## Summary

Keep this PR open. Current main still lacks the Feishu-specific `match.accountId` routing clarification, and the PR is not implemented or superseded. The proposed direction is useful, but the wording should distinguish the routing default bucket from `channels.feishu.defaultAccount`, which the Feishu docs describe as outbound API selection.

## Best Possible Solution

Keep this PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain that omitted `match.accountId` maps to the default routing bucket rather than all accounts, explicit ids match concrete Feishu runtime account ids, and `accountId: "*"` matches any configured Feishu account. Also distinguish this from `channels.feishu.defaultAccount`, which controls outbound/default API account selection, and coordinate terminology with #49769 while preserving a Feishu-specific example in `docs/channels/feishu.md`.
