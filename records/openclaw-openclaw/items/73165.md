## Summary

Keep this PR open. Current main does not implement the proposed MCP readiness gate for advertised skills, and the latest PR patch is still a live implementation candidate rather than obsolete work.

## Best Possible Solution

Continue review rather than close. If maintainers want this core safety behavior, keep the readiness gate but require tests for skill-level denial precedence, owner scoping, server fallback, missing readiness, and stale-readiness policy. The smoke script should remain opt-in and documented, and unrelated package-boundary/test-alias changes should be justified or split out before merge.

## Risks / Open Questions

- Merging the current readiness logic could advertise a skill explicitly marked `callable: false` if the associated server record is `callable: true`.
- The new smoke script is an opt-in command-execution surface that spawns `mcporter` from PATH and should be reviewed before it is wired into automation.
- The package-boundary, vitest alias, cache-trace, and unrelated test edits broaden the PR scope beyond the MCP readiness gate and should be explained, split, or rebased away.
