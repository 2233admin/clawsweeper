## Summary

Keep this PR open. It is a focused implementation candidate for restoring `openai-codex/gpt-5.3-codex-spark` and `openai-codex/gpt-5.4-mini`; current `main` still suppresses those Codex OAuth routes, the runtime does not resolve them, docs/tests still describe them as unsupported, and the shared manifest suppression planner still lacks the explicit-row precedence the PR adds.

## Best Possible Solution

Keep this PR open for normal maintainer review. If live validation holds, the best path is to land a focused change that restores explicit `openai-codex` catalog/runtime support for `gpt-5.3-codex-spark` and `gpt-5.4-mini`, preserves direct `openai/gpt-5.3-codex-spark` suppression until that route is verified, documents the route distinction, and covers the manifest suppression precedence with regression tests.
