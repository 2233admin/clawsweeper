## Summary

Keep this draft PR open. Current main still explicitly suppresses the Codex OAuth routes the PR restores, and current tests/docs assert they are unsupported. The PR is focused on the implicated OpenAI plugin/model-catalog paths and is not obsolete on main.

## Best Possible Solution

Keep this PR open for normal maintainer review. If live validation holds, the best path is a focused implementation that restores explicit openai-codex catalog rows for gpt-5.3-codex-spark and gpt-5.4-mini, lets same-manifest explicit rows override stale suppressions, keeps direct OpenAI API Spark suppressed until verified, and updates the associated docs and regression tests.
