## Summary

Current main still matches the reported configured ACP session-key collision path for inherited Discord thread routing, and the same-author fix PR #64322 remains the right follow-up path rather than closing this issue now.

## What This Changes

The issue asks for Discord threads inheriting a configured parent-channel ACP binding to receive distinct persistent ACP session keys instead of sharing the parent channel's ACP process.

## Best Possible Solution

Keep this open until #64322 or an equivalent fix lands. The end state should preserve parent-channel routing inheritance while materializing inherited Discord thread ACP bindings with thread-scoped persistent session keys, plus regression coverage for two thread ids under one parent producing different targetSessionKey values.
