## Summary

Keep this PR open. Current main still has the exact mismatch the PR targets: the `nodes` tool schema makes `node` optional, but the `describe` branch still requires `node` and calls the resolver without the default-node fallback. The PR is focused and paired with open issue #51903, but it still needs a small regression test for `action: "describe"` with no `node` before it is ready to land cleanly.

## Best Possible Solution

Keep the PR open, add a focused test in `src/agents/tools/nodes-tool.test.ts` for `{ action: "describe" }` with no `node` that asserts `resolveNodeId` is called with `allowDefault=true` and `node.describe` receives the resolved node id, then review/land this PR to close the paired open issue #51903.
