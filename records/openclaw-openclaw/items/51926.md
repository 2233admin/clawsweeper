## Summary

Keep this PR open. Current main still has the exact schema/implementation mismatch the PR targets: the `nodes` tool schema accepts omitted `node`, but the `describe` action still hard-requires `node` and calls `resolveNodeId` without enabling the existing default-node fallback. The PR is a focused implementation candidate for open issue #51903, but it should gain a regression test before landing.

## Best Possible Solution

Keep this PR open, add a targeted regression test for `tool.execute("call-1", { action: "describe" })` that asserts `resolveNodeId` is called with `undefined` and `allowDefault=true` and that `node.describe` receives the resolved node id, then review and land the focused PR to close #51903.
