## Summary

Keep this PR open. Current main still does not implement the Slack DM message-tool threading fix: DM context keeps only the native Slack `D...` channel id, the auto-thread resolver still rejects `user:` targets, and current tests explicitly encode that rejection. The PR remains a plausible implementation candidate, but it needs a clean rebase and should drop or split unrelated dependency/CI changes before maintainer review.

## Best Possible Solution

Keep this PR open, but require a clean, focused rebase: preserve Slack DM `currentChannelId` as the native `D...` id for inferred Slack actions, add an additive generic DM/current-recipient target field or equivalent SDK-safe threading context, make `resolveSlackAutoThreadId` match `user:` sends against that field, and add focused regression tests for Slack DM message-tool media/attachment sends. Drop or split the unrelated `@pierre/diffs`, lockfile, and platform-test changes.
