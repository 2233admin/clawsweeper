## Summary

Keep this PR open. Current main at 7a23b2d945cc2d2e5c452e4641213e71de8dcc66 still does not implement the Slack DM message-tool threading fix: DM tool context keeps only the native Slack D... channel id, the Slack auto-thread resolver still rejects user: targets, and current tests assert that rejection. The PR remains a plausible implementation candidate, but it needs a focused rebase onto the current extensions/slack layout and should split the unrelated dependency/CI changes before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a focused rebase: preserve Slack DM currentChannelId as the native D... id for inferred Slack actions, add a generic current-recipient/current-target field or equivalent SDK-safe threading context, make resolveSlackAutoThreadId match user: sends against that field, and add focused regression tests for Slack DM message-tool media/attachment sends. Split the @pierre/diffs patch, lockfile changes, and unrelated platform-test guards into a separate maintainer-reviewed PR if they are still needed.
