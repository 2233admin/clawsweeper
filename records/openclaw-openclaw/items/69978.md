## Repair Work Prompt

Implement a focused TUI watchdog control fix for this issue. Current main still has only the internal `streamingWatchdogMs` option, no `/watchdog` command, no persisted TUI preference, and no per-run warning dedupe.

Scope:
- Add same-run duplicate suppression in `src/tui/tui-event-handlers.ts`: once the streaming watchdog warning is emitted for a run id, do not emit it again for that run. Clear the warned set on `finalizeRun`, `terminateRun`, and session-key changes. Preserve the #69081 behavior that tool/lifecycle proof-of-life rearms the watchdog and reconnect recovery reloads history without warning.
- Add a local `/watchdog` TUI command with status, `off`/`0`, and duration arguments such as `30s`, `60s`, `120s`, and `2m`. Validate duration suffixes, reject too-small nonzero values below 5s, and provide completions.
- Persist the TUI-local preference under the resolved OpenClaw state directory, preferably in a small TUI settings helper/module rather than global config schema. Do not add a new global config key unless maintainers explicitly choose to fold this into #68596.
- Wire the active timeout preference into `createEventHandlers` so changes affect the current TUI session without restart where feasible.
- Update TUI help/docs and add a single-line changelog entry with `Thanks @JIRBOY` and any other reporter credit if the fix intentionally covers #68596 or #69026.
- Treat #69026 as related partial source work, not as the desired final behavior; do not remove all watchdog warning visibility unless the command/config explicitly disables it.

Do not change unrelated Gateway, provider, or global timeout semantics.
