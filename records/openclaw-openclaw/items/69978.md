## Summary

Keep #69978 open. Current main still has the TUI streaming watchdog as an internal 30s handler option, no `/watchdog` command or persistence path, and no per-run duplicate-warning suppression. Related #68596 and #69081 overlap the timeout/status area, but neither fully supersedes this issue's combined runtime-control and warning-dedupe request.

## Best Possible Solution

Keep the issue open unless maintainers explicitly consolidate it into a canonical TUI watchdog follow-up. The best implementation path is in `src/tui`: add per-run watchdog warning suppression cleared on finalize/terminate/session switch, expose a supported timeout/disable control through `/watchdog` or the broader #68596 config surface, persist the preference, and add focused command, handler, docs, and watchdog regression tests.
