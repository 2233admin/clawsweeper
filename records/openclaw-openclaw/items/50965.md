## Summary

Keep this PR open. Current main still has the cron-remove false-success path, and the PR remains a focused but incomplete implementation candidate because it changes only the service layer while the CLI remove aliases still do not fail the command when removal did not happen.

## Best Possible Solution

Keep this PR open for author or maintainer follow-up, or replace it with a complete fix: make cron service removal return a typed not-found failure without persisting or rearming on a no-op, update the result contract and relevant consumers, make the remove aliases exit non-zero when `ok` or `removed` is false, add focused service plus CLI regression tests, and deliberately decide whether name-based removal should be supported like `cron show`.
