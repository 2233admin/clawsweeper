## Summary

Keep this PR open. Current main still has the cron remove silent-success path, and the proposed PR is a plausible service-layer fix but is incomplete because it does not update the CLI exit behavior that reviewers already flagged.

## Best Possible Solution

Update the PR or replace it with a complete fix on main: make cron service removal return a real not-found failure without persisting or rearming on a no-op, propagate that through the typed result, make cron rm/remove/delete exit non-zero when ok/removed is false, and add service plus CLI regression tests for the missing-job path.
