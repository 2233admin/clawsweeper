## Summary

Keep this PR open. Current main still does not implement the PR’s intended startup dangerous-flag warnings for `gateway.auth.mode="none"` or trusted-proxy auth with empty/absent `allowUsers`, and the config schema still accepts whitespace-only `trustedProxy.userHeader`. The PR remains a focused gateway hardening candidate, but it needs to be rebased onto the current dangerous-flag collector shape and covered with targeted tests.

## Best Possible Solution

Keep this PR open and review it as a focused gateway hardening candidate. Rebase it onto current main, add the auth-mode and trusted-proxy allowlist detections to the current dangerous-flag collector while preserving `undefined` and `[]` semantics, decide whether schema-level whitespace-only `userHeader` rejection belongs in addition to the runtime guard, and add targeted tests for `src/security/dangerous-config-flags.test.ts`, `src/gateway/server-startup-log.test.ts`, and config schema validation as appropriate.
