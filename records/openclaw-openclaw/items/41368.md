## What This Changes

The PR modifies `extensions/feishu/src/bitable.ts` to add Feishu Bitable single-record deletion, batch-record deletion, and field deletion helpers, schemas, and tool registrations.

## Best Possible Solution

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Restore token lookup hints on delete schemas:** `extensions/feishu/src/bitable.ts:589-613`
  - body: The new delete schemas use bare `app_token` and `table_id` descriptions, unlike existing Bitable schemas that point the model to `feishu_bitable_get_meta` and the URL table parameter. Because these tools delete data, losing that guidance increases the chance of targeting the wrong table.
  - confidence: 0.83
- **[P3] Return the batch delete response records:** `extensions/feishu/src/bitable.ts:474-476`
  - body: `batchDeleteRecords` reports only the input count and drops `res.data?.records`. The SDK returns per-record delete results, so callers lose API-confirmed IDs and get weaker confirmation than the create/update helpers provide.
  - confidence: 0.78
