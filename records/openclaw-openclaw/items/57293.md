## Summary

Keep this PR open. Current main still has the reported runtime schema mismatch: `approvalRunningNoticeMs` is documented and typed for global `tools.exec`, but the strict `ToolExecSchema` is built from `ToolExecBaseShape`, which still omits the field and only adds it to `AgentToolExecSchema`. The PR should not be closed as implemented. It also mixes in systemd `EnvironmentFile=` work; that part appears separately addressed on main with a safer implementation and regression coverage, so the config fix should be rebased or split before landing.

## Best Possible Solution

Keep the PR open, but rework it into the focused config fix: move `approvalRunningNoticeMs` into `ToolExecBaseShape`, regenerate the base schema if needed, and add a runtime `ToolsSchema` regression test for the global `tools.exec` path. Drop or separately review the systemd changes because main already has a safer `EnvironmentFile=` implementation with coverage.
