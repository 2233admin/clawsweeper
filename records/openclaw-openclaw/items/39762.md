## Summary

Keep #39762 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still does not implement Feishu read-only group mode: the config/docs surface is absent, accepted group messages still create real reply dispatchers in both broadcast and single-agent paths, and outbound Feishu target resolution has no group-reply guard. The PR remains actionable because the author addressed earlier broadcast and whitespace feedback in the discussion, while the later raw `oc_*` chat-id bypass and unrelated `.secrets.baseline` churn still need maintainer review before merge.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to Feishu config/schema/docs, enforce a no-op dispatcher for accepted group messages in both broadcast active-agent and single-agent paths, block explicit and bare Feishu `chat_id` outbound targets when enabled, add targeted regression tests for those paths, and remove or separately justify the `.secrets.baseline` churn before merge.
