## Summary

Current main still drops wake target fields for the agent-facing `cron` tool and Gateway `wake` method. The lower cron Gateway adapter can route `agentId`/`sessionKey` when it receives them, but the tool, protocol handler, and `CronService.wake` path still pass only `mode` and `text`, so the report remains valid.

## Best Possible Solution

Keep this issue open. The best path is to review or expand PR #70268 so it covers both the hook wake mapping case and the agent-facing cron wake action, or otherwise land a focused PR that threads optional `agentId` and `sessionKey` through the cron tool schema, Gateway wake protocol handler, `CronService.wake`, and existing Gateway cron routing adapter with regression tests.
