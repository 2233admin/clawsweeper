## Summary

Keep open. Current main does not fully implement the branch’s Windows test/runtime alignment, and the branch still has unresolved blocking review findings in command/path resolution; this needs maintainer review rather than cleanup closure.

## What This Changes

The branch updates Windows scheduled-task and path-related tests while changing executable lookup, home-prefix expansion, and stable Homebrew Node path resolution helpers.

## Best Possible Solution

Refresh or replace the branch with a focused patch that keeps the useful Windows scheduled-task test alignment, preserves unmatched tilde tokens, uses PATHEXT only for extension lookup rather than rejecting already-qualified files, and adds targeted regression coverage for both command-resolution cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unmatched tilde tokens:** `src/infra/home-dir.ts:73`
  - body: Wrapping the replacement in `path.resolve(...)` also resolves inputs the regex intentionally leaves untouched. A value like `~alice/bin/tool` becomes a cwd-anchored path, and callers use this helper for executable and allowlist paths, so an unresolved token can silently target a different filesystem path.
  - confidence: 0.92
- **[P2] Skip PATHEXT checks for qualified files:** `src/infra/executable-path.ts:41`
  - body: Passing the caller env into `isExecutableFile` makes Windows reject an already-found file when its extension is not in `PATHEXT`. With `PATHEXT=.CMD`, a concrete `node.exe` path can return undefined even though the executable exists; PATHEXT should drive extension search, not invalidate qualified paths.
  - confidence: 0.88
- **[P3] Normalize formula casing before the fallback:** `src/infra/stable-node-path.ts:13`
  - body: The Cellar matcher is now case-insensitive, but the default formula fallback still checks `formula === "node"`. A path that only matched because of the `i` flag, such as a `Cellar/Node/.../node.exe` path, skips the `<prefix>/bin/node.exe` fallback.
  - confidence: 0.62

## Security Review

Status: needs_attention

Summary: No workflow, lockfile, dependency, publishing, or release-script changes were found, but the runtime exec path changes affect approval and executable-resolution semantics.

Concerns:

- **[low] Home-prefix expansion can retarget approval paths:** `src/infra/home-dir.ts:73`
  - body: The proposed `path.resolve(...)` wrapper can turn an unmatched `~user/...` token into a cwd-relative absolute path. Because this helper feeds exec allowlist and command-resolution paths, that can change what an approved pattern or command target refers to.
  - confidence: 0.82
