## Summary

Keep open. Current main still has the reported Control UI streaming layout path: tool-boundary text is stored as stream segments, `buildChatItems` emits segment text and the live tail as separate stream items, and every stream item renders through a full assistant group. The linked layout fix PR #63986 was closed unmerged, and the current tests do not exercise segmented live streaming through the real grouped DOM path.

## What This Changes

The issue asks for Control UI streaming assistant responses to stay in one in-flight assistant bubble/group instead of temporarily rendering multiple assistant bubbles before final history collapse.

## Best Possible Solution

Represent one active assistant turn as a single outer assistant group in the Control UI, with committed stream segments, visible tool cards, reading indicator, and live tail rendered inside that group. When tool calls are hidden, merge segment text and live text into one updating streaming bubble, and add regression coverage that asserts segmented live streaming produces exactly one outer `.chat-group.assistant`.
