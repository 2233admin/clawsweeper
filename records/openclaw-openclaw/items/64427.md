## Summary

Keep open. Current main still documents and accepts Slack `dmHistoryLimit`, but the Slack monitor only plumbs `historyLimit` and still gates history injection on `isRoomish`, which excludes 1:1 Slack DMs. Related #67020 tracks the same remaining gap, but this issue has the older and more complete reproduction, so it should remain the canonical tracker unless maintainers choose otherwise.

## What This Changes

Request that the Slack plugin honor `channels.slack.dmHistoryLimit` by hydrating new 1:1 DM sessions with prior Slack DM conversation history.

## Best Possible Solution

Implement this inside `extensions/slack`: resolve an effective DM history limit from `channels.slack.dms.<user>.historyLimit` before `channels.slack.dmHistoryLimit`, fetch recent `conversations.history` for fresh top-level `channel_type: "im"` sessions, exclude the current inbound event, include prior user and bot messages as Slack DM context, and feed both prompt body context and `InboundHistory` consistently with existing Slack history handling. Keep broader cross-session channel visibility, such as #7359, out of this fix unless maintainers explicitly broaden the scope.
