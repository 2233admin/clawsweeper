## Summary

Keep open. Current main still exposes Slack `dmHistoryLimit` as a public config surface, but the Slack monitor only derives and carries `historyLimit`, and message preparation only injects pending history when `isRoomish` is true, which excludes `channel_type: "im"` 1:1 DMs.

## What This Changes

The issue requests that Slack 1:1 DM sessions honor `dmHistoryLimit` by hydrating recent DM history when a fresh session starts.

## Best Possible Solution

Implement the fix inside `extensions/slack`: compute an effective DM history limit from `channels.slack.dms.<user>.historyLimit` before `channels.slack.dmHistoryLimit`, hydrate recent `conversations.history` only for fresh top-level `channel_type: "im"` sessions, format the prior messages through the existing inbound envelope/history contract, and add focused Slack monitor tests. Keep the broader cross-session/channel visibility work tracked by #7359 separate.
