## Summary

Keep open. Current main still has adjacent managed/click download capture, but direct browser `/navigate` remains URL-only, rethrows non-policy `page.goto()` errors, and has no CDP download protocol hook or regression coverage for Content-Disposition attachment navigation. The linked fix PR #48132 is closed unmerged, while #64558 shipped adjacent click-triggered/managed-profile download behavior rather than this direct navigation path.

## What This Changes

The issue asks the browser plugin to treat CDP/Playwright navigation-triggered downloads as successful, save them to the configured download directory, and return structured download metadata instead of surfacing the Playwright download-starting error.

## Best Possible Solution

Extend the browser plugin's Playwright/CDP navigation flow so a direct attachment navigation is captured as a managed download, returns structured download metadata from `/navigate`, and preserves the existing navigation policy, output-root, and explicit-waiter protections.
