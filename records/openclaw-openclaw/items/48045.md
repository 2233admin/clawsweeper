## Summary

Keep open. Current main handles explicit browser download waits and unmanaged click-triggered downloads, but the reported direct `/navigate` case still goes through `page.goto()` and rethrows Playwright's `Download is starting` navigation abort instead of returning `download.triggered`. Prior PRs addressed parts of this space, but the exact navigation-download behavior remains unmerged and needs a security-aware fix.

## Best Possible Solution

Keep this issue open and implement a browser-plugin fix for download-triggered navigation: detect Playwright's download navigation abort, return a structured success marker from `/navigate`, preserve all initial/redirect/final navigation policy checks, and add regression coverage for both allowed attachment downloads and blocked redirect-to-attachment cases.
