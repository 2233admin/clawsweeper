## Summary

Keep open. Current main still routes Feishu `user:ou_...` targets as `open_id`, but the Feishu send and `feishu_chat` paths still lose Lark SDK Axios response bodies, leaving HTTP 400 failures as generic errors. The reporter's follow-up shows native contact lookup succeeds while `im.message.create` returns vendor `code: 9499`, and a v2026.4.27 commenter reports Feishu still failing, so this remains an active Feishu plugin bug/diagnostic gap.

## What This Changes

The issue asks for Feishu direct user targets via the shared `message` tool to send successfully instead of returning HTTP 400.

## Best Possible Solution

Preserve Feishu vendor error bodies for message sends and `feishu_chat` lookups first, then use the exposed `code`/`msg`/`log_id` to decide whether the DM failure needs a send payload fix, a bot setup/permission docs update, or a separate package/runtime fix for the v2026.4.27 dispatch error.
