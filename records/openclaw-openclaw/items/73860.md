## Summary

Keep open. The added Feishu response body changes the shape of the issue: current main already routes `user:ou_...` as `open_id`, but it still lets Lark/Axios HTTP 400 errors surface as generic `Request failed with status code 400` messages, so the useful Feishu `code`/`msg`/`log_id` can be lost. The actual `im.message.create` code 9499 cause may still be Feishu scopes, tenant visibility, or bot-DM policy, so closing as implemented would be premature.

## Best Possible Solution

Keep this issue open. First land a narrow Feishu-plugin diagnostics fix that unwraps Lark/Axios HTTP error bodies for `message` sends and `feishu_chat`, with regression tests for code/msg/log_id preservation. Then use the exposed Feishu 9499 log id to decide whether the remaining direct-message failure needs docs for Feishu scopes/policy or a separate target-routing fix.
