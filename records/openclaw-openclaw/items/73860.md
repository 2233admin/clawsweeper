## Summary

Keep open. The report is current against v2026.4.26 and current main still has the same Feishu user-target send path: `target=user:ou_...` is normalized to `receive_id_type: "open_id"`, then sent through Feishu `im.message.create`. I found no post-report change to the Feishu send/target/chat files, and existing tests do not prove the exact `message` action shape succeeds against Feishu user DMs. The remaining work needs the Feishu/Lark response body or log_id to distinguish a code bug from app scopes, tenant visibility, or bot-DM policy.

## Best Possible Solution

Keep open and investigate in the Feishu plugin. Capture the full Feishu/Lark response body for both `im.message.create` and `contact.user.get`, add a regression test for `message` action send with `target=user:ou_...`, then either fix a target/ID-type mismatch, document required app scopes and tenant constraints, or improve Feishu SDK/Axios error unwrapping so users see the actionable vendor error.
