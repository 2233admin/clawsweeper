## Summary

Keep open. Current main still has the reported Signal daemon restart race, and PR #71863 is an open, unmerged fix that explicitly closes this issue, so the issue should remain open until that PR or an equivalent Signal-owned fix lands.

## What This Changes

Make Signal daemon shutdown asynchronous during config/SIGUSR1 restarts so the old signal-cli process exits, or is SIGKILLed after a bounded timeout, before a replacement daemon starts.

## Best Possible Solution

Keep this issue open and review/land PR #71863, or land an equivalent focused Signal fix that makes daemon stop async/idempotent, waits for signal-cli exit after SIGTERM, escalates to SIGKILL after a bounded timeout, and awaits that shutdown through monitor teardown before channel restart.
