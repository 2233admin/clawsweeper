## Summary

Keep this PR open. Current main still lacks the PR's intended daily/idle lazy-reset hook dispatch: automatic rollovers only fire session lifecycle hooks, the manual reset hook path remains command-gated, and the bundled session-memory hook still filters to `new`/`reset` only. The PR is an active, maintainer-repaired candidate for the missing behavior, not obsolete work.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for lazy daily/idle session reset hooks. After it lands, use it to close or supersede the older overlapping #50933 and the related daily/idle session-memory reports it covers.
