The PR discussion also has recent maintainer context preserving this branch as the canonical contributor path, so the right cleanup is to review/land it or explicitly supersede it, not close it.

Keep this PR open and review/land it as the canonical fix, or explicitly supersede it with a narrower equivalent PR.
