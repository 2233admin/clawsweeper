## Summary

Keep this PR open. Current main does not implement the intended lazy daily/idle reset hook dispatch: stale rollovers archive the old session and fire session lifecycle hooks, but internal command hooks and `before_reset` remain tied to manual `/new`/`/reset`, and `session-memory` still filters/documents only `command:new` and `command:reset`. The discussion also has recent maintainer context preserving this branch as the canonical contributor path, so the right cleanup is review/land it or explicitly supersede it, not close it.

## Best Possible Solution

Keep this PR open and review/land it as the canonical fix, or explicitly supersede it with a narrower equivalent PR. The implementation should make lazy daily/idle rollovers fire internal `command:daily`/`command:idle` events and `before_reset` without blocking the triggering reply, update `session-memory` docs/metadata/filtering, and keep regression tests/changelog aligned. After merge, close or supersede the overlapping automatic-reset memory reports and resolve #50933/#71792 against the chosen canonical path.
