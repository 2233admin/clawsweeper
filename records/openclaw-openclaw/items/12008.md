## Summary

Keep open. Current main still lacks a typed/schema-validated Gemini safety-settings config surface, and the direct Google Generative AI transport still builds request bodies without `safetySettings`. PR #49529 remains the active implementation candidate for this capability.

## Best Possible Solution

Keep this issue open until PR #49529 or an equivalent maintainer-reviewed implementation lands. The concrete path is a typed and schema-validated `models.providers.*.safetySettings` config surface, generated schema/help/docs updates, runtime mapping only for `google-generative-ai` requests, explicit per-request precedence, and focused regression tests for config validation and payload injection.
