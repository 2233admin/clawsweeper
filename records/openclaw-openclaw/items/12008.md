## Summary

Keep open. Current main still lacks a typed, schema-validated, documented Gemini safety-settings configuration surface, and the direct Google Generative AI transport still builds requests without `safetySettings`; the open related PR remains an implementation candidate rather than a merged fix.

## What This Changes

The issue requests provider- or model-level Gemini safety-settings configuration that maps to Gemini API safety settings for harassment, hate, sexually explicit, and dangerous-content thresholds.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs coverage, Google-only request injection, explicit payload precedence, and targeted config/runtime regression tests.
