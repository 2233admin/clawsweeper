## Summary

Keep open for maintainer review, but this PR is not mergeable as-is: the current diff has CI-confirmed syntax and lint failures, and the gateway-only guard still misses the service-side atomic update path described in the linked cron bug.

## What This Changes

The PR adds a cron expression validation helper, calls it from the `cron.add` and `cron.update` gateway handlers, and adds Croner validation tests.

## Best Possible Solution

Fix this at the cron service boundary by staging the post-patch job, validating the candidate schedule before mutating stored state, and preserving repair paths for already-corrupt jobs; keep gateway handling focused on request normalization and error shaping.
