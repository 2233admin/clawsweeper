## Summary

Keep open. Current main still has the reported gap: `message_sending` runs through the typed plugin hook runner, but there is no internal/user HOOK.md pre-send dispatch. An open unmerged PR, #73570, explicitly references this issue with closing syntax, so the best maintainer path is to review and amend that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review/amend #73570. The fix should add a tested internal pre-send hook bridge in the shared outbound delivery path and the inbound/webchat before-deliver path, define whether user hooks listen on `message:sending`, `message_sending`, or both, document whether the internal hook is observe-only or can mutate/cancel, and add regression coverage for normal, rewritten, and cancelled outbound messages.
