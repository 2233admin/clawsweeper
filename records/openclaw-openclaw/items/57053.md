## Summary

Keep this issue open. Current main still exposes `session_status` as a `/status`-style text card with `details.statusText`; it does not provide structured fields such as `remaining_seconds`, `remaining_tokens`, `elapsed_seconds`, or `session_timeout_seconds`. The related open #57705 covers sub-agent concurrency capacity, which is adjacent but not a replacement for this session time/token budget request.

## Best Possible Solution

Keep this issue open and implement a typed budget contract in the session tool surface: either add a `budget` object to `session_status.details` or introduce `session_budget`. Define exact semantics for elapsed/remaining wall-clock time and token headroom, add focused tests in `src/agents/openclaw-tools.session-status.test.ts`, and document the contract in `docs/concepts/session-tool.md`. Coordinate with #57705 only for any separate sub-agent capacity fields.
