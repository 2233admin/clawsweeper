## Summary

Keep this PR open. Current main still does not implement the requested 0o644 mode for user-facing Write/Edit outputs, and the PR is not obsolete. The existing PR also remains incomplete because sandboxed Write/Edit still has no mode propagation path and the sandbox mutation helper still creates write temp files with 0o600.

## Best Possible Solution

Keep the PR open and request a focused revision: add a nullable-safe mode option through writeFileWithinRoot, the legacy path, and the pinned helper; pass 0o644 from both host and sandboxed user-facing Write/Edit operations; extend SandboxFsBridge/buildPinnedWritePlan/the mutation helper to carry that mode; keep all internal defaults at 0o600; coordinate with #44077; and add targeted permission regression tests for host Write/Edit, sandbox Write/Edit, explicit mode: 0, and private internal defaults.
