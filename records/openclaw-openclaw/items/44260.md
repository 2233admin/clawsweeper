## Summary

Keep this PR open. Current main still does not implement 0o644 for user-facing Write/Edit outputs, and the PR branch is useful but incomplete because it only updates host workspace Write/Edit call sites while sandboxed Write/Edit still writes through private 0o600 sandbox mutation paths. The related sandbox permission regression in #44077 remains open and should be handled with the same focused mode-plumbing fix.

## What This Changes

The PR branch adds optional file-mode plumbing to the safe write path, passes 0o644 from host workspace Write/Edit calls, and changes the pinned helper mode fallback to nullish coalescing.

## Best Possible Solution

Revise the PR so both host workspace and sandboxed user-facing Write/Edit can create new regular files with 0o644, while all internal/sensitive writes continue to default to 0o600 and existing-file mode preservation stays intact. The fix should include nullable-safe mode plumbing through the safe write and sandbox bridge layers plus focused permission regression tests tied to #44077.
