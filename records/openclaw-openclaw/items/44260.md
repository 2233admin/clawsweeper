## Summary

Keep this PR open. Current main still lacks the requested 0o644 user-facing Write/Edit mode path, and the PR is not obsolete. The current PR direction is plausible, but it still needs sandbox Write/Edit mode propagation and regression tests before it should land.

## Best Possible Solution

Keep the PR open and request a focused revision: add a mode option through writeFileWithinRoot and the sandbox filesystem bridge/buildPinnedWritePlan path, pass 0o644 only from user-facing host and sandbox Write/Edit new-file creation, preserve internal 0o600 defaults and explicit mode: 0 with nullish checks, and add targeted permission tests for host Write/Edit, sandbox Write/Edit, and private/default write paths.

## Risks / Open Questions

- Merging the PR as-is would likely leave sandboxed Write/Edit sessions creating owner-only files, so the Syncthing/shared-mount readability problem could still reproduce when sandbox mode is active.
- The broader 0644 mode must stay limited to user-facing Write/Edit document writes; internal state, auth, session, and secret/config writes should continue to default or explicitly write as 0600.
- Without permission regression tests for host and sandbox Write/Edit, explicit mode: 0, and sensitive default-private paths, future changes could silently broaden or regress file modes.
