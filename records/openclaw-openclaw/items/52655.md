## Summary

Keep this PR open. Current main does not implement the requested per-agent queue lanes: `agents.list[]` has no `lane` or `laneConcurrency` config, embedded runs without an explicit lane still fall back to the process-wide `main` lane, and gateway startup only configures the built-in lanes. The PR is still open and unmerged, and its diff adds the missing config, resolvers, lane application, and call-site wiring, including the follow-up runner/memory runner scope discussed in #55566.

## Best Possible Solution

Keep the PR open. The best path is to rebase it onto current main, remove the ignored full generated baseline artifacts, update only the expected generated schema/hash/doc surfaces, then review whether per-agent lanes should land as the product answer for #16055 and the narrower follow-up behavior discussed in #55566.
