## Summary

Keep this PR open. Current main still has the reported main-session cron duplication path: the cron timer enqueues the raw systemEvent with a plain cron context key, heartbeat prompt construction peeks that queue and wraps cron-tagged events in the scheduled-reminder prompt, and the reply path can still drain the same event into System lines. The PR is a focused implementation candidate for the open linked bug, not obsolete or superseded.

## Best Possible Solution

Review and land this PR, or apply an equivalent focused fix on main that prevents main-session cron `payload.kind="systemEvent"` jobs from being wrapped a second time by heartbeat prompt construction while preserving the existing wrapper for cron events that are not already injected into the active main-session transcript. Keep regression coverage for `sessionTarget="main"`, `wakeMode="now"`, and internal/no-delivery cron behavior.
