## Summary

Keep this issue open. Current main has adjacent safeguards for config validation, update service stopping, and the cited lossless-claw regression, but it still lacks the requested standalone no-bind Gateway preflight command and shared startup validation path that exercises plugin/context-engine resolution before serving traffic.

## What This Changes

The issue requests `openclaw gateway preflight [--json]` to validate Gateway config, plugin/context-engine startup, and port availability without binding the listener or touching session state.

## Best Possible Solution

Add `openclaw gateway preflight [--json]` as a first-class Gateway CLI command backed by a shared no-bind startup validation routine. The end state should parse and validate config, resolve active SecretRefs/auth, load the same startup plugin and context-engine registration paths in a no-mutation mode, probe configured port bindability without accepting connections, return stable human/JSON results with exit code 0/1, and document the command on the public Gateway CLI page before composing it into a later safe-update flow.
