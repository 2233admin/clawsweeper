## Summary

Keep this issue open. Current main still runs the reported pi-tools workspace-paths and safe-bins tests under WSL, and the focused WSL guard/cleanup hardening described in related closed PRs is not present in the checked-out source.

## Best Possible Solution

Keep this issue open until a focused test-only fix lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The best path is to harden temp-dir creation/cleanup, use src/infra/wsl.ts helpers for any narrowly justified WSL guard, and verify the affected tests no longer hang or race under WSL.
