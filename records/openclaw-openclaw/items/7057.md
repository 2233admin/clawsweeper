## Summary

Keep this issue open. Current main still leaves the reported WSL-sensitive pi-tools workspace-paths and safe-bins tests active without WSL guards or hardened temp cleanup, and the two related implementation PRs in the provided timeline are closed unmerged.

## Best Possible Solution

Keep this issue open until a focused test-only patch lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The best path is to harden temp-dir creation/cleanup, use the existing src/infra/wsl.ts helpers for any narrowly justified WSL guard, and verify the affected tests under WSL or an appropriate Windows/WSL lane without reviving unrelated changes from #7085.
