Keep this PR open. ## What This Changes

The PR adds `openclaw sessions rm <key>` and `openclaw sessions clear` CLI registration, a new sessions deletion command module with store/agent/all-agents/dry-run/JSON options, orphan transcript removal, and command-registration/behavior tests.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased successor. The best implementation is a focused `openclaw sessions rm <key>` and `openclaw sessions clear` CLI that deletes only the requested sessions, avoids unrelated maintenance pruning during targeted deletion, reuses existing archive/trajectory/runtime cleanup contracts where appropriate, returns stable JSON, documents the commands at https://docs.openclaw.ai/cli/sessions, and adds targeted registration plus command behavior tests.
