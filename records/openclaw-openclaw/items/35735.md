## Summary

Keep open. Current main still lacks the proposed `openclaw sessions rm` and `openclaw sessions clear` CLI surface; the existing Gateway `sessions.delete` RPC is related runtime plumbing but does not provide the offline/store-targeted single and bulk deletion workflow. The PR remains useful, but the discussion has unresolved behavior and output-shape review comments that need maintainer review or a rebased successor before merge.

## What This Changes

This PR adds CLI registration/tests and a new sessions deletion command module for `openclaw sessions rm <key>` and `openclaw sessions clear`, with store/agent/all-agents/dry-run/JSON options and orphan transcript cleanup.

## Best Possible Solution

Ship a focused sessions deletion CLI that deletes only matched sessions, preserves store consistency, reuses the Gateway/archive/runtime cleanup contracts where appropriate, returns stable JSON, documents the new commands at https://docs.openclaw.ai/cli/sessions, and includes targeted registration plus command behavior tests.
