## Summary

Keep open. Current main has packaged docs, local docs path discovery, prompt guidance, and config schema lookup, but it still lacks the requested native, version-matched, local-first docs retrieval layer for agents.

## What This Changes

Request to bundle version-matched OpenClaw docs with releases and expose local-first docs search/get/schema/API retrieval to agents for onboarding.

## Best Possible Solution

Define a release-packaged docs manifest/search index, expose local-first docs search/get/schema/API lookup through a maintained agent-facing surface, make live Mintlify fallback explicit, and cover the installed-version behavior with protocol/tool/docs tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime reproduction because this is a feature request; source inspection gives high-confidence proof that the current implementation has path discovery and config lookup but no native local-first docs retrieval layer.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the bundle/index contract and agent-facing API; the local-first retrieval direction is sound, but prompt guidance plus live CLI search is not the requested maintainable solution.
