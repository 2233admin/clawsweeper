## Summary

Keep open. Current main still ships loop detection disabled by default and still lacks read-specific out-of-range offset recovery, so the PR is not obsolete. It should not merge as-is because the branch mixes two behavior changes, carries stale/generated local-only docs artifacts, and the read-offset wrapper can discard already-read adaptive pages; overlapping open loop-default PRs need maintainer selection.

## What This Changes

The PR enables tool-call loop detection by default, updates related tests/docs/config metadata and generated baselines, and wraps out-of-range read offsets into diagnostic tool results.

## Best Possible Solution

Land two focused, rebased changes: one canonical loop-detection default decision that preserves current run-id scoping and aligns tests, schema help, generated checksums, and docs; and a separate read-offset recovery fix inside adaptive paging that returns a diagnostic without losing partial output.
