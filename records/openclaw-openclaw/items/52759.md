## Summary

Keep this PR open. Current main still leaves tool-loop detection disabled by default and the latest release does too, so the main behavior change is not implemented. The linked read-offset issue appears partly obsolete because main now converts thrown tool errors into model-visible diagnostic results, but this PR still needs maintainer follow-up: rebase onto current run-scoped loop detection, coordinate with overlapping #70877, and likely split or drop the read-offset patch.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but ask for a rebase and likely split. The loop-detection-default change should preserve current main's run-id scoping, update tests/docs/schema together, and be reviewed alongside #70877 and other overlapping default-on proposals. The read-offset portion should either be dropped as covered by the generic diagnostic adapter for #52680 or reworked as a focused read-tool enhancement that preserves partial adaptive output and has dedicated tests.
