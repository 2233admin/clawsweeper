## Summary

Keep this issue open. Current main still renders chat `/status` and `session_status` through the shared status-card text builder with no `session.statusCard`, `agents.defaults.statusCard`, hide flag, section filter, or JSON-only mode for those chat/tool surfaces. The CLI `openclaw status --json` path is separate, and #69445 was already closed as superseded by this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open until OpenClaw adds or rejects an explicit, documented status-card visibility control for chat `/status` and `session_status`. A complete fix should preserve current rendered-card behavior by default, choose one public config shape, update schema/types/help/docs, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for chat-command and agent-tool paths while keeping CLI `openclaw status --json` separate.
