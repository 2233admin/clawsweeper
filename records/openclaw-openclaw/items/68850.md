## Summary

Keep open. Current main still has the noisy displayName path, so implemented-on-main is not justified; however the patch applies synthetic-name filtering to explicit session labels as well as displayName, which conflicts with the session label contract. The existing Codex review comment already identifies that regression, and #51124 is related but broader/open rather than a canonical replacement.

## What This Changes

The branch adds a synthetic transport-name detector and tests so the Control UI session selector ignores `webchat:g-agent-...` names and uses main/subagent fallback labels.

## Best Possible Solution

Apply the visual cleanup in the current `ui/src/ui/chat/session-controls.ts` path by suppressing only synthetic transport-derived `displayName` values, preserving explicit `label` values as authoritative, and add focused helper plus chat selector tests for displayName-only synthetic rows and synthetic-looking user labels.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep explicit labels authoritative:** `ui/src/ui/app-render.helpers.ts:886`
  - body: The new synthetic-name filter clears `row.label` as well as `row.displayName`. Session labels are user-authored via `sessions.patch` and are exposed separately from displayName, so a valid label like `webchat:g-agent-control-tower-main` would be silently replaced by fallback text and could lose the user's chosen disambiguator. Apply the synthetic filter only to transport-derived displayName values.
  - confidence: 0.9
