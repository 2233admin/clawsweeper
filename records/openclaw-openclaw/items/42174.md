## Summary

Keep this PR open. Current main still rejects Windows drive-letter Docker bind sources in both config validation and runtime sandbox validation, so the PR is not obsolete or implemented on main. The report is concrete and still maps to the current sandbox bind path, but the current PR diff changes a security-sensitive host bind validation boundary and needs maintainer review, Windows-aware root enforcement, and regression tests before it is mergeable.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a maintainer-owned patch. The best fix should reuse the existing bind-spec parser in config validation, update runtime validation and host-path canonicalization with Windows-aware allowed-root enforcement, add regression coverage for valid Windows drive-letter binds and blocked escape cases, and update sandbox docs only if native Windows Docker Desktop bind support is officially supported.
