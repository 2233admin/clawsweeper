## Summary

Keep this PR open. Current main still rejects Windows drive-letter Docker bind sources in both the config schema and runtime sandbox validation, so the PR is not obsolete or implemented on main. The PR addresses a concrete Windows Docker Desktop custom-bind bug, but its diff touches the sandbox host-bind security boundary and still needs maintainer review plus regression coverage before it can be safely merged.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The best fix should reuse the existing bind-spec parser in config validation, define a shared Windows-aware absolute-host-path predicate, update runtime validation and host-path canonicalization with safe allowed-root enforcement, add regression tests for accepted and blocked Windows bind cases, and update sandbox docs only if native Windows Docker Desktop bind support is officially supported.
