## Summary

Keep this PR open. Current main still does not implement the intended Windows Docker bind fix: config validation still splits `D:/...` at the drive-letter colon, and runtime sandbox validation still rejects bind sources that do not start with `/`. The PR is still relevant, but because it changes sandbox host-bind security behavior, it needs maintainer review or a replacement patch with Windows-safe allowlist and canonicalization coverage.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The implementation should reuse the existing bind-spec parser in config validation, add a shared Windows-aware absolute host-path helper, preserve blocked-path and allowed-root enforcement for drive-letter paths, canonicalize existing ancestors safely on Windows, compare drive-letter paths with correct case semantics, and add regression tests for accepted and rejected Windows bind cases.
