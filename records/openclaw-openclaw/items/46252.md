## Summary

Keep open. Current main is only partially archive-aware: the broad `usage.cost` scan includes reset/deleted transcript archives, but the per-session `sessions.usage` path still resolves one transcript file and scans only that file, so an active `<id>.jsonl` plus sibling `.jsonl.reset.*` archive remains undercounted in the dashboard. The related #46605 fix PR is closed unmerged, and the reporter's active-plus-reset reproduction still matches the current control flow.

## What This Changes

The issue requests that OpenClaw cost and usage reporting include `.jsonl.reset.<timestamp>` archive transcripts created by `/new` or `/reset` so daily spend is not undercounted.

## Best Possible Solution

Keep the archive-aware global scan, then add a shared per-session transcript-segment resolver that aggregates every usage-counted segment for a session id: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`, while continuing to exclude `.jsonl.bak.*`, compaction checkpoints, trajectory artifacts, and store backups. Use that infra path for `sessions.usage` cost summaries so dashboard totals, filters, exports, and gateway RPCs agree.

## Reproduction Assessment

Yes. The issue gives concrete active-plus-reset files and costs, and current source still reproduces the path statically because `loadSessionCostSummary({ sessionId })` resolves the primary transcript and scans only that file.

## Solution Assessment

Yes for the repair direction, not for current main. The narrow maintainable fix belongs in infra session transcript resolution and aggregation used by `sessions.usage`, not as a UI-only adjustment, because gateway totals and filtered dashboard views depend on the same summaries.
