## Summary

Keep open. Current main partially fixed archive discovery for the global `usage.cost` summary, but the main dashboard/session totals still flow through `sessions.usage`, which dedupes to one transcript per session and `loadSessionCostSummary()` still returns the primary `.jsonl` when it exists. That matches the reporter’s later partial-cost reproduction: same-session `.jsonl.reset.*` archives remain omitted from per-session and unfiltered dashboard totals.

## Best Possible Solution

Keep this issue open and fix the remaining `sessions.usage`/dashboard path so per-session and unfiltered dashboard totals aggregate all usage-counted transcript segments for the same session id: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`, excluding `.jsonl.bak.*`, trajectory files, and compaction checkpoints. Add a regression where one session has both a primary transcript and a reset archive and assert `sessions.usage` totals include both.
