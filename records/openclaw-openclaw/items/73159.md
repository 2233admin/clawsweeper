## Summary

Keep open. Current main still lacks the read-tool missing-path guidance and message-tool unsupported-action preflight this PR adds; the patch is focused and useful, with only a non-blocking duplicate-action display issue left to polish before merge.

## What This Changes

The PR adds missing-path guidance around the read tool, pre-dispatch unsupported-action validation for the message tool, and focused regression tests for both paths.

## Best Possible Solution

Land a focused version that keeps the read-tool guidance and message-action preflight, preserves the send/broadcast and unknown-discovery pass-through behavior, deduplicates the supported-action display list, and retains the two regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the missing read path still flows through executeReadWithAdaptivePaging without a wrapper, and unsupported channel actions still reach runMessageActionForTool; the PR adds targeted tests for both observable paths.

## Solution Assessment

Yes, with a small polish fix. The proposed changes use the existing read/message tool wrappers and channel discovery seam rather than changing plugin dispatch broadly; the supported-action display list should use a Set before sorting.
