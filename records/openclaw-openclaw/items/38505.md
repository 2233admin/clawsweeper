## Summary

Keep this issue open. Current main still wires Discord inbound text through a MessageCreateListener-only path, does not register a MessageUpdateListener, and the inbound replay guard keys by account/channel/message id, so a same-message-id edit needs an explicit edit-aware dispatch path. Related #53654 overlaps but is broader and still open, so it is useful coordination context rather than a strong reason to close this narrower report.

## Best Possible Solution

Keep this issue open and implement it in the bundled Discord plugin. Add a Carbon MessageUpdateListener, ignore non-content updates, route qualifying edited messages through the existing Discord authorization/preflight/session path with edited content as authoritative, make inbound replay/dedupe edit-aware, and add focused tests for mention-added edits, DMs or active threads, unchanged-content updates, bot/self edits, and duplicate update delivery. Coordinate with #53654 if delete-to-cancel is handled in the same PR, but do not close this narrower issue until implementation lands.
