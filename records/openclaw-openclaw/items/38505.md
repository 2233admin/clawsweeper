## Summary

Keep #38505 open. Current main still does not register a Discord `messageUpdate` listener or provide an edit-aware inbound dispatch path, and the bundled Discord plugin's replay guard still dedupes by account/channel/message id. Related #53654 overlaps, but it is broader, still open, and does not clearly supersede this narrower report.

## Best Possible Solution

Keep the issue open and implement this in the bundled Discord plugin. Add a Carbon `MessageUpdateListener`, ignore non-content updates, route qualifying edited messages through the same Discord preflight/authorization/session path with edited content as authoritative, make inbound replay/dedupe edit-aware, and add focused tests for mention-added edits, DM or active-thread edits, unchanged-content updates, and duplicate update handling. Coordinate with #53654 if delete-to-cancel is implemented in the same pass.
