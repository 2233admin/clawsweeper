## Summary

Keep open. Current main still forwards wizard select and multiselect labels directly, so the intended adapter hardening remains useful, but this PR is not merge-ready because its new Clack mock omits the current `autocomplete` export used by searchable select, which should make the wizard test module fail before assertions run.

## What This Changes

This PR adds prompt-option label normalization in `src/wizard/clack-prompter.ts` and expands `src/wizard/clack-prompter.test.ts` with mocked Clack prompt coverage for blank label fallbacks.

## Best Possible Solution

Land a central prompt-adapter label normalizer that covers regular and searchable select/multiselect paths, with focused unit coverage for each path and wording that accurately distinguishes a proven crash from defensive fallback behavior.
