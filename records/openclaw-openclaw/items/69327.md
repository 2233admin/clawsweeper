## Summary

Keep open. Current main still has the same unresolved product boundary: `sessions_spawn(sandbox="inherit")` is documented and implemented as a sandboxed/unsandboxed guard, while the child run resolves its sandbox independently from the target agent config. Generic Docker env, setupCommand, config-hash recreation, and workspace inheritance exist, but there is no parent resolved Docker sandbox context/env channel for subagent inheritance.

## Best Possible Solution

Keep this open for maintainer design review and a focused fix. The likely direction is to define `sandbox="inherit"` semantics explicitly, add same-agent subagent regression coverage for `sandbox.docker.env` and `setupCommand` after recreate, and avoid leaking parent env/config into cross-agent children unless a reviewed policy allows it.
