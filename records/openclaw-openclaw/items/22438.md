## Summary

Keep open. Current main still has no `agents.defaults.bootstrapTier` schema/config/docs surface or tier-aware resolver, and the discussion now has both an open closing PR and fresh wire-level evidence that bootstrap size affects tool dispatch on local models.

## What This Changes

Request to add configurable tiered bootstrap loading with `minimal`, `standard`, and `full` levels so main, subagent, and cron sessions can receive different workspace context.

## Best Possible Solution

Review and either land or deliberately replace PR #22439 with an additive, documented bootstrap-tier contract that preserves existing defaults, resolves the effective tier from session type plus config override, wires tier selection through bootstrap filtering and extra-file hooks, updates schema/types/help/generated docs/public docs, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source-inspection verification path: current main has no tier symbol, schema key, or resolver path that reads a bootstrap-tier config value. The latest discussion also supplies concrete runtime evidence for the token-pressure behavior this feature is meant to address.

## Solution Assessment

Unclear until maintainer review of PR #22439 and adjacent bootstrap-extra-file/default-context work. A config-driven tier surface is a plausible narrow solution, but default changes and `standard` versus `full` semantics need product agreement.
