## Summary

Keep open. Current main still lacks the requested `agents.defaults.bootstrapTier` config/schema/docs surface and a tier-aware bootstrap resolver, and open PR #22439 is already the direct implementation candidate with closing syntax. The latest issue comment adds useful wire-level evidence that bootstrap size can affect tool dispatch on local models, which strengthens the case for maintainer review rather than cleanup closure.

## What This Changes

Request to add tiered bootstrap loading through `agents.defaults.bootstrapTier` (`minimal` | `standard` | `full`) so main, subagent, cron, and hook-extended sessions can control how much bootstrap context is injected.

## Best Possible Solution

Review and either land or deliberately replace PR #22439 with an additive, documented bootstrap-tier contract that preserves current defaults, resolves the effective tier from session type plus config override, wires tier selection through bootstrap filtering and hook-loaded extras, updates schema/types/help/generated docs/public docs, and adds focused coverage for main, subagent, cron, lightweight, and extra-file behavior.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source-inspection verification path: current main has no tier symbol, schema key, or resolver path that reads a bootstrap-tier config value. The newest comment also supplies concrete runtime evidence for the broader token-pressure problem this feature is intended to address.

## Solution Assessment

Unclear until maintainer review of PR #22439 and the adjacent extra-file/default-context work. A config-driven tier surface is a plausible narrow solution, but default changes and `standard` versus `full` hook semantics need explicit product agreement.
