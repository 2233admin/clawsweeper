## Summary

Keep open. Current main still has the manifest-based runtime-deps selection gap for package.json-only library extensions, so this PR is not obsolete; however, the latest provided diff intentionally changes plugin-gate semantics for automatic native dependency staging and still needs maintainer resolution of the documented startup/repair contract before merge.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extensions in bundled runtime-deps scanning, updates doctor/runtime-deps tests, and documents a library-extension gate exception.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainers decide the library-extension policy: package.json-only bundled library packages that opt into staged runtime dependencies should be discoverable by the intended packaged scan and doctor paths, with docs and tests matching the chosen gate semantics and the existing lazy public-artifact runtime preparation left intact.
