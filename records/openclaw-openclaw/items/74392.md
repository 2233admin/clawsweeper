## Summary

Keep open. Current main still lacks the package.json `stageRuntimeDependencies` fallback for manifest-less bundled library extensions in the packaged runtime-deps scan and doctor repair paths, while this PR is a focused implementation for that gap. It should get maintainer review and validation rather than cleanup closure because it intentionally defines gate-bypass policy for library extensions and sits next to recently merged runtime-deps staging work.

## What This Changes

The PR teaches bundled runtime-deps scanning and `openclaw doctor --fix` to include manifest-less bundled library extension directories that declare `openclaw.bundle.stageRuntimeDependencies: true`, with regression tests for scan, repair, and plugin-gate bypass behavior.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer confirmation of the library-extension gate semantics: package.json-only bundled library packages that explicitly opt into staged runtime dependencies should be included in packaged startup scans and doctor repair, with tests and docs aligned to the chosen policy while preserving the lazy public-artifact runtime path.
