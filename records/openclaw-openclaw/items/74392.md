## Summary

Keep this PR open. Current main still lacks package.json `stageRuntimeDependencies` selection for manifest-less bundled library extensions, so the requested repair remains relevant, but the PR now needs maintainer handling before merge because it bypasses explicit plugin-disable gates and predates the current main runtime-deps repair refactor.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extensions in bundled runtime-deps scan and doctor repair paths, documents a library-extension gate exception, and adds regression tests.

## Best Possible Solution

Port the package.json-only library-extension selection into current main's shared bundled runtime-deps selector, decide and document the intended gate semantics, keep the lazy public-artifact runtime preparation intact, and validate scan/doctor behavior with regression tests against the rebased branch.
