## Summary

Keep this PR open. Current main still misses the packaged scan/doctor coverage for manifest-less library extensions, but the latest PR needs one docs correction and explicit maintainer acceptance of the native dependency staging gate exception before merge.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extensions in bundled runtime-deps scan and doctor repair paths, documents the gate-bypass semantics, and adds regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow change after moving the library-extension documentation to the correct doctor/unscoped-scan scope and confirming the intended operator contract for built-in library packages that are not user-facing plugins.

## Reproduction Assessment

Yes. A packaged-root fixture with `dist/extensions/media-understanding-core/package.json`, `stageRuntimeDependencies: true`, and no `openclaw.plugin.json` reproduces the current-main scan gap by inspection: current selection never reads that package-level flag.

## Solution Assessment

No, not yet. The code direction is narrow for the remaining packaged scan/doctor gap, but the docs need to stop implying Gateway startup prestage coverage unless the PR adds a startup/bootstrap regression and implementation for that path.
