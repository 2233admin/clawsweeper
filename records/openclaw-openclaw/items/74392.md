## Summary

Keep this PR open. Current main still does not include manifest-less bundled library packages in the packaged runtime-deps scan or doctor repair path, while the PR targets that real gap. The remaining question is maintainer policy and overlap with #74246, because the latest PR head intentionally bypasses plugin activation gates for these library deps.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library packages in bundled runtime-deps scanning and doctor repair, plus regression tests.

## Best Possible Solution

Review this as a narrow runtime-deps repair or fold it into #74246: manifest-less bundled library packages that intentionally opt into staged runtime dependencies should be discoverable by packaged startup scans and doctor repair, with explicit tests for the chosen allow/deny/global-disable policy and the existing on-demand public-artifact preparation kept intact.
