## Summary

Keep this PR open. Current main still skips manifest-less bundled library packages in the packaged runtime-deps scan and doctor repair path, while this PR is a focused implementation that addresses that gap. The remaining blocker is maintainer confirmation that library extensions should bypass plugin enable/deny/allow gates, plus normal CI/Testbox validation.

## What This Changes

The PR adds `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extensions in bundled runtime-deps selection and `openclaw doctor --fix`, with regression tests for scan, repair, and gate-bypass cases.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer confirmation of the library-extension policy: manifest-less bundled packages that explicitly opt into staged runtime dependencies should be included in packaged scans and doctor repair, with tests and docs matching the selected gate semantics while preserving the lazy public-artifact runtime path.
