## Summary

Keep open. Current main still has the manifest-less runtime-deps gap for media-understanding-core, but this PR has unresolved maintainer policy questions around plugin gate bypasses and needs to be ported to the current split runtime-deps helpers before it can land.

## What This Changes

The PR adds package.json stageRuntimeDependencies handling for manifest-less bundled library packages in bundled runtime-deps scanning and doctor repair, with tests and docs asserting those library packages bypass plugin gates.

## Best Possible Solution

Land this PR or an equivalent narrow port after maintainer policy review: package.json-only bundled library packages that opt into staged runtime dependencies should be discoverable by the chosen packaged scan and doctor repair paths, with tests and docs matching the final allow/deny/global-disable semantics and the existing lazy public-artifact runtime preparation left intact.
