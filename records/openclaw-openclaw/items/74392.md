## Summary

Keep open. Current main still has the manifest-less library runtime-deps scan gap, and this active PR is the right kind of implementation vehicle, but it still needs maintainer resolution on library-extension gate semantics and startup-scope proof before merge.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library packages in bundled runtime-deps scanning and doctor repair, documents a plugin-gate exception, and adds regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainers decide the library-extension policy: package.json-only bundled library packages that opt into staged runtime dependencies should be discoverable by the intended packaged scan and doctor paths, with docs and tests matching the chosen gate semantics and the existing lazy public-artifact runtime preparation left intact.
