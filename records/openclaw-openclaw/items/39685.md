## Summary

Keep #39685 open. Current main still does not expose or enforce the requested global/per-agent `allowedDomains` and `denyDomains` egress firewall for OpenClaw-managed `web_fetch` or managed/provider `web_search`. The only nearby domain allowlist is the narrow Codex-native `tools.web.search.openaiCodex.allowedDomains` payload filter, and the lower-level SSRF hostname allowlist remains available but unwired to this requested config surface.

## Best Possible Solution

Keep the issue open and implement a core web-tool network policy: add typed/schema/help/docs support for global and per-agent `allowedDomains` and `denyDomains`, resolve the policy consistently with other tool policies, map allowlists into `SsrFPolicy.hostnameAllowlist` for `web_fetch`, add denylist precedence, make cache keys and redirect behavior policy-aware, and explicitly document managed/provider-native `web_search` limits. Add focused tests for schema validation, global/per-agent merge behavior, allow/deny precedence, wildcard matching, redirects, cache separation, and provider limitations.
