## Summary

Keep open. Current main still has SSRF primitives and narrow provider-specific domain filters, but it does not expose or enforce the requested global/per-agent allowedDomains plus denyDomains egress firewall for OpenClaw-managed web_fetch and web_search. The related draft PR #67421 remains an implementation candidate rather than a close reason because it is narrower and not merged.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-reviewed core web-tool network policy with typed config, strict schema/help/docs, global and per-agent resolution, deny-before-allow domain matching, redirect and cache-key enforcement for web_fetch, and explicit semantics for managed web_search versus provider-native search. Review #67421 as a possible partial building block, but close this issue only after the broader policy lands or is superseded by a narrower canonical tracker.
