## Summary

Keep this issue open. Current main still has SSRF primitives and a narrow Codex-native search `allowedDomains` setting, but it does not expose or enforce the requested global/per-agent `allowedDomains` plus `denyDomains` egress firewall for OpenClaw-managed `web_fetch` and `web_search`. The adjacent draft PR #67421 is still open and narrower than this issue, so it is an implementation candidate rather than a close reason.

## Best Possible Solution

Keep this issue open. The best implementation is a first-class core web-tool network policy with typed config, strict schema/help/docs, global and per-agent resolution, deny-before-allow domain matching, redirect and cache-key enforcement for `web_fetch`, and explicit behavior for managed versus provider-native `web_search`. Review #67421 as a possible partial building block, but close this issue only after the broader policy lands or is superseded by a canonical follow-up.
