## Summary

Keep open. Current main still has the Discord voice prompt gap: voice transcript runs resolve Discord channel config for authorization, but the successful voice path does not carry the per-channel `systemPrompt` into `agentCommandFromIngress`; the normal Discord text/native paths already route that prompt through `GroupSystemPrompt` and `extraSystemPrompt`.

## What This Changes

Discord voice transcript turns should apply the same per-channel `systemPrompt` override that Discord text-channel ingress applies.

## Best Possible Solution

Update Discord voice ingress so successful guild voice transcript runs reuse the trusted resolved channel config, build the same per-channel prompt used by text/native ingress, and pass it as `extraSystemPrompt` to the agent while keeping STT transcript text and channel metadata out of trusted system prompt content.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: configure a Discord guild voice channel with `systemPrompt`, process a voice segment, and inspect the `agentCommandFromIngress` options; current main has no path that can include that prompt.

## Solution Assessment

Yes. The narrow maintainable fix is in the Discord plugin: return or otherwise carry the resolved channel config from voice authorization, reuse `buildDiscordGroupSystemPrompt`, and pass the result through the existing `extraSystemPrompt` ingress field rather than changing the shared agent runner.
