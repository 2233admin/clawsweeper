## Summary

Keep open. Current main still has the reported Discord realtime voice gap: the voice transcript path calls the agent directly without carrying the resolved per-channel Discord system prompt, while the normal Discord text/native ingress path already resolves and forwards that prompt.

## What This Changes

Discord realtime voice transcript turns should apply the same per-channel `systemPrompt` override that Discord text-channel ingress applies.

## Best Possible Solution

Update Discord voice ingress so successful guild voice transcript runs reuse the trusted resolved channel config, build the same per-channel prompt used by text/native ingress, and pass it as `extraSystemPrompt` to the agent while keeping STT transcript text and channel metadata out of trusted system prompt content.
