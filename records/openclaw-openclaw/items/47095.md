## Summary

Keep open. Current main still has the reported Discord voice gap: voice transcript runs resolve Discord channel config for authorization, but the successful agent run never receives the resolved per-channel `systemPrompt`, while Discord text and native-command ingress already build `GroupSystemPrompt` and the shared reply runner forwards it as `extraSystemPrompt`.

## What This Changes

The issue asks for Discord voice transcript turns to apply per-channel `systemPrompt` overrides the same way Discord text/native message ingress does.

## Best Possible Solution

Carry the trusted resolved Discord channel config through the voice authorization path, derive the same trimmed group system prompt used by text/native ingress, and pass it as `extraSystemPrompt` to `agentCommandFromIngress` for guild voice transcript runs.
