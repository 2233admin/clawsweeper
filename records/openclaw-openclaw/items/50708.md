## Summary

Keep this PR open. Current main still formats `lastExitStatus` and `lastExitReason` for a runtime whose status is `running`, while systemd runtime loading still preserves `ExecMainStatus` and `ExecMainCode` even when `ActiveState=active`. The PR’s current formatter-level approach is still useful because it addresses the user-facing stale status text without deleting shared runtime fields used by other diagnostics.

## Best Possible Solution

Review and land a formatter-level fix, preferably this PR or an equivalent patch, with regression coverage in `src/daemon/runtime-format.test.ts`. Keep preserving raw systemd last-exit fields in `readSystemdServiceRuntime` so restart health, onboarding, and doctor diagnostics can still use them when appropriate.
