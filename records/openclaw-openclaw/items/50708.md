## Summary

Keep open. Current main still renders `lastExitStatus` and `lastExitReason` for a runtime whose status is `running`, so the PR’s intended user-visible fix is not implemented on main.

## Best Possible Solution

Review and land this PR or an equivalent formatter-level fix, with `src/daemon/runtime-format.test.ts` coverage that suppresses stale exit details when `status` is `running` while preserving exit diagnostics for stopped or failed runtimes. Keep `readSystemdServiceRuntime` returning raw systemd exit fields for restart health and other diagnostics.
