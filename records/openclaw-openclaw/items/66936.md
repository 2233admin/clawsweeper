## Summary

Keep open. Current main fixes the unresolved SecretRef crash and the plain JSON `agents list` path, but the remaining CLI hang class is still exposed because human `agents list`, bare `openclaw agents`, and `agents add` still reach plugin-loading paths. Related PR #74220 is open for the text list/bare agents slice, so this should not close before that lands and the residual add path is handled or explicitly split.

## What This Changes

The issue asks for `openclaw agents list` to tolerate SecretRef-backed Telegram bot tokens and for `openclaw agents` subcommands to exit cleanly after completing their work.

## Best Possible Solution

Land or adapt #74220 for bare `openclaw agents` and text `agents list`, then either add a narrow `agents add` startup-policy or teardown fix if that path still reproduces, or split it into a focused follow-up. Closure should wait for built CLI smoke proof that list/add/bind/delete exit without external timeout when a plugin that previously held handles is installed.
