## Summary

Current main still matches the reported browser navigate race: the browser tool, CLI client, and `/navigate` route pass only URL/target id, while the Playwright implementation calls `page.goto` without `waitUntil`, leaving Playwright's default `load` wait state and a 20s client/request timeout in the path. This should stay open as a focused browser plugin bug rather than stale cleanup.

## Best Possible Solution

Keep this issue open and fix the browser plugin navigate path: validate a navigate wait state, forward it through tool/CLI/client/proxy/API/server to Playwright `page.goto`, align request timeout slack with the navigation timeout, and preserve existing navigation policy and target-resolution behavior. Add targeted regression tests and a user-facing changelog entry when implemented.
