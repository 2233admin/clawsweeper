## Summary

Keep this PR open. Current main still has the Slack interaction liveness gap: block actions and modal lifecycle handlers enqueue session-scoped system events but do not request a heartbeat wake, while this PR is the active rebased implementation candidate that adds the missing guarded wake behavior and regression coverage.

## Best Possible Solution

Keep this PR as the canonical implementation candidate. Land this PR or an equivalent patch that captures `enqueueSystemEvent(...)`'s boolean result in the Slack block-action and modal lifecycle handlers, calls `requestHeartbeatNow({ reason: "exec-event", sessionKey })` only when an event was actually queued, and keeps the SDK export, docs, changelog, and Slack interaction regression tests aligned.
