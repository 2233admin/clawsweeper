## Summary

Keep this PR open. Current main still requires Android API 31, while this PR is a recent, active implementation candidate for API 26 support with user testing and unresolved maintainer product-scope judgment.

## Best Possible Solution

Keep this PR open as an active implementation candidate. The best path is maintainer product judgment on whether OpenClaw should support API 26, then a focused Android compatibility pass with version guards, targeted lower-API tests or emulator proof, and review of the DNS fallback and manifest/runtime behavior before merge. If maintainers decide to stay API 31+, close it only with that explicit product decision.
