## Summary

Keep this PR open. Current main still has the central reported behavior: native Anthropic PDF uses raw `fetch()` against a configurable base URL with `x-api-key`, and native Gemini PDF builds a `generateContent?key=...` URL and raw-fetches it. The PR is security-relevant and focused, but its current strict-guard patch is stale against the newer timeout/provider-transport work and needs revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open and request a rebased revision: route native Anthropic and Gemini PDF calls through the guarded provider HTTP path, preserve the 120s timeout and provider proxy/transport semantics, move Gemini auth out of the URL into `x-goog-api-key`, add provider-specific audit contexts and cleanup, and add focused SSRF/header regression tests.
