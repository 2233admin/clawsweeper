## Summary

Current main has not implemented this PR's intended built-in memory-file optimization. The memory read code has moved from the PR's old `src/memory/*` paths into the memory host SDK, and the current `readMemoryFile()` still reads the whole markdown file with `fs.readFile` before slicing line windows. QMD has a separate streaming partial-read path, but that does not cover the built-in `readMemoryFile()` path this PR targets.

## Best Possible Solution

Keep this PR open as useful work, but require a rebase/update against current main. The best implementation is to add streaming line-window reads in the current memory host SDK read-file path, reuse `buildMemoryReadResultFromSlice`, preserve current default line and character-cap semantics, mirror/package the change consistently, and add a regression test proving windowed built-in reads do not call `fs.readFile`.
