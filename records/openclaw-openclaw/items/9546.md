## Summary

Current main still does not implement a WebChat option to disable, dismiss, customize, or replace the New messages banner. The banner state is still derived directly from scroll position, rendered from that state, absent from persisted UI settings and strict gateway.webchat config, and covered by tests that assert the current indicator behavior.

## Best Possible Solution

Keep this issue open. The best next step is maintainer product direction on the setting surface and semantics, then a narrow implementation that preserves the current default, adds a persisted WebChat new-message indicator mode, feeds it into scheduleChatScroll/renderChat, and updates focused tests plus WebChat/config docs.
