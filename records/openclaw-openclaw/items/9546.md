## Summary

Current main still does not implement a WebChat option to disable, dismiss, customize, or replace the “New messages” banner. The indicator is still derived directly from scroll state, rendered when that state is true, and absent from persisted UI settings, the strict gateway.webchat config schema, public WebChat docs, and focused tests.

## Best Possible Solution

Keep this issue open. A narrow implementation should add a persisted WebChat new-message indicator mode, feed it into scheduleChatScroll and renderChat, expose it in the appropriate Control UI or gateway.webchat config surface, and update focused scroll/render tests plus the public WebChat/config docs.
