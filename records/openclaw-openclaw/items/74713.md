## Summary

Keep open. This PR has the protected `maintainer` label, so it is not cleanup-close material, and patch review found a P2 correctness issue in the new configured-provider matching that should be addressed before merge.

## What This Changes

The PR centralizes bundled runtime-deps selection for doctor, `openclaw plugins deps`, and startup repair to include configured bundled providers, updates docs/changelog/tests, and makes external runtime mirror file refresh atomic.

## Best Possible Solution

Update the configured-provider collection and manifest-provider comparison to use the canonical provider normalizer, add regression coverage for alias model refs or provider config keys, then rerun the targeted runtime-deps checks before maintainer merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use provider normalization for configured providers:** `src/plugins/bundled-runtime-deps-selection.ts:202-203`
  - body: The new provider selection compares only lowercased strings, but provider refs are canonicalized elsewhere with `normalizeProviderId` (`bedrock`/`aws-bedrock` -> `amazon-bedrock`, etc.). A config using one of those supported aliases will not match a bundled manifest provider such as `amazon-bedrock`, so doctor and `plugins deps` will still skip the provider deps this PR is meant to repair. Please normalize both collected config refs and manifest providers with the provider normalizer and add an alias regression test.
  - confidence: 0.88
