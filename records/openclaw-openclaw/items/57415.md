## Summary

Keep open. Current main still does not expose `openclaw agent --cwd`, and this PR is useful, but the latest head still has a concrete P2 issue on the new gateway workspace-selection surface: trusted `workspaceDir` overrides are authorized but forwarded as raw trimmed strings instead of being normalized or rejected at ingress.

## What This Changes

The PR adds `openclaw agent --cwd <dir>`, forwards it into local and gateway agent runs, extends the gateway agent protocol plus generated Swift models with `workspaceDir`, and adds CLI/gateway regression tests.

## Best Possible Solution

Land a rebased version that preserves spawned-session workspace inheritance, authorizes public gateway overrides with a trusted owner signal, normalizes or rejects `workspaceDir` at gateway ingress before dispatch, keeps generated protocol clients/docs aligned, and verifies the focused CLI/gateway regression tests.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting CLI registration, forwarding, and the agent RPC schema; the PR bug is reproducible from the diff by sending an owner-authorized gateway `agent` request with `workspaceDir: "."` or `"~/repo"` and observing that the raw string reaches `agentCommand` for later gateway-process resolution.

## Solution Assessment

No, not yet. The CLI-side normalization and owner authorization are the right direction, but the accepted gateway override should also be normalized with the same path resolver or restricted to absolute paths before `trustedWorkspaceOverride` is assigned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:104`
  - body: The server accepts `workspaceDir` from owner-authorized callers but returns the trimmed raw value. A non-CLI admin client can send `.` or `~/repo`; because `agentCommand` later resolves that path in the gateway process, the run can target the daemon's cwd/home rather than the caller's intended workspace. Normalize at gateway ingress, or require absolute paths, before assigning the override.
  - confidence: 0.9
