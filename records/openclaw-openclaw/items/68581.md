## Summary

Keep open. Current main still implements Android location as foreground-only: the Android mode enum has no `always` value, legacy `always` is downgraded to `whileUsing`, background `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, and the manifest lacks Android background-location/foreground-service-location support. The linked #71015 is adjacent metadata work, not a replacement for this background-fetch request.

## What This Changes

Request Android node support for `location.enabledMode: always`, including an Always settings mode, Android background-location permission flow, foreground-service-backed background `location.get`, tests, and docs updates.

## Best Possible Solution

Keep this open for an explicit Android product/platform decision. If accepted, implement it in `apps/android` by adding an `Always` location mode, Android 11+ background-location permission handling, foreground-service-location manifest/flavor support, a short-lived service-backed background `location.get` path, regression tests for both `whileUsing` and `always`, and updated public location docs.
