## Summary

Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the embedded attempt, compaction, and BTW provider-stream paths still omit the session identity already in scope. The PR is a focused additive core plugin API proposal tied to open #73488 and downstream glueclaw transcript-bleed reports, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, thread them through `registerProviderStreamForModel` from embedded attempt, compaction, and BTW paths, keep one-off image/simple-completion paths undefined where appropriate, add focused passthrough tests, and document the provider-plugin trust boundary before closing #73488.
