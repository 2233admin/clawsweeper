## Summary

Keep open. Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the provider stream registration path still drops the session identity that the embedded attempt, compaction, and BTW paths already have. This PR is a focused additive provider-plugin API proposal tied to open #73488 and the linked downstream glueclaw transcript-bleed reports, not something already implemented on main or cleanly movable to ClawHub without a core API change.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, thread them through `registerProviderStreamForModel` from embedded attempt, compaction, and BTW paths, leave one-off image/simple-completion paths undefined where appropriate, add focused passthrough tests, and document the provider-plugin trust boundary before closing #73488.
