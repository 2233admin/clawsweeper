## Summary

Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` on provider `createStreamFn` context, while the linked open issue #73488 describes a concrete same-author failure mode for provider plugins that need conversation-scoped state. The PR is a focused candidate implementation, not something already shipped or superseded.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, add narrow regression coverage for `ProviderCreateStreamFnContext` on the main attempt, compaction, and BTW paths, document the sensitivity of `sessionKey`, then land the additive optional SDK fields so #73488 can close after merge.
