## Summary

Keep this PR open. Current main does not implement the PR’s central WhatsApp ACP work: the WhatsApp inbound handler still resolves only the normal route, the WhatsApp channel plugin does not expose a configured-binding adapter, and WhatsApp group history still drops media metadata before ACP dispatch. The related merged ACP MCP work does not supersede these WhatsApp-specific changes, while the PR discussion also shows security-sensitive review findings that would need to be resolved before any landing path.

## Best Possible Solution

Keep this PR open, but require it to be rebased or replaced with a narrower current-main implementation that adds WhatsApp configured ACP binding support, preserves media metadata through WhatsApp history/reply context, wires ACP hook behavior intentionally, and resolves the outstanding attachment-path and session-unbind security review comments.
