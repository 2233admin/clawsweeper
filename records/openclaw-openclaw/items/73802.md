Keep open. This is a fresh, concrete Discord embedded exec-approval delivery regression against v2026.4.26, and current main still does not prove the native Discord approval card/buttons are delivered when the approval remains pending and manual `/approve` works.

Keep this issue open. The best path is a focused maintainer review of intended Discord multi-account approval routing, followed by a regression test and fix in the Discord native approval path so a configured exec approval either records a delivered Discord card/button or produces a visible fallback/diagnostic when native delivery yields zero entries.
