## Summary

Keep open. This is a fresh, specific regression report against v2026.4.26, and current main does not provide enough proof that the missing Discord native approval card/buttons are fixed. The code has a native Discord approval runtime, but the gateway can still keep an approval pending through a native client or Discord turn-source route without proving a Discord card was delivered, which matches the report's “approval exists, manual /approve works, no card appears” shape.

## Best Possible Solution

Keep the issue open and fix or verify the Discord native approval delivery path for configured embedded Discord exec approvals. The best maintainer path is a focused regression test that simulates a Discord-originated exec approval with enabled approvers and asserts that the native approval runtime sends a card/button payload, plus failure handling that reports an explicit unavailable/fallback state when no card can be delivered.
