## Summary

Keep this issue open. Current main has useful lower-layer controls, including trusted-proxy identity, operator scopes, SecretRefs/redaction, per-agent auth stores, DM/session isolation, sandbox/tool policy, and group `toolsBySender`, but it still documents a single trusted-operator gateway model and does not implement shared-gateway admin/user/guest RBAC, per-user credential isolation, or audit semantics. The latest maintainer discussion also explicitly keeps this tracker open, while external wrappers such as Pinchy, teamclaw, and openclaw-mission-control are useful prior art rather than a core resolution.

## Best Possible Solution

Keep this issue open as the broad product/security tracker. A complete solution should define gateway and channel caller identity, role- or scope-based authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documented trust boundaries, and regression coverage across gateway auth, config read/write paths, HTTP APIs, and channel command surfaces. If maintainers choose an external-wrapper or ClawHub-first direction, document that explicitly and identify any missing plugin/API surfaces needed to support it outside core.
