## Summary

Current main still does not implement this PR’s intended Slack behavior. Slack room history is pending-only: skipped mention-gated messages are buffered, but processed `requireMention=false` channel messages are not retained after dispatch, and the current regression test still asserts that the second always-on channel message does not see the first as history. The PR remains plausible, but it needs maintainer product review and a rebase into the current `extensions/slack` layout before it should be merged or closed.

## Best Possible Solution

Keep this PR open for maintainer review. If always-on Slack channel context is desired, rebase the change into `extensions/slack/src/monitor/message-handler/*`, carry the resolved mention policy into dispatch or record the processed message during prepare, retain `requireMention=false` room messages in both reply and no-reply outcomes with deterministic ordering, update the Slack regression tests, and revise the public group/Slack docs away from the pending-only model. If maintainers prefer pending-only semantics, close the PR with that explicit product rationale.
