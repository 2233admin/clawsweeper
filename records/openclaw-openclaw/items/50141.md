## Summary

Keep this PR open. Current main already preserves provider-qualified values through the Control UI model picker, but it does not implement the PR's stated `/model` slash-command display changes: bare `/model` still prints the current model and available models as unqualified `m.id` strings. The PR also has recent author follow-up after rebase and addressed review comments, so it remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to either land or cherry-pick the remaining slash-command changes and tests, while recognizing that the core Control UI model-picker behavior is already implemented on current main.
