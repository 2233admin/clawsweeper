## Summary

Keep open. Current main has already fixed the Control UI picker path, but the bare `/model` slash command still prints current and available models as unqualified IDs. The PR still contains useful remaining slash-command behavior and regression coverage, and its diff is scoped to UI TypeScript and tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reduce, land, or cherry-pick the remaining `/model` slash-command change so current and available model displays use `buildQualifiedChatModelValue()` with session/catalog provider data, while preserving the PR's guard against incorrectly borrowing the defaults provider for legacy session rows.
