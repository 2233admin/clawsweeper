## Summary

Keep this PR open. Current main still lacks the intended migration-guidance classification, but the PR patch would introduce a security-adjacent false negative by downgrading exact env-template findings even when the resolved secret is plaintext stored in `config.env` or `env.vars` in the same config file.

## What This Changes

The PR adds a `MIGRATION_RECOMMENDED` secrets-audit finding, summary count, CLI/docs output, and tests for exact `${ENV_VAR}` templates on SecretRef-capable config fields.

## Best Possible Solution

Revise the audit so exact env templates backed by external shell/dotenv values produce migration guidance, while templates whose effective value comes from plaintext `config.env` or `env.vars` produce a plaintext finding or a distinct source-specific finding. Add regression coverage for both `env.vars` and flat `env.*` precedence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve plaintext findings for config.env-backed templates:** `src/secrets/audit.ts:277`
  - body: This branch classifies any authored exact `${VAR}` as `MIGRATION_RECOMMENDED`, but config reads apply `config.env` before substitution. If `openclaw.json` stores `env.OPENAI_API_KEY: "sk..."` and `models.providers.openai.apiKey: "${OPENAI_API_KEY}"`, the resolved credential came from plaintext in the same config file; because audit does not separately scan `config.env`, this suppresses the only plaintext finding.
  - confidence: 0.9
