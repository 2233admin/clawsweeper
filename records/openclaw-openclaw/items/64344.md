## Summary

Keep this PR open. Current main has not implemented the requested audit classification, and the proposed patch is focused, but the diff introduces a security-adjacent false negative: exact `${ENV_VAR}` templates backed by plaintext `config.env` values can be downgraded from `PLAINTEXT_FOUND` to migration guidance.

## What This Changes

The PR adds a `MIGRATION_RECOMMENDED` secrets-audit code, count, CLI/docs output, and tests for exact `${ENV_VAR}` templates on SecretRef-capable config fields.

## Best Possible Solution

Revise the PR so exact templates backed by external environment sources produce migration guidance, while templates whose effective value comes from plaintext `config.env` storage still produce a plaintext finding or a distinct source-specific finding, with regression tests covering both `env.vars` and flat `env.*` precedence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve plaintext findings for config.env-backed templates:** `src/secrets/audit.ts:277`
  - body: This branch classifies any authored exact `${VAR}` as `MIGRATION_RECOMMENDED`, but `readConfigFileSnapshot` applies `config.env` before substitution. If `openclaw.json` stores `env.OPENAI_API_KEY: "sk..."` and `models.providers.openai.apiKey: "${OPENAI_API_KEY}"`, the resolved target came from plaintext in the same config file; because secrets audit does not separately scan `config.env`, this new branch suppresses the only plaintext finding.
  - confidence: 0.89

## Security Review

Status: needs_attention
