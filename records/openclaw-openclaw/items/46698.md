## Summary

Keep open. The user problem is still valid on current main, but this PR is now aimed at a removed legacy provider path, so it would not fix the active GitHub Copilot login implementation without being ported into the plugin-owned auth flow.

## What This Changes

The branch adds a `--wait` flag to the GitHub Copilot login shortcut and changes GitHub device-flow polling to delay before polling and honor `slow_down` interval responses.

## Best Possible Solution

Port the polling contract and remote-terminal wait UX into `extensions/github-copilot/login.ts` and the current provider-auth command path, with focused tests, docs, and changelog credit for the contributor, while leaving token storage and unrelated providers unchanged.
