## Summary

Keep open. The PR addresses real stability gaps that are still present on current main, but the proposed patch is stale and incorrect: the Gmail watcher backoff reset happens on `spawn`, so immediate crash loops still restart at 5s, the branch removes active i18n coverage, and it lacks the required changelog entry.

## What This Changes

The PR wraps subagent output polling read errors, adds warning logs for best-effort config fallback, adds Gmail watcher restart backoff, attempts to clear a media cleanup interval, and removes the UI i18n registry test.

## Best Possible Solution

Rebase or replace the PR with a narrow runtime stability fix: implement Gmail watcher backoff that resets only after a healthy uptime window, preserve i18n registry coverage, drop the obsolete media-server edit, and carry forward the subagent/config improvements with focused tests and a changelog entry.

## Reproduction Assessment

Yes for the blocking PR regression: a local child_process check confirms `spawn` fires before `exit` for an immediately exiting child, so the PR's reset point is too early. The other claimed gaps were verified by source inspection on current main rather than full live reproduction.

## Solution Assessment

No. The PR direction is partly useful, but resetting Gmail watcher backoff on `spawn` is not the narrowest maintainable fix; reset after a non-trivial healthy run window and keep unrelated test coverage intact.
