## Summary

Keep this issue open. Current main still lets the Amazon Bedrock memory embedding auto-detection path call the AWS SDK default credential chain without an `AWS_EC2_METADATA_DISABLED` guard or temporary IMDS suppression, so the reported 169.254.169.254 probe remains plausible. The linked fix PR #64944 was closed unmerged, and current tests do not cover the requested IMDS-disable behavior.

## Best Possible Solution

Keep this issue open. Implement a focused Amazon Bedrock plugin fix around `extensions/amazon-bedrock/embedding-provider.ts` so passive `hasAwsCredentials()` detection honors `AWS_EC2_METADATA_DISABLED`, avoids IMDS probing when no explicit AWS marker is present, restores any temporary environment changes, and does not weaken real Bedrock runtime authentication. Add regression tests in `extensions/amazon-bedrock/embedding-provider.test.ts` for the disabled-IMDS path and env restoration; update Bedrock or memory docs only if the auto-detection contract changes.
