## Summary

Current main does not implement the requested DeepSeek provider policy artifact. The DeepSeek catalog already carries the intended V4 context-window and pricing metadata, but config materialization only gets provider-owned defaults through `provider-policy-api.js`; DeepSeek has no such file, so generic model defaults can still fill missing configured model metadata with zero cost and the default context window.

## Best Possible Solution

Keep this open and land a narrow DeepSeek plugin-owned fix: add `extensions/deepseek/provider-policy-api.ts` that uses the existing DeepSeek catalog to hydrate missing context window, max output, cost, input/reasoning/compat/api metadata for matching configured DeepSeek model rows while preserving explicit user overrides. Add targeted tests proving `materializeRuntimeConfig` or `applyModelDefaults` no longer turns DeepSeek V4 configured rows into 200k/zero-cost rows.
