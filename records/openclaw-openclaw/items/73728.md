## Summary

Keep this issue open. Current main still defaults the shared plugin discovery and manifest snapshot cache controls to 1000 ms, the capability/web provider snapshot paths still expire using the shorter resolved TTL, and I did not find a gateway-specific longer default or launcher override. The linked #73729 and #73730 reports are adjacent cache-miss issues, #73731 was a separate closed duplicate-load path, and broader open reports such as #72310/#73835 do not supersede this focused default-policy request.

## Best Possible Solution

Keep this open for a focused maintainer/product decision. The best fix is either to raise the shared plugin discovery/manifest snapshot defaults or add an explicit gateway-mode default/launcher env path, then align src/plugins/cache-controls.ts, src/plugins/discovery.ts, and src/plugins/manifest-registry.ts. Add targeted tests for unset-env defaults and capability/web snapshot expiration, and update the plugin internals docs at https://docs.openclaw.ai/plugins/architecture-internals with the intended gateway cache window and invalidation model.
