## Summary

Keep this issue open. Current main still defaults both shared plugin discovery and manifest snapshot cache TTLs to 1000 ms, the reported capability-provider and web-provider snapshot caches still expire from the shorter resolved TTL, and I found no gateway launcher override that raises those defaults. The related open PRs #73847 and #73853 are useful adjacent work, but they target web-provider identity misses and repeated provider registry loads, not this default TTL policy.

## Best Possible Solution

Keep this open for a focused maintainer decision. The best path is to choose either a longer shared default or an explicit gateway-mode default/launcher env policy, then update src/plugins/cache-controls.ts plus any lower-level discovery/manifest defaults that should match. Add targeted tests for unset-env defaults and snapshot expiration, document the intended gateway cache window at https://docs.openclaw.ai/plugins/architecture-internals, and coordinate with #73847/#73853 so adjacent cache fixes do not conflict.
