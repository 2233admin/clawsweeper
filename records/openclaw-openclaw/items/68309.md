## Summary

Keep open. Current main still has the unguarded OpenAI-completions `delta.content` append path, so the reported `[object Object]` corruption remains a valid product bug. The linked open PR #68418 uses closing syntax and directly targets this bug, so this issue should remain open until that PR, or an equivalent fix, is reviewed and landed.

## What This Changes

The issue reports that `mistral/mistral-small-latest` with thinking enabled emits repeated `[object Object]` text before the normal assistant answer, even when the UI thinking display is hidden.

## Best Possible Solution

Land a focused parser fix, likely PR #68418 or an equivalent patch, that preserves normal string `delta.content` behavior while safely unpacking array/object typed content into text and thinking blocks, with a regression test in `src/agents/openai-transport-stream.test.ts`.
