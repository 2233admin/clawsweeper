## Summary

Keep this issue open. Current main has the adjacent #68474 mitigation for long SDK Retry-After sleeps and per-provider request timeouts, but it still does not expose a typed per-provider or per-candidate retry-count budget for model provider SDK calls. The proposed config shapes would be rejected by strict schemas, and the OpenAI/Azure SDK client path currently forwards timeout/fetch only, not a maxRetries-style option.

## Best Possible Solution

Implement a typed model-provider retry budget, preferably under provider request configuration such as models.providers.<id>.request.retry.maxRetries or a clearly documented models.providers.<id>.maxRetries. Plumb it into SDK client/request construction where the pinned SDKs support it, define explicit no-op or direct-fetch behavior where they do not, update schema labels/help/generated docs and retry/failover docs, and add regression tests proving a low retry budget advances fallback quickly on 429/529-style failures.
