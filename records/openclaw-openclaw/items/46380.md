## Summary

Keep this PR open. Current main still routes MiniMax VLM image-understanding requests through raw `fetch()` to a host derived from `apiHost`, `MINIMAX_API_HOST`, or `modelBaseUrl`, so the SSRF/key-exfiltration concern is not implemented on main. The PR is security-relevant and narrow, but should be rebased/revised to preserve current timeout and proxy behavior, add `auditContext`, and include regression tests.

## Best Possible Solution

Keep the PR open and request a current-main revision: use `fetchWithSsrFGuard` through the proxy-aware guarded helper or an equivalent local pattern, include `auditContext: "minimax-vlm"`, preserve timeout handling, release dispatchers in a clear `finally`, and add tests for private/cloud-metadata `apiHost`, `MINIMAX_API_HOST`, and `modelBaseUrl` blocking plus HTTP(S)_PROXY compatibility.
