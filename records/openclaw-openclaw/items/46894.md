## Summary

Keep open. Current main still treats `/model status` and `/model list` as non-empty model arguments in the Control UI local slash-command executor, so they flow into `sessions.patch` instead of the existing no-arg model-info path. There is also an open implementation candidate, #47069, whose body says it closes this issue, so the best cleanup path is to review and land or close that PR rather than closing this issue first.

## Best Possible Solution

Keep this issue open and review the paired implementation candidate #47069 first. The desired fix belongs in the Control UI local slash-command executor: route `/model status`, `/model list`, and likely `/model info` through the same model-info display path as bare `/model`, add regression tests proving those inputs do not call `sessions.patch`, and keep normal `/model <model>` session patching unchanged.
