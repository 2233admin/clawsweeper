## Summary

Keep open. Current main still follows the reported Control UI path: `/model` is a local command, `/model status` and `/model list` are passed through as non-empty args, and `executeModel` sends any non-empty arg to `sessions.patch` as a model value. The issue is also paired with open implementation PRs, especially #47069 with closing syntax, so it should remain open until maintainers land, replace, or close that fix path.

## What This Changes

Treat Control UI `/model status` and `/model list` as informational model commands instead of sending `status` or `list` to `sessions.patch` as model names.

## Best Possible Solution

Land or replace the paired Control UI fix so `status`, `list`, and any maintainer-approved informational aliases route through the model-info path, with focused executor tests proving those inputs do not call `sessions.patch` while real `/model <provider/model>` changes still work.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic on current main: `/model status` parses as local command `model` with args `status`, then the local executor calls `sessions.patch` for that non-empty arg, matching the reporter's gateway error logs.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `ui/src/ui/chat/slash-command-executor.ts` with colocated regression tests; the shared channel directive path already handles `list` and `status`, so this does not need a new core protocol or plugin API.
