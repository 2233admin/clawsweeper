## Summary

Keep open. Current main still does not install Homebrew from the skill dependency onboarding path: it only offers to display the Homebrew install command, then the gateway-backed installer returns a missing-brew failure for brew-backed skills. The standalone shell installer can bootstrap Homebrew during initial OpenClaw installation, but that is a separate path and does not cover later onboarding/configure, gateway `skills.install`, or Skills UI installs.

## What This Changes

The issue requests automatically installing Homebrew when a brew-backed skill dependency install needs `brew` and it is missing.

## Best Possible Solution

Keep this open until maintainers decide the policy for skill dependency onboarding. If auto-bootstrap is allowed, add an explicit macOS confirmation flow in the gateway-backed skill install/onboarding path, reuse the standalone installer’s admin/failure expectations, re-detect `brew`, and cover CLI, gateway, and Skills UI behavior. If auto-bootstrap is not allowed, change the prompt and docs to present Homebrew as a manual prerequisite instead of an install step.
