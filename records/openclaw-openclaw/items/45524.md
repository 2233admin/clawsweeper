## Summary

Keep open. Current main has not implemented the requested startup dangerous-flag warning for explicit `agents.defaults.sandbox.mode="off"`, and the PR remains useful, but the proposed patch is not merge-ready because one added trusted-proxy warning misses the same omitted-`allowUsers` case that runtime auth and the security audit already treat as allow-all.

## What This Changes

The PR adds startup dangerous-config warnings for explicit sandbox mode off, gateway auth mode none, and an explicit empty trusted-proxy `allowUsers` list in `src/security/dangerous-config-flags.ts`.

## Best Possible Solution

Rebase the change onto current main, add the core-owned dangerous flags in the current dangerous-flag helper structure, cover explicit `sandbox.mode="off"` without warning on an omitted sandbox config, and align the trusted-proxy warning with runtime/audit semantics by treating both missing and empty `allowUsers` as no allowlist if that extra warning stays in scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - body: Trusted-proxy auth normalizes `allowUsers ?? []`, so leaving `gateway.auth.trustedProxy.allowUsers` unset still accepts any authenticated proxy user.
