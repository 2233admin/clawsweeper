## Summary

Keep open. Current main still has the hard-coded `20000` recovery hint, so the PR targets a real gap, but the latest branch still emits comma-grouped values such as `50,000` for a JSON5 integer config field and its new tests assert those grouped strings.

## What This Changes

The PR changes the context-overflow recovery helper to compute a tiered `reserveTokensFloor` recommendation from the primary model context window, and adds tests for 1M, 200k, 100k, smaller, and unknown context-window paths.

## Best Possible Solution

Keep the context-window-aware guidance if maintainers accept the tiers, but emit the suggested `reserveTokensFloor` as an ungrouped JSON5 numeric literal or a full parseable config snippet, update the assertions to reject grouped values, and add the required changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is clear from the PR diff: `buildContextOverflowResetHint()` formats the suggested config value with `toLocaleString("en-US")`, while current config loading uses JSON5 and the schema requires an integer number.

## Solution Assessment

No. The direction is narrow and useful, but the current patch is not the best mergeable form because the suggested value is formatted for display instead of copy-pasteable configuration.
