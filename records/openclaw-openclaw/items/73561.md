## Summary

Keep open. Current main still emits the hard-coded `20000` overflow-recovery hint, so this PR is not obsolete, but the latest patch should not merge as-is because it formats the recommended config number with commas. OpenClaw config is JSON5 and `reserveTokensFloor` is validated as a number, so a hint like `50,000` is not directly copyable as the numeric config value the message tells users to set.

## What This Changes

The PR replaces the hard-coded context-overflow `reserveTokensFloor` hint with model-context-window tiers and adds regression tests for 1M, 200k, 100k, smaller, and unknown context windows.

## Best Possible Solution

Keep the context-window-aware guidance, but emit the suggested `reserveTokensFloor` as an ungrouped integer or a full parseable JSON5 config snippet, then run the targeted agent-runner execution test and changed gate before merging if the tier policy is accepted.
