## Summary

Keep open. Current main still emits the hard-coded `reserveTokensFloor` recommendation of `20000`, and the existing tests only check that the generic hint mentions `reserveTokensFloor`, not that it adapts to model context size. The PR is a focused implementation candidate for the linked context-overflow guidance problem, but it still needs targeted tier tests and locale-safe, config-parseable number formatting before landing.

## Best Possible Solution

Keep this PR open as a plausible focused fix for the open context-overflow guidance gap tracked by #65839 and the related duplicate #72106. Ask for changes: add tests for the tier boundaries and unknown-context fallback, replace locale-dependent formatting with a config-parseable numeric value, and confirm the context-window lookup uses the most accurate runtime/session evidence available before landing.
