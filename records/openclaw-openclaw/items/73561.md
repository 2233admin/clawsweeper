## Summary

Keep this PR open. Current main still emits the hard-coded `reserveTokensFloor` value of `20000` in the generic context-overflow recovery hint, so the PR is not obsolete or implemented on main. The PR is a plausible implementation candidate for the open canonical guidance issue, but it needs review changes before landing: tests for the tiered values and a locale-safe, config-parseable number format.

## Best Possible Solution

Keep the PR open as a plausible fix for the canonical context-overflow guidance issue.
