## Summary

Keep this issue open. Current main still parses Feishu/Lark DM root/thread metadata but routes ordinary direct messages by sender Open ID only, while topic-derived peer IDs, parent peers, thread bootstrap, MessageThreadId, receive lanes, and debounce partitioning remain group/topic-group scoped. A related open PR, #63148, appears to be the active implementation candidate, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Request Feishu/Lark direct messages to optionally isolate gateway sessions by topic/thread ID instead of grouping every DM from the same sender into one session.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled Feishu plugin, preferably by reviewing or porting #63148: add an explicit opt-in DM topic/session-scope config, preserve sender-scoped defaults, derive direct-message conversation IDs from sender plus canonical topic/root/thread metadata, propagate that ID through routing, configured/runtime bindings, replies, thread context, receive lanes, debounce keys, tests, and Feishu docs.

- Feishu/Lark direct-message topic event semantics still need confirmation before choosing canonical precedence among root_id, thread_id, parent_id, and message_id fallbacks.

- A correct fix should cover receive lanes and debounce partitioning as well as route keys, otherwise simultaneous DM topics could still serialize or merge unexpectedly.
