## Summary

The issue should stay open: current main still has no typed activation metadata, no parser/snapshot state for it, and no request-time path that filters plugin tools from matched skills. This is an active public skill/plugin contract request rather than conservative cleanup.

## What This Changes

Requests a new `metadata.openclaw.activates.plugins` skill frontmatter contract so matched skills can activate plugin tools and bundled skills per turn.

## Best Possible Solution

Design and land a backward-compatible activation contract that parses and stores skill activation metadata, threads activated plugin ids into request-time tool planning, and documents/tests the precedence rules around tool policy and plugin-skill behavior.
