## Summary

Keep open. Current main still resolves the macOS Gateway LaunchAgent plist under the resolved HOME directory and bootstraps that same path, with no boot-volume fallback for external APFS home directories. The reporter’s concrete error 5 case and boot-volume workaround remain actionable.

## Best Possible Solution

Implement a shared macOS LaunchAgent location resolver that detects when the resolved HOME LaunchAgents directory is not launchd-bootstrappable and stages the managed Gateway plist on a boot-volume per-user LaunchAgents path. Use that resolver consistently in install, force reinstall rollback, restart/handoff, uninstall, status/doctor, service audit, update helper, and macOS app plist snapshot code. Add focused unit coverage for external-HOME path selection and manual validation on macOS with an external APFS home.
