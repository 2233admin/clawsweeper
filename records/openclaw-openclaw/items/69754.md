## Summary

Keep open. Current main supports normal Feishu interactive card payloads, but Feishu document/comment targets still degrade approval-style interactive payloads to text without explicit surface guidance, and the generic text fallback can still reduce value buttons to labels. Related Feishu card support in #72667 addressed the normal native-card path from #58298, not this degraded approval-copy problem.

## What This Changes

Improve Feishu approval fallback UX so users get clear guidance when interactive approval buttons are unavailable or approval degrades to text.

## Best Possible Solution

Land a narrow Feishu fallback UX fix: when approval-style interactive buttons degrade to text on Feishu document/comment surfaces, preserve exact `/approve ...` command values and add explicit copy explaining that interactive buttons are unavailable in that Feishu surface, whether same-chat text approval is valid, and when Web UI or terminal UI is the fallback. Update focused Feishu/approval docs and regression tests without changing approval authorization semantics.
