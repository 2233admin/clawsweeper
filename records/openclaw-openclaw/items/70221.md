## Summary

Keep this issue open. Current main still supports only a height hint for Control UI `[embed ...]` previews: the parser/type shape has `preferredHeight` but no width field, `renderPreviewFrame` only receives/applies height, and the iframe CSS remains `width: 100%`. No paired implementation PR appears to be open, and the request is narrow enough for one focused Control UI fix PR.

## Best Possible Solution

Implement a bounded `width`/`preferredWidth` presentation hint for Control UI hosted embeds: parse `width="..."` from `[embed ...]` and structured canvas preview payloads where appropriate, carry it through shared preview/sidebar types, render it with responsive constraints in the preview container/frame, and update tests, docs, prompt guidance, and changelog with reporter attribution. Keep sandboxing, external URL policy, and non-web channel behavior unchanged.
