## Summary

Keep open. Current main still carries Control UI hosted embeds as a height-only presentation path: `width` is not documented, parsed into the preview contract, retained by UI/sidebar types, or applied by the iframe renderer, and the iframe CSS still forces container width.

## What This Changes

Request to add a width parameter to the `[embed]` directive so Control UI hosted iframe previews can render at desktop-friendly widths instead of always filling the chat column.

## Best Possible Solution

Ship a bounded `preferredWidth` presentation hint end to end for hosted Control UI embeds, preserving the existing iframe sandbox, external URL policy, and non-web channel behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `[embed ref="cv_demo" title="Demo" height="600" width="900" /]`: current main parses shortcode attributes but only normalizes and renders height, while CSS keeps the iframe at `width: 100%`.

## Solution Assessment

Yes. The narrowest maintainable fix is to add a bounded width hint through the existing canvas preview contract, renderer, sidebar path, prompt/docs, and focused tests rather than allowing arbitrary inline style or changing iframe security policy.
