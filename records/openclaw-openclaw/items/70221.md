## Summary

Keep #70221 open. Current main still supports only `height`/`preferredHeight` for `[embed ...]` canvas previews; there is no width field in the shortcode parser, normalized preview types, sidebar content, iframe rendering, docs, prompt guidance, or focused tests. The provided related items are only broad title-search overlaps and do not supersede this request.

## Best Possible Solution

Implement this in the core Control UI rich-output path rather than closing it: add a bounded `width`/`preferredWidth` contract in `src/chat/canvas-render.ts`, carry it through normalized chat/sidebar types, apply it in `ui/src/ui/chat/tool-cards.ts` and `ui/src/ui/views/markdown-sidebar.ts` with responsive constraints, then update `docs/reference/rich-output-protocol.md`, webchat prompt examples, and focused parser/render tests.
