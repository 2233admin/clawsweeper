## Summary

Keep this PR open. Current main has not implemented the proposed showAlerts-based webchat behavior, but the branch’s one-flag swap is not safe as-is because it collapses the documented showOk/showAlerts split and can suppress OK-only heartbeat acknowledgments when users intentionally enable them.

## What This Changes

This PR changes Gateway webchat heartbeat chat suppression from `showOk` to `showAlerts` and updates Gateway agent-event tests for heartbeat chat output.

## Best Possible Solution

Preserve the documented heartbeat visibility contract in webchat: OK-only heartbeat acknowledgments should follow `showOk`, non-OK alert content should follow `showAlerts`, and default behavior should continue to hide OKs while showing real alerts. Broader system-event audience classification should stay with the related consumer-path work rather than being folded into this small PR.

Add focused tests in `src/gateway/server-chat.agent-events.test.ts` for default OK suppression, `showOk=true/showAlerts=false` OK visibility, `showOk=true/showAlerts=false` alert suppression, and `showOk=false/showAlerts=true` alert delivery after stripping `HEARTBEAT_OK`.
