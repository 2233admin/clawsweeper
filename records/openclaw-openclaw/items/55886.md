Keep this PR open. The PR remains a focused implementation candidate, with one functional review point around the exported Telegram rate-limit helper narrowing.

## Best Possible Solution

Keep this PR open for maintainer review or refresh. The best path is to land a focused Telegram plugin change in `extensions/telegram/src/sendchataction-401-backoff.ts` that adds per-handler transient cooldown for network, 429, and temporary 5xx chat-action failures, preserves the existing 401 suspension contract, keeps rejected cooldown starts visible to the shared typing guard, and includes targeted handler/network-error tests plus the changelog credit.
