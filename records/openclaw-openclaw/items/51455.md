## Summary

Keep open. Current main still omits `standardFontDataUrl` in the active bundled `document-extract` PDF.js call, the fallback page-render path remains reachable, the latest release has the same omission, and open PR #70936 explicitly closes this issue while #74355 is another active implementation candidate.

## What This Changes

The issue requests passing a filesystem-path `standardFontDataUrl` to PDF.js in the bundled PDF extraction fallback renderer and adding regression coverage.

## Best Possible Solution

Land one narrow plugin-owned fix that resolves the installed `pdfjs-dist/standard_fonts/` directory as a slash-terminated filesystem path, passes it through the local PDF.js type and `getDocument` call, and adds focused regression coverage that fails without the option.
