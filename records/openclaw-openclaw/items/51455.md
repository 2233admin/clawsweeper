## Summary

Keep open. Current main and the latest release still omit `standardFontDataUrl` in the bundled `document-extract` plugin's PDF.js `getDocument` call, while the page-render fallback remains active and PDF.js expects a slash-terminated standard-font asset location. Open PR #70936 explicitly closes this issue and PR #74355 is another active implementation candidate, so the issue should stay open until a maintainer consolidates and lands or closes those PRs.

## What This Changes

Pass a valid `standardFontDataUrl` for `pdfjs-dist/standard_fonts/` into the PDF extraction fallback renderer and add regression coverage for Node PDF.js standard-font warnings.

## Best Possible Solution

The bundled `document-extract` plugin should resolve the installed `pdfjs-dist` package root to a slash-terminated `standard_fonts/` filesystem path, pass that as `standardFontDataUrl` through the local `PdfJsModule.getDocument` type and call site, and include focused real-module regression coverage proving standard-font PDFs extract text without the warning.
