## Summary

Keep open. Current main still has the active PDF fallback renderer calling PDF.js without `standardFontDataUrl` before page rendering, and the existing regression tests do not assert the PDF.js document options. The issue is already paired with open PR #70936, which has GitHub closing linkage for this issue and a focused fix shape, so the maintainer path is to review/land or close that PR rather than auto-close this issue or queue duplicate work.

## Best Possible Solution

Keep this issue open while #70936 is reviewed. The best fix is a narrow change in `extensions/document-extract/document-extractor.ts` that resolves the slash-terminated `pdfjs-dist/standard_fonts` directory from the installed package root, passes it as `standardFontDataUrl` to `getDocument`, updates the local PDF.js type shim if needed, and adds regression coverage in `extensions/document-extract/document-extractor.test.ts` plus any calling-seam tests needed. Close this issue only after that PR or an equivalent fix merges.
