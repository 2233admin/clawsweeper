## Summary

Keep open. Current main has an adjacent `openclaw backup create/verify` surface, but it does not include this PR's macOS keep-awake helper, Docker-specific backup/restore scripts, or Docker migration documentation. The contribution remains useful, but the restore script needs archive extraction hardening before it should merge.

## What This Changes

The PR branch adds Docker install documentation plus `scripts/openclaw-keepawake.sh`, Docker migration backup/restore scripts, and shared shell helpers for macOS-hosted Docker operation.

## Best Possible Solution

Keep the PR open and harden the migration restore path before merge, preferably by reusing or mirroring the existing backup verifier's archive path validation and then adding focused validation for the new shell helpers/docs.

## Reproduction Assessment

Yes for the review finding: static inspection of the PR branch shows `tar -xzf` runs at `scripts/migrate/restore-openclaw.sh:102` immediately after sidecar checksum verification and before any path/link validation. Runtime reproduction is not required to confirm that validation is missing from the proposed code path.

## Solution Assessment

No, not as currently proposed. The feature direction is reasonable, but the restore script should validate tar entries before extraction or delegate to a safer existing backup verification/extraction path before it handles sensitive OpenClaw config and workspace data.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Validate archive paths before extraction:** `scripts/migrate/restore-openclaw.sh:102`
  - body: The restore helper accepts an arbitrary `--archive` and extracts it with `tar -xzf` before inspecting member paths. A sidecar checksum only proves the archive matches that sidecar; it does not reject absolute paths, traversal entries, symlinks, or hardlinks before extraction. Please list and reject unsafe entries before extracting, or reuse the existing backup verifier semantics.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete archive-extraction hardening issue in the new restore helper.

Concerns:

- **[medium] Unsafe tar extraction before path validation:** `scripts/migrate/restore-openclaw.sh:102`
  - body: `restore-openclaw.sh` extracts the provided archive into a temporary directory before validating member paths or link entries. Because the script restores sensitive local config/workspace state, it should reject unsafe tar entries before extraction rather than relying only on a companion checksum and post-extraction `SHA256SUMS`.
  - confidence: 0.87
