## Summary

Keep this PR open. Current main still does not implement the PR’s central UI behavior: live `state: "error"` events only set hidden error state, send failures still create normal text bubbles, assistant error-only history has no `errorMessage` fallback for display, and grouped chat has no distinct model-error label or styling. The related #55923 remains open and overlaps only the live-error append path, while shipped #70848 is a narrower runner-side propagation fix rather than this UI/history/rendering work.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI pieces into the canonical inline-error work. The best implementation should append structured assistant error messages for live run failures and direct send failures, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, add distinct grouped chat error styling/labeling, preserve current sanitized connect-error formatting, coordinate with open #55923 and shipped #70848, and keep focused controller/extraction/render regression tests.
