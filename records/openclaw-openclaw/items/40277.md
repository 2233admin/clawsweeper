## Summary

Keep open. Current main still does not implement this PR's intended client-side inline model-error behavior: live Gateway error events are not appended into chat history, persisted assistant error-only records do not become visible text, and grouped chat rendering still lacks a highlighted error bubble. The shipped #70848 runner fix is useful but narrower, and open #55923 overlaps only part of the live-error path.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused behavior into the canonical inline-error work. The best implementation should append structured assistant error messages for live `state: "error"` events, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current structured connect-error formatting, coordinate with #55923 and the shipped #70848 runner fix, and add focused controller/extraction/render tests on current main.
