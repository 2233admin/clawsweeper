## Summary

Keep open. Current main still does not implement this PR's intended inline model-error UI: live `state: "error"` events only set hidden `lastError`, persisted assistant `stopReason: "error"` / `errorMessage` metadata is not extracted into visible chat text, and grouped chat rendering has no highlighted model-error bubble or empty-detail fallback. The shipped #70848 fix is narrower runner/provider propagation work, and open #55923 overlaps only part of the live-error append behavior.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused behavior into the canonical inline-error work. The best implementation should append structured assistant error messages for live chat `state: "error"` events, render persisted assistant `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current structured connect-error formatting, coordinate with #55923 and the shipped #70848 runner fix, and add focused controller/extraction/render tests on current main.
