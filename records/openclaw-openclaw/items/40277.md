## Summary

Keep open. Current main still does not implement this PR's full inline model-error behavior: live chat error events still only update hidden error state, persisted assistant error metadata is not extracted into visible chat text, and grouped rendering has no highlighted model-error bubble or empty-detail fallback. The shipped #70848 work fixes a narrower runner/provider path, and open #55923 overlaps only part of the live-error append behavior, so this PR still contains useful focused UI work after a rebase.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused behavior into the canonical inline-error work. The best implementation should append structured assistant error messages for live chat `state: "error"` events, render persisted assistant `stopReason: "error"` / `errorMessage` entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current structured connect-error formatting, coordinate with #55923 and the already-shipped #70848 runner fix, and add focused controller/extraction/render tests on current main.
