## Summary

Keep this PR open. Current main still clears/records chat error events as hidden controller state instead of appending structured assistant error messages, send failures still render as ordinary assistant text, persisted assistant error-only history has no extraction fallback, and grouped chat has no distinct model-error label or styling. The PR's latest diff appears to address the Greptile edge-case and duplicated-prefix review comments, but the behavior is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, then either rebase and land its focused UI pieces or fold them into the canonical inline-error work. The best implementation should append structured assistant error messages for live run failures and send failures, render persisted `stopReason: "error"` / `errorMessage` history entries including empty-detail fallbacks, add distinct grouped chat error styling/labeling, preserve `formatConnectError`, coordinate with #55923, and keep focused controller/extraction/view regression tests.
