## Summary

Keep open. Current main still clears Dashboard chat run state on gateway hello, the hello snapshot still lacks active chat-run recovery data, and the composer still derives Stop/Queue from `chatRunId`. The issue is also paired with open, unmerged PR #41067 using closing syntax, so this issue should stay open until maintainers review, land, replace, or close that PR.

## What This Changes

Gateway Dashboard chat should preserve Stop/Queue busy controls for an active session after tab/view switches or reconnect-style restoration.

## Best Possible Solution

Keep this issue open and treat PR #41067 as the active implementation candidate. The best fix should add an additive active chat-run snapshot to Gateway hello, populate it from server active run state, restore matching UI `chatRunId`/busy state on reconnect or tab return, preserve Stop/Queue behavior, update generated protocol/client artifacts if affected, and cover the Gateway/UI tests before maintainer review.
