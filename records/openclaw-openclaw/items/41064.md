## Summary

Keep this issue open. Current main still clears Dashboard chat run state after gateway hello, the hello snapshot has no active chat-run recovery field, and the composer busy calculation still does not treat abortability as busy. The issue is also explicitly paired with open PR #41067, so the right maintainer path is to review, update, land, or close that PR rather than close this issue independently.

## Best Possible Solution

Keep this issue open and treat PR #41067 as the implementation candidate. The likely fix should add an additive active chat-run snapshot to the Gateway hello contract, populate it from server active run state, restore matching UI chatRunId/busy state on reconnect or tab return, make composer busy state include abortability, and update the focused Gateway/UI tests and protocol artifacts before maintainer review.
