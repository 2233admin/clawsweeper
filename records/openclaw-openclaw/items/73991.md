## Summary

Keep open. Current main still lacks config-backed per-agent/default command lanes, so this PR remains a relevant implementation candidate for #43235, but the proposed patch is not merge-ready because it preserves an untrusted lane override at the gateway boundary and has concrete lane-resolution defects called out by the diff review.

## What This Changes

The PR adds `agents.defaults.commandLane` and `agents.list[].commandLane` schema/docs/types, routes gateway and auto-reply agent runs through resolved command lanes, registers configured lane concurrency, and adds tests plus a changelog entry.

## Best Possible Solution

Keep this PR open as the active #43235 implementation path, but require focused changes before merge: make foreground command lanes server-controlled or allowlisted, fix lane/maxConcurrent inheritance semantics, reject or remap reserved lane ids consistently, align resolved agent config with default inheritance, then rerun the changed gate.

## Reproduction Assessment

Yes. A high-confidence reproduction is a config with `agents.defaults.commandLane: { id: "inbound", maxConcurrent: 3 }` and an agent override `{ id: "agent:heavy" }`, which the PR resolves as `agent:heavy` with maxConcurrent `3`; a gateway `agent` request with a unique `lane` also bypasses the configured lane because `request.lane` wins.

## Solution Assessment

No, not as-is. The product direction is the right narrow place to solve #43235, but the implementation should not let caller-supplied lanes bypass configured isolation and should only inherit default concurrency when the resolved lane is actually the default lane.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep request lanes server-controlled:** `src/gateway/server-methods/agent.ts:1083-1084`
  - body: The new resolver gives `request.lane` precedence over configured agent lanes. Since the command queue creates state for any new lane string and each lane has its own concurrency slot, an authenticated caller can send unique lane names to bypass the configured/default lane cap; ignore this field for network ingress or validate it against a server-side allowlist.
  - confidence: 0.86
- **[P2] Stop inheriting maxConcurrent across lane ids:** `src/config/agent-command-lanes.ts:54`
  - body: When defaults configure `{ id: "inbound", maxConcurrent: 3 }` and an agent sets only `{ id: "agent:heavy" }`, this fallback applies the inbound lane's cap to `agent:heavy`. Only inherit `defaults.maxConcurrent` when the agent did not override the lane id.
  - confidence: 0.9
- **[P2] Reject reserved command lane ids:** `src/config/zod-schema.agent-runtime.ts:827-830`
  - body: The schema accepts any non-empty lane id, but `resolveGlobalLane("cron")` remaps execution to `cron-nested` while startup applies this config to `cron`. Reserved ids such as `cron` should be rejected or normalized so configured concurrency applies to the lane that actually runs.
  - confidence: 0.78
- **[P3] Inherit default commandLane in resolved agent config:** `src/agents/agent-scope-config.ts:125`
  - body: `ResolvedAgentConfig.commandLane` is the only newly inheritable field that skips `agentDefaults?.commandLane`, so consumers of the resolved config see `undefined` even when a default command lane applies. Mirror the existing default fallback pattern used by nearby fields.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff has a concrete resource-exhaustion concern because network callers can still supply arbitrary lane names ahead of the new server-configured lane policy.

Concerns:

- **[high] Untrusted lane override can bypass configured caps:** `src/gateway/server-methods/agent.ts:1084`
  - body: `request.lane` is accepted from the gateway RPC payload and wins over the configured per-agent/default lane. The command queue creates a new lane state for any string with its own concurrency slot, so repeated unique lane names can bypass intended lane caps and increase concurrent work.
  - confidence: 0.86
