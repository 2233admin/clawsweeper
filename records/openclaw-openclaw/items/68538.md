## Summary

Keep this PR open for maintainer review. Current main still lacks a stalled-defer timeout for Discord native commands, but the proposed 1500 ms abort can drop commands that Discord could still acknowledge inside the upstream 3-second interaction window, and the branch needs review-comment fixes plus a port over current main's native-command refactor.

## What This Changes

The PR timeboxes Discord native slash-command `interaction.defer()` before dispatch, adds native-command lifecycle logging, wraps chunked follow-up sends in the interaction guard, and adds a regression test for a stalled defer.

## Best Possible Solution

Land a rebased Discord plugin fix that prevents an indefinitely stalled defer from blocking the ACK path, preserves commands whose defer succeeds within Discord's initial-response window, uses a closed timeout classification, keeps follow-up guard consistency, and retains focused regression coverage for stalled and near-deadline defer behavior.
