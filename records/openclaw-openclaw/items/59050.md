## Summary

Keep this issue open. Current main still does not implement configurable workspace bootstrap file ordering: bootstrap discovery and final prompt rendering use fixed ordering, the strict config schemas expose only injection/size/truncation controls, and the hook workaround cannot preserve arbitrary known-file order because the renderer re-sorts files before rendering.

## Best Possible Solution

Keep this issue open for a conservative core implementation: add documented `agents.defaults.bootstrapFileOrder` and likely `agents.list[].bootstrapFileOrder`, validate supported bootstrap basenames, thread the setting through bootstrap context assembly and final prompt rendering before fixed sorting, preserve current default order and deterministic fallback for unlisted files, decide HEARTBEAT/cache-boundary semantics, and update schema/help/generated docs plus tests proving SOUL.md can render before AGENTS.md while default behavior remains unchanged.
