## Summary

Keep open. Current main still does not implement Sysbox/sysbox-runc, does not accept an `agents.defaults.sandbox.docker.runtime` selector, and does not pass Docker `--runtime` when creating sandbox containers. Main now has a generic sandbox backend seam and OpenShell, so a Sysbox backend plugin may be possible, but the issue discussion explicitly treats this as the remaining canonical runtime-architecture decision rather than completed by Docker-socket sandboxing, sandbox-common, or OpenShell.

## Best Possible Solution

Keep this open until maintainers choose the product boundary. The two concrete paths are: add a conservative core `agents.defaults.sandbox.docker.runtime` selector with types, strict schema, config resolution/hash behavior, Docker create `--runtime` wiring, docs, and tests; or keep core generic and implement Sysbox as a sandbox backend plugin using `registerSandboxBackend`, with host prerequisites and security tradeoffs documented and any missing SDK gaps tracked separately.
