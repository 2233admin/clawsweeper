## Summary

Keep open. Current main still does not implement Sysbox/sysbox-runc, does not accept `agents.defaults.sandbox.docker.runtime`, and does not pass Docker `--runtime` when creating sandbox containers. The newer sandbox backend/plugin seam may be the right route for a Sysbox implementation, but the issue discussion already treats this as the canonical runtime-architecture tracker rather than completed by Docker-socket sandboxing or sandbox-common work.

## Best Possible Solution

Keep this open until maintainers choose a concrete path: either add a conservative core `agents.defaults.sandbox.docker.runtime` selector with types, strict schema, config resolution/hash behavior, Docker create `--runtime` wiring, docs, and tests; or move Sysbox to a sandbox backend plugin built on `registerSandboxBackend`, with host prerequisites and security tradeoffs documented and any missing SDK gaps tracked separately.
