## Summary

Keep open. Current main still has no command-lane circuit breaker or maxExecutionMs timeout API, so the PR is not implemented or obsolete. The branch is useful core queue hardening, but it is not merge-ready because the provided review discussion flags correctness blockers around uncancelled timed-out work and a documented priority option that is not implemented.

## What This Changes

The PR branch adds CommandLaneTimeoutError/maxExecutionMs support plus CommandLaneCircuitBreakerError depth/wait threshold rejection to src/process/command-queue.ts and expands the command queue tests.

## Best Possible Solution

Keep the PR open, but require a revised queue-hardening implementation before merge: define a cancellable or abort-aware timeout contract, remove or implement priority ordering, keep the breaker focused on depth and queued-wait thresholds, and add regression tests for depth trip, wait-time trip, retryAfterMs, default main-lane behavior, and no overlap after timeout.
