## Summary

Keep open. Current main still has source-, doc-, and test-backed non-off implicit API thinking defaults for reasoning-capable models, cron paths can still resolve missing payload thinking through that default resolver, and Microsoft Teams still lacks the reasoning-payload suppression guard used by other channel delivery paths.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking and reasoning visibility default-off unless explicitly configured, add global/default audit controls, honor agent defaults for cron jobs, and prevent reasoning blocks from reaching chat channels.

## Best Possible Solution

Define one explicit policy across runtime, cron, docs, and channels: API thinking is off unless explicitly configured or truly required by a provider/model; cron agentTurn jobs inherit the same resolved agent/global default as normal agent turns; chat channels suppress reasoning-marked payloads unless they deliberately implement a reasoning lane; and release notes call out billing-impacting default changes prominently.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: resolve an unconfigured reasoning-capable model or Claude 4.6 model, run an agentTurn cron payload without thinking, and pass an isReasoning payload through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because it keeps non-off implicit thinking defaults, has incomplete default-agent cron inheritance, and leaves Teams without the reasoning-payload suppression guard.
