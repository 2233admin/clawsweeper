## Summary

Keep open. Current main partially addresses the report by adding a global reasoning visibility default and making Anthropic Opus 4.7 thinking default to off, but the broader issue is still present: model-capability reasoning visibility can still flip on, implicit thinking still defaults non-off for reasoning-capable models and cron turns without explicit thinking, and Microsoft Teams still lacks a reasoning-payload suppression guard.

## What This Changes

The issue requests opt-in reasoning/thinking defaults for Claude and other reasoning-capable models, a global reasoning default, startup or doctor warnings for accidental reasoning/thinking costs, and suppression or separation of reasoning blocks in chat surfaces.

## Best Possible Solution

Keep the issue open and align the product contract across code, tests, docs, and channels: make API thinking and reasoning visibility opt-in unless a provider declares a mandatory or true default policy, keep global/per-agent overrides explicit, add cron/startup or doctor auditing for costly implicit thinking, and suppress reasoning-only payloads in Teams the way Discord and Slack already do. Coordinate the global visibility knob with #13607, but do not treat that narrower issue as covering the cron and channel-leak parts.
