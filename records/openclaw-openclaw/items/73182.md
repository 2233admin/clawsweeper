## Summary

Keep open. Current main still contains the capability-based default that can turn reasoning visibility on for reasoning-capable catalog models when no directive/session/per-agent reasoning setting exists and thinking resolves to off; tests assert that behavior. The global `agents.defaults.reasoningDefault` requested by the issue is still not schema-backed, and Microsoft Teams still renders reply payload text without a reasoning-payload suppression guard. The exact Anthropic billing claim needs a targeted payload test because OpenClaw passes API thinking through `thinkLevel` separately, but the reported default/visibility/product-boundary problem is not implemented or superseded.

## Best Possible Solution

Keep this issue open and fix the core reply/model-selection path: reasoning visibility should stay `off` unless explicitly set or a provider profile declares a true default/mandatory reasoning policy. Add schema/resolution/docs for `agents.defaults.reasoningDefault` if maintainers accept the global knob, add Teams suppression for flagged reasoning payloads, and cover Anthropic Claude unset-config behavior with request-payload and reply-delivery regression tests. Coordinate the global-default piece with #13607 rather than closing this broader report.

## Risks / Open Questions

- The exact Anthropic spend increase depends on the pi-coding-agent/Anthropic transport payload; the fix should add a regression test proving unset config does not request extended thinking for the affected Claude models unless thinking is explicitly enabled.
- Changing this default can affect providers that rely on the current capability-based behavior, so the fix should distinguish provider/model support from provider/model default or mandatory reasoning rather than blanket-removing provider hooks.
