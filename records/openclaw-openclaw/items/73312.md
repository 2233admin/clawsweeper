## Risks / Open Questions

- Merging the current PR shape could leave delivered Matrix replies unchanged if renderPresentation and sendPayload remain only on the Matrix runtime object and are not exposed through the registered Matrix plugin outbound facade.
- Forwarding arbitrary Matrix extraContent from payload channelData is event-content sensitive because Matrix send merges it over base content; this needs allowlisting or construction of only the com.openclaw.presentation key.
- The PR tests cover matrixOutbound directly, but the remaining regression is at the facade/core-delivery boundary, so a delivered-reply or plugin-facade test should be added before landing.
