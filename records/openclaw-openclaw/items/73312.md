## Summary

Keep this PR open. Current main still lacks Matrix MessagePresentation metadata delivery, and the branch is not obsolete, but the submitted change only adds runtime adapter/test coverage; normal core delivery still would not see the new Matrix hooks through the channel facade. The security pass also leaves a merge-blocking review point because the proposed payload path forwards arbitrary Matrix extraContent into event content that is currently merged over generated Matrix fields.

## What This Changes

The PR adds Matrix outbound MessagePresentation rendering and sendPayload support in extensions/matrix/src/outbound.ts, with outbound tests for presentation metadata, mediaUrls, and audioAsVoice handling.

## Best Possible Solution

Revise the PR so Matrix exposes presentationCapabilities, renderPresentation, and sendPayload through the channel facade or an expanded shared runtime delegate, constrain Matrix extraContent forwarding to the intended com.openclaw.presentation metadata, and add coverage at the registered-channel/core-delivery boundary while keeping the mediaUrls and audioAsVoice regressions covered.
