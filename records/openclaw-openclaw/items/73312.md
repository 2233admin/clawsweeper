## Summary

Keep this PR open. Current main still does not implement Matrix MessagePresentation metadata delivery, and the PR is not obsolete. The current PR shape still needs revision because it adds runtime Matrix outbound methods and tests, but current core delivery loads the Matrix channel facade, whose outbound surface still does not expose presentationCapabilities, renderPresentation, or sendPayload. Security pass found no workflow, dependency, lockfile, script, package-publishing, or supply-chain changes, but the Matrix extraContent forwarding should be constrained before merge.

## Best Possible Solution

Request revisions rather than close: expose Matrix presentationCapabilities, renderPresentation, and sendPayload through the Matrix channel facade or extend createRuntimeOutboundDelegates; add facade/core-delivery coverage proving MessagePresentation reaches Matrix events; preserve the PR’s existing fixes for version ordering, mediaUrls, and audioAsVoice; and restrict Matrix extraContent forwarding to the intended com.openclaw.presentation payload before merge.
