## Summary

Keep this PR open. Current main still does not implement Matrix MessagePresentation metadata delivery, and the PR is not obsolete. The submitted work remains relevant, but it still needs revisions because the Matrix channel facade does not expose the new presentation/sendPayload methods to core delivery, and the PR's Matrix extraContent path should be constrained before merge.

## Best Possible Solution

Keep this PR open and request revisions: expose Matrix presentationCapabilities, renderPresentation, and sendPayload through the Matrix channel facade or extend createRuntimeOutboundDelegates; add facade/core-delivery coverage proving MessagePresentation reaches Matrix event extraContent; keep the mediaUrls, audioAsVoice, and version-ordering fixes; and restrict Matrix extraContent forwarding to the intended com.openclaw.presentation payload before maintainer merge review.
