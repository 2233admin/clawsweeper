## Summary

Keep open. Current main still has the Telegram/Feishu channel gap this PR targets, but the proposed fallback is incomplete against the current config contract because it misses normalized agent IDs and the now-supported global reasoning default.

## What This Changes

The PR updates Telegram and Feishu reasoning-preview resolution to use per-agent `agents.list[].reasoningDefault` as a fallback, passes config and agent IDs into Feishu call sites, and adds extension tests for config fallback plus explicit session `off` behavior.

## Best Possible Solution

Update the branch so Telegram and Feishu resolve the effective reasoning visibility with the same precedence current main exposes: explicit session state first, then normalized per-agent config, then `agents.defaults.reasoningDefault`, then `off`; Feishu should still enable live preview only when the effective value is `stream`.
