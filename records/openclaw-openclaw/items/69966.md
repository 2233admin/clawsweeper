## Summary

Keep open. Current main still keeps bootstrap profile knobs on `agents.defaults` only: per-agent config/types do not expose the requested fields, strict schema does not allow them under `agents.list[]`, runtime bootstrap/context-injection paths resolve defaults-only values, and the Control UI profile presets patch only defaults. The related open PR #69988 is useful precedent for per-agent override resolution, but it is scoped to compaction and does not implement this request.

## What This Changes

Allow individual `agents.list[]` entries to override bootstrap profile settings such as `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` instead of forcing all agents to share `agents.defaults`.

## Best Possible Solution

Implement direct optional per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` under `agents.list[]`; resolve them for the effective session agent id with fallback to `agents.defaults`; update runtime bootstrap/context-injection paths, Control UI profile application, schema/help/docs/generated baselines, and focused tests. Defer named reusable profile registries unless maintainers explicitly choose that larger product layer.
