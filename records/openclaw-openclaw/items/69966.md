## Summary

Current main still treats `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` as global `agents.defaults` bootstrap profile settings. The per-agent config type and strict schema do not accept these fields under `agents.list[]`, runtime bootstrap/context-injection paths read defaults-only values, the Control UI profile presets patch only defaults, and docs still present bootstrap budgets as defaults-only while listing only nearby per-agent `skillsLimits` and `contextLimits`. The adjacent open compaction PR #69988 does not cover this bootstrap-profile request.

## Best Possible Solution

Keep this issue open and implement a focused core config/runtime/UI feature: add optional direct per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` under `agents.list[]`; resolve them by effective session agent id with fallback to `agents.defaults`; update Control UI so the bootstrap profile surface can apply global defaults or selected-agent overrides; and document/test inheritance. Defer named reusable profile registries unless maintainers explicitly want that product layer.
