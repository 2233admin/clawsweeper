## Summary

Keep open. Current main still lacks the shared daily-memory/session-summary helper surface and several downstream consumers remain on local or canonical-only rules, so the PR’s intended behavior is not implemented on main. Because this is an open XL implementation touching gateway startup, memory persistence, plugin-facing runtime exports, scripts, docs, and path-boundary logic, the next step is maintainer review rather than closure or an automated replacement PR.

## What This Changes

The PR adds shared daily-memory/session-summary parsing, probing, provenance, startup-selection, and backfill helpers, exports them through the memory host runtime surface, and rewires startup context, session-memory, doctor, memory-core dreaming/REM/short-term flows, bootstrap injection, docs, and scripts to use those rules.

## Best Possible Solution

Land, split, or replace this with a maintainer-approved implementation that centralizes daily-memory/session-summary classification and provenance, applies it consistently across startup/session-memory/doctor/dreaming/REM/short-term flows, and keeps plugin-facing file helpers narrowly scoped with explicit workspace-boundary guarantees and regression coverage.
