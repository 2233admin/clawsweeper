## Summary

Keep open. Current main still lacks the shared daily-memory/session-summary model, recent-daily index, startup selector, and downstream rewiring proposed here, and the PR remains an open XL implementation touching gateway, memory runtime, memory-core, docs, and scripts. This is not obsolete or safely closeable; it needs maintainer review of scope, validation, and the path-boundary/security-sensitive parts before merge.

## What This Changes

The PR adds shared daily-memory/session-summary parsing, probing, provenance, startup-selection, and backfill helpers, exports them through the memory host runtime surface, and rewires startup context, session-memory, doctor, memory-core dreaming/REM/short-term flows, bootstrap injection, docs, and scripts to use those rules.

## Best Possible Solution

Land, split, or replace this with a maintainer-approved implementation that centralizes daily-memory/session-summary classification and provenance, applies it consistently across startup/session-memory/doctor/dreaming/REM/short-term flows, and preserves explicit workspace-boundary guarantees for plugin-facing file helpers with focused regression coverage.
