## Summary

Keep open. Current main has partial mitigations for OAuth refresh contention, session/run queueing, and background-task visibility, but it still does not fully resolve the reported multi-agent orchestration cluster. The concurrent `openclaw agents add` path still computes from an early config snapshot and commits via a stale-base conflict guard rather than a serialized re-read/merge transaction, and open PR #44997 remains an implementation candidate for that exact config-write symptom.

## Best Possible Solution

Keep this issue open. Review and either land, rewrite, or close #44997 for the serialized config mutation piece, with regression coverage proving concurrent non-interactive `openclaw agents add` calls preserve every agent and binding. Then split or link the remaining orchestration work explicitly: session/run queue semantics, local lock/failover boundaries, OAuth/auth-noise boundaries, CLI-owned child-process cancellation and visibility, and subagent completion recovery in coordination with #44925 and the now-shipped parts of #45200.
