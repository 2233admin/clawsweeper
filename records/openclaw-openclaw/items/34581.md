## Summary

Keep this PR open. Current main still uses strict `express.json()` in the Microsoft Teams webhook and only handles 413 parser errors, so the malformed Bot Framework JSON escape path described by the PR is not implemented on main. The PR remains useful, but it changes an authenticated webhook parsing and acknowledgement boundary, so it needs Teams maintainer/security review before merge.

## What This Changes

This PR changes the Microsoft Teams webhook from `express.json()` to raw-body parsing with a two-pass JSON escape repair path, updates the lifecycle test mock, and adds regression coverage for malformed Bot Framework activity JSON.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase/review it on current main, preserve bearer/JWT validation before body parsing and the existing size limit, repair only truly bare invalid JSON escapes, return HTTP 200 only for intended authenticated Bot Framework parse failures, and add integrated middleware coverage before landing.
