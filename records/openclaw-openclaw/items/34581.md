## Summary

Keep this PR open. Current main still uses strict `express.json()` for the Microsoft Teams webhook and has no raw-body JSON escape repair path, while the PR targets the right Teams-owned webhook surface. It needs Teams maintainer review because it changes authenticated webhook parsing and acknowledgement behavior for malformed Bot Framework activities.

## What This Changes

The PR changes the Microsoft Teams webhook to parse raw JSON bodies, repair bare invalid JSON escape sequences before adapter processing, acknowledge unrecoverable authenticated JSON parse failures, update the lifecycle Express mock, and add JSON-repair regression tests.

## Best Possible Solution

Land a reviewed Teams-owned fix that preserves bearer/JWT validation before body parsing and the existing size limit, repairs only truly bare invalid JSON escapes, and has direct middleware coverage for repaired and unrecoverable authenticated Bot Framework payloads.
