## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams webhook bodies with `express.json()` and does not implement the PR's raw-body parse, JSON escape repair, or authenticated HTTP 200 acknowledgement for unrecoverable Bot Framework JSON. The PR addresses a plausible current Teams delivery/backoff failure, but it needs a rebase plus focused functional and security review before merge.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is to rebase it onto current main, preserve the current auth-before-body-parse hardening and `MSTEAMS_WEBHOOK_MAX_BODY_BYTES`, implement repair only for truly bare invalid JSON escapes, return HTTP 200 only after authenticated unrecoverable Bot Framework JSON, and add integrated regression coverage for the webhook behavior before landing.

## Risks / Open Questions

- Security-sensitive webhook behavior: the PR changes an internet-facing Microsoft Teams endpoint from strict JSON parsing to raw body parsing, repair, and HTTP 200 fallback, so review must preserve bearer precheck, Bot Framework JWT validation before body parsing, and the existing body-size limit.
- Functional risk: acknowledging unrecoverable JSON with HTTP 200 prevents Azure backoff but can also drop an activity silently; logs and tests should make that tradeoff explicit.
- Coverage risk: the PR should include integrated middleware/adapter tests for repair success, unrecoverable authenticated acknowledgement, adapter non-invocation, middleware ordering, and payload-size handling, not only copied helper behavior.
