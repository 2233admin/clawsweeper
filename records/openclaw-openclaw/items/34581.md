## Summary

Keep this PR open. Current main still uses the strict Microsoft Teams `express.json` webhook parser, forwards non-413 parse errors, and has no raw-body two-pass JSON escape repair or authenticated HTTP 200 fallback for unrecoverable Bot Framework JSON. The PR is an active, focused implementation candidate, but it changes a security-sensitive webhook parsing boundary and still needs Teams maintainer/security review plus integrated middleware coverage before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, review the Teams webhook security boundary, keep auth/JWT validation before raw parsing, preserve the size limit, repair only truly bare invalid JSON escapes, acknowledge unrecoverable authenticated Bot Framework JSON with HTTP 200 only where intended, and add integrated Express middleware regression coverage before landing.
