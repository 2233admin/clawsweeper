## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams Bot Framework webhook bodies with `express.json()` and has no raw-body parser, JSON escape repair helper, or authenticated HTTP 200 fallback for unrecoverable malformed JSON. The PR remains a focused implementation candidate, with useful follow-up fixes in the discussion for the regex edge case and test mock, but it still needs maintainer review/rebase against current webhook auth hardening and integrated regression coverage.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is to rebase it onto current main, preserve the current auth-before-body-parse hardening and body-size limit, implement repair only for truly bare invalid JSON escapes, return HTTP 200 only after authenticated unrecoverable Bot Framework JSON, and add integrated webhook regression coverage before landing.

## Risks / Open Questions

- The PR changes an internet-facing Microsoft Teams webhook from strict JSON parsing to raw body parsing and repair, so a rebase must preserve bearer precheck, Bot Framework JWT validation before body parsing, and `MSTEAMS_WEBHOOK_MAX_BODY_BYTES`.
- Returning HTTP 200 for unrecoverable JSON intentionally acknowledges and drops that activity; logging and tests should make that tradeoff explicit.
- The added PR test mirrors the helper directly; integrated middleware coverage is still needed for repair success, unrecoverable authenticated acknowledgement, adapter non-invocation, middleware ordering, and payload-size handling.
