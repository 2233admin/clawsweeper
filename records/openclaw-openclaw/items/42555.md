## Summary

Keep this PR open, but it is not merge-ready. Current main still hardcodes the Slack assistant-thread typing status, while the branch targets the removed `src/slack` shim path and exposes a Slack-only setting through core agent identity instead of the Slack plugin config surface.

## What This Changes

The PR adds an optional `identity.typingStatuses` config field and resolver, then changes Slack dispatch to send a randomly selected configured status instead of the hardcoded typing text.

## Best Possible Solution

Ship this through the active Slack plugin as a typed, validated Slack status/loading-message config contract, or define a generic channel status contract first if maintainers want per-agent personalities across channels. The implementation should route through `extensions/slack` status handling, reject empty entries, update Slack docs/generated config metadata, and cover default plus configured payloads in Slack monitor tests.
