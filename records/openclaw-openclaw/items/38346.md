## Summary

Keep open. Current main still does not provide a Telegram mode that observes or ingests all visible group messages while separately allowing only selected senders to trigger agent turns, commands, tools, or visible replies.

## What This Changes

The issue requests a Telegram group mode where the bot can observe or ingest all group messages for future context while only allowlisted users can trigger responses or actions.

## Best Possible Solution

Keep this open for bundled Telegram plugin product/security review. If approved, add an explicit opt-in observe/ingest-before-trigger mode for configured Telegram groups/topics, keep trigger authorization separate for agent turns, native/text commands, tool execution, and visible replies, and update Telegram config types/schema/docs plus regression tests.
