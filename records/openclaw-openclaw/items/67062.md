## Summary

Keep open. Current main covers the reporter's concrete `multimedia.nt.qq.com.cn` example via the default `*.qq.com.cn` QQ Bot media allowlist, but QQ Bot still has no supported `channels.qqbot.mediaAllowHosts` setting or runtime merge point. The remaining request is valid and security-sensitive because inbound and outbound fallback media downloads still use a fixed SSRF policy.

## What This Changes

Add `channels.qqbot.mediaAllowHosts` so QQ Bot media downloads can merge user-configured host allowlist entries with the default QQ/Tencent SSRF media policy.

## Best Possible Solution

Implement the option inside the bundled QQ Bot plugin with a conservative default policy plus a documented, normalized host-merge mechanism that is applied to both inbound attachment downloads and outbound URL fallback downloads. The config, schema, manifest metadata, generated channel metadata, docs, and focused tests should all move together.
