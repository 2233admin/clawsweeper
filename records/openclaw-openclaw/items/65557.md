## Summary

Keep this issue open. Current main has the chat/session model picker, a global `agents.defaults.models` allowlist, and some per-agent default/fallback routing, but it still does not implement account-scoped provider/model allowlists, account defaults, wildcard account policy, or account-aware server-side rejection. The related #7424 is still open and narrower per-agent work, not a substitute for this admin/account policy request.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is to define an effective account/user provider-model policy in config/schema, expose it in Control UI admin settings, filter `models.list` and the chat picker by that effective policy, and enforce the same policy in `sessions.patch`, `/model`, `chat.send`/run overrides, and OpenAI-compatible model override paths. Coordinate with #7424 so per-agent and per-account policy compose cleanly.
