## Summary

Keep this issue open. Current main has the chat/session model picker and a global `agents.defaults.models` model allowlist, but it still lacks account-scoped provider/model allowlists, per-account defaults, wildcard policy, and account-aware server-side rejection; the related #7424 is narrower per-agent allowlist work and does not supersede this account/admin policy request.

## What This Changes

Request provider and model selection in session UI plus admin-controlled per-account provider/model allowlists, per-account defaults, global fallback defaults, and server-side rejection for disallowed selections.

## Best Possible Solution

Keep this open for maintainer design of a single effective provider/model policy that combines global defaults, account/user restrictions, and any per-agent restrictions, then expose it in config/admin UI and enforce it consistently in catalog listing and every model override path.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. Source, docs, and tests give high-confidence verification that current main has only global model allowlisting plus the session picker, not the requested account-scoped policy.

## Solution Assessment

Unclear until maintainers define the account identity model and precedence rules. A UI-only filter would not be sufficient; the maintainable solution needs server-side enforcement shared by `models.list`, `sessions.patch`, chat/run override paths, and OpenAI-compatible overrides.
