## Summary

Keep open. Current main still evaluates the image, video, and music provider lists before the shared resolver can return a complete explicit modelConfig, and the paired open PR #74097 is already the intended implementation path for this issue.

## What This Changes

The issue asks OpenClaw to defer image, video, and music capability provider listing until model-config candidate resolution actually needs providers, avoiding unnecessary plugin registry work when explicit modelConfig is complete.

## Best Possible Solution

Review and land #74097, or an equivalent narrow patch, so resolveCapabilityModelConfigForTool accepts a provider thunk, invokes it only after complete explicit modelConfig fails, updates the three media generation callers, and adds regression coverage for both the complete explicit-config and incomplete candidate-resolution paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with complete agents.defaults image/video/music modelConfig, the three callers still evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the explicit config.

## Solution Assessment

Yes. The lazy provider thunk proposed by the paired PR is the narrowest maintainable fix because it preserves provider-based candidate resolution when modelConfig is incomplete while avoiding provider discovery when explicit config is already complete.
