## Summary

Keep open. Current main still eagerly evaluates the image, video, and music provider lists before the shared resolver can return a complete explicit modelConfig, and the paired implementation PR #74097 is still the tracked fix candidate in the provided GitHub context.

## What This Changes

Request to defer image/video/music capability provider listing during media tool model-config resolution so complete explicit modelConfig can short-circuit without plugin-provider discovery.

## Best Possible Solution

Land #74097 or an equivalent narrow change that passes provider thunks into the shared resolver, invokes them only after explicit modelConfig fails, and preserves provider-based candidate resolution when modelConfig is incomplete.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with complete agents.defaults image/video/music modelConfig, the three callers still evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the explicit config.

## Solution Assessment

Yes. A lazy provider thunk is the narrowest maintainable fix because it avoids unnecessary plugin capability discovery on the explicit-config path while keeping the existing fallback/candidate behavior intact.
