## Summary

Keep open. Current main still eagerly computes image, video, and music provider arrays before the shared explicit-modelConfig short-circuit can run, and the open paired implementation PR #74097 still references this issue with closing syntax, so this should remain open until that PR merges or is replaced.

## What This Changes

The issue requests lazy provider listing for image, video, and music generation tool model-config resolution so complete explicit defaults do not trigger unnecessary plugin capability-provider discovery each turn.

## Best Possible Solution

Land #74097 or an equivalent narrow change that makes the shared media model-config resolver accept a provider thunk, invokes it only when explicit config is incomplete, updates the three media tool callers, and keeps regression coverage for both the explicit-config and candidate-resolution paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: each create*GenerateTool path calls a resolver that eagerly evaluates listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return a complete explicit modelConfig.

## Solution Assessment

Yes. The lazy provider thunk proposed by the paired PR is the narrowest maintainable fix because it preserves provider-based candidate resolution when modelConfig is incomplete while avoiding discovery work when explicit config is already complete.
