## What This Changes

This PR preserves an explicit empty `skills.allowBundled` array, treats it as blocking all bundled skills while leaving workspace skills unaffected, and adds a prompt regression test.

## Best Possible Solution

Land a rebased version of this minimal config fix with the regression test, preserving the documented distinction between an omitted bundled allowlist and an explicitly empty bundled allowlist.

## Reproduction Assessment

Yes. Current-main source gives a deterministic reproduction: `skills.allowBundled: []` normalizes to `undefined`, then bundled skills are allowed because `isBundledSkillAllowed()` treats `undefined` as unrestricted.

## Solution Assessment

Yes, after rebase. Preserving empty arrays through normalization and checking only `undefined` as unrestricted is the narrow maintainable fix for the documented contract; the branch just needs to be refreshed against current main.
