## Summary

Close as ClawHub/plugin-scope rather than bundled core. The branch adds a new bundled Vesicle channel, but current main already has the external channel plugin, setup-entry, install, and ClawHub/npm paths needed for this integration, and the repo vision says optional plugins should usually be hosted outside core unless there is a clear product, security, or missing-API reason. The diff does not identify a missing core extension seam, and it also includes invalid CLI examples in the new docs.

## What This Changes

The PR adds a bundled `extensions/vesicle` channel plugin for Vesicle native iMessage health probes and outbound text sends, plus config/secret metadata, focused tests, generated channel metadata, lockfile/sidecar registration, and channel docs.

## Best Possible Solution

Keep OpenClaw core unchanged for this integration and publish Vesicle as an external channel plugin through ClawHub or npm using the existing channel plugin SDK. If external publication uncovers a genuinely missing generic SDK seam, that seam should be proposed separately as a small, plugin-neutral core change.

No. Adding Vesicle as another bundled core channel is not the narrowest maintainable path under the current plugin strategy; an external channel plugin is the better fit, with only generic SDK gaps returning to core if needed.
