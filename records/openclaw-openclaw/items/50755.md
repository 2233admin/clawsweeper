## Summary

Keep this PR open. Current main does not implement per-agent session maintenance overrides: agent entries still do not accept a maintenance config, the runtime resolver still reads only global session.maintenance, and cleanup/save paths still use global maintenance unless a caller passes a fully resolved override. The tracking issue #52775 being closed is not enough to close this PR because the PR’s intended behavior is not present on main.

## Best Possible Solution

Keep the PR open and review or update it as the implementation candidate for per-agent session maintenance. The best path is to add the agent maintenance schema/type surface, resolve overrides from a single loaded config snapshot with normalized agent IDs, thread the resolved policy through normal session writes and cleanup targets, define safe behavior for shared stores and explicit --store cleanup, and add regression tests plus docs/schema updates. If maintainers no longer want this product surface, close it manually with that product decision rather than as cleanup.
