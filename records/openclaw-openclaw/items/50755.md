## Summary

Keep open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still does not implement this PR’s intended per-agent session maintenance overrides. The config schema/type/docs expose `session.maintenance` as global-only, the resolver has no `agentId` path, and the store/cleanup paths still resolve global maintenance policy. The closed tracking issue #52775 does not supersede this PR because the behavior remains absent on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish this PR or replace it with a maintainer-approved implementation that adds `agents.list[].maintenance` schema/type/generated-schema/docs support, resolves per-agent overrides from one normalized config snapshot, threads the selected policy through normal session writes and cleanup, handles shared stores and explicit `--store` safely, preserves legacy `pruneDays` precedence correctly, supports intentional disk-budget disabling semantics, normalizes agent ids consistently, and adds regression tests for those cases.
