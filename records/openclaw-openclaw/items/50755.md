## Summary

Keep open. Current main still does not implement the PR's intended per-agent session maintenance overrides: `agents.list[]` has no `maintenance` schema/type field, maintenance resolution is global-only, and cleanup/store paths still use the global `session.maintenance` policy. The closed tracking issue #52775 does not make this PR obsolete because the behavior is still absent on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either finish the feature on this PR or supersede it with a narrower maintainer-approved implementation that adds `agents.list[].maintenance` schema/type/docs support, resolves per-agent maintenance from a single normalized config snapshot, defines safe behavior for shared stores and explicit store paths, threads the selected policy through normal write/load and cleanup paths, and adds regression tests for destructive cleanup behavior.
