## Summary

Keep open. Current main still performs a single Feishu message resource download with the supplied `image|file` type and has no 5xx retry, `type=media` fallback, or video-file heuristic for iOS photo-library videos sent as file messages. The linked bug #49855 remains open in the provided GitHub context, and this PR is still a plausible implementation candidate rather than obsolete. Security review found no CI, dependency, release, publishing, or secret-handling changes in the provided diff.

## Best Possible Solution

Keep this PR open as a Feishu bug-fix candidate, or consolidate it with #50164/#49855 after maintainer review. The eventual fix should prove the correct Feishu download strategy for iOS file-type videos, cover real 5xx error shapes and production call sites with targeted tests, and avoid unsupported SDK parameter assumptions.
