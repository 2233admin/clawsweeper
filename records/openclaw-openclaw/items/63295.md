## Summary

Current main should keep this open. Bare `/new` and `/reset` now return a local acknowledgement before model execution, so the original bare-prompt repro path has changed, but the requested sender-envelope behavior is still not implemented for reset prompts that do reach the model. Solving the original personalized greeting request now needs a maintainer product decision because it conflicts with current static-ack behavior and tests.

## Best Possible Solution

Maintainers should decide whether bare `/new` and `/reset` are intended to stay local acknowledgements or should support a model-generated first reset greeting. If model-generated reset turns remain supported, update `src/auto-reply/reply/get-reply-run.ts` so reset prompts that reach the model include `inboundUserContext`, then add focused coverage for reset-tail or soft-reset sender metadata plus Telegram/native command context. If static acknowledgement is intended, document that personalized bare reset greetings are not supported and route customization through the broader reset-prompt work.
