## Summary

Keep open. Current main still lacks the OpenRouter hooks this branch proposes, and the reported dynamic-model defaulting problem remains plausible, but the PR has blocking correctness gaps around reasoning visibility and fresh-cache behavior before it can merge.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports a cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the PR so dynamic OpenRouter capability metadata is available before thinking and reasoning defaults resolve, native OpenRouter reasoning blocks remain visible by default when intended, and any SDK/cache exposure is narrow, documented, and covered by cached plus fresh-cache dynamic-model tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:171-176`
  - body: Returning `"low"` for a reasoning-capable OpenRouter model makes `thinkingActive` true in `get-reply-directives.ts`, and current main then skips `resolveDefaultReasoningLevel()`. The subscriber only extracts native thinking when `includeReasoning` or `streamReasoning` is enabled, so the native `reasoning_content` this PR is trying to show can still be dropped unless the user sets `/reasoning` explicitly.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:307-308`
  - body: `getAllCachedOpenRouterModels()` calls `ensureOpenRouterModelCache()` and immediately returns `cache ?? new Map()`. On a fresh machine, that only starts the OpenRouter API fetch in the background, so catalog augmentation can run with an empty map before `prepareDynamicModel()` later awaits the per-model warm-up; first use of an uncached dynamic model can retain the old off defaults.
  - confidence: 0.78
