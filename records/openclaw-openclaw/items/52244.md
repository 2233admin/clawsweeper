## Summary

Keep open. Current main has adjacent OpenRouter reasoning/default infrastructure, but it does not implement this PR's intended change for OpenRouter dynamic models: the OpenRouter plugin still lacks `augmentModelCatalog` and `resolveDefaultThinkingLevel`, the capability cache does not expose `getAllCachedOpenRouterModels`, and default reasoning still depends on catalog entries that dynamic models like `moonshotai/kimi-k2.5` do not have.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rework or rebase the contributor change so OpenRouter-owned cached capabilities can feed the model catalog/default reasoning path without exposing unnecessary SDK surface, then add focused coverage for a dynamic OpenRouter reasoning model that is absent from the static catalog.
