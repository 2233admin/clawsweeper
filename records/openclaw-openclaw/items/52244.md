## Summary

Keep open. Current main still does not implement the proposed OpenRouter dynamic-model default fix, but the PR as written has blocking behavior gaps: inferred thinking can still leave reasoning visibility off, and the all-cached accessor can return an empty catalog on a fresh cache before default resolution runs.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports an all-cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the PR so dynamic OpenRouter capability metadata is available before thinking and reasoning defaults are resolved, preserve the intended reasoning visibility behavior in the embedded/TUI path, and keep any SDK export narrow, documented, and covered by fresh-cache and cached dynamic-model tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` here for dynamic OpenRouter reasoning models makes `thinkingActive` true before the model reasoning fallback runs. Since `getReplyDirectives()` only calls `resolveDefaultReasoningLevel()` while thinking is inactive, and the embedded subscriber only extracts native thinking when `reasoningMode` is `on` or `stream`, the reasoning blocks this PR is trying to surface can still be dropped by default.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking `ensureOpenRouterModelCache()`, so a fresh machine with no disk cache returns an empty map while the API fetch runs in the background. Default resolution happens before `prepareDynamicModel()` awaits the per-model warm-up, so first use of an uncached dynamic reasoning model can still default off.
  - confidence: 0.78
