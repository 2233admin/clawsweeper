## Summary

Keep this PR open. Current main still routes ordinary tool-card side-panel content through markdown, and the shared markdown renderer still collapses JSON code fences into a disclosure, so the PR is addressing a real remaining UI behavior rather than duplicating the earlier full-command fixes.

## What This Changes

The branch adds a dedicated `tool` sidebar content variant, renders tool-card input and output as expanded sidebar blocks with raw fallback, and adds focused UI tests and CSS for that view.

## Best Possible Solution

Land a reviewed version that keeps generic markdown JSON collapse intact while giving tool cards a dedicated, escaped sidebar renderer with regression coverage for expanded input, output, empty states, and raw fallback.

## Reproduction Assessment

Yes. On current main, a tool card with JSON input or output is serialized into markdown and then rendered through the JSON-collapsing markdown path, so the side panel can be checked with a small tool-card fixture rather than a live gateway run.

## Solution Assessment

Yes. A dedicated `tool` sidebar content kind is narrower and safer than disabling JSON collapse globally, and the diff preserves the existing canvas/markdown routes while adding focused tests for the tool route.
