## Summary

Keep this PR open. Current main does not implement the requested OpenAI Responses replay id normalization, and the PR now covers both history replay and the live stream wrapper, but the latest patch still has a correctness issue: malformed non-`fc_` item ids can be normalized into `fc_*` ids after the existing no-reasoning downgrade has already run.

## What This Changes

The PR adds an OpenAI Responses replay tool-call id normalizer, wires it into `sanitizeSessionHistory` and the live stream replay wrapper, adds regression tests, and updates transcript hygiene docs.

## Best Possible Solution

Normalize malformed OpenAI Responses ids before pi-ai conversion while preserving the existing reasoning-pair rule: after dropping stale reasoning, normalize malformed/overlong ids, then strip `fc_*` item ids when replayable reasoning is absent. Cover both persisted-history replay and same-run stream continuation cases.

## Reproduction Assessment

Yes. #74665 provides a deterministic payload-capture repro, and the PR-specific ordering issue can be reproduced with the added no-reasoning `call_...|notfc_...` fixture: the PR returns `call_*|fc_*` even though current OpenAI replay policy strips item ids when reasoning is missing.

## Solution Assessment

No. Normalizing before pi-ai splits ids is the right direction, but applying the normalizer after `downgradeOpenAIFunctionCallReasoningPairs` lets malformed item ids bypass the existing no-reasoning downgrade; reorder the passes or rerun the downgrade after normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve the no-reasoning item-id downgrade:** `src/agents/pi-embedded-runner/replay-history.ts:654-660`
  - body: The new normalizer runs after `downgradeOpenAIFunctionCallReasoningPairs`. For a malformed no-reasoning id like `call_...|notfc_...`, the downgrade leaves the suffix alone, then normalization turns it into `|fc_*`, so pi-ai will send `function_call.id` even though the existing OpenAI policy strips `fc_*` ids when replayable reasoning is absent. Normalize before the pair downgrade, or run the downgrade again after normalization, and update the new no-reasoning tests to expect the item id to be omitted.
  - confidence: 0.84
