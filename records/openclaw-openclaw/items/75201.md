## Summary

Keep open. Current main still runs the two `runCliAgent` cleanup hooks sequentially in one `finally` block, so the PR addresses a real remaining cleanup-continuation gap after the merged timeout-fallback work. I found no blocking correctness or security finding in the proposed diff, but final CI was not fully settled at review time.

## What This Changes

The PR wraps CLI live-session and bundle MCP run-end cleanup in independent warning-only guards and adds a regression test that later cleanup still runs when live-session cleanup rejects.

## Best Possible Solution

Land a narrow cleanup-isolation change in `runCliAgent` so each run-end resource cleanup gets its own best-effort guard and diagnostics, with the added seam test covering continuation after a cleanup rejection.

## Reproduction Assessment

Yes. The high-confidence reproduction is a unit seam: call `runCliAgent` with both cleanup flags enabled, make `closeClaudeLiveSessionForContext` reject, and observe current main skip `closeMcpLoopbackServer` while the PR asserts it still runs.

## Solution Assessment

Yes. Independent guards around each cleanup hook are the narrowest maintainable fix for preserving later cleanup; aggregating and rethrowing cleanup errors would keep old failure semantics but would still risk losing the successful agent result for run-end cleanup.
