## Risks / Open Questions

- The PR changes effective Tailscale exposure behavior: when `preserveFunnel` is true and a matching external Funnel route is detected, OpenClaw skips applying Serve and also skips `resetOnExit` cleanup for that run. The PR documents/logs this, but it still needs maintainer product/security review.
- The parser depends on Tailscale `funnel status --json` shape, especially `AllowFunnel` and `Web.*.Handlers.*.Proxy`. The PR adds unit coverage and addressed bot review concerns, but the PR body says no live tailnet restart was verified.
- The PR updates generated config/schema/help surfaces but not `docs/gateway/tailscale.md`; maintainers may want a user-facing docs note before merge.
