Keep this PR open. Current main still unconditionally reapplies Tailscale Serve in serve mode, while the PR provides a narrow opt-in preserve path with parser/test/config coverage; the remaining action is maintainer review plus completion of exact-head checks.

## What This Changes

The PR adds optional `gateway.tailscale.preserveFunnel` config, Tailscale Funnel status parsing, serve-mode skip wiring, generated config help/schema updates, regression tests, and a changelog entry.

## Best Possible Solution

Review and land this PR after exact-head checks finish, preserving the password-only OpenClaw-managed Funnel policy while giving token-auth operators an opt-in way to keep externally managed Funnel routes across gateway restarts.

## Reproduction Assessment

Yes. The high-confidence reproduction is `gateway.tailscale.mode="serve"`, an externally configured Tailscale Funnel route for the gateway port, then a gateway restart; current main re-runs `tailscale serve --bg --yes <port>` with no existing-Funnel precheck.

## Solution Assessment

Yes. The proposed opt-in `preserveFunnel` path is the narrowest maintainable fix because it avoids broadening `mode="funnel"` auth policy and keeps default Serve/reset behavior unchanged.
