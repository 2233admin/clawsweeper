## What This Changes

The PR adds an optional `gateway.tailscale.preserveFunnel` setting, Funnel-status parsing, serve-mode skip logic, config schema/help/changelog updates, and focused regression tests, with a few incidental formatting/no-op edits outside the Tailscale path.

## Best Possible Solution

Land the narrow opt-in preserve-Funnel behavior after rebase and review, keep OpenClaw-managed `mode="funnel"` password-only, document the new option on the Tailscale Gateway page, and avoid bundling unrelated cleanup in the same merge.

## Reproduction Assessment

Yes. The high-confidence reproduction is `gateway.tailscale.mode="serve"`, an externally configured Tailscale Funnel route for the gateway port, then a gateway restart; current main statically re-runs `tailscale serve --bg --yes <port>` without checking existing Funnel state.

## Solution Assessment

Yes, with a docs/rebase cleanup caveat. The opt-in precheck is narrower than loosening the password-only OpenClaw-managed Funnel policy, and the helper fails closed to the existing Serve behavior when status parsing or command execution fails.
