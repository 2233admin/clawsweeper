## Summary

Keep this issue open because the reported reserve-token asymmetry is still present in current main, and an open replacement PR already targets the same compaction-gate bug but has not merged.

## What This Changes

The issue reports that memoryFlush and preflight compaction gates ignore configured `agents.defaults.compaction.reserveTokens` and compute thresholds from `reserveTokensFloor` instead.

## Best Possible Solution

Land PR #74010 or an equivalent narrow core fix that resolves one effective compaction reserve from configured `reserveTokens` plus `reserveTokensFloor`, uses it consistently for preflight and memoryFlush thresholds, and keeps regression coverage for `reserveTokens > reserveTokensFloor` and small-window threshold clamping.
