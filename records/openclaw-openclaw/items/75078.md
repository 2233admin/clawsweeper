The branch adds media-aware Feishu dedupe keys in the receive handler and final message handler, plus a regression test for two audio events sharing a message_id but using different file_key values.

## Best Possible Solution

Land a Feishu-owned media-aware dedupe key after fixing lint and targeted validation, ideally keeping the key computation shared or otherwise hard to drift between the receive handler and final handler.
