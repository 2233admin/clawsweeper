## Summary

Current main still preserves blank/non-inferable tool-call names and still assigns `call_auto_*` fallback ids, so the reported NVIDIA/Kimi failure shape remains. The issue is also paired with open closing PR #56391, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that NVIDIA Kimi K2.5 can emit blank tool-call names that become `call_auto_*` fallback ids and repeated `Tool not found` loops.

## Best Possible Solution

Keep this issue open and review/land PR #56391 or merge an equivalent maintainer-approved fix that converts blank or whitespace-only tool names without an inferable id into a deterministic non-blank failure sentinel before dispatch, with regression coverage proving the blank-name plus `call_auto_*` loop cannot recur. Keep parallel-tool-calls compatibility work separate.
