## Summary

Keep this PR open. Current main still generates the Windows self-restart helper with a task-existence query followed by retrying `schtasks /Run`, but it does not check whether the scheduled task is already running before issuing `/Run`. The CLI restart path still writes a restart intent, ends the scheduled task, and then runs it again, while a SIGTERM with that intent is treated as a gateway restart and can route through the same Windows self-restart helper. The PR is a focused one-file candidate for the reported race, though it still needs maintainer review for the remaining PowerShell/cmd robustness feedback.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a hardened version of the running-task guard in `src/infra/windows-task-restart.ts` with focused coverage for the already-running task case, localized Windows status behavior, task names containing shell-sensitive characters, and PowerShell profile suppression. Close only after current main contains an equivalent fix for the duplicate Windows scheduled-task `/Run` race.
