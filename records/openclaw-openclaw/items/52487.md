## Summary

Keep open. Current main still has the Windows scheduled-task duplicate-start race this PR targets, but the proposed probe still needs hardening because it embeds the task name in a generated batch/PowerShell command without cmd-safe handling and runs PowerShell with user profiles enabled.

## What This Changes

The PR inserts a PowerShell `Get-ScheduledTask` running-state probe into the generated Windows scheduled-task self-restart script before retrying `schtasks /Run`.

## Best Possible Solution

Keep the narrow Windows restart fix, but revise the generated helper to run a profile-free, noninteractive PowerShell state check and pass the task name through a cmd-safe variable/assignment or the same validation pattern used by the update restart helper. Add focused tests for default task names, unsafe/custom task names, and guard ordering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote or validate the task name before the probe:** `src/infra/windows-task-restart.ts:47`
  - body: The new probe embeds `taskName` directly in a `.cmd` line and only doubles PowerShell single quotes. Values from `OPENCLAW_WINDOWS_TASK_NAME` or profile-derived task names containing `%`, `!`, or `"` can be expanded or break the outer `-Command` string before PowerShell sees them, so the guard can miss an already-running task or run unintended batch text. Pass the value through a cmd-safe assignment or validate it like the update restart helper.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:47`
  - body: `powershell -Command` loads user and system profiles before `Get-ScheduledTask`. Profile output containing `Running` can make `findstr` take the cleanup path when the task is not running, and slow or interactive profile code can block the helper before it reaches `/Run`. Add `-NoProfile` and preferably `-NonInteractive` for this generated restart probe.
  - confidence: 0.82

## Security Review

A crafted local `OPENCLAW_WINDOWS_TASK_NAME` or profile-derived task name could alter the generated batch command when the restart helper runs, so the probe should pass the value through a safe assignment or reject unsafe names before embedding it.
