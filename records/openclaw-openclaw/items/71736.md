## Summary

Keep this open. Current main has the merged generic foundation (`registerControlUiDescriptor` plus `plugins.uiDescriptors`), but that surface is still narrower than the RFC: it only projects generic descriptors for session/tool/run/settings surfaces, with no first-class chat modes, input guards, event classifiers, approval renderer slots, status-surface vocabulary, or Control UI client consumption. The issue is also recent, active, and paired with the same-author open RFC PR #71731; 1fanwang's linked fork prototype is useful context but explicitly covers only a panel/status-surface slice.

## Best Possible Solution

Keep this issue open as the Control UI contribution-slot design thread. Maintainers should decide whether the shipped `registerControlUiDescriptor`/`plugins.uiDescriptors` foundation is the intended base, then either narrow this RFC into explicit follow-ups or implement the missing slot vocabulary and client behavior for chat modes, approval cards/renderers, event classifiers, input guards, status surfaces, plugin actions, disablement semantics, and sanitization fixture coverage.
