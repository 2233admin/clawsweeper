## Summary

Keep this open. Current main ships a useful generic `registerControlUiDescriptor`/`plugins.uiDescriptors` foundation, but it does not implement the broader Control UI contribution-slot vocabulary requested here, and the discussion is still active with open same-author follow-up PRs.

## What This Changes

Request to add safe, data-driven Control UI plugin contribution slots for chat modes, approval cards/renderers, event classifiers, input guards, status surfaces, and related plugin-owned UI actions.

## Best Possible Solution

Keep this as the Control UI contribution-slot design thread. Maintainers should decide whether the existing generic descriptor foundation is the base abstraction, then either narrow the RFC around that contract or extend it with explicit descriptor kinds and client behavior for chat modes, approval cards/renderers, event classification, input guards, status surfaces, plugin actions, disablement, and sanitization fixtures.
