## Summary

Keep open. Current main has adjacent Feishu fixes, but it does not implement the two requested behaviors: broadcast-only Feishu `@all` is explicitly not treated as a bot mention, and Feishu direct-message and group sessions remain separate by design. The configured Feishu broadcast path is opt-in and observer-style for non-active agents, so it is not equivalent to every Feishu bot/agent visibly receiving and replying to an `@all` group message.

## Best Possible Solution

Keep this issue open for maintainer product judgment and split it into two concrete decisions: whether Feishu should offer an explicit opt-in `@all` or multi-mention fan-out mode where every targeted bot/agent receives and replies once, and whether Feishu should add opt-in DM/group session linking or instead document that cross-surface continuity belongs in memory/session tools rather than shared transcripts. Coordinate the multi-bot portion with related Feishu cross-bot broadcast design work such as #43563 rather than closing this report prematurely.
