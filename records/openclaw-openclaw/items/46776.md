## Summary

Keep open. Current main still matches the reported behavior: Feishu broadcast-only `@all` is deliberately not a bot mention, configured broadcast dispatch only gives the active agent a real Feishu reply path while observer agents are silent, and direct versus group conversations still route to separate session keys. The related #43563 covers the multi-agent broadcast half, but it does not cover the requested direct/group context continuity.

## What This Changes

The issue reports that Feishu group `@all` or multi-agent messages do not reach or visibly wake every OpenClaw agent, and that the same agent does not share context between direct and group conversations.

## Best Possible Solution

Define two explicit maintainer-owned outcomes: an opt-in Feishu multi-agent fan-out mode coordinated with #43563, and a separate opt-in continuity model for direct/group context, likely through session linking or memory/session tools rather than automatic transcript sharing across all Feishu surfaces.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: broadcast-only Feishu `@all` parses as not mentioning the bot, mention-gated group handling returns before dispatch, and direct/group routing builds distinct session keys.

## Solution Assessment

Unclear. The current implementation is deliberate isolation and does not satisfy the requested workflow, but the narrowest maintainable fix depends on maintainer choices around visible fan-out, loop control, and privacy.
