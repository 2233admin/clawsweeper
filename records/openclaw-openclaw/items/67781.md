## Summary

Keep this PR open, but do not merge it as submitted. Current main still lacks a Google Vertex ADC marker-stripping wrapper and still returns no auto-reply output for some empty final payload paths, so the PR is directionally useful; however, the patch replaces the existing Google thinking stream wrapper and treats several intentionally empty post-delivery states as failures.

## What This Changes

The PR adds a Google provider stream wrapper and tests to strip the Vertex ADC auth marker, plus auto-reply runner guards and a regression test that turn empty or non-renderable final payloads into a friendly error.

## Best Possible Solution

Revise this PR, or replace it with a narrow maintainer-owned patch, so Vertex ADC marker stripping composes with the existing Google thinking wrapper and the auto-reply failure path only applies to truly terminal non-silent runs that produced no delivered output and no renderable final payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Compose the Vertex wrapper with Google thinking:** `extensions/google/provider-registration.ts:63-66`
  - body: The new provider-level `wrapStreamFn` is added after `GOOGLE_GEMINI_PROVIDER_HOOKS`, so it replaces `createGoogleThinkingStreamWrapper`. Non-Vertex Google calls now return `undefined` from the hook, and Vertex calls only strip the marker, so the existing Gemini thinking payload sanitizer is no longer applied. Compose the marker stripping with the existing wrapper instead of overwriting it.
  - confidence: 0.92

  - body: This branch treats every post-filter empty reply as `run_failed` when `silentExpected` is false. `buildReplyPayloads` intentionally returns an empty list after successful block streaming, direct block-key dedupe, and same-target messaging-tool suppression, so this can send a false error after the reply was already delivered or intentionally suppressed.
  - confidence: 0.9
