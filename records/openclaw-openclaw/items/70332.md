## Summary

Keep open. Current main still does not implement this PR's central Dream Diary hardening: request-scoped fallback still appends the first raw snippet or promotion, diary dedupe still fingerprints timestamp plus body and append still always inserts, and managed daily dreaming stripping still starts only from markers and treats any heading as a boundary. The PR is related to #70523, but not fully superseded by it, and the existing review concern around italic diary lines remains a real merge risk.

## Best Possible Solution

Keep this PR open for normal maintainer review and revision. The best path is to coordinate the fallback/raw-dump boundary with #70523, update Dream Diary fallback prose and body-only dedupe so only the timestamp line is ignored, add append-time duplicate prevention, make managed daily block stripping heading-level-aware including legacy heading-only sections, and preserve italic body lines in the fingerprint tests before landing.
