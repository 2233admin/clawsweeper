## Summary

Keep this issue open. Current main still does not enforce full absolute paths in user-visible assistant replies: shared display helpers still shorten home paths, skill prompt paths are intentionally compacted to `~/...`, final reply normalization sanitizes text without expanding paths, and tilde expansion is limited to host file-tool execution. The related #38495 PR is useful prior art, but it was closed unmerged and `OPENCLAW_EXPAND_PATHS` is absent from current main.

## Best Possible Solution

Keep this issue open. Maintainers should choose the product boundary, then implement a focused final-reply policy or formatter, or a documented config switch, with regression coverage that distinguishes user-visible final replies from intentional internal prompt compaction and accepted tilde tool inputs. #38495 remains useful prior art but should not be reused as a broad helper toggle without reviewing CLI/display blast radius.
