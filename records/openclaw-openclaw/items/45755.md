## Summary

Keep open. Current main still does not render the agent identity emoji inline with the assistant name in the Control UI chat message group footer/header. The backend identity path preserves `emoji`, and the Sessions table now uses it, but the chat UI normalizes/passes only assistant name and avatar into transcript rendering. The related attempted fix, #45898, was closed unmerged and targeted the agent selector dropdown rather than the chat message header called out in the issue discussion.

## Best Possible Solution

Keep this issue open and land a focused Control UI fix that preserves `identity.emoji` through bootstrap/RPC UI state and renders it inline with the assistant name in chat transcript group footers/headers, with targeted tests. Keep the existing Sessions-table emoji implementation, but do not treat it as satisfying this chat-message regression.
