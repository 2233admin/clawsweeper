## Summary

Keep this open. Current main still accepts `agentId` and `sessionKey` on hook mappings but drops those fields for `action: "wake"`, and open PR #70268 is an unmerged implementation candidate that explicitly targets this issue.

## What This Changes

The issue asks for wake-mode webhook mappings to honor configured `agentId` and `sessionKey` routing, or to reject/warn when those fields are unsupported instead of silently dispatching to the main session.

## Best Possible Solution

Review and land PR #70268 or an equivalent narrow fix that threads wake mapping `agentId` and `sessionKey` through mapping resolution, policy validation, dispatch-session normalization, and `dispatchWakeHook`, while preserving the existing main-session fallback and adding regression coverage for mapped wake routing and templated session-key guardrails.
