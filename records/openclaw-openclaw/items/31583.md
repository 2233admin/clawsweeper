## Summary

Current main still blocks the reported sensitive-pattern skill env keys before host exec can inherit them. An open PR, #50960, targets this exact root cause with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Ensure exec subprocesses inherit environment variables configured under `skills.entries.<skill>.env`, including sensitive names such as `GOG_KEYRING_PASSWORD`, during host-run skill execution.

## Best Possible Solution

Keep this issue open and review/land #50960 or an equivalent maintainer-reviewed fix. The likely code change belongs in `src/agents/skills/env-overrides.ts`: add explicitly configured `skills.entries.<skill>.env` keys to the sensitive-key allowlist before sanitization, while preserving the always-blocked dangerous host/runtime env policy. Add regression coverage for undeclared configured `_PASSWORD` and `_TOKEN` keys and verify host exec inherits the applied run-scoped environment.
