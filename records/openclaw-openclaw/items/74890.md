## Summary

Keep this open. Current main still has the macOS-owned write gap: the Swift direct writer replaces the config before treating `gateway-mode-removed` as suspicious, and local `ConfigStore.save` still falls back to that direct writer after a gateway `config.set` failure. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining repair is narrow and Swift-focused.

## What This Changes

The issue asks OpenClaw to prevent dashboard/macOS config writes from replacing a valid config with partial roots missing required envelope fields such as `meta` or `gateway.mode`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and `ConfigStore.save`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to seed a config with `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`. `ConfigStore.save` can route a stale or partial root into the same writer after local gateway `config.set` fails.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the macOS direct writer and local fallback while leaving the already guarded browser Control UI `baseHash` path and TypeScript gateway rejection path intact.
