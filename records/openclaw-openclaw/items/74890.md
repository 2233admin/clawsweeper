## Summary

Keep open. Current main still allows the macOS direct/local config writer path to replace the live config before treating `gateway-mode-removed` as suspicious, and `ConfigStore.save` can still fall back to that direct writer when the gateway `config.set` path fails.

## What This Changes

The issue asks for Dashboard/Control UI/macOS app config writes to preserve or reject required config envelope fields such as `meta` and `gateway.mode` instead of writing partial root snapshots.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is to harden the Swift direct-write/fallback path to preserve or reject required envelope fields, while leaving the already guarded browser Control UI and TypeScript gateway writer paths intact.
