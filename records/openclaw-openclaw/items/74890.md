## Summary

Keep this issue open. Current main still has a macOS direct/local config-write path that can commit a root config missing `gateway.mode`; the stronger browser Control UI and TypeScript gateway writer protections do not cover that fallback.

## What This Changes

The issue asks OpenClaw to prevent Dashboard/macOS config writes from stripping mandatory root envelope fields such as `meta` and `gateway.mode` before replacing `openclaw.json`.

## Best Possible Solution

Add a shared macOS pre-commit safety check around direct/local config saves: merge against a fresh config or reject writes that would remove required envelope fields before replacing `openclaw.json`, then cover `OpenClawConfigFile` and `ConfigStore` fallback behavior with Swift regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes the payload and only records `gateway-mode-removed` afterward.

## Solution Assessment

Yes, the requested direction is the right fix boundary. The safest implementation is a shared guard in `OpenClawConfigFile.saveDict` and/or the `ConfigStore` local fallback, while leaving the browser Control UI `config.set` base-hash path intact.
