## Summary

Keep this issue open. Current main still has the macOS-owned unsafe path: `OpenClawConfigFile.saveDict` writes the supplied root before treating `gateway-mode-removed` as an audit-only anomaly, and `ConfigStore.save` can still fall back to that direct writer after a gateway `config.set` failure. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining work is narrow and source-level actionable in the Swift local save path.

## What This Changes

The issue asks OpenClaw to prevent Dashboard/Control UI/macOS config writes from stripping required root envelope fields such as `meta` and `gateway.mode` and triggering last-good restore churn.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes for the remaining fix boundary. The narrowest maintainable solution is a macOS direct-writer/fallback change that preserves or rejects destructive envelope loss before disk replacement, while leaving the already guarded browser Control UI and TypeScript gateway writer paths intact.
