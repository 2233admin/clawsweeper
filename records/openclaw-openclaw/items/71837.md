## Summary

Keep this PR open. Current main still lacks the central REM-to-grounded promotion behavior, but the branch has concrete blocking review findings around synthetic query identity, confidence weighting, and REM meta filtering that should be fixed before merge.

## What This Changes

The PR updates memory-core dreaming and session extraction to strip wrapper scaffolding, expand REM denoise/bias logic, feed REM candidate truths into grounded short-term promotion signals, adjust deep promotion gates, and add focused tests.

## Best Possible Solution

Land a rebased implementation that preserves the useful scaffold denoise and REM-to-grounded promotion design while using stable synthetic REM query identity, confidence-scaled grounded signal strength, and complete meta stopword coverage before changed-gate validation.

## Reproduction Assessment

Yes. The branch diff itself gives a high-confidence reproduction path: selecting the same REM candidate on multiple day buckets produces multiple synthetic query hashes, marginal REM truths receive signalCount 3, and a `system` concept tag is no longer filtered from REM themes.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a stable REM query id:** `extensions/memory-core/src/dreaming-phases.ts:1856-1857`
  - body: The grounded REM path writes `query: __dreaming_rem__:${remDayBucket}`. Because `recordGroundedShortTermCandidates` hashes that query, the same candidate selected on multiple days gains distinct synthetic query hashes and can satisfy diversity gates without distinct recall contexts. Keep the synthetic query stable and rely on `dayBucket` for day-scoped dedupe.
  - confidence: 0.9
- **[P2] Scale REM grounded signal strength:** `extensions/memory-core/src/dreaming-phases.ts:1855`
  - body: Every REM truth that passes the 0.45 threshold is written with `signalCount: 3`, so a marginal truth and a high-confidence truth get the same grounded promotion weight. Derive the count from `calculateCandidateTruthConfidence(entry)` or use a lower floor for borderline candidates.
  - confidence: 0.82
- **[P2] Keep system out of REM themes:** `extensions/memory-core/src/dreaming-phases.ts:1467`
  - body: The PR adds `system` to `REM_POSITIVE_TAGS` but does not add it to `REM_META_TAGS`, while current main explicitly blacklists `system` reflection tags. That lets transcript/runtime meta language become a boosted REM theme, which is the opposite of this denoise change.
  - confidence: 0.84
