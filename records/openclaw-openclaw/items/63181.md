## Summary

Current main still has the reported prompt-contract mismatch: the system prompt section is titled `Current Date & Time` while its only payload is `Time zone: ...`. The separate `session_status` guidance and message-envelope timestamping help temporal grounding, but they do not remove or clarify the misleading section label that this issue targets.

## What This Changes

Rename or clarify the timezone-only system prompt section so cache-stable timezone injection is not presented as current date/time context.

## Best Possible Solution

Keep the timezone-only cache-stable behavior, but make the prompt contract honest: rename the section to `Time Zone` or add an adjacent instruction that current date/time is intentionally absent and should be fetched with `session_status`. Update the system prompt tests and date/time docs in the same PR.
