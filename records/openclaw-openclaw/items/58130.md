## Summary

Current main has a Control UI and an Agents tab, but the requested CoPaw-like centralized multi-agent management UI is not complete: the web UI can inspect/select agents and edit some per-agent settings, while agent add/delete and MCP registry management remain documented CLI/config surfaces. This is core Control UI product work, not a ClawHub auto-close.

## Best Possible Solution

Keep open and refine it into native Control UI work: define an OpenClaw-specific multi-agent/team management dashboard with create/delete agent flows, channel-account binding/unbinding, per-agent skill management, and MCP registry CRUD using existing gateway/config contracts where possible. Avoid treating this as ClawHub work because the requested surface is the built-in web Control UI.
