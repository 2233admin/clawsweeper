## Summary

Keep this PR open. Current main still lacks the no-root Feishu topic MessageThreadId fallback and the outbound thread reply propagation this PR is trying to add, but the branch needs maintainer review because main has since added a native payload-send path that is not covered by the same replyInThread logic.

## What This Changes

The PR updates the Feishu plugin to persist message_id as MessageThreadId for no-root topic-thread sessions and to propagate replyInThread through outbound threadId fallback paths, with focused Feishu tests.

## Best Possible Solution

Keep the PR alive but request a rebase/update: preserve the guarded message_id fallback for topic sessions with replyInThread enabled, apply one shared trimmed replyInThread decision across Feishu text, markdown-card, structured-card, native payload, media, and local-image sends, and add regression coverage for the native payload path before merging.
