## Summary

Keep open: the PR addresses real current-main `/v1/responses` client-tool gaps, and the earlier final-response snapshot regression called out in review appears fixed on the latest PR head. ## What This Changes

The PR changes the `/v1/responses` client-tool path to terminate Pi’s tool batch on client handoff, emit function-call argument delta/done SSE events, parse upstream argument `.done` events, and add regression tests plus a changelog entry.

## Best Possible Solution

Keep the implementation direction, add the required changelog credit, update the public OpenResponses SSE event list for the new function-call argument events, then run the targeted agent/gateway tests and normal changed gate before merge.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction path: client-tool stubs return a pending result without Pi's terminate hint, and streamed `/v1/responses` function calls lack the dedicated argument events strict parsers consume.

## Solution Assessment

Mostly yes. `terminate: true` matches Pi's batch termination contract and the added delta/done events match the locked OpenAI Responses stream types; the remaining issues are release-note/docs hygiene, not a different code approach.
