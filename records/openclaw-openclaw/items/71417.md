## Summary

Keep open. Current main still has the reported silent auto-resume behavior for `openclaw agent --agent <id>` when no explicit fresh-session control is supplied, and neither the CLI/protocol surface nor the docs expose a `--new-session`/`--fresh` flag, resume banner, or structured resumed metadata.

## What This Changes

Request to make `openclaw agent` fresh-session intent and default session resume behavior discoverable when `--session-id` is omitted.

## Best Possible Solution

Ship a backward-compatible fresh-session path: keep default resume behavior, add `--new-session` with a `--fresh` alias, surface clear resume metadata/banner when an existing session is reused, and document/test the behavior across Gateway and `--local`.

## Repair Work Prompt

Fix the silent resume behavior for `openclaw agent` conservatively. Add `--new-session` with `--fresh` as an alias to the `agent` command, plumb a `forceNewSession` (or equivalently named) boolean through `AgentCliOpts`, `agentViaGatewayCommand`, Gateway `AgentParamsSchema`, the Gateway agent handler, and local embedded `agentCommand`/`resolveSession` paths. When the flag is set, do not reuse an existing fresh `sessionEntry.sessionId` for the resolved session key; generate a fresh session id and treat the run as a new session while preserving the existing session key and current transcript/store safety rules. When an existing session is reused without the flag, surface structured metadata such as `resumed: true`, reused `sessionId`, `sessionKey`, and last-activity/start timestamp where available, and print a one-line resume banner to stderr from CLI-facing paths without contaminating `--json` stdout. Preserve the current default auto-resume behavior for compatibility; do not flip the default to fresh, do not alter delivery-channel routing semantics, and do not regress the related explicit `--session-id` behavior from #71415. Update `docs/cli/agent.md`, `docs/tools/agent-send.md`, focused CLI/Gateway/local tests, and add a single-line changelog fix entry crediting @mayank6136 unless a maintainer says no changelog is needed.
