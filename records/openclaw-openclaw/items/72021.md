## Summary

Keep open. Current main still gates promotion with aggregate signalCount, which includes recall, daily, and grounded signals, and the provided context shows open PR #74114 explicitly targets this issue but has not merged. The issue should remain open until that PR lands or maintainers choose a different memory-promotion semantic.

## What This Changes

The issue requests that memory-core short-term promotion stop treating daily/session signal counts as real recall counts for promotion gating and scoring semantics.

## Best Possible Solution

Review #74114 as the canonical implementation path. If maintainers accept recall-only gate semantics, land that change with matching tests and then decide whether score thresholds, promote-explain output, and docs need follow-up calibration.
