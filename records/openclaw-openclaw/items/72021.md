## Summary

Keep open. Current main still gates `minRecallCount` with aggregate `recallCount + dailyCount + groundedCount`, and #74114 is an open unmerged PR with closing syntax for this issue, so the issue should remain open until maintainers review, land, or reject that PR.

## What This Changes

The issue asks for short-term memory promotion to distinguish real user recalls from daily/session/grounded aggregate signals when applying promotion gates and scoring.

## Best Possible Solution

Use #74114 as the active implementation thread: decide whether `minRecallCount` should mean real user recalls only, then land a recall-only gate with aligned tests/docs/diagnostics and any needed score calibration, or document that aggregate signals are intentionally allowed threshold evidence.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and an existing test show a `recallCount: 0`, `dailyCount: 3` candidate can pass the default promotion gates. The exact reporter score values require their recall-store fixture or a new equivalent fixture.

## Solution Assessment

Unclear. A recall-only `minRecallCount` gate is the narrowest code fix for the false-candidate half of the report, but it changes current tested daily/grounded semantics and should be reviewed through #74114 before merging.
