## Summary

Keep this PR open. Current main still awaits bounded primary model prewarm inside the `sidecars.channels` startup span before calling `startChannels()`, so the PR’s intended non-blocking behavior is not implemented on main. The PR is focused, active, and the prior review discussion only found a changelog-attribution issue that the author says was fixed in the second commit.

## Best Possible Solution

Review and land this PR, or an equivalent gateway startup change, after normal targeted validation. The remaining product behavior is clear: keep primary model prewarm best-effort and bounded, but schedule it in the background so Slack and other chat channels do not wait on model discovery/prewarm latency.
