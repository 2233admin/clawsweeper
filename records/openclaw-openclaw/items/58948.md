## Summary

Keep PR #58948 open. Current main still does not strip fenced code blocks in shared `stripMarkdown`, and BlueBubbles still calls that helper before sends, so the user-visible fenced-backtick problem is not implemented on main. The PR is a plausible focused fix, but it still needs maintainer/reviewer follow-up because its final regex would drop inline triple-backtick spans such as `Run ```npm i``` now`. The broader #56283 overlaps but remains open and does not supersede this focused BlueBubbles/stripMarkdown fix yet.

## Best Possible Solution

Keep the PR open and ask for a narrow update: add fenced-code stripping to the shared `stripMarkdown` path used by BlueBubbles/TTS/LINE, but constrain the fence matcher to real block fences, preserve inline triple-backtick spans, and add regression coverage for exec approval payloads plus adjacent inline-span behavior. Maintainers can later decide whether to land this focused helper fix independently or fold it into the broader plain-text-channel pipeline in #56283.
