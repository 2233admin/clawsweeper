Keep open. Current main still lacks the channel MCP stdout isolation fix, and the production change matches the existing plugin-tools stdio pattern; the main caveat is that the added test covers tools/list over an in-memory transport rather than proving stdout isolation.

## What This Changes

The PR routes channel MCP server logging to stderr before config loading and adds an in-memory tools/list capability test for the channel MCP server.

Land the scoped channel MCP stderr routing on current main, ideally with a regression assertion that `routeLogsToStderr()` runs before runtime config loading or that real stdio stdout remains JSON-RPC-only; keep the linked #68587 bug paired until the fix merges.
