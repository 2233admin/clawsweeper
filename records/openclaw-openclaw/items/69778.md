## Summary

Keep open. Current main still resumes restart-aborted subagent sessions solely from active run membership plus `abortedLastRun`, without any age bound or `agents.defaults.taskRecovery.maxAgeMinutes` config, and regression/docs still preserve recovery for stale aborted runs. No open PR referencing this issue was found in the read-only search.

## What This Changes

The issue requests a bounded gateway recovery policy so restart-interrupted CLI subagent tasks older than a configured age are marked lost instead of being automatically resumed.

## Best Possible Solution

Implement a bounded restart-aborted subagent/task recovery policy in the core recovery path: add a documented config knob with a conservative default, keep fresh interrupted work recoverable, mark stale interrupted run/task records terminal as lost without synthetic redispatch, log a concise startup/recovery skip summary, and update docs plus regression coverage for both stale-skip and fresh-resume behavior.

## Repair Work Prompt

Treat #69778 as the canonical bug. Implement bounded recovery for gateway-restart-aborted CLI/subagent work so old prompts are not automatically redispatched after a long outage. Add a configurable max age under the agent/task config surface, default it conservatively (about 120 minutes unless existing config conventions suggest a better name/value), and apply it where restart-aborted subagent orphan recovery decides whether to synthesize a resume. Fresh interrupted runs should still recover; stale interrupted runs should become terminal/lost without calling `callGateway`/synthetic resume. Keep task registry, subagent run registry, cancellation, announcements, and maintenance coherent, and log a concise skipped-stale recovery summary if the local logging surface supports it. Update docs at `docs/tools/subagents.md` and `docs/automation/tasks.md`; update generated config docs/schema only through the repo generators if required by the normal config workflow. Add a user-facing changelog Fixes entry crediting `@FelixRen-FY` if this repo currently expects changelog entries for bug fixes. Do not implement durable partial generation resume, do not change unrelated task retention cleanup, and do not weaken recent restart recovery.
