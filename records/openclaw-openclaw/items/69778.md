## Summary

Keep open. Current main still resumes restart-aborted subagent work from active run membership plus `abortedLastRun` without an age bound, and current tests/docs continue to treat stale restart-aborted runs as recoverable. The newer discussion adds a second real-world incident with side-effecting channel delivery, while the related #74864 report is complementary rather than a duplicate fix for this age-bound bug.

## What This Changes

The issue requests an age-bounded gateway recovery policy so old interrupted CLI subagent tasks are marked lost instead of being resumed automatically after restart.

## Best Possible Solution

Add a bounded restart-aborted recovery policy in the core subagent/task recovery path: keep fresh interrupted work recoverable, mark stale interrupted run/task records terminal as lost without synthetic redispatch, expose a documented config default, and add regression coverage plus docs for both stale-skip and fresh-resume behavior.

## Reproduction Assessment

Yes. A high-confidence reproduction is to persist an active subagent run older than the proposed threshold with a child session marked `abortedLastRun: true`, restart the registry/gateway, and observe the synthetic resume path; current tests already preserve similar stale recovery for a three-hour-old run.

## Solution Assessment

Yes. An age gate at the orphan-recovery decision point, with coherent task/run terminalization and config/docs/tests, is the narrowest maintainable fix; filtering only docs or task restore would leave the redispatch path intact.

## Repair Work Prompt

Treat #69778 as the canonical bug. Implement bounded recovery for gateway-restart-aborted CLI/subagent work so old prompts are not automatically redispatched after a long outage. Add a configurable max age under the agent/task config surface, default it conservatively around 120 minutes unless existing config conventions suggest a better name/value, and apply it where restart-aborted subagent orphan recovery decides whether to synthesize a resume. Fresh interrupted runs should still recover; stale interrupted runs should become terminal/lost without calling `callGateway` or sending the synthetic resume message. Keep task registry, subagent run registry, cancellation, announcements, and maintenance coherent, and log a concise skipped-stale recovery summary if the existing logging surface supports it.
