## Summary

Keep open. Current main still recovers restart-aborted subagent sessions without an age bound, and regression coverage still preserves a 3-hour-old aborted restored run for recovery. The related #66909 was a docs clarification, not a bounded-recovery implementation.

## Best Possible Solution

Implement a bounded recovery policy for restart-aborted subagent/task recovery: keep recent interrupted work recoverable, but mark or finalize stale interrupted work as lost/failed without redispatching it. The age check should cover the synthetic resume path, be configurable through the agent/task config schema, update docs, and include regression tests for both stale-skip and fresh-resume behavior.

- Start in `src/agents/subagent-orphan-recovery.ts` and the restore/sweep paths in `src/agents/subagent-registry.ts` / `src/agents/subagent-registry-helpers.ts`; make sure the direct synthetic resume path cannot fire for old runs.
