## Summary

Keep open. Current main still auto-recovers restart-aborted subagent sessions without an age bound, and the tests explicitly preserve/recover a 3-hour-old aborted restored run. The related #66909 was only a docs clarification and does not supersede this concrete safety bug.

## Best Possible Solution

Keep the issue open. The best fix belongs in the subagent/task recovery owner path: add an age-bound recovery policy, likely under `agents.defaults.subagents` or a generic task recovery config, mark stale interrupted runs/tasks `lost` instead of dispatching synthetic resume prompts, surface operator-visible skip counts, and update docs/tests so short restart recovery remains possible while old prompts do not auto-execute.
