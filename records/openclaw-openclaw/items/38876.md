## Summary

Keep open. Current main still has the string-only `extractToolText` path, so embedded `toolresult` / `tool_result` blocks with array-shaped `text` or `content` still produce empty output. This PR remains the focused contributor path, but the current PR diff should still get maintainer/security review for bounded joining before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Amend or review it to extract array-shaped `text` and `content` fragments with explicit bounds, keep the focused regression tests in `ui/src/ui/chat/tool-cards.node.test.ts`, and land it with the changelog credit once the security concern is resolved or accepted.
