## Summary

Keep this PR open. Current main still has the Web UI tool-card gap the PR targets: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped `content[0].text` payloads still fall through as empty output. The PR remains the active implementation path and is not implemented on main or superseded by the closed unmerged related PR.

## Best Possible Solution

Keep this PR open as the focused implementation path. Review the bounded-output concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, preserves regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and keeps contributor credit in the changelog.
