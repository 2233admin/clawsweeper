## Summary

Keep this PR open. Current main still has the Web UI tool-card bug the PR targets: embedded `toolresult` / `tool_result` blocks only extract output when `text` or `content` is already a string, so array-shaped `content[0].text` payloads still render as empty output. The PR is the active contributor path and has recent maintainer repair/merge-ready context, not an obsolete duplicate.

## Best Possible Solution

Keep this PR open as the canonical implementation path. Review the bounded-output concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, preserves regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, and keeps contributor credit in the changelog.
