## Summary

Keep this PR open. Current main still has the Web UI tool-card gap: embedded `toolresult` / `tool_result` blocks only extract string `text` or string `content`, so array-shaped `content[0].text` still falls through to the `No output` sidebar fallback. The PR remains the active, maintainer-updated implementation path, with regression tests added on the branch; it is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open as the focused implementation path. Review the bounded-output concern, then land this branch or an equivalent maintainer patch that extracts and joins array-shaped `text` and `content` fragments in `ui/src/ui/chat/tool-cards.ts`, preserves the added regression tests in `ui/src/ui/chat/tool-cards.node.test.ts`, and keeps the contributor-credit changelog entry. Do not close it based on the closed linked bug or closed-unmerged #38228.
