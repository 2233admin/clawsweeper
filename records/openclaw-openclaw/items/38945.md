## Summary

Keep open. The Unicode tokenizer bug still exists in current main's active `memory-core` implementation, while this PR still changes the obsolete `src/memory/*` memory files. The PR is useful source work, but it needs to be ported into `extensions/memory-core` and cover the shared search planner tokenizer too.

## What This Changes

This PR updates the old `src/memory` MMR and FTS tokenizer helpers and tests to NFC-normalize input and match Unicode letters, combining marks, numbers, and underscores.

## Best Possible Solution

Port the narrow Unicode/NFC tokenizer fix into `extensions/memory-core`: preserve existing CJK unigram/bigram MMR tokens, add Unicode letter/mark/number tokenization and NFC normalization, update both FTS tokenizer paths including `manager-search.ts`, add focused regressions for accented Latin, combining-mark scripts, and NFC/NFD equivalence, then land the updated PR or a narrow replacement with credit to @jarimustonen.
