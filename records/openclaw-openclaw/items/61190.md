## Summary

Keep open. Current main still resolves directory matches with the initially detected target kind instead of the matched directory entry kind, so this PR’s central fix is not implemented. The PR is focused and matches a plausible routing bug for directory-backed channel plugins, including the documented community DingTalk plugin path, but the latest `channel` to `group` normalization still needs maintainer judgment because current outbound session routing distinguishes those target kinds.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the directory-entry-kind fix and regression coverage after deciding whether `entry.kind === "channel"` should preserve `channel` for outbound routing or intentionally normalize to `group` with a documented contract.
