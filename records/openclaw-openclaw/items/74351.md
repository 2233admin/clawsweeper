## Summary

Current main still blocks IPv6 ULA DNS answers even when `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` is true. The report is valid, but changing this would broaden an SSRF policy boundary and needs maintainer/security review before an automated fix.

## What This Changes

The issue requests that the web fetch SSRF fake-IP proxy exemption cover IPv6 ULA fake-IP responses, or expose a separate ULA exemption, so `web_fetch` works through dual-stack fake-IP proxies.

## Best Possible Solution

Keep this issue open for maintainer/security design. The safest direction is likely a separate, default-off IPv6 ULA fake-IP opt-in with schema/type/docs/audit/cache/test coverage, or an explicit decision to keep ULA blocked and document that `allowRfc2544BenchmarkRange` intentionally covers only `198.18.0.0/15`.
