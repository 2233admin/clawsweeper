## Summary

Keep open. Current main still has the reported split-state behavior: `OPENCLAW_STATE_DIR` is honored by the general state-dir resolver, but the exec-approval file and socket defaults still resolve from hard-coded `~/.openclaw` paths. The earlier ClawSweeper comment is consistent with the current source and docs inspection.

## What This Changes

The issue asks for `exec-approvals.json` and its related state to follow `OPENCLAW_STATE_DIR` instead of always using `~/.openclaw`.

## Best Possible Solution

Make exec approval file and socket default resolution state-dir-backed via `resolveStateDir()`, preserve explicit `socket.path` values, add a legacy read or migration plan for existing home-based approval files, and update the approval docs plus regression tests so doctor and storage semantics agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR=~/openclaw`, `resolveStateDir()` resolves the override, while `ensureExecApprovals()` and `saveExecApprovals()` still write through `resolveExecApprovalsPath()` to `~/.openclaw/exec-approvals.json`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is to move only the default exec approval file/socket locations under the active state dir, with an explicit compatibility plan for the legacy home-based file.
