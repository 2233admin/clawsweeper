## Summary

Keep open. Current `main` still has the reported split-state behavior: `OPENCLAW_STATE_DIR` is honored by the shared state-dir resolver, but exec approval file/socket defaults still resolve through hard-coded `~/.openclaw` paths. The two related fix PRs linked from the timeline are both closed unmerged, so there is no implemented or pending canonical fix to close against.

## What This Changes

The issue reports that `exec-approvals.json` ignores `OPENCLAW_STATE_DIR` and is written under `~/.openclaw`, causing split-state doctor warnings.

## Best Possible Solution

Make the default exec approval file and socket path derive from `resolveStateDir()` while preserving explicit `socket.path` values, add a deliberate legacy read or migration path for existing `~/.openclaw/exec-approvals.json`, and update tests plus approval docs so doctor and storage semantics agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR=~/openclaw`, `resolveStateDir()` resolves the override, while `ensureExecApprovals()` and `saveExecApprovals()` still write through `resolveExecApprovalsPath()` to the hard-coded home-based file.

## Solution Assessment

Unclear until maintainers decide compatibility semantics. The narrow maintainable direction is state-dir-backed defaults plus an explicit legacy policy/socket-token migration plan, not a broad config or doctor-only workaround.
