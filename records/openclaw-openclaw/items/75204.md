## Summary

Keep open. Current `main` still has the reported split-state behavior: the shared state resolver honors `OPENCLAW_STATE_DIR`, but the exec approval file/socket defaults, read/write helpers, tests, and docs still point at `~/.openclaw`, and the related fix PRs were closed unmerged.

## What This Changes

The issue reports that `exec-approvals.json` ignores `OPENCLAW_STATE_DIR` and is written under `~/.openclaw`, creating split state directories.

## Best Possible Solution

Make the default exec approvals file and socket derive from `resolveStateDir()` when no explicit path is configured, preserve explicit `socket.path` values, safely read or migrate an existing home-based approvals file, and update focused tests plus the public exec approvals docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR` set, `resolveStateDir()` resolves the override, while `ensureExecApprovals()` and `saveExecApprovals()` still write through the home-based `resolveExecApprovalsPath()`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is state-dir-backed defaults plus explicit legacy handling so existing host exec policy is not silently lost.
