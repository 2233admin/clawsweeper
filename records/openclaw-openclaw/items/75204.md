## Summary

Keep open. Current main still has the reported split-state behavior: `OPENCLAW_STATE_DIR` is honored by the shared state resolver, but exec approval file/socket defaults and related policy reporting still point at `~/.openclaw`; the three related fix PRs were closed unmerged.

## What This Changes

The issue requests that `exec-approvals.json` and related exec-approval state honor `OPENCLAW_STATE_DIR` instead of being written under `~/.openclaw`.

## Best Possible Solution

Route the default exec approval file and default socket path through `resolveStateDir()` while preserving explicit `socket.path` values, define a legacy read or migration path for existing home-based approval files, and update focused tests plus approvals docs so doctor, CLI, gateway, and node-host behavior agree.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with `OPENCLAW_STATE_DIR=~/openclaw`, `resolveStateDir()` resolves the override, while `ensureExecApprovals()` and `saveExecApprovals()` still write through `resolveExecApprovalsPath()` to `~/.openclaw/exec-approvals.json`.

## Solution Assessment

No current solution is in place. The narrow maintainable fix is state-dir-backed defaults plus explicit legacy policy preservation; changing only the constants is likely incomplete for existing installs.
