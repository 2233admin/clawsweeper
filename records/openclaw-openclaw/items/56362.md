## Summary

Keep this PR open. Current main still has the slash-only reset-hook fallback that the PR is trying to change, and the bang-trigger regression tests from the PR are not present on main. The PR also still needs maintainer review on the sessionFile path semantics before it is safe to land.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to land a narrow `get-reply.ts` fix that emits/reset-classifies hooks for all configured reset triggers, add focused Discord `!new` and `!reset` regression coverage, and either drop or rework the `session-reset-service.ts` change so custom `sessionFile` paths continue to preserve their intended location while still clearing archived history.
