## Summary

Keep open. Current main still leaves Moonshot/Kimi opted out of replay tool-call ID sanitization, so a WhatsApp group transcript containing repeated raw IDs such as `exec:26` can bypass the generic duplicate-ID rewrite. Adjacent fixes exist, and strict sanitization now preserves Kimi native IDs, which makes a focused Moonshot-safe fix feasible.

## Best Possible Solution

Keep this issue open and land a focused fix that makes Moonshot replay history duplicate-safe while preserving valid native Kimi tool-call IDs. The likely direction is to enable or add a Moonshot-safe replay sanitizer now that strict mode preserves Kimi IDs, with a regression using duplicated raw `exec:26` assistant/tool-result history on a Moonshot OpenAI-compatible policy and a WhatsApp group session context.
