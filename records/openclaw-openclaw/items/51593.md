## Summary

Keep open. Current main has adjacent duplicate-tool-call mitigations, WhatsApp group-session isolation, and Moonshot/Kimi native ID preservation, but it still does not prove the reported WhatsApp group + Moonshot/Kimi raw `exec:26` duplicate replay path is fixed. Moonshot currently opts out of the generic sanitizer that rewrites repeated raw tool-call IDs.

## Best Possible Solution

Add a focused regression for a WhatsApp group session using the Moonshot provider with replay history containing duplicated raw `exec:26` assistant tool-call IDs. If current Kimi now emits only native `functions.<name>:<index>` IDs and the replay passes, close as implemented with that proof; otherwise add a Moonshot-safe sanitizer/repair that preserves valid native Kimi IDs while rewriting or dropping repeated non-native raw IDs before replay.
