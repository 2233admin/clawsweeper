## Summary

Keep open. Current main still has the macOS companion update/config behavior described by the issue: signed builds default the Sparkle updater on, one default-on UI preference controls both automatic checks and automatic downloads, and the Swift config reader detects update-channel-only clobbers but returns the clobbered config instead of restoring the previous one. Core TypeScript config I/O now has recovery for this clobber shape, but that does not fully cover the companion app path.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split persisted update settings and UI so automatic checks, downloads, install-on-quit/restart, and beta-channel opt-in are explicit. Route companion config reads/writes through the core config I/O recovery path or port equivalent last-known-good restore behavior into OpenClawConfigFile, then add signed-build/update-channel and update-channel-only clobber regression coverage.
