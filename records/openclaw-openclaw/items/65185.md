## Summary

Keep this PR open. Current main still has the iOS manual-gateway TLS upgrade and mixed Swift auth precedence behavior the PR is trying to fix, and the PR is a plausible implementation candidate for #47887. It is not merge-ready because the submitted test block is outside the Swift test suite, the existing `.local` assertion still contradicts the proposed behavior, and the `.ts.net` plaintext allowance conflicts with current mobile tailnet/public transport guidance.

## What This Changes

The PR changes iOS manual gateway TLS host classification, adds Swift gateway security tests, and changes OpenClawKit GatewayChannel auth selection so explicit passwords suppress bootstrap-token auth.

## Best Possible Solution

Keep this PR as a focused candidate for the iOS/Swift part of #47887, but revise it before merge: place tests inside the suite, update the stale `.local` expectation, align private-LAN plaintext behavior with the shared host classifier and current Tailscale security policy, and either fix the TypeScript auth/setup-code counterpart in a coordinated PR or leave an explicit linked follow-up.
