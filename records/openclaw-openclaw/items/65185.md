## Summary

Keep this PR open. Current main still has the underlying iOS manual TLS and mixed-auth behavior, so the work is not obsolete, but the proposed patch is not merge-ready because it breaks the iOS test target, leaves a contradictory TLS assertion, and changes mobile plaintext WebSocket policy without matching the current security model.

## What This Changes

The PR changes iOS manual gateway TLS host classification, adds Swift gateway security tests, and changes OpenClawKit GatewayChannel auth selection so explicit passwords suppress bootstrap-token auth.

## Best Possible Solution

Revise this PR into a focused iOS/Swift fix that compiles, updates the stale TLS expectation, adds Swift auth-precedence coverage, and aligns the private-LAN versus tailnet cleartext boundary with the documented mobile security model; leave the TypeScript setup-code/client auth work linked as a separate follow-up unless it is handled here too.
