## Summary

Keep open. Current main has warning/detection and documentation for text-shaped tool calls, but it does not implement the requested strict retry/block/routing policy when a model fails to emit real structured tool calls; the related discussion also frames this as security-sensitive agent-loop design rather than stale cleanup.

## What This Changes

The issue requests stricter agent-loop handling so models cannot satisfy tool-required tasks by narrating or text-simulating tool calls instead of emitting real `tool_calls` protocol entries.

## Best Possible Solution

Define a maintainer-approved strict-tool policy that keeps the existing no-blind-text-execution rule, adds configurable block/warn/retry behavior for text-shaped tool attempts or clearly tool-required prompts, documents provider-specific limits, and covers the agent-loop, local-model, and hook-enforcement paths with focused tests.
