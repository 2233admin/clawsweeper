## Summary

Keep open. Current main has diagnostics, documentation, and local-model compatibility workarounds for text-shaped tool calls, but it still does not implement the requested strict protocol-boundary behavior: retrying, forcing, blocking, or routing when a model emits simulated tool-use prose instead of a structured tool call. The issue discussion also frames this as a security-sensitive tool-execution boundary, so it needs maintainer design before an automated fix is safe.

## What This Changes

The issue asks OpenClaw to detect model text that simulates tool use and enforce real tool invocation through structured tool calls, retries, or deterministic routing.

## Best Possible Solution

Define a maintainer-approved protocol-boundary policy: structured tool calls remain the only generic executable path, text-shaped tool calls are detected and surfaced, forced `tool_choice` retries are opt-in and provider/model scoped, and provider-specific compatibility exceptions are documented, hook-preserving, and covered by focused tests.
