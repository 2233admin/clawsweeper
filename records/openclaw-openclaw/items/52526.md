## Summary

Current main still appears to have the reported CLI result-surface mismatch: `agent --json` payloads are built from the raw CLI output before the CLI transcript write path runs `before_message_write`, and delivery serializes those existing payloads unchanged. The related open PR #56031 is by the reporter but is scoped to pi embedded runner `--expect-final`, so it should not close this CLI-specific issue.

## Best Possible Solution

Keep this issue open and fix the CLI result path so `agent --json` returns the same finalized assistant text that the transcript write path persists after `before_message_write`. A conservative fix would thread the finalized assistant message from the guarded write/persistence path back into the CLI run result, or re-read the last persisted assistant after the guarded write, then add a regression test that rewrites or blocks the final assistant message and asserts both session JSONL and JSON `payloads[].text` agree. Coordinate with #56031 only for shared helper ideas; it does not close this CLI surface.
