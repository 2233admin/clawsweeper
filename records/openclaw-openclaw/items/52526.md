## Summary

Keep open. Current main at 7e5c3753 still builds `agent --json` payloads from the CLI runner's raw output before the CLI transcript persistence path could provide a hook-finalized assistant message. The related open PR #56031 is scoped to the pi embedded runner `--expect-final` payload path, so it does not close this CLI-specific report.

## Best Possible Solution

Keep this issue open and fix the CLI result path so `openclaw agent --json` returns the same finalized assistant text that the transcript persistence path writes after `before_message_write`. A focused fix should thread the finalized assistant message from CLI transcript persistence back into the command result, or read the just-persisted assistant message after guarded persistence, then update JSON `payloads` and `meta.finalAssistantVisibleText` consistently. The pi embedded runner PR #56031 can be used for helper ideas, but it is not the canonical fix for this CLI path.
