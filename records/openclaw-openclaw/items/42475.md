## Summary

Keep this issue open. Current main still provides cost visibility and external/provider workaround paths, but it does not implement native per-agent daily/monthly spend budget config or a gateway/model-call enforcement gate that can warn, block, or pause before spend occurs.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The likely path is a documented agents.defaults and agents.list[] budget config, durable per-agent spend accounting or reservation before every model attempt/fallback, explicit warn/block/pause behavior, status and doctor visibility, and regression coverage across chat.send, OpenAI-compatible gateway endpoints, cron, heartbeat, subagents, concurrency, missing pricing metadata, and provider fallback chains. If maintainers prefer plugin-first, first add or document a blocking pre-model-call budget API plus status/doctor projection so an external plugin can enforce the same boundary reliably.
