## Summary

Current main still does not implement per-agent daily/monthly model-spend budget enforcement at the gateway. OpenClaw has cost visibility, provider quota/cooldown handling, disk-budget cleanup, and plugin hooks, but no accepted budget config, durable pre-spend ledger/reservation, or gateway/model-call gate that can warn/block/pause an agent before spending. The issue discussion’s ATXP, nornr.com, and vibe-billing comments are useful prior art and workarounds, not evidence that native enforcement exists; related #66012 is closed and focused on optimization/visibility rather than hard enforcement.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The likely implementation path is a documented agents.defaults and agents.list[] budget config contract, durable per-agent spend accounting or reservation before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway turns, cron, heartbeat, subagents, concurrency, missing pricing metadata, and provider fallback chains. If maintainers choose plugin-first, add or document the supported blocking pre-model-call budget API and projection surfaces first.

## Risks / Open Questions

- A transcript-scan-only implementation would likely race concurrent runs, retries, cron, heartbeat, subagents, and fallback chains; hard enforcement probably needs a durable per-agent ledger or reservation before each model attempt.
- Maintainers still need product decisions for approximate pricing, missing cost metadata, zero-cost/subscription providers, media/tool spend, provider pricing drift, and daily/monthly reset boundaries.
- A plugin-first direction may be viable only if it deliberately covers the blocking pre-model-call path plus status/doctor projection; current main does not ship that as a complete budget feature.
