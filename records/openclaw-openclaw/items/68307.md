## Summary

Keep this PR open. Current main still has the reported gap: the commands system-prompt path calls the shared bootstrap resolver without a warning callback, while adjacent embedded runner and CLI paths already pass `makeBootstrapWarn`; the PR is a narrow implementation of that missing wiring and is not obsolete.

## What This Changes

This PR imports the existing bootstrap warning helper and subsystem logger in the commands system-prompt builder, then passes a WARN-level truncation callback into `resolveBootstrapContextForRun`.

## Best Possible Solution

Land this narrow wiring change, or an equivalent main change, so command/system-prompt bootstrap truncation uses the same `makeBootstrapWarn` WARN-level logging path as the embedded runner, compaction, and CLI runner surfaces, with targeted coverage if maintainers want an explicit regression test.
