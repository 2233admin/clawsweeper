## Summary

Keep open. Current main still lacks the PR's chain-wide `.error`/`.cause` context-overflow detection, but the latest PR revision still has a blocking security issue: the embedded-runner path opts into raw nested messages and then uses the matching raw segment as diagnostic text that is logged.

## What This Changes

The PR adds an error-chain message collector and uses it for context-overflow and failure classification across model fallback, embedded overflow compaction, and auto-reply handling, with related tests plus small system-event trust and changed-lanes/MiniMax test updates.

## Best Possible Solution

Revise the branch so raw chain candidates are used only for classification, while any diagnostic, log, reset-helper, stored failure, Control UI, or user-facing text comes from `formatErrorMessage()`, `redactSensitiveText()`, or an equivalent redacted display path. Keep chain-wide `.error`/`.cause` overflow detection and the billing/rate-limit precedence tests, and add a focused regression for nested overflow text that also contains a token.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep raw chain messages out of overflow diagnostics:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: `collectErrorChainMessages(promptError, { redact: false })` feeds the selected `errorText`, and that text is logged by the existing context-overflow diagnostic. A nested provider cause that both matches overflow and contains an Authorization header or API key will bypass `formatErrorMessage()`/`redactSensitiveText()` and leak into logs; use raw candidates only for matching, then return a redacted display string.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR still introduces a concrete secret-redaction regression in the embedded-runner overflow diagnostic path.

Concerns:

- **[high] Raw nested error messages can leak to logs:** `src/agents/pi-embedded-runner/run.ts:1296`
  - body: The embedded runner opts into raw chain messages for prompt-error overflow detection and then propagates the matching raw segment as diagnostic text. Provider and transport errors can include bearer tokens, API keys, or serialized request metadata in nested `.cause` or `.error.message` fields, so this should stay classifier-only or be redacted before surfacing.
  - confidence: 0.9
