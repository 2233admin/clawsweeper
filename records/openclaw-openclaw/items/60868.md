## Summary

Keep this PR open. Current main has partial nested-error support, but the runtime overflow paths still depend on one formatted message and do not implement the PR’s full cycle-safe `.error` / `.cause` chain inspection behavior.

## Best Possible Solution

Keep the PR open for maintainer review or a focused revision. The best implementation is a shared, cycle-safe candidate helper, likely reusing or extending `collectErrorGraphCandidates()`, that lets model fallback, embedded-runner overflow recovery, and auto-reply classification inspect `.error` and `.cause` segments without losing billing/rate-limit precedence or structured `code`/status hints, with targeted regression tests for Error instances, object causes, transport-shaped `.error` payloads, and mixed billing/overflow chains.
