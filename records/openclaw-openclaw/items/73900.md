## Summary

Keep open. Current main still preserves the reported routing path: cron forces `target: "last"`, the gateway adapter shallow-merges that override over configured heartbeat defaults, and outbound heartbeat resolution treats inherited `heartbeat.to` as an explicit destination before using the bound session route. The issue is also paired with open closing PR #73968, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `sessionTarget: "main"` cron `systemEvent` heartbeat wakes can inherit global heartbeat destination fields and send Telegram topic reminders to a DM instead of the bound topic route.

## Best Possible Solution

Keep this open and land #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main-session `systemEvent` wakes, with regression coverage proving a topic-bound session stays on its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and observed logs, and the current-main source path statically reproduces the merge and explicit-destination precedence without needing a live Telegram run.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` wakes is the narrowest maintainable direction; changing global heartbeat `target: "last"` semantics would be broader and riskier.
