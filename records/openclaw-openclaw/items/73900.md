## Summary

Keep open. Current main still preserves inherited heartbeat destination fields when cron forces `target: "last"`, and the issue is paired with open PR #73968, which should be reviewed, landed, replaced, or closed before this issue is closed.

## What This Changes

Fix cron main-session `systemEvent` heartbeat wakes so a forced `target: "last"` route does not inherit global heartbeat destinations and replies stay on the bound session route.

## Best Possible Solution

Keep this issue open and land #73968 or an equivalent narrow fix that clears inherited explicit destination fields for cron-forced `target: "last"` main-session `systemEvent` wakes, with regression coverage proving a topic-bound session stays on its group/topic route when global heartbeat defaults include a DM `to`.

## Reproduction Assessment

Yes. The issue provides concrete config, cron payload, session key, and observed logs; current main also statically reproduces the route by combining the cron `target: "last"` override, shallow heartbeat-default merge, and outbound `explicitTo` handling.

## Solution Assessment

Yes. Clearing inherited explicit destination fields only for cron-forced `target: "last"` main-session systemEvent wakes is the narrow maintainable fix; changing all heartbeat `target: "last"` semantics would be broader and riskier.
