## Summary

Keep open. Current main still routes main-session cron system-event reminders through the heartbeat `cron-event` prompt and `getReplyFromConfig(...)`, so the reported model-cost path is still present; the related PR #15403 is closed unmerged and addresses shell rc/env loading rather than this path.

## What This Changes

The issue asks OpenClaw to avoid using ChatGPT/OpenAI model calls as a string formatter for recurring cron/watch reminders that could be delivered deterministically.

## Best Possible Solution

Keep this open for maintainer product judgment, then either add an explicit cron option for deterministic/simple reminder delivery or narrowly detect clearly simple main-session reminder payloads and deliver them through existing outbound seams without a model call. Preserve the model-mediated heartbeat path for cron payloads that need reasoning, summarization, tools, or internal-only handling, and cover routing, event draining, target=last, and visibility behavior with focused tests.
