## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat `cron-event` prompt and `getReplyFromConfig(...)`, so the reported OpenAI/ChatGPT cost path remains present. The fix needs maintainer judgment because a broad deterministic bypass could skip intended reasoning, internal handling, or delivery-policy behavior for more complex cron events.

## What This Changes

The issue requests a cheaper deterministic formatting and delivery path for recurring cron/watch reminders instead of sending simple reminder text through ChatGPT/OpenAI model calls.

## Best Possible Solution

Define and implement a narrow low-cost delivery path for simple main-session cron reminders, preferably through an explicit cron option or conservative simple-payload detector, while retaining the existing model-backed `cron-event` path for complex, internal, tool-using, or reasoning-dependent events. Update the cron/heartbeat docs and focused tests to make that boundary clear.
