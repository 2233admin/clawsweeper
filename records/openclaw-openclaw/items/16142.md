## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported repeated OpenAI/ChatGPT cost behavior remains present.

## What This Changes

Request to stop using ChatGPT/OpenAI calls as a string formatter for repeated cron-driven Slack/email reminder delivery by adding a deterministic low-cost formatting path.

## Best Possible Solution

Introduce a narrowly scoped low-cost mode or conservative simple-reminder detector for main-session cron reminders, with docs and focused tests defining which events may be delivered deterministically and which must keep the current model-backed cron-event behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged cron system event, run heartbeat, and observe Provider cron-event plus a getReplyFromConfig call; existing tests assert this behavior.

## Solution Assessment

Unclear. Deterministic formatting is directionally right for simple reminders, but the maintainable fix needs a product-defined boundary so complex cron events keep current model-backed semantics.
