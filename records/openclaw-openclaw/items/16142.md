## Summary

Keep open. Current main still routes main-session cron system-event reminders through the heartbeat reply/model path, and the existing maintainer-visible review comment also kept this open for product follow-up. The related #15403 is closed unmerged and is about shell rc/env loading, not avoiding model calls for simple reminder formatting.

## Best Possible Solution

Keep this open for maintainer product judgment, then implement a narrow deterministic delivery path only for clearly simple main-session reminder payloads, or add an explicit cron payload/option that chooses deterministic delivery. Preserve model-mediated cron events for payloads that need reasoning or tools, and cover routing, event draining, delivery visibility, and target=last behavior with focused regression tests.
