## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat cron-event prompt and `getReplyFromConfig`, so the reported model-cost behavior remains present. The previous keep-open reviews still apply because there is no relevant diff from the last reviewed SHA to current main on the cron/heartbeat files or docs, and the remaining fix needs a product boundary for which reminders may bypass the model.

## What This Changes

The issue requests a lower-cost deterministic path for simple cron/reminder messages so OpenClaw does not repeatedly use the OpenAI-backed model as a string formatter.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, either as an explicit cron option or a conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Document the boundary and add focused cron/heartbeat coverage for both paths.
