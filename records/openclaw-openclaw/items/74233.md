## Summary

Keep this open. Current main still has the reported host-side behavior: context-engine assembly updates the active messages, but the embedded runner still passes the pre-assembly snapshot into the hard overflow precheck, and the helper still lets the larger unwindowed estimate win. The issue is also paired with open closing PR #74255, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Make context-engine assembled messages authoritative for prompt admission so raw pre-assembly history cannot hard-fail the overflow precheck unless the engine explicitly opts into that policy.

## Best Possible Solution

Land or adapt #74255 so successful context-engine assembly is prompt-authoritative by default, raw pre-assembly history becomes explicit opt-in or separate maintenance/debt telemetry, and the precheck tests/docs reflect that final contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction shape: provide a small assembled message set and a larger pre-assembly snapshot, then call the current precheck path and observe that the larger unwindowed estimate drives compaction.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix. A generic context-engine prompt-authority flag or default contract update is preferable to a lossless-claw-specific bypass or removing overflow protection entirely.
