## Summary

Keep open. Current main still contains the reported behavior, and the issue is already paired with open PR #74255, which uses closing syntax and proposes the host-side fix, so this should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that context-engine assembled messages become prompt-authoritative for overflow precheck admission instead of being bypassed by larger pre-assembly history.

## Best Possible Solution

Review and land or replace #74255 with a generic host-side fix that separates prompt-admission size from raw-history maintenance debt, updates the context-engine contract/docs, and changes the regression test so a small assembled prompt is admitted unless an engine explicitly opts into pre-assembly overflow authority.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by calling `shouldPreemptivelyCompactBeforePrompt()` with small assembled `messages` and larger `unwindowedMessages`; the current unit test already asserts the problematic `compact_only` outcome.

## Solution Assessment

Unclear until maintainer review of #74255. The proposed assembled-by-default plus explicit preassembly opt-in direction appears narrow and aligned with the documented assemble contract, but the docs’ full-transcript safeguard wording needs to be reconciled deliberately.
