## Summary

Current main still contains the reported behavior: context-engine assembly can replace the active session messages, but the embedded runner still passes the pre-assembly snapshot into the overflow precheck, and the precheck intentionally uses the larger estimate. This should remain open because open PR #74255 already references this issue with closing syntax and is the right implementation path to review before closing.

## What This Changes

Request to make context-engine assembled messages prompt-authoritative for overflow precheck, while separating raw-history pressure from hard prompt admission.

## Best Possible Solution

Keep this issue open and review PR #74255 as the canonical implementation candidate. If maintainers accept the contract direction, land or revise that PR so assembled context is prompt-authoritative by default, raw-history pressure is handled as a separate maintenance signal or explicit opt-in, and the current locking test becomes a regression for the small-assembled-prompt case.
