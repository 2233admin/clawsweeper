## Summary

Current main still contains the reported behavior: context-engine assembly can replace the active messages, but the embedded runner still passes a pre-assembly snapshot into the overflow precheck and the precheck intentionally uses the larger estimate. The issue should stay open because #74255 is an open closing PR for the same host-side fix.

## What This Changes

Make context-engine assembled messages authoritative for prompt overflow admission so raw pre-assembly history cannot force compaction/reset unless explicitly requested by the engine contract.

## Best Possible Solution

Review and either land #74255 or an equivalent generic fix that separates prompt-admission size from raw-history maintenance debt, updates the context-engine contract/docs, and replaces the current locking test with regressions for default assembled authority plus explicit raw-history opt-in.

## Reproduction Assessment

Yes. Current main has a focused source/test reproduction: `preemptive-compaction.test.ts` constructs a small assembled `messages` array with a larger `unwindowedMessages` array and expects compaction from the larger estimate; a runner-level reproduction would use a context engine that returns a small assembled prompt from an overflowing raw transcript.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: make assembled context prompt-authoritative by default while preserving an explicit opt-in for engines that want raw pre-assembly history to hard-gate overflow. The open paired PR appears to target that boundary, so the next step is normal maintainer review rather than closing this issue.
