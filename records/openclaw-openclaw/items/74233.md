## Summary

Keep open. Current main still snapshots pre-assembly history before context-engine assembly and feeds it into the hard pre-prompt overflow precheck, whose helper prefers the larger unwindowed estimate; open closing PR #74255 directly targets this and should be reviewed or landed before this issue closes.

## What This Changes

The issue requests that context-engine assembled messages be treated as the prompt-authoritative view for overflow precheck instead of hard-gating on larger pre-assembly history.

## Best Possible Solution

Land or adapt #74255 so successful context-engine assembly is prompt-authoritative by default, raw pre-assembly history becomes explicit opt-in or maintenance/debt telemetry, and the context-engine docs, public SDK contract, changelog, and regression tests match the final behavior.

## Reproduction Assessment

Yes. Source and test inspection give a high-confidence reproduction path: use a context engine that assembles a small under-budget message window while the runner still passes a larger pre-assembly snapshot to the precheck helper, whose current test asserts the larger unwindowed estimate wins.

## Solution Assessment

Yes for the requested direction, but current main is not solved. A generic context-engine contract path like #74255 is the narrow maintainable fix; a lossless-claw-specific bypass or removal of overflow protection would be less safe.
