## Summary

Keep this PR open. Current main has the scope-escalation protection, but it still uses the existing `null` / `{ ok: false, reason }` contracts rather than the structured error-object contract proposed here. The PR is not obsolete, but it needs rework before it can land because the provided review/diff shows parse-breaking edits and a public error-message side channel in revoke handling.

## Best Possible Solution

Keep the PR open, but ask for a focused rework: preserve or deliberately migrate the token result discriminators, add structured details only where callers can safely consume them, keep public rotate/revoke denial messages generic while logging detailed server-side context, fix the parser errors, and update the focused infra/gateway tests to cover both scope escalation and missing device/role cases.
