## Summary

Keep open. Current main still lacks a native `fallbackRunbook` config key, fallback-only system-prompt injection, and startup/doctor validation of configured fallback model refs, although normal bootstrap context and fallback observability are now present.

## What This Changes

The issue requests a fallback-only runbook field in agent model config plus diagnostics for invalid fallback model IDs.

## Best Possible Solution

Add a typed, documented fallback-only prompt field for default and per-agent model config, inject it only when the active candidate is a fallback while preserving normal bootstrap context, and add non-blocking doctor/models diagnostics for fallback refs missing from known provider catalogs.
