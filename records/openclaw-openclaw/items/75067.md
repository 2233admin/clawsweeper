## Summary

Keep this PR open, but do not merge it as-is. Current main still has the 60s pricing timeout and no disk cache/circuit breaker, so the work is not obsolete; however, the proposed disk-cache warm path can bypass current config pricing, and the circuit breaker does not schedule automatic retries at its stated cooldown. The PR also explicitly depends on the still-open #73486 pricing-deferral change, so it needs normal Gateway maintainer review and a rebase/coordination pass.

## What This Changes

The PR lowers Gateway model-pricing fetch timeout to 10s, adds a home-directory disk cache and in-memory circuit breaker for OpenRouter/LiteLLM pricing refreshes, adds `startupBlockedReason` to startup-pending readiness responses, and updates gateway tests.

## Best Possible Solution

Keep the resilience direction, but apply current config/manifest pricing before trusting disk cache, use remaining TTL/cooldown-aware scheduling for disk and circuit-breaker paths, and coordinate with #73486 so pricing refresh is not on the startup-critical path. Add targeted tests for config-pricing precedence with a fresh cache and fake-timer proof that automatic retry happens after the circuit cooldown.

## Reproduction Assessment

Yes for the review findings: the PR head shows the fresh-cache return before `collectSeededPricing`, and the circuit-open branch calls the 24h `scheduleRefresh` helper. Runtime tests were not executed because this sweep was read-only, but both issues are directly reproducible with focused unit tests using a fresh cache and fake timers.

## Solution Assessment

No. The proposed direction is reasonable, but this implementation is not the narrowest maintainable fix until disk-cache warming preserves current config precedence and the circuit breaker actually retries on its advertised cooldown.

Schedule the next attempt for the remaining circuit duration or use a helper that accepts the cooldown.
