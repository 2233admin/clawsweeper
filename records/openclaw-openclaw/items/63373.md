## Summary

Keep open. Current main still does not cover the exact reported uppercase square-bracket transcript marker shape, `[TOOL_CALL] ... [/TOOL_CALL]`, on the shared assistant-visible or Telegram final-delivery paths. Existing fixes cover adjacent XML tool-call leaks, bracketed local-model blocks with JSON plus `[END_TOOL_REQUEST]`/named closing tags, and WebChat synthetic repair-result filtering, but those do not close this Telegram leak report.

## Best Possible Solution

Keep this open and add a narrowly tested shared user-visible sanitization guard for internal square-bracket transcript markers such as `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]`, including the exact non-JSON Telegram payload shape from the report. The best code direction is a shared assistant-visible sanitizer update plus a final outbound-delivery defense-in-depth guard, with tests that preserve literal examples/code blocks and do not regress existing XML/local-model stripping behavior.
