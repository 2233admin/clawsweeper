## Summary

Keep open: current main still lacks a user-visible sanitization guard for the exact uppercase square-bracket transcript block shape reported for Telegram, and the related WebChat/XML fixes do not cover this delimiter family.

## What This Changes

Prevent raw internal `[TOOL_CALL]`/`[TOOL_RESULT]` transcript blocks from reaching Telegram or other user-visible delivery paths after transcript corruption or repair.

## Best Possible Solution

Add a narrowly scoped shared user-visible sanitizer for uppercase square-bracket internal transcript blocks, cover the exact non-JSON Telegram payload from the report, and add outbound delivery defense-in-depth without breaking literal code/prose examples or existing XML/local-model stripping behavior.

## Reproduction Assessment

Yes for the sanitizer gap: the exact reported `[TOOL_CALL] {tool => ...} [/TOOL_CALL]` shape is not matched by current assistant-visible, bracketed-block, Telegram, or final outbound sanitizers. The full live timing trigger is still unclear.

## Solution Assessment

Yes, the best fix direction is a shared assistant-visible sanitizer update plus a final outbound-delivery guard, with focused Telegram and shared sanitizer tests. A Telegram-only filter would leave the same internal marker family exposed on other delivery surfaces.
