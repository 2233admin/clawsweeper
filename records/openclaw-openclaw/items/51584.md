## Summary

Keep this PR open. Current main does not implement the intended fallback: `sandbox explain` still resolves inferred channels only through `normalizeAnyChannelId`, and that helper only checks the active runtime plugin registry. The PR is not obsolete; it needs maintainer review, regression coverage, and possibly a small adjustment for raw legacy `telegram:slash:<id>` CLI input.

## Best Possible Solution

Review and likely land an adjusted version of this PR for #51245: add a static bundled-channel fallback in `sandbox explain` active-channel inference, cover built-in channels when the runtime plugin registry is empty, and add regression tests for `agent:main:telegram:slash:<id>` plus raw `telegram:slash:<id>` if that CLI input remains supported.
