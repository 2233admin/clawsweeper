## Summary

Current main still lacks the requested TUI external-editor compose flow. The TUI editor has no Ctrl+E/onExternalEditor hook, runTui only suspends the TUI for the local auth flow, slash commands do not include an edit/editor/compose substitute, and the public TUI docs do not document $VISUAL/$EDITOR handoff. Related #55836 is adjacent vi/vim in-place editing, not a duplicate or shipped replacement.

## Best Possible Solution

Implement the feature in the core TUI composer: add a deliberate shortcut or command, suspend the TUI, write the current composer text to a temporary markdown prompt file, launch $VISUAL then $EDITOR and wait, load the edited content back into the composer without auto-submitting, show clear status/error messages, document the shortcut in the public TUI guide, and keep #55836 separate for vi/vim in-place editing mode.
