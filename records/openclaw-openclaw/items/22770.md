## Summary

Current main still does not implement the requested external-editor compose flow. The TUI editor has no Ctrl+E/external-editor hook, the runTui suspension path is only used for local auth, slash commands and docs do not expose an editor handoff, and the related #55836 issue is adjacent vi/vim input-mode work rather than a substitute.

## What This Changes

The issue requests an external-editor compose mode for the TUI that opens $VISUAL/$EDITOR with the current prompt text, then loads edited content back into the composer without auto-submitting.

## Best Possible Solution

Keep this issue open. After maintainer signoff on shortcut or slash-command UX and the local-process/temp-file security boundary, implement the core TUI composer flow: suspend the TUI, write current composer text to a temporary markdown prompt file, launch $VISUAL then $EDITOR and wait, read the edited content back into the composer without auto-submitting, show clear status/errors, and document the workflow in the public TUI guide. Keep #55836 separate for in-place vi/vim keybindings.
