## Summary

Keep this PR open. Current main still has the reported gateway-rpc timeout validation bug: malformed explicit `--timeout` values are coerced to `NaN` and then treated as unset by the gateway call timeout resolver. The PR remains an active implementation candidate, and the overlapping timeout-validation PRs are open rather than merged superseders.

## Best Possible Solution

Keep this PR open for maintainer review, landing, or explicit supersession. The best fix is strict positive-integer validation at the gateway-rpc CLI boundary, preserving the omitted-option default at 30000 ms, rejecting explicit empty/whitespace and partial numeric strings, and covering affected shared parser consumers with focused regression tests. If #40953 becomes the canonical broader fix, close this PR only after that path lands or maintainers deliberately supersede it while preserving contributor credit.
