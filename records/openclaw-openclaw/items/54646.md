## Summary

Keep this PR open. Current main still has the gateway-rpc timeout validation bug: the CLI option advertises a 30000 ms default, but the runtime still coerces `opts.timeout` with `Number(...)`, so malformed values can reach the gateway call path and fall back to 10000 ms. The latest PR context also shows maintainer follow-up pushing a narrow repair to this branch, so this remains an active implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open for maintainer review or deliberate supersession. The best implementation is strict positive-integer validation at the gateway-rpc CLI boundary, preserving the omitted-option default at 30000 ms, rejecting explicit empty/whitespace and partial numeric strings, and keeping focused regression coverage for gateway-rpc plus any shared parser consumers such as cron and status.
