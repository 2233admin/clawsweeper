## Summary

Keep open. Current main still drops image blocks from ACP tool call results: `extractToolCallContent()` only emits text content, and the ACP translator uses that helper for both partial and final tool updates. The PR is focused on the missing behavior and is not obsolete, though the remaining whitespace-only image payload review should be addressed before merge.

## Best Possible Solution

Keep this PR open and finish it in the ACP mapper: emit `ImageContent` from `extractToolCallContent()` for valid non-blank image blocks, preserve text and fallback behavior, add unit tests for text-only, image-only, mixed, missing, empty, and whitespace-only image fields, then run the targeted ACP mapper/translator tests before merge.
