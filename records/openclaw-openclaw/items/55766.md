## Summary

Keep this PR open. Current main still drops image blocks from ACP tool-call result content, and this PR is a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and finish it, or land an equivalent maintainer patch: add image-block forwarding in `extractToolCallContent()` for both partial and final ACP tool updates, validate `data` and `mimeType` as trimmed non-empty strings, preserve existing text and fallback behavior, add focused mapper tests for text/image/mixed/malformed/whitespace/fallback cases, and then run the targeted ACP mapper test plus the appropriate changed gate for this surface.
