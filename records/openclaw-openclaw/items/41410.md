## Summary

Keep this PR open. Current main still does not implement Telegram forum topic deletion through the message action surface, and the PR targets a real missing bundled Telegram plugin capability. It is not ready to close as implemented or merge as-is because the live PR remains dirty against main, includes stale pre-plugin-layout paths, and still has unresolved destructive-ID parsing review feedback.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to rebase/port it onto the current Telegram plugin layout: add `topic-delete` to the shared action contract and outbound target spec, expose it from `extensions/telegram/src/channel-actions.ts`, implement `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, add the `deleteForumTopic` runtime branch in `extensions/telegram/src/action-runtime.ts`, update Telegram docs and focused tests, and harden destructive ID parsing to decimal-only integers. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
