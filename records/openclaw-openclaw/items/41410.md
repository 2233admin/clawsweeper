Current main still does not implement this PR's intended Telegram forum-topic deletion support. The PR remains useful implementation work, but it needs a current-main port plus final safety cleanup around destructive ID parsing before it should merge.

## Best Possible Solution

The best path is to port the focused Telegram forum-topic deletion slice onto current main in `extensions/telegram/src/channel-actions.ts`, `extensions/telegram/src/action-runtime.ts`, and `extensions/telegram/src/send.ts`; add the accepted `topic-delete`/legacy delete routing surface to shared action metadata only if maintainers approve it; update `docs/channels/telegram.md`; and add targeted tests for gating, cfg/account/proxy pass-through, message-delete fallback behavior, and decimal-only explicit ID validation. Keep #11414 as the broader forum-topic lifecycle umbrella.
