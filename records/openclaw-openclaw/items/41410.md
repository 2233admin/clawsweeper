## Summary

Keep this PR open. Current main at d22a851253b0225e72847dc6b1a7b74d8035e214 still does not implement Telegram forum topic deletion through the message action surface: Telegram exposes `delete`, `topic-create`, and `topic-edit`, but no `topic-delete` or `deleteForumTopic` action, no `deleteForumTopicTelegram` transport helper, and no docs/tests advertising topic deletion. The PR remains a useful implementation candidate for the deletion slice of open umbrella #11414, but it still needs a current-main port and maintainer review of the destructive ID-parsing feedback.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to rebase/port it onto current main by adding a Telegram `topic-delete` or `deleteForumTopic` action path, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, updating the shared action contract only if maintainers want the public alias, documenting the user-facing behavior, and adding focused tests for message delete vs topic delete routing, delete gating, cfg/account pass-through, and decimal-only explicit ID validation. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
