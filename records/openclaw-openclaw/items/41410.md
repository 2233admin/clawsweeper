## Summary

Keep this PR open. Current main at db40ec404a9178294dba3121758ff59459983a11 still does not implement Telegram forum topic deletion: the shared message-action contract, Telegram action map, runtime branches, transport helper, docs, and tests all stop at message deletion plus topic create/edit. The PR is useful implementation work, but it needs a rebase/port off removed core files and one more destructive-ID hardening pass before it is merge-ready.

## Best Possible Solution

Keep this PR open as useful implementation work. Ask for a rebase/port to the current Telegram plugin-owned layout: add `topic-delete` to the shared action list and target spec, map and discover it in `extensions/telegram/src/channel-actions.ts`, implement `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, add a `deleteForumTopic` branch in `extensions/telegram/src/action-runtime.ts`, update Telegram docs and focused tests, and harden destructive ID parsing to explicit base-10 integer strings. Keep the broader close/reopen forum-topic management work tracked by #11414.
