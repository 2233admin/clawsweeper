Keep this PR open. Current main still does not implement dynamic GitHub Copilot text-model catalog discovery: the Copilot provider catalog returns an empty model list, the static default model table still assigns 128K/8192 limits, unknown Copilot models still use the conservative synthetic fallback, and xhigh thinking remains a hardcoded model-id allowlist. The PR remains a focused implementation candidate, but it needs maintainer review for the authenticated /models fetch, token-derived base URL handling, capability mapping, and transport selection.

## Best Possible Solution
