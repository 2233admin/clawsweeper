## Summary

Keep this PR open. Current main still does not implement authenticated GitHub Copilot text-model discovery from the Copilot `/models` API: the provider catalog returns an empty model list, static defaults still use 128000/8192 limits, unknown Copilot models still use the conservative synthetic fallback, and thinking levels still depend on a hardcoded xhigh allowlist. The PR is a relevant implementation candidate, but it needs maintainer review because it adds an authenticated provider network call, a process-wide model-capability cache, and new capability/thinking mapping semantics.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches the authenticated text-model catalog through provider-owned hooks, maps API limits/capabilities/transport/thinking profiles into OpenClaw model definitions, preserves conservative fallback behavior when discovery fails, scopes runtime caching correctly, and includes focused tests plus any user-facing changelog/docs updates maintainers require.
