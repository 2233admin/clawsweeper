## Summary

Keep this PR open. Current main still does not implement the intended dynamic GitHub Copilot text-model catalog: the provider catalog returns no runtime model rows, model listing falls back to static defaults, unknown Copilot models get synthetic 128K/8192 limits, and xhigh thinking remains hardcoded by model id. The PR remains a focused implementation candidate, with normal maintainer review needed for the authenticated /models fetch and capability mapping.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined implementation inside the GitHub Copilot plugin that fetches the authenticated /models catalog, maps capabilities into OpenClaw model definitions and thinking profiles, keeps the existing conservative fallback when discovery fails, and validates the new token/network path.
