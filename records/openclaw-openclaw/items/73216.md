## Summary

Keep this PR open. It is still open and unmerged, and both the checked-out main tree and live remote main still lack authenticated GitHub Copilot text-model discovery from the Copilot `/models` API. Current Copilot cataloging returns no text models, static defaults still use 128000/8192 limits, unknown models still fall back to a conservative synthetic model, and thinking levels still rely on a hardcoded xhigh allowlist. The PR is a relevant implementation candidate, but it needs maintainer review because it adds an authenticated provider network call, process-level model-capability caching, and a few unrelated SDK/test edits outside the stated Copilot scope.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches authenticated text-model catalog data through provider-owned hooks, maps limits/capabilities/transport/thinking profiles into OpenClaw model definitions, preserves conservative fallback behavior when discovery fails, scopes runtime caching to the relevant credential/baseUrl context, and includes focused tests plus any user-facing changelog/docs updates maintainers require.
