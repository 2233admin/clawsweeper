## Summary

Keep open. Current main does not implement this PR's intended dynamic GitHub Copilot model catalog: the bundled provider still returns an empty runtime catalog, relies on a synthetic unknown-model fallback with 128K/8192 defaults, and exposes thinking levels from a hardcoded Copilot xhigh allowlist. The PR is a plausible implementation candidate using existing provider hooks, with review needed for the new authenticated /models fetch and capability mapping.

## Best Possible Solution

Keep the PR open for normal maintainer review. If the feature is accepted, land a refined version that keeps the dynamic Copilot model catalog inside the GitHub Copilot plugin, uses the existing provider hooks, validates the authenticated `/models` fetch path, and preserves fallback behavior when auth or discovery fails.
