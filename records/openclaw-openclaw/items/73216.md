## Summary

Keep this PR open. Current main still does not implement authenticated GitHub Copilot text-model discovery from the Copilot `/models` API: the provider catalog still returns no text models, static defaults still use 128000/8192 limits, unknown Copilot models still use the conservative synthetic fallback, and thinking levels still use a hardcoded xhigh allowlist. The PR is a relevant implementation candidate, and the author already addressed two earlier Greptile findings in force-pushes, but it still needs maintainer review because it adds an authenticated provider network call, process-level capability caching, and unrelated plugin-SDK/test changes outside the stated Copilot scope.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land a refined GitHub Copilot plugin implementation that fetches authenticated text-model catalog data through provider-owned hooks, maps limits/capabilities/transport/thinking profiles into OpenClaw model definitions, preserves conservative fallback behavior when discovery fails, scopes runtime caching to the relevant credential/baseUrl context, and includes focused tests plus any user-facing changelog/docs updates maintainers require.
