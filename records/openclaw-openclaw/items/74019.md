## Summary

Keep open. Current main still matches the reported path: `llm-task` passes the caller config into a `disableTools: true` embedded run, the embedded runner intentionally builds an empty tool set, then the empty-allowlist guard still sees config-derived `tools.alsoAllow` / agent tool policy and can emit the reported `No callable tools remain` error.

## Best Possible Solution

Land a narrow embedded-runner fix: when a run intentionally disables tools for JSON-only/model-only execution, ignore config-derived allowlist sources for the empty-tool guard, while keeping runtime `toolsAllow` fail-closed and preserving normal `tools.alsoAllow` behavior for tool-enabled runs. Add focused regression coverage and a changelog fix entry crediting @technicallysecure.
