## Summary

Keep this issue open. Current main still has override-first context-token resolution in the live agent, auto-reply, gateway default, and session-persistence paths, and maxTokens is still resolved across provider/model and transport-specific code rather than by a shared conservative minimum resolver.

## What This Changes

The issue requests resolving effective contextTokens and maxTokens as the minimum of agent config, provider model config, discovery metadata, and safe defaults instead of letting higher-priority config values exceed model limits.

## Best Possible Solution

Add a shared positive-token-limit resolver for contextTokens and maxTokens, wire it through run setup, auto-reply, session persistence/status/defaults, and provider request construction, and document the conservative cap behavior so configured high defaults remain safe with smaller model limits.
