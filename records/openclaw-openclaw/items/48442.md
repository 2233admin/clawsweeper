## Summary

Keep open. Current main does not implement the PR's proposed `agents.defaults.systemPrompt.identityLine` or `disabledSections` surface, and the PR is still incomplete as an implementation candidate because the new config was not added to the strict runtime schema or threaded through production prompt builders. The related identity-preamble issue #43562 remains open, so the underlying product problem is still live rather than implemented on main.

## Best Possible Solution

Keep this PR open only as an implementation candidate, not as ready-to-merge. The best path is to update it against current main with a documented schema/help/generated-config contract for the chosen prompt customization surface, thread it through embedded, CLI, compact, and auto-reply prompt assembly, add validation plus runtime tests, and align it with the still-open identity-preamble issue #43562. Maintainers should also decide whether section disabling, especially `safety`, is an acceptable public config surface or whether the scope should be limited to a safer identity-line override.
