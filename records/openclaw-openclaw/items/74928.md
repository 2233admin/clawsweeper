## Summary

Keep open. Current main still has only phase-based assistant visibility metadata, and the latest release tag still lacks a public/internal assistant audience contract at the shared parser, schema, projection, and delivery-selection surfaces.

## What This Changes

Add a shared public/internal assistant audience contract and enforce it at visible-text extraction and channel delivery while preserving legacy phase fallback and silent-token behavior.

## Best Possible Solution

Define one shared assistant audience contract, parse and encode it alongside phase metadata, propagate it through provider stream/replay, transcript/history projection, OpenResponses surfaces, UI extraction, and channel delivery, then enforce public-only visible selection with an explicit legacy phase-only fallback.

## Reproduction Assessment

Yes. A synthetic assistant text block with `textSignature` JSON containing `v: 1`, an id, `phase: "final_answer"`, and `audience: "internal"` follows the current parser/extractor path because audience is ignored and final-answer text is selected as visible.

## Solution Assessment

Yes for the requested direction, but not as a quick delivery-only patch. The maintainable fix is a maintainer-reviewed shared contract across parsing, projection, schemas, and delivery so internal text fails closed before any user-visible surface sees it.
