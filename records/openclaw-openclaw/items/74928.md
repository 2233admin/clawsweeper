## Summary

Keep open. Current main still has phase-aware visible-text handling but no explicit public/internal assistant audience contract in the shared signature parser, provider stream encoders, replay/conversion path, or delivery extraction helpers. The reporter's follow-up package smoke evidence matches the current source-level gap, and the related commentary/phase issues do not supersede this broader delivery-boundary contract request.

## What This Changes

The issue requests an explicit public/internal assistant audience contract at visible-text extraction and channel delivery boundaries, including fail-closed handling for unknown audience values and silent suppression for exact NO_REPLY text.

## Best Possible Solution

Define one assistant audience contract in the shared message/signature layer, carry it through provider stream parsing and replay conversion, and enforce it at all user-visible extraction and delivery boundaries while preserving a documented legacy phase-only fallback for older transcripts.

## Reproduction Assessment

Yes. A source-level synthetic assistant message with textSignature JSON containing v: 1, an id, phase: "final_answer", and audience: "internal" reproduces the gap because current parsing ignores audience and visible extraction selects final_answer text.

## Solution Assessment

Unclear as a ready implementation, but the requested direction is sound. The narrowest maintainable solution is a maintainer-reviewed cross-boundary contract rather than a sanitizer-only patch, because audience must be encoded, parsed, persisted, replayed, and enforced consistently.
