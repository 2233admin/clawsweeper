## Summary

Keep open. Current main still implements a phase-only assistant visibility contract; it does not expose, preserve, or enforce a public/internal assistant audience boundary at the shared signature, provider/replay, history projection, or channel delivery layers. The reporter’s sanitized package smoke results match the current source-level behavior, and the related leak reports cover adjacent symptoms rather than superseding this broader boundary contract.

## What This Changes

The issue requests an explicit public/internal assistant audience contract so internal assistant text and silent sentinels cannot be selected for user-visible delivery.

## Best Possible Solution

Define a single assistant audience contract in the shared message/signature layer, carry it through provider stream parsing, replay conversion, transcript/history projection, and channel delivery, then enforce internal suppression and invalid-audience fail-closed behavior at every user-visible extraction boundary while preserving documented legacy phase-only fallback.

## Reproduction Assessment

Yes. A source-level synthetic assistant message with textSignature JSON containing v: 1, an id, phase: "final_answer", and audience: "internal" reproduces the gap because current parsing ignores audience and visible extraction selects final_answer text.

## Solution Assessment

Unclear as an implementation because no patch is proposed here, but the requested direction is the right class of fix. The narrow maintainable solution is a reviewed cross-boundary audience contract, not a delivery-only text scrubber.
