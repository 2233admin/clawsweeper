## Summary

Keep PR #59523 open. Current main still has the reported ClawDock gap: `_clawdock_compose()` explicitly includes only `docker-compose.yml` and optional `docker-compose.extra.yml`, while the Docker setup path can generate `docker-compose.sandbox.yml` when sandboxing is enabled. The PR’s one-line additive fix remains relevant and is not superseded by the related reports provided.

## Best Possible Solution

Keep the PR open or refresh it against current main, add the `docker-compose.sandbox.yml` conditional immediately after the existing `docker-compose.extra.yml` handling in `scripts/clawdock/clawdock-helpers.sh`, and add a focused regression check or documented manual verification that ClawDock commands include the sandbox overlay when it exists.
