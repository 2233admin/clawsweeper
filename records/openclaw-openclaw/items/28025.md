## Summary

Keep open. Current main still lacks `channels.slack.thread.replyOnToolUse` or an equivalent tool-use-gated Slack reply target; Slack replies are still routed by `replyToMode`, incoming thread state, and a reply plan built before tool lifecycle callbacks run.

## What This Changes

The issue requests a Slack config option that automatically sends tool-using agent responses to a thread while leaving purely conversational responses top-level.

## Best Possible Solution

Keep this open for maintainer product review. If accepted, implement it inside the Slack plugin as a default-false typed config option, carry a reliable per-turn tool-use signal into reply target planning before the first Slack transport action, preserve explicit thread replies and existing reply-mode semantics, update Slack docs and generated config metadata, and add focused tests for DMs, channels, existing threads, native streaming, draft preview streaming, progress callbacks, and reply-mode interactions.
