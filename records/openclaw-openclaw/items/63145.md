## Summary

Keep open. Current main still lacks the requested `models status --probe --all` behavior: the status CLI does not register or parse a status-level `--all`, the command options do not carry a `probeAll` flag, and the probe planner still selects one model per provider. The related #63259 PR is an open implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Add `openclaw models status --probe --all` to send minimal live requests to every configured primary, fallback, image, and relevant per-agent model, then report per-model health.

## Best Possible Solution

Land #63259 or an equivalent focused fix that adds `--all` to both CLI entry paths, threads a `probeAll` option through `modelsStatusCommand` and `runAuthProbes`, expands probe targets across all configured model candidates while preserving existing default behavior and filters, and updates tests plus the public models CLI docs.
