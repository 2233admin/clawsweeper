## Summary

Keep open. Current main still lacks `openclaw models status --probe --all`: the CLI does not expose a status-level `--all` option, `modelsStatusCommand` has no `probeAll` option to pass through, and the probe planner still selects one candidate model per provider. The open, unmerged PR #63259 is the current implementation candidate, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

Request adding `openclaw models status --probe --all` to live-probe every configured primary, fallback, and image model with per-model status and JSON output while preserving current `--probe` behavior.

## Best Possible Solution

Keep this open and review #63259 or a successor PR. The implementation should add a status-level `--all` flag in both CLI paths, thread a `probeAll` option through `modelsStatusCommand` into `runAuthProbes`, expand probe targets across all configured primary/fallback/image candidates while preserving provider/profile filters and current default behavior, and update focused CLI, target-builder, JSON, and docs coverage.

## Reproduction Assessment

Yes. The missing capability is reproducible by static inspection of current main: neither CLI path accepts `models status --probe --all`, and the probe target builder still selects `direct[0]` once per provider.

## Solution Assessment

Yes. A narrow `--all` flag that threads a `probeAll` option into the existing target builder is the maintainable direction; it reuses the current live probe machinery instead of adding a separate monitoring or provider-specific curl path.
