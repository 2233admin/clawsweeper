The PR changes Discord native slash-command zero-output fallbacks from success-style `Done.` replies to an explicit no-visible-reply warning and adds regression coverage for those paths.

Decide whether Discord native zero-output commands should warn instead of acknowledge; if yes, port the change onto current main by updating both the direct plugin empty-reply branch and `dispatchDiscordNativeAgentReply`, with regression coverage that does not depend on `/status` falling through the dispatcher.
