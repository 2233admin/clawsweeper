## Summary

Keep this PR open. Current main still lacks the proposed initial-replay/preexisting-message suppression in `subscribeEmbeddedPiSession`, and the adjacent transcript-only `delivery-mirror`/`gateway-injected` filter does not cover ordinary assistant messages already present in `session.messages`. The related #50176 appears to be the same earlier contributor attempt, but it is closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Review and land a main-compatible version of this PR, or an equivalent core fix, with the regression test for subscribing to an existing session that replays historical assistant lifecycle events. Keep the existing transcript-only filters, but add the preexisting-message/initial-replay guard so ordinary historical assistant messages are not delivered outward again.
