## Summary

Keep open. The report is fresh, detailed, and current main still has unmerged implementation candidates for the same behavior. Source inspection shows the cron embedded-run path still passes the run-scoped session key without an explicit sandbox policy key, while the embedded runner uses that key for sandbox resolution and Docker image inspection still throws if the Docker backend path is reached.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent fix. The preferred fix should pass the correct sandbox policy key for automated runs, guarantee sandbox.mode off cannot initialize Docker/backend code, and add focused regression coverage for cron, heartbeat, and sub-agent startup with Docker unavailable.
