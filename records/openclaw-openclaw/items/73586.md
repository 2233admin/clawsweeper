## Summary

Keep open. The report is fresh, detailed, and current main is not proven implemented: PR #73627 still explicitly targets this issue and is unmerged, with PR #73671 as an alternate open candidate. Current main still passes only the run-scoped cron session key into the embedded runner, while the embedded runner still falls back from `sandboxSessionKey` to `sessionKey`, and the Docker image-inspect helper still throws when Docker is unavailable if the sandbox backend path is reached.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent fix. The preferred fix should pass the correct sandbox policy key for automated isolated runs, guarantee `sandbox.mode: "off"` cannot initialize Docker/backend code, and add targeted regression coverage for cron, heartbeat, and sub-agent startup when Docker is unavailable.
