## Summary

Keep open. The issue is recent, detailed, and the current main branch still does not contain a targeted fix or regression proof for the reported cron/heartbeat/sub-agent Docker probe when sandbox mode is off. The intended resolver guard exists, but that is exactly the path the report says is being bypassed.

## Best Possible Solution

Keep this issue open. Add a focused regression test that mocks Docker/backend calls and exercises cron, heartbeat, and sub-agent session startup with `agents.defaults.sandbox.mode: "off"`; then move any Docker capability/image/container preflight behind the same `resolveSandboxRuntimeStatus(...).sandboxed === true` short-circuit or pass the correct runtime policy session key everywhere.
