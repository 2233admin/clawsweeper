## Summary

Keep open. The report is fresh, detailed, and still has an open implementation PR with closing syntax. In the checked-out current main, cron isolated runs still pass only the run-scoped session key into the embedded runner; the embedded runner still falls back from missing `sandboxSessionKey` to that run key before resolving sandbox context; and the Docker image-inspect helper still throws when Docker is unavailable if the sandbox backend path is reached. Maintainers should review and land or close the existing fix candidates rather than close this issue early.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent fix. The preferred fix should pass the canonical sandbox policy key for automated isolated runs, ensure `sandbox.mode: "off"` cannot initialize Docker/backend code, and add targeted regression coverage for cron, heartbeat, and sub-agent startup when Docker is unavailable.
