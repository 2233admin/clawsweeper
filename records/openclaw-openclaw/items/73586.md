## Summary

Keep open. The report is recent, detailed, and matches a real high-impact contract: sandbox mode `off` should mean host execution with no sandbox runtime dependency. Current main still has the same sandbox-resolution guard and Docker image probe behavior as the reported release, and I did not find a landed fix or targeted regression proof for the cron/heartbeat/sub-agent bypass described here.

## Best Possible Solution

Keep the issue open and add a focused regression test that mocks Docker/sandbox backend calls and exercises the automated session entry points with `agents.defaults.sandbox.mode: "off"`. The fix should ensure every cron, heartbeat, sub-agent, and harness path calls Docker only after `resolveSandboxRuntimeStatus(...).sandboxed === true`; if a separate preflight path is bypassing that guard, move it behind the same resolver or short-circuit it for mode off.
