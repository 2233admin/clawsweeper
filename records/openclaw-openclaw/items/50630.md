## Summary

Keep open. Current main still allows the reported `gateway.auth.mode="none"` plus `gateway.tailscale.mode="serve"` startup path, and there is an open PR (#50631) with closing syntax for this issue. Per policy, this issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and route it through review of PR #50631 or an equivalent narrow gateway hardening PR. The fix should reject `gateway.auth.mode="none"` when `gateway.tailscale.mode="serve"` is active, preserve local loopback-only `mode: none`, preserve existing verified Tailscale identity behavior for Control UI/WebSocket auth when a real auth mode is configured, and update `src/gateway/server-runtime-config.test.ts` plus gateway docs if the startup contract changes.
