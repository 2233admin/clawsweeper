## Summary

Keep this PR open. Current main still lacks the requested Slack API error detail formatting, and the latest PR head addresses the earlier `retryAfter` SDK-shape review by reading it as a top-level field. The patch is not merge-ready because its new formatter replaces several redacting `formatErrorMessage` call sites with raw `err.message` and raw structured fields, which can leak token-shaped values in logs or status output.

## What This Changes

Adds a Slack-local `formatSlackError()` helper with unit tests and routes Slack probe, scope, startup, stream, reaction, slash reply, and thread-resolution error paths through it to include Slack SDK error codes, scope details, retry timing, and HTTP status fields.

## Best Possible Solution

Land a Slack-plugin-local formatter that appends the Slack SDK structured fields but redacts the final string before returning it, including the non-Error fallback branch, and add a regression test with a Slack-shaped token in `err.message` or metadata. Keep the broader delivery queue preservation work tracked separately through #62391 or a generic core error serialization seam.
