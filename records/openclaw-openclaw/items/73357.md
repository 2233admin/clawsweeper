## Summary

Keep this PR open. Current main still does not implement graceful ENOSPC/inotify handling for memory-core file watchers, and the PR is a focused active bug-fix candidate, but it still needs maintainer review before landing because its global unhandled-rejection classifier can suppress message-only disk-full failures and it does not address the QMD memory watcher path.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to handle memory watcher errors at the watcher source, tighten or avoid global ENOSPC message-only suppression so ordinary disk-full write failures remain fatal/visible, add negative tests for disk-full messages without watcher context, and either add QMD watcher parity or explicitly scope it into a follow-up before landing.

## Risks / Open Questions

- Landing the PR as-is may suppress unrelated message-only disk-full failures because the global classifier treats 'no space left on device' and 'enosp' as transient without requiring a watcher/inotify signal.
- The PR only adds an error handler to the builtin memory watcher; QMD-backed memory has a separate chokidar watcher with the same missing error-listener pattern on current main.
- No live inotify-exhaustion reproduction was run in this read-only review; the decision is based on source, PR diff, provided discussion, and targeted GitHub API inspection.
