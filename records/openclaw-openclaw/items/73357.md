## Summary

Keep this PR open. Current main does not implement the requested file-watcher ENOSPC handling: the global unhandled-rejection classifier still only treats network and SQLite failures as transient, and the memory-core watcher still lacks an `error` listener. The PR is a focused, recent bug-fix candidate, but it still needs maintainer review because its global message-only classifier may suppress unrelated disk-full errors and QMD watcher parity should be checked.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to tighten file-watch classification so disk-full write failures are not globally swallowed, add direct tests for message-only disk-full negatives and watcher `error` handling, and consider applying the same watcher error handling to QMD memory before landing.

## Risks / Open Questions

- The PR’s global message-only classifier can still suppress unrelated disk-full failures when the error message contains `no space left on device` or `enosp` but lacks an `ENOSPC` code.
- The PR only adds an error handler to the builtin memory watcher; QMD-backed memory has a separate chokidar watcher with the same missing `error` listener pattern on current main.
- No workflow, lockfile, dependency-source, install-script, release, publishing, permissions, or secrets-surface changes were observed in the PR diff; the security-relevant review point is overly broad fatal-error suppression.
