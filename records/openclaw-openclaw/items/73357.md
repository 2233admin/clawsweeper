## Summary

Keep this PR open. Current main still lacks graceful ENOSPC/inotify watcher handling for the memory-core watcher, and the live PR remains the focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to tighten the file-watch transient classifier so generic disk-full errors are not globally swallowed, keep regression tests for watcher positives and disk-full negatives, add direct watcher error-handler coverage if feasible, and decide whether QMD watcher parity belongs in this PR or a linked follow-up before merge.
