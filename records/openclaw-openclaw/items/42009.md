## Summary

Keep this PR open. Current main still resolves `openclaw agent --agent <id> --to <target>` through the implicit selected-agent main session before the `--to`-derived key can win, so the central change in this PR is not implemented on main. #72414 fixed the related configured-default-agent path, but its own context treats this explicit `--agent + --to` case as distinct rather than superseded.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to rebase it onto current main, preserve explicit `--session-key` precedence, derive selected-agent `--to` keys before the implicit agent-main fallback, keep qualified `--to` rebinding to the selected agent scope, and retain regression coverage for those cases alongside #72414's default-agent compatibility tests.
