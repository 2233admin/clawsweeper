## Summary

Keep open. Current main still has no supported tools-level umask, file mode, dir mode, or filePermissions config for agent-created user-facing files. The existing config and docs only cover exec routing/security and workspace containment, while host write/edit, apply_patch, workspace-safe writes, and sandbox bridge writes either inherit process defaults or use fixed 0600-style behavior. This remains core work because the requested behavior spans built-in filesystem and exec/write surfaces, not an optional plugin-only integration.

## Best Possible Solution

Keep this issue open and implement a core, documented, per-agent-aware file creation policy for user-facing workspace output, likely under tools.fs or a narrow tools.fileCreation block. Validate octal umask/file/dir values, apply the policy to host write/edit, apply_patch, sandbox bridge writes, and exec-created user-output where feasible, while preserving existing restrictive modes for sensitive OpenClaw-managed files. Add regression tests for host filesystem tools, sandbox bridge writes, apply_patch, and controlled-umask exec output, plus docs that distinguish this from config/credential permission hardening.
