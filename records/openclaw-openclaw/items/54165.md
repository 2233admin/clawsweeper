## Summary

Keep this PR open. The heartbeat leak is still present on current main and the PR contains useful source work, but the patch is not merge-ready because it mixes the narrow heartbeat sanitizer with Telegram streaming hook changes that can emit `message:sent` before the final preview content is settled.

## What This Changes

The PR adds heartbeat-mode stripping of `[TOOL_CALL]` and `[TOOL_RESULT]` bracket blocks in `buildReplyPayloads`, and also wires Telegram draft-stream `message:sent` hook emission with test coverage.

## Best Possible Solution

Land the heartbeat bracket-block sanitizer in `src/auto-reply/reply/agent-runner-payloads.ts` with focused regression coverage, and move any Telegram streaming hook changes into a separate PR that explicitly preserves the existing preview-finalization semantics from #50917.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `buildReplyPayloads` with `isHeartbeat: true` and a text payload containing `[TOOL_CALL]...[/TOOL_CALL]`; current main preserves the text because the heartbeat branch passes payloads through unchanged.

## Solution Assessment

No. The heartbeat sanitizer is the right narrow direction, but this PR is not the best patch because it also changes Telegram streaming hook timing and can report send-time preview content instead of finalized reply content.
