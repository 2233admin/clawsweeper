## Summary

Keep open: current main still does not strip the reported heartbeat bracket blocks, but this PR should not merge as-is because it mixes the valid heartbeat fix with stale Telegram streaming hook changes that can emit `message:sent` before the final preview content is settled.

## What This Changes

The PR adds regex stripping for `[TOOL_CALL]` and `[TOOL_RESULT]` blocks in heartbeat reply payloads and also wires Telegram draft-stream `message:sent` hook emission with a dispatch test.

## Best Possible Solution

Land a narrow heartbeat sanitizer in the auto-reply payload path with regression coverage for bracket-format heartbeat replies, and keep Telegram streaming hook behavior on the existing preview-finalized path unless a separate PR reopens that design against the current #50917 semantics.
