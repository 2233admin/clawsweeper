## Summary

Keep this PR open. Current main still has the Kimi/OpenAI-compatible malformed tool-call argument repair path, but it does not implement the PR's central fail-closed behavior for unrecoverable malformed final JSON: current code preserves preexisting streamed arguments, only reapplies cached repaired arguments to `stream.result()`, and has no cleared-index tracking for final snapshots. The PR is also now in tension with the shipped #54491 preservation behavior, so it needs maintainer product/security judgment and a rebase onto the split helper file rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the Kimi policy explicitly: whether unrecoverable malformed final JSON should always clear all dispatch/final snapshots, or whether the shipped #54491 preservation behavior should remain. If fail-closed is desired, rebase the implementation into `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts` and add focused coverage for event `toolCall`, `partial`, `message`, and `stream.result()` final snapshots without regressing valid JSON, prefixed repair, and cached-repair invalidation cases.
