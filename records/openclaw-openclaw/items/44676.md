## Summary

Keep this PR open. Current main still does not implement the PR's central fail-closed behavior for unrecoverable malformed Kimi tool-call JSON, and later shipped code/tests now explicitly preserve preexisting streamed tool arguments in some malformed-reevaluation cases. The PR remains a plausible security/behavior hardening candidate, but it needs maintainer policy judgment and a rebase onto the current split helper module.

## Best Possible Solution

Keep this PR open as a focused candidate. A maintainer should decide whether malformed final Kimi tool-call JSON must fail closed even when streaming already exposed non-empty arguments; if yes, rebase the change into `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts` with focused coverage for `event.toolCall`, partial/message snapshots, and `stream.result()` final messages, while preserving the current valid/preamble repair cases and explicitly reconciling the #54491 preservation behavior.
