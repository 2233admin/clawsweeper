## Summary

Keep this PR open. Current main still does not start typing for direct tool-dispatched skill commands: that path executes and returns inside `handleInlineActions` before the normal agent-run typing signaler is used. The PR is focused and useful, but the existing review discussion is valid: the proposed placement should be revised so typing failures cannot abort the tool and so `typingMode: "never"` / `suppressTyping` are respected.

## Best Possible Solution

Keep the PR open, but request a policy-aware revision: compute or pass the same typing policy/mode used by normal runs into `handleInlineActions`, start typing before `tool.execute()` only when allowed, isolate typing-start errors in a nested non-fatal try/catch, and add focused inline tool-dispatch tests for allowed typing, `typingMode: "never"`/suppressed typing, and typing-start failure still executing the tool.
