## Summary

Keep open. Current main already contains the blank-prompt guard from merged #74168, but it still lacks the PR's format-cooldown exemption and fallback tool-support behavior. The branch is useful, but it should not merge as-is because the unconditional embedded-runner `requiresTools: true` would skip configured no-tool fallback models that current runner/docs intentionally support as text-only degraded fallbacks.

## What This Changes

The PR branch exempts `format` failover reasons from auth-profile cooldown, adds a `requiresTools` model-fallback filter passed from the agent command path, and adds focused tests for cooldown and tool-support fallback filtering.

## Best Possible Solution

Keep the format-error auth-health exemption, preserve current main's merged blank-prompt guard, and change the fallback tool filter so it only skips no-tool models when the specific run truly requires callable tools rather than for every embedded fallback chain. Update the model-failover docs to match the new cooldown semantics before merge.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main returns `format` from `resolveAuthProfileFailureReason`, and the PR head unconditionally passes `requiresTools: true`, which would skip a configured fallback model with `compat.supportsTools: false` even though the current attempt runner can run such models with tools disabled.

## Solution Assessment

No. The format cooldown change is narrow, but the tool fallback implementation is too broad; the safer fix is to derive `requiresTools` from actual explicit tool requirements or otherwise preserve documented text-only fallback behavior.
