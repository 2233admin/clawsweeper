## Summary

Keep open. Current main already covers the blank replay/session-file repair path, but it still lacks the two remaining behaviors this PR targets: `format` failover reasons still flow into auth-profile cooldown, and model fallback resolution still does not skip fallback models that explicitly declare no tool support.

## What This Changes

The PR adds a `requiresTools` fallback filter for configured fallback models, passes it from the embedded agent command path, and exempts `format` failover reasons from auth-profile cooldown with focused tests.

## Best Possible Solution

Land a focused patch that keeps current main's blank-message sanitization, prevents client-side format payload bugs from poisoning shared auth-profile health, filters explicitly tool-incapable fallback models for tool-requiring embedded runs, and aligns the model-failover docs with the new cooldown semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `resolveAuthProfileFailureReason({ failoverReason: "format" })` returns `format`, and configured fallback refs are added without consulting `supportsModelTools`.

## Solution Assessment

Yes, with a small documentation follow-up. The PR's final code direction is narrow and uses existing helpers; the safer completion is to update the model-failover docs so operator-facing cooldown behavior matches the new `format` policy.
