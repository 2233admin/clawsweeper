## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, with only fixed SOUL.md guidance and no configurable elevation or exclusion surface. The contributor supplied useful design and the linked implementation PR was closed unmerged, while the latest maintainer-side discussion still points to an unresolved prompt/API product boundary.

## What This Changes

Request configurable elevation of selected context files into dedicated prompt sections before Project Context, with matched files excluded from the flat context unless explicitly included.

## Best Possible Solution

Maintainers should define a documented deterministic context-elevation contract, then implement it where prompt assembly and agent config meet, with tests proving ordering, exclusion, missing-file handling, cache placement, and instruction-priority wording.
