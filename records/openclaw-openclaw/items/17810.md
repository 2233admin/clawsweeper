## Summary

Keep open. Current main still renders SOUL.md, IDENTITY.md, USER.md, and MEMORY.md as ordered but flat Project Context files; it does not expose configurable context-file elevation, dedicated prompt sections, or Project Context exclusion. The related implementation PR #18110 is closed and unmerged, and the remaining decision is core prompt/config contract versus a better plugin/hook surface.

## Best Possible Solution

Keep this issue open for maintainer product direction. If accepted in core, add a documented typed config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering, config schema/help/generated docs, and public docs. If maintainers prefer externalization, expose or document a stable plugin/context-engine path that can map bootstrap files to prompt sections and exclude them from Project Context without replacing the whole system prompt.
