## Summary

Keep open. Current main still renders the named bootstrap files as ordered but flat Project Context entries, exposes no configurable elevation/exclusion surface, and the direct implementation PR is closed unmerged; the remaining decision is maintainer product/API direction rather than conservative cleanup closure.

## What This Changes

Requests a configurable `context.elevation` surface that maps bootstrap files such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` into dedicated prompt sections and can exclude them from flat Project Context rendering.

## Best Possible Solution

Define a deterministic, documented context-file elevation contract, then implement it in the maintainer-approved surface with schema/help labels, prompt rendering, exclusion behavior, prompt-cache placement, tests, and public docs kept in sync.

## Reproduction Assessment

Yes. A focused static reproduction is to pass `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles` into `buildAgentSystemPrompt`; current main sorts them and renders them as `## <path>` entries under `# Project Context`, and existing tests assert that flat model.

## Solution Assessment

Unclear. The requested typed config is plausible and narrow, but the best maintainable fix depends on whether maintainers want this as core prompt config or as a plugin/context-engine/hook contract that can both elevate and exclude bootstrap files without replacing the whole prompt.
