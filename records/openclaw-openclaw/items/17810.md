## Summary

Keep open. Current main still renders the named bootstrap files as ordered but flat Project Context entries, the requested configurable elevation/exclusion surface is absent from prompt params, config schema, tests, and docs, and the related implementation PR #18110 is closed unmerged. The remaining decision is the accepted prompt/context API shape, not cleanup closure.

## What This Changes

The issue requests configurable elevation of selected context/bootstrap files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of rendering them as ordinary Project Context files.

## Best Possible Solution

Define a documented deterministic context-file elevation contract covering match semantics, ordering, exclusion, missing-file behavior, prompt-cache placement, and priority-scoping, then implement it either in agent prompt assembly/config or as a narrow plugin/context-engine API with aligned tests and docs.
