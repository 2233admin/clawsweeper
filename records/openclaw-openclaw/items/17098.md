## Summary

Keep this PR open. Current main still does not include an xCloud hosting guide, VPS card, docs navigation entry, or redirects, so the contribution is not obsolete. The PR should not merge as-is because the proposed guide still mislabels Claude Code as an AI provider and documents a third-party hosted Control UI credential flow without verifying OpenClaw's current allowed-origin, gateway auth, and device-pairing requirements.

## What This Changes

Adds an xCloud hosting guide under `docs/install`, registers xCloud navigation and redirects in `docs/docs.json`, and adds an xCloud card to the VPS provider picker.

## Best Possible Solution

Publish an xCloud hosting guide only after a maintainer or xCloud/provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the page to document the exact auth, origin, device-pairing, remote-access, Telegram allowlist, and provider-credential behavior.

  - confidence: 0.95

  - confidence: 0.86
