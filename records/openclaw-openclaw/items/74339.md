## Summary

Keep open for normal maintainer review. Current main still has the duplicate-prone ACPX Codex startup override path, the linked bug remains open, and this PR is a focused implementation with green PR checks; it has not been superseded or implemented on main.

## What This Changes

The PR adds ACPX Codex command-key detection so existing `-c model` and `-c model_reasoning_effort` flags are not appended again, with regression tests for full and partial configured-command cases.

## Best Possible Solution

Land this PR or an equivalent narrow ACPX fix after maintainer review, keeping the regression coverage and making the configured-command versus runtime-override precedence explicit if maintainers want a different rule.

## Reproduction Assessment

Yes. Current main can be checked statically: a configured Codex ACP command that already includes `-c model=... -c model_reasoning_effort=...` will receive another pair when runtime model/thinking options are active; issue #74305 also provides a user repro and workaround.

## Solution Assessment

Unclear. The PR is the narrowest fix for duplicate `-c` keys, but the configured-value precedence for conflicting runtime overrides is a maintainer policy choice rather than something proven by the current docs/tests.
