## Summary

Keep open. Current main still leaves the user-facing `openclaw system event` / gateway `wake` path without explicit session targeting, while internal exec/node completion paths are only partially relevant because they already receive a session key. The issue also has open PR #50818 with closing syntax, so cleanup policy says it should remain open until that PR or a replacement is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open and use PR #50818, or a narrower replacement, as the implementation candidate. The fix should provide an explicit, validated session-targeted external completion path for `system event` / `wake` or a first-class async completion callback, thread the target session into `enqueueSystemEvent` and `requestHeartbeatNow`, and add targeted coverage for CLI params, gateway wake handling, cron wake routing, and the supported retry/failure behavior for local websocket callbacks.
