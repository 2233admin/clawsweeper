## Summary

Keep open. Current main has session-scoped internals for several exec/node/ACP completion paths, but the user-facing `openclaw system event` / gateway `wake` entry point still has no explicit session-targeting surface, and open PR #50818 is an unmerged implementation candidate that references this issue with closing syntax.

## Best Possible Solution

Keep this issue open and review, land, or close PR #50818 or a narrower replacement. The implementation should add an explicit, validated session-targeted external completion path for `system event` / `wake` or a first-class async completion callback, thread the target into `enqueueSystemEvent` and `requestHeartbeatNow`, and update CLI docs plus targeted protocol/CLI/gateway/cron/exec coverage.
