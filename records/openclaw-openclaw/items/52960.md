## Summary

Keep this issue open. Current main has chat streaming, live tool cards, queue/Stop controls, compaction/fallback indicators, session lifecycle snapshots, and reconnect recovery, but it still does not expose the requested compact active-run progress model in the Control UI chat surface: phase/latest action/last activity/elapsed/waiting target/subagent summary/reconnect-during-run visibility remain missing.

## Best Possible Solution

Keep this issue open and implement a small Control UI active-run progress model fed by Gateway chat, agent lifecycle, tool/subagent, delivery, and reconnect signals. Render a compact strip near the thread or composer with current phase, latest meaningful action, last activity, elapsed time, waiting target/subagent summary, and reconnect count; cover it with focused Control UI and Gateway tests and document the behavior on the public Control UI docs page.
