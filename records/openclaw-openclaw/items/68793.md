## Summary

Keep open. Current main has partially reduced QQ Bot WebSocket payload noise by keeping raw dispatch JSON at debug, but the normal inbound message handler still sends user-visible message content through the info log path, so the central privacy/noise request remains unresolved.

## What This Changes

The issue requests lowering QQ Bot default log verbosity and preventing full inbound chat payloads from appearing in info-level logs.

## Best Possible Solution

Fix this inside the QQ Bot plugin by making default info logs metadata-only for inbound processing and routine command detection, keeping full message bodies and raw gateway payloads behind debug or an explicit verbose path. Add focused tests around the QQ Bot gateway or logger boundary so info logs expose identifiers, type, and content length rather than user text.

The observable bug is that normal QQ Bot info logs can include inbound user chat content: `extensions/qqbot/src/engine/gateway/gateway.ts` logs `event.content` via `log.info`, and `extensions/qqbot/src/engine/commands/slash-command-handler.ts` logs urgent command text snippets at info.
