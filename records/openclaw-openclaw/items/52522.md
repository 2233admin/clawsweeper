## Summary

Keep this PR open. Current main still bounds bootstrap file reads to the resolved workspace directory, so the proposed widened symlink behavior is not shipped. The PR is a focused bug-fix candidate, but it needs maintainer review or revision because the proposed default-workspace-root boundary can weaken prompt isolation for nested configured workspaces.

## Best Possible Solution

Keep the PR open and ask for maintainer review or a revision that threads an explicit intended bootstrap boundary through the run/config path instead of hard-coding DEFAULT_AGENT_WORKSPACE_DIR. If shared parent bootstrap files are desired for multi-agent child workspaces, land it with tests for accepted in-boundary symlinks, rejected workspace escapes, and nested configured workspace isolation.
