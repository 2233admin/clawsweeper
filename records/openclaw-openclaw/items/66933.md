## Summary

Keep this PR open. Current main still lacks the global `tools.toolsBySender`, per-agent `agents.<id>.tools.toolsBySender`, channel-scoped sender key type, and sender-policy filtering step in both the main and bundled tool pipelines. The PR is an active implementation candidate for a real tool-capability boundary change, not conservative cleanup material.

## Best Possible Solution

If the product direction is accepted, rebase it onto current main, settle the explicit `channel:<provider>:<senderId>` contract, update schemas/generated config metadata and docs, add focused regression tests for the main and bundled tool pipelines, and resolve the remaining channel alias plus no-sender wildcard semantics before merge.
