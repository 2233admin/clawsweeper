## Summary

Keep this PR open. Current main still has the Telegram progress gap: Codex item events can carry structured tool metadata, but the auto-reply callback drops it and Telegram falls back to the generic title, so the proposed narrow fix remains useful and has no blocking finding from this read-only review.

## What This Changes

The PR forwards item-event `meta` through auto-reply options, renders Telegram tool progress from structured `name` plus `meta`, and adds a focused Telegram dispatch regression test.

## Best Possible Solution

Land a narrow implementation that keeps existing `progressText` and `summary` precedence, forwards structured item metadata through the generic auto-reply callback, and lets Telegram own the channel-specific tool-progress fallback with focused regression coverage.

## Reproduction Assessment

Yes. A Codex item event shaped like `{ kind: "tool", title: "Tool", name: "read", meta: "from README.md" }` reproduces the current-main failure path: `meta` is emitted upstream, dropped by the runner, and Telegram chooses `Tool` before `read`.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix I found because it adds one callback field, copies it at the existing event boundary, and changes only Telegram's display fallback for structured tool events.
