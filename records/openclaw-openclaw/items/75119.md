## Summary

Keep this PR open: current main still drops Codex item-event `meta` before Telegram can render it, and the PR is a narrow implementation that addresses the observable generic `Tool` progress line without touching unrelated surfaces. ## What This Changes

The PR adds optional `meta` to auto-reply item events, forwards that field from agent item events, formats Telegram tool progress from `name` plus `meta` before falling back to generic titles, and adds a focused Telegram dispatch regression test.

## Best Possible Solution

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable path: it preserves existing `progressText` and `summary` precedence, forwards an already-emitted optional field, and changes only Telegram's fallback for structured tool items.
