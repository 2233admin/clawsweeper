## Summary

Keep open. Current main still exhibits the Telegram preview gap this PR targets, and the diff is a narrow implementation with no blocking correctness findings; the remaining action is maintainer review and targeted validation rather than cleanup closure.

## What This Changes

The PR adds an optional `meta` field to auto-reply item events, forwards Codex item-event meta through the runner, formats Telegram tool progress from structured tool name plus meta before generic titles, and adds a Telegram dispatch regression test.

## Best Possible Solution

Land the narrow callback-and-Telegram formatting change after maintainer review and the focused Telegram dispatch test passes; keep broader channel parity, such as Matrix preview formatting, out of this PR unless maintainers explicitly want the scope expanded.

## Reproduction Assessment

Yes. A Codex item event shaped like `{ kind: "tool", title: "Tool", name: "read", meta: "from README.md" }` is enough to reproduce the current-main failure path: `meta` is dropped before Telegram, and Telegram then prefers the generic `Tool` title.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix for the reported Telegram behavior because it adds an optional callback field, forwards the already-emitted Codex data, and only changes Telegram's fallback order after `progressText` and `summary` remain preferred.
