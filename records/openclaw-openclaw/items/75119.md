## Summary

Keep open for normal maintainer review. Current main still drops structured item-event `meta` before Telegram sees it, so Codex tool events whose title is the generic `Tool` cannot render the actionable `read from README.md` style progress line; the proposed patch is narrow and I found no blocking review findings.

## What This Changes

The PR forwards item-event `meta` through reply options and uses structured tool `name` plus `meta` for Telegram preview progress, with a regression test for Codex-style read progress.

## Best Possible Solution

Land this narrow Telegram/core propagation fix after targeted validation, keeping the existing preview sanitizer and fallback order while allowing structured Codex tool metadata to reach the Telegram progress line.

## Reproduction Assessment

## Solution Assessment

Yes. The proposed path is the narrowest maintainable fix: carry the already-emitted event metadata through the existing reply-options contract and consume it only in Telegram’s preview formatter before falling back to generic titles.
