## Summary

Keep open. Current main and the latest stable release still expose the shared message tool `channel` parameter as an undescribed optional string, so the requested agent-visible schema guidance has not landed; the prior implementation PRs are closed unmerged.

## What This Changes

The issue requests adding description metadata, and possibly a plugin-aware enum, to the shared message tool `channel` parameter so agents pass provider/plugin ids rather than destination ids.

## Best Possible Solution

Land a narrow core schema/test update: keep `channel` optional, add TypeBox description text saying it expects a messaging provider or channel plugin id such as `slack` or `discord` rather than a destination id, and add a focused assertion. Defer enum support unless it is derived from plugin-aware channel values.

## Reproduction Assessment

Yes. A high-confidence reproduction is source/schema inspection on current main: `createMessageTool()` exposes `buildRoutingSchema()`, whose `channel` property has no description or enum metadata.

## Solution Assessment

Yes for the description portion. The narrowest maintainable fix is schema metadata plus a focused test; a hard-coded static enum is not the best path unless it is generated from registered channel/plugin values.
