## Repair Work Prompt

Fix the message tool `channel` schema metadata for this issue. In `src/agents/tools/message-tool.ts`, keep `channel` optional but add a TypeBox description explaining that it expects a messaging provider or channel plugin id such as `slack` or `discord`, not the destination conversation/channel/user id. Do not add a hard-coded static enum unless it is derived from the active channel/plugin registry and preserves external plugin ids/aliases. Do not change channel selection runtime behavior.
