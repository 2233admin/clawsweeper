## Summary

Keep open. Current main still performs one SearXNG category request and returns a successful zero-result payload without retrying the general category; the existing core provider fallback only handles unavailable providers, thrown errors, or structured missing-key errors.

## What This Changes

The issue requests a SearXNG/web-search fallback that retries the same query as general search when a category-specific search returns zero results.

## Best Possible Solution

Add the retry in the SearXNG plugin client: after an empty categorized response, retry once with categories set to general when the original effective category set is non-empty and does not already include general, while preserving timeout, cache, endpoint validation, count, language, wrapping, and error behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a mocked runSearxngSearch call with categories: "weather" where the SearXNG JSON body is {"results":[]}; current code builds one categorized URL and returns count: 0 without a second general request.

## Solution Assessment

Yes. A provider-local fallback is the narrowest maintainable solution because zero results can be a valid successful result in the generic provider loop, while this report is about SearXNG category-engine compatibility.
