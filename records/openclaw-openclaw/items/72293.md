## Summary

Keep open. Current main partially carries terminal `stopReason` into lifecycle/session state, but the parent-visible wait and subagent completion paths still classify a lifecycle `end` with `stopReason: "aborted"` as successful unless `aborted: true` is also present. An open PR already references this issue with closing syntax, so the issue should remain open until that implementation is reviewed and merged or rejected.

## What This Changes

The issue requests that aborted empty-content subagent terminal turns be classified and announced as aborted or failed instead of successful done/no-op completion.

## Best Possible Solution

Land a focused fix that treats terminal assistant `stopReason: "aborted"` as a parent-visible failed or killed outcome across lifecycle emission, `agent.wait`, subagent registry completion, and announce formatting, while preserving existing retry-grace behavior for transient aborts. Add regression coverage for an empty-content aborted assistant turn flowing through producer, wait, registry, and announcement classification.
