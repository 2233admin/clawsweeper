## Summary

Keep this open. Current main still has a source-level abort classification gap: an embedded terminal assistant turn with `stopReason: "aborted"` can be emitted as lifecycle `phase: "end"`, while `agent.wait` and the subagent registry classify that terminal event as successful unless `aborted: true` is also present. The issue is also paired with an open linked fix PR using closing syntax, so it should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks OpenClaw to treat aborted, empty-output subagent terminal runs as parent-visible failures instead of successful completion with no announce.

## Best Possible Solution

Land or replace a narrow core fix that normalizes final assistant `stopReason: "aborted"` into an existing failed/killed parent-visible outcome across lifecycle emission, `agent.wait`, subagent registry completion, announce formatting, and focused regression tests, without adding a new public status union or automatic provider retry policy unless maintainers explicitly expand the scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to emit lifecycle `phase: "end"` with `stopReason: "aborted"` and no `aborted: true`; current main preserves the stop reason but `agent.wait` and the subagent registry classify the run as successful.

## Solution Assessment

Yes. The best narrow fix is to propagate aborted terminal metadata into the existing failure/killed path and assert that parent-visible wait/announce consumers do not treat it as `ok`; the linked open PR is the current implementation candidate, so the issue should not close before it lands or is replaced.
