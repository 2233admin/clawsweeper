## Summary

Keep this PR open. Current main and the latest release do not implement Cyrillic/Greek confusable folding for external-content boundary markers, so the PR is not obsolete. The PR is focused on the right security surface, but it needs maintainer review/rebase onto the current index-mapped sanitizer and should address the remaining lowercase Cyrillic gap raised in review before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/adapt the confusable folding into the current `foldMarkerTextWithIndexMap` pipeline, include a complete marker-alphabet confusable set including Cyrillic small te, and add regression tests for uppercase/lowercase Cyrillic and Greek spoofed start/end markers plus surrounding-text preservation. Then run the targeted external-content test file and the appropriate changed gate before merge.
