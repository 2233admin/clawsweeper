## Summary

Keep this PR open. Current main still does not implement progressive disclosure for long Control UI markdown messages: it keeps the existing hard markdown limits and renders markdown directly in chat text containers. The PR is still potentially useful Web UI work, but it should not be closed as implemented or merged without addressing the performance and toggle/layout concerns raised in review comments.

## Best Possible Solution

Keep this PR open for focused Web UI maintainer review or contributor revision. A mergeable implementation should preserve the cheap fallback for very large or actively streaming content, avoid parsing and mounting hundreds of thousands of characters by default, key expansion state to stable reactive message identity, keep controls outside masked content, handle streaming completion predictably, and add long-message regression tests if the behavior ships.
