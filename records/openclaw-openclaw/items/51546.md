## Summary

Keep this PR open. Current main still does not implement progressive disclosure for long markdown messages in the Control UI: chat markdown renders directly in plain `.chat-text`, markdown still hard-truncates at 140,000 chars, JSON auto-parse remains capped at 20,000 chars, and the stylesheet has no show-more/show-less disclosure classes. The PR is not obsolete or implemented on main, but the review discussion shows it needs revision before it could be merged.

## Best Possible Solution

Keep this PR open for focused Web UI maintainer review or request a revision. The best path is a long-message disclosure implementation that avoids eagerly parsing and mounting hundreds of thousands of characters, keeps expansion state in Lit/reactive state, places controls outside masked content, handles streaming-to-complete transitions predictably, and adds regression coverage for long chat messages.
