## Summary

Keep open. Current main still lacks the interactive SearXNG JSON-format setup note, and the latest PR patch addresses the earlier timing, SecretRef, and quickstart concerns, but it should not merge until the new public provider field is reconciled with the Plugin SDK API baseline and any maintainer docs decision.

## What This Changes

This PR adds a `credentialNote` field to web search provider metadata, sets it for SearXNG, displays it before credential collection, and adds SearXNG/provider setup flow tests.

## Best Possible Solution

Land a provider-owned pre-credential note for SearXNG, update the generated Plugin SDK API baseline for the intentional provider contract change, and add public SDK docs only if maintainers want third-party plugin authors to rely on this field.
