## Summary

Keep this PR open. Current main still lacks the proposed high-level `sessions_send` target-selection guidance in both the model-facing tool description and public session-tool docs, while the runtime currently enforces the mutual-exclusion behavior that the PR documents. Related #63339 is complementary schema-field guidance, and newer #74009 may become the broader runtime-precedence path but is still open and needs changes, so it is not a clean superseding implementation yet.

## What This Changes

This PR adds target-selection guidance for `sessions_send` in the public session-tool docs and model-facing tool description preset, telling callers to choose one of `sessionKey` or `label` and prefer `sessionKey` when known.

## Best Possible Solution

Keep this PR available for maintainer review, then either land the narrow current-behavior clarification or fold it into #63339/#74009 once maintainers decide whether `sessions_send` should remain mutually exclusive or move to `sessionKey` precedence.
