## Summary

Keep this PR open for maintainer review. Current main still lacks the reset-command preflight-compaction bypass, so the branch is not obsolete; however, the proposed guard also covers `/reset soft`, while current source, docs, and tests treat soft reset as transcript-preserving rather than a full session-clearing reset.

## What This Changes

The PR adds a `/new` and `/reset` preflight-compaction bypass in `runPreflightCompactionIfNeeded`, adds focused regression tests, and adds a changelog fix entry for the Discord reset timeout case.

## Best Possible Solution

Land a narrow preflight bypass only after it matches the real reset semantics: full `/new` and full `/reset` should avoid wasted compaction before session rotation, while `/reset soft` needs an explicit maintainer decision, dedicated coverage, and comment wording that matches the canonical reset detection paths.
