## Summary

Keep this issue open. The same reporter has an open, unmerged implementation PR (#67916) tied to this issue, and current main still contains the reported client-side hard-deny path when `localMediaPreviewRoots` is empty before bootstrap completes.

## Best Possible Solution

Keep this issue open while #67916 remains open. Maintainers should review and either land that PR or an equivalent focused fix that treats empty `localMediaPreviewRoots` as not-yet-loaded, uses the server metadata endpoint as the authority during that bootstrap window, preserves the existing populated-roots denial behavior, and adds a regression test for the empty-roots case.
