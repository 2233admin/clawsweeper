## What This Changes

The issue requests a Control UI fix so local assistant media attachments fall through to the server metadata check when bootstrap `localMediaPreviewRoots` have not loaded yet.

## Best Possible Solution

Keep this issue open until PR #67916, or an equivalent focused replacement, lands. The fix should treat empty `localMediaPreviewRoots` as a not-yet-loaded bootstrap state, use the authenticated server metadata endpoint as the authority, preserve populated-roots client-side denial for truly outside paths, and add a regression test for the empty-roots case.
