## Summary

Keep this PR open, but do not merge it as-is. The linked heartbeat prompt leak is valid; the patch filters the marker in channel visibility bookkeeping rather than in the actual outbound send path, and it also changes the public plugin SDK surface without updating the API baseline.

## What This Changes

The PR moves the heartbeat transcript prompt constant into infra/plugin-sdk reach, filters that exact string in Telegram/Slack/Discord delivered-text visibility helpers, and adds Apple Notes/ClawHub documentation edits.

## Best Possible Solution

Move heartbeat prompt suppression into the shared heartbeat/outbound delivery path before any channel send, keep meaningful heartbeat alert delivery intact, add a Telegram regression for the linked reproduction, and regenerate plugin SDK API baseline if the constant remains a public SDK export.

## Reproduction Assessment

Yes. The linked issue provides a concrete Telegram DM configuration and release, and current-main source inspection shows the reported marker is not guarded at the shared heartbeat/channel send boundary.

## Solution Assessment

No. Filtering shouldTreatDeliveredTextAsVisible is not the narrowest maintainable fix because that hook does not prevent routing or direct heartbeat outbound delivery; a shared pre-send guard with regression coverage is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Suppress the marker before delivery:** `extensions/telegram/src/channel.ts:285-287`

Put the guard in the actual heartbeat/outbound delivery path and cover the Telegram repro.

  - confidence: 0.84
