## Summary

Keep open. Current main still has the Matrix cross-signing UIA limitation this PR targets, and the linked companion issue remains valid, but the patch should not merge as-is because the new MAS capability probe bypasses Matrix's guarded transport path and the changelog entry misses the required attribution.

## What This Changes

The PR updates Matrix crypto bootstrap to parse UIA 401 stages, surface MAS cross-signing reset errors and URLs, detect MAS through auth metadata, add regression tests, and add an unreleased Matrix changelog entry.

## Best Possible Solution

Keep the PR open and request changes: preserve the MAS/MSC3967 behavior, but run the auth_metadata probe through Matrix's guarded transport, add the required changelog credit, update the Matrix verification docs for MAS reset recovery, then rerun focused Matrix bootstrap tests and the changed gate before merge.
