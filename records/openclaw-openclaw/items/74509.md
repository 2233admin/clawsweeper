## Summary

Keep open. Current main still has the old Matrix cross-signing UIA fallback and the PR addresses the valid MAS/MSC3967 gap from #74504, but the PR head still introduces a blocking password-handling regression by trimming an opaque Matrix password before submitting password UIA.

## What This Changes

The PR changes the Matrix plugin’s cross-signing bootstrap to detect MAS via auth metadata, parse UIA 401 stages and sessions, surface typed MAS reset errors with URLs, add regression tests, and add a Matrix changelog entry.

## Best Possible Solution

Request changes to preserve the raw configured Matrix password in the submitted auth dict while using trimming only for the configured-password presence check, add a regression test for leading/trailing whitespace in channels.matrix.password, document MAS reset recovery in the Matrix verification docs, then rerun the focused Matrix bootstrap tests and changed gate.

## Reproduction Assessment

Yes. The current-main bug is reproducible from #74504’s UIA 401 body advertising org.matrix.cross_signing_reset and the current callback path in crypto-bootstrap.ts; the PR regression is reproducible with a unit callback where the configured Matrix password intentionally contains leading or trailing whitespace.

## Solution Assessment

No, not as currently patched. Stage-aware UIA handling, session threading, typed MAS reset errors, and the guarded auth_metadata probe are the right direction, but trimming an opaque credential before submission is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-227`
  - body: The patch assigns `params.password?.trim()` to `password` and then sends that value in the `m.login.password` auth dict. Matrix passwords are opaque credentials and current main submits the configured value unchanged, so accounts with intentional leading or trailing whitespace would fail UIA after this change. Use trimming only to decide whether a password is configured, but submit the original `params.password`.
  - confidence: 0.88
