## Summary

Keep open. The PR addresses a real current-main Matrix MAS/UIA gap, but the current diff introduces a credential handling regression by trimming the configured Matrix password before submitting `m.login.password` UIA.

## What This Changes

The PR changes Matrix cross-signing bootstrap to detect MAS via `auth_metadata`, parse UIA 401 stages and sessions, surface typed MAS reset errors with URLs, add regression tests, and add a Matrix changelog entry.

## Best Possible Solution

Request changes to preserve the raw configured Matrix password in the submitted auth dict while using trimming only for the presence check, add a regression test for leading/trailing whitespace in `channels.matrix.password`, document MAS reset recovery in the Matrix verification docs, then run the focused Matrix bootstrap tests and changed gate.

## Reproduction Assessment

Yes. The central bug is reproducible from current-main code plus the companion report's 401 UIA body, and the PR regression is reproducible with a unit test where a configured Matrix password intentionally contains leading or trailing whitespace.

## Solution Assessment

No, not as currently patched. The stage-aware UIA handling, session threading, typed MAS reset error, and guarded `auth_metadata` probe are the right direction, but trimming an opaque credential before submission is not the narrowest safe implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-227`
  - body: The patch assigns `params.password?.trim()` to `password` and then sends that value in the `m.login.password` auth dict. Current main submits the configured password unchanged, and Matrix passwords are opaque credentials, so accounts with intentional leading or trailing whitespace would fail UIA after this change. Use trimming only to decide whether a password is configured, but submit the original `params.password`.
  - confidence: 0.88
