## Summary

Keep this PR open. Current main still does not implement the intended lifecycle-end Gateway final-message fix: a `NO_` lead fragment can be suppressed during delta streaming, remain buffered, then be emitted in the final chat payload because finalization disables lead-fragment suppression. The latest branch remains the active implementation candidate, with useful maintainer/contributor context, but the clientRunId scoping security note needs resolution before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to resolve the clientRunId scoping concern, keep regression coverage for lifecycle-end `NO_`/`NO_RE` suppression and intentional uppercase `NO` preservation, rebase if needed, then run the normal Gateway changed gate before merge.
