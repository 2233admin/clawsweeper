## Summary

Keep this PR open. Current main still does not implement the intended Gateway final-message fix: a suppressed `NO_`/`NO_RE` control-token lead fragment can remain buffered during streaming and then be emitted in the final chat payload because finalization disables lead-fragment suppression. The branch remains the active implementation candidate, but the clientRunId scoping security note needs maintainer resolution before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to rebase if needed, keep regression coverage for lifecycle-end `NO_` suppression and intentional uppercase `NO` preservation, resolve or explicitly justify the clientRunId scoping security note with code-backed reasoning, then run the normal Gateway changed gate before merge.
