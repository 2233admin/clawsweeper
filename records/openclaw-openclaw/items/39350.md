## Summary

Keep this PR open. Current main still does not implement the PR’s intended Gateway fix: a suppressed `NO_REPLY` lead fragment such as `NO_` can remain buffered and then be emitted as a final Control UI/WebChat assistant message when lifecycle ends before the full token arrives. The latest PR head remains the active implementation candidate, with targeted regression coverage, but it still deserves maintainer review for the client-run-id scoping security note before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The maintainer path is to rebase it on current main, keep regression coverage for `NO_` lifecycle-end suppression and uppercase `NO` preservation, address or explicitly dismiss the clientRunId scoping security note with code-backed reasoning, then run the normal Gateway changed gate before merge.
