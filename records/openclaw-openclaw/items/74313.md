## Summary

Keep open. Current main still has the reported mismatch: doctor classifies on-disk agent dirs only from the configured default id plus `agents.list`, while no-`--agent` model status still resolves through the legacy `agents/main/agent` path. The linked implementation PR #74438 is open and unmerged in the provided context, so this issue should stay open until maintainers land or close that PR.

## What This Changes

The issue reports unsafe `openclaw doctor` cleanup guidance for `agents/main` when a non-main configured default agent exists but compatibility auth/session state can still be live.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so `models status` and probe default-agent resolution match the documented contract while doctor exempts only the live compatibility `agents/main` store and preserves warnings for true orphan dirs.
