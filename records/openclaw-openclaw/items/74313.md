## Summary

Keep open. Current main still has the reported mismatch: doctor classifies an on-disk `agents/main/agent` directory as orphaned when `main` is not configured, while omitted `models status` still resolves auth/model state through the compatibility `main` agent directory. The issue also has an open, unmerged closing PR, #74438, so the issue should remain open until that PR lands or is replaced.

## What This Changes

`openclaw doctor` should stop presenting a live compatibility `agents/main` store as removable orphaned state when a named non-`main` agent is configured as the default.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the current live compatibility `main` store and still reports real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still reads through the compatibility `main` path.

## Solution Assessment

Yes. The narrow linked PR direction is the best current path because it aligns the command path, probe path, doctor diagnostic, docs contract, and regression tests without removing the legacy compatibility store.
