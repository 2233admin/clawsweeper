## What This Changes

The issue requests that doctor stop presenting the live compatibility `agents/main` state directory as stale/removable when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues warning on true orphan directories.

## Reproduction Assessment

Yes. Configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main’s `listOrphanAgentDirs` will classify `main` as orphan while no-`--agent` `models status` still resolves through the legacy `main` store helper.
