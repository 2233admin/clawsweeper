Keep open. Current main still has the reported doctor/models-status mismatch, and the provided GitHub context shows #74438 is an open, unmerged closing PR for the same behavior, so this issue should remain open for normal review and landing rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop `openclaw doctor` from presenting the live compatibility `agents/main` store as removable orphaned state when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves the auth/model store through the legacy `main` compatibility path.

## Solution Assessment

Yes. The best path is the narrow linked PR shape: align the models status/probe agent scope with the documented default-agent contract and add a targeted doctor exemption, rather than suppressing all orphan-agent warnings or closing before merge.
