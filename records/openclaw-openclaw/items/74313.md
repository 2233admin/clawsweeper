## Summary

Keep open. Current main still has the reported doctor/models-status mismatch, and #74438 is an open, unmerged fix PR linked to this issue, so the issue should remain open for normal maintainer review and landing.

## What This Changes

The issue asks OpenClaw to stop presenting the live compatibility `agents/main` store as removable orphaned state when a non-`main` default agent is configured.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that aligns `models status`, `models status --probe`, and doctor orphan classification around the configured default-agent contract while preserving real orphan warnings.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves through the compatibility `main` store.

## Solution Assessment

Yes. The narrowest maintainable solution is to align the command/path-resolution seams and exempt only the live compatibility store, rather than suppressing all orphan-agent diagnostics.
