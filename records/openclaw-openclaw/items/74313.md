## Summary

Keep open. Current main still has the reported mismatch: doctor can classify an unconfigured `agents/main/agent` directory as orphaned while `models status` and probe paths still fall back to the compatibility main auth store when no explicit agent is passed. The GitHub context also shows #74438 is an open, unmerged closing PR for this exact behavior, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw doctor` and `openclaw models status`/`--probe` to agree on live agent state when the configured default agent is not `main`, without suggesting removal of an active compatibility store.

## Best Possible Solution

The desired end state is a narrow agent-scope fix: omitted `models status` and `models status --probe` should resolve the same configured-default-or-env-overridden agent directory, and doctor should continue warning on real orphan dirs while exempting only the live compatibility `main` store.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by configuring a non-`main` default agent, leaving `agents/main/agent` on disk, and tracing current main: doctor reports `main` as orphaned while status/probe auth paths still resolve through the compatibility main dir.

## Solution Assessment

Yes. The linked #74438 direction is the narrow maintainable fix because it aligns command resolution with the documented configured-default contract instead of broadly suppressing orphan checks or merely softening the warning text.
