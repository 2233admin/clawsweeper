## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web search provider, but the requested ZAI/GLM native web_search path is still absent from the ZAI plugin manifest, runtime registration, shared contract expectations, tests, and docs. The issue discussion and closed prior PRs are useful context, but they do not supersede the remaining ZAI work.

## What This Changes

Request support for provider-native web search passthrough for ZAI/GLM and Google/Gemini when no separate search provider is configured.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, credential/config handling, docs, and provider registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
