## Summary

Keep open. Current main implements the Gemini half through the bundled Google/Gemini web_search provider, but the ZAI/GLM half is still absent from the ZAI plugin manifest, runtime registration, shared registration contract, tests, and docs. ## What This Changes

The issue requests native web_search passthrough for ZAI/GLM and Google/Gemini providers so model-owned search can work without a separate third-party search key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing `webSearchProviders` plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, changelog, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
