## Summary

Keep this PR open. Current main does not implement the PR's intended Discord thread-bound ACP relay suppression: the relay still enqueues `:progress`, `:stall`, and `:resumed` system events when parent streaming is enabled, and the spawn path does not pass any Discord/thread-specific suppression option. The linked issue discussion and PR review context support the bug and the proposed narrow fix, but this work has not landed on main.

## Best Possible Solution

Rebase or reimplement the PR on current main with a narrow Discord thread-bound suppression seam in the ACP parent relay wiring. Keep non-Discord and non-thread parent streaming unchanged, preserve relay logging and terminal lifecycle notices, and add regression coverage for explicit thread context plus thread-bound parent-session-key cases.
