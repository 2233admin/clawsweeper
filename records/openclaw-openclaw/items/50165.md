## Summary

Keep this open. Current main has fixes for descendant subagent announce timing and task visibility, but the remaining report is still a real lifecycle-contract gap: subagent completion is finalized from the child agent-run outcome, while child-owned background exec/process sessions are tracked separately and can outlive the child run.

## What This Changes

This issue asks OpenClaw to keep subagents visibly in progress until the delegated user-relevant work, including long-running child-owned work, is actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in /tasks and session_status, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancel, cleanup, and regression coverage.
