## Summary

Keep this open. Current main still treats the child agent-run outcome as authoritative for subagent task completion, while backgrounded exec/process sessions are tracked in a separate scoped process registry; the related merged fixes cover descendant subagent announce timing and embedded-run cleanup ordering, not the generic child-owned background-process liveness gap described here.

## What This Changes

The issue requests reliable subagent lifecycle/status semantics so delegated subagents do not appear complete before child-owned background exec or other user-relevant work has actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep agent-run completion authoritative and surface child-owned background processes as separate active work in /tasks and session_status, or defer subagent finalization through a generic owned-process liveness seam with bounded timeout, cancellation, cleanup, and regression coverage.
