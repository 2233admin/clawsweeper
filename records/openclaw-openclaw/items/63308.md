## Summary

Keep open. Current main still exposes `sessionKey`, `label`, and `agentId` as independent optional `sessions_send` parameters and still hard-rejects calls that include both `sessionKey` and `label`. The latest release tag shows the same guard, and open PR #63377 uses closing syntax for this issue, with related runtime follow-up still open in #64699/#64846.

## Best Possible Solution

Keep this issue open while maintainers review or consolidate the paired PRs. The strongest fix is likely executor-side canonicalization where a present `sessionKey` is authoritative, plus clearer schema/description guidance and regression coverage for mixed `sessionKey`+`label` calls; then close this through the merged PR.
