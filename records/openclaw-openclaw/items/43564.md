## Summary

Keep open. Current main still lacks ACP skill-context injection: the `sessions_spawn` ACP schema has no skill selector, ACP spawn params and runtime turn input have no skill/context field, and the ACP path forwards only the task text before the normal native skill snapshot path runs. The prior ClawSweeper review remains accurate; the relevant files have no diff from that reviewed state to current main.

## What This Changes

Request to let `sessions_spawn` with `runtime: "acp"` inject selected, automatic, or default OpenClaw skill context into the spawned external harness session.

## Best Possible Solution

Add a documented ACP skill-context capability after policy is settled: start with a typed explicit selector, reuse existing skill snapshot filtering and prompt limits, pass context through a structured ACP seam or tightly bounded prompt-prefix fallback, and add tests/docs for explicit lists, missing skills, truncation, isolation, and host-runtime safety. Treat automatic relevance and default skill injection as follow-up policy work unless maintainers explicitly choose them for the first API.
