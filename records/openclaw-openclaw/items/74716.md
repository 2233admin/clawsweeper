## Summary

Keep this PR open. It is protected by the maintainer label and still changes a privileged Gateway-to-Mac MCP process path; the latest head fixed the previously reported open-timeout close path, but the node MCP HTTP proxy still ignores request aborts while startup is pending, which can leave a Mac-hosted backend alive after the HTTP caller has gone away.

## What This Changes

The PR adds node-hosted MCP protocol and proxy plumbing, Mac app Computer Use MCP hosting, Codex thread MCP config injection, pairing approval updates, generated Swift models, regression tests, and docs.

## Best Possible Solution

Keep the branch open for maintainer/platform review, make aborted node MCP HTTP requests cancel pending startup and close any transport/session they created, add a regression test for abort-before-open, then rerun the targeted gateway MCP tests, protocol check, and exact-head CI before merge.

## Reproduction Assessment

Yes. On the PR head, start a `/mcp/node/<node>/<server>` HTTP request, let it reach `ensureStarted()`, then abort the HTTP client before `node.mcp.session.open.result`; the abort signal is not checked during startup, so the proxy can still open and retain the Mac-hosted MCP session.

## Solution Assessment

No. The architecture may be the right product direction, but this is not the best mergeable form until HTTP aborts are treated like lifecycle cancellation for the node-hosted MCP startup path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cancel MCP startup when the HTTP request aborts:** `src/gateway/node-mcp-http-proxy.ts:107`
  - body: `forward()` waits for `ensureStarted()` before it registers any abort-aware waiter, so a loopback HTTP client that disconnects while `node.mcp.session.open` is pending is ignored. The gateway can still finish opening the Mac-hosted MCP process, send the abandoned JSON-RPC request, and retain the proxy session until the loopback server shuts down. Check `signal.aborted` around startup and close the transport/session when startup belongs to an aborted request.
  - confidence: 0.88

## Security Review
