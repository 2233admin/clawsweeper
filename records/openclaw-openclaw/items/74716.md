## Summary

Keep open. Current main does not implement the native OpenClaw.app-hosted Codex Computer Use route, the PR carries the protected `maintainer` label, and the latest head still has a blocking node-hosted MCP lifecycle bug where a terminal close frame before open result leaves startup waiting for timeout.

## What This Changes

The PR adds node-hosted MCP protocol/proxy support, macOS OpenClaw.app Computer Use package hosting and transfer, Codex thread MCP injection, generated Swift protocol models, SDK/runtime types, docs, and regression tests.

## Best Possible Solution

Keep the native node-hosted MCP architecture only after terminal native MCP lifecycle frames settle pending opens immediately, then complete maintainer/platform review of the protocol, macOS host, package validation, generated models, docs, and CI before merge.

## Reproduction Assessment

Yes for the review finding: start a node MCP session and deliver `node.mcp.session.closed` for the same session before `node.mcp.session.open.result`. The PR head leaves the pending open unresolved, so startup waits for the open timeout instead of failing on the terminal close frame.

## Solution Assessment

No. Routing permission-sensitive Computer Use through OpenClaw.app matches the stated direction, but the current PR is not ready until `handleMcpSessionClosed` also clears and resolves matching pending opens with the close/error result.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Settle pending MCP opens when the node closes:** `src/gateway/node-registry.ts:452-459`
  - body: `handleMcpSessionClosed` only removes the active session and notifies `onClosed`; it leaves a matching `pendingMcpOpens` entry alive. If the native host exits and sends `node.mcp.session.closed` before `node.mcp.session.open.result`, `NodeMcpClientTransport.start()` and the HTTP proxy keep waiting until the open timeout even though the session is already terminal. Clear and resolve the pending open in this handler, with a closed-before-open regression test.
