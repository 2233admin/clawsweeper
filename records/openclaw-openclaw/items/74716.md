## Summary

Keep open. The PR has the protected `maintainer` label and the latest head still contains a concrete protocol mismatch in the macOS MCP closed-session payload, so it needs an author fix plus maintainer/platform review before merge.

## What This Changes

The PR adds node-hosted MCP session protocol and proxy plumbing, Mac app Computer Use MCP hosting, Codex thread MCP config injection, pairing/protocol/docs updates, and regression coverage.

## Best Possible Solution

Fix the closed-session `signal` contract on the PR branch, then rerun the protocol, Gateway MCP proxy/transport, Codex app-server injection, and macOS Swift validation before maintainer platform review decides whether to land the native node-hosted MCP design.

## Reproduction Assessment

Not applicable as a feature PR, but the review finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes the Mac host send a numeric `signal`, while the Gateway closed-event validator accepts only a string or null.

## Solution Assessment

No, this is not the best mergeable form as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or deliberately broaden the protocol schema and regenerate clients if numeric signals are the intended contract.
