## Summary

Keep open. This is a protected maintainer-labeled PR for a privileged macOS/Gateway MCP routing change, and latest head still has a protocol mismatch that should block merge until fixed and re-reviewed.

## What This Changes

The PR adds node-hosted MCP session protocol/proxy plumbing, Mac app Computer Use MCP hosting, Codex thread MCP config injection, pairing metadata, generated Swift protocol models, docs, and regression tests.

## Best Possible Solution

Fix the closed-session `signal` contract on the PR branch, then rerun protocol, Gateway MCP proxy/transport, Codex app-server injection, and macOS validation before maintainer platform review decides whether to land the native node-hosted MCP design.

## Reproduction Assessment

Not applicable as a feature PR, but the review finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes the Mac host send a numeric `signal`, while the Gateway closed-event validator accepts only a string or null.

## Solution Assessment

No, this is not the best mergeable form as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or deliberately broaden the protocol schema and regenerate clients if numeric signals are the intended contract.
