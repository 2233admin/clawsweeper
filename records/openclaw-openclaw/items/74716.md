## Summary

Keep open. This is a protected maintainer-labeled PR for a broad privileged Mac/Gateway/Codex MCP routing path, and the latest head is not merge-ready because the Mac closed-session frame still violates the Gateway protocol schema.

## What This Changes

This PR adds node-hosted MCP session protocol/proxy plumbing, Mac app Computer Use MCP hosting, Codex thread MCP config injection, node pairing updates, generated Swift protocol models, regression tests, docs, and a plugin SDK API baseline refresh.

## Best Possible Solution

Align the Mac closed-event payload with the Gateway protocol contract, rerun exact-head Gateway/protocol/Codex/macOS validation, and land only after maintainer platform review accepts the native node-hosted MCP design.

## Reproduction Assessment

Not applicable as a feature PR, but the review finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes the Mac host send a numeric `signal`, which the Gateway closed-event validator rejects because the schema requires a string or null.

## Solution Assessment

No, this is not the best mergeable form as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or intentionally broaden the protocol schema and regenerate clients consistently if numeric signals are the desired contract.

## Review Findings

Overall confidence: 0.88
