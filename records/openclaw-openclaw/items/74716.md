## Summary

Keep open. This PR has the protected `maintainer` label, current main does not already implement node-hosted MCP sessions for Codex Computer Use, and the latest PR head still has a protocol mismatch that can drop `node.mcp.session.closed` cleanup frames.

## What This Changes

The branch adds node-hosted MCP session protocol and loopback proxy plumbing, macOS OpenClaw.app Computer Use package hosting, Codex thread MCP injection, SDK/runtime types, docs, generated Swift models, and regression tests.

## Best Possible Solution

Make the node-hosted MCP closed-session contract consistent, then review the native process ownership, package validation, pairing/admin-scope policy, generated protocol models, docs, and macOS/Gateway CI before merge.

## Reproduction Assessment

Not applicable as a feature PR, but the blocking finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes Swift send a numeric `signal`, and the Gateway validator accepts only string or null.

## Solution Assessment

No, the PR is not mergeable as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or intentionally broaden the schema and regenerate clients if numeric signals are the desired contract.
