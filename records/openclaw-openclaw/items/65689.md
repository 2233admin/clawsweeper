## Summary

Keep this PR open. Current main still drops `errorMessage` from Swift chat history models, so the intended bug fix is not implemented; however the submitted fallback needs to be gated to true assistant error turns before merge.

## What This Changes

The PR adds `errorMessage` to the Swift `OpenClawChatMessage`, uses it as fallback assistant text in chat rendering, passes it through message-copy sites, and adds a ChatViewModel regression test for history refresh after an error turn.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve `errorMessage`, pass it through all Swift chat copy/reconcile sites, render a sanitized fallback only for assistant error turns with no visible text, and add regression coverage for error-history refresh plus stale-success/tool-only cases.
