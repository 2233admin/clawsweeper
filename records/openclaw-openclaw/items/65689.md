## Summary

Keep this PR open. Current main still does not preserve `errorMessage` on `OpenClawChatMessage`, so persisted OpenClawKit chat history can lose detailed provider failures after refresh. The PR targets a real gap, but its current fallback is too broad because it can render stale/background `errorMessage` values on non-error or tool-only assistant turns.

## Best Possible Solution

Keep this PR open and request a narrow revision: preserve `errorMessage` on `OpenClawChatMessage`, carry it through copy/reconcile sites, render or synthesize fallback text only for assistant error turns with no visible text, and add regression coverage for both the intended history-refresh error case and stale-success/tool-only `errorMessage` cases.
