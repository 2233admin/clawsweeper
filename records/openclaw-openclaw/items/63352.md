## Summary

Keep this issue open. Current main still has generic exact tool-and-params loop detection, known polling/no-progress detection, ping-pong detection, unknown-tool repeats, and a global no-progress breaker, but it does not implement the requested exec-side read-only command classifier, cwd/path normalization, line-window collapsing, same-file or same-file-pair thresholds, or warning-only synthetic result before another redundant readback command executes. The open #64426 is a companion first-class read-tool proposal, not a replacement for this exec-runtime request.

## What This Changes

Add a soft per-turn exec/readback detector that classifies obvious read-only shell commands, normalizes target file paths, and warns before repeated same-file or file-pair rereads burn quota.

## Best Possible Solution

Keep this open and implement a narrow core exec/tool-loop enhancement: classify obvious read-only shell readbacks, normalize cwd plus target file paths while collapsing line-window variants, track same-file and same-file-pair counts per run/session, return a warning-only synthetic result before the next redundant read executes, emit structured telemetry, and add focused coverage for thresholds, alternating pairs, progress resets, and false-positive cases.
