## Summary

Keep open. Current main still has generic tool-call loop detection, run-scoped history, unknown-tool repeat handling, ping-pong detection, and exec outcome hashing, but it does not implement the requested exec readback classifier that normalizes file targets, collapses line-window variants, tracks same-file or same-file-pair rereads, and returns a warning-only synthetic result before another redundant readback command runs.

## What This Changes

The issue proposes adding a warning-only exec-runtime detector for repeated read-only shell commands that reread the same file or file pair within one turn.

## Best Possible Solution

Keep this open and design a narrow core guard that classifies only obvious read-only exec commands, normalizes cwd plus target files, collapses line-window variants, tracks same-file and same-file-pair rereads per run/session, emits structured telemetry, and nudges the model with a warning-only result before executing another redundant readback.
