## Summary

Keep this PR open. Current main still does not implement the PR's intended server-side dual-bind behavior for tailnet/custom bind hosts, while local gateway callers still target loopback. The PR remains a plausible focused fix candidate, but it overlaps with open client-side host-resolution work in #50751 and changes documented locality/security expectations, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose either this server-side dual-bind model or the client-side host-resolution model in #50751, then land one coherent fix with regression coverage for tailnet/custom bind internal calls, aligned docs/security expectations, and targeted live proof for gateway management plus `sessions_spawn` paths.
