## Summary

Current main still does not implement this PR's intended server-side dual-bind behavior. `resolveGatewayListenHosts()` returns only the non-loopback bind host, while local gateway calls still target `127.0.0.1`; the PR remains a plausible implementation candidate, but it overlaps open #50751's client-side host-resolution approach and needs a maintainer choice on the network/locality model.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one coherent fix model: server-side dual-bind as proposed here, client-side bind-aware host resolution as in #50751, or a deliberate combination. The chosen fix should include regression coverage for tailnet/custom bind internal calls, updated docs/security wording, and targeted live proof for CLI management plus `sessions_spawn`.
