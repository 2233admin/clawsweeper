## Summary

Keep this issue open. The report is fresh, detailed, and still matches current main: Feishu warms persistent dedupe state and dispatches into normal agent sessions, but there is no Feishu-specific quarantine or repair path for corrupt runtime/dedup state or stale Feishu session rows before dispatch.

## What This Changes

Prevent stale or corrupt Feishu runtime, dedup, or session state from causing sustained gateway CPU loops, with safe quarantine, logging, and repair guidance.

## Best Possible Solution

Keep this open and investigate in the Feishu plugin first. The best fix is a Feishu-owned defensive path that detects corrupt Feishu dedupe/runtime state, quarantines or rebuilds it with a clear warning, and either isolates stale `agent:*:feishu:*` session rows before dispatch or adds a narrowly scoped doctor/session repair that archives only Feishu-owned artifacts while preserving App ID and secret.
