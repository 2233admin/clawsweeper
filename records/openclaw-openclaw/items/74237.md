## Summary

Keep this open. Current main still lacks a Feishu-owned repair/quarantine path for corrupt local Feishu state or stale direct Feishu session entries, and #74397 is an open same-author implementation PR with closing syntax, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

Prevent stale or corrupt Feishu runtime/dedup state and Feishu-scoped direct channel session state from putting the gateway into a sustained hot loop.

## Best Possible Solution

Review and land #74397 or an equivalent Feishu-owned doctor/state repair that detects corrupt `~/.openclaw/feishu` JSON and stale direct Feishu session entries, archives or rebuilds only that scoped state, and keeps credentials and unrelated sessions intact.
