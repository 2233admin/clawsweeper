## Summary

Keep open. Current main still lacks a Feishu doctor/state repair adapter and only has dedupe warmup, generic session repair, and a separate empty-message guard; #74397 is an open unmerged same-author PR with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that stale or corrupt Feishu channel/session state stop being able to drive the gateway into a sustained high-CPU loop.

## Best Possible Solution

Land #74397 or an equivalent Feishu-owned doctor repair that detects corrupt Feishu local JSON/state and suspicious Feishu direct channel session artifacts, archives only Feishu-scoped repair targets, preserves credentials and unrelated sessions, and adds focused Feishu doctor coverage.

## Solution Assessment
