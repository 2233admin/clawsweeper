## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-431`
  - body: The PR replaces the `createHttp1*` dispatcher helpers with raw Undici constructors in `createPinnedDispatcher`. Current main centralizes `allowH2: false` in those helpers because guarded networking relies on HTTP/1-only dispatcher overrides, so this can re-enable HTTP/2 on SSRF-guarded pinned fetch paths. Keep the helpers or explicitly preserve the HTTP/1-only option.
  - confidence: 0.88
- **[P1] Isolate recovery before mission cycles:** `src/claw/runtime.ts:76`
  - body: `drainClawRuntimeQueue` awaits `recoverInterruptedMissions()` before `runMissionCycles()` and only sets `state.recovered = true` after recovery succeeds. If one bad mission throws during recovery, every wake exits through the shared catch before other queued or active missions can progress. Handle recovery failures separately so one corrupt recovery record does not stall the whole runtime.
  - confidence: 0.86
- **[P2] Reload Claw dashboard only when needed:** `ui/src/ui/app-gateway.ts:424-425`
  - body: This handler reloads the full Claw dashboard for every `claw.*` event on every connected UI, even when the user is on another tab. Runtime cycles can emit multiple Claw events, so unrelated tabs will fan out repeated mission, audit, and artifact RPCs in the background. Gate this on the Claw tab or apply targeted event updates.
  - confidence: 0.78

## Security Review

Concerns:

The PR's `createPinnedDispatcher` switches to raw Undici constructors, which can drop that protection on direct and proxy pinned-dispatcher paths.
