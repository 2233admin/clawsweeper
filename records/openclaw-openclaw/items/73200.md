## Summary

Keep this PR open for normal maintainer review. Current main has not implemented the PR’s behavior, and the branch changes user-facing task-audit severity while leaving the published audit policy and changelog incomplete.

## What This Changes

The PR changes task audit so all `lost` tasks report `warn`, adds a 1-second tolerance for `startedAt < createdAt` races, and updates the colocated audit tests for the new severity order and timestamp cases.

## Best Possible Solution

Land this only if maintainers want warning-only `lost` audit findings; in that case update the public task-audit docs and changelog with the new policy. If not, keep the current warn/error split and consider merging only the timestamp-tolerance portion if direct audit inputs still need it.
