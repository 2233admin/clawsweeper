## Summary

Keep open. Current main partially addresses session-corpus noise and scan checkpointing, but the central request is still not implemented: daily and session ingestion still enter short-term recall with fixed source scores, and repeated session content can still be treated as distinct when it appears at different transcript positions.

## What This Changes

Request to improve memory-core dreaming by filtering or deduplicating noisy session corpus entries, scoring ingested content by quality and recency, and rotating scan budget so session corpus entries do not stall behind daily logs.

## Best Possible Solution

Keep the current session sanitization and checkpointing, then finish the memory-core ingestion layer with deterministic content-level dedupe, source/content/recency-aware scores before `recordShortTermRecalls`, and regression coverage for large multi-file session corpus rotation under tight budgets.
