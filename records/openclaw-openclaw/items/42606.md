## Summary

Keep this PR open. The requested noVNC bootstrap hardening is still valid, but current main has not implemented it on the active browser plugin path, and this PR still targets the removed `src/browser` path rather than `extensions/browser`.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and keep the existing observer-token redirect semantics. Then review and land that current-path patch rather than merging the stale-path branch.

## Risks / Open Questions

- Merging this branch as written would not harden the active `/sandbox/novnc` route because it changes the removed `src/browser` path.
- Until ported, the token-gated noVNC bootstrap still serves an inline redirect script without a document CSP, script nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`.
- The port must preserve the current one-time observer token behavior, avoid adding a `Location` header, and keep the noVNC password in the URL fragment rather than query string or response headers.
