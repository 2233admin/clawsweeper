## Summary

Keep this open. Current `main` still serves the active `/sandbox/novnc` bootstrap from the browser plugin without the requested CSP nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`; the branch’s useful hardening is aimed at stale `src/browser` paths instead of the live `extensions/browser` implementation. The prior maintainer comments in the provided thread match the current code review: the idea is valid, but it needs to be ported before merge.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and preserve the existing observer-token redirect behavior.

## Risks / Open Questions

- Merging the branch as written would not change the active noVNC observer route because the live implementation is under `extensions/browser`, not `src/browser`.
- Until this is ported, the token-gated noVNC bootstrap still serves an inline redirect script without a document CSP, script nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`.
- The port needs to preserve the one-time observer token behavior, avoid a `Location` header, and keep the noVNC password in the URL fragment only.
