## Summary

Keep this PR open. The hardening is still valid, but current main has not implemented it on the live browser plugin path; the branch changes stale `src/browser` files while `/sandbox/novnc` is now served from `extensions/browser` without the requested CSP nonce, `nosniff`, or frame-deny headers.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and preserve the current observer-token redirect behavior.

## Risks / Open Questions

- Merging the branch as written would not change the active `/sandbox/novnc` route because current main serves it from `extensions/browser`, not `src/browser`.
- Until the patch is ported, the token-gated noVNC bootstrap still serves an inline redirect script without a document CSP, script nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`.
- The port must preserve the existing one-time observer token behavior, avoid a `Location` header, and keep the noVNC password in the URL fragment only.
