## Summary

Keep this issue open. Current main still has per-agent tool-name allow/deny plus positive exec allowlist handling, including shell-script path allowlist matching, but it does not expose or enforce a typed per-agent negative policy for shell wrapper/script paths. This remains a real core exec-policy gap, not an implemented feature, duplicate, stale report, or ClawHub/plugin item.

## Best Possible Solution

Keep this issue open for maintainer design and implementation. The best direction is a typed, documented per-agent exec path deny policy that reuses existing shell wrapper/script candidate resolution, applies before approval prompts and durable approval reuse, makes deny win over allowlist/safe-bin/approval paths, covers gateway and node exec, and adds regression tests plus docs for `sh|bash <script>` and interpreter script operands.
