## Summary

Keep this PR open. Current main still lets `cron.add` and `cron.update` pass schema-valid but syntactically invalid cron schedules through to the cron service, where Croner parsing can throw and fall into the WebSocket catch-all as `UNAVAILABLE`. The proposed PR is a narrow implementation candidate for the linked bug, and the provided diff does not show security-sensitive workflow, dependency, lockfile, permission, package-publishing, or secrets changes.

## Best Possible Solution

Review and land this PR or an equivalent gateway-boundary fix. The fix should validate cron schedules for `cron.add` and `cron.update` before mutating the cron service, map Croner schedule parse failures to `INVALID_REQUEST`, preserve existing valid `cron`, `every`, and `at` semantics, and keep focused regression coverage in `src/gateway/server-methods/cron.validation.test.ts`. After merge, #74066 can close through the PR.
