## Summary

Keep this PR open. Current main still accepts schema-valid but syntactically invalid cron expressions through the gateway handler and lets Croner parse failures escape through the WebSocket catch-all as UNAVAILABLE. The PR is a narrow implementation candidate for #74066, with focused handler and regression-test changes, and its diff does not introduce security-sensitive workflow, dependency, lockfile, publishing, permission, or secrets changes.
