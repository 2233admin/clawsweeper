## Summary

Keep this PR open. Current main still does not implement the PR’s intended opt-in Linuxbrew Docker support, and the linked Docker skill-install bug remains open. The PR is still relevant, but the submitted diff is not merge-ready because it adds unpinned remote installer execution and has concrete Dockerfile hazards already called out in review.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. The best path is to choose one canonical fix for #14593: revise this PR or a replacement to add Docker-safe opt-in Linuxbrew support, or intentionally adopt an alternate product behavior, with conditional environment handling, setup/docs coverage, focused tests, and an explicit supply-chain policy for any Homebrew installer execution.

- The linked Docker skill-install bug remains unresolved until a revised Docker-safe implementation lands; #55940 and #57553 are related open alternatives but are not merged.
