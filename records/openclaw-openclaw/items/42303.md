## Summary

Keep this PR open for maintainer follow-up. Current main still lacks the requested opt-in Linuxbrew Docker build support, and the linked Docker skill-install bug remains active. The PR is relevant, but it is not merge-ready because the submitted Dockerfile adds risky build-time remote installer execution and has concrete functional hazards already called out in review.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Choose one canonical fix for #14593: revise this PR or a replacement to provide Docker-safe opt-in Linuxbrew support, or intentionally adopt an alternate product behavior such as hiding/falling back from brew-only skill installs in Linux containers. The final path should include conditional environment handling, Docker setup/docs coverage, focused tests, and an explicit supply-chain policy for any Homebrew installer execution.
