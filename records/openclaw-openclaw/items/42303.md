## Summary

Keep this PR open. Current main still does not implement the requested opt-in Linuxbrew Docker build support, and the linked Docker skill-install bug remains open. The PR is a relevant implementation candidate, but it is not merge-ready because the submitted Dockerfile introduces security-sensitive remote installer execution and concrete functional hazards already called out in review.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. Choose a canonical fix for #14593: either revise this Docker build approach to provide safe opt-in Linuxbrew support with conditional environment handling, docs, and explicit installer integrity policy, or land a different product path such as Linux/container fallback behavior or hiding brew-only installs in containers. Once a canonical fix merges, close or supersede the remaining implementation PRs against that shipped path.

- Closing this PR now would remove one active implementation path while #14593 and alternative fixes #55940 and #57553 remain unresolved.
