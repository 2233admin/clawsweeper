## Summary

Keep this PR open. Current main still does not implement the intended opt-in Linuxbrew Docker build/setup support, and the linked Docker skill-install bug remains open. The PR is relevant, but it is stale against the current Docker setup layout and is not merge-ready as-is after the security/functional review pass.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is to pick the canonical fix for #14593: either a safe opt-in Linuxbrew Docker build/setup flow in the current Dockerfile and scripts/docker/setup.sh, with conditional runtime environment handling, docs, regression coverage, and installer integrity policy; or a different shipped product path such as hiding brew-only installs in Linux containers or providing non-brew Linux installers. After one path lands on main, close or supersede the remaining PRs against that shipped implementation.
