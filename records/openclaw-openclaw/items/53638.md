## Summary

Keep open. Current main implements the channel/group portion through the shared `channels.modelByChannel` surface, including the Discord channel use case raised in discussion, but the original per-DM request is still not wired because direct-message identifiers are not passed into model override resolution.

## What This Changes

The issue requests persistent model selection overrides for individual channel, group, and direct-message conversations in OpenClaw configuration.

## Best Possible Solution

Complete the generic `channels.modelByChannel` resolver for direct conversations by deriving stable DM candidate keys from finalized context, persisted session origin, native provider identifiers, direct delivery targets, and the current direct session key, then document those accepted key forms without adding provider-specific nested `model` fields.
