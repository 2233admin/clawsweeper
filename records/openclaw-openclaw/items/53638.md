## Summary

Keep open. Current main partially satisfies the request through the shared `channels.modelByChannel` surface for channel/group IDs, including the Discord channel use case raised in discussion, but ordinary DM sessions still do not supply direct-peer identifiers to the resolver, so persistent per-DM model pins remain missing.

## What This Changes

Request persistent config-based model overrides for specific Slack/WhatsApp channels, groups, and DMs, with channel/group/DM-specific resolution ahead of the global default.

## Best Possible Solution

Complete the generic `channels.modelByChannel` resolver for direct conversations by deriving normalized DM candidate keys from finalized message context, stored session origin, native provider IDs, and direct session keys while preserving existing session override and channel/group precedence. Document the accepted DM key forms and add Slack, WhatsApp, Discord, and generic resolver regression coverage without adding provider-specific `model` fields.
