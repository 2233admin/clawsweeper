## Summary

Keep this issue open. Current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 still has active-schema validation, doctor legacy repair, unknown-key pruning, and a Control UI pending-change diff, but it does not implement the requested `openclaw doctor --config-diff` CLI workflow or a version-to-version migration report driven by `meta.lastTouchedVersion`. The related open issues are adjacent rather than superseding this CLI migration-diff request.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly reject a CLI upgrade-diff workflow. A complete solution should add `openclaw doctor --config-diff` command plumbing, docs, and tests; compare the active config against the current schema and a trustworthy prior-version baseline from `meta.lastTouchedVersion`; report unknown keys, invalid enum values, removed/renamed/new keys, and migration suggestions; and reserve `--config-diff --fix` for explicit safe migrations with backups. Coordinate with #39780 for Control UI UX and with concrete config-regression reports like #72872, but do not close this CLI feature as already covered by them.
