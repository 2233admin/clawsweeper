## Summary

Keep this issue open. Current main at 7e5c3753f6f797d4faf7e22eefe5a11f5a3a17ed still has current-schema doctor repair, unknown-key stripping, config version stamping, and a Control UI pending-change diff, but it does not implement `openclaw doctor --config-diff` or a version-to-version migration report driven by `meta.lastTouchedVersion`. Related #39780 and #72872 are adjacent, not superseding this CLI migration-diff request.

## Best Possible Solution

Keep this open for maintainer design and implementation of a CLI upgrade-diff workflow. A complete solution should add `openclaw doctor --config-diff` plumbing, docs, and tests; define a trustworthy prior-version baseline from `meta.lastTouchedVersion` or an explicit migration inventory; report unknown keys, invalid enum values, removed/renamed/new keys, and actionable migration suggestions; and reserve `--config-diff --fix` for explicit safe migrations with backups. Coordinate with #39780 for Control UI UX and concrete regression reports like #72872, but do not close this CLI feature as already covered by them.
