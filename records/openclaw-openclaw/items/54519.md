## Summary

Keep open. Current main still lacks the requested hard per-account read-only channel mode: shared channel config, WhatsApp config/schema, gateway send/poll paths, shared outbound delivery, and WhatsApp transport helpers still have no generic account-level outbound-deny gate. The related #71382 PR remains open and WhatsApp-scoped, so it is useful implementation context rather than a reason to close this broader issue.

## What This Changes

Request a per-account channel configuration mode that continues receiving inbound messages while blocking all outbound delivery at the gateway or transport level.

## Best Possible Solution

Ship a maintainer-approved channel/account outbound policy in the shared config/schema/docs/UI metadata, then enforce it before every outbound surface: automatic replies, gateway sends, polls, message actions, reactions, typing/presence, read receipts, heartbeat/status sends, and plugin-owned transport side effects where feasible. Use #71382 as the WhatsApp subset reference, but close this broader issue only after the generic policy or an explicit maintainer-scoped replacement lands with tests and docs.
