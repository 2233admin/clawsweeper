## Summary

Keep this issue open. Current main still hardcodes inline dreaming phase output to `memory/YYYY-MM-DD.md`, and the memory-core config schema/docs do not expose an `inlineRelativePath`, `inlineTarget`, or equivalent path-template setting. The related memory flush-path cleanup is not the same surface: `memoryFlushWritePath` comes from the active memory plugin flush plan, while this issue is about memory-core dreaming phase markdown storage.

## Best Possible Solution

Implement this in the memory-core owner module as a conservative `dreaming.storage.inlineRelativePath` or equivalent target selector. Keep the current default path, validate templates as workspace-relative markdown paths with a required date token, preserve the existing managed markers, and document how it interacts with Obsidian daily-note layouts.
