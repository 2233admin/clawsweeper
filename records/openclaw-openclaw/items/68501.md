## Summary

Keep this issue open. Current main still preserves explicit inline dreaming mode but hardcodes inline light/REM output to `memory/{YYYY-MM-DD}.md`, and the memory-core config schema does not expose an inline path or target setting; the related memory flush-path cleanup does not cover this dreaming-specific writer.

## What This Changes

The issue requests configurable inline dreaming output paths for memory-core so dreaming blocks can target layouts such as `memory/daily-memory/{date}.md` while preserving managed markers and documenting Obsidian daily-notes interaction.

## Best Possible Solution

If maintainers want to support this workflow, add a memory-core-owned dreaming storage path option that defaults to the current `memory/{date}.md`, safely resolves only workspace-relative templates, keeps the existing managed block markers, and documents the Obsidian daily-notes interaction.
