## Summary

Keep this open. Current main still hardcodes inline dreaming output to `memory/YYYY-MM-DD.md`; the existing config covers enablement, cadence, model, timezone, and storage mode, but not the inline target path. This should stay in the bundled `memory-core` plugin rather than close as ClawHub work, because the requested behavior is for an existing memory-core dreaming mode and current plugin APIs do not expose this exact internal write target without reimplementing the dreaming loop.

## What This Changes

The issue requests a configurable memory-core dreaming inline output path, such as `memory/daily-memory/{date}.md`, while preserving managed dreaming markers and documenting Obsidian Daily Notes interaction.

## Best Possible Solution

Add a backward-compatible memory-core-owned inline target setting, preferably under `dreaming.storage`, with the current `memory/{date}.md` behavior as the default, safe relative path validation, preserved light/REM managed markers, focused tests, and docs covering the Obsidian `memory/daily-memory/{date}.md` layout.
