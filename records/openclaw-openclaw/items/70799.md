## Summary

Keep this PR open. Current main still lacks the PR’s helper-entry guards in `extensions/bluebubbles/src/chat.ts`: `markBlueBubblesChatRead` and `sendBlueBubblesTyping` immediately delegate to `sendBlueBubblesChatEndpointRequest`, which only checks cached Private API status after creating a `BlueBubblesClient`. The PR is narrow and still distinct from the open #69932, which guards monitor typing call sites but does not cover mark-read or the exported helper boundary.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a small helper-level guard for `markBlueBubblesChatRead` and `sendBlueBubblesTyping`, or fold the mark-read/helper guard into the broader BlueBubbles Private API fallback work in #69932. Before merge, confirm the live null-cache/account-id behavior and add or adjust a targeted regression test if the current mock path does not exercise the production failure.
