## Summary

Current main does not implement the PR's requested outer guards for BlueBubbles mark-read and typing calls. The affected inbound path still calls those helpers and logs failures at error level, and the cached Private API status helper still returns null when status is unknown, so the reported production failure path remains plausible and unsuperseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to fix the BlueBubbles-owned inbound mark-read/typing behavior in extensions/bluebubbles, either by landing a narrowly tested version of this guard or by adding a stronger root-cause fix that prevents Private-API-only read/typing calls from surfacing as user-visible fallback replies when Private API is disabled.
