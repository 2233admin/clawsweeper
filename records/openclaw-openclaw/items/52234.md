## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: reset cleanup clears followup queues and drain callbacks before the replacement session is written, queued followups still execute against their captured session binding, and current streaming/tool/block delivery lacks reset-aware stale-output suppression. The PR is a plausible implementation candidate, refreshed from closed unmerged #51655, but it is not obsolete cleanup material.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but require a revised maintainer-reviewed patch before merge. The best path is a smaller, race-tested implementation that preserves queued followups through sessions.reset, pauses and resumes preserved drains with ownership for overlapping resets, rebinds queued and in-flight work through canonical latest-session resolution, suppresses or aborts stale partial/reasoning/tool/block/final deliveries, preserves threading/session overrides, and adds targeted regression coverage for reset races.
