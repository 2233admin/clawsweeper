## Summary

Keep this PR open. Current main still implements the opposite reset behavior for the central request: gateway session reset clears followup queues and drain callbacks instead of preserving paused queued replies across the replacement session. The PR is also not safe to close as implemented because the latest review context includes unresolved reset-path regressions at the PR head, including a P0 cleanupSessionBeforeMutation call-context issue and a P1 reset-entry override preservation issue.

## Best Possible Solution

Keep the PR open for maintainer review or a revised implementation. The best path is a smaller, current-main patch that preserves queued followups across sessions.reset, pauses/ref-counts and resumes drains around the reset mutation, rebinds queued and in-flight work through canonical session resolution, suppresses or aborts stale partial/block/tool/final deliveries, and preserves existing session overrides, with targeted tests for each race.
