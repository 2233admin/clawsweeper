## Summary

Keep this PR open. Current main still has the reported Docker ownership gap: the optional browser install path creates `/home/node/.cache/ms-playwright` as root, chowns only that child directory, and later runs the image as the non-root `node` user. The related duplicate child PR was closed unmerged, and the broader Docker setup PR is still open and does not prove this exact Dockerfile ownership fix has landed.

## Best Possible Solution

Review and land this focused Dockerfile ownership change if targeted Docker validation confirms the browser-enabled image leaves `/home/node/.cache` owned by or writable to the runtime `node` user. Add a narrow Dockerfile assertion or build proof for the parent cache ownership contract if feasible.
