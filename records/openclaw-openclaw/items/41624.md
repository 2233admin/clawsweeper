## Summary

Keep this PR open. Current main still has the child-only Playwright cache ownership line in the optional browser-enabled Docker layer, the runtime image still switches to `USER node`, and the Docker docs still document `/home/node/.cache/ms-playwright`; the PR remains a focused but infrastructure-sensitive fix that needs maintainer Docker review rather than closure.

## What This Changes

This PR changes the optional `OPENCLAW_INSTALL_BROWSER` Dockerfile path to `chown -R node:node /home/node/.cache` instead of only `/home/node/.cache/ms-playwright` after installing Playwright Chromium.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land this focused parent-cache ownership fix with a narrow `src/dockerfile.test.ts` assertion and Docker validation for `OPENCLAW_INSTALL_BROWSER=1`, or explicitly supersede it with #42771 if maintainers prefer moving Playwright browsers to `/opt/playwright`; coordinate with #61167 only for adjacent compose/setup env wiring.
