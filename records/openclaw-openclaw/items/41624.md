## Summary

Current main still has the Docker cache ownership gap this PR targets. The optional browser-install block creates `/home/node/.cache/ms-playwright` as root, only chowns that child directory, and the runtime image later switches to `USER node`; the PR’s one-line change to chown `/home/node/.cache` has not landed or been superseded.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The maintainer path is to review the one-line Docker ownership change, request or attach targeted Docker proof for `OPENCLAW_INSTALL_BROWSER=1`, and merge it if validation confirms `/home/node/.cache` is writable by the runtime `node` user without unintended image ownership side effects.
