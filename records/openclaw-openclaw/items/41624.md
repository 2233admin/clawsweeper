## Summary

Keep this PR open. Current main still creates and chowns only `/home/node/.cache/ms-playwright`, so the parent `/home/node/.cache` ownership problem described by the PR remains possible in the browser-enabled Docker image. The diff is narrow, but Docker image permission changes are infra-sensitive and there is an open competing direction in #42771 that would move the browser cache to `/opt/playwright`, so this needs maintainer Docker review rather than cleanup closure.

## What This Changes

This PR widens the optional Docker browser-install ownership step from `chown -R node:node /home/node/.cache/ms-playwright` to `chown -R node:node /home/node/.cache`.

## Best Possible Solution

Land one coherent Docker browser-cache policy: either merge this focused parent-cache ownership fix with a narrow Dockerfile regression assertion and browser-enabled image validation, or supersede it with #42771 if maintainers prefer `/opt/playwright`; keep docs and Docker setup examples aligned with whichever path is chosen.
