## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the optional Docker browser-install path and only chowns that child directory before switching to `USER node`; the PR's parent-cache ownership change has not landed. The latest release tag still has the same child-only chown. The one-line PR diff is functionally scoped and has no obvious supply-chain or secrets regression, but it is Docker image ownership behavior and should get maintainer Docker validation.
