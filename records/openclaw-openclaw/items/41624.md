## Summary

Keep open. Current main still chowns only `/home/node/.cache/ms-playwright`, while this PR is a focused one-line Dockerfile fix that makes the parent cache directory writable for the runtime `node` user. The patch looks functionally correct, but it should add the required user-facing changelog entry and get Docker/browser-install validation before merge.

## What This Changes

The PR changes the optional Docker browser-install block to chown `/home/node/.cache` instead of only `/home/node/.cache/ms-playwright`.

## Best Possible Solution

Merge the focused ownership fix after adding release-note credit and validating the browser-enabled Docker build path; keep any broader Playwright path migration isolated to #42771 or a replacement design.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: the Dockerfile creates `/home/node/.cache/ms-playwright` as root, chowns only the child directory, and then runs the final image as `node`, leaving parent-cache writes vulnerable in browser-enabled containers.

## Solution Assessment

Yes. For the current documented `/home/node/.cache/ms-playwright` path, widening the ownership target to `/home/node/.cache` is the narrowest maintainable fix; the safer addition is release-note coverage and targeted Docker validation, not a broader Docker rewrite in this PR.
