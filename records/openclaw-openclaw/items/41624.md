## Summary

Keep this PR open. Current main still has the Docker cache ownership gap the PR targets: the optional browser install creates `/home/node/.cache/ms-playwright` as root and only chowns that child directory, while the runtime image later runs as `node`. The related duplicate PR was closed unmerged as a child, and this PR remains the active focused implementation candidate.

## Best Possible Solution

Review and land this focused Dockerfile change if targeted Docker validation confirms the optional browser-install image has `/home/node/.cache` writable by the runtime `node` user, ideally adding a narrow Dockerfile assertion or build proof for the ownership contract.
