## Summary

Keep open. Current main still writes Gateway file logs as a single JSONL stream and formats that stream only at read time in the CLI/UI; there is no persistent derived plain-text companion file. The earlier duplicate is closed, the same-author implementation PR is closed unmerged, and the merged date-boundary log discovery PR is adjacent but does not satisfy this request.

## What This Changes

The issue requests a derived human-readable formatted log file beside the existing JSONL file log while leaving the JSONL path and consumers unchanged.

## Best Possible Solution

Land a focused logger enhancement that derives a readable companion path from the active logging file, writes redacted plain-text lines beside the unchanged JSONL stream, preserves CLI/Control UI/script JSONL consumers, and documents/tests redaction, rotation, date-boundary behavior, and cleanup semantics.
