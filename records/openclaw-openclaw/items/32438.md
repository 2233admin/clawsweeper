## Summary

Keep this issue open. Current main still writes only the JSONL gateway log file, docs still describe file logs as JSON objects per line, and the existing CLI/Control UI paths depend on parsing that JSONL file rather than writing a human-readable sidecar. The related earlier issue #29378 is closed and the paired implementation PR #32442 is closed unmerged, so there is no canonical open PR or shipped fix to close against.

## Best Possible Solution

Keep this issue open and implement a focused logging enhancement: derive a human-readable sidecar path from the active JSONL log path, write formatted redacted lines beside the unchanged JSONL file, document the behavior, and keep CLI/Control UI JSONL consumers unchanged. The sidecar should follow current rolling/day-change/rotation behavior rather than introducing an unbounded second log surface.
