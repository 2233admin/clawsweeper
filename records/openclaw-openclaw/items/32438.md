Implement a focused replacement PR for this issue: add a persistent human-readable formatted log file beside the existing Gateway JSONL file when file logging is enabled. Keep `logging.file` and the JSONL format unchanged, and do not change `logs.tail`, Control UI, or scripts to consume the sidecar. Derive the sidecar path from the active file path by inserting `-formatted` before the extension, write redacted plain-text lines in the requested `YYYY-MM-DD HH:mm:ss [level] [subsystem] message` shape, and handle current dated rolling paths plus size-based rotation/pruning deliberately.
