## Summary

Keep this issue open. Current main still classifies ended subagent orchestrators with pending descendants as active, and the native and ACP spawn quota paths use that active-run count. The related #71252/#71264 stale-unended orphan work fixed a different liveness case and deliberately preserved pending-descendant activity semantics.

## Best Possible Solution

Keep the issue open and resolve it with an explicit subagent liveness policy split: ended orchestrators with pending descendants can remain cleanup-aware for announce and descendant settlement, but should not appear as active user work or consume the requester session's spawn quota if maintainers accept the reporter's expected behavior. Update `subagent-list`, `subagent-registry-queries`, native and ACP spawn quota behavior, regression tests, and `docs/tools/subagents.md` together.
