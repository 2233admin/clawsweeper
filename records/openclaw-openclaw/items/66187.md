## Summary

Keep this issue open. Current main still treats ended subagent orchestrators with pending descendants as active, and both native and ACP spawn quota paths consume that active-run count. The related stale-unended orphan fix for #71252/#71264 addressed runs with no `endedAt`; it deliberately leaves this ended-with-pending-descendants policy in place.

## Best Possible Solution

Keep this open and resolve it with a maintainer-approved liveness policy split: ended orchestrators with pending descendants can remain cleanup-aware for announce/settlement, but should not appear as active user work or consume the requester session's spawn quota if maintainers accept the reporter's expected behavior. Update `src/agents/subagent-list.ts`, `src/agents/subagent-registry-queries.ts`, native and ACP spawn quota paths, regression tests, and `docs/tools/subagents.md` together.
