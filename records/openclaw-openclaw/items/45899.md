## Summary

Keep this PR open. Current main has the broader `MEDIA:` local-path documentation, but it still does not implement the PR's Feishu-specific blocked-local-media hint or one-time final-reply fallback. The Feishu reply dispatcher still sends media without `mediaLocalRoots` and its error path only logs, closes streaming, and clears typing state.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, preserve the newer `MEDIA:` documentation model, and decide whether to land the narrow Feishu runtime hint/fallback alongside or after the related Feishu `mediaLocalRoots` plumbing.
