## Summary

Current main still preserves the reported TUI footer behavior: `formatTokens(null, 200000)` renders `tokens ?/200k` because missing totals are labeled `?` and percent is only calculated for numeric totals. The footer calls that formatter directly with `sessionInfo.totalTokens` and `sessionInfo.contextTokens`. Gateway transcript fallback improves sparse sessions with recoverable nonzero usage, but it does not remove the reachable known-context/missing-total TUI state. The related PR #43044 proposed the narrow fallback but is closed and unmerged in the provided context, so this issue should stay open.

## Best Possible Solution

Keep this issue open. Apply a narrow TUI formatter fix in `src/tui/tui-formatters.ts`: when `context` is known and `total` is null or undefined, render `tokens 0/<ctx> (0%)` instead of `tokens ?/<ctx>`. Add focused regression coverage in `src/tui/tui-formatters.test.ts` and a refresh-path test where defaults provide `contextTokens` but the selected session row has no `totalTokens`.
