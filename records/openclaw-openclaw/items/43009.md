## Summary

Current main still contains the reported TUI footer behavior: `formatTokens(null, 200000)` renders `tokens ?/200k`, and the footer passes nullable session token fields directly into that formatter. The linked fix PR #43044 is closed and unmerged, and there is still no focused formatter regression covering the known-context/missing-total case.

## Best Possible Solution

Keep this issue open. The narrow maintainer fix should make the TUI footer match status semantics for the known-context/missing-total case, either by carrying an explicit zero total into `sessionInfo` or by updating `src/tui/tui-formatters.ts` to render `tokens 0/<ctx> (0%)` when the selected session has a known context and no total. Add focused coverage in `src/tui/tui-formatters.test.ts` and a refresh-path regression in `src/tui/tui-session-actions.test.ts` where defaults provide `contextTokens` and the selected session row has no `totalTokens`.
