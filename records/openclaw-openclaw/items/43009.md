## Summary

Current main still has a reachable TUI footer state that renders `tokens ?/<context>` when the context window is known but total tokens are missing. The related fix PR was closed unmerged, and current tests do not cover the proposed formatter fallback.

## Best Possible Solution

Keep this issue open. The narrow maintainer fix belongs in `src/tui/tui-formatters.ts`: when `context` is known and `total` is null or undefined, render `tokens 0/<ctx> (0%)` instead of `tokens ?/<ctx>`. Add focused coverage in `src/tui/tui-formatters.test.ts`, plus a refresh-path regression where defaults provide `contextTokens` and the selected session row has no `totalTokens`.
