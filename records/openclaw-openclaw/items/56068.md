## Summary

Keep this open. Current `origin/main` still validates and exposes `ui.seamColor` on backend paths, but the Control UI bootstrap contract, bootstrap loader, theme application code, and CSS tokens do not transport or apply that value. The linked fix PR was closed unmerged, so the requested web UI behavior is not implemented on main.

## What This Changes

Make configured `ui.seamColor` update Control UI accent styling instead of leaving web UI accents at the built-in theme defaults.

## Best Possible Solution

Define the Control UI scope for `ui.seamColor`, then wire the optional value through the bootstrap endpoint and apply root accent CSS variables only when configured, preserving existing defaults and browser-local imported themes when absent. Add regression coverage for the gateway payload and jsdom theme application path, plus docs/changelog updates if the web behavior is shipped.
