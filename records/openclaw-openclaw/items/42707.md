## Summary

Keep this PR open. Current main still lacks Telegram subagent lifecycle hook registration, and resumed agent runs still do not hydrate requester routing from persisted session delivery context, so the PR’s central Telegram `sessions_spawn(thread=true, mode="session")` behavior is not implemented. Main has landed some supporting infrastructure, including Telegram thread-binding state and shared session-binding adapters, but the missing hook bridge and resume hydration remain. The related feature tracker #42986 is still open, and this PR remains a relevant but stale implementation candidate.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. Rebase it onto current main; add Telegram `registerFull` lifecycle hook wiring using plugin SDK/local Telegram API facades; reuse the current Telegram thread-binding manager and shared session-binding adapter infrastructure; add any needed prewarm helper behind a public/local boundary; hydrate `resolveAgentRunContext` from the resolved `SessionEntry` on `--session-id` resumes; normalize legacy Telegram target prefixes; and add focused tests for Telegram `sessions_spawn(thread=true, mode="session")`, resume routing hydration, delivery-target resolution, and unbind cleanup. Keep #42986 as the user-facing tracker until a corrected implementation lands.
