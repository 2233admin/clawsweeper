## Summary

Current main still lacks a Feishu handler for `im.chat.access_event.bot_p2p_chat_entered_v1`. The default Feishu WebSocket path registers a fixed handler map, passes that dispatcher to the Lark SDK WebSocket client, and no code/docs/tests currently mention the reported event. The related implementation PR #42434 is closed unmerged, so the issue remains actionable.

## Best Possible Solution

Keep this issue open. Add a narrow Feishu-owned no-op or low-noise handler for `im.chat.access_event.bot_p2p_chat_entered_v1` in `extensions/feishu/src/monitor.account.ts`, with regression coverage proving the handler is registered on the WebSocket monitor path and does not stop the Feishu channel or Gateway transport. A fresh focused PR or carefully extracted patch is preferable to reviving closed mixed-scope PR #42434 wholesale.
