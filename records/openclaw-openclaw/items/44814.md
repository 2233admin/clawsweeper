## Summary

Keep this PR open. Current main partially replaced the old hardcoded 5,000,000-byte call sites with a shared resolver, but it does not implement the PR's intended WebUI/Control UI config surface and still rejects the linked 7,322,379-byte image through the current 6 MiB image cap. The PR is stale relative to main, but it remains useful prior art for #71142 rather than something that should be closed as implemented.

## Best Possible Solution

Keep this PR open as useful prior art, but require a current-main refresh. The best path is to coordinate with #71142, choose a maintainer-approved public config surface such as `gateway.controlUi` or a shared chat attachment policy, route `chat.send`, Gateway `agent.request`, and node-event `agent.request` through one resolver, reconcile the 6 MiB image cap and transport/staging ceilings, update strict zod/types/generated schema/docs, and add regression tests for default behavior, raised-limit acceptance, oversized rejection, and user-facing error reporting.
