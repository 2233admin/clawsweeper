## Summary

Keep this PR open. Current main partially supersedes the old hardcoded 5,000,000-byte call sites with `resolveChatAttachmentMaxBytes(cfg)`, but it does not implement the PR's proposed `gateway.webui.attachmentMaxBytes` config surface and the concrete 7,322,379-byte image case still exceeds the fixed 6 MiB image cap. The PR remains useful prior art for #71142, but needs a current-main refresh and maintainer API decision before merge.

## Best Possible Solution

Keep this PR open as useful prior art, but do not merge it as-is. The best path is to coordinate with #71142, choose a maintainer-approved public policy surface, likely the existing `agents.defaults.mediaMaxMb` or a documented `gateway.controlUi` field, route `chat.send`, Gateway `agent.request`, and node-event `agent.request` through one resolver, reconcile the fixed 6 MiB image cap with the configured policy, expose an upload policy to the Control UI, add browser-side prevalidation, document the effective limits on the public Control UI docs page, and add regression tests for defaults, raised-limit acceptance, oversized rejection, and transport-ceiling behavior.
