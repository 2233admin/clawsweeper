## Summary

Keep this PR open. Current main still does not implement the intended google-vertex ADC runtime-auth fix: ADC resolves to the non-secret `gcp-vertex-credentials` marker, embedded run and compaction can still store that marker as runtime auth, and the Google transport still serializes raw auth strings as `x-goog-api-key`. The PR also needs a current-main rebase because the run path has moved to `run/auth-controller.ts`, and the provided diff includes an unrelated GitHub Copilot compaction token exchange that needs maintainer review.

## Best Possible Solution

Keep this PR open, but require a rebase and current-main implementation. The best fix should handle the current `gcp-vertex-credentials` non-secret marker, or exchange ADC into real request/Bearer auth, in both `src/agents/pi-embedded-runner/run/auth-controller.ts` and `src/agents/pi-embedded-runner/compact.ts`; preserve real google-vertex API keys; add focused regression tests for run and compaction; and remove or split the unrelated GitHub Copilot compact-flow token change.
