## Summary

Keep this PR open. Current main still does not implement the intended google-vertex ADC runtime-auth fix: ADC resolves to the non-secret `gcp-vertex-credentials` marker, the embedded run and compaction paths can still store that marker as runtime auth when no provider runtime hook returns a real credential, and the Google transport still serializes raw strings as `x-goog-api-key`. The PR is also stale against the current `run/auth-controller.ts` split and carries an unrelated Copilot compact-flow token exchange concern that needs maintainer review.

## Best Possible Solution

Keep this PR open as a useful bug-fix candidate, but require a rebase and current-main implementation. The best fix should suppress non-secret google-vertex ADC markers, or exchange ADC into real request/Bearer auth, in both `src/agents/pi-embedded-runner/run/auth-controller.ts` and `src/agents/pi-embedded-runner/compact.ts`; preserve real google-vertex API keys; add focused regression tests for run and compaction; and remove or split the unrelated Copilot compact-flow token change.
