## Summary

Keep open. Current main still starts native `sessions_spawn` child runs without passing `inputProvenance`, while the existing inter-session provenance contract and `sessions_send` path already support the requested metadata. The issue is also paired with open draft PR #47087 using closing linkage, so it should remain open until that PR or an equivalent patch lands.

## What This Changes

Attach explicit inter-session `inputProvenance` to child tasks created by `sessions_spawn`, preserving source session/channel where available and setting `sourceTool` to `sessions_spawn`.

## Best Possible Solution

Land #47087 or an equivalent narrow patch that constructs `inter_session` provenance at the native spawn boundary, passes it into the child gateway `agent` request, preserves requester session/channel lineage when present, and adds focused regression coverage in the existing subagent spawn test seam.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to call `spawnSubagentDirect()` through the existing mocked gateway seam and inspect the child `agent` params; current main does not include `inputProvenance`.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing `InputProvenance` contract in `spawnSubagentDirect()` rather than redesigning provenance or broader multi-agent policy.
