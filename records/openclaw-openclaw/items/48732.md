## Summary

Keep this PR open. Current main still does not contain the PR's healthcheck-specific instruction to use user-local timestamps in alerts, reports, and logs. The linked issue was closed based on generic cron current-time injection, but that is not the same as implementing this PR's intended skill wording, especially because cron context still includes a UTC anchor that a model can copy into user-facing output.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current `skills/healthcheck/SKILL.md`. The best implementation is an additive core rule telling healthcheck to render user-facing alert/report/log timestamps in the user's local timezone from `Current time:` or `agents.defaults.userTimezone`, while allowing UTC as a secondary comparison value when useful.
