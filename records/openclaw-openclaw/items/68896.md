## Summary

Keep open. Current main does not implement the requested short-click auto-send plus 3-second hold-to-continuous dictation UX. The old browser dictation button and STT state have been removed, while the remaining Control UI voice surface is the separate realtime Talk control, so this now needs maintainer product/design judgment rather than cleanup closure.

## What This Changes

Request a smarter Control UI voice input button where short click records one message and auto-sends on pause, while a 3-second hold enters continuous auto-send voice mode with interim transcript and mode-specific tooltip copy.

## Best Possible Solution

Maintainers should decide whether Control UI should restore browser dictation as a one-shot/continuous accessible mic UX or keep realtime Talk as the only browser voice surface. If accepted, implement the behavior in source with explicit modes, pointer/touch/keyboard accessibility, no global recognition state, targeted chat view tests, and aligned Control UI docs.
