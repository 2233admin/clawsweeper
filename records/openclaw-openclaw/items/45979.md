## Summary

Keep this PR open. Current main still contains the shrinking loop-bound bug in gateway dedupe cleanup, and that maintenance timer is wired into normal gateway startup. The PR is a narrow one-file fix with no observed security-sensitive scope, but it should gain a focused regression test before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the minimal precomputed `excess` fix in `src/gateway/server-maintenance.ts` and add a focused regression test in `src/gateway/server-maintenance.test.ts` that overfills the dedupe map, advances the cleanup timer, and asserts the cache trims to `DEDUPE_MAX` with the oldest entries removed.
