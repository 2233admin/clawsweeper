## Summary

Keep this PR open. Current main still has the exact shrinking-loop-bound bug this PR targets in gateway dedupe cleanup, and the affected maintenance timer is wired into normal gateway startup. The PR is a narrow one-file functional fix with no observed security- or supply-chain-sensitive changes, but it should gain a focused regression test before landing.

## Best Possible Solution

Keep this PR open for review. The best path is to land the minimal precomputed `excess` fix in `src/gateway/server-maintenance.ts` and add a focused regression test in `src/gateway/server-maintenance.test.ts` that overfills the dedupe map, advances the cleanup timer, and asserts the cache trims to `DEDUPE_MAX` with the oldest entries removed.
