## Summary

Keep open. Current main still lacks the proposed IPv4-only BYOH clarification in the gateway CLI docs, gateway config type docs, and no-GUI onboarding hint, while the current runtime code supports the PR premise that the BYOH-facing `lan`, `tailnet`, and `custom` paths are IPv4-oriented today. The provided PR diff appears narrowly scoped and does not introduce workflow, dependency, lockfile, package, release, or other security-sensitive changes.

## What This Changes

The PR updates gateway CLI documentation, gateway config TSDoc, the no-GUI Control UI SSH hint, and an onboard helper test to state that BYOH `lan`, `tailnet`, and `custom` bind paths are currently IPv4-only and to recommend an IPv4 sidecar/proxy for IPv6-only hosts.

## Best Possible Solution

Adapt and land a docs-focused patch that documents the current IPv4-only BYOH contract in the CLI/config/onboarding surfaces, keeps any implementation comments aligned with the current `src/gateway/control-ui-links.ts` location, and leaves native IPv6 BYOH support as a separate behavior change if maintainers want it.
