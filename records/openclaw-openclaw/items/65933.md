## What This Changes

The PR changes host exec security resolution to ignore model-supplied `security` arguments except for operator-granted elevated full mode, and adds regression coverage plus a new `maxSecurity` policy helper/test.

## Best Possible Solution

Land a narrow fix that treats configured exec security as the final operator policy while preserving the explicit elevated-full override, removes the unused imports and rejected `maxSecurity` helper/test, keeps focused regression tests for full-plus-allowlist and allowlist behavior, and updates schema/docs wording if the model-facing `security` field remains accepted only for compatibility.

## Solution Assessment

No, not in its current form. Ignoring model-supplied security while keeping elevated full as the only override is the right runtime direction, but the branch needs the CI-blocking imports and stale `maxSecurity` API removed before it is the best mergeable solution.
