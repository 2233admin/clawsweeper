## Summary

Keep this PR open. Current main still lets the exec tool's model-supplied `security` argument reduce the configured exec policy via `minSecurity`, so the reported `tools.exec.security="full"` plus model `security:"allowlist"` failure remains real. The PR is directionally correct, but its head still needs maintainer security review and cleanup before it should land.

## What This Changes

The PR changes exec handling to ignore model-supplied security arguments in favor of configured exec security, adds a security-floor regression test file, and adds a new `maxSecurity` helper with tests.

## Best Possible Solution

Land a narrow exec-policy fix that makes configured/session exec security authoritative for model tool calls, preserves the explicit operator-granted elevated-full override, removes the unused `maxSecurity` helper/tests, adds regressions for `full` with model `allowlist`, `allowlist` with model `deny`, and `deny` with model `full`, and makes the model-facing `security` schema/docs compatibility decision explicit.
