## Summary

Keep issue #59164 open. Current main still routes `api.runtime.subagent.run()` through the plugin Gateway subagent runtime to the generic `agent` RPC path, while the subagent lifecycle registry is only populated by the dedicated subagent spawn paths. The `subagent_ended` hook is emitted from `SubagentRunRecord` lifecycle/completion handling, so the SDK path still appears unable to fire the hook on run completion.

## Best Possible Solution

Keep the issue open and add a focused Gateway/plugin-runtime fix: when `api.runtime.subagent.run()` dispatches a subagent-shaped `agent` run, register an appropriate `SubagentRunRecord` or route through the existing subagent spawn lifecycle helper, then add a regression test proving `api.on("subagent_ended")` fires once when that SDK-started run completes. If maintainers intentionally do not want lifecycle hooks for SDK runs, update the plugin SDK docs to state that limitation explicitly.
