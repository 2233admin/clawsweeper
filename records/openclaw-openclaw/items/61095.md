## Summary

Keep this issue open. Current main still builds host exec environments from `process.env`, the inherited-env sanitizer still does not block `OPENCLAW_SERVICE_*` or `OPENCLAW_SYSTEMD_UNIT`, and the credential code still treats `OPENCLAW_SERVICE_KIND=gateway` as a gateway-runtime signal. The provided GitHub context also shows an open same-author PR with closing syntax for this issue, so policy says the issue should remain open until that PR or a successor lands.

## Best Possible Solution

Keep the issue open while maintainers review and either land #62204 or a successor that strips internal gateway service markers from exec-spawned host environments, preserves `OPENCLAW_SHELL=exec`, updates the generated macOS policy if the shared policy changes, and adds regression coverage for both the auth-precedence failure and clean CLI process exit.
