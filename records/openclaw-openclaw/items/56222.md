## Summary

Keep this issue open. Current main still does not expose a gateway runtime version in the operator-facing `openclaw gateway status` JSON/text path, and an open unmerged implementation PR (#73225) already targets this issue, so the right maintainer action is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Review PR #73225 as the implementation candidate. The best fix is an additive, backward-compatible status change that propagates the runtime gateway version through `probeGatewayStatus`, `gatherDaemonStatus`, JSON/text printing, and the MCP-facing status surface, with focused tests for handshake metadata and `status.runtimeVersion` fallback paths.
