## Summary

Keep open. Current main still does not expose `agentId` on typed message hook context, so the PR addresses a real remaining plugin contract gap; it needs maintainer review, a rebase onto the current hook type split, docs, and one outbound identity fix before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts, threads them through inbound and outbound message hook delivery, and adds mapper, dispatch, and delivery tests.

## Best Possible Solution

Land a rebased additive contract update in the current hook type files that exposes optional `ctx.agentId`, maps it from inbound session identity and outbound resolved delivery identity, keeps mirror/session identity internally consistent, documents the new field, and covers mirror, session-only, and unset paths in targeted tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Keep mirror identity from a single source:** `src/infra/outbound/deliver.ts:411`
  - body: When `mirror.sessionKey` is present but `mirror.agentId` is absent, this falls back to `session.agentId`, so hooks can receive a mirror session key paired with the sender session's agent. That breaks the stated mirror-wins identity contract for plugins that correlate sends by session and agent; derive the agent from the selected mirror session key or leave it unset until that identity is available.
  - confidence: 0.72
