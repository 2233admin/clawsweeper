## Summary

Keep this PR open. Current main still only exposes `sessionKey` for typed message-hook contexts in some paths; it does not expose `agentId`, inbound `message_received` does not thread the resolved session agent into the public typed context, and outbound `message_sending` / `message_sent` do not pass session or mirror identity through the public typed hook context. The PR remains a focused implementation candidate, with the existing review note about adding session-only outbound coverage still relevant.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, add optional `agentId` to the public typed message hook context and canonical mapper types, thread resolved inbound session/agent identity into `message_received`, resolve outbound hook context once with mirror identity preferred over direct session identity, pass that same context to `message_sending` and `message_sent`, add no-session/session-only/mirror-wins tests for success and failure paths, and update the public message hooks docs.
