## Summary

Keep open. Current main still falls back when ACP session metadata is missing, so the PR is not obsolete, but the latest PR head has concrete P2 correctness issues and one security-sensitive side effect before it is merge-ready.

## What This Changes

This PR adds lazy ACP session initialization for ACP-configured agents in TUI reply dispatch and CLI agent command execution, exposes runtime/default fields through resolved agent config, and adds resolver/dispatch regression tests.

## Best Possible Solution

Keep the lazy ACP initialization direction, but update the branch onto current main and narrow initialization to side-effect-safe turns: skip raw model runs, recompute the CLI ACP prompt body after lazy re-resolution, include inline image-only first turns, and keep targeted tests for each path.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main returns null on ACP resolution kind "none", and the PR head paths show raw model runs can initialize ACP before skipping the ACP ready branch, while inline image-only first turns return before inline image attachment resolution.

## Solution Assessment

No. The PR is solving the right missing activation path, but it is not yet the best mergeable solution because it introduces raw-run side effects and misses two first-turn routing cases that should be fixed in the same branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Gate CLI lazy init out of raw runs:** `src/agents/agent-command.ts:472`
  - body: This condition can initialize ACP for `modelRun` / `promptMode: "none"` executions, but the later ACP branch is skipped for raw runs. That can start an external ACP runtime and write metadata before continuing through the raw/embedded path, so add the same `!isRawModelRun` gate here.
  - confidence: 0.84
- **[P2] Recompute the CLI ACP body after lazy init:** `src/agents/agent-command.ts:498`
  - body: After lazy init re-resolves the session to `ready`, `runTurn` still sends the `body` computed while ACP resolution was `none`. Internal-event turns should use `resolveAcpPromptBody(...)` for ACP, so recompute or defer body selection after the lazy re-resolve.
  - confidence: 0.88
- **[P2] Include inline images in the empty-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:391-392`
  - body: This guard returns null when there is no prompt text and no inbound media, but the normal ACP path also supports `params.images` via inline image attachments. A first image-only TUI/WebChat turn with inline images will still fall back to embedded instead of initializing ACP.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: No dependency, workflow, lockfile, or publishing changes were found, but the CLI lazy-init block can unexpectedly start an external ACP runtime for raw model runs.

Concerns:

- **[medium] Raw model runs can start ACP unexpectedly:** `src/agents/agent-command.ts:472`
  - body: The lazy CLI init condition is not gated by `!isRawModelRun`, so `modelRun` or `promptMode: "none"` can call `initializeSession()` for an ACP-configured session before the later ACP branch is skipped. That creates ACP metadata and may launch an external harness on a path that otherwise stays out of ACP.
  - confidence: 0.84
