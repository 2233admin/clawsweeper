## Summary

Keep open: current main still has the TUI ACP lazy-init gap this PR is trying to fix, and the branch direction is useful, but the latest PR head still introduces concrete CLI and first-turn dispatch regressions that should block merge.

## What This Changes

This PR adds lazy ACP session initialization for agents configured with `runtime.type: "acp"` in TUI reply dispatch and CLI agent command execution, exposes runtime/default fields through `resolveAgentConfig`, and adds resolver/dispatch regression tests.

## Best Possible Solution

Keep the lazy ACP initialization work, but narrow it to side-effect-safe turns: initialize only after policy and no-op checks, skip raw model runs, choose the CLI ACP prompt body after any lazy re-resolve, include inline image-only turns in dispatch, and add targeted tests for those cases before maintainer review/merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Recompute the CLI ACP body after lazy init:** `src/agents/agent-command.ts:497-498`
  - body: The CLI computes `body` while `acpResolution` is still `none`, so it uses the embedded/internal-context form. If this new block initializes and re-resolves to `ready`, the ACP branch later sends that stale body to `runTurn`; subagent/internal event turns should use `resolveAcpPromptBody(...)` instead. Move body selection until after lazy init or recompute it when init changes the resolution.
  - confidence: 0.88
- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:472`
  - body: This lazy-init condition is not gated by `!isRawModelRun`. For `modelRun` or `promptMode: "none"`, an ACP-configured session with missing metadata can start an external ACP runtime and write metadata, then skip the later ACP-ready branch and continue down the raw/embedded path. Add the same raw-run exclusion to the init block.
  - confidence: 0.84
- **[P2] Include inline images in the first-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:391-392`
  - body: The new guard returns `null` when there is no prompt text and no inbound media, but the normal ACP path also supports `params.images` via `resolveAcpInlineImageAttachments(...)` a few lines later. A first image-only TUI/WebChat turn with inline images will still fall back to embedded instead of initializing ACP. Include inline images in the no-op check or move attachment resolution before it.
  - confidence: 0.83
