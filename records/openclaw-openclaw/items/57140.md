## Summary

Keep this PR open. Current main does not already implement the PR’s central normalization change: the OpenAI WebSocket transport helper still compares raw provider/modelApi strings, and the regression test for whitespace/casing is absent. The work is still potentially useful, but it needs to be rebased onto the current stream-resolution split and preserve the newer endpoint/baseUrl guard.

## Best Possible Solution

Port only the still-valid normalization behavior into current main: normalize provider with normalizeProviderId and trim/lowercase modelApi inside shouldUseOpenAIWebSocketTransport, keep the existing baseUrl endpoint-class restriction and current openai-codex policy unless maintainers explicitly change it, add regression tests for whitespace/casing plus non-public baseUrls, and place any downstream defense-in-depth guard in src/agents/pi-embedded-runner/stream-resolution.ts after the refactor.
