## Summary

Keep this issue open. Current `main` mitigates the doctor orphan comparison by canonicalizing both sides, but session path resolution still returns the real symlink target and `resolveAndPersistSessionFile()` persists that value. The provided GitHub context also shows open PR #49796 with closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, or explicitly superseded.

Keep this issue open while maintainers review, land, or supersede PR #49796.
