## Summary

Keep #40039 open. Current main still does not contain the proposed symptom-first troubleshooting section for assistants that reply but lack broader tools, so the docs gap remains. The PR is salvageable, but it should be revised before merge because current main supports the `minimal` tool profile and does not expose the `Global tools profile` status row that the PR text tells users to inspect.

## Best Possible Solution

Keep #40039 open and ask for a small revision: add `minimal` to the troubleshooting guidance, align the status-check wording with current `openclaw status` behavior or sequence it after #39996, and keep detailed profile definitions linked to the canonical tools/config docs so the troubleshooting page stays compact and accurate.
