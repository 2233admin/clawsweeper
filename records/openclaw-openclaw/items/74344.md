## Summary

Current main and shipped releases already implement the requested Telegram polling watchdog escape hatch. The v2026.4.15 hardcoded 90s threshold was changed by PR #57737: main now defaults to 120000 ms and exposes bounded top-level plus per-account `channels.telegram.pollingStallThresholdMs` config, with tests and public docs.

## What This Changes

Allow operators to override or clearly document the Telegram polling stall watchdog threshold so slow first `getUpdates` startup does not force a repeated polling-runner rebuild loop.

## Best Possible Solution

Close this as implemented and keep the shipped `channels.telegram.pollingStallThresholdMs` contract. Operators on older builds such as v2026.4.15 should upgrade to a release containing #57737, then set the top-level or per-account Telegram threshold if their environment still has slow first-poll windows.
