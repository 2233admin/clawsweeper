## Summary

Keep open. Current main still chooses a matching macOS camera deviceId before applying the requested facing, so an iPhone Continuity Camera exposed as one external/unspecified device can still ignore `--facing front`. The related closed items only cover the macOS `camera.snap` allowlist, not front/back selection for Continuity Camera.

## Best Possible Solution

Keep this open for a focused macOS camera investigation. First verify on real macOS plus iPhone Continuity Camera whether AVFoundation exposes a selectable front camera path; if it does, implement that in `CameraCaptureService` and cover it with narrow selection tests. If it does not, make the CLI/tool/docs explicit that `facing` cannot be honored for single external/unspecified Continuity devices, preferably with an actionable error or warning instead of silently returning the rear camera.
