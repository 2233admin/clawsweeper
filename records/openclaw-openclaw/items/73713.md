## Summary

Keep open. Current main still has the reported diagnostic gap: the embedding command uses the default CLI wrapper that prints `String(err)`, and the shared remote embedding HTTP path still lets pre-response transport failures bubble without provider context or nested cause/code formatting.

The issue asks OpenClaw to audit the Voyage `infer embedding create` path on Node 24 and preserve underlying fetch causes such as Undici `UND_ERR_*` details instead of showing only `TypeError: fetch failed`.

## Best Possible Solution

Keep this issue open and implement a focused diagnostic fix: use redacted, cause-aware formatting in the default CLI error path and/or wrap pre-response remote embedding transport failures with provider context while preserving the original cause and stable codes.
