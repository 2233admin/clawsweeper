## Summary

Keep open. Current main does route the Voyage embedding CLI through the memory embedding provider and guarded remote HTTP path, but the reported diagnostic problem is still present: provider fetch errors bubble to the CLI and the shared CLI wrapper prints `String(err)`, which drops `error.cause` and `UND_ERR_*` details. The provided related reports are adjacent fetch-failed surfaces, not a merged canonical fix for this Node 24 Voyage embedding CLI issue.

## Best Possible Solution

Keep the issue open. The best fix is to preserve provider fetch causes in the `infer embedding create` path, likely by using the existing cause-aware `formatErrorMessage`/redaction path in the CLI error handler and adding a regression test for `TypeError: fetch failed` with an `UND_ERR_INVALID_ARG` cause. In parallel, audit the Voyage remote embedding path for any custom `fetchImpl` or dispatcher mixing and run a live Node 24 Voyage CLI smoke once credentials are available.
