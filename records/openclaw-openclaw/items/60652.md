## Summary

Current main still has the reported Discord discovery gap. `describeDiscordMessageTool` advertises `send`, message read/edit/delete, and other Discord actions, but it does not include `upload-file` or `sendAttachment`; the Discord handler still accepts file inputs only through the existing `send` path. The provided issue context also shows multiple open PRs with closing syntax for this issue, so the issue should remain open until maintainers choose and land or close a canonical fix.

## Best Possible Solution

Keep the issue open and review one canonical Discord fix, likely #61087 or #60808. The accepted change should advertise `upload-file` in Discord message tool discovery under the chosen gate, add or confirm an explicit dispatcher path that delegates to the existing Discord attachment send backend, and update Discord-specific contract/unit tests so agents can discover the file-send capability.
