## Summary

Current main resolves the reported `pnpm check` failure path through the shared tsgo wrapper: normal production typechecking now runs with declaration emit disabled unless a caller explicitly asks for declarations. The linked annotation-only PR #62612 was closed unmerged, but the user-visible failure from 2026.4.6 is fixed by the maintained tsgo policy and that policy first shipped in `v2026.4.10`.

## What This Changes

The issue reports that `pnpm check` on OpenClaw 2026.4.6 failed during tsgo with TS2883 inferred type portability errors in xAI helpers and the exec tool surface.

## Best Possible Solution

Keep the shared tsgo wrapper policy as the maintained fix for normal source-tree `pnpm check`; if maintainers later require declaration emit for bundled extension sources, track that as a separate declaration-build issue rather than reopening this check failure.

## Reproduction Assessment

Yes. The issue includes concrete 2026.4.6 `pnpm check` output, and current-main source inspection gives a high-confidence verification path: `pnpm check` now routes through `tsgo:prod` and `scripts/run-tsgo.mjs`, which injects `--declaration false` before tsgo runs.

## Solution Assessment

Yes. Disabling declaration emit in the normal check wrapper is the narrow maintained fix for the reported `pnpm check` failure, while explicit declaration builds still opt back in with `--declaration true`.
