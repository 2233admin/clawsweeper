## Summary

Keep this PR open. Current main still lacks the requested usage-panel long-name behavior: skill, tool, and file names in the context breakdown render without full-value title attributes, and the current consolidated usage stylesheet has no scoped ellipsis rule for those labels. The PR remains a focused UI fix, but it needs to be rebased/ported onto the current stylesheet layout before it can land.

## Best Possible Solution

Keep the PR open and port it onto current main: add the three `title` attributes in `ui/src/ui/views/usage-render-details.ts`, add a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, and do a focused visual check with long skill, tool, and file names before landing.
