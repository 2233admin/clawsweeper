## Summary

Keep this PR open. Current main has moved WhatsApp Web auth code into the pluginized `extensions/whatsapp` surface, but it still does not implement the PR's intended symlink rejection for primary WhatsApp credential reads or the auth-directory mode hardening.

## Best Possible Solution

Keep this PR open and ask for a current-main adaptation. The best path is to add a shared WhatsApp credential-file reader that rejects symlinked `creds.json` and `creds.json.bak` before all presence/status/identity/age/backup reads, create and best-effort chmod the WhatsApp auth directory to `0o700`, preserve the existing atomic save flow, and add regression tests for symlinked primary credentials, symlinked backups, normal files, and directory mode behavior.
