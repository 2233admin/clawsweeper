## Summary

Keep this PR open. Current main still has not shipped the central WhatsApp credential symlink hardening: the implementation moved from the old `src/web/*` files to `extensions/whatsapp/*`, and the current pluginized credential readers still use `stat`/`readFile` paths that follow symlinked `creds.json` files. The PR is a focused security-hardening candidate, but it needs rebasing/adaptation to the current WhatsApp plugin paths and regression tests.

## Best Possible Solution

Keep this PR open and ask for a current-main adaptation. The best path is to harden the pluginized WhatsApp credential surface by rejecting symlinked `creds.json` and `creds.json.bak` before all status/identity/age/presence reads, create and best-effort chmod the WhatsApp auth directory to `0o700`, preserve the existing atomic save behavior, and add focused regression tests for symlinked primary credentials, backup restore, normal regular files, and directory mode handling.
