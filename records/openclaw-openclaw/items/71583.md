## Best Possible Solution

Keep the repair inside the QQBot plugin: bound the access-token POST with an abortable timeout, surface token-stage failures through runtime status, add a `probeAccount` diagnostic for the token endpoint, and document `bots.qq.com:443` egress checks without adding Tencent-specific behavior to core gateway code.
