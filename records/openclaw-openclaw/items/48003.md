## Summary

Keep open. Current main still gates steer injection on `isStreaming`, so active non-streaming tool execution can still fall through to followup queue behavior. The Discord inbound path also still serializes same-session jobs behind `KeyedAsyncQueue` before they reach the reply pipeline. Open PR #73365 is the right implementation candidate, but it is unmerged, so this issue should stay open.

## Best Possible Solution

Keep this issue open and use #73365, or a similarly narrow replacement, as the canonical fix path. The fix should make steer injection lifecycle-aware for active non-streaming runs, preserve compaction/startup/teardown safeguards, and add focused regression coverage for `queueEmbeddedPiMessage`, `runReplyAgent`, and the Discord same-session inbound path or an equivalent live proof.
