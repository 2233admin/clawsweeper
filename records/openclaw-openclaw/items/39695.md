## Summary

Keep open. Current main improves fresh local onboarding and logs fake text-form tool calls, but it still does not prove the reported Discord-bound path can execute `exec` and persist the expected JSONL session transcript. Explicit `tools.profile: "messaging"` remains chat-only and continues to exclude runtime tools.

## Best Possible Solution

Keep this open until maintainers decide the intended product boundary: either add a narrow doctor/config diagnostic for agents whose messaging profile cannot satisfy SOUL instructions that require `exec`, land a carefully scoped legacy-profile migration, or add a Discord-bound regression proving an inbound Discord turn with `coding`/`alsoAllow` can execute `exec` and persist the JSONL transcript. Avoid silently broadening messaging-profile command execution without an explicit security decision.
