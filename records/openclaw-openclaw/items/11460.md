## Summary

Keep this issue open. Current main still only supports outbound WhatsApp `react`; it does not advertise or handle the generic `message reactions` query action for WhatsApp, does not store/query historical WhatsApp reactions, and does not listen for inbound WhatsApp reaction events. Related PRs #46881 and #48570 are still open/unmerged, and the earlier #13641 attempt was closed unmerged, so they are useful implementation input but not shipped fixes.

## Best Possible Solution

Keep this issue open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action for WhatsApp, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through the normal access-control and routing pipeline, and add focused WhatsApp tests plus docs. Treat #46881 and #48570 as useful implementation candidates, not shipped superseding fixes.
