## Summary

Keep this issue open. Current main still only advertises and handles outbound WhatsApp `react`; it does not support the generic `reactions` query action, does not store/query historical WhatsApp reactions, and does not attach an inbound WhatsApp reaction listener. The related implementation attempts #46881 and #48570 remain useful input but are open/unmerged, while #13641 was closed unmerged.

## Best Possible Solution

Keep this issue open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action for WhatsApp, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through the normal access-control and routing pipeline, and add focused WhatsApp tests plus docs. Treat #46881 and #48570 as implementation candidates to review or consolidate rather than as shipped superseding fixes.
