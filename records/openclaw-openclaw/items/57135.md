## Summary

Keep this PR open. Current main does not clearly implement the PR’s stated user-facing fix for AJV/JSON-Schema `additionalProperties` validation errors: the formatter still passes through `must NOT have additional properties` without adding the offending key name. The PR also bundles unrelated Feishu prompt/error-message changes and config validation behavior, so the best path is a narrower, rebased fix with targeted tests.

## Best Possible Solution

Rework or replace this PR with a narrow change in `src/plugins/schema-validator.ts` that formats AJV `additionalProperties` errors using `error.params.additionalProperty`, then add targeted tests proving channel/plugin config errors show the offending key names. Keep Feishu localization and plugin allowlist policy changes out of this PR unless maintainers explicitly ask for them.
