## Summary

Keep open. Current main has useful Google Meet hardening for setup/status/speech gating and the merged #73256 partial fix, but it still does not provide the central requested product outcome: a supported reliable headless/listen-first Meet participant with documented identity semantics and end-to-end audio or transcription health proof.

## What This Changes

The issue requests reliable Google Meet headless agent joining with documented identity/join modes, listen/transcribe health checks, and eventual bidirectional realtime voice.

## Best Possible Solution

Keep this as Google Meet plugin product work. The desired end state is a documented identity and meeting-entry model, a chosen official Media API versus browser-participant strategy, status that separates browser/lobby/admission/provider/audio/transcript state, preflight checks that fail on missing real audio or transcript movement, and live validation proving actual Meet audio or transcription.

Unclear. ## Solution Assessment

No. Current main is a useful partial hardening path, but it is not the complete or narrowest maintainable solution for the requested reliable headless/listen-first participant because the realtime path still depends on Chrome/profile/virtual-audio behavior and bridge-side byte counters.
