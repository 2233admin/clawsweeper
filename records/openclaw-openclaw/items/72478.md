Keep open. Current main contains useful Google Meet hardening after the merged observe-mode and speech-readiness work, but the central request is still not implemented: a supported reliable headless/listen-first Meet participant with documented identity/join semantics and end-to-end audio or transcription proof remains product work.

## What This Changes

The issue requests a reliable supported Google Meet agent participant path with headless/listen-first join modes, identity and lobby state, and audio/transcription health checks.

## Best Possible Solution

Keep this open as Google Meet plugin product work. The desired end state is a documented identity and meeting-entry model, a chosen official Media API versus hardened Chrome-node participant strategy, status that separates browser/lobby/admission/provider/audio/transcript state, preflight checks that fail on missing real audio or transcript movement, and live validation proving actual Meet audio or transcription.
