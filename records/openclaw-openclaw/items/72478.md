## Summary

Keep open. Current main has meaningful Google Meet plugin support and the related #73256 follow-up hardens observe-only setup and realtime speech-output checks, but the central request remains broader: a supported headless or listen-first participant with live admission, audio, and transcription health proof.

## What This Changes

The issue requests a reliable Google Meet headless or listen-first agent join flow with explicit identity, admission, audio, transcription, and failure-health reporting.

## Best Possible Solution

Keep this as Google Meet plugin product/bug work in `extensions/google-meet`: define the supported identity and admission model, add a real listen-first/live-transcription path or document the API boundary, make status prove live transcript or caption movement when available, and keep improving Chrome-node realtime health with end-to-end admission plus actual audio movement proof.
