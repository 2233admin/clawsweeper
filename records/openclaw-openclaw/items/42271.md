## Summary

Keep this PR open. Current main still resolves Feishu media only from the triggering message, then fetches quoted/replied messages only for text context. The PR remains a focused implementation candidate for the bundled Feishu plugin quoted-media gap, and the related prior reports/attempts support that this is a real user workflow rather than stale drive-by work.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The implementation should preserve raw fetched parent-message content or structured media keys, resolve quoted/replied media after quoted-context visibility checks pass, build the final agent media payload after all media collection, and add quoted-media regression coverage.
