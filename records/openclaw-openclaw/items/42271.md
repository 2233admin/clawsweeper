## Summary

Keep this PR open. Current main still does not resolve media from Feishu quoted/replied messages: it resolves media only from the triggering message, builds the agent media payload before fetching the parent message, and fetched parent messages expose only parsed placeholder content without raw media keys. The PR targets a real bundled Feishu plugin gap with useful live verification and prior related attempts, but it needs rebasing onto the current Feishu file split and focused regression coverage before it is landable.

## Best Possible Solution

Keep this PR open or replace it with an equivalent rebased Feishu plugin patch. The fix should preserve raw fetched parent-message content or structured media keys, resolve quoted media only after group-context visibility checks pass, build the final media payload after all current and quoted media collection, account for current audio-preflight ordering, and add regression coverage for quoted image/file media plus blocked quoted-context senders.
