## Summary

Keep this PR open. Current main still resolves media only from the triggering Feishu message, builds the agent media payload before the quoted/replied parent is fetched, and fetched parent messages still expose only parsed placeholder text such as `[file message]` without the raw `image_key`/`file_key` JSON needed to download the quoted resource. The PR discussion and related reports describe a real Feishu group-chat workflow, and the provided patch is a focused bundled Feishu plugin fix rather than obsolete work.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin patch. The fix should preserve raw parent-message media content or structured media keys, resolve quoted media after `shouldIncludeFetchedGroupContextMessage` allows the fetched parent, build the final media payload after all media collection, account for current audio-preflight ordering, and add quoted-media regression coverage.
