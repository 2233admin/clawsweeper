## Summary

Keep this PR open. Current main still resolves media only from the inbound Feishu message itself, not from fetched quoted/replied parent messages, so the PR’s intended Feishu group-chat media behavior is not implemented. The PR remains a focused implementation candidate for https://github.com/openclaw/openclaw/pull/42271 on the bundled Feishu plugin inbound path.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The best maintainer path is to adapt it onto current main by preserving raw fetched parent-message content or structured media keys, resolving quoted/replied media only after quote visibility checks pass, building the final agent media payload after all media collection, preserving existing Feishu routing/thread behavior, and adding quoted-media regression coverage.
