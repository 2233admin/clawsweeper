## Summary

Keep this PR open. Current main still downloads media only from the inbound Feishu message, then fetches quoted/replied parent messages only for text context. The PR remains a focused implementation candidate for the bundled Feishu plugin quoted-media gap, with prior reports and closed attempts confirming the same user workflow is real and not shipped yet.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The maintainer path is to rebase the idea onto current main by preserving raw fetched parent-message content or structured media keys, resolving quoted/replied media after quoted-context visibility checks pass, building the final agent media payload after all media collection, and adding quoted-media regression coverage.
