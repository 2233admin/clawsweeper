## Summary

Keep open. Current main and the latest release already address the immediate Talk mode microphone regression, but they still do not implement this PR's main configurable Permissions-Policy capability. The PR remains useful, with one non-blocking docs gap to address before merge.

## What This Changes

The PR adds a configurable `gateway.http.securityHeaders.permissionsPolicy` option, threads it through config schema/runtime/gateway HTTP handling, adjusts the default Permissions-Policy behavior, and adds focused gateway tests.

## Best Possible Solution

Land a rebased, tightly scoped version that exposes `permissionsPolicy?: string | false` through types, zod/generated metadata, runtime resolution, HTTP header handling, public gateway docs, and focused tests, while preserving or clearly documenting the final default header semantics.
