## Summary

Keep this issue open. Current main still sends last-known-good config recovery notices into prompt-visible system events and, on the restart-sentinel auto-recovery path, also sends the same text out-of-band to the originating channel. The requested `gateway.systemEvents.recoveryNotice` policy knob is not present in the strict config schema, exported Gateway config types, or Gateway docs, and the related items are adjacent rather than superseding this request.

## Best Possible Solution

Implement a focused Gateway config policy on current main: add `gateway.systemEvents.recoveryNotice` with modes like `user` (current default), `log`, and `off`; gate both the prompt-visible system-event surface and the out-of-band restart-sentinel recovery notice; cover the existing startup/reload `Config recovery warning` enqueue path; preserve restart continuations and recovery behavior; document the operator tradeoff; and add targeted tests for all modes.

## Clownfish Work Prompt

Implement the recovery-notice policy requested by this issue. Observable problem: after last-known-good config recovery, OpenClaw can inject technical recovery text into the next agent prompt and, on the restart-sentinel auto-recovery path, also deliver it out-of-band to the originating channel, which breaks customer-facing personas. Add a focused `gateway.systemEvents.recoveryNotice` config knob with modes `user` (default/current behavior), `log`, and `off` unless the existing config naming conventions strongly suggest a better local name. Scope: update the strict Zod schema, exported Gateway config types, docs/help/reference surfaces as appropriate, and the recovery notice runtime. Gate both surfaces in `src/gateway/server-restart-sentinel.ts`: the `enqueueRestartSentinelWake` system-event call and `deliverRestartSentinelNotice` outbound call. Also gate the existing startup/reload `Config recovery warning` path in `src/gateway/config-recovery-notice.ts`, `src/gateway/server-startup-config.ts`, and `src/gateway/server-reload-handlers.ts` so the policy is not partial. Preserve restart continuations, last-known-good recovery, sentinel cleanup, and current behavior by default. For `log`, suppress prompt/outbound notices and emit a single useful Gateway log line; for `off`, suppress prompt/outbound notices without extra notice logging beyond existing recovery logs. Add focused tests for all three modes, including restart-sentinel prompt/outbound behavior and startup/reload config recovery warning behavior. Update docs under `docs/gateway/configuration.md` or the generated reference source used by this repo. Changelog: add a user-facing `### Changes` bullet with proper reporter attribution if project policy requires it for this feature. Do not change unrelated recovery policy, plugin-local rollback behavior, channel delivery routing, or restart continuation behavior.
