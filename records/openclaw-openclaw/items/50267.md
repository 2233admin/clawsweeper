## Summary

Keep this PR open. Current main still does not pass endpoint-aware attribution headers into the image-understanding direct `complete()` call, so the OpenRouter image-tool problem remains. The PR is still pointed at a real bug, but its patch is stale and should be reworked against the current request-policy helper rather than closed as implemented.

## Best Possible Solution

Rework this PR against `src/media-understanding/image.ts`: resolve headers with `resolveProviderRequestHeaders` or `resolveProviderRequestPolicyConfig` using `provider`, `api`, `baseUrl`, `capability: "image"`, and `transport: "media-understanding"`; pass the resulting headers into both direct `complete()` attempts; add regression tests for OpenRouter default/native-route attribution and custom-base-url non-injection.
