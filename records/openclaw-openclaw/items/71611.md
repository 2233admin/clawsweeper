## Summary

This PR should stay open because the underlying Windows memory-core EBUSY bug is still valid on current main, but the proposed patch is not mergeable: it introduces invalid TypeScript in three changed spots and would drop WAL/SHM sidecar handling plus backup-path uniqueness if repaired incorrectly.

## What This Changes

The PR adds a memory-core rename retry/copy fallback helper and routes atomic-reindex file moves through it for Windows EBUSY/EPERM/EEXIST handling.

## Best Possible Solution

Repair this PR or open a narrow replacement that ports the cron rename retry/fallback behavior to memory-core while preserving base/WAL/SHM sidecar handling and random UUID backup paths, then add focused regression tests for retry, copy fallback, sidecar moves/removal, and rollback safety. Keep the unsafe-reindex/vector-table issue in #70559 separate.

## Reproduction Assessment

Yes. The PR defects reproduce by static inspection of the patch as invalid TypeScript, and the underlying product bug has concrete Windows reproduction steps and EBUSY logs in #64187 and related #74559.

## Solution Assessment

No as submitted. Retrying EBUSY and falling back on Windows rename failures is a reasonable narrow direction, but this patch must restore the original path construction, sidecar behavior, UUID backup uniqueness, and regression coverage before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Restore sidecar path template literals:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:31-32`
  - body: The new `const source = ${sourceBase};` and `const target = ${targetBase};` lines are invalid TypeScript. They also drop the `suffix` value, so WAL and SHM sidecars would no longer move with the base SQLite DB if this were repaired incorrectly.
  - confidence: 0.99
- **[P1] Keep removing WAL and SHM sidecars:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:45`
  - body: `fs.rm(${basePath}, ...)` is invalid TypeScript and no longer includes the per-file suffix. Restore the template literal so cleanup removes the base DB, WAL, and SHM files together.
  - confidence: 0.99
- **[P1] Preserve unique backup paths:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:49`
  - body: The new backup path expression is invalid TypeScript and removes `randomUUID()`. Without a unique backup filename, overlapping swaps can collide on the same backup path and break rollback safety.
  - confidence: 0.98
