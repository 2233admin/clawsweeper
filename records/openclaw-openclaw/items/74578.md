## Summary

Keep open. Current main still routes user-facing replies through a sanitizer that does not suppress the internal replay placeholder, and this PR targets the right shared boundary, but the added regex can leave one placeholder behind when standalone placeholder lines are adjacent.

## What This Changes

The branch adds a shared sanitizer filter for standalone `[tool calls omitted]` lines, regression tests for placeholder stripping and inline mentions, and a changelog fix entry for #74573.

## Best Possible Solution

Update the sanitizer so every standalone placeholder line is removed, including consecutive lines, while keeping ordinary inline mentions unchanged and leaving the `turns.ts` replay placeholder semantics internal-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Strip consecutive placeholder lines:** `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:43-44`
  - body: The new regex consumes the newline after each matched placeholder line. With adjacent standalone placeholders, the next global match starts after that newline, so the following placeholder no longer has a leading line boundary and remains visible. For example, `A\n[tool calls omitted]\n[tool calls omitted]\nB` becomes `A\n[tool calls omitted]\nB`. Use a non-consuming end-of-line boundary/lookahead or a line-wise filter, and add a consecutive-lines test.
  - confidence: 0.88
