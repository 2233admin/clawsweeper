## Summary

Keep open. The PR targets a valid user-facing leak and puts the fix in the shared sanitizer, but the patch derives the returned reply text from `stripped.trim()`, which changes existing whitespace-preservation behavior and conflicts with current sanitizer tests.

## What This Changes

The PR changes `sanitizeUserFacingText` to remove the internal `[tool calls omitted]` placeholder before user-facing reply and error formatting paths.

## Best Possible Solution

Strip the exact internal placeholder from the display copy in `sanitizeUserFacingText` while keeping the existing final-output whitespace contract and leaving replay transcript repair in `turns.ts` unchanged. Add focused sanitizer tests for placeholder-only output, placeholder removal around visible text, and the existing whitespace-preservation case.
