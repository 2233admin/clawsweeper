## Summary

Keep this issue open. Current main has adjacent Plugin SDK and context-engine lifecycle hooks, but it still does not expose the requested workspace/internal `HOOK.md` events for `subagent:ended`, `agent:turnComplete`, or tool-call thresholds. The discussion is recent and active, including a memory-layer use case, a linked external `knowledge-graph` implementation, and the reporter offering a workaround, so this needs maintainer API/product judgment rather than cleanup.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely path is a narrow, documented internal-hook bridge for `subagent:ended`, plus explicit `agent:turnComplete` and tool-count/threshold semantics if maintainers want workspace `HOOK.md` automation. If maintainers prefer plugin-only lifecycle automation, close later with an explicit product decision and docs pointing users to Plugin SDK hooks.
