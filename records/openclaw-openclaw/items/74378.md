## Summary

Keep open. The report is current, has a concrete Windows/npm reproduction for supported short-lived CLI commands, and an open unmerged PR with closing syntax already targets the same behavior; current main still lacks a merged process-lifetime fix.

## What This Changes

The issue requests that short-lived Windows npm CLI invocations such as `node .../openclaw.mjs status` and `node .../openclaw.mjs --version` terminate after completion instead of leaving `node.exe` processes alive.

## Best Possible Solution

A focused lifecycle fix should make supported short-lived CLI commands release or avoid any process-wide resources that keep Node alive on Windows npm installs, with Windows smoke proof for `status` and `--version`, while preserving long-running gateway/provider behavior.

## Reproduction Assessment

Yes. The reporter provides a concrete Windows npm reproduction using `status` and `--version`, with process observations showing the gateway is separate; I did not live-run Windows in this read-only Linux review.

## Solution Assessment
