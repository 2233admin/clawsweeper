## Summary

Keep open. The Windows/npm process-lifetime report is current and actionable, current `main` still has the routed `status` command returning through the shared CLI lifecycle without a merged completion/teardown fix, and PR #74425 is still open and unmerged with closing syntax for the same bug.

## What This Changes

The issue reports that short-lived OpenClaw CLI commands on Windows npm installs remain alive as `node.exe ... openclaw.mjs <command>` processes after completing.

## Best Possible Solution

Review PR #74425 against the current CLI lifecycle and either land a focused fix or replace it with narrower teardown for resources that keep short-lived Windows npm CLI commands alive, with Windows smoke proof for `status` and `--version` while preserving long-running gateway behavior.

## Reproduction Assessment

Yes. The reporter gives a concrete Windows npm reproduction for `node <npm-global-openclaw>/openclaw.mjs status` and `--version`, including process observations after 3 and 13 seconds with the gateway healthy and separate; I did not live-run Windows in this read-only Linux review.
