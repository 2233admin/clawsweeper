## What This Changes

The issue asks that short-lived Windows npm CLI commands such as `status` and `--version` terminate cleanly instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Land or replace the linked PR with a focused lifecycle fix that lets Windows npm `status` and `--version` complete without lingering `node.exe` processes, preserves intentional long-running commands, and includes Windows smoke coverage plus output-completeness checks for piped or redirected output.
