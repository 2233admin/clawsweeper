## Summary

Keep open. Current main still shows the same short-lived CLI lifecycle gap: `status` is routed as a fast local command, but the shared startup path can bootstrap process-wide Undici/proxy state before routing and the finalizer only stops the managed proxy and memory managers. PR #74425 is an open, unmerged implementation candidate that explicitly links this issue, so the issue should remain open until that PR is reviewed and either landed or rejected.

## What This Changes

Windows npm-installed short-lived CLI commands such as `status` and `--version` reportedly leave `node.exe ... openclaw.mjs` processes alive after command completion.

## Best Possible Solution

Review PR #74425, or replace it with a targeted lifecycle fix that lets short-lived commands exit naturally after cleaning up any process-wide CLI proxy/Undici/debug resources, while preserving long-running gateway and agent behavior. Add regression coverage for routed `status`/version paths plus a Windows npm smoke check.
