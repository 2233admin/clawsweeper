Keep open. The report is current, has a concrete Windows npm reproduction, current main still returns naturally after the CLI entrypoint without a merged lifecycle fix, and open PR #74425 remains the linked implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests that short-lived Windows npm CLI commands such as `status` and `--version` terminate cleanly instead of leaving `node.exe ... openclaw.mjs` processes alive.

## Best Possible Solution

Review PR #74425 against the current CLI lifecycle and either land a focused fix or replace it with targeted teardown for the resource that keeps short-lived Windows npm CLI commands alive, while preserving long-running gateway/provider behavior and adding Windows smoke coverage for `status` and `--version`.

## Solution Assessment
