## Summary

Keep open. Current main still has the reported headless fallback gap, and two open PRs (#72802 and #73227) explicitly target it with closing syntax, so this issue should remain open until one fix lands or maintainers choose between them.

## What This Changes

The issue reports that `openclaw dashboard` on WSL2/headless output shows only the clean Control UI URL, leaving users without the gateway token when clipboard/browser token delivery fails.

## Best Possible Solution

Land a CVE-safe dashboard fallback that provides manual auth guidance when token handoff via clipboard/browser fails, without logging the token or `#token=`, then close this issue through the merged PR and supersede the duplicate open PR.
