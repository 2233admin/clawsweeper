## Summary

Keep open. Current main still prints only the bare dashboard URL to logs and sends the tokenized URL only through clipboard/browser delivery; in the reported WSL/headless failure path, those delivery channels can fail and current output provides no manual token recovery. Two open PRs already use closing syntax for this bug, so the issue should remain open until a maintainer lands or rejects one of those implementations.

## What This Changes

The issue asks for `openclaw dashboard` to provide a token-authenticated dashboard URL or recovery guidance when WSL/headless execution prints only a bare Gateway URL.

## Best Possible Solution

Review and dedupe the two open dashboard-token fallback PRs, then land one narrow fix that adds CVE-safe manual auth guidance only when tokenized delivery fails while preserving the existing contract that runtime logs and SSH hints never include the token value or `#token=` fragment.
