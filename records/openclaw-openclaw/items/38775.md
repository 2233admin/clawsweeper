## Summary

Current main still does not implement usage-aware multi-account auth-profile load balancing. Anthropic usage snapshots are fetched and shown on status/reporting surfaces, but request-time Gateway auth selection remains order/type/lastUsed plus reactive cooldown/failover. The related #41135 is useful broader pool-routing context, but it is still open and not a merged implementation.

## Best Possible Solution

Keep this open and consolidate design with #41135 rather than auto-closing. The best path is an additive, generic auth-profile routing contract that can consume provider-owned usage snapshots as scoring inputs, preserve current defaults/session stickiness unless opted in, support proactive thresholds and fallback for profiles without usage data, expose route decisions/status, and update config/schema/docs before enabling Anthropic OAuth usage-aware balancing.
