## Summary

Keep open. Current main still lacks Chat Completions client-tool passthrough, tool_choice parsing, OpenAI-compatible tool_calls output, and the role:"tool"/tool_call_id follow-up loop. The provided GitHub context also shows PR #66278 is open, unmerged, and uses closing syntax for this issue, so this should remain open until that PR or an equivalent gateway change lands.

## Best Possible Solution

Keep this issue open and treat PR #66278 as the active implementation candidate. The maintainer path is to review, revise, and land that PR or an equivalent change in src/gateway/openai-http.ts, with focused tests in src/gateway/openai-http.test.ts for tools/tool_choice parsing, non-stream message.tool_calls, streaming delta.tool_calls, and role:"tool" follow-up messages with tool_call_id.
