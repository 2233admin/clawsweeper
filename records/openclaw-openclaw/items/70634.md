## Summary

Keep this issue open. Current main still uses a FIFO per-lane command queue, and `sessions_send`/A2A turns are distinguishable as `inputProvenance.kind === "inter_session"` but are not assigned a lower queue priority. The KinthAI comment is useful supporting evidence that the same starvation pattern appears in production and that a two-tier queue is a practical fix.

## Best Possible Solution

Implement a conservative two-tier command-lane priority in current main: keep existing FIFO behavior for equal-priority work, place human/channel/control turns ahead of `sessions_send`/`inter_session` agent-originated turns for the same target session, and add regression tests that prove a later foreground human turn runs before already-queued background A2A work while background FIFO order remains stable.

## Clownfish Work Prompt

Fix the human-message starvation bug in agent session lanes. Current main serializes per-session agent turns through `src/process/command-queue.ts` with FIFO `push`/`shift`; `sessions_send` and A2A follow-ups carry `inputProvenance.kind === "inter_session"` and `sourceTool: "sessions_send"` but are not queued behind human/channel/control turns. Implement a conservative queued-work priority tier: preserve FIFO within equal priority, do not preempt an already-active run, and demote known agent-originated inter-session work behind normal foreground turns for the same lane. Prefer a generic command-queue option such as `priority`/`tier` that defaults to existing foreground behavior, then have `runEmbeddedPiAgent` pass a background tier when `params.inputProvenance?.kind === "inter_session"`. Keep the change generic; do not hard-code channel/plugin ids. Add focused tests proving: a foreground enqueue behind queued background work runs first after the active task completes; background work remains FIFO; existing clear/reset/drain semantics still reject or preserve the right entries; and `sessions_send`/A2A calls still pass inter-session provenance and now receive the lower queue tier. Add a user-facing changelog fix entry if the repo's active changelog is being maintained for this release, crediting @ewrurwteurwU and @kinthaiofficial, not Codex. Do not bundle budget throttling/back-pressure into this PR.
