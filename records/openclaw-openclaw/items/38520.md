## Summary

Keep open. Current main still has adjacent compaction safeguards, but not the requested configurable pre-compaction warning thresholds, visible structured handoff turn, or bounded/audited compaction deferral mechanism.

## What This Changes

The issue requests configurable pre-compaction warning thresholds, a structured pre-compaction handoff turn, and a bounded deferral mechanism for imminent compaction.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete solution should define validated warning/critical threshold config, channel/UI delivery semantics for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should deliberately reuse or replace the existing memoryFlush, notifyUser, session:compact hooks, compaction provider, and context-engine maintenance surfaces, then update config schema/types/help/docs and focused runtime tests.
