## Summary

Keep this PR open. Current main still lacks both intended changes: restart-sentinel continuation dispatch does not force automatic source delivery for group/topic continuations, and explicit restart callers cannot pass a per-request deferral timeout to prevent an indefinitely pending pre-restart counter. The PR is a focused draft bug-fix candidate with regression coverage, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely maintainer path is to land a narrow fix that forces restart-sentinel continuation turns to use automatic source delivery, adds group/topic regression coverage, and bounds explicit restart deferral with a reviewed per-request timeout for `gateway.restart` and `/restart`.
