## Summary

Keep open. Current main still lacks the restart-continuation reply-delivery override and per-request explicit restart deferral bound, so the PR is not obsolete; however, the reviewed PR head still has concrete merge blockers around coalesced timeout state and duplicated explicit-timeout constants.

The PR adds automatic source reply delivery for restart-sentinel continuation turns, passes a 30s per-request deferral timeout from explicit restart callers, and adds regression tests for Telegram group topics and restart deferral.

## Best Possible Solution

Revise this PR so restart-sentinel continuations explicitly request automatic source delivery, explicit restart deferral timeout state is centralized and mutable across coalesced scheduled restarts, and the tests cover direct and coalesced timeout ordering. Maintainers should also decide whether the already-in-flight continuation loss in #74424 belongs in this PR or a narrower follow-up.
