## Summary

Keep this PR open. Current main still lacks both intended behaviors: restart-sentinel continuation dispatch does not force automatic source delivery for group/topic conversations, and explicit restart callers still cannot pass a per-request deferral timeout. The PR is still a focused implementation candidate, but the latest review context flags a coalesced-restart gap that should be resolved before merge.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to amend it so coalesced explicit restarts also carry the per-request deferral timeout, share the explicit timeout constant from one scheduler-owned location, keep the automatic source delivery override for restart-sentinel continuations, and land with the focused regression tests described in the PR.
