## Summary

Keep open. Current main still does not force automatic source delivery for restart-sentinel continuation replies and does not expose a caller-specific explicit restart deferral timeout; the live PR is relevant, but it still needs a small revision because the per-request timeout is not preserved through pending-restart coalescing and the 30s constant is duplicated.

## What This Changes

This PR adds automatic source reply delivery for restart-sentinel continuation turns, passes a 30s deferral timeout from explicit restart callers, and adds regression coverage for Telegram-style group topics plus per-request restart deferral.

## Best Possible Solution

Revise this PR, or a narrow replacement branch, so restart-sentinel continuations keep the automatic source-delivery override while explicit restart deferral timeout state is centralized and survives rescheduling/coalescing; keep targeted regression coverage for group-topic replies, direct per-request timeout, and the coalesced explicit-restart ordering before landing.
