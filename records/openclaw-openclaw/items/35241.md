## Summary

Keep this PR open. Current main still does not expose `openclaw agent --session-key`, while the embedded agent command, session resolver, Gateway protocol, and Gateway handler already understand `sessionKey`. The PR remains relevant, but it should be refreshed with validation, docs/help, and focused CLI wrapper tests before merge.

## Best Possible Solution

Keep this PR open and refresh it as a small CLI surface patch: register `openclaw agent --session-key <key>`, add `sessionKey` to `AgentCliOpts`, treat it as a valid selector, pass it through `resolveSessionKeyForRequest()` and embedded fallback, add shared validation for malformed and mismatched agent session keys, and update CLI docs/help plus focused tests. Handle `--no-fallback` separately.
