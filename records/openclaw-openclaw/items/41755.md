## Summary

Current main still does not restore Control UI abortability after refreshing during an active stream. The prior ClawSweeper analysis remains current: reconnect clears the local run id, neither hello nor chat.history carries an active-run snapshot, and later deltas update stream text without making the run abortable.

## Best Possible Solution

Keep this issue open. Add an authoritative active-run hydration path for Control UI on page load/reconnect, likely through hello/bootstrap or chat.history, returning the active run id plus buffered stream state for the visible session. The UI should restore chatRunId, chatStream, and chatStreamStartedAt before rendering controls, with tests for refresh during active streaming and terminal-event cleanup.
