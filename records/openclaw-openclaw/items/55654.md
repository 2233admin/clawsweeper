## Summary

Keep open. Current main still does not expose an agent-facing `feishu_bitable_delete_record` tool. The Feishu Bitable plugin registers get/list/create/update plus app/field creation, while record deletion is only used internally during new-Bitable cleanup. Related PRs #40697 and #41368 remain implementation candidates rather than shipped main behavior.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `extensions/feishu/src/bitable.ts`: add a typed helper around `client.bitable.appTableRecord.delete`, register `feishu_bitable_delete_record` through the existing Bitable tool helper with `app_token`, `table_id`, `record_id`, and optional `accountId`, return a structured success result, and add focused Feishu plugin tests for registration, routing, SDK call parameters, and error handling. Review or consolidate #40697 and #41368 before landing.
