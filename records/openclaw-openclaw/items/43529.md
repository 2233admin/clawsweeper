## Summary

Keep open. The timeout death-spiral problem still exists on current main, and this PR is a useful but not merge-ready fix: it can reset sessions for mixed-cause fallback failures, misses timeout `isError` payloads without `meta.error`, and needs rebase/type-call-site repair against current main.

## What This Changes

The PR adds a timeout reset callback through `runReplyAgent` and `runAgentTurnWithFallback`, returns timeout-specific retry text, and adds focused run-reply tests for thrown timeout, `meta.error` timeout, reset-unavailable fallback text, and heartbeat behavior.

## Best Possible Solution

Implement timeout session rotation on current main using structured all-model timeout evidence, handle both `meta.error` and timeout `isError` payload surfaces before the run loop treats the turn as successful, and preserve existing heartbeat, transient HTTP retry, billing, rate-limit, auth, compaction, role-ordering, and external-chat failure behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require all attempts to time out before resetting:** `src/auto-reply/reply/agent-runner-execution.ts:589-595`
  - body: `isTimeoutError(err)` runs on the aggregate fallback error text. A fallback summary can contain one timed-out attempt plus auth, billing, rate-limit, or other failures, so this can rotate and discard the active transcript for a non-timeout exhaustion. Use `FallbackSummaryError.attempts` and require every relevant attempt to have timeout evidence before resetting.
  - confidence: 0.91
- **[P2] Handle timeout payloads before treating the run as successful:** `src/auto-reply/reply/agent-runner-execution.ts:475-480`
  - body: The embedded runner can return a timeout as an `isError` payload with no `meta.error`. This new branch only checks `runResult.meta?.error`, then the loop breaks as a successful run, so that timeout surface keeps the same bloated session and leaves the reported retry loop unresolved.
  - confidence: 0.88
- **[P2] Update direct callers for the required reset callback:** `src/auto-reply/reply/agent-runner-execution.ts:91`
  - body: Adding `resetSessionAfterTimeout` as a required parameter means existing direct `runAgentTurnWithFallback` test callers no longer satisfy the parameter object. The PR needs to update the shared test helper/call sites with a stub callback or the test typecheck lane will fail after rebase.
  - confidence: 0.84
