## Summary

Keep this PR open. Current main still lacks the proposed external plugin seam for runtime provider auth overrides, and the existing runtime-auth hook is owned by the selected provider plugin rather than by a separate trusted plugin that wants to supply per-call credentials, base URL, or request headers for built-in providers.

## What This Changes

The PR adds a plugin SDK registration API for provider runtime auth overrides, wires override lookup into model auth resolution, propagates optional base URL and request headers, and adds focused auth/runtime/media-understanding tests.

## Best Possible Solution

Land a maintainer-approved additive plugin SDK/core seam for trusted plugins to provide per-call provider credentials, base URL, and request headers for existing providers, with documented semantics and focused tests for ordering, throw logging, mode validation, reentrancy, alias handling, model context, availability, and side-path fallback behavior.
