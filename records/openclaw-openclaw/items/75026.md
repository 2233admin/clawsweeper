## Summary

Keep open. Current main still has the MiniMax Anthropic-compatible usage propagation gap, and #75136 is an open, unmerged PR that explicitly targets this issue, so the issue should remain open until that fix or an equivalent transport-boundary fix lands.

## What This Changes

MiniMax Anthropic-compatible provider token usage should be persisted and shown in the Control UI Usage page instead of staying at zero while message counts update.

## Best Possible Solution

Land #75136 after normal review, or land an equivalent narrow fix in `src/agents/anthropic-transport-stream.ts` that normalizes compatible-provider usage at the Anthropic stream boundary, including alias-shaped and `message_stop` usage, before assistant snapshots and session persistence. The Usage page should continue to read transcript-backed session usage rather than infer tokens in the UI.

## Reproduction Assessment

Yes. The issue provides exact MiniMax CN Anthropic-compatible setup details and nonzero raw provider usage, and current-main source tracing shows the transport can drop alias-shaped or `message_stop` usage before session aggregation.

## Solution Assessment

Yes. Fixing usage at the Anthropic stream transport boundary is the narrow maintainable direction; a Control UI-only workaround would leave session JSONL, status summaries, and cost totals incorrect.
