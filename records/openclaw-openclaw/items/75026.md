## Summary

Keep open. Current main still leaves a credible MiniMax Anthropic-compatible usage propagation gap: MiniMax API-key configs use the Anthropic messages path, but that transport records only native Anthropic usage keys before the session accumulator and Control UI usage summary consume transcript usage.

## What This Changes

Request that MiniMax Anthropic-compatible token usage be persisted and shown in the Control UI Usage page instead of showing zero tokens while message counts still update.

## Best Possible Solution

Fix the provider/transport-side usage capture so MiniMax Anthropic-compatible streaming responses produce a nonzero assistant usage snapshot before session persistence, with regression coverage for the MiniMax event shape. Keep the Control UI transcript-backed rather than inferring token totals from message counts or provider identity.

## Solution Assessment
