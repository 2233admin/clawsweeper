## Summary

Keep open. Current main still has the MiniMax Anthropic-compatible usage propagation gap, and the open unmerged PR that targets this report should be reviewed or replaced rather than closing the issue early.

## What This Changes

The issue reports that MiniMax’s Anthropic-compatible endpoint returns token usage but OpenClaw persists zero token totals, so the Control UI Usage page shows only message counts.

## Best Possible Solution

Review, rebase if needed, and land PR #75136, or an equivalent narrow fix in `src/agents/anthropic-transport-stream.ts` that normalizes compatible-provider usage before assistant snapshots and session persistence. Add deterministic SSE coverage for MiniMax-shaped alias usage and `message_stop` usage, with the Control UI continuing to consume transcript-backed totals.

## Reproduction Assessment

Yes. The issue gives exact setup details and raw nonzero provider usage, and current-main source tracing shows those aliases are still not propagated on the Anthropic stream path before session aggregation.

## Solution Assessment

Yes. Fixing usage at the Anthropic stream transport boundary is the narrow maintainable solution; a Control UI-only workaround would leave session logs, status summaries, and cost totals wrong.
