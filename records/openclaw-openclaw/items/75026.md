## Summary

Keep open. Current main still appears to lose MiniMax Anthropic-compatible usage before session aggregation, and open PR #75136 is already the implementation candidate but has not merged.

## What This Changes

MiniMax Anthropic-compatible provider sessions should persist nonzero token usage so the Control UI Usage page shows token totals instead of 0 while message counts continue to work.

## Best Possible Solution

Review and land PR #75136, or an equivalent narrow fix, so the Anthropic stream transport normalizes compatible-provider usage aliases before assistant snapshots and session persistence. The Usage page should continue reading transcript-backed usage rather than inferring tokens from provider identity or message counts.

## Reproduction Assessment

Yes. The reporter supplied nonzero raw provider usage and zero persisted JSONL usage, and current-main source can reproduce the loss with an Anthropic-compatible SSE stream that reports prompt_tokens/completion_tokens or final usage outside the native message_delta fields.

## Solution Assessment

Yes. Normalizing usage at the Anthropic stream transport boundary is the narrow maintainable fix; changing only the Control UI would leave session logs, status, cost summaries, and downstream aggregators wrong.
