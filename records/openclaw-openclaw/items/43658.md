## Summary

Keep open. Current main still uses a single unbound GitHub Copilot API-token cache; a valid cache hit returns before considering the incoming GitHub token/profile, so profile rotation can still reuse another account's Copilot token. This issue also has same-author open PR #43659 with closing syntax, so the best maintainer path is to review, land, or close that implementation candidate rather than close this issue early.

## Best Possible Solution

Keep this issue open and resolve it through #43659 or an equivalent maintainer-reviewed patch. The fix should bind cached Copilot API tokens to the selected GitHub token/profile, preferably with a non-secret fingerprint or per-profile cache partition, treat legacy unbound cache entries as misses, and add focused regression tests proving a rotated profile triggers a fresh token exchange instead of reusing another profile's cached token.
