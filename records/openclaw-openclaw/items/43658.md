## Summary

Keep this issue open. Current main still returns any usable shared GitHub Copilot API-token cache entry without binding it to the selected GitHub token or auth profile, so the reported profile-rotation failure remains present. The issue is also actively paired with open same-author PR #43659 using closing syntax, so the maintainer path is to review, land, or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and resolve it through #43659 or an equivalent maintainer-reviewed patch. The fix should bind cached Copilot API tokens to the selected GitHub token/profile, treat legacy unbound cache entries as misses or otherwise safe, update both active helper paths if both remain used, and add focused regression tests for cross-profile cache bypass.
