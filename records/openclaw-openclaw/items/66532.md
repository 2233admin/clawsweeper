## Summary

Keep open. Current main still has the requested embedded-run coordination helpers internally, and even has adjacent harness/runtime and Gateway abort surfaces, but the injected `api.runtime.agent` contract still exposes only the embedded run starters and session store helpers. The CoClaw plugin context remains a valid example of a trusted plugin needing a stable in-process runtime API instead of reading the `openclaw.embeddedRunState` global.

## What This Changes

Expose the existing embedded-agent abort, wait, active-state, and queue helpers through `api.runtime.agent` so trusted plugins can coordinate in-process embedded runs without reading global state.

## Best Possible Solution

Add a documented, additive, session-scoped plugin runtime contract for embedded-run coordination: lazy-bind the existing helpers on `api.runtime.agent`, declare their types without exposing bulk abort overloads, preserve the current queue success/failure semantics, and update runtime shape tests, plugin runtime mocks, and public plugin runtime docs. Keep the broader runId-based `agent.abort` API tracked separately in #52412.
