## Summary

Keep open. Current main fixes one reported sub-problem, the stale runtime auth snapshot after saving rotated OAuth credentials, but the central Gemini OAuth refresh complaint is not fully implemented: automatic profile resolution still downgrades refresh failures to debug-only logging, the google-gemini-cli plugin still has no provider-owned refreshOAuth hook, and Gemini is still not included in the auth cooldown bypass policy. Recent user comments and the closed-unmerged PR #42610 support treating this as still actionable rather than implemented.

## Best Possible Solution

Keep this issue open for a focused Gemini OAuth refresh fix. The best path is to add a regression test for an expired google-gemini-cli auth profile through the embedded runtime/provider resolution path, surface refresh failures at warn/error or user-visible status level instead of debug-only, decide whether google-gemini-cli needs a provider-owned refreshOAuth implementation, and make an explicit product decision on Gemini cooldown behavior.
