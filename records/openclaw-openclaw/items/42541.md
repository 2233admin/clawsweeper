## Summary

Keep open. Current main has fixed part of the shared OAuth plumbing, especially runtime snapshot refresh after saved token rotation, but the Gemini OAuth refresh report is not fully resolved. The automatic profile-resolution path still downgrades refresh failures to debug-only logging, the google-gemini-cli provider still does not own a refreshOAuth hook despite the SDK seam existing, and Gemini is still not part of the provider cooldown bypass policy.

## Best Possible Solution

Keep this issue open. The best maintainer path is a focused google-gemini-cli OAuth refresh regression that drives an expired auth profile through the embedded runtime/profile-resolution path, preserves and surfaces the underlying refresh error at an operator-visible level, verifies successful refresh updates disk plus runtime snapshots, and makes an explicit provider/product decision on Gemini cooldown handling. If pi-ai remains the refresh backend, pin the upstream error behavior; otherwise add a google-gemini-cli refreshOAuth hook in the Google plugin.
