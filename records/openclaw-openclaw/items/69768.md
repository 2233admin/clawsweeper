## Summary

Current main still does not strip JSON Schema conditional keywords from Gemini tool schemas, so the reported Gemini cron/tool 400 path remains a real code issue. The failure-notification docs are present, and the current fallback code has moved since the original report, but #70497 is still open and unmerged with the focused cleaner/test fix.

## What This Changes

The issue asks OpenClaw to address Gemini Flash cron tool-payload HTTP 400 failures from conditional tool schemas and to improve guidance around model choice and cron failure notification visibility.

## Best Possible Solution

Keep this issue open and review/land #70497 after ensuring the final change is focused, or merge an equivalent narrow patch that adds `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` with regression coverage in `src/agents/schema/clean-for-gemini.test.ts`. The failure-destination docs are already present, so a docs-only close would leave the schema rejection path unresolved.
