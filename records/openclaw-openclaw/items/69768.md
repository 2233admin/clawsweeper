## Summary

Keep #69768 open. Current main still does not strip the `if`/`then`/`else` conditional JSON Schema keywords identified in the issue discussion and proposed by closed-unmerged PR #69964, so the core Gemini tool-schema rejection path is not fully fixed. Some adjacent behavior has moved forward: format errors are now failover-worthy, and cron failure destinations are documented, but the reported Gemini payload serialization gap remains actionable.

## Best Possible Solution

Keep the issue open and land a narrow agents fix: add `if`, `then`, and `else` to Gemini schema cleaning, add a regression test proving nested conditionals are removed while valid properties are preserved, and run the targeted schema/tool tests plus the changed gate. The existing model-failover and cron failure-destination docs can stay as supporting mitigation, not as the closure reason.
