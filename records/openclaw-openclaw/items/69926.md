## Summary

Keep open. Current main still does not implement WhatsApp per-group `allowFrom`: the public WhatsApp group config type/schema and generated metadata only accept `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`, while inbound authorization still resolves one account/channel-level `groupAllowFrom` with fallback to `allowFrom`. The related deny/blocklist and multi-account inheritance work do not supersede this per-group sender-authorization request.

## Best Possible Solution

Keep this issue open. The best implementation is an additive WhatsApp plugin change: add optional `allowFrom` to `WhatsAppGroupConfig`, the Zod schema, generated config metadata, and docs; resolve the effective sender allowlist from the matched WhatsApp group entry before falling back to `groupAllowFrom`/`allowFrom`; cover both root `channels.whatsapp.groups` and `channels.whatsapp.accounts.<id>.groups` with regression tests. Coordinate the multi-account inheritance edge with #69874, but do not close this as a duplicate because per-group sender authorization remains separate work.
