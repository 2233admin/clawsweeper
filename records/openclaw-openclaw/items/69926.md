## Summary

Keep open. Current main does not implement WhatsApp per-group `allowFrom`: the WhatsApp group config type/schema only accepts `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`, and inbound group authorization still resolves one account/channel-level `groupAllowFrom` with fallback to `allowFrom`. The request is recent, concrete, and not superseded by the related global deny/blocklist or multi-account inheritance items.

## Best Possible Solution

Keep this issue open. The best implementation is a WhatsApp-plugin change: add optional `allowFrom` to `WhatsAppGroupConfig`, the Zod schema, generated config metadata, and docs; resolve the effective group sender allowlist from the matched WhatsApp group entry before falling back to `groupAllowFrom`/`allowFrom`; cover both root and `accounts.<id>.groups` with regression tests. Coordinate the multi-account inheritance edge with #69874, but do not close this as a duplicate because the per-group sender authorization feature remains separate.
