## Summary

Keep this PR open. Current main still implements Signal through native signal-cli JSON-RPC plus SSE only, while this PR remains the canonical implementation candidate for bbernhard/signal-cli-rest-api REST/WebSocket container support. This PR adds a Signal adapter/container client, config/docs/tests, and a `ws` dependency to support bbernhard/signal-cli-rest-api REST `/v2/send` and WebSocket `/v1/receive/{account}` alongside native signal-cli JSON-RPC/SSE.
