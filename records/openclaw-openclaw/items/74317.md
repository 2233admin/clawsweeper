## Summary

Keep open. The latest PR is an active XL bundled-plugin feature with security-sensitive network, credential, dynamic-code, dependency, and product-ownership implications; current main does not already include it, and the latest head still has merge-blocking review issues around SecretRef-aware credentials and unbounded initialization waits.

## What This Changes

The PR adds a bundled `extensions/guardrails` plugin with `before_dispatch` allow/block checks, blacklist/HTTP/dynamic-import connectors, package metadata, static asset packaging, lockfile updates, and tests.

## Best Possible Solution

Maintainers should decide the product/security ownership model first, then either require the PR to become a properly documented official bundled security plugin with SecretRef-aware credentials, bounded initialization fallback, dependency review, and docs/changelog alignment, or route the implementation to external ClawHub distribution on the existing plugin hook surface.

## Reproduction Assessment

Yes for the blocking review defects: configure `http.apiKey` as a SecretRef object and the PR schema/runtime types do not support it, or use an HTTP/import backend whose `init()` never settles and the first matching `before_dispatch` waits indefinitely. For the feature itself, reproduction is not applicable because current main does not include a bundled guardrails plugin.

## Solution Assessment

No, not as-is. The plugin-shaped approach uses an appropriate seam, but a bundled security plugin should not merge before credential handling, timeout/fallback semantics, docs, dependency review, and maintainer/secops ownership are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make provider keys SecretRef-aware:** `extensions/guardrails/openclaw.plugin.json:26-28`
  - body: The new HTTP connector accepts provider API keys from plugin config, but the manifest declares both global and per-channel `apiKey` fields as plain strings and the runtime type is `apiKey: string`. Operators therefore cannot use the existing SecretRef/redaction pipeline for guardrail provider tokens, forcing plaintext config or causing SecretRef objects to be rejected/dropped.
  - confidence: 0.92
- **[P2] Bound HTTP provider initialization waits:** `extensions/guardrails/index.ts:178`
  - body: `makeHttpBackendFn` awaits `entry.initPromise` before calling `adapter.check`, but `timeoutMs` is only passed into `check`. If a registered provider's `init()` never settles, the first matching `before_dispatch` stalls indefinitely instead of returning the configured fail-open/fail-closed fallback.
  - confidence: 0.88
- **[P2] Bound import connector initialization waits:** `extensions/guardrails/index.ts:322`
  - body: The import connector waits for `createImportBackend` and the module's optional `init(args)` before the `Promise.race` timeout wrapper is installed. A bad custom checker can hang external-channel dispatch during initialization rather than honoring `timeoutMs` and `fallbackOnError`.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds credential-bearing outbound checks and trusted dynamic code execution without fully integrating SecretRefs or bounding initialization waits.

Concerns:

- **[high] Plaintext provider API keys:** `extensions/guardrails/openclaw.plugin.json:26`
  - body: `http.apiKey` and per-channel override keys are declared as plain strings and are not listed in `configContracts.secretInputs`, so guardrail provider tokens would bypass the existing SecretRef discovery/redaction contract.
  - confidence: 0.92
- **[medium] Unbounded initialization can stall dispatch:** `extensions/guardrails/index.ts:178`
  - body: HTTP and import initialization promises are awaited from the `before_dispatch` path without the plugin's configured timeout, making a hung provider/module an availability risk at the external-channel ingress boundary.
  - confidence: 0.88
- **[medium] Trusted local code execution surface:** `extensions/guardrails/src/import-connector.ts:23`
  - body: The import connector loads operator-specified local modules through `jiti`; that may be acceptable for trusted deployments, but bundled inclusion needs explicit docs and threat-model clarity around trusted code, path expectations, hot reload, and lifecycle behavior.
  - confidence: 0.82
