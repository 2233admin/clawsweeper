## Summary

Keep open for maintainer and security review. Current main does not already ship the proposed guardrails plugin, and the existing hook surface can support a plugin-shaped implementation, but this PR adds an XL bundled security feature with outbound HTTP checks, API-key handling, dynamic imported policy code, new runtime dependencies, and a linked open security-labeled design issue. That is product/security ownership work, not conservative cleanup.

## What This Changes

This PR adds a bundled `extensions/guardrails` plugin that registers a `before_dispatch` allow/block check with blacklist, HTTP-provider, and dynamic import connectors, plus package metadata, runtime asset packaging, lockfile updates, and tests.

## Best Possible Solution

Maintainers should first decide the ownership model: either promote this to an official bundled security plugin with secops review, documented `before_dispatch` ingress semantics, resolved review findings, and full packaging/config docs, or route the implementation to an external ClawHub plugin using the existing hook surface.
