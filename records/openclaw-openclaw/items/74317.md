## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Load the guardrails hook when configured:** `extensions/guardrails/openclaw.plugin.json:5`
  - body: The manifest sets `activation.onStartup` to `false` and does not declare any `activation.onConfigPaths`. Current startup resolution only imports root hook plugins when startup activation or configured activation paths select them, so enabling `plugins.entries.guardrails` would not load `index.ts` and the `before_dispatch` guardrail would never register. Add startup activation or a config activation path so configured guardrails actually run.
  - confidence: 0.9
- **[P2] Strip common invisible bypass characters:** `extensions/guardrails/src/normalize.ts:1`
  - body: The blacklist connector only removes U+200B/C/D, U+2060, and U+FEFF before matching. A blocked keyword interleaved with soft hyphen, LRM/RLM, bidi controls, isolates, or Hangul fillers will not match, so the new safety plugin can pass input it is meant to block. Expand the strip set and add table-driven tests.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bundled guardrails plugin and still has a concrete silent-inactive activation bug plus dynamic-code and outbound-provider surfaces that need maintainer/security review.

Concerns:

- **[high] Configured guardrails can silently stay inactive:** `extensions/guardrails/openclaw.plugin.json:5`
  - body: Because the manifest disables startup activation without adding config activation paths, an operator can configure the plugin and still get no pre-dispatch enforcement. For a guardrail feature, silent non-enforcement is a security-relevant false sense of protection.
  - confidence: 0.9
- **[medium] Dynamic import connector executes configured local code:** `extensions/guardrails/src/import-connector.ts:26`
  - body: The import connector loads an operator-specified script through `jiti.import`. That may be acceptable for trusted deployments, but bundling it as an official security plugin expands the code-execution surface and should get explicit policy review.
  - confidence: 0.78
- **[low] New runtime dependencies need supply-chain review:** `extensions/guardrails/package.json:8`
  - body: The plugin adds `@monyone/aho-corasick` and `jiti` to the runtime dependency graph. That is consistent with the implementation, but it should be reviewed before an official bundled security plugin ships.
  - confidence: 0.72
