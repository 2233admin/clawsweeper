## Summary

Keep this PR open. Current main still does not implement a context-engine result field or stream boundary that lets an engine select the model-visible tool schema subset. Main now passes `availableTools: Set<string>` into `assemble()`, which partially addresses the earlier review feedback about tool awareness, but it is prompt-guidance metadata rather than a schema override mechanism.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. A good implementation should add a backwards-compatible context-engine contract for per-turn tool schema selection, pass sufficient live tool information or clearly define a name-based override, honor empty-subset semantics deliberately, filter at the model-request boundary without broadening executor policy, and add focused PI/Codex/runtime tests plus docs for the behavior.
