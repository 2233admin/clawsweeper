## Summary

Keep this PR open for normal maintainer review. ## What This Changes

The branch adds a cron startup WARN log when an agentTurn payload omits `lightContext: true` and a non-empty `toolsAllow` list.

## Best Possible Solution

Land a narrow diagnostic after fixing the warning scope or wording, then treat creation/update-time validation, timeout defaults, or stronger docs as separate product follow-up for the linked issue if maintainers want prevention rather than run-time diagnosis.

## Reproduction Assessment

Yes at the code-path level: an isolated `agentTurn` without `lightContext` leaves `bootstrapContextMode` unset, and a missing/non-empty `toolsAllow` distinction controls whether the embedded runner keeps the full prompt and skills context. The exact 60 second timeout remains hardware/workspace dependent.

## Solution Assessment

Mostly yes, with a small adjustment. A run-time warning is a low-risk diagnostic fix, but the cleanest version should either gate the warning to `sessionTarget === "isolated"` or make the message apply to all agentTurn cron jobs that share this path.
