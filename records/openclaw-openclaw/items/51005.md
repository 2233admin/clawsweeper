## Summary

Keep this PR open. Current main still rejects `freshness`, `date_after`, and `date_before` before the Brave `llm-context` request path, while Brave's own LLM Context docs list `freshness` and custom date ranges as supported query parameters. The PR is still a focused candidate to port/rebase, not obsolete.

## Best Possible Solution

Keep the PR open and ask the contributor or a maintainer to rebase/port it onto current main, preserving the existing `ui_lang` rejection while allowing Brave `llm-context` `freshness` and date range filters with focused regression tests and docs updates.
