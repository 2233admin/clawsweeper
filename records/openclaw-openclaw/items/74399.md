## Summary

Keep open. Current main still starts the classic `openclaw onboard` wizard without an upfront ETA or long-running setup expectation, and the user-facing onboarding docs describe the setup scope without warning that provider auth, channels, daemon install, skills, plugins, and network work can take a while. The related open issues are adjacent rather than canonical duplicates.

## What This Changes

Request to add an upfront duration or timeline expectation during `openclaw onboard` so new users know setup may take a long time.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding flow before the main prompts begin, using conservative wording such as setup can take several minutes or longer depending on selected providers, channels, daemon install, skills, plugins, and network conditions; mirror that expectation in the CLI onboarding docs without promising an exact 40-minute duration.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: `openclaw onboard` reaches `setupWizardCommand` -> `runInteractiveSetup` -> `runSetupWizard`, where the first wizard output has no ETA or long-running setup note.

## Solution Assessment

Yes. A small UX/docs change is the narrowest maintainable solution; a fixed 40-minute promise would be less safe than a conservative duration expectation unless maintainers add timing data.
