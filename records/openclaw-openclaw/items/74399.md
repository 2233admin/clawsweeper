## Repair Work Prompt

Fix the openclaw/openclaw issue requesting an upfront ETA or long-running setup expectation for `openclaw onboard`. Add conservative copy early in the classic CLI wizard before the main prompts begin, likely near `runSetupWizard` after the intro/header and before risk acknowledgement or first substantial prompt. The copy should say setup can take a while because provider auth, channel setup, daemon install, skills, and optional dependencies may run, but do not promise a hard-coded 40-minute duration unless maintainers have timing data. Mirror the expectation in the public CLI onboarding docs. Keep the change scoped to UX copy, docs, and focused tests; do not change provider/channel install behavior, daemon policy, or the modern Crestodian path unless needed by existing tests. Add or update a focused test in `src/wizard/setup.test.ts` proving the early note is emitted, and update docs under `docs/start/wizard.md`, `docs/cli/onboard.md`, and/or `docs/reference/wizard.md` where the onboarding steps are described.
