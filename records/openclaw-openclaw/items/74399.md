Keep open. Current main still lacks an upfront duration expectation in the classic onboarding flow, the relevant docs also omit ETA or long-running setup copy, and the related title-search items are adjacent onboarding work rather than canonical duplicates. ## What This Changes

The issue requests an early `openclaw onboard` message that warns beginners setup can take a long time, around 40 minutes.

## Best Possible Solution

Add a concise early note to the classic CLI wizard that sets expectations before prompts begin, mentions that provider auth, channel setup, daemon install, skills, and optional dependencies can take a while, and points users to skip flags or later configuration where appropriate; mirror that expectation in the public onboarding docs if the runtime copy changes.

Implement a focused fix for this issue: `openclaw onboard` should set beginner expectations early in the classic interactive CLI wizard before the user is deep in setup. Add a concise note near the start of `runSetupWizard` after the existing intro/security flow or another clearly early point that fits the Clack UX. Mention that setup time varies and can take a while because provider auth, channel setup, daemon install, skills, and optional dependency installs may run; avoid asserting a universal exact 40-minute ETA unless maintainers have timing data.
