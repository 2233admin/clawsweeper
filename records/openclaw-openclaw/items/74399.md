## Summary

Keep open. Current main still lacks an upfront duration or long-running setup expectation in the default interactive `openclaw onboard` flow, and the relevant CLI onboarding docs describe the steps without warning that provider auth, channels, daemon work, search, skills, and optional dependency setup can take a while. The existing ClawSweeper comments already identified the same gap, and the current checkout still matches that assessment.

## What This Changes

Request to add an ETA or early expectation-setting timeline for users when `openclaw onboard` starts.

## Best Possible Solution

Add a concise early note in the classic CLI wizard before substantive prompts begin, saying setup can take a while when provider auth, channels, daemon install, web search, skills, and optional dependencies are included. Mirror the same expectation in the public onboarding docs, preferably using cautious range/qualitative wording rather than a fixed 40-minute promise unless timing data supports it.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by tracing `openclaw onboard` through `src/commands/onboard.ts` into `runSetupWizard`, where the first UI output has no ETA note and later code still runs multiple potentially long setup phases.

## Solution Assessment

Yes. The narrowest maintainable fix is a small wizard copy plus docs update; a full progress estimator or exact 40-minute countdown would be harder to keep accurate and is not necessary to solve the reported beginner-experience gap.
