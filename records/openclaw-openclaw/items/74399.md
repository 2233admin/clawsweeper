## Summary

Keep this issue open. Current main shows the normal onboarding header, security acknowledgement, QuickStart summary, step-specific notes, and progress spinners for install work, but it does not show an upfront ETA or long-running setup warning before `openclaw onboard` begins the flow. The related title-search items are adjacent onboarding/timeline work, not a canonical duplicate of this request.

## What This Changes

Add an upfront ETA or long-running setup warning to `openclaw onboard` so first-time users know setup may take substantial time.

## Best Possible Solution

Add an early, concise onboarding note for the classic CLI wizard that sets expectations before the flow starts, mentions the steps most likely to take time, and points users to skip flags or later configuration where appropriate; mirror the same expectation in the public onboarding docs if the user-facing copy changes.
