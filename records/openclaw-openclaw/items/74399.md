## Summary

Keep open. Current main still does not show an upfront ETA or long-running setup expectation when the classic `openclaw onboard` flow starts, and the relevant onboarding docs also omit a duration warning. This is a narrow, current UX/docs gap suitable for a focused repair rather than closure.

## What This Changes

Request to show an upfront duration or long-running setup expectation during `openclaw onboard`, with matching onboarding documentation.

## Best Possible Solution

Add a concise early note to the classic CLI onboarding flow before the main prompts begin, explaining that setup can take several minutes and may take longer for provider auth, channels, daemon install, skills, plugins, and network-bound steps; mirror the same expectation in the CLI onboarding docs without changing runtime/provider behavior.

## Reproduction Assessment

Yes. Source review gives a high-confidence reproduction path: `openclaw onboard` enters `setupWizardCommand` -> `runInteractiveSetup` -> `runSetupWizard`, where the first visible wizard actions contain no ETA or long-running setup note before prompts begin.

## Solution Assessment

Yes, a small UX/docs change is the narrowest maintainable solution. The safer alternative to the reporter's fixed 40-minute wording is conservative duration copy that warns about variable setup time without overpromising.
