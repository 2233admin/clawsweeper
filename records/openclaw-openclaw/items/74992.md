## Summary

Keep this PR open. The reported missing-cost problem is real against the current model resolution path, but the patch is not mergeable as submitted because it replaces the skipPiDiscovery registry factory path with a direct call to a private upstream constructor.

## What This Changes

The PR normalizes resolved model cost metadata to zero-valued defaults, adds coverage for configured provider models that omit cost, and changes the skipPiDiscovery empty ModelRegistry construction path.

## Best Possible Solution

Keep the zero-cost normalization direction, but construct the empty skipPiDiscovery registry through the upstream public factory API, preserving the existing fallback behavior and adding the proposed regression coverage for configured models with omitted or partial cost metadata.

## Reproduction Assessment

Yes. A configured provider model with `api` and no `cost` can flow from `buildInlineProviderModels` into Pi transports, and the pinned Pi `calculateCost` implementation dereferences `model.cost` directly; the PR's targeted test is an appropriate reproduction, though I did not run it because this review is read-only.

## Solution Assessment

No, not as submitted. Defaulting missing cost fields in `normalizeResolvedModel` is a narrow maintainable fix, but the unrelated constructor replacement should use `PiModelRegistryClass.create(authStorage, "")` or retain the public `inMemory`/`create` fallback to remain type-correct.
