## Summary

Keep this issue open. Current main still lacks first-class outgoing Telegram custom emoji support: there is no shared custom-emoji Markdown IR span, no Telegram `entities` or `caption_entities` send surface, and the shared message tool cannot request custom emoji entities. The related open and closed items are useful context, but they do not supersede the remaining message/caption/reply work.

## What This Changes

Add Telegram custom emoji support for bot messages, captions, and replies, including custom emoji identifiers/entities and a usable agent-facing send path.

## Best Possible Solution

Choose the product/API representation first, then implement validated Telegram custom emoji support through the Telegram plugin send/delivery/edit/streaming paths and the shared message tool, with docs and focused regression tests that preserve existing HTML fallback behavior.
