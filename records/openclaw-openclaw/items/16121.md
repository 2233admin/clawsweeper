## Summary

Keep this issue open. Current main still lacks first-class outgoing Telegram custom emoji support across the Markdown/IR renderer, direct Telegram send options, media captions, and the shared message tool. Related #40848 remains open for the narrower entities surface, and closed-unmerged PR #13905 is useful prior art rather than a shipped fix.

## What This Changes

Request support for Telegram Bot API custom emojis in Telegram messages, captions, and replies, including custom emoji identifiers and a way to send/render them where normal emoji are supported.

## Best Possible Solution

Keep this issue open for maintainer API-shape review. Decide whether Telegram custom emoji should be represented as a Telegram-owned validated `entities`/`caption_entities` surface, a shared Markdown/IR custom emoji span, or both; then thread it through the Telegram message tool, direct send helpers, captions, reply delivery, edits/streaming previews, docs, and focused Telegram tests while coordinating with #40848, #10944, and prior PR #13905.
