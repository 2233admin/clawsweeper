## Summary

Keep this issue open. Current main still lacks first-class outgoing Telegram custom emoji support for messages, captions, and replies: shared Markdown IR has no custom-emoji span, Telegram formatting emits the existing HTML subset, send/caption paths still use HTML parse mode without entity arrays, and the shared message tool has no custom emoji or Telegram entity input surface. The closed #13905 PR is useful prior art, #40848 was closed as a duplicate of this tracker, and open #47562 is scoped to reactions rather than message/caption rendering.

## What This Changes

Request support for Telegram custom emojis in outgoing messages, captions, and replies, including custom emoji identifiers and rendering/sending according to the Telegram Bot API.

## Best Possible Solution

Define the public representation first, then implement validated Telegram custom emoji entities through the Telegram plugin send, caption, reply, edit, delivery, and streaming paths plus the shared message tool, with docs and focused regression tests that preserve current HTML fallback behavior.
