## Summary

Keep this issue open. Current main still performs synchronous skill discovery and SKILL.md parsing on each loader call, with only session-level snapshot reuse and watcher invalidation above the loader. The author-linked PR #59547 is closed and unmerged in the provided GitHub context, and its head commit is not present in this checkout.

## Best Possible Solution

Keep the issue open and review a fresh, scoped implementation in the skill loader: cache direct single-skill loads by source and resolved directory, validate hits with SKILL.md metadata, rerun containment and anti-symlink checks on hits, prune entries under scanned roots, add bounded eviction, and cover unchanged, modified, deleted, oversized, symlink-escape, nested-root, and eviction cases. Session snapshot/watch reuse should remain a higher-level optimization, not the substitute for the requested loader cache.

## Risks / Open Questions

- A future loader cache needs conservative invalidation and must preserve the existing realpath and anti-symlink containment checks before serving cached entries.
- Caching nested fallback-loaded skills without explicit dependency tracking could serve stale or unsafe results, which is why the issue's proposed direct-single-skill-only scope remains a useful acceptance criterion.
