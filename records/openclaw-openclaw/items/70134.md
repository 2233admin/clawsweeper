## Summary

Keep PR #70134 open. Current main still uses auth-profiles.json mtime in the models config ready-cache fingerprint, so the PR's central semantic-fingerprint behavior is not implemented or superseded. The PR discussion also contains useful review/author follow-up on profile order, mixed JWT audiences, legacy apiKey/keyRef normalization, and an unresolved OAuth token-content invalidation tradeoff that needs normal maintainer review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best path is a core models-config change that replaces auth-profiles.json mtime with a semantic auth-profile fingerprint while preserving models.json mtime invalidation, profile insertion-order semantics, normalized legacy apiKey/keyRef handling, and a deliberate OAuth token-content policy, with focused regression tests around each cache-invalidating case.
