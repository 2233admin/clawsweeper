## Risks / Open Questions

- The PR diff is based on an older `models-config.ts` fingerprint signature and must preserve current-main workspace, plugin metadata, provider-discovery provider list, and provider-discovery timeout inputs when rebased.
- The OAuth branch fingerprints `access` and `refresh` as presence booleans while current main uses the actual OAuth access token for discovery; a token rotation with the same identity and expiry bucket could skip replanning even if catalog entitlements changed.
- The PR handles secret material by hashing and length metadata rather than raw values. That is better than leaking plaintext, but the fingerprint output should remain in-memory and unlogged because hashes of low-entropy env values can still be sensitive.
