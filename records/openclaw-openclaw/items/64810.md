## Summary

Keep open. Current main still lacks the active reply-run guard described in the report, and the provided GitHub context shows #64963 is an open, unmerged PR with closing syntax for this issue, so it remains the implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open while maintainers review/rebase/land #64963 or apply the equivalent focused fix directly. The likely code path is to have src/infra/heartbeat-runner.ts, after heartbeat preflight resolves sessionKey and before the session-lane queue check, skip with requests-in-flight when resolveActiveReplyRunSessionId(sessionKey) is truthy, then add the regression test for an active reply operation with otherwise idle queues.
