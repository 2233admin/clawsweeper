## Summary

Keep open. Current main has a non-QR setup-code path for copy/paste mobile pairing, but it does not implement the requested `openclaw invite` command, short human-dictated invite codes, cloud invite registration/resolution endpoints, or `gateway.cloud` config. The related cloud relay issue is still open, so this needs maintainer product/security judgment rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer product/security review. The likely best path is to either design `openclaw invite` together with the open cloud relay work, reusing the existing bootstrap-token and `openclaw devices approve` flow, or have a maintainer explicitly close the request in favor of the documented `openclaw qr --setup-code-only` setup-code path if short cloud invite codes are out of scope.

## Risks / Open Questions

- The existing setup-code flow is close enough to solve copy/paste pairing, but it does not cover the short phone-dictated cloud invite-code UX requested here.
- Cloud-discoverable invite codes would need explicit security, expiry, collision, abuse-rate-limit, and backend trust design because current setup codes contain short-lived bootstrap material and docs say to treat them like a password while valid.
