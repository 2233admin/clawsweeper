## Summary

Keep this PR open. Current main still does not implement the proposed Kimi-specific OpenAI-compatible tool-schema compaction: the shared tool-parameter normalizer only applies Gemini cleanup and compat-declared keyword stripping, the provider-owned normalizeToolSchemas seam exists but NVIDIA and Moonshot do not register a Kimi schema normalizer, and the related NVIDIA NIM Kimi regression remains open in #55942. The PR is not obsolete, but it likely needs a rebase onto the current provider-owned schema-normalization layout and stronger validation because the related bug report says this compaction may still be insufficient.

## Best Possible Solution

Keep this PR open as an implementation candidate, but rebase it onto the current provider-owned schema-normalization path. The best fix should live either in a shared plugin-SDK helper reused by the relevant providers or directly in the NVIDIA/Moonshot provider hook if maintainers want it owner-local, preserve the already-discussed $ref inlining and provider-alias safeguards, and validate against #55942 before merge.
