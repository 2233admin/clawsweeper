## Summary

Keep this PR open. Current main still builds the WebChat session dropdown from stored session rows plus the active session key, not from all configured agents, and the PR remains the active implementation candidate for the configured-agent dropdown bug tracked by #41933.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best fix belongs in `ui/src/ui/chat/session-controls.ts`: combine `sessions.list` rows with `agents.list`, synthesize normalized per-agent main-session keys from the configured `mainKey`, avoid per-agent switch targets for global-scope sessions, use normalized dedupe/row lookup, and keep focused UI regression tests.
