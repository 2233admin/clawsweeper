## Summary

Keep this PR open. Current main still builds the WebChat session dropdown from persisted session rows plus the active session key, so configured agents without stored sessions are still missing. The related configured-agent tracker remains open, and this PR is still the active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best fix belongs in `ui/src/ui/chat/session-controls.ts`: combine `sessions.list` rows with `agents.list`, synthesize normalized per-agent main-session keys using the configured `mainKey`, skip per-agent switch targets for global-scope sessions, normalize dedupe/row lookup, and keep focused regression tests plus changelog credit.
