## Summary

Current main still matches the reported false positive. `gateway probe` can probe both an SSH tunnel target and the configured remote URL for the same logical gateway, and `buildGatewayStatusWarnings` still emits `multiple_gateways` from the count of reachable targets rather than from any gateway identity comparison. The issue is concrete, recent, source-backed, and narrow enough for a focused fix PR.

## Best Possible Solution

Keep this open and fix the gateway-status owner path. Classify reachable targets by a conservative non-null gateway self identity, suppress or downgrade `multiple_gateways` when all reachable targets share that identity, and preserve the warning for distinct identities or unknown/mixed identity cases. Add focused output tests for same-identity SSH tunnel plus configured remote URL and for genuinely distinct reachable gateways.
