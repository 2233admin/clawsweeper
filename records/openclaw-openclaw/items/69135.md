## Summary

Current main still matches the reported bug: `gateway probe` can probe an SSH tunnel URL and a configured remote URL for the same logical gateway, then emits `multiple_gateways` solely because more than one target is reachable. The warning path does not compare the `self` gateway identity that the probe already extracts.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/commands/gateway-status/output.ts`: group reachable probe results by a stable gateway identity when `self` presence has host/IP, suppress or downgrade the warning when all reachable targets share that identity, and keep `multiple_gateways` for distinct identities or identity-less ambiguous multiple targets. Add focused coverage in `src/commands/gateway-status/output.test.ts` or the command suite for `sshTunnel` plus `configRemote` returning the same `self`, and update the gateway probe warning docs/copy if a same-gateway multi-transport informational message is added.

## Evidence
