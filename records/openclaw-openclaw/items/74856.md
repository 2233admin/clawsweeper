## What This Changes

This PR normalizes nullish plugin command handler results to an empty reply payload, adds nullish guards in text and Telegram native command consumers, and adds regression tests for undefined handler results.

## Best Possible Solution

Land one canonical fix that treats nullish plugin command handler returns as an empty reply at the execution boundary, keeps downstream command consumers tolerant of mocked or wrapped executors, and includes regression coverage for both generic text commands and Telegram native commands.

## Reproduction Assessment

Yes. A plugin command handler with no explicit return reproduces the source-level path on current main: the executor returns `undefined`, then text and Telegram command consumers access properties on the result.

## Solution Assessment

Yes. Normalizing at `executePluginCommand` is the narrowest maintainable fix because it protects all command surfaces using that helper; the added consumer guards are redundant after the central fix but acceptable defense around mocks or alternate wrappers.
