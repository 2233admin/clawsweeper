## Summary

Keep open. Current main does not implement the EV-002B Skill Workshop hardening, and the PR is an active implementation candidate, but the latest head still has a blocking security issue in the final write path: the temp-file write happens through a pathname after a containment check, so a symlink swap can redirect that temp write outside the skill root before the later check throws.

## What This Changes

The PR adds realpath containment, stale-write checks, prompt-budget enforcement, SDK exports, docs, and tests for Skill Workshop, while also carrying merge-support changes across QA, gateway RPC, bundled plugin resolution, proxy, channel, and test surfaces.

## Best Possible Solution

Update the EV-002B implementation so Skill Workshop still owns the procedural write funnel and SDK/docs contract, but commits `SKILL.md` and support files through an existing pinned/root-bound write primitive or an equivalent no-follow directory-fd writer; split unrelated merge-support changes if maintainers want a narrower security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Use a pinned write for the final Skill Workshop commit:** `extensions/skill-workshop/src/skills.ts:144`
  - body: `atomicWriteInsideRoot()` still checks containment, then calls `fsPromises.writeFile(tempPath, ...)` through a path string. If the checked skill directory is replaced with a symlink between the check and this write, the temp file is created outside the skill root before the later check throws. Route the final commit through a root-bound no-follow writer, such as the existing pinned write helper, so no write occurs outside the verified root.
  - confidence: 0.82
