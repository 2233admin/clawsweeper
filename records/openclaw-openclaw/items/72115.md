## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review finding: use an OPENCLAW_BUNDLED_PLUGINS_DIR value such as `~/bundled`; `resolveBundledPluginsDir` expands it via `resolveUserPath`, while the new facade-loader check compares against `path.resolve` of the raw string and can disable runtime dependency staging incorrectly.

## Solution Assessment

Is this the best way to solve the issue? Not yet. The Skill Workshop hardening approach is aligned with the existing root-bound write primitives, but the proposed patch should normalize the bundled override with the same resolver as the plugin root discovery code and needs maintainer scope/security review before merge.

## Review Findings

Full review comments:

- **[P2] Normalize bundled overrides before comparing paths:** `src/plugin-sdk/facade-loader.ts:149-156`
  - body: `OPENCLAW_BUNDLED_PLUGINS_DIR` is resolved with `resolveUserPath` in `resolveBundledPluginsDir`, so values like `~/bundled` can produce an absolute override root. Use the same user-path/realpath normalization before deciding to skip runtime-dependency staging.
