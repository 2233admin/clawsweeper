## Summary

Keep this PR open. Current main still does not warn when `openclaw cron add --message ...` omits `--agent`; the merged cron session-agent inference work is adjacent tool-path behavior, not a replacement for this CLI warning. The linked cron UX issue remains open, and the PR is still a plausible focused implementation candidate after refresh.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Refresh it on current main, add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification so it applies only to `agentTurn` jobs without an explicit nonblank `--agent`, emit the warning on stderr, keep wording accurate for configurable defaults, and add focused tests in `src/cli/cron-cli.test.ts` for missing, blank, explicit-agent, session-key, and system-event cases.
