## Summary

Keep this PR open. Current main still has the timeout propagation gaps the PR targets, but the PR is not safe to close as implemented or merge as-is because review-blocking timeout semantics remain unresolved.

## Best Possible Solution

Keep the PR open for revision or replacement. A mergeable fix should forward numeric agent timeouts only where the caller is actually waiting, preserve fire-and-forget target runtime defaults unless maintainers intentionally change that contract, align session-lock waiting with the effective run budget without doubling it, and add regression coverage for gateway validation, `timeoutSeconds: 0`, A2A helper waits, and contended session locks.

## Risks / Open Questions

- The PR should not be merged as-is because it forwards `timeout` as a string while the gateway contract expects a numeric integer.
- The proposed fire-and-forget behavior would force `timeoutSeconds: 0` target runs to 30 seconds instead of preserving the current default agent runtime budget, which can break legitimate background work.
- Passing the full run timeout into session lock acquisition can let a contended run spend the whole caller budget waiting for the lock and then still start model execution, preserving part of the misleading timeout behavior.
- The proposed A2A helper change grants target runs the full timeout while the helper wait remains capped at 60 seconds, leaving long target runs active after the caller has returned `undefined`.
- No supply-chain or secrets regression was apparent from the PR file list, but the runtime resource-budget semantics need maintainer review.
