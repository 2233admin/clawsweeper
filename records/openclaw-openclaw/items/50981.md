## Summary

Keep this PR open. Current main still has the timeout propagation gaps the PR targets: `sessions_send` starts nested `agent` runs without forwarding `timeoutSeconds`, `runAgentStep` starts its nested `agent` run without a target timeout and then caps only the wait, and `runEmbeddedAttempt` still calls `acquireSessionWriteLock` without `timeoutMs`, so the lock helper falls back to its default wait. The PR is relevant, but the current diff is not merge-ready because review comments identify real functional regressions around string-valued `agent.timeout`, fire-and-forget target budgets, and timeout budget accounting.

## Best Possible Solution

Keep the PR open for maintainer revision or replacement. A mergeable fix should forward a numeric `agent.timeout` only when appropriate, preserve fire-and-forget target-run semantics unless maintainers intentionally change them, count session-lock wait against the effective run/caller budget, align A2A target and wait budgets, and add regression coverage for gateway validation, fire-and-forget sends, contended session locks, and A2A helper waits.
