## Summary

Keep open. Current main still drops text-only tool result payloads for ordinary group chats because `shouldSendToolSummaries` follows the group/Slack/forum progress gate without checking the session verbose level. The PR addresses a real gap supported by the DingTalk AI Card validation context, but its branch is stale and would regress current main's Slack non-DM, Telegram forum-topic, and native direct delivery behavior if carried forward as-is.

## What This Changes

The PR imports verbose-level normalization and changes `dispatch-from-config.ts` so stored `/verbose on` or `/verbose full` can allow group-chat tool summaries while preserving default group suppression and native-source suppression.

## Best Possible Solution

Layer explicit verbose `on` or `full` onto the current main dispatch gate so ordinary non-Slack group chats can receive text tool summaries only after user opt-in, while preserving Slack non-DM suppression, Telegram forum-topic delivery, native direct delivery, exec approvals, and media-only fallback. Keep the broader persistent config-option discussion in #54663 separate.
