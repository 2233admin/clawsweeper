## Summary

Keep open. The reported bug is still real on current main, but this PR branch is stale against the dispatch path and would regress newer shared-surface behavior if rebased mechanically, so it needs a targeted repair rather than closure or merge as-is.

## What This Changes

The PR imports `normalizeVerboseLevel` and changes `dispatch-from-config.ts` so stored `/verbose on` or `/verbose full` can bypass group-chat tool-summary suppression while keeping native command-source sessions suppressed.

## Best Possible Solution

Extend current main's shared verbose/tool-summary gate so explicit stored verbose `on` or `full` enables text tool summaries for ordinary non-Slack group chats, while preserving Slack non-DM suppression, Telegram forum-topic delivery, native direct delivery, exec approval delivery, media-only fallback, and preview-progress suppression.

## Reproduction Assessment

Yes. A focused unit test can seed a group session entry with `verboseLevel: "on"`, emit `opts.onToolResult({ text: "..." })`, and assert `dispatcher.sendToolResult` receives the text payload; current main's existing group suppression test covers the negative side but lacks that positive case.

## Solution Assessment

No. The PR's current condition is not the best fix because it replaces current main's richer surface gate with a group/native-only gate; the safer repair is to layer verbose opt-in into the current shared gate and add focused regression coverage.
