## Summary

Keep open. Current main still acknowledges ACP child dispatch before the ACP turn has durable transcript state, and the initial user prompt is only appended after `acpManager.runTurn` completes. A stalled or failed ACP/Codex turn can therefore still leave `sessions_history` empty after an accepted spawn.

## Best Possible Solution

Keep this issue open and fix the ACP spawn/turn contract so an accepted ACP child has durable diagnostic history before `runTurn` can stall. The narrowest path is to persist the initial task as a child user transcript row, or expose an equivalent child-visible task event through `sessions_history`, before returning accepted or before entering the asynchronous ACP turn. Add regression coverage for stalled/failed pre-output ACP turns and for successful turns to prove the prompt is not duplicated.
