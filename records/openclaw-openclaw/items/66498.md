## Summary

Keep open. Current main still rejects Gemini `web_search` freshness/date filters before request construction, still sends an empty Google Search grounding tool with no temporal range, and the Gemini docs still describe freshness as unsupported. The broader open #17925 is adjacent native-search work, but it is not a canonical replacement for this narrower Gemini freshness gap.

## What This Changes

Request Gemini `web_search` to honor OpenClaw freshness/date parameters by passing an equivalent temporal filter to Google Search grounding.

## Best Possible Solution

Implement the support inside the Google plugin path: normalize supported OpenClaw freshness/date inputs into a Gemini Google Search grounding interval, pass it in the request, include the normalized interval in the Gemini cache key, and update the Gemini/web docs plus focused provider tests.
