## Summary

Keep this PR open. Current main still does not implement the intended behavior: `agents.defaults.models` entries are still strict `alias`/`params`/`streaming` objects, generated schema still rejects additional model metadata fields, and `applyModelDefaults()` still does not merge passthrough fields from agent defaults into provider model definitions. The PR remains a focused implementation candidate for #44468, with useful review discussion already addressing the mutation-preservation bug, but it still needs the normalized model-ref lookup concern and generated schema/docs follow-through resolved before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for allowing provider-level model metadata overrides through `agents.defaults.models`. The best path is to revise override lookup to normalize provider/model refs consistently with existing model-selection helpers, retain mutation-preservation coverage, add focused tests for `contextWindow`, `maxTokens`, novel passthrough fields, and normalized provider aliases, then update generated schema/help/docs before maintainer review and merge.
