## Summary

Keep open. This is a recent, detailed gateway-host exec/macOS APFS stale-read report, and current main still matches the affected YOLO host-exec shape: gateway `security=full` plus `ask=off` skips script preflight, builds a fresh shell `-c` child argv for non-sandbox exec, and reaches `child_process.spawn` through the supervisor for each run. I found no current source, docs, changelog, or test evidence of a stale-read mitigation, cache invalidation, retry, `F_NOCACHE`/fcntl handling, or regression coverage for atomic rename followed by reads from separate gateway-host exec calls. The prior ClawSweeper comment already kept this open, and the provided related-item context covers different stale-state/watch/log/session-lock topics rather than this APFS cross-exec read race.

## Best Possible Solution

Keep this issue open for focused maintainer investigation. The next useful path is a macOS/APFS repro comparing v2026.4.19, v2026.4.20, the latest release, and current main under gateway-host exec with `security=full` and `ask=off`. If confirmed, fix the core gateway-host exec/runtime path or add a scoped diagnostic/workaround, and add regression coverage around atomic rename followed by reads from separate exec calls.
