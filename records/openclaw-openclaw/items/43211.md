## Summary

Current main still does not implement the PR's requested hyphen-stripped model allowlist behavior. The PR remains a plausible fix candidate, but it should stay open for maintainer review/rework because the proposed unconditional variant expansion changes a model authorization boundary and can make distinct provider IDs share allowlist entries.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The best path is to add a concrete reproduction for the runtime path that strips provider hyphens, then fix that path directly or model the equivalence through explicit provider/plugin alias metadata. Regression coverage should prove legitimate aliases resolve while distinct providers such as `my-provider` and `myprovider` remain separately allowlisted unless explicitly linked.
