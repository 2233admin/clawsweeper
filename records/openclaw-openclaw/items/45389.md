## Summary

Keep open. Current main has adjacent protections for high-context timeout loops, including timeout-triggered compaction, bounded in-run retry/failover, provider/profile cooldowns, and a separate WebSocket transport cooldown, but it still does not implement the requested configurable session-level circuit breaker that counts consecutive LLM timeouts across runs, marks or pauses the session as degraded, and emits a system event/webhook. Related PR #42933 remains the closest open implementation candidate rather than proof this is fixed.

## Best Possible Solution

Keep this issue open and consolidate it with the active session-breaker work, preferably by reviewing/reworking #42933 or naming a canonical tracker. The best implementation would add a documented per-agent/default timeout breaker threshold, persist per-session consecutive LLM timeout state across heartbeat/cron/interactive runs, reset on success or explicit operator action, mark or pause degraded sessions, and emit a sanitized system event/webhook notification. Ryce's recovery comment should inform the reset/degraded semantics so operators have a clean pre-spiral recovery path.
