## Summary

Current main still only partially covers the request. WebChat/Control UI can switch sessions through a chat-header select and the separate Sessions page, and the backend has a `sessions.create` API for dashboard sessions. It does not implement the requested Discord-style session sidebar, and the chat New session button still sends `/new` instead of creating a new dashboard session and switching to it. The supplied timeline also shows open maintainer PR #73725 as the active follow-up for the WebChat create/switch slice, so closing this issue before that lands would be premature.

## Best Possible Solution

Keep this issue open while maintainers review or land #73725 for the focused WebChat New Chat `sessions.create` create/switch path. After that lands, reassess the remaining sidebar/list UX in the Control UI chat/sidebar code and decide whether WebChat should gain a first-class sidebar session surface and/or thread-bound subagent hooks, or whether thread binding should stay limited to channel-backed threads.
