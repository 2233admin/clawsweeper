## Summary

Current main still only partially covers the request. Control UI/WebChat can switch sessions through a chat-header select and the Sessions page, and the backend has `sessions.create`, but the left sidebar is still static navigation, the chat New session button still sends `/new`, and `sessions_spawn(thread: true)` still depends on channel thread-binding hooks rather than a WebChat thread surface. Open maintainer PR #73725 is the active follow-up for the narrower New Chat create/switch path, so closing this before that lands would be premature.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, land, or close #73725 for the focused WebChat New Chat `sessions.create` create/switch path. After that lands, reassess the remaining sidebar/list UX in Control UI and decide whether WebChat should get a first-class sidebar session surface and/or a WebChat-specific thread-binding model, or whether persistent thread-bound subagent sessions should stay limited to channel-backed thread surfaces.
