## Summary

Keep this issue open. The report is same-day, has multiple corroborating comments, and current main does not show a source, test, changelog, or release-backed fix for the specific Feishu reaction/reply latency regression from 2026.4.22 to 2026.4.26.

## Best Possible Solution

Keep this open and triage it as a Feishu latency regression. Ask for a current-main or next-release retest with redacted config and timings for inbound receipt, debounce flush, sender-name lookup, typing reaction add/remove, streaming card start/update/close, model first token, and final delivery. If streaming-card startup/backoff is the delay, change the Feishu reply path so fallback is fast and covered by a regression test; if typing reactions or sender lookups dominate, adjust defaults, parallelize, cache, or document a stronger migration path.
