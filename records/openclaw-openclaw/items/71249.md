## Summary

Keep open. Current main still lacks the generic turn-start cleanup for stale `pluginDebugEntries`, the `updateSessionStoreEntry()` preserve-activity option needed for metadata-only clears, and the verbose-refresh suppression path after a cleanup failure. The PR remains a focused runtime/session-store bug fix with relevant regression coverage, and the supplied review thread shows the one mutation concern was addressed in a later PR commit. The security pass did not find unrelated workflow, dependency, package, permission, secret, or code-execution changes in the PR diff.

## What This Changes

This PR adds a preserve-activity merge option for session entry updates, clears turn-scoped plugin debug entries at reply turn start, suppresses stale debug rehydration after clear failures, documents the turn-scoped contract, and adds session-store plus reply-runner regression tests.

## Best Possible Solution

Review, rebase if needed, and land or revise this focused fix so plugin debug entries are treated as turn-scoped runtime metadata: clear them generically at reply turn start, preserve activity timestamps for that metadata-only clear, and prevent verbose refresh from replaying stale disk state after cleanup failure while keeping Active Memory's writer behavior plugin-owned.
