## Summary

Current main still has the reported gap. OpenAI TTS speed is already forwarded from provider config and Talk/provider overrides into the `/audio/speech` request body, but `[[tts:speed=...]]` is not parsed by the OpenAI speech provider, so the directive path cannot create an OpenAI speed override.

## Best Possible Solution

Keep this open and implement a narrow OpenAI-owned fix in `extensions/openai/speech-provider.ts`: handle `speed` in `parseDirectiveToken`, require `policy.allowVoiceSettings`, parse a finite number, validate OpenAI's `0.25..4.0` range, return `{ speed }`, and add focused regression tests for preferred-provider and explicit `provider=openai` directives plus disabled, invalid, out-of-range, and request-body serialization behavior. Update the TTS docs/OpenAI field reference and changelog if the public directive/config wording changes.

## Clownfish Work Prompt

Fix the OpenAI TTS speed directive gap for this issue. Current main forwards OpenAI TTS speed from provider config and Talk/provider overrides, but `[[tts:speed=1.5]]` and `[[tts:provider=openai speed=1.5]]` do not reach OpenAI because `extensions/openai/speech-provider.ts` parses only OpenAI voice/model directive keys.

Expected implementation boundary:
- Update `extensions/openai/speech-provider.ts` only as needed to handle `speed` in `parseDirectiveToken`.
- Gate the directive on `ctx.policy.allowVoiceSettings`; when disabled, mark it handled without applying an override.
- Parse a finite numeric value, warn on invalid input, validate OpenAI's native `0.25..4.0` range, and return provider overrides as `{ speed: value }`.
- Do not move this into core `src/tts/directives.ts`, do not change ElevenLabs/MiniMax speed semantics, and do not introduce a new plugin API unless existing provider parser context is insufficient.
- Add focused tests for: preferred OpenAI `[[tts:speed=1.5]]`, explicit `[[tts:provider=openai speed=1.5]]`, disabled `allowVoiceSettings`, invalid value warning, out-of-range handling, and final OpenAI `/audio/speech` request body serialization.
- Update `docs/tools/tts.md` if the OpenAI provider field/directive wording needs clarification.
- Add a single-line changelog fix entry in the active version crediting `Thanks @useramuser`.
