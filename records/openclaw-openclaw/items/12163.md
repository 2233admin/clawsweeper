## Summary

Keep this issue open. Current main forwards OpenAI TTS speed when it comes from provider config or Talk/provider overrides, but the reported `[[tts:speed=1.5]]` directive path still does not reach OpenAI because the OpenAI speech provider parser handles voice/model only and returns unhandled for `speed`.

## Best Possible Solution

Keep this issue open. Implement OpenAI-owned speed directive handling in `extensions/openai/speech-provider.ts`: accept `speed` only when `policy.allowVoiceSettings` is enabled, parse a finite number, validate OpenAI's `0.25..4.0` range, return `{ speed }` as provider overrides, and add focused tests for preferred OpenAI `[[tts:speed=1.5]]`, explicit `[[tts:provider=openai speed=1.5]]`, disabled voice settings, invalid/out-of-range values, and final request-body serialization. Update the OpenAI TTS docs field reference if the implementation changes the public config/directive wording.
