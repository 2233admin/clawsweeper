## Summary

Keep open. Current main still does not implement the PR's intended fail-closed Discord behavior: bot identity fetch failures are swallowed into an undefined bot id, startup continues with that optional id, and guild mention-required drops remain guarded by `botId`. The PR is a narrow implementation candidate for the open security-labeled #42219, with useful prior context from #46856 and #49218 carried forward in the body.

## Best Possible Solution

Keep this PR open for review and land it, or land an equivalent focused Discord plugin fix. The right shape is to make `fetchDiscordBotIdentity` fail closed on rejected or malformed `fetchUser("@me")`, remove the redundant `botId` guard from the mention-required drop path, keep cleanup before lifecycle startup intact, and retain focused regressions for rejected identity fetch, no usable id, and absent-botId mention-pattern gating. After merge, #42219 can close as implemented; #46856 and #49218 remain useful prior-credit context.
