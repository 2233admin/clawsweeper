## Summary

Keep open. Current main still has the threshold-only Control UI chat auto-follow behavior this PR targets: no manual-scroll release lock, no wheel-intent handler, no tight return-to-bottom reacquire gate, and no expanded regression coverage for those cases. The PR is open, unmerged, and is a focused UI/test change for the same behavior reported in #37500 and #60837.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review the scroll state-machine patch against current Control UI behavior, request any needed fixes, then land it or apply equivalent manual-scroll release/reacquire handling with the focused `ui/src/ui/app-scroll.test.ts` coverage on main.
