## Summary

Keep this PR open. Current main still does not solve the prepend/root-index use case because the Feishu insert schema requires `after_block_id` and the runtime only inserts after an existing anchor block, but the proposed patch has a concrete validation bug: the new child-position `index` accepts negative and fractional numbers instead of the non-negative integer shape used by the nearby positional schema.

## What This Changes

The PR changes the Feishu `feishu_doc` insert action so `after_block_id` is optional and a new root-child `index` is forwarded when no anchor block is supplied.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving anchored insert behavior, adding focused coverage for root-index and batched insert paths, updating bundled Feishu doc skill guidance, and adding a user-facing changelog entry crediting the contributor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: `index` is a 0-based child position, but `Type.Number()` accepts fractional and negative values, so invalid inputs bypass tool validation and fail later in Feishu. Use `Type.Integer({ minimum: 0 })`, matching the existing `upload_image` index schema.
  - confidence: 0.93
