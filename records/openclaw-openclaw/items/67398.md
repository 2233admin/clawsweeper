## Summary

Keep open. Current main still has the heartbeat outbound-session routing gap, so this PR is not obsolete, but the proposed one-line change needs maintainer review because it changes the key used by delivery policy/media access unless the base session is preserved separately.

## What This Changes

This PR changes heartbeat outbound delivery context to use `runSessionKey` and adds isolated-session regression tests for fresh-base and wake re-entry delivery keys.

## Best Possible Solution

Land a reviewed heartbeat fix that routes outbound metadata and internal hooks to the stable isolated `:heartbeat` run key while preserving the base session for delivery policy/media semantics, with regression coverage for the outbound key plus explicit policy or mirror behavior if those are intended guarantees.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve the base session policy key:** `src/infra/heartbeat-runner.ts:979`
  - body: Changing `session.key` to `runSessionKey` also changes the key used by outbound payload planning and media access whenever no `policyKey` is provided. For isolated group/channel heartbeat sessions, that can make base-conversation policies stop matching once the key gains the `:heartbeat` suffix. Pass the base session as `policySessionKey` or equivalent while using the isolated key for delivery metadata/hooks.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff is small and does not touch dependencies or CI, but it changes a session key that delivery policy and media-access checks consume.

Concerns:

- **[medium] Preserve policy lookup while changing session identity:** `src/infra/heartbeat-runner.ts:979`
  - body: `deliverOutboundPayloads` falls back from `policyKey` to `session.key` for payload planning and uses `session.key` for media-access policy context. Switching heartbeat delivery to `<base>:heartbeat` without preserving the base key can bypass or misapply base conversation policy in isolated heartbeat deliveries.
  - confidence: 0.74
