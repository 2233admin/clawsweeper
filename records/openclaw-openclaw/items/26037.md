## Summary

Keep this issue open. Current main still has only partial Qwen/Bailian Coding Plan thinking support: the Qwen wrapper can emit top-level `enable_thinking`, but the named Coding Plan models are still cataloged as non-reasoning and the custom-provider compat schema still does not expose a Qwen thinking format path.

## What This Changes

Request official OpenClaw support for Ali Bailian/Qwen Coding Plan models with thinking/reasoning enabled, including an official way to use DashScope `enable_thinking`.

## Best Possible Solution

Implement a focused provider-owned Qwen/Bailian Coding Plan fix: verify which Coding Plan models support DashScope top-level `enable_thinking`, mark only those bundled catalog entries or compat paths as thinking-capable, map OpenClaw thinking levels to `enable_thinking` for bundled Qwen and documented custom-provider configs, and update Qwen docs plus targeted regression tests.
