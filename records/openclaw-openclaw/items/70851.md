## Summary

Keep this PR open. Current main still does not implement the intended behavior: the chat picker only hides global/unknown and cron rows, current tests still expect subagent rows to appear in picker options, and the Control UI session row type still lacks the gateway's `spawnDepth` metadata or any explicit `visibility=internal` classification. The PR is a useful implementation candidate, but its UI-only heuristic should be refined toward the source-side classification contract described in the PR body.

## Best Possible Solution

Keep this PR open and revise it toward the PR body's minimal end-state: add a gateway/session-row internal visibility or classification contract for subagent, ACP, and node-backed runtime sessions; have the normal chat picker hide inactive internal rows by default while keeping the currently selected internal row visible; and preserve an admin/debug path to list internal sessions. Use parsed key helpers or source metadata rather than broad substring matching, and keep the broader picker split tracked by #64121.
