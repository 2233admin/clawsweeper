## Summary

Keep PR #70851 open. Current main still builds the chat header session picker from `sessions.list` rows without filtering subagent, ACP, node-backed, or `spawnDepth` internal sessions; it only skips global/unknown rows and cron rows unless selected. Gateway rows now carry `spawnDepth`, but the picker does not consume it and there is no explicit `visibility=internal` contract on the row. The PR is therefore not obsolete, though its own discussion correctly frames the UI-only heuristic as an interim mitigation that should be refined toward gateway-side internal classification plus picker filtering.

## Best Possible Solution

Keep the PR open, but steer it toward the stated minimal end state: add an explicit internal visibility/classification field to gateway session rows for subagent, ACP, and node-backed runtime sessions; expose it through the UI type; hide those rows in `resolveSessionOptionGroups` unless they are the active session; keep admin/debug sessions surfaces able to request or show internal rows; and replace broad substring matching with shared parsed session-key helpers plus focused tests, including the `spawnDepth` fallback.
