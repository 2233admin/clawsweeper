## Summary

Keep open. Current main still lacks length caps for prompt-visible plugin hook injections, and the PR remains relevant hardening work; the branch should be refreshed or replaced because current main now aggregates more prompt-visible fields and sources than this PR covers.

## What This Changes

The PR adds a 50,000-character clamp and warning for selected plugin hook prompt strings, applies an extra combined `prependContext` clamp in the embedded runner, and adds phase/legacy hook truncation tests.

## Best Possible Solution

Keep this PR open and refresh it, or land a successor that clamps every prompt-visible plugin injection field after shared aggregation for both embedded runner and CLI paths. The fix should preserve hook priority semantics, warn with a useful model-visible truncation marker, cover phase/legacy/combined/queued/turn/heartbeat sources with focused tests, and add a changelog credit if it ships.
