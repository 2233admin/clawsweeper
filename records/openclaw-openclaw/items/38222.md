## Summary

Keep open. Current main still has no hook prompt-context length enforcement, and prompt-visible plugin text is still merged and applied unbounded across the shared hook runner, embedded runner, and CLI preparation path. The PR addresses a real hardening gap, but it should not be closed as implemented or merged without maintainer review because the supplied diff covers only part of the current prompt mutation surface.

## What This Changes

The PR adds a 50,000-character clamp helper for selected plugin hook prompt strings, applies it to `before_prompt_build`/`before_agent_start` and the embedded runner's combined `prependContext`, and adds hook truncation tests.

## Best Possible Solution

Land a refreshed version or successor that applies a shared post-aggregation limit to every prompt-visible plugin injection field across phase, legacy, queued, turn-prepare, heartbeat, embedded-runner, and CLI paths, while preserving hook ordering, warning operators clearly, adding a model-visible truncation marker, and covering the behavior with focused tests and changelog credit.
