## Summary

Keep open. Current main still supports only binding-level DM scoping, while Discord guild/channel peers remain isolated into channel session keys; the requested cross-surface session unification is valid but needs maintainer product and privacy review before implementation.

## What This Changes

The issue requests a binding-level session key, alias, or scope override so a Discord guild/channel route binding can share an agent session with Telegram or other direct-message surfaces.

## Best Possible Solution

Add an explicit opt-in route-binding session target contract in config schema/types and route resolution, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged default isolation.
