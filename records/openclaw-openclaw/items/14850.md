## Summary

Keep open. Current main still does not provide a binding-level session target, alias, or scope that can intentionally route a Discord guild/channel conversation into an agent main or DM session. The implemented binding override remains `session.dmScope`, which is schema-limited and resolver-limited to direct-message session scoping, while current tests and docs continue to preserve isolated channel/guild session keys. The existing maintainer review also kept this open because the remaining capability crosses product and privacy/security boundaries.

## What This Changes

Allow route bindings to override or alias the session key so a bound guild/channel conversation and a direct-message surface for the same agent can share conversational continuity.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged unconfigured channel behavior.
