## Summary

Keep open. Current main still sends Feishu text events through the inbound debouncer before command dispatch, and non-debounced command messages with an active same-key chain are queued behind that chain. The issue is also explicitly paired with open PR #43512, so it should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Feishu text commands such as `/stop`, `/new`, and `/status` should bypass Feishu queueing and execute immediately during an active agent run.

## Best Possible Solution

Land or replace #43512 with a current-main Feishu fix that detects command-only text before same-key inbound debounce serialization, or otherwise routes `/stop`, `/new`, and `/status` to an immediate control path while preserving Feishu auth, mention handling, dedupe, and normal-message serialization. Add regression coverage for commands sent during an active same-chat run.
