Keep this issue open. Current main has improved stale-index behavior, but the provider-backed hybrid FTS path still filters keyword rows by the active embedding model, the exact hybrid keyword regression coverage remains skipped, and open PR #48328 is still the implementation candidate linked to this issue.

Keep this issue open while #48328, or a newer current-path memory-search PR, is reviewed. The fix should explicitly decide whether FTS keyword rows should ignore embedding provider/model while vector search remains model-filtered, then add active regression coverage for hybrid exact-keyword recall across provider/model changes and retain the current stale-index sync improvements.
