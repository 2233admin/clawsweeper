## Summary

Keep PR #70058 open. Current main still constructs the skills watcher with native chokidar events only, has no OPENCLAW_SKILLS_WATCH_POLLING or interval handling, and docs/tests do not cover an opt-in polling fallback. The PR remains a concrete core bug-fix candidate; it also appears to have addressed the bot review comments about undefined and sub-1 polling intervals, but it should be narrowed before merge because it carries an unrelated QA scenario change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrowed version of the core skills watcher polling fallback in src/agents/skills/refresh.ts with the associated tests and docs, while removing or separating the unrelated QA scenario change. This belongs in OpenClaw core because skills snapshot invalidation is core runtime behavior, not a ClawHub/plugin concern.
