## Summary

Keep this PR open. Current main implements the basic Feishu tool precedence change, but it still does not implement the PR's final review-driven guard that only trusts inherited account context for Feishu-origin sessions. The remaining work is scoped to the Feishu plugin and protects against cross-channel account-id collisions selecting the wrong Feishu tenant.

## Best Possible Solution

Keep this PR open for maintainer review or a narrow replacement patch. Rebase it onto current main, preserve explicit accountId > enabled Feishu-origin contextual account > configured defaultAccount, thread messageChannel through Feishu tool account resolution, and add targeted tests for Feishu-origin contextual routing, non-Feishu fallback, explicit override, unknown-account fallback, and normalized account ids.
