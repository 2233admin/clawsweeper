## Summary

Keep open. Current main still does not implement the PR's central capability: prompt-build hooks only receive prompt/messages and can only return prompt/system-prompt mutations, while runtime tool narrowing remains limited to existing run params. The PR is useful and has real supporting discussion, including the MCP-heavy deployment comment, but it is still open, unmerged, merge-dirty, and has unresolved review findings around preserving hook prompt mutations/provider transforms and including client-hosted tools in the hook-visible tool catalog.

## What This Changes

This PR extends `before_prompt_build` hooks to receive available tool and skill names, return `toolsAllow`/`skillsAllow`, and apply those allowlists during embedded runner prompt/session setup with focused tests.

## Best Possible Solution

Land a revised version that exposes a stable narrowing-only prompt-build hook contract, composes skill filtering with existing system-prompt mutations and provider transforms, defines or includes client-hosted tools in the available-tool catalog, updates public plugin docs/SDK tests, and validates the embedded runner path after rebasing onto current main.
