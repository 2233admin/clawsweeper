## Summary

Keep open. Current main only contains the narrower zero-valued poll-default fix, while the broader poll-noise and Discord empty-modal problems remain partly unresolved; this PR has useful source work, but its component cleanup is placed on an older core `params.components` path and would miss the current Discord parser boundary.

## What This Changes

The PR updates message send handling to reject explicit `pollQuestion` on `send`, strip non-question poll fields and empty component skeletons from send params, and adds poll/send regression coverage.

## Best Possible Solution

Use this PR as source material, but adapt the implementation to current main: handle poll-noise sanitization in the shared send path while preserving a clear error for explicit poll intent, and handle empty Discord modal/component skeletons in the Discord plugin parser/runtime. Add focused regression tests for #42820 and #43015 before closing the linked issues.
