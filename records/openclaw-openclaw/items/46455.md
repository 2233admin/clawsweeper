## Summary

Keep this PR open. Current main only shipped the narrower zero-default poll fix; it still rejects non-question/non-zero poll noise on `action: "send"`, and Discord still has an empty-modal error path if a component payload reaches the plugin parser. The PR remains a useful implementation candidate for #42820 and #43015, but it needs current-main adaptation around the Discord plugin boundary.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase/adapt it to current main: in core, ignore or strip model-populated non-question poll noise for `action="send"` while preserving the corrective error for explicit `pollQuestion`; in the Discord plugin, ignore empty `components.modal` skeletons at the parser boundary without dropping valid array or container-only component payloads. Add focused regression tests for #42820 and #43015 before closing those linked issues.
