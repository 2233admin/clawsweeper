## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: `message.send` still rejects non-question/non-zero poll noise, and Discord still errors on non-empty empty-modal component skeletons. The PR remains a useful implementation candidate, but it needs current-main adaptation because Discord component handling now lives in the Discord plugin boundary.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. Rebase/adapt it to current main: in core, ignore or strip model-populated non-question poll noise for `action="send"` while preserving a corrective error for explicit `pollQuestion`; in the Discord plugin, ignore empty `components.modal` skeletons without dropping valid array or container-only component payloads. Add focused regression tests for the linked poll and Discord component failures, then close #42820 and #43015 after a fix lands.
