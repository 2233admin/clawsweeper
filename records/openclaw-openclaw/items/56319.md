## Summary

Keep this PR open. Current main does not implement the PR’s central behavior: Telegram exec approval prompts are resolved by clearing inline buttons, but the approval message is not deleted after approval or denial. The PR is a plausible implementation candidate for the linked bug, but it is stale against the current native approval runtime and needs a focused port plus review cleanup before it can be merged or closed.

## Best Possible Solution

Port a focused version of the Telegram deletion behavior into `extensions/telegram/src/approval-handler.runtime.ts`: clear inline buttons immediately, delete the resolved approval message after the intended short delay for both approved and denied resolutions, use `deleteMessageTelegram`, handle timer lifecycle safely, and add targeted tests for the native approval finalizer/callback path. Keep the linked bug open until that focused fix lands, and remove or split unrelated docs changes from this PR.
