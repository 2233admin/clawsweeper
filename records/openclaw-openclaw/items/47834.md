## Summary

Keep open. Current main still runs `before_tool_call` with the incoming browser tool params only; it does not enrich `browser` `act`/`type` calls with the current tab URL before execution. The browser act route can return a policy-checked URL after execution, which explains the credential-vault cache workaround, but that does not solve the cold-cache pre-injection domain check requested here. Related PR #30323 did not land as a safe shipped fix and had unresolved maintainer/security concerns.

## Best Possible Solution

Keep this open for a maintainer-reviewed, narrow browser/plugin API fix: expose a policy-checked current tab URL to `before_tool_call` for browser `act`/`type` decisions, or provide a supported hook-time browser URL query seam. The implementation should stay in the browser plugin/tool-hook boundary, avoid unguarded Playwright/CDP URL reads, preserve existing param rewriting/blocking semantics, and include focused tests for flattened and `request` act params, stale/missing targetIds, node/proxy behavior, and fail-closed credential-domain use cases.
