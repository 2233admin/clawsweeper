## Summary

Keep open. Current main still does not provide a trustworthy current page URL to `before_tool_call` hooks before browser `act`/`type` executes; the hook runner forwards only incoming params, while the browser route resolves policy-filtered tab URLs only inside the route response path. The reporter's credential-vault workaround is useful and fail-secure, but it confirms the native hook-time contract gap remains.

## What This Changes

Request to expose the browser's current tab URL to `before_tool_call` hooks for browser `act`/`type` actions, or provide an equivalent query seam, so credential plugins can enforce domain pins before injecting text.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests covering managed browser, existing-session, node-proxy, blocked/private URL, and cold-target behavior.
