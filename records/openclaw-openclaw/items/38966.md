## Summary

Current main still does not refresh the TUI after an external session reset. The gateway now emits subscriber-gated `sessions.changed` reset events, but the TUI neither subscribes to session-change events nor dispatches/handles `sessions.changed`. The linked implementation PR #40472 is open and explicitly closes this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or close #40472. The fix should make the TUI subscribe to `sessions.changed`, dispatch session-change events, ignore unrelated sessions, refresh history/session metadata on current-session `new` or `reset`, clear stale active run/stream/local-run state, and add focused TUI/gateway regression coverage.
