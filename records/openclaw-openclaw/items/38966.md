## Summary

Current main still does not complete the reported external reset refresh path. The gateway emits subscriber-gated `sessions.changed` events after `sessions.reset`, but the TUI gateway client does not subscribe to session-change events and the TUI dispatcher/event-handler surface still ignores `sessions.changed`. The issue is paired with open implementation PR #40472, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the TUI to detect session resets initiated by another client, refresh or clear history and token/session state, and clear stale run state for the current session.

## Best Possible Solution

Keep this issue open and review, land, or close #40472. The implementation should make the TUI subscribe to `sessions.changed`, dispatch and handle current-session `new`/`reset` events, refresh history/session metadata, clear stale active run/stream/local-run state, ignore unrelated sessions, and add focused regression coverage.
