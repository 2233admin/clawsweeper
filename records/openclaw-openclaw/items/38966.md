## Summary

Keep #38966 open. Current main partially addresses the gateway side by emitting subscriber-gated `sessions.changed` events after `sessions.reset`, but `openclaw tui` still neither subscribes to those session events nor dispatches/handles them. The linked fix PR #40472 is still the relevant open work in the provided GitHub context, so the user-visible external reset refresh bug is not shipped on main.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should subscribe the TUI to session change events, handle `sessions.changed` for the current session when the reason represents reset/new session lifecycle, reload history/session metadata, clear stale run/stream/local-run state, ignore unrelated sessions, and add focused TUI/gateway regression coverage.
