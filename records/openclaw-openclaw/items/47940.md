## Summary

Keep open. Current main has append-only heartbeat transcripts and filters only HEARTBEAT_OK no-op pairs from replay context. It still runs heartbeat in the main session by default, keeps meaningful sent heartbeat/tool turns in history, and exposes isolatedSession only as an opt-in workaround rather than making each configured heartbeat cycle independently evaluate as requested.

## Best Possible Solution

Keep this as an OpenClaw core heartbeat bug. The best fix is to make scheduled heartbeat cycles independent in the default path while preserving append-only transcripts, delivery routing, and HEARTBEAT.md/workspace injection. A likely implementation is heartbeat-run replay filtering that excludes prior heartbeat-generated sent/tool turns from subsequent heartbeat prompt context, or a deliberate default isolated-heartbeat context if maintainers accept that product change. Add regression coverage for sent/tool heartbeat followed by another scheduled heartbeat proving the prior sent work cannot cause a false ok-token skip.
