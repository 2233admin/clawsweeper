## Summary

Keep open. Current main still has the problematic normal agent path: agent tool registration passes a workspaceDir into createImageTool, and the image tool can still resolve a media://inbound reference as a workspace-relative path before loadWebMedia can hydrate it. The linked implementation PR remains open and unmerged, and #74123 corroborates the same malformed workspace media:/inbound failure shape.

## What This Changes

Fix the agent image-tool pipeline so inbound media:// image references from Telegram sessions are hydrated and passed to the configured image model instead of being treated as missing image data.

## Best Possible Solution

Land PR #63497 or an equivalent narrow patch that preserves canonical media://inbound references through image-tool workspace normalization, lets the shared media loader resolve them with the existing inbound media guards, and adds regression coverage for createOpenClawTools/createImageTool with workspaceDir.

## Reproduction Assessment

Yes. The source-level reproduction is a normal agent image tool created with workspaceDir and invoked with image media://inbound/<id>; current code can route that reference through path.resolve(workspaceDir, normalizedRef), matching the malformed workspace media:/inbound path reported in #74123.

## Solution Assessment

Yes. The narrowest maintainable fix is to keep media://inbound refs intact until loadWebMedia/resolveInboundMediaReference handles them, with regression coverage for the workspaceDir path; broad Telegram, provider, or model changes are not needed.
