## Summary

Keep open. Current main still has the reported agent-session edge: `createOpenClawTools` passes `workspaceDir` into the image tool, and `createImageTool` can still treat `media://inbound/<id>` as a relative workspace path before `loadWebMedia` can resolve the managed inbound media URI. The linked PR #63497 is still open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #63497 or land an equivalent narrow fix. The fix should preserve or resolve `media://inbound` refs before workspace-relative path rewriting in `src/agents/tools/image-tool.ts`, add regression coverage for the `createImageTool` or `createOpenClawTools` path with `workspaceDir` and `fsPolicy: { workspaceOnly: true }`, and keep the shared inbound-media validation guards intact.
