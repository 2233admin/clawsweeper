## Summary

Keep this issue open. Current main has meaningful cron catch-up, concurrency, and session-store OOM mitigations, but it still does not implement the reported Gateway heap-headroom behavior: no `gateway.nodeMaxOldSpaceMb`-style config, no generated service `NODE_OPTIONS`, and no `--max-old-space-size` insertion in the node/systemd Gateway command path. The related open #42853 remains the closest implementation candidate, while #69404 is an adjacent session-store OOM fix rather than a complete fix for this issue.

## Best Possible Solution

Keep this issue open as the canonical tracker for Gateway heap headroom under cron pressure. The best maintainer path is to review and land #42853 or an equivalent narrow fix that adds a typed `gateway.nodeMaxOldSpaceMb`/auto-detected heap contract, threads it through service install planning, safely injects only `--max-old-space-size` into generated service environments or node args, and preserves the existing cron catch-up/concurrency/session-maintenance safeguards. Broader session-store payload work should remain coordinated with #55334.
