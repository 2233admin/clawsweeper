## Summary

Keep open. Current main has useful adjacent mitigations for pricing opt-out, Telegram HTTP/1 transport hardening, polling recovery, and diagnostics, but the reported chronic cross-subsystem Gateway degradation is still not proven fixed. Pricing still starts from the initial runtime-services path when enabled, the remote pricing timeout remains 60s, and the broader slow RPC / Telegram runtime symptoms are not fully covered by the narrower open pricing-startup PR.

## What This Changes

Investigate and fix chronic Gateway runtime degradation where pricing fetches, Telegram polling/sends, and normal Gateway RPCs stall in long-running Windows Node processes despite fast standalone network probes.

Keep this as the canonical broad Gateway runtime degradation tracker.
