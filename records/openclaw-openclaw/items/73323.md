## Summary

Keep this issue open. Current main has adjacent mitigations and diagnostics, but it still does not contain a source-backed fix for the broader Windows gateway runtime degradation covering pricing fetch 60s timeouts, Telegram polling/send stalls, and slow normal gateway RPCs.

## Best Possible Solution

Keep this open for gateway/runtime maintainer triage. Review or land #73486 only as a narrower pricing-startup mitigation, then continue the broader investigation with long-running Windows gateway instrumentation for event-loop delay, handle/socket counts, undici dispatcher state, Telegram polling/send timing, pricing refresh timing, and normal RPC latency during the recurring stall cycle.
