## Summary

Keep this PR open. Current main still renders and patches channel config forms at the channel root, so it has not already implemented the PR's intended account-scoped form resolution. The PR is also a plausible fix candidate for the still-relevant UI/account-path problem, but the review discussion points to real correctness blockers that should be addressed before merge.

## Best Possible Solution

Keep this PR open as a candidate, but require a corrected implementation before merge: pass a preferred/default account id into the config section, resolve account-scoped values when present, fall back both schema and value handling to the root channel config for root-backed channels, ensure patch paths match the object being displayed, and add regression tests for account-scoped and root-fallback cases.
