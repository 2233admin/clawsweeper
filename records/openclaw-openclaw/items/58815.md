## Summary

Keep PR #58815 open. Current main still has the Android GatewaySession job lifecycle shape the PR is trying to fix, and the request is a plausible app reconnect bug, but the submitted patch is not closeable as obsolete or merge-ready because review already identified a real race in the proposed implementation.

## Best Possible Solution

Keep the PR open or replace it with a maintainer fix. The best implementation belongs in `GatewaySession`: make job ownership atomic or synchronized, snapshot and clear the job synchronously in `disconnect()` before asynchronous cancellation, ensure `connect()` cannot mistake a stale inactive job for a live runLoop, and add a regression test for immediate disconnect -> connect using the suite's contextual timeout helper. Merge only after the Android GatewaySession test lane validates the behavior.
