## Summary

Keep this PR open. Current main still does not implement the Mattermost message-tool thread inheritance this PR targets, the paired bug report #45082 remains open, and this abandoned PR is still a plausible implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open for maintainer triage or contributor pickup. The best path is to rebase the Mattermost-owned fix onto current main, add a Mattermost `threading.buildToolContext` that supplies normalized current channel id, active thread root, account-scoped `replyToMode`, and `hasRepliedRef`, add `resolveAutoThreadId` if mirror/session routing needs the implicit thread before send, preserve explicit reply precedence, and add focused regression tests for all/first/off, explicit reply targets, same/different-channel guards, and first-reply state.
