## Summary

Keep this PR open. Current main still does not apply `messages.responsePrefix` in the isolated cron direct delivery path: cron builds `payloadsForDelivery` from `deliveryPayloads` or `synthesizedText`, strips silent reply tokens, and passes those payloads straight to `deliverOutboundPayloads`. Normal reply routing does resolve and apply the prefix, so the PR is still addressing a real consistency gap. Security pass: the provided PR only touches a cron runtime file and a colocated Vitest test file, with no CI, dependency, lockfile, release, package metadata, secret handling, or downloaded-code execution changes visible in the diff.

## Best Possible Solution

Review and land this PR, or apply an equivalent fix on main that resolves the effective `messages.responsePrefix` for the cron delivery target, prefixes text cron announce payloads idempotently, leaves media-only payloads unchanged, and adds focused regression coverage for global, channel/account, empty, already-prefixed, short-prefix, and trailing-whitespace cases.
