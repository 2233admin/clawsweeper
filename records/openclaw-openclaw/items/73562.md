## Summary

Keep this open. Current main and the latest release still use the direct `readline.question()` prompt path for non-forced plugin uninstall, and the explicitly linked fixing PR is open and unmerged, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

`openclaw plugins uninstall <id>` should fail cleanly with exit 1 and actionable guidance when confirmation stdin closes before an answer, while preserving piped answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, has plugin uninstall convert that specific condition into exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the stale-entry cleanup in #73551 separate unless a maintainer intentionally broadens the work.

## Reproduction Assessment

Yes. Current main still reaches `promptYesNo()` before plugin uninstall mutations, and a read-only `node:readline/promises` closed-stdin smoke reproduces exit 13 while piped input resolves normally.

## Solution Assessment

Yes at the design level. Handling EOF in the shared prompt primitive and translating that specific error in `plugins uninstall` is narrower than a blanket non-TTY gate because piped `y`/`n` answers should continue to work.
