## Summary

Keep this open. Current main still has the closed-stdin prompt failure shape, and the linked closing PR is open and unmerged, so the issue should remain open until PR #73566 or an equivalent fix lands.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly when confirmation stdin closes, while preserving piped answers and `--force`.

## Best Possible Solution

Land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, has plugin uninstall convert that condition into exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests.

## Reproduction Assessment

Yes. Current main still awaits the shared prompt in the non-forced uninstall path, and a read-only `node:readline/promises` closed-input smoke reproduced exit 13 while piped input resolved normally.

## Solution Assessment

Yes at the design level. Handling EOF in the shared prompt primitive and catching that specific condition in `plugins uninstall` is narrower than a blanket non-TTY gate because piped `y`/`n` answers should continue to work.
