## Summary

Keep this open. Current main still directly awaits the shared readline confirmation helper in the non-forced plugin uninstall path, the latest release tag has the same behavior, and PR #73566 is open and unmerged with closing syntax for this issue.

## What This Changes

Make `openclaw plugins uninstall <id>` handle closed confirmation stdin cleanly without Node exit 13 while preserving piped answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, has plugin uninstall convert that condition into exit 1 with interactive or `--force` guidance, and adds focused prompt plus uninstall regression tests. Keep the stale-entry cleanup in #73551 separate unless a maintainer intentionally broadens the same PR.

## Reproduction Assessment

Yes. Current main still reaches a direct `readline.question()` await in the non-forced uninstall path, and a read-only Node smoke reproduces exit 13 for closed input while newline-delimited `n` input resolves normally.

## Solution Assessment

Yes at the design level. Handling EOF in the shared prompt primitive and catching that specific condition in `plugins uninstall` is narrower than a blanket non-TTY gate because piped `y`/`n` answers should continue to work.
