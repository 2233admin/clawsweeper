## Summary

Keep open. The memory-search half still addresses a real current-main gap, but the webhooks half is superseded by merged #67941 and the branch is draft/stale against the current synchronous registration design.

## What This Changes

The PR branch forwards active `OpenClawConfig` into memory-search embedding provider lookup, adds regression tests, and splits webhooks registration into synchronous inline-secret handling plus async fallback resolution.

## Best Possible Solution

Rebase and narrow to the memory-search fix: use the cfg-aware memory embedding provider runtime resolver for primary, fallback, and multimodal validation lookup, keep #67941's current webhooks request-time SecretRef design, and add a focused regression proving cfg is forwarded without broad plugin activation.
