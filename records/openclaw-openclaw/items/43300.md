## Summary

Keep this issue open. Current main has tactical trusted-proxy mitigations: local-direct password fallback and explicit `allowLoopback` support for same-host proxy requests with real identity headers. It still does not implement the headerless internal identity path requested by the report, and the provided GitHub context shows an open fork PR, BingqingLyu/openclaw#2181, that uses closing syntax for this issue. Per cleanup policy, that PR should be reviewed, merged, closed, or explicitly superseded before this issue is closed.

## Best Possible Solution

Keep this issue open while maintainers review, merge, close, or explicitly supersede BingqingLyu/openclaw#2181 and decide whether #69066 owns the longer-term internal service-identity work. If maintainers decide the shipped password fallback plus `allowLoopback` docs are the accepted resolution, close only after the open closing PR is resolved or marked superseded.
