## Summary

Current main still appears vulnerable to stale workspace skill paths after an iCloud-to-local workspace migration. The session skill snapshot is reused by version and skill filter only, while the persisted/runtime snapshot shape still lacks workspace or source-root provenance. Related sandbox skill-path reports overlap conceptually, but they do not supersede this workspace-migration bug.

## Best Possible Solution

Keep #39719 open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source-root provenance to SkillSnapshot and SessionSkillSnapshot, compare it against the current resolved workspace before reuse, invalidate snapshots when agents.defaults.workspace or per-agent workspace changes, apply the same behavior across agent command, auto-reply, and cron/embedded consumers, and add a regression test with stale absolute SKILL.md paths from the old iCloud workspace.
