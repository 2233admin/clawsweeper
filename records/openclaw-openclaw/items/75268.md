## Summary

Keep this open. Current main still has the reported OpenAI-compatible stream parsing gap, and the issue is now explicitly paired with open PR #75339, so the terminal outcome should be to review and land or replace that PR before closing this issue.

## What This Changes

Prevent Mistral thinking/native-reasoning structured chunks from appearing and persisting as repeated `[object Object]` text in replies, session transcripts, and memory.

## Best Possible Solution

Normalize structured content at the OpenAI-compatible stream boundary so `thinking` chunks become hidden thinking blocks, `text` chunks become visible assistant text, and unknown objects are never coerced into `[object Object]`; then close this issue only after PR #75339 or an equivalent fix lands.

## Reproduction Assessment

Yes. The issue gives concrete versioned steps, and static current-main inspection gives a focused mocked reproduction: feed a Mistral-style `delta.content` array with typed thinking/text blocks into the completions stream parser and assert no `[object Object]` enters output text.

## Solution Assessment

Yes. The best fix layer is the shared OpenAI-compatible stream parser, not memory cleanup alone, because the corruption starts in live assistant text before persistence and memory ingestion.
