## Summary

Keep open. The requested feature is still absent from current main, and this PR is not safe to merge as-is because it accepts per-agent `compaction` and `contextPruning` settings without applying the effective values in the normal Pi compaction/pruning runtime paths. The already-linked #69988 overlaps on compaction but does not cover `contextPruning`, so this needs maintainer consolidation rather than cleanup closure.

## What This Changes

The PR adds per-agent `compaction` and `contextPruning` config type/schema fields, extracts shared Zod schema definitions, and returns those fields from agent config resolution.

## Best Possible Solution

Land one canonical implementation that accepts both per-agent keys, merges each agent override over `agents.defaults`, passes the effective settings into Pi settings and embedded extension factories, updates config docs/generated surfaces, and leaves #52732 open until that implementation merges.
