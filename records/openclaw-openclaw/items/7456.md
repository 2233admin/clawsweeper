## Summary

Keep this issue open. Current main still exposes the classic `openclaw onboard` wizard as a forward/cancel flow: prompt types have no Back sentinel or `allowBack`, the Clack adapter only handles cancellation, `runSetupWizard` remains sequential, `WizardSession` has no history/previous API, and the Gateway wizard protocol only publishes `wizard.start`, `wizard.next`, `wizard.cancel`, and `wizard.status`. The related Back-navigation PR was closed unmerged, and the resume/dialog related items do not ship the requested in-session back navigation with preserved answers.

## Best Possible Solution

Keep this issue open. The best path is an additive typed Back/previous contract through `WizardPrompter`, Clack prompts, `WizardSession`, Gateway wizard schema/handlers, generated clients, and UI clients, then refactor the setup flow into a state-machine/history model that preserves answers when moving backward and forward. PR #7762 is useful prior art but should be revisited rather than treated as shipped.
