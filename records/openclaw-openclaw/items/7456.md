## Summary

Keep this issue open. Current main still does not provide in-session Back navigation for the classic `openclaw onboard` wizard: the prompt contract has no back sentinel, the Clack adapter only handles cancellation, `WizardSession` only supports next/answer/cancel, the Gateway protocol only exposes start/next/cancel/status, and the setup flow remains a sequential prompt flow. The closed PR #7762 is useful prior art but was not merged.

## Best Possible Solution

Keep this issue open. The best implementation is an additive typed Back/previous contract through the wizard prompt layer, Clack adapter, `WizardSession`, Gateway wizard protocol/handlers, generated clients, and UI clients, plus a setup-flow state/history model that preserves answers when moving backward and forward. PR #7762 should be used as prior art, not treated as shipped.
