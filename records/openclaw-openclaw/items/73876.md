## Summary

Keep open. Current main still has the reported Moonshot naming mismatch, and the currently open linked PR #74946 is the right place to review the active fix before this issue closes.

## What This Changes

The issue asks OpenClaw to accept OpenRouter-style `moonshotai/<model>` direct Moonshot refs without breaking existing `moonshot/<model>` configs.

## Best Possible Solution

Land #74946 or an equivalent tested compatibility fix that accepts `moonshotai/<model>` as direct Moonshot while keeping `moonshot` canonical for plugin id, auth profiles, config keys, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `moonshotai/kimi-k2.6` is split at the first slash, `normalizeProviderId` leaves `moonshotai` unchanged, and the Moonshot catalog/auth surfaces are registered under `moonshot`.

## Solution Assessment

Yes. A compatibility alias is safer than renaming the provider because it preserves existing `moonshot/...` configs while resolving the copied OpenRouter slug; the issue should stay open until the linked fix PR is merged or replaced.
