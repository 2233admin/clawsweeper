## Summary

Keep this issue open. Current main has Gateway subscription primitives and shared DM session routing, but the bundled TUI still only dispatches `chat`, `chat.side_result`, and `agent` events, has no TUI backend subscription methods for `sessions.subscribe` / `sessions.messages.subscribe`, and filters visible chat/Btw events to the current session. The related open #40678 tracks the same broader opt-in visibility idea, closed #51691 was routed back to this broader tracker, and open maintainer PR #73402 is narrower active-session live transcript work rather than the full TUI/Telegram/iMessage unified inbox request.

## Best Possible Solution

Keep this open as the broader TUI/Gateway product tracker unless maintainers manually consolidate it with #40678. The best implementation path is an explicit opt-in TUI observer or mirror mode built on the existing Gateway `sessions.subscribe` and/or `sessions.messages.subscribe` primitives, with channel/session labels and clear privacy boundaries for Telegram, iMessage, and other channels. Review #73402 separately as a narrower active-session live transcript fix.
