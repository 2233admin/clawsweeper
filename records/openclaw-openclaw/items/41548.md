## Summary

Keep this open. Current main has shared-session and route-docking pieces, plus Gateway session event primitives, but it still does not implement the requested unified live inbox/message sync across TUI, Telegram, and iMessage. The TUI only renders active chat/agent event families for its current session and does not subscribe to or render session.message/sessions.changed updates as cross-channel activity. Related #40678 and #51691 remain open coordination points, not merged replacements.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer deliberately consolidates it into #40678 or #51691. The likely implementation is an opt-in TUI unified inbox or observer mode that subscribes to Gateway session.message/sessions.changed events for a selected agent/session scope, renders source channel/session labels, and keeps existing delivery-route privacy rules intact.

## Risks / Open Questions

- A naive unified inbox could leak private channel context across local TUI, Telegram, and iMessage surfaces. The feature should be opt-in, source-labeled, and preserve delivery-route and DM-isolation safeguards.
- There is real overlap with #40678 and #51691. Maintainers should choose whether this issue remains the umbrella or is explicitly consolidated into one canonical tracker, but there is not enough maintainer signal here to auto-close it as duplicate.
