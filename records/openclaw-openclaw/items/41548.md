## Summary

Current main still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. The Gateway and Control UI have session subscription primitives, but the TUI still only handles chat, chat.side_result, and agent events for the active session; it does not subscribe to or render session.message/sessions.changed updates across channel sessions. Related #40678 and #51691 remain open coordination points, not merged replacements.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer deliberately consolidates it into #40678 or #51691. The likely implementation is an opt-in TUI unified inbox or observer mode that subscribes to gateway session.message/sessions.changed events for a selected agent/session scope, renders source channel/session labels, and preserves delivery-route privacy rules.

## Risks / Open Questions
