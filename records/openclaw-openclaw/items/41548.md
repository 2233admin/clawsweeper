## Summary

Current main still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. OpenClaw has shared gateway session storage and subscription APIs, but the TUI still listens only to its active chat stream, filters events by the current session, and does not subscribe to or render gateway session.message/sessions.changed updates as cross-channel activity. Related #40678 and #51691 remain open coordination points rather than shipped replacements.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue. The likely implementation is an explicit opt-in unified inbox or observer mode that subscribes to gateway session.message/sessions.changed events for the selected agent/session scope, renders source channel/session labels, and preserves delivery-route privacy rules. Coordinate the TUI observer scope with #40678 and the broader real-time surface handoff with #51691.
