## Summary

Keep this issue open. Current main has shared session routing and Gateway/WebChat session event primitives, but the TUI still does not implement the requested unified live inbox across TUI, Telegram, and iMessage. It only renders the active session’s `chat`/`chat.side_result`/`agent` stream and does not subscribe to or handle `session.message`/`sessions.changed` as an inbox feed. Related #40678 and #51691 are still open, and #40678 is a same-author overlapping issue, so this should not be auto-closed without maintainer consolidation.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue unless a maintainer manually consolidates it into #40678 or #51691. The likely fix is an opt-in TUI observer or unified inbox mode that subscribes to Gateway `sessions.subscribe` and/or `sessions.messages.subscribe`, renders `session.message` and `sessions.changed` activity with channel/session labels, and keeps the existing active-session TUI behavior as the default.
