## Summary

Keep this PR open. Current main still does not implement Discord account-level agentId routing: routing only reads persisted bindings, the Discord account type/schema do not accept account-level agentId, docs still show manual Discord bindings, and the status snapshot type does not expose an account binding. The PR remains a plausible implementation candidate, but it needs rebase and maintainer review for the remaining routing/status/security-sensitive edge cases noted in review comments.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, add Discord account-level agentId to the accepted config contract, synthesize effective route bindings in the routing layer without writing them back to config, preserve explicit binding precedence, filter disabled/tokenless accounts or define that behavior explicitly, propagate the effective binding through the real channel-status surfaces, and update the multi-agent/Discord docs plus focused routing/status tests.
