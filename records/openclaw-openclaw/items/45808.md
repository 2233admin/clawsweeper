## Summary

Current main still does not implement this PR’s intended Discord account-level agent routing. The PR remains a real implementation candidate: the routing entry point still returns only persisted bindings, the Discord config schema/type still lacks `accounts.<id>.agentId`, docs still instruct users to write manual `bindings`, and no synthesizer exists on main. The discussion is useful: early review found the core direction reasonable and the author addressed several comments, but later review concerns around status visibility and tokenless-account synthesis still need maintainer review before merge.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to add Discord account-level `agentId` to the accepted config contract, synthesize effective route bindings in the routing layer without writing them back to config, preserve explicit `bindings` precedence, skip unsafe disabled/tokenless accounts if outbound binding would be affected, expose the effective binding through the real channel-status surfaces if that remains part of the goal, and land with focused routing/status tests plus docs updates.
