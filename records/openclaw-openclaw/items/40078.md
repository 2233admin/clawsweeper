## Summary

Keep #40078 open. Current main still lacks a Discord guild/channel `preflightTranscribe` option or equivalent silent history-ingestion mode. Discord now has audio preflight for DMs and spoken mention detection, but unmentioned `requireMention` guild audio still records the original `<media:audio>` history entry and returns before agent dispatch.

## Best Possible Solution

Keep #40078 open. Add an explicit Discord guild/channel option such as `preflightTranscribe` across config types, strict schema, generated metadata, docs, and focused tests. In Discord preflight, after access checks and before the no-mention return, transcribe eligible unmentioned audio when enabled, store transcript text in pending guild history instead of `<media:audio>`, suppress agent dispatch, and prove a later @mention receives that transcript in `InboundHistory`.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications, so any implementation should remain explicit opt-in and run only after guild/channel/member access checks.
- The implementation should carry preflight state forward so the same attachment is not transcribed a second time if it later enters the normal reply pipeline.
