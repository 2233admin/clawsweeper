## Summary

Keep this issue open. Current main still does not provide a Discord guild/channel `preflightTranscribe` option or equivalent silent history-ingestion mode. Discord can preflight-transcribe audio for DM/mention detection, but unmentioned `requireMention` guild audio that does not mention the bot still records the original `<media:audio>` history entry and returns before dispatch.

## Best Possible Solution

Implement an explicit Discord guild/channel option such as `preflightTranscribe` in the bundled Discord plugin. Add the field to config types, strict schema, generated metadata/help, and Discord docs. In Discord preflight, after authorization and before the no-mention return, transcribe eligible unmentioned audio when enabled, store the transcript text in pending guild history instead of `<media:audio>`, suppress agent dispatch, and add focused tests proving a later mention receives the transcript in history context.
