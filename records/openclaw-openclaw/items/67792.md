## Summary

Keep open. Current main still builds the Telegram model button fallback label from the provider-local model id, and the paired fix PR #72752 is open and unmerged with closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for Telegram model-selection buttons to show the full `openrouter/provider/model` ref for nested OpenRouter model ids instead of dropping the `openrouter/` prefix.

## Best Possible Solution

Review and land #72752, or an equivalent narrow Telegram plugin fix, so nested provider-local ids fall back to canonical `provider/model` labels while preserving display-name overrides, callback data, selection resolution, pagination, truncation, and current-model marking.
