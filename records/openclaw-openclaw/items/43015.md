## Summary

Keep open. Current main still leaves the central poll/schema coupling in place: the shared message tool schema exposes poll creation fields, and the send path still rejects when non-empty, true, or non-zero poll-looking parameters are present. The Discord components exposure is partially mitigated, but this issue also has open PR #46455 with closing syntax, so the right maintainer path is to review, land, replace, or explicitly close that implementation candidate rather than close this issue now.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise from ordinary `action: "send"` calls, preserves explicit poll/component validation, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
