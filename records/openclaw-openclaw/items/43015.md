## Summary

Keep open. Current main still partially matches the reported failure: the generic message tool schema still includes shared poll creation fields, and `action: "send"` still rejects when non-empty/true/non-zero poll-looking params are present. The Discord `components` schema exposure appears partially addressed on main, but the poll/schema coupling remains and the provided GitHub context shows open PR #46455 as an unmerged implementation candidate for this issue.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise from ordinary `action: "send"` calls, preserves strict validation for explicit poll/component paths, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
