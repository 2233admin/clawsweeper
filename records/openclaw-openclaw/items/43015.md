## Summary

Keep open. Current main partially addresses the report by hiding Discord `components` from the generic message tool schema and ignoring completely empty Discord component objects in the lower runtime, but the central poll/schema coupling remains: the shared `message` tool schema still exposes poll creation fields, and `action: "send"` still throws when non-empty/true/non-zero poll-looking params are present. The provided GitHub context also lists open PR #46455 with closing syntax for this issue, so this issue should remain open while that implementation candidate is reviewed, landed, replaced, or explicitly closed.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise from ordinary `action: "send"` calls, preserves strict validation for explicit poll/component actions, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
