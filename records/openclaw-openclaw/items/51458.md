## Summary

Current main still has the reported Slack outbound message-action failure: `send` turns `interactive` or `presentation` into Slack blocks, then rejects the action when `media` is also present. The Slack reply payload path already has a media-first/final-blocks delivery shape, but message actions do not use it, and the related open PR is inbound file hydration work rather than an outbound fix.

## Best Possible Solution

Keep this issue open and fix it in the Slack-owned outbound message-action path. The best implementation should route `send` with `media` plus `interactive` or `presentation` through the same media-first/final-blocks behavior used by Slack reply payload delivery, or an equivalent compound action, with focused regression coverage for `media + interactive`, `media + presentation`, and partial-failure/retry semantics.
