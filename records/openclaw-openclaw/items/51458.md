## Summary

Keep open. Current main still rejects the reported Slack message-action shape before delivery, and the related open Slack file PR is inbound media hydration work rather than an outbound send fix.

## What This Changes

The issue requests that Slack message `send` support file attachments together with interactive or presentation controls instead of throwing when those controls render to Block Kit blocks.

## Best Possible Solution

Implement the fix in the Slack-owned message action path by routing `send` with `media` plus `interactive` or `presentation` through the existing media-first/final-blocks behavior, or an equivalent helper, while keeping the operation represented as one logical message action with focused regression coverage.

## Reproduction Assessment

Yes. A deterministic code reproduction is enough: call Slack message `send` with `to`, `media`, and `interactive` or `presentation`; `handleSlackMessageAction` renders blocks and throws before invoking Slack delivery.

## Solution Assessment

Unclear as currently proposed. The two-pass direction matches existing Slack outbound payload delivery, but the maintainable fix should not merely remove the guard because the lower runtime also rejects the same shape and partial-failure retry semantics need to remain coherent.
