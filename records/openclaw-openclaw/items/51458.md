## Summary

Current main still has the reported Slack outbound message-action failure. Slack `send` converts `interactive` or `presentation` into Block Kit blocks and rejects the action when `media` is also present; the lower Slack action runtime and concrete send implementation still reject `blocks + mediaUrl`. The media-first/final-blocks behavior exists in the Slack reply payload path, but message actions do not use it. The related open PR #62792 is inbound Slack file hydration work and explicitly says it makes no outbound Slack behavior changes, so it is not a canonical fix for this issue.

## Best Possible Solution

Keep this issue open. Implement the fix in the Slack-owned message action path by routing `send` with `media` plus `interactive` or `presentation` through the same media-first/final-blocks behavior used by Slack reply payload delivery, or an equivalent compound Slack action that preserves account, thread, media access, ordering, idempotency, and transcript intent. Add focused regression tests for message `send` with `media + interactive` and `media + presentation`, plus partial-failure/retry coverage for the two-call delivery shape.
