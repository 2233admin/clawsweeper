## Summary

Keep this PR open for normal maintainer review. Current main still passes the raw Active Memory `timeoutMs` into the embedded runner even though the surrounding watchdog and hook timeout already include setup grace, so the branch is addressing a real remaining inconsistency rather than work that is obsolete on main.

## What This Changes

The PR adds a shared Active Memory embedded-recall timeout helper and updates the implementation and regression tests so the embedded runner, outer watchdog, and prompt hook all use `timeoutMs + setupGraceTimeoutMs`.
