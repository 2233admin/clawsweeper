## Summary

Keep #38302 open. Current main still does not implement per-account Slack native command prefixes or account-level native command name mappings. Slack account config accepts only the existing slashCommand fields, the strict schema and generated metadata reject additional native naming keys, the native command registry resolves names only by provider, and Slack registration/dispatch does not pass any account-specific native name data. The issue also has a second user confirming the same multi-app Slack collision, while the two related implementation PRs #38326 and #38391 are closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled Slack plugin plus the native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, either nativePrefix-derived names or an explicit nativeNames map; thread resolved account naming through native spec generation, Slack registration, argument-menu payloads, and findCommandByNativeName dispatch; preserve current unprefixed behavior and the /status to /agentstatus override; update Slack setup docs; and add focused multi-account tests proving prefixed native commands route to the intended built-in commands.
