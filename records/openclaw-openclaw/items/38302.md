## Summary

Current main still lacks the requested per-account Slack native command prefix or native-name mapping. Slack slash-command config exposes only enabled/name/sessionPrefix/ephemeral, the strict schema and generated metadata would reject the proposed account-level field, and Slack native registration/dispatch still resolve native names only by provider "slack". The linked implementation attempts #38326 and #38391 are closed and unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep this open and implement the feature in the bundled Slack plugin plus the shared native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, thread it through native spec generation, Slack registration, argument-menu values, and native dispatch lookup, preserve current unprefixed behavior and the /status -> /agentstatus provider override, update Slack/slash-command docs and generated config metadata, and add focused multi-account tests proving prefixed or mapped native commands route to the intended built-ins.
