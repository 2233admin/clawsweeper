## Summary

Keep open. Current main still lacks the requested per-account Slack native slash-command prefix or mapping. Slack slash-command config only exposes enabled/name/sessionPrefix/ephemeral, the strict schema and generated metadata would reject `nativePrefix` or `nativeNames`, and Slack native registration/dispatch still resolves built-in native names by provider only. The linked implementation attempts #38326 and #38391 are closed and unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep this open and implement a focused Slack/shared-native-command fix: add a validated account-level native naming option, thread it through native spec generation, Slack command registration, native dispatch lookup, and argument-menu values, preserve current defaults and `/agentstatus`, then update generated config metadata plus Slack/slash-command docs and focused multi-account tests.

## Clownfish Work Prompt

Implement per-account Slack native slash-command naming for multi-agent Slack apps. Current main rejects `channels.slack.accounts.<id>.slashCommand.nativePrefix` / `nativeNames` and registers/dispatches Slack native commands by provider-only names, so multiple Slack apps cannot each own `/reset`, `/compact`, `/stop`, etc. Expected boundary: keep the behavior in the bundled Slack plugin plus the shared native-command contract only. Add a validated account-level Slack slash-command native naming option; prefer a flexible mapping if maintainers have already leaned that way, but support the issue's prefix use case. Thread the resolved account native names through native spec generation, Slack command registration, `findCommandByNativeName` dispatch, and Slack argument-menu callback values. Preserve existing unprefixed defaults, `commands.native` behavior, native skills behavior, and the Slack `/status` -> `/agentstatus` override. Update generated config metadata and docs (`docs/channels/slack.md`, `docs/tools/slash-commands.md`, and any config reference docs surfaced by docs:list). Add focused tests for two Slack accounts with distinct native names/prefixes proving registration and dispatch route to the same built-in commands, plus compatibility tests for no mapping and `/agentstatus`. Do not move Slack-specific policy into generic core beyond the typed seam needed to pass account-aware native naming.
