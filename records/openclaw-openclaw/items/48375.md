## Summary

Keep open. Current main still does not implement the custom Tailscale control-server feature: the gateway schema/types, CLI, runtime merge/validation path, schema UI hints, and Tailscale docs only cover the existing `mode` and `resetOnExit` behavior. The linked issue #48367 was closed as tracker consolidation in favor of this open implementation PR, and the PR discussion still has substantive behavior/security review feedback to resolve before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to rebase it onto current main, resolve the remaining review comments, decide whether `gateway.tailscale.controlUrl` is an enforced runtime contract or documentation/validation aid, add the schema/UI hints plus focused validation/runtime/CLI tests, and update the public Tailscale docs if maintainers accept the feature.
