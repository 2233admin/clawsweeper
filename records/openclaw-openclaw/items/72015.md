## Summary

Keep this open. Current main has meaningful mitigations for Active Memory timeouts, timeout partials, circuit breaking, and QMD startup laziness, but the central reliability/defaults request remains unresolved and an open draft implementation PR still targets this issue with closing syntax.

## What This Changes

The issue requests safer Active Memory and QMD defaults for multi-agent gateways by bounding or avoiding blocking per-message recall, reducing inherited-model latency risk, enforcing strict budgets, limiting recall concurrency, and preventing QMD startup bursts.

## Best Possible Solution

Keep the issue open until maintainers land #73667 or a replacement that makes pre-reply Active Memory safe under load, keeps recall budgets strict from the user's perspective, gives operators an explicit fast-model/default-warning path, drops or deduplicates stacked recall work, and keeps QMD startup lazy or explicitly bounded for broad multi-agent deployments.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: enable Active Memory for an eligible chat session without config.model, use a slow inherited session or primary model, and the before_prompt_build hook awaits the embedded recall before the main reply path can proceed.

## Solution Assessment

No. Current main is a partial mitigation, not the best complete solution, because blocking recall and inherited-model defaults remain; the active implementation candidate is #73667, but maintainers should verify whether its narrower deadline and QMD startup changes satisfy the full reliability-defaults request.
