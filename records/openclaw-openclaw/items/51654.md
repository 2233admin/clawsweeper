## Summary

Keep this issue open. Current main still does not provide a session-level env contract for ACP `sessions_spawn`: the tool schema and docs omit `env`, the ACP spawn parameter type cannot carry it, and the ACP session manager still calls `runtime.ensureSession` without forwarding env even though the lower-level ACP runtime type already has an optional env field.

## Best Possible Solution

Add a complete ACP session env contract: expose `env?: Record<string, string>` for `sessions_spawn` when `runtime: "acp"`, validate/redact it, thread it through `SpawnAcpParams` and `AcpInitializeSessionInput`, forward it into `runtime.ensureSession`, document merge/override behavior on https://docs.openclaw.ai/tools/acp-agents, and add focused tests at the tool schema/execution, ACP spawn, manager, and acpx-runtime boundary.

## Risks / Open Questions

- Per-session env can contain credentials, so implementation should validate string keys/values, define merge/override precedence, and avoid persisting or echoing secret values in transcripts, task records, logs, or status output.
- The low-level runtime type already exposes `env`, so a partial fix could look complete while still dropping env at the tool schema, `SpawnAcpParams`, ACP manager, or documentation boundary.
