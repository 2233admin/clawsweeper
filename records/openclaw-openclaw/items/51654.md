## Summary

Current main still does not implement session-level environment variables for ACP `sessions_spawn`. The public tool schema omits `env`, the ACP spawn and manager types cannot carry it, and the manager calls `runtime.ensureSession` without env even though the low-level ACP runtime input already has an optional env seam. The relevant files are unchanged from the prior ClawSweeper keep-open review.

## Best Possible Solution

Add a complete ACP session env contract: expose `env?: Record<string, string>` for `sessions_spawn` when `runtime: "acp"`, validate keys/values, define merge/override behavior against the parent Gateway/acpx environment, redact secret values from all persisted/logged surfaces, thread the map through `SpawnAcpParams`, `initializeAcpSpawnRuntime`, `AcpInitializeSessionInput`, and `runtime.ensureSession`, and document the behavior on `https://docs.openclaw.ai/tools/acp-agents`. Add focused tests for the tool schema/execution, ACP spawn handoff, manager forwarding, and acpx runtime boundary.
