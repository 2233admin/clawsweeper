## Summary

Current main still does not implement the requested first-party `openclaw backup restore <archive>` workflow. The backup CLI registers and documents `create` and `verify` only; command descriptors, generated CLI docs, and parser tests all omit `restore`. The discussion’s keepmyclaw link is a useful community workaround, and open PR #59280 is a plausible implementation path, but neither makes the core CLI feature shipped yet.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` command or maintainers explicitly consolidate it into a canonical native backup/restore tracker. The likely path is to review and either land or close PR #59280, with implementation in the backup command surface, docs at `docs/cli/backup.md`, and focused tests for parsing, validation, dry-run, force/conflict behavior, staging, and rollback.
