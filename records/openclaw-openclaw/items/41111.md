## Summary

Current main still lacks a first-party `openclaw backup restore <archive>` command. The backup CLI, command descriptor, docs, and parser tests all expose only `create` and `verify`. The keepmyclaw ClawHub plugin mentioned in the discussion is a useful workaround, and open PR #59280 remains a plausible implementation path, but neither makes the requested native core CLI workflow shipped.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` command or maintainers explicitly consolidate it into a canonical backup/restore tracker. The likely path is to review and either land or close PR #59280, with implementation in the backup command surface, docs at `docs/cli/backup.md`, and focused tests for parsing, validation, dry-run, force/conflict behavior, staging, and rollback.
