## Summary

Keep this open. Current main still exposes only `openclaw backup create` and `openclaw backup verify`; there is no native `openclaw backup restore <archive>` command in the CLI registration, command descriptor, docs, generated CLI index, or parser tests. The keepmyclaw ClawHub plugin is useful prior discussion and PR #59280 is a plausible implementation path, but neither means the requested first-party restore workflow has shipped.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` command or maintainers explicitly consolidate the work into a canonical backup/restore tracker. The practical path is to review and either land or close PR #59280, with implementation in the backup CLI surface, docs at `docs/cli/backup.md`, generated CLI index updates, and focused tests for parsing, validation, dry-run, conflict, force, staging, and rollback behavior.
