## Summary

Current main still does not implement Slack tool-level assistant thread status updates. Slack dispatch sets `assistant.threads.setStatus` to the static `"is typing..."` at reply start and clears it at reply stop; tool-start events are already available in the reply pipeline, but they currently feed reactions and draft-preview progress rather than Slack assistant thread status text. The earlier direct implementation PR #33420 is closed unmerged, and adjacent open work #42555/#73054 does not supersede this specific request.

## Best Possible Solution

Keep this issue open and implement the narrow Slack-owned wiring: add a Slack-local tool-name-to-status-label helper, update `ctx.setSlackThreadStatus` from Slack dispatch on tool-start when an assistant thread target exists, preserve the existing start/clear behavior, avoid update-phase spam, and add focused Slack coverage for the status call sequence. If maintainers prefer configurable static loading messages or draft-preview progress as the intended UX, record that product decision and link the canonical replacement before closing this issue.
