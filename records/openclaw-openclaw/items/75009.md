## Summary

Keep this PR open. Current main does not have the requested WhatsApp DM reply-mode config, and the PR is a useful implementation path, but the fallback currently cannot tell whether the agent already sent a same-chat `message(action=send)` reply, so it can post the suppressed final reply anyway.

## What This Changes

The PR adds `channels.whatsapp.dmVisibleReplies` with account overrides, routes configured WhatsApp DMs through `message_tool_only`, adds a suppressed-final fallback, and updates prompts, docs, config schema, and targeted tests.

## Best Possible Solution

Keep the feature direction, but revise the fallback so it is gated by a per-turn signal that records successful same-chat WhatsApp `message(action=send)` delivery; preserve the automatic default and add coverage for the explicit-send-plus-final-text case.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is: enable `dmVisibleReplies: "message_tool"`, have the agent call `message(action=send)` to the current WhatsApp DM and also return final text; the PR fallback has no state that records that explicit send, so it can deliver the suppressed final reply too.

## Solution Assessment

No. The PR is close, but the fallback is not yet the narrowest maintainable solution because it infers absence of a visible message from dispatcher state that does not include the message tool send path; a same-chat send tracker or equivalent hook is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track same-chat message sends before fallback:** `extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:420-426`
  - body: In `message_tool` mode, `didSendReply` is only flipped by `deliverOutboundPayload`, while a model's `message(action=send)` goes through the outbound message-action runner. If the model sends to the current DM and also returns ordinary final text, this fallback still posts the suppressed final reply, duplicating visible output and breaking the documented "only when no same-chat send happened" behavior. Gate the fallback on an explicit same-chat message-send signal.
  - confidence: 0.86
