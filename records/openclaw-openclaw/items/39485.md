## Summary

Keep this PR open. Current main still has no Discord accountId-to-existing-agent fallback after explicit binding tiers, and the PR remains a focused implementation candidate for #39428. It should not merge as-is because the proposed helper can route to a synthesized non-existent agent when no agents are configured and it labels implicit routing as an explicit account binding.

## Best Possible Solution

Keep this PR open or supersede it with a guarded maintainer patch for #39428. The fix should add a narrow Discord-only implicit fallback after explicit binding tiers, skip accountId "default", require the candidate to exist in agents.list, use a distinct match reason such as "implicit.account", update routing docs/contracts as needed, and add regression coverage for positive matching, default-account behavior, and empty-agent-list behavior.
