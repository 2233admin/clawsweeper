## Summary

Keep this PR open for maintainer review. Current main still aggregates the latest usable descendant fallback reply from each distinct child and joins multiple child outputs, while this PR changes that user-visible fallback to return only the freshest overall child reply; current tests and docs still align with aggregation-scoped behavior, and review comments on the PR head identify material fan-out and interim-filtering risks.

## What This Changes

The PR changes cron isolated-agent descendant fallback behavior to choose only the freshest settled descendant reply instead of joining multiple child replies, adds regression tests for that behavior, and includes a small shared singleton type cast.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a focused cron fallback change that suppresses stale same-child historical replies while preserving one usable latest reply per distinct child for fan-out runs, with a narrower acknowledgement filter and tests covering same-child stale history, multi-child aggregation, silent replies, and true interim acknowledgements; if maintainers explicitly want freshest-only delivery, update the docs and aggregation tests to make that product decision clear.
