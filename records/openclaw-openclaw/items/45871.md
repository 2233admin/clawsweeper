## Summary

Keep open. Current main still enforces a 5-turn ceiling, so this PR is not obsolete, but the branch is not ready to merge because it leaves generated/public config schema surfaces and the current config reference at the old range and edits localized docs that repo policy says are maintained elsewhere.

## What This Changes

The PR raises the configurable `session.agentToAgent.maxPingPongTurns` ceiling from 5 to 20 in the runtime clamp, source Zod schema, type/help text, and session-tool docs while keeping the default at 5.

## Best Possible Solution

If maintainers accept a 20-turn ceiling, land a rebased update that changes the runtime clamp and source validation together, regenerates checked-in config schema/doc baseline artifacts, updates the current English config reference and source docs, removes localized docs edits from this repo, and adds focused boundary coverage for accepting 20 and rejecting or clamping values above 20.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: a value above 5 is rejected by `SessionSchema` and would be clamped to 5 by `resolvePingPongTurns`; tests were not run because this review was required to stay read-only.

## Solution Assessment

No, not as currently proposed. Raising the runtime and validation ceiling is the narrow technical direction if the cap is accepted, but the branch must also update generated/public config surfaces, current English docs, and i18n-policy compliance before merge.
