## Summary

Current main still learns Telegram forum topic names for inbound context, but it does not persist or project those names into session metadata, gateway session rows, or the Control UI session selector. The issue remains valid and is the canonical tracker; related PRs #7407 and #43105 are closed unmerged, while open PR #51124 is adjacent UI cleanup and does not complete the topic-name metadata path.

## What This Changes

Show human-readable Telegram forum topic names in the session dropdown instead of raw topic-qualified Telegram session keys or indistinguishable group-level labels.

## Best Possible Solution

Keep this issue open as the canonical tracker and implement a narrow Telegram/session metadata fix: reuse the existing cached TopicName, map it into a generic thread/topic display label for topic-qualified sessions, persist or derive it in session metadata, expose it through sessions.list, and make the Control UI selector prefer that explicit label. Keep Telegram-specific discovery in the Telegram plugin and avoid hard-coding Telegram policy in core beyond a generic label seam.
