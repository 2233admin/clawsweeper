## Summary

Keep this issue open. Current main captures and persists Telegram forum topic names for inbound agent context, but it still does not carry those names into session metadata, gateway session rows, or the Control UI session dropdown. The related implementation PRs are closed unmerged, while duplicate reports were already closed back toward this tracker.

## Best Possible Solution

Keep this issue open as the canonical tracker and implement a narrow bundled Telegram/session-metadata fix: reuse the cached TopicName from inbound Telegram context, persist or derive topic display metadata for topic-qualified sessions, preserve explicit label/displayName overrides, expose the result through sessions.list, and add regression coverage for Telegram topic extraction, session metadata persistence, gateway rows, and Control UI label resolution.
