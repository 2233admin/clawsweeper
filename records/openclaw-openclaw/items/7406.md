## Summary

Keep open. Current main captures Telegram forum topic names and exposes them to inbound/prompt metadata, but the session metadata, gateway session rows, and Control UI selector still do not carry or prefer a durable topic display label, so the reported dropdown problem remains valid.

## What This Changes

The issue requests human-readable Telegram forum topic names in session dropdown labels instead of raw or topic-id-based session keys.

## Best Possible Solution

Add a generic thread/topic display-label field through session metadata, populate it from Telegram's cached TopicName for forum-topic sessions, expose it through sessions.list, and have the Control UI selector prefer that explicit label while preserving existing non-Telegram fallbacks.
