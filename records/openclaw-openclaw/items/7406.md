## Summary

Keep open. Current main learns Telegram forum topic names for inbound context, but the value still does not flow into stored session display metadata, gateway session rows, or the Control UI selector, so the dropdown problem remains.

## What This Changes

Show Telegram forum-topic sessions in the session dropdown with human-readable group/topic labels instead of raw topic-qualified session keys.

## Best Possible Solution

Add a narrow generic thread/topic display-label path in session metadata, populate it from Telegram's cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving existing fallbacks.
