## Summary

Keep open. Current main still uses a shared OpenClaw media root for inbound channel media, bundled channel paths still write to the literal `inbound` subdirectory, managed inbound media references still reject nested/scoped IDs, sandbox staging still rewrites attachments into flat `media/inbound/<filename>` paths, and docs still describe flat/shared inbound staging. The closed PR #46162 is useful prior work but was not merged, while #25712 and #67194 point back here as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical core media isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, update sandbox staging and public docs, and add regression coverage. PR #46162 can be used as prior art, but it should be reopened, reviewed, or replaced before this issue closes.
