## Summary

Keep open. Current main still stores inbound media under the shared OpenClaw media root, bundled channel callers still pass the literal `inbound` bucket, and the managed media read-side still rejects nested/scoped inbound IDs. The issue remains the canonical tracker for a real multi-agent privacy/correctness gap.

## Best Possible Solution

Keep this issue open as the canonical core media isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, update sandbox staging and public docs, and add regression coverage. PR #46162 is useful prior art, but it was closed unmerged and should be reopened, reviewed, or replaced before this issue closes.
