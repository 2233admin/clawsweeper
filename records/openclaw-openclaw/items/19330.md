## Summary

Keep this issue open. Current main still uses a shared inbound media store across bundled channel downloads, agent-scoped media roots still include the global media directories, managed inbound references still reject scoped/nested IDs, and sandbox staging still flattens media to media/inbound/<basename>. The closed PR #46162 was an unmerged implementation attempt, while #60154 is only a narrower Feishu-specific follow-up.

## Best Possible Solution

Keep this issue open as the canonical cross-channel media-isolation tracker. A maintainer-quality fix should define a backward-compatible scoped inbound-media contract, thread sanitized account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, constrain agent media roots appropriately, preserve or intentionally remap scope through sandbox staging, update docs/config help, and add regression coverage. Keep #60154 as a Feishu-specific slice unless the generic fix fully covers it.
