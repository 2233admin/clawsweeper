## Summary

Keep this issue open. Current main still uses a shared, flat inbound media store across bundled channel callers, the managed-media reference layer still rejects scoped/nested inbound paths, and sandbox staging still rewrites accepted media to a flat `media/inbound/<basename>` path. The existing comments and related context also point the same way: closed PR #46162 is useful prior art but was not merged, and open #60154 is a narrower Feishu account-isolation request rather than a replacement for this cross-channel tracker.

## Best Possible Solution

Keep this issue open as the canonical core media-isolation tracker. A maintainer-quality fix should define a backwards-compatible scoped inbound-media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve or intentionally map scope through sandbox staging, update docs/config help, and add regression coverage. PR #46162 is useful prior art; #60154 should inform the Feishu-specific slice but should not replace this broader issue.
