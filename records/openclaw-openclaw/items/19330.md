## Summary

Keep this issue open. Current main still uses a shared flat inbound media store for representative bundled channel downloads, still rejects scoped/nested managed inbound references, and still stages sandbox media into a flat `media/inbound/<basename>` path. The related context points to this issue as the broad tracker; #60154 is only a narrower Feishu slice, and #46162 was closed without merging.

## Best Possible Solution

Keep this issue open as the canonical core media-isolation tracker. A maintainer-quality fix should define a backward-compatible scoped inbound-media contract, thread sanitized account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, preserve or intentionally remap scope through sandbox staging, update docs/config help, and add regression coverage. Keep #60154 as the Feishu-specific configuration slice unless the generic fix fully covers it.
