## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from stored session rows plus the active session key, not from all configured agents, so configured agents with no stored session row can still be missing.

## Best Possible Solution

Keep this issue open and finish the active implementation path. The fix should update `ui/src/ui/chat/session-controls.ts` so `resolveSessionOptionGroups` combines stored `sessions.list` rows with `state.agentsList?.agents`, synthesizes normalized per-agent main-session keys for configured agents without rows, dedupes case-insensitively, preserves global/unknown/cron filtering, and adds regression coverage for configured agents with no existing session.
