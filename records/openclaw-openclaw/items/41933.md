## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from `sessions.list` rows plus the active session key; `agents.list` is loaded and used for labeling, but not used to add configured agents that have no stored session row. The backend `sessions.list` path still reads the session store, so the reported missing-agent case remains plausible. The provided GitHub context also shows open PR #41955 with closing syntax for this issue, so the issue should stay open until that PR lands, closes, or is superseded.

## Best Possible Solution

Keep this issue open and review, land, or supersede the active implementation path. The likely fix belongs in `resolveSessionOptionGroups`: after adding stored sessions and the active key, iterate `state.agentsList.agents`, add canonical main-session options for configured agents without session rows, dedupe keys case-insensitively, preserve existing global/unknown/cron filtering, and add a regression test for configured agents with no stored session row. PR #41955 and related PR #48671 should be evaluated against that narrow behavior before closing the issue.
