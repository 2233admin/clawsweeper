## Summary

Current main still does not implement the requested Control UI/WebChat dropdown behavior. The dropdown resolver builds options from stored session rows plus the active session key, while `agents.list` is only used for group labels. Configured agents that have no stored session row are therefore still omitted. Related implementation work remains open in #48671, with earlier direct attempts #41955 and #42041 closed unmerged.

## Best Possible Solution

Keep this issue open and review/land #48671 or an equivalent focused patch. The fix should merge `state.agentsList.agents` into `resolveSessionOptionGroups` as normalized per-agent main-session options, dedupe against stored sessions and the active key, preserve current global/unknown/cron filtering, respect the configured main key semantics, and add regression coverage for configured agents with no stored session row.
