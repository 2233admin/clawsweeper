## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from `sessions.list` rows plus the active session key, while `agents.list` is available only for labeling sessions that already made it into the dropdown. The issue has concrete reproduction details and an open PR with closing syntax, so it should remain open until a narrow implementation lands or is superseded.

## Best Possible Solution

Keep this issue open and review, refresh, or supersede the active implementation path. The fix should add configured `agents.list` entries to `resolveSessionOptionGroups` as canonical main-session options, dedupe against existing `sessions.list` rows case-insensitively, preserve current filtering for global, unknown, cron, and active sessions, and add regression coverage for configured agents with no stored session row.
