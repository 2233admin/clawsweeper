## Summary

Keep this issue open. Current main still builds the Control UI chat session dropdown from stored `sessions.list` rows plus the active session key, while `agents.list` is available but only used for labeling options that already exist. The provided discussion also shows open implementation candidates with closing syntax, so this should stay open until one lands or is superseded.

## Best Possible Solution

Keep this issue open and review or land the active implementation path, preferably the refreshed #48671 branch or a successor. The fix should add configured `agents.list` entries to `resolveSessionOptionGroups` as canonical agent main-session keys, dedupe existing `sessions.list` rows, preserve current global/unknown/cron filtering, and add regression coverage for configured agents with no stored session.
