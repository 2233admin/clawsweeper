## Summary

Keep open. Current main does not implement this PR’s intended token/password path: device-less shared-secret operator sessions outside the narrow direct-local backend exception still lose self-declared scopes, and tests encode that behavior. The PR touches a security-sensitive gateway auth boundary and should be reviewed or adapted, not auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review alongside #51396. The best path is to decide the supported policy for non-local shared-secret operator clients, then either rebase/adapt this PR against current main with token and password regression coverage plus a remote backend write-RPC proof, or explicitly close it in favor of a canonical PR once that PR actually covers the same token/password behavior.
