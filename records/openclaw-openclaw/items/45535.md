## Summary

Keep this PR open. Current main has adjacent OpenAI-style usage normalization, but it still does not implement this PR's central mixed-alias behavior: an earlier zero `input_tokens` value still wins before a later positive `prompt_tokens`, and the current CLI usage parser still lacks `prompt_tokens` / `completion_tokens` aliases. The PR also needs a refresh against the current `src/agents/cli-output.ts` parser and must preserve the current zero-only JSONL guard.

## Best Possible Solution

Update this PR or replace it with a fresh patch against current main that adds positive-before-zero alias selection in `src/agents/usage.ts`, adds `prompt_tokens`/`completion_tokens` alias support in `src/agents/cli-output.ts`, and adds regression tests for both the mixed `input_tokens: 0` plus `prompt_tokens: N` case and JSONL streams where later all-zero usage must not overwrite earlier nonzero usage.
