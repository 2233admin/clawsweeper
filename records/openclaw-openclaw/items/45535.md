## Summary

Keep this PR open. Current main still has the mixed-alias bug this PR is trying to address: `normalizeUsage` selects `input_tokens` before `prompt_tokens` with a nullish chain, so an earlier `input_tokens: 0` still blocks a later positive `prompt_tokens`. The current CLI output parser also still lacks `prompt_tokens` / `completion_tokens` aliases. The submitted PR is stale against the current parser location and has a known JSONL zero-overwrite concern, so the best path is a refreshed patch rather than closing it as implemented.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that implements positive-before-zero alias selection in `src/agents/usage.ts`, adds `prompt_tokens`/`promptTokens` and `completion_tokens`/`completionTokens` support in `src/agents/cli-output.ts`, and adds regression tests for both the mixed `input_tokens: 0` plus `prompt_tokens: N` case and the zero-only JSONL stream guard.
