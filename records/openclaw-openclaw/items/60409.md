## Summary

Keep this PR open. Current main still reads only resolved primary transcript files for `chat.history` and title derivation, while reset flows still archive old transcripts as `.reset.<timestamp>` files; the PR addresses that live gap with a narrow fallback and targeted tests. The related open PRs cover complementary archive visibility, memory-hook, session-log, and compaction-chain surfaces rather than superseding this focused read-path change.

## What This Changes

The PR exports transcript candidate classification, adds a latest reset-archive lookup helper, uses it in `readSessionMessages` and title-field derivation when no primary transcript exists, and adds gateway session-utils tests for canonical, legacy, non-canonical, stale, and title fallback cases.

## Best Possible Solution

Land this patch or an equivalent narrow gateway fallback after maintainer review confirms the reset-archive visibility semantics. Preserve primary transcript precedence, stale-session exclusion, timestamp validation, and regular-file filtering, then coordinate archive selector, compaction-chain, memory-hook, and archive UX follow-ups through #45003, #68765, #71537, and #73883.
