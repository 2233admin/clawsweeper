## Summary

Keep open. Current main still forwards the cron agentTurn message directly into isolated execution, where non-string runtime values would stringify in the prompt, so the proposed fix is not obsolete. The PR should not merge as-is because the fallback path logs serialized prompt content and the malformed-payload path still lacks focused regression coverage.

## What This Changes

This PR modifies `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from `text`, `message`, or `content` fields, JSON-stringify and debug-log unrecognized shapes, and pass the extracted value to the isolated runner.

## Best Possible Solution

Land a tested cron fix that preserves the public `message: string` contract: normalize or repair malformed stored payloads before execution where possible, and keep any defensive extraction at the detached execution boundary narrow and free of prompt-content logging.
