## Summary

Keep this issue open. Current main still sends CLI-driven gateway-executed plugin message actions through a 10,000 ms gateway RPC wrapper, and the `openclaw message send --media` path does not expose or pass a longer timeout for slow external channel uploads such as the reported WeCom large-file send.

## Best Possible Solution

Keep the issue open and fix the OpenClaw core CLI/gateway boundary so `openclaw message send`, especially media sends handled by gateway-executed channel plugins, can use a longer gateway RPC timeout. Preserve the plugin boundary, avoid WeCom-specific code, and add focused regression tests plus docs/changelog for the user-facing CLI behavior.
