## Summary

Keep this issue open. Current main still has a 10,000 ms gateway wrapper on CLI-driven gateway plugin message actions, and `openclaw message` does not expose or forward a longer timeout for the reported WeCom large-file send path. The WeCom runtime is external, but the observed error is from OpenClaw core's gateway RPC wrapper before the plugin action can complete.

## Best Possible Solution

Keep this issue open and fix the OpenClaw core CLI/gateway boundary: allow `openclaw message` sends, especially media sends handled by gateway-executed plugins, to use a longer RPC timeout while preserving the existing plugin boundary and avoiding WeCom-specific logic in core.
