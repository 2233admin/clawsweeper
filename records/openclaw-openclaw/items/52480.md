## Summary

Keep open. Current main still only retries Discord shared requests on Carbon `RateLimitError`, and the paired bug #52396 is still open. The PR is still a plausible implementation candidate, but the provided final diff broadens a shared retry runner used by non-idempotent Discord POSTs, so it needs maintainer review or a safer idempotent/send split rather than cleanup closure.

## Best Possible Solution

Keep this PR and the paired open bug #52396 open until maintainers either land a focused fix or close the pair together. The best implementation path is to add Discord transient-error classification with separate retry semantics for idempotent reads/edits/probes versus non-idempotent sends/thread creates, plus regression tests covering retryable network failures and duplicate-send avoidance.
