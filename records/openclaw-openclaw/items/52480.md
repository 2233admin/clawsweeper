## Summary

Keep open. Current main still retries Discord REST calls only for the internal `RateLimitError`, so the intended transient-error fix is not implemented, but this PR is not merge-ready because the added test imports an undeclared RateLimitError class and the production change applies broad transient retries to non-idempotent Discord send/thread POSTs.

## What This Changes

The PR adds a Discord transient-error regex to the shared retry runner and adds a new retry unit test file for rate-limit, transient, permanent, and exhausted retry cases.

## Best Possible Solution

Use a Discord retry policy that preserves `retry_after` handling, retries safe transient failures, and explicitly separates idempotent REST/lookup/edit/delete retry behavior from non-idempotent message/thread/component/poll/sticker/voice POST behavior, with tests for both retry and no-retry cases.
