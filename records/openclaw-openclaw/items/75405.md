## Summary

Close as implemented on current main and shipped in v2026.4.29. The Telegram native slash-command path now routes through fresh bindings, targets the routed chat session, and marks native commands as explicit visible replies, which addresses the reported 2026.4.27 silent drop for group slash commands bound to non-main agents.

## What This Changes

Telegram native slash commands in group chats bound to non-main agents should route to the bound agent and reply visibly instead of being silently dropped.

## Best Possible Solution

Keep the #73672 behavior in place and ask affected users to upgrade to v2026.4.29 or newer; any broader Telegram group fallback work should remain separate from the now-fixed native slash-command path.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.27 reproduction path, and current-main source/tests verify the fixed path by checking group native-command auth, routed command target sessions, and visible native-command delivery.

## Solution Assessment

Yes. Treating channel-validated native commands as explicit visible replies while preserving owner allowlists is narrower and safer than the broader suppressed-source fallback attempted in the closed unmerged #73583 branch.
