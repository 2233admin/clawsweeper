## Summary

Keep this PR open. Current main still does not implement the proposed Control UI stale-run watchdog, and the linked same-author bug #42213 remains open with this PR as the implementation candidate. The backend `agent.wait` capability exists, but the UI still clears `chatRunId` only through terminal chat events or reconnect-scoped cleanup, so queued messages can remain gated behind a stale active run.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main, address the review comments, preserve the `agent.wait`-backed stale-run recovery semantics, and keep regression coverage for stale-finished recovery, still-active timeout retry, and watchdog cancellation/reset before maintainer review.
