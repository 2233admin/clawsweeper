## Summary

Keep this PR open. Current main still lacks the proposed Control UI stale-run watchdog, and the linked bug remains plausible: `chatRunId` still makes the UI busy, queued sends still return early while busy, and the UI still clears the run only from matching terminal chat events or reconnect cleanup. The PR is an implementation candidate for open #42213 rather than obsolete.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213, with #42011 as related context. Rebase it onto current main, address the watchdog review comments, update the security-impact wording to mention the existing authenticated `agent.wait` probe, and preserve regression coverage for stale-finished recovery, active-run timeout retry, and watchdog cancellation/reset before maintainer landing review.
