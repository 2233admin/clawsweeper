## Summary

Keep this issue open. Current main has moved Pi to 0.70.5 and shipped an opt-in active-transcript byte guard, but the embedded gateway path still calls `SessionManager.open()` on the active JSONL and the upstream 0.70.5 implementation still fully reads/parses the same file once in `open()` and again through the constructor. The related #73510 is a broader stuck-session recovery report and does not supersede this narrower duplicate-load crash path.

## What This Changes

The issue asks OpenClaw to avoid gateway crashes when large embedded-session JSONL transcripts are reopened and rebuilt, especially the duplicate full-file load in `SessionManager.open()`.

## Best Possible Solution

Add a narrow OpenClaw session-manager open helper or upstream Pi fix that reads only the JSONL header for cwd and constructs/reuses the manager without a second full parse, then cover the embedded run path with a large-transcript regression proving it reaches compaction/truncation or a controlled error instead of exiting the gateway.
