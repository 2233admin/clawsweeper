## Summary

Keep open. Current main still only partially addresses the Google Chat Workspace Add-ons report: app-url/add-on auth exists, but public setup docs and the setup wizard still omit the required `appPrincipal`, and native Add-ons wrapper parsing is still limited to `chat.messagePayload`. The discussion workaround from SmileyChris is useful evidence because it handles additional Add-ons payload variants that current main still does not normalize. Open PR #65058 overlaps the parser subset, but it is partial and does not address this issue's docs/setup gap.

## What This Changes

The issue asks OpenClaw to document and natively support Google Chat Workspace Add-ons app-url JWT principal binding and wrapped event-body formats without requiring an external translation proxy.

## Best Possible Solution

Keep the issue open and finish Google Chat plugin compatibility in the owner plugin: document `audienceType: "app-url"` plus numeric `appPrincipal` discovery on the public Google Chat page, add setup/CLI guidance or input for `appPrincipal`, broaden Add-ons wrapper normalization for supported Chat payload variants or intentionally acknowledge unsupported non-message events, and cover those paths with focused tests while keeping strict principal binding by default.
