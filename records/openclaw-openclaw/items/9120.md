## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run collect queueing, but not the requested user-controlled continuation marker that buffers marked fragments until a later unmarked message. The issue also already has an Apr 26, 2026 maintainer-side automated review by steipete explicitly keeping it open for the same missing behavior.

## Best Possible Solution

Keep this open for maintainer/product judgment. A complete fix likely belongs in shared inbound handling or a documented plugin/API seam: add typed messages.continuationMarker and messages.continuationTimeout config/schema/docs, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, flush on the first unmarked message or timeout, define media/control-command behavior, and add focused tests across inbound policy, debouncer behavior, queue interaction, and typing/status behavior.
