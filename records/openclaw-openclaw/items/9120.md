## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run collect queueing, but not the requested user-controlled continuation marker that buffers marked fragments until a later unmarked message arrives. The issue discussion also already contains an Apr 26, 2026 maintainer-side Codex review keeping it open for the same missing behavior.

## Best Possible Solution

Keep this open for maintainer/product judgment. A complete implementation should add typed messages.continuationMarker and messages.continuationTimeout config/schema/help/docs if the product direction is accepted, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, flush on the first unmarked message or timeout, define media/control-command behavior, and cover inbound policy, debouncer, queue interaction, and typing/status behavior with focused tests.
