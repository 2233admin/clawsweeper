## Summary

Keep this PR open. Current main still does not expose `api.runtime.llm.complete()` or context-engine LLM capabilities, and the related product/API request remains open, but the proposed patch is not merge-ready because it adds a new credential-using inference path without settled agent binding, permission, cost, and observability semantics.

## What This Changes

The PR adds a plugin runtime LLM completion facade, exports new SDK types, and injects an LLM helper into context-engine compaction, after-turn, and maintenance runtime contexts.

## Best Possible Solution

Ship an additive LLM runtime seam only after the API is bound to the active agent/session, documented in the plugin runtime docs, covered by SDK/API baseline updates, and tested for default and non-default agent model/auth resolution. If maintainers choose a different API shape, keep #45865 as the canonical design discussion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the runtime config snapshot for completions:** `src/plugins/runtime/runtime-llm.runtime.ts:18`
  - body: `runtime-llm.runtime.ts` calls `loadConfig()` directly from `src/plugins/runtime`, but the config-boundary guard scans this tree and rejects ambient `loadConfig()` calls outside the small compat allowlist. This will fail the deprecated-internal-config-api check and also bypasses the active runtime config snapshot; pass or bind the runtime config instead.
  - confidence: 0.9
- **[P2] Bind completions to the active agent:** `src/plugins/runtime/runtime-llm.runtime.ts:19`
  - body: When callers omit `agentId`, the new facade resolves the config default agent. The same facade is injected into context-engine hooks for any active session, so a non-default agent can silently run completions with another agent's model/auth profile unless the injected capability is bound to the active session agent or requires explicit agent selection.
  - confidence: 0.86
- **[P2] Commit the SDK API baseline update:** `src/plugin-sdk/index.ts:63-67`
  - body: Exporting `PluginLlmComplete*` from the public SDK changes the generated API baseline, but the PR does not include the updated `docs/.generated/plugin-sdk-api-baseline.sha256`. `pnpm plugin-sdk:api:check` will report drift until the baseline hash is regenerated and committed.
  - confidence: 0.82
