## Summary

Keep open. Current main still lacks a first-class one-way dispatch or handoff mode for `sessions_send`. The agent-facing schema has no `mode`, `replyBack`, `announce`, `dispatch`, `sessions_deliver`, or `sessions_enqueue` control; `timeoutSeconds: 0` still returns pending announce delivery and starts the A2A flow; and the A2A helper still performs reply-back turns plus an announce step. Related open issues cover adjacent announce suppression or duplicate reverse-call safety, but they do not supersede this distinct per-call delivery semantic.

## Best Possible Solution

Implement an explicit delivery intent for `sessions_send`, such as `mode: "conversation" | "dispatch"`, or add a separate `sessions_deliver`/`sessions_enqueue` primitive. The dispatch path should enqueue/persist the message into the target session and return a minimal delivery ack without calling `runSessionsSendA2AFlow`, without requester-side reply reinjection, and without announce unless explicitly requested. Add focused coverage in `src/agents/openclaw-tools.sessions.test.ts` and `src/agents/tools/sessions-send-tool.a2a.test.ts`, update `docs/concepts/session-tool.md`, and keep #39476 as the separate duplicate/reverse-call guard track.
