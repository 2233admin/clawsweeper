- Changing launchd to `SuccessfulExit=false` without updating the supervised restart flow can break explicit `openclaw gateway restart`, because current main exits with status 0 and relies on launchd `KeepAlive` to relaunch.
- The PR mixes the launchd fix with unrelated sandbox Docker changes, widening review and rollback scope.
- Security/supply-chain pass: the sandbox diff adds remote Docker image pull/tag behavior for `debian:bookworm-slim`, which could silently replace the intended OpenClaw sandbox image with a degraded third-party base image when Docker is available but the custom image is missing.
- Existing review comments identify concrete cleanup issues in the branch: the Docker fallback, an unused variable in `doctor-sandbox.ts`, and a misplaced duplicate type import in `browser.ts`.
