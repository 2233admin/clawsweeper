## Summary

Keep this PR open. Current main still uses `KeepAlive=true`, so the launchd crash-loop/debugging problem is not implemented on main. However, this PR is not ready to close or merge as-is: the proposed `SuccessfulExit=false` plist conflicts with the current supervised restart path, which exits cleanly under launchd and relies on `KeepAlive` to relaunch, and the branch also includes unrelated sandbox Docker changes with a supply-chain/runtime behavior risk.

## Best Possible Solution

Keep the PR open, but narrow it to the launchd problem. The best fix should preserve explicit `gateway restart` behavior while adding a targeted crash-loop/debugging path, likely by changing both the LaunchAgent policy and the supervised restart flow together or by adding a separate backoff/disable-after-repeated-failures mechanism. The unrelated sandbox Docker commits should be removed or moved to a separate PR with their own tests and security review.
