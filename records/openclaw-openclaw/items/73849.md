## Summary

Keep this PR open, but do not merge it as-is. Current main does not implement the proposed launchd `SuccessfulExit=false` plist change, but the PR conflicts with the current supervised restart/update path that exits with status 0 and relies on `KeepAlive=true`, and it also mixes in unrelated sandbox Docker changes with a concrete supply-chain/runtime regression.

## What This Changes

This PR changes the macOS LaunchAgent plist, audit check, and launchd test from boolean `KeepAlive=true` to `KeepAlive` with `SuccessfulExit=false`, while also modifying sandbox Docker/browser/doctor handling for Docker daemon and image availability.

## Best Possible Solution

Keep the launchd problem open and replace this with a focused daemon-only fix that preserves explicit `openclaw gateway restart` and update restarts while improving crash-loop/debuggability, for example by coordinating the plist policy with the supervised restart handoff or adding a targeted backoff/disable-after-repeated-failures path. The sandbox Docker changes should stay out of this PR and continue to honor the no-silent-Debian-substitution contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: run under launchd, trigger the current supervised restart/update path, observe it exits with status 0, and apply `SuccessfulExit=false`; launchd semantics then no longer relaunch that clean exit. The Docker regression is also reproducible by missing `openclaw-sandbox:bookworm-slim` with Docker available, where the PR pulls and tags plain Debian instead of failing fast.

## Solution Assessment

No. The proposed plist change is not the narrowest maintainable solution because it changes clean-exit behavior without updating the restart/update flow that depends on clean exits, and the PR also carries unrelated sandbox changes. A safer solution is a focused launchd lifecycle change with tests for both crash-loop debugging and intentional restarts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve launchd-supervised relaunches:** `src/daemon/launchd-plist.ts:181`
  - body: Changing the generated LaunchAgent to `SuccessfulExit=false` means launchd will not relaunch after a clean exit. Current supervised restart and update paths return `mode: "supervised"` and then call `exitProcess(0)`, relying on `KeepAlive=true` to bring the gateway back, so this patch can turn `openclaw gateway restart` or update restarts into a stopped service unless those paths are changed too.
  - confidence: 0.92
- **[P1] Do not substitute the default sandbox image:** `src/agents/sandbox/docker.ts:320-327`
  - body: When the default sandbox image is missing and Docker is running, this branch pulls `debian:bookworm-slim` and tags it as `openclaw-sandbox:bookworm-slim`. That contradicts the current sandbox contract because the default image carries Python and helper tooling that plain Debian lacks, so users can silently get a degraded sandbox.
  - confidence: 0.94
- **[P2] Drop the unused and duplicate browser imports:** `src/agents/sandbox/browser.ts:29-129`
  - body: The browser change adds unused Docker imports and then adds a second `SandboxConfig` type import after `SandboxConfig` is already imported at the top of the file. Clean this up before the branch can pass the repository's import/lint/type checks.
  - confidence: 0.84
- **[P3] Remove the unused lowercase copy:** `src/commands/doctor-sandbox.ts:89`
  - body: `lower` is assigned but never read; `isDockerDaemonUnavailable` is called with `stderr`. This should be removed so the changed doctor path does not introduce an avoidable lint/typecheck failure.
  - confidence: 0.86
