## Summary

Keep open. Current main still lacks the SDK-facing `tools.invoke` RPC requested by the linked member-authored issue, and this PR is the active implementation path, but the patch has a blocking SDK request-building bug on a direct tool execution surface: an unexpected `params.name` value can override the explicit tool name argument before the request is sent.

## What This Changes

The PR adds a `tools.invoke` Gateway RPC, SDK helper/types, shared direct-tool invocation core reused by HTTP `/tools/invoke`, generated Swift params, docs, changelog, and targeted tests for scope, policy, and plugin-approval behavior.

## Best Possible Solution

Fix the SDK builder so the explicit `oc.tools.invoke(name, params)` argument cannot be overridden by params, then land the additive RPC only after maintainer review confirms the `operator.write` scope, admin-only owner treatment, HTTP parity, approval-required result contract, and generated protocol artifacts are acceptable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the explicit SDK tool name:** `packages/sdk/src/client.ts:185`
  - body: Because `gatewayParams` is spread after `{ name: toolName }`, any JavaScript caller or cast can pass `{ name: "other-tool" }` in the params object and silently invoke a different tool than the explicit first argument. On this direct tool execution surface, strip or reject `params.name`, or spread params first so the explicit `name` argument always wins.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff adds a sensitive direct tool invocation RPC and the SDK helper currently allows an unexpected params key to override the explicitly requested tool name.

Concerns:

- **[medium] SDK params can override tool name:** `packages/sdk/src/client.ts:185`
  - body: The new SDK helper spreads caller params after the trusted tool-name argument. If an app forwards a user-controlled params object, a hidden `name` field can switch the invoked tool while the call site appears to request another tool.
  - confidence: 0.88
