## Summary

Keep this PR open. Current main does not implement the proposed `before_identity_resolve` hook or an equivalent dynamic peer-identity seam: hook names/types omit it, the hook runner has no identity-resolution phase, the routing SDK exports no helper, and route/session-key construction still only uses static `session.identityLinks`. The PR remains a plausible implementation candidate, but it needs maintainer review because the discussion identifies real routing/session correctness follow-ups around blank canonical IDs, secondary Slack/WhatsApp paths, and multi-plugin execution behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused additive plugin contract: add a typed identity-resolution hook, expose a narrow SDK helper, thread a `resolvedPeerId` override through route/session-key construction, integrate all relevant direct-message paths for Slack, MS Teams, and WhatsApp, and add contract/channel tests that cover normalization, blank results, fallback to `identityLinks`, multi-plugin priority, and secondary routing paths.
