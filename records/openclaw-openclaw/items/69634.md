## Summary

Keep open. Current main still has the overlap-prone layout contract: assistant bubbles render the floating message action bar, and CSS has a `.chat-bubble.has-copy` padding rule, but the current renderer does not apply `has-copy`. The PR's intended fix is still relevant, though the branch is stale against the current `grouped-render.ts` shape and should be rebased or folded into the newer related work without dropping current bubble classes.

## What This Changes

The PR adds the existing `has-copy` class to Control UI chat bubbles when markdown copy or expand actions are available.

## Best Possible Solution

Land this layout fix, or an equivalent change in the current renderer, by adding `has-copy` to the existing multiline `bubbleClasses` array whenever `hasActions` is true, while preserving `chat-bubble--tool-shell`; add a focused regression assertion for assistant bubbles with copy/expand actions.
