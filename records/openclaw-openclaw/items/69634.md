## Summary

Keep #69634 open. Current main still renders message-level chat actions, and the CSS still reserves space only for `.chat-bubble.has-copy`, but `renderGroupedMessage` does not add `has-copy` to `bubbleClasses`. The related code-block copy button and handler exist, but the specific overlap fix proposed by this PR is not implemented on main. Related PRs #69689 and #69888 are closed/unmerged duplicate attempts, and the linked bug #69605 remains the canonical open issue.

## Best Possible Solution

Keep the PR or linked bug #69605 open. The best maintainer path is to apply the `has-copy` class when `canCopyMarkdown || canExpand` is true in `ui/src/ui/chat/grouped-render.ts`, and add a focused `grouped-render.test.ts` assertion that assistant bubbles with message-level actions receive `has-copy` while bubbles without actions do not.
