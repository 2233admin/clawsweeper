## Summary

Keep this PR open. The underlying Control UI gap is still real on current main, but the branch is dirty against main and its latest diff still has two concrete correctness regressions in agent-session key construction and thinking-option preservation.

## What This Changes

This PR adds a chat-controls agent select, moves the session/model/thinking selectors into the same controls row, and adds CSS sizing for the new agent control.

## Best Possible Solution

Add the agent picker to the current shared chat session-controls module, keep the existing model/thinking selector behavior intact, normalize full agent session keys down to their main key before building target keys, and add focused tests for multi-agent switching from plain and agent-prefixed sessions.

## Reproduction Assessment

Yes. The key bug is reproducible by code inspection with a plain active session and `sessionDefaults.mainSessionKey` set to `agent:main:main`, which the PR passes as a `mainKey` and turns into `agent:beta:agent:main:main`; the thinking bug is reproducible with a session row that advertises custom `thinkingOptions` or `thinkingDefault`.

## Solution Assessment

No. The feature direction is valid, but this branch is not the best fix as-is because it duplicates stale selector logic and drops current-main session-specific thinking behavior; the safer path is a rebase or narrow replacement on top of `ui/src/ui/chat/session-controls.ts`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Strip full agent keys before switching agents:** `ui/src/ui/app-render.helpers.ts:178`
  - body: When the active session is plain or legacy, `resolveSidebarChatSessionKey` can return an already-prefixed value such as `agent:main:main`. Passing that as `mainKey` builds nested targets like `agent:beta:agent:main:main`, so switching agents can open an unintended empty session instead of the selected agent's main session.
  - confidence: 0.88
- **[P2] Preserve session-provided thinking choices:** `ui/src/ui/app-render.helpers.ts:740-751`
  - body: The new thinking selector derives options and defaults only from provider/catalog data. `sessions.list` rows can provide `thinkingOptions`, `thinkingLevels`, and `thinkingDefault`, so sessions with values such as `xhigh` or provider-specific levels lose supported choices and can show the wrong default.
  - confidence: 0.9
