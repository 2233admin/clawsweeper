## Summary

Keep this PR open. Current main has partial turn-source delivery-plan hardening and generic `currentChannelId` fallback, but it has not implemented the PR's session-patch behavior or focused agent-handler regressions. The branch also has an unresolved route-field atomicity concern from security review, so it remains useful but should not be closed as already implemented.

## Best Possible Solution

Keep this PR open. Maintainers should resolve the Aisle accountId/to atomicity concern, adapt the branch to current `src/infra/outbound` turn-source delivery helpers, and then either land this branch or apply an equivalent narrow fix in `src/gateway/server-methods/agent.ts` with the focused gateway regression tests.
