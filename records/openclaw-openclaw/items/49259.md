## Summary

Keep this issue open. Current main still has manual selected-session deletion and generic store cleanup, but it does not provide the requested Dashboard Sessions prune workflow with a 48-hour default age, orphan-candidate preview, or Telegram/Discord channel/topic existence checks. Related #50310 is still open and narrower Discord cleanup work, not a replacement for this broader Dashboard feature.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-designed, additive prune workflow: add a preview RPC that returns stale orphan candidates with explicit reasons, expose safe plugin-owned Telegram/Discord reachability checks through a generic seam, revalidate candidates during apply, and wire Dashboard Sessions to show a 48-hour default age field, candidate preview, and explicit Prune action. Keep #50310 as narrower Discord cleanup context rather than a duplicate.
