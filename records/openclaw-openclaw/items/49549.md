## Summary

Keep this PR open. Current main still does not implement WhatsApp `allowTo` or any equivalent outbound-only allowlist, and the open PR remains a focused implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open. The best path is to review and rebase it against current main, decide whether OpenClaw wants the full `allowTo` config/schema/runtime/docs/tests model or the narrower #48387 behavior, and require the directory-resolution gap to be fixed or explicitly split to a canonical follow-up before landing.
