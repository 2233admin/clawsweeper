## Summary

Keep open. Current main still does not implement WhatsApp `allowTo` or any equivalent outbound-only allowlist, so this PR remains a relevant implementation candidate for the linked allowFrom/outbound coupling problem. The PR discussion is useful: earlier null/action/cron issues were addressed on the branch, but the latest review still raises a real directory-resolution gap for name/picker/autocomplete flows.

## Best Possible Solution

Keep this PR open. The best path is to review it against current main, require the remaining directory-resolution behavior to be implemented or explicitly split to a canonical follow-up, then land the WhatsApp `allowTo` config/schema/runtime/docs/tests if maintainers accept the outbound authorization model.
