## Summary

Keep this PR open, but do not merge it as-is. The linked blank-tool-name loop is a real agent-runtime problem that current main still leaves unresolved, while this patch introduces blocking regressions by converting recoverable blank streaming placeholders into a normal `_blank` dispatch name and by leaving `_blank` available for real tool-name collisions.

## What This Changes

The branch changes embedded agent tool-call normalization to replace blank or whitespace-only tool names with a `_blank` sentinel, treats that sentinel as incomplete for unknown-tool loop classification, and updates related tests.

## Best Possible Solution

Rework blank-name handling so unrecoverable blank tool calls stop retrying without becoming a dispatchable real tool name. The fix should preserve ID-based recovery for streaming deltas, reserve or otherwise make any sentinel non-callable, keep replay pruning semantics intact, and add focused regression tests for late ID arrival, `_blank` tool-name collision, and replay sanitization.

## Reproduction Assessment

Yes. Current main has unit-level coverage around `wrapStreamFnTrimToolCallNames` for blank names, ID inference, ambiguous IDs, and whitespace-only names; the PR regressions can be reproduced with targeted stream tests that mutate the same tool-call block from blank-name/no-id to blank-name/valid-id, plus an allowlist containing `_blank`.

## Solution Assessment

No. The requested direction is valid, but using a normal `_blank` string as the fallback is not the best fix because it can collide with real tools and is written too early into mutable streaming blocks. A safer fix should defer terminal invalid-name handling until recovery is impossible or use a non-dispatchable path that the allowlist cannot match.

  - body: The sentinel is a normal string, and `collectAllowedToolNames` accepts trimmed plugin and client tool names without reserving `_blank`. If a session has a real `_blank` tool, malformed blank model output will dispatch that tool instead of failing closed.
  - confidence: 0.88

  - body: Replay sanitization calls this normalizer too. With no allowlist, a blank replayed tool call with an ID now normalizes to `_blank` and is kept as a valid non-empty name, where current main drops it after trimming to empty.

## Security Review

Status: needs_attention

Summary: The diff touches agent tool dispatch and can convert malformed blank model output into execution of a real `_blank` tool if such a plugin or client tool is allowlisted.

Concerns:

- **[medium] Sentinel can collide with a real tool:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `_blank` is not reserved before dispatch, while the session allowlist accepts arbitrary non-empty trimmed tool names. A plugin or client tool named `_blank` could be invoked from malformed blank model output, which is an unintended tool-execution path.
  - confidence: 0.88
