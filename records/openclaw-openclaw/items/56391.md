## Summary

Keep this PR open. Current main still preserves unresolved blank/whitespace tool names, and the existing unknown-tool loop guard treats blank names as incomplete rather than as unknown-tool retries. The PR remains an active implementation candidate for the open blank-name loop issue, but it needs maintainer review of the sentinel-collision and streaming recovery concerns before it should land.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate for the blank tool-name loop. The best fix should preserve id-based recovery for streamed tool calls, avoid dispatching a real `_blank` tool on malformed model output, and add regression tests for blank/no-id, whitespace-only, ambiguous id, late-arriving id, and `_blank` allowlist collision cases.

## Risks / Open Questions

- A literal `_blank` dispatch sentinel can collide with a real plugin/client tool named `_blank` unless the name is reserved or handled outside normal dispatch.
- Mutating streamed blank names to `_blank` may make later chunks with an id skip the blank-name inference path, turning recoverable calls into false unknown-tool failures.
- The linked bug issue remains open and this PR uses closing syntax for it, so closing the PR as obsolete would remove the current implementation candidate without a shipped replacement.
