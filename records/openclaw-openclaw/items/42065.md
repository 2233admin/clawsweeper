## Summary

Keep this PR open. Current main has substantial Control UI i18n support, generated zh-CN locale output, and an Overview-only language selector, but it still does not implement the PR's central user-facing change: a global topbar locale switcher that works across tabs. The PR also needs rebase/rework for the current generated i18n pipeline and the async locale persistence issue noted in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase or rewrite it against current main, use `scripts/control-ui-i18n.ts` as the single sync/check workflow, decide the product placement for a global language picker, implement the topbar switcher if maintainers want that behavior, localize the remaining topbar/theme/update strings through `ui/src/i18n/locales/en.ts` plus generated locale output, and fix the async `setLocale`/settings persistence ordering before review.
