## Summary

Keep this PR open. Current main has substantial Control UI i18n support, generated locale bundles, and an Overview-card language picker, but it still does not implement the PR's central user-facing request: a global locale switch in the topbar that works across tabs. The PR also needs rework because main now uses a generated Control UI locale pipeline instead of a separate zh-CN checker/report script.

## Best Possible Solution

Keep this PR open and ask for a rebase or rewrite against current main. The best path is to use `scripts/control-ui-i18n.ts` as the only sync/check workflow, decide the product placement for the language picker, implement a global topbar switcher if maintainers want it, localize the remaining topbar/sidebar/update strings through `ui/src/i18n/locales/en.ts` plus generated locale output, and persist settings only after a successful locale load or with explicit failure handling.
