## Summary

Keep this PR open. Current main has substantial Control UI i18n support and generated zh-CN locale output, but it still does not implement the PR's central user-facing change: a global language switcher in the topbar that works across tabs. The PR also needs rebase/rework because main now uses the generated `scripts/control-ui-i18n.ts` pipeline rather than the PR's separate zh-CN checker/report workflow.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to rebase or rewrite it against current main, use `scripts/control-ui-i18n.ts` as the single sync/check workflow, decide the product placement for the language picker, implement a global topbar switcher if maintainers want that behavior, localize the remaining topbar/sidebar/update strings through `ui/src/i18n/locales/en.ts` plus generated locale output, and fix the async locale persistence issue before merge.
