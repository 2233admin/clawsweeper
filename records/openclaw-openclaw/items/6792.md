## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract: plugin manifests do not expose or normalize `configPatch`, install results cannot carry it, and install persistence only updates plugin enablement, slot selection, install records, and registry refresh. The provider-auth `configPatch` prior art is real but remains scoped to provider setup flows, and the linked implementation PR #43421 was closed unmerged while the product/security contract for third-party config writes remains unresolved.

## What This Changes

The issue requests a manifest-level install-time `configPatch` field so plugin installs can apply required OpenClaw config changes automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup without allowing silent arbitrary writes. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests and docs.
