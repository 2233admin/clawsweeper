## Summary

Keep open. Current main still lacks manifest-level install-time `configPatch`: the manifest contract and loader do not expose it, install results cannot carry it, install persistence only mutates plugin install state, and docs still tell users to configure plugin config manually after install. The request is concrete but needs maintainer design because it is a trust-sensitive core plugin API change with unresolved merge/array semantics.

## Best Possible Solution

Keep this issue open for maintainer design of a core plugin-install/API contract. The best path is to add a typed manifest-level config intent or equivalent install-time plugin config API, sanitize and validate it before writing, define merge/conflict/array semantics, show an auditable diff and skip path, document uninstall/manual-revert guidance, and add regression coverage for preserving unrelated config across all install sources.
