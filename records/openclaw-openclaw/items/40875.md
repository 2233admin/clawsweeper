## Summary

Keep this PR open. Current main still lacks heartbeat-specific model fallback guidance, but this PR is not ready to close as implemented or merge as-is: the docs layout has moved to Mintlify `ParamField` blocks, the proposed wording overstates the cross-provider failure case because the configured primary can still be appended, and the changelog entry lacks the required attribution.

## Best Possible Solution

Request a narrow revision: rebase onto current main, place the note inside the current heartbeat `model` `ParamField`, align wording with `docs/concepts/model-failover.md`, explicitly cover no override, same-provider overrides, different-provider overrides already in configured fallbacks, skipped unrelated different-provider fallbacks, and configured-primary retry when no explicit fallback override is supplied. Keep this docs-only unless runtime behavior is intentionally changed, and add the required changelog attribution.
