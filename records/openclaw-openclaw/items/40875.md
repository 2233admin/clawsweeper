## Summary

Keep this PR open. Current main still lacks heartbeat-specific fallback guidance in `docs/gateway/heartbeat.md`, so the docs gap is real, but the submitted patch should not be merged as-is: it targets the older list-style field notes, its strongest warning is inaccurate against current candidate-chain behavior, and the changelog entry does not meet the repo attribution rule.

## Best Possible Solution

Request a narrow revision rather than closing: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, state that no override and same-provider overrides follow the standard fallback chain, state that different-provider overrides skip unrelated configured fallbacks unless the override is already in the configured fallback chain, and mention that the configured primary can still be tried as the final candidate. Fix or remove the changelog entry. Keep #23718 separate as the broader feature request for structured heartbeat fallback configuration.
