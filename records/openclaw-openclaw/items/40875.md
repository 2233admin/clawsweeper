## Summary

Keep this PR open. Current main still does not document heartbeat-specific model fallback behavior in the heartbeat field docs, but the submitted patch should not be merged as-is because it targets the older bullet-list docs structure and its final sentence is inaccurate for the current fallback implementation.

## Best Possible Solution

Request a narrow revision rather than closing: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, state that no override and same-provider overrides follow the standard chain, state that different-provider overrides skip unrelated configured fallbacks unless the override is already in the configured fallback chain, and mention that the configured primary can still be tried as the final candidate. Fix or remove the changelog entry.
