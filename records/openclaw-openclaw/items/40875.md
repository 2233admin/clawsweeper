## Summary

Keep this PR open. Current main still has the heartbeat docs gap the PR is trying to address, but the submitted patch should not be merged as-is because it targets the older list-style heartbeat docs and its strongest warning is inaccurate against the current fallback candidate logic.

## Best Possible Solution

Request a narrow revision instead of closing: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, state that no override and same-provider overrides follow the standard fallback chain, state that different-provider overrides skip unrelated configured fallbacks unless the override is already in the configured fallback chain, and mention that the configured primary can still be tried as the final candidate. Fix or remove the changelog entry.
