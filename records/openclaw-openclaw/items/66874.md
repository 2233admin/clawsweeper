## Summary

Keep open. Current main still supports the bundled Anthropic Claude CLI backend, but the official Docker image and Docker release workflow do not install Claude Code, and the Docker/provider/backend docs still lack a Docker-specific recipe for persistent `claude` installation plus `~/.claude` credential persistence. The earlier ClawSweeper reviews remain accurate; #2440 is adjacent credential-detection history, not a canonical tracker for this Docker image/docs gap.

## What This Changes

The issue requests official Docker support for Claude CLI reuse, either by preinstalling pinned `@anthropic-ai/claude-code` in the OpenClaw image or by documenting a persistent Docker install and credential path.

## Best Possible Solution

Add Docker-specific Claude CLI guidance that covers persistent binary installation, `~/.claude` credential persistence, explicit `claude-cli.command` plus `agentRuntime.id: "claude-cli"` config, onboarding verification, and Anthropic API-key fallback; treat image preinstallation as a separate maintainer product/security decision.

## Repair Work Prompt

Create a narrow docs-only fix for openclaw/openclaw#66874. Add Docker-specific Claude CLI guidance without changing the official Docker image or release workflow. In `docs/install/docker.md`, add a section explaining how Docker users can persist a Claude Code install and credentials across image upgrades: cover a persistent npm prefix or `/home/node` named volume, `~/.claude/.credentials.json` persistence via `/home/node` or an explicit `.claude` bind/named volume, how to verify `claude --version` and `claude auth status --text` inside the container, and an example `agents.defaults.cliBackends.claude-cli.command` plus `agents.defaults.agentRuntime.id: "claude-cli"` config. Cross-link from `docs/providers/anthropic.md` and `docs/gateway/cli-backends.md` to the Docker section. Include Anthropic API-key fallback guidance for long-lived Docker gateways. Do not add `npm install -g @anthropic-ai/claude-code` to `Dockerfile`, do not change `.github/workflows/docker-release.yml`, do not add dependency pins or lockfile changes, and do not edit generated docs. Changelog expectation: no changelog entry for docs-only guidance; if maintainers expand scope to image preinstall, stop for manual review.
