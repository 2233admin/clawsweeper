## Summary

Keep this PR open. Current main still routes leading Markdown image syntax through the bang/bash paths in the TUI, bash parser, command handler, and ACP bypass logic, while the current tests do not cover the reported regression. The PR remains a relevant implementation candidate, but it needs rebase/adaptation to current moved surfaces.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, add a shared narrow Markdown-image predicate used by the TUI submit handler, bash parser/handler, and ACP bypass path, and land regression coverage proving `![alt](url)` is normal text while `![ -f file ]` remains a valid bang bash command.
