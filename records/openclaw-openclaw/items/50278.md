## Summary

Keep this PR open. Current main still routes any non-lone leading `!` line through TUI bang handling, and the channel/ACP bash paths still treat bang-prefixed text as bash without a Markdown-image exception. The PR is not obsolete; the later PR commits also appear to address the review concern about preserving shell-test inputs like `![ -f file ]`.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to review or land the corrected head, preserving the narrower `![...](...)` Markdown-image exclusion across TUI, bash command parsing, command routing, and ACP bypass while keeping documented `! <command>` bash behavior intact.
