## Summary

Current main still does not implement a Control UI/WebChat setting to disable auto-scroll, a top/off scroll mode, or a Scroll to Top action. Browser Control UI has bottom-follow/manual-scroll preservation and a New messages button that jumps down, while native SwiftUI WebChat remains bottom-pinned. Related TUI work and the open browser scroll-interrupt PR do not supersede this broader feature request.

## Best Possible Solution

Keep this issue open. The best path is a focused product/design decision followed by implementation across browser Control UI and native SwiftUI WebChat: add an explicit chat scroll preference and/or Scroll to Top action, persist the local UI choice where appropriate, preserve existing manual-scroll/new-message behavior, and only add gateway config/docs if maintainers want operator policy. Keep TUI-specific work in #44130 and browser scroll-interrupt hardening in #72957 separate unless maintainers intentionally consolidate them.
