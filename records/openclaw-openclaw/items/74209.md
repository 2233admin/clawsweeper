## Summary

Keep open. Current main has useful Bonjour guardrails and startup timeout handling, but it still keeps Bonjour and other default-enabled bundled plugins in the normal startup/runtime-dependency path, while the maintainer comment and later macOS data point identify the unresolved problem as broader default bundled-plugin/minimal-profile policy rather than only the Bonjour watchdog loop.

## What This Changes

The issue requests that upgrades avoid enabling or initializing a large unused bundled-plugin set, and that optional plugins such as Bonjour cannot block gateway startup or normal operation when they are unconfigured or unused.

## Best Possible Solution

Define a maintainable upgrade/startup policy that loads and stages only configured or explicitly enabled operational plugins by default, preserves intentional core exceptions, keeps explicit disables authoritative, and documents a minimal-profile or safe-mode recovery path for operators.

## Reproduction Assessment

Yes for the remaining policy shape: current main statically shows Bonjour as default-on/startup and tests assert default-enabled startup/runtime-dependency behavior. The exact constrained Linux restart churn was not live-reproduced in this read-only review.

## Solution Assessment

No, closing this as already solved would not be the best solution. The current Bonjour timeout and restart caps are useful defensive mitigations, but the broader default bundled-plugin/minimal-profile behavior remains unchanged.
