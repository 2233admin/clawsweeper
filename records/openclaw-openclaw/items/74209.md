## Summary

Keep open. Current main has useful mitigations around Bonjour and bundled runtime-dependency staging, but it still default-enables and startup-loads the Bonjour plugin, and the bundled default-on plugin surface remains broad. The issue's central request has not been implemented on main and needs maintainer policy/compatibility review before a narrow fix is selected.

## Best Possible Solution

Keep this issue open and have maintainers choose the intended default policy: either make Bonjour and other optional bundled services opt-in, or keep them default-on but prove they cannot delay readiness, restart the gateway, or stage unused runtime dependencies. The likely implementation area is the Bonjour manifest/advertiser plus Gateway startup plugin planning and bundled runtime-dependency selection, with regression coverage for a Linux packaged install using a small configured plugin set.
