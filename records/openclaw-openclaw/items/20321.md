## Summary

Keep this issue open. Current main still has no bundled Telegram `groupPolicy: "members"` contract: the public config type, strict schema, generated metadata, docs, inbound group-message path, and native-command authorization still cover only `open`, `disabled`, and `allowlist`. The closed implementation PRs and the `openclaw-chat-only-trigger` community workaround are useful prior art, but they are not a shipped core implementation.

## What This Changes

The issue requests a Telegram `groupPolicy: "members"` mode that lets bots respond in newly created groups only when all current group participants are trusted users.

## Best Possible Solution

Define and land a bundled Telegram `members` policy that updates config types/schema/generated metadata, inbound and native-command authorization, fail-closed Telegram membership verification, security-audit treatment, docs, and focused tests while preserving existing `open`, `disabled`, and `allowlist` behavior.
