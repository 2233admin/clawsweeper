## Summary

Keep open. Current main still reuses the session id as the run id for node `agent.request` events, and the current test suite still asserts that old behavior. The PR’s intended gateway fix is not implemented or superseded on main, and the provided diff is a small targeted runtime/test change with no visible supply-chain or secrets-handling expansion.

## Best Possible Solution

Keep this PR open and review it as a focused gateway bug fix. The best implementation is to generate a fresh `randomUUID()` for each node `agent.request` run in `src/gateway/server-node-events.ts`, keep the existing session id for session continuity, update the colocated test to assert the run id is distinct from the session id, and then run the targeted gateway test plus the normal changed gate before landing.
