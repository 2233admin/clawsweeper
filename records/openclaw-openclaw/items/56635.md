## Summary

Keep open. Current main still omits `--exclude-labels` from the Gmail watcher command, the config schema has no `excludeLabels` escape hatch, and multiple open PRs explicitly reference this issue as implementation candidates.

## Best Possible Solution

Review and land one canonical fix, preferably the configurable version: add `hooks.gmail.excludeLabels` with default `['SPAM','TRASH','DRAFT','SENT']`, wire it through runtime config and overrides into `buildGogWatchServeArgs()`, update config labels/help/docs as needed, and add focused tests for default, custom, and empty label behavior. Keep this issue open until that implementation merges.
