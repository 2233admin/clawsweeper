## Summary

Keep this PR open. Current main and the latest release still reject `messages.groupChat.historyLimit: 0`, while the docs describe `0` as the disable value, and the PR now covers both the Zod validator and generated schema surface with a focused regression test. The diff is narrow and the security pass did not find unrelated code-execution, dependency, workflow, secret, or packaging changes.

## What This Changes

The PR changes `GroupChatSchema.historyLimit` from a positive integer to a nonnegative integer, updates the generated base config schema from `exclusiveMinimum: 0` to `minimum: 0`, and adds boundary tests for zero, positive, and negative values.

## Best Possible Solution

Land this narrow schema fix, including the generated base schema update and boundary test, so both runtime Zod parsing and schema-driven config consumers accept `0` as the documented way to disable group chat history; then close #65305 as implemented.
