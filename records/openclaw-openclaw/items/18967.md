## Summary

Current main still does not provide the requested native parent-scoped send path. It has announce-chain delivery, ACP-only parent streaming, and broad opt-in ways to expose `sessions_send`, but no trusted API that derives the direct parent and enforces parent-only, same-tree, no-chaining behavior.

## What This Changes

Allow sub-agents to send mid-run progress updates or clarifying questions only to their direct parent session through a dedicated parent-scoped `sessions_send` path.

## Best Possible Solution

Keep this issue open for maintainer design. The best implementation is a dedicated `sessions_send_parent` tool or a `sessionKey: "parent"` resolver that derives the direct parent from trusted spawn/session metadata, rejects missing-parent, sibling, ancestor, and cross-agent targets, avoids automatic forwarding beyond that parent, and updates policy, resolver, docs, and regression tests.
