## Summary

Keep this issue open. Current main still returns `queuedFinal: false` for the plugin-bound `inbound_claim` handled path, Telegram still sends silent-reply fallback when `!queuedFinal` and no delivery was recorded, and the existing regression test still asserts the reported false result. The issue is also paired with open unmerged PR #70288, so maintainer policy keeps it open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and update PR #70288 or land an equivalent maintainer patch that gives successful plugin-bound inbound claims a durable “async response committed” signal all the way through `dispatchInboundMessageWithBufferedDispatcher` to Telegram, then update dispatch and wrapper/Telegram regression tests. Treat typing indicators or timeout fallback for claimed-but-never-delivered plugin responses as a separate follow-up contract decision.
