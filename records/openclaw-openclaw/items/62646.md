## Summary

Close as implemented on current main: later shipped QMD changes remove the phantom `memory-alt-main` default path, prefer `--mask`, and add safer stale/missing collection repair coverage. The source branch itself should not merge because its splice-based pruning has blocking correctness and data-boundary regressions.

## What This Changes

The PR changes QMD collection compatibility to prefer `--mask`, mutates `managedCollectionNames` after `ensureCollections()` so searches use only listed existing collections, and adds focused QMD manager/compat tests.

## Best Possible Solution

Keep the shipped current-main approach: a single canonical default root memory collection, mask-first QMD collection creation, and targeted stale/missing collection repair. Any remaining metadata-free QMD conflict should be handled with a narrower repair that preserves scoped collection filters instead of pruning the runtime list from incomplete sidecar metadata.

## Reproduction Assessment

Yes. The original reported path is covered by current tests that assert default QMD config no longer includes `memory-alt-main`, search does not query that phantom collection, and missing managed collections trigger repair plus one retry.

## Solution Assessment

No, the PR branch is not the best way to solve it. Current main's shipped config/repair changes are safer than mutating `managedCollectionNames` from `listCollectionsBestEffort()` output because that output is intentionally incomplete on older or degraded QMD installs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve managed names when listing is unavailable:** `extensions/memory-core/src/memory/qmd-manager.ts:468-470`
  - body: `listCollectionsBestEffort()` intentionally returns an empty map when `qmd collection list --json` is unsupported. Filtering `managedCollectionNames` only through `existing.has(name)` then drops usable collections after an `already exists` add failure, so older QMD installs can skip valid searches as if no managed collections exist.
  - confidence: 0.9
- **[P1] Refresh collection scope before retrying search:** `extensions/memory-core/src/memory/qmd-manager.ts:471-475`
  - body: `search()` captures the collection array before the first attempt, while this patch mutates that same array in place during missing-collection repair. If reconciliation empties it, the retry can run with no `-c` filters and return unmanaged QMD collections instead of failing closed.
  - confidence: 0.86
