## Summary

Keep this PR open. Current main has adjacent gateway timeout/close-message cleanup, but it still does not thread `GatewayClient.onConnectError` into `callGateway` close errors, and the real `GatewayClient` constructor still normalizes explicit `deviceIdentity: null` to `undefined`. The PR also needs revision because its loopback/no-token identity omission can disable cached device-token auth for paired local sessions.

## Best Possible Solution

Keep this PR open for revision or replacement with a narrow updated PR: preserve explicit `deviceIdentity: null` in the real `GatewayClient` constructor, capture `GatewayClient.onConnectError` in `callGateway` and include it in the final close error, and retain current main's backend/shared-token loopback carveout so paired local device-token sessions continue to work.
