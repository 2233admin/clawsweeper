## Summary

Keep open. Current main has adjacent auth-profile rotation/cooldown support, but it still does not implement this PR’s central contract: the successful final auth profile from an embedded fallback rotation is not exposed as `finalAuthProfileId` and is not persisted back to the session `authProfileOverride` after a successful run. The PR also needs rework before merge because its diff includes an XL command-path refactor with a security-sensitive model-override guard regression called out in review.

## Best Possible Solution

Do not close this as implemented. The best path is a narrow rebased fix on current main: carry the resolved final auth profile from the embedded runner into the agent result, persist it to the session `authProfileOverride` after successful fallback rotation, preserve `authProfileOverrideSource: "user"` when appropriate, and add focused regression coverage. The rework should drop the unrelated `src/commands/agent.ts` wholesale refactor and preserve the existing ingress `allowModelOverride` guard.
