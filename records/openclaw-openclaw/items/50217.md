## Summary

Keep this PR open. Current main still does not implement the central behavior: embedded runs do not expose a final winning auth profile id, and the post-run session-store update path has no input or write that would persist a rotated winner back to `authProfileOverride`. The PR remains useful, but it should be reworked narrowly because the current diff also expands `src/commands/agent.ts` and the review discussion flagged a provider/model override authorization regression risk.

## Best Possible Solution

Keep the PR open, but rework it into a narrow auth-profile persistence patch: propagate the resolved final winning profile id from the embedded attempt/run result, pass it into post-run session persistence, write it to `authProfileOverride`, preserve `authProfileOverrideSource: "user"` when the session started user-pinned, and add focused regression tests. Keep the command-module refactor and any `allowModelOverride` authorization changes out of this fix unless reviewed separately.
