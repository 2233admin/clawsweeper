## Summary

Keep this PR open. Current main still exposes the message-tool `asVoice` input, but the shared send path, CLI, gateway protocol, gateway send handler, direct `sendMessage` path, and generated Swift send params do not preserve it as `audioAsVoice`. The provided PR is still a focused implementation candidate for open #73448, not obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #73448. The best path is an additive pass-through from message-tool/CLI `asVoice` to downstream `audioAsVoice` across `handleSendAction`, `executeSendAction`, `sendMessage`, gateway `send`, protocol schema, generated Swift models, docs, and focused regression tests, with no unrelated infrastructure or dependency changes.
