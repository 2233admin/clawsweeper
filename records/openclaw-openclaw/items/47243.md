## Summary

Keep open. Current main still does not implement the requested Control UI session timestamp plus last-message preview, and this PR remains directionally useful for the linked feature request. It should not merge as-is because the patch mixes in an unrelated dependency override and enables preview transcript reads/rendering without the bounds and display normalization expected by the existing backend contract.

## What This Changes

This PR adds `lastMessagePreview` to the Control UI session row type, requests `includeLastMessage` from `sessions.list`, renders a timestamped preview under the session key, and adds a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed Control UI implementation: remove the unrelated package override, request previews only for a bounded/capped session set, propagate `lastMessagePreview` through the UI row/event surface, normalize and truncate displayed preview text, make timestamp behavior explicit, and add focused controller/view coverage. Coordinate with #47120 and #48396 so only one corrected implementation ships.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: This PR adds a new `pnpm.overrides` entry for `punycode` but does not update `pnpm-lock.yaml`. Frozen installs will fail with a lockfile config mismatch, and the dependency-resolution change is unrelated to the session preview UI work, so it should be removed or split into a dedicated dependency PR with the lockfile updated.
  - confidence: 0.94
- **[P2] Gate preview reads behind a bounded list:** `ui/src/ui/controllers/sessions.ts:42`
  - body: `includeLastMessage` performs transcript file I/O per returned session. Because the Control UI omits `limit` when the filter is `0` or all sessions, this change can make every refresh tail-read every transcript in large stores. Only request previews when the result set is bounded or otherwise capped.
  - confidence: 0.9
- **[P2] Clamp preview text before rendering:** `ui/src/ui/views/sessions.ts:456`
  - body: The view renders `lastMessagePreview` directly into the Sessions table. Backend extraction can return a long message body, so a single large final message can expand rows and make the table hard to scan. Normalize whitespace and truncate the preview before rendering, with the full value only in a tooltip if needed.
  - confidence: 0.86
