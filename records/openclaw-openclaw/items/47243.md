## Summary

Keep open. Current main still does not implement the Control UI timestamp plus last-message preview requested by this PR: the backend has an opt-in `includeLastMessage` path, but the web UI does not request, type, or render that field. The PR remains directionally useful for #47120, but it needs revision before merge because the submitted diff includes an unrelated package override and enables preview reads without a bounded/capped session list.

## Best Possible Solution

Keep the PR open as a useful implementation candidate for #47120, but require a narrower revision: remove the unrelated `punycode` override, request `includeLastMessage` only for a bounded/capped session list, add `lastMessagePreview` to the UI row type and session event field handling if needed, render a normalized/truncated preview with a clear timestamp treatment, and add focused controller/view tests for the request parameters and rendered preview.

## Risks / Open Questions

- Merging the PR as posted would mix a package-resolution override into a small UI feature, increasing supply-chain surface and likely breaking frozen installs without a lockfile update.
- Always requesting `includeLastMessage` can trigger transcript tail reads for every listed session when the UI limit is unbounded.
- Rendering transcript previews without normalization or truncation can create oversized table rows and hurt scanability.
