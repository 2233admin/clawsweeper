## Summary

Keep this PR open. Current main still loads internal workspace hooks from only `defaultWorkspaceDir` during Gateway startup, while the PR’s central change is multi-workspace startup loading for configured agent workspaces. The linked bug report is closed, but the main-branch code has not absorbed the behavior this PR proposes.

## Best Possible Solution

Keep this PR open as a useful fix candidate. The best path is a rebase or replacement patch against current main that integrates multi-workspace hook loading in `src/gateway/server-startup-post-attach.ts`, uses `listAgentWorkspaceDirs`, keeps shared hooks loaded once, loads workspace-local hooks once per configured workspace, scopes local handlers to matching event/session/agent workspace context, preserves explicit opt-in and current no-override semantics, isolates bad workspace failures, and adds focused regression coverage for non-default workspace loading and startup events.

- The PR is stale against current main because the startup integration point is now `src/gateway/server-startup-post-attach.ts`, not the file named in the original diff.
