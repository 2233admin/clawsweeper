Current main still does not implement the PR’s intended behavior: there is no `messages.responseFooter`, no inherited `agents.defaults.responseUsage`, `/usage full` is still based on per-response token/cache/cost data plus session key only, and successful block streaming can still return before the usage footer is appended.
