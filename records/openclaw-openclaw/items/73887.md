## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve final TTS when the prepared marker write fails:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If `recordProjectorPreparedSyntheticFinal()` throws here, `run()` retries with a fresh coordinator. The checkpoint already skips the earlier block effects, but the coordinator never rebuilds accumulated block text, so `resolveSyntheticFinalPayload()` returns null and final-mode TTS is silently lost. Persist the resend payload before skipping, or avoid advancing/skipping the block effects until the prepared-final marker is durable.
  - confidence: 0.87
