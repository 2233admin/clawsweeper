## Summary

Keep open. Current main still lacks a Control UI cross-agent relationship/debugging surface: Agents remains a selected-agent panel flow, Sessions remains a flat table, and Gateway exposes useful session metadata but no safe relationship/artifact API joining allowAgents, workspace files, shared-workspace/TASK artifacts, and live status. The latest reporter comment and external openclaw-studio RFC make this a product/API direction question, not a conservative close.

## What This Changes

The issue requests a Control UI cross-agent relationship view for durable multi-agent teams, with delegate/dataflow/sequence edges and follow-up editing, tools, skills, and chat slices.

## Best Possible Solution

Keep this open for maintainer product/API review. The best path is to first scope a read-only MVP relationship view backed by safe Gateway methods for agent config, allowAgents, session parent/child metadata, and explicitly approved workflow artifacts; editing, tool/skill mutation, and per-agent chat should be split into follow-up issues or PRs after the core model is approved.
