## Summary

Keep open. Current main still lacks the requested Control UI cross-agent relationship view, and the latest author comment/prototype reframes the request as a product and architecture decision around independent durable agents rather than only a Sessions scaling issue. This is not a safe stale/duplicate/ClawHub close because #52803 tracks a related but narrower session-ops hierarchy surface, while this issue asks for config, delegation, dataflow, sequence, and live status to be modeled together.

## What This Changes

Add a Control UI Studio-style cross-agent view that joins delegate edges from subagent allowlists, per-agent workspace/config files, shared-workspace/TASK dataflow and sequence, and live session status, with later editing/chat slices.

## Best Possible Solution

Keep this open for maintainer product review. If maintainers want multi-agent Studio in core, the best first implementation is a read-only, explicitly allowlisted topology contract that reuses existing session metadata and agent-file APIs where possible, with config mutation, symlink mutation, tools/skills editing, and chat split into later issues after API/security review.

## Reproduction Assessment

Not applicable as a bug reproduction. The missing feature is verified by source/docs inspection: current Control UI surfaces agents and sessions separately but has no route, panel, Gateway method, or docs for a cross-agent topology view.

## Solution Assessment

Unclear at this stage. The full proposed five-slice implementation is too broad to call the best fix before maintainers decide whether this belongs in core Control UI, but a read-only topology MVP would be the narrowest maintainable starting point if the direction is accepted.
