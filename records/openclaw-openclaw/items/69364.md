## Summary

Current main still does not implement the requested Control UI cross-agent relationship view. The UI has selected-agent panels, core workspace-file editing, tools/skills controls, and a flat Sessions table backed by Gateway subagent metadata, but no graph/orchestration tab that joins allowAgents/subagent delegation, per-agent config, shared-workspace/TASK artifacts, and live status. The request overlaps open umbrella #52803 but is more concrete and should remain open for maintainer product/API direction.

## Best Possible Solution

Keep this open for maintainer product review. The best path is to define a read-only MVP first: a Control UI relationship view backed by safe Gateway methods for agent config, subagent/session parent-child metadata, and only explicitly approved shared-workspace/TASK artifacts. Editing, tools/skills mutation, and per-agent chat should be separate follow-up slices with API and security review, and #52803 should link to this as the concrete graph/config-debugging proposal if maintainers want that structure.
