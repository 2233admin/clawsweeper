## Summary

Keep open. Current main still has the subagent-output race/leak path, and this contributor PR is an active implementation candidate, but the patch has concrete correctness defects and overlaps with #73783 and #74340, so it should not be closed or merged without maintainer or author revision.

## What This Changes

The PR imports `readLatestAssistantReply` into `src/agents/subagent-announce.ts` and adds a best-effort one-shot latest-assistant-reply check before announce status construction.

## Best Possible Solution

Land one canonical subagent announce/output fix that performs a bounded, normalized post-wait refresh before constructing completion events, preserves `ANNOUNCE_SKIP` and `NO_REPLY` semantics, covers child-findings and no-findings paths including frozen-result capture, and adds focused regression tests for the auto-compaction raw-tool-output race.

## Reproduction Assessment

Yes. #74073 gives a concrete live reproduction using Grok/DeepSeek with large Tavily `web_search` output, and the source path is clear: current main can select `latestRawText` before a later assistant reply is available.

## Solution Assessment

No. The PR targets the right layer, but the one-shot raw latest-assistant read skips existing normalization, does not update the `childCompletionFindings` value that feeds `findings`, and does not cover the broader capture path discussed in #73783.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize refreshed assistant replies before assigning:** `src/agents/subagent-announce.ts:464-471`
  - body: At this point `reply` has already gone through `stripAndClassifyReply`, but `latestReply` is raw assistant text from history. If the refreshed text contains a leading or embedded `NO_REPLY` plus real content, this block writes the raw token back into the announcement because the guards only catch pure silent replies.
  - confidence: 0.86
- **[P2] Refresh the value that actually feeds findings:** `src/agents/subagent-announce.ts:459-475`
  - body: The retry also runs when `childCompletionFindings` is set, but `findings` is still computed as `childCompletionFindings || reply || "(no output)"` below. Any refreshed `reply` is discarded in that path, so the PR does not actually unify the child-findings branch.
  - confidence: 0.78
