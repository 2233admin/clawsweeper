## Summary

Keep open. Current main still has the linked subagent announce race shape, and this PR is relevant but not correct as-is: the new one-shot refresh can reintroduce raw silent tokens and is ignored when child completion findings win output selection. Related PR #74340 overlaps the same leak area, so maintainers should choose or revise one canonical fix path.

## What This Changes

The PR imports `readLatestAssistantReply` and adds a best-effort one-shot latest assistant reply check in `runSubagentAnnounceFlow` before announcement status construction.

## Best Possible Solution

Land one canonical subagent announce/output fix that performs a bounded, normalized post-wait refresh before constructing completion events, preserves existing `ANNOUNCE_SKIP` and `NO_REPLY` semantics, covers both child-findings and no-findings paths, and adds regression tests for raw tool JSON followed by delayed post-compaction assistant output.

## Reproduction Assessment

Yes. The code path is reproducible at source level: current main can select `latestRawText` in `readSubagentOutput` before finalizing `findings`, and #74073 supplies the live Grok/DeepSeek plus Tavily scenario that triggers the race.

## Solution Assessment

No. The PR targets the right layer, but the one-shot raw latest-assistant read skips existing normalization, does not affect `childCompletionFindings`, and does not poll for delayed post-compaction output; a bounded normalized refresh with tests is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize refreshed assistant replies before assigning:** `src/agents/subagent-announce.ts:464-471`
  - body: At this point `reply` has already gone through `stripAndClassifyReply`, but `latestReply` is raw assistant text from history. If the latest assistant reply contains a leading or embedded `NO_REPLY` plus real text, this block sees it as different and writes the raw token back into the announcement because the guards only catch pure silent replies. Run the refreshed text through the same normalization before assigning it.
  - confidence: 0.86
- **[P2] Refresh the value that actually feeds findings:** `src/agents/subagent-announce.ts:459-475`
  - body: The new retry also runs when `childCompletionFindings` is set, but `findings` is still computed as `childCompletionFindings || reply || "(no output)"` below. Any refreshed `reply` is discarded in that path, so the PR does not actually unify the child-findings branch. Either refresh the findings value too or limit this retry to the no-findings path.
  - confidence: 0.78
