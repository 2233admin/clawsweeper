## Summary

Current main still only partially addresses the requested edit/read stability work. OpenClaw adds mismatch hints and post-write recovery around the upstream edit tool, and the pinned upstream package has a narrow fuzzy match path, but the broad whitespace-collapsing fallback, explicit preflight validation contract, read line/keyword positioning, and small-edit guidance remain unimplemented. The related open PR #67276 is scoped to clearer mismatch warnings for #66915 and explicitly does not change edit semantics.

## Best Possible Solution

Keep this open as a broader edit/read stability enhancement. The best path is maintainer product/API review, then either upstream or wrapper implementation of whitespace-tolerant matching with uniqueness and offset-mapping safeguards, focused tests for ambiguous normalized matches, optional read line/keyword positioning if accepted, and public docs for any recommended small-edit workflow. Treat #67276/#66915 as a narrower warning-surfacing track, not the full fix.
