## Summary

Current main still only partially addresses the requested edit/read stability improvements. OpenClaw wraps the upstream edit tool with mismatch hints and post-write recovery, and the pinned upstream tool normalizes line endings plus a narrow fuzzy match set, but the requested broad whitespace-collapsing fallback, explicit preflight validation contract, read line/keyword positioning, and 1-3 line edit guidance are still not implemented. A related open PR, #67276, targets the narrower user-facing mismatch warning tracked by #66915 and does not close this broader feature request.

## Best Possible Solution

Keep this issue open as the broader edit/read stability enhancement. The best path is to define the product contract for whitespace-tolerant edit matching, implement it upstream or in the OpenClaw wrapper with uniqueness and offset-mapping safeguards, add focused tests for ambiguous whitespace matches plus read line/keyword positioning if accepted, and update public tool docs if recommended edit workflow changes. Review #67276/#66915 as a narrower UI-warning improvement, not as the full fix for this request.
