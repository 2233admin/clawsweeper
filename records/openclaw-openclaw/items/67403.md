## Summary

Keep this PR open. Current main still requires contiguous full-query matches in the memory-wiki local search path, so the token-overlap behavior proposed here has not shipped. The closed bug #64453 now points to this PR as the canonical implementation path, and the earlier same-scope PR #65569 is closed unmerged.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a refined memory-wiki change that replaces full-string wiki candidate gates with normalized token-overlap matching in extensions/memory-wiki/src/query.ts, preserves exact-phrase ranking, adds the focused regression tests already proposed here, and resolves or consciously accepts the digest fallback and threshold review comments.
