## Summary

Keep this PR open. Current main still requires the full normalized query string to appear contiguously in memory-wiki digest and full-page search paths, and the token-overlap behavior proposed by the PR is not implemented or superseded. The linked bug is closed as superseded by this PR, and the earlier same-scope PR is closed unmerged, leaving this PR as the active implementation candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a refined memory-wiki fix that adds normalized token-overlap matching across digest and full-scan paths with focused regression tests, while addressing or explicitly accepting the digest fallback and minimum-threshold review concerns.

- The PR uses a fixed two-token minimum for all multi-term queries, which may over-admit stop-word-heavy false positives for longer natural-language queries unless refined or explicitly accepted.
- The PR base is older than current main and should be rebased or refreshed before merge.
