Keep this PR open. Current main still uses full normalized query-string matching in the memory-wiki search paths, so the token-overlap behavior has not shipped. The PR remains a plausible implementation candidate, but its latest diff now also includes unrelated browser, Signal installer, and channel contract helper changes that need maintainer scope/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or justify the unrelated browser, Signal, and channel test-helper edits, then land this memory-wiki token-overlap fix or an equivalent patch that replaces full-string-only candidate gates, preserves exact phrase priority, continues body fallback scanning, rejects zero-token matches, and ships the focused regression tests.
