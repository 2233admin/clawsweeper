## Summary

Keep open. Current main still has the reported Control UI render-state gap: terminal chat events clear the live text/run fields but not the UI-only tool artifacts, and the renderer appends prior stream segments plus the full current stream. This PR is an active contributor implementation for the linked WebChat report #72341, so the appropriate next step is maintainer review and validation rather than closure or a separate repair lane.

## What This Changes

The PR updates Control UI chat rendering and controller logic to clear live tool/text artifacts on terminal chat events, trim cumulative live stream prefixes, and add regression coverage in chat controller and chat item tests.

## Best Possible Solution

Land a reviewed Control UI fix that removes stale UI-only live tool/text artifacts when owned chat runs terminate, preserves `toolStreamOrder` long enough for gateway history reload decisions, and handles cumulative live deltas with focused regression coverage for final, aborted, error, and renderer trimming paths.
