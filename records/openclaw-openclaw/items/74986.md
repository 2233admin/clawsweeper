## Summary

Keep open. ## What This Changes

The issue reports that `openclaw infer model list`, `openclaw infer model run`, and `openclaw agents list` can hang before network I/O while the `openclaw-infer` child process spins at 100% CPU on OpenClaw 2026.4.27.

## Best Possible Solution

Review and land or revise PR #75022 if the metadata-only infer catalog path is correct, then retest the reporter’s full command matrix on the affected environment. Any remaining `infer model run` or all-agent spin should be split into a focused follow-up with stack/profile evidence around provider-auth/runtime loading or shared initialization.
