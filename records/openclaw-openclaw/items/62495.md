## Summary

Keep open. Current main still emits a warn-level console message for escaped workspace skill paths, and the existing regression test asserts that warning. The linked PR #62518 addressed the same noise but is closed unmerged, so the requested cleanup has not landed; the only uncertainty is that the report describes a Markdown-link trigger while current code evidence points to symlink/SKILL.md realpath validation.

## What This Changes

The issue requests eliminating noisy startup warnings for intentional sibling skill references from workspace skill documentation.

## Best Possible Solution

Preserve the outside-root containment behavior, but make benign intentional sibling skill references quiet during normal startup and keep explicit security/audit visibility for actual workspace skill escapes.

## Reproduction Assessment

Unclear as written: current source and tests give a high-confidence reproduction for symlinked workspace skill directories or SKILL.md files that resolve outside the skill root, but I did not find evidence that pure Markdown body links are parsed as skill-path inputs.

## Solution Assessment

Unclear for the exact Markdown-link framing. The maintainable direction is to avoid loading outside-root skills, reduce duplicate startup noise only for the benign/known case, and add regression coverage so the security containment remains observable.
