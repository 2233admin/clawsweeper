## Summary

Keep open. Current main has adjacent pieces: explicit `/new` and `/reset`, automatic/manual compaction, and a Control UI high-context notice with a Compact action. It does not implement the requested topic-change detector, smart new-session reminder, or automatic session switch at 95% context usage. Related #51485 is a separate Telegram topic-labeling request, not a canonical tracker for this behavior.

## Best Possible Solution

Keep this open for maintainer product review. A good implementation would likely be opt-in, reuse fresh token snapshots and existing reset/session APIs, start with a non-destructive reminder rather than silent auto-switching, and define clear UI/channel behavior plus safeguards for false-positive topic changes.
