## Summary

Keep open. Current main still does not include the proposed Trivy/KICS/TruffleHog workflow template or TruffleHog staged-content pre-commit guide. Existing OpenClaw security docs and CI cover related maintainer checks with detect-secrets, private-key detection, zizmor, CodeQL, and dependency audit, but they are not equivalent user-copyable scanner templates. The PR also remains security-sensitive because the current patch still has a staged-path handling bug and an unpinned action reference in a security workflow template.

## Best Possible Solution

Keep this PR open for secops/docs maintainer review. If this template belongs in OpenClaw docs, update it to stream NUL-delimited staged paths directly, pin every GitHub Action reference to an immutable SHA, add docs front matter/navigation for the user-facing guide, and align the wording with the existing detect-secrets, zizmor, dependency-audit, and OpenClaw security audit docs. If generic third-party scanner templates are out of scope for core docs, close it manually with that product rationale.
