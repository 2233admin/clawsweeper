## Summary

Keep open. The report is recent, concrete, and targets the current native macOS SwiftUI chat stack rather than the older scrolling crash that was already closed. Current main still embeds `OpenClawChatView` in the macOS companion app and the chat list still uses SwiftUI `ScrollView`/`LazyVStack` plus `scrollPosition` bottom-pinning and animated updates without an active scrollbar-drag guard. I found no source, docs, tests, merged PR, or release-diff evidence that scrollbar-thumb dragging has been fixed.

## What This Changes

The issue requests that dragging the macOS companion app chat scrollbar thumb through chat history should not freeze the app UI.

## Best Possible Solution

Keep this open and investigate on a real macOS companion app build with a long chat history. The likely fix belongs around the native SwiftUI chat scroll path: avoid fighting user-driven scrollbar interaction by gating or deferring bottom-pinning while the user is actively scrolling/dragging, or replace the macOS message list with an `NSScrollView`-backed implementation if SwiftUI `scrollPosition` remains unstable. Add focused regression coverage where feasible and require manual scrollbar-thumb verification before closing.
