## Summary

Keep this PR open for maintainer review. Current main has addressed the old two-minute pain point by raising the shared exec approval default to 30 minutes, but it still does not expose or wire a public configurable approval-timeout key, so the PR’s central feature is not implemented on main. The branch is also touching a security-sensitive approval window and has an unresolved product/API choice around the public config surface, already reflected by prior maintainer review and the still-open #51287 follow-up.

## What This Changes

This PR proposes `tools.exec.approvalTimeoutMs` and threads it through exec approval request, gateway/node host execution, CLI invoke defaults, config schema, labels, help text, and TypeScript types.

## Best Possible Solution

Choose one supported, bounded public timeout surface, document it against the current 30-minute default, thread the resolved value through agent, gateway, node, and transport waits, and cover propagation plus expiry behavior with focused tests. If maintainers prefer the #51287 shape instead, explicitly supersede this PR rather than closing it as already implemented.
