## Summary

Current main still does not implement a user-facing configurable exec approval timeout. The PR targets a real remaining request and is linked to the still-open canonical feature issue #51287, but it needs maintainer review/rebase before it can land: current main has no `tools.exec.approvalTimeoutMs` or `approvals.timeoutMs` schema surface, the runtime still sends the shared fixed timeout, and the PR’s proposed docs/default/bounds are stale against the current 30-minute default.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly supersede it under #51287 if maintainers choose a different public key. A landable fix should pick one supported config surface, use bounded validation, align help/docs with the current 30-minute default, thread the resolved timeout through agent, gateway, node, and transport waits, and add focused propagation and expiry tests.
