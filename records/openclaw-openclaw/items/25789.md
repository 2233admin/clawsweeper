## Summary

Keep open. Current main still does not provide a user-facing configurable exec approval timeout: neither `tools.exec.approvalTimeoutMs` nor `approvals.timeoutMs` exists in the typed config or strict schemas, and the active approval paths still use the fixed 30-minute default. This PR remains a plausible implementation candidate for the open feature request #51287, but it needs maintainer review for config shape, default/docs alignment, bounds, propagation tests, and the security-sensitive timeout behavior before merge.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it under #51287 if maintainers choose a different implementation. A complete fix should choose the public config shape, add bounded validation and generated schema/help/labels/docs, thread the resolved timeout through gateway and node exec approval registration plus transport waits, align the documented default with current main, and add focused propagation tests.
