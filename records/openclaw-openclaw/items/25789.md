## Summary

Keep open. Current main still lacks a user-facing configurable exec approval timeout: there is no `tools.exec.approvalTimeoutMs` or `approvals.timeoutMs` config surface, and the exec approval path still uses the fixed 30-minute default plus an internal protocol `timeoutMs`. The PR is still a plausible implementation candidate, but it needs maintainer review, a rebase onto the current exec/node paths, bounded validation, docs/default alignment, and tests.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it under #51287. A complete fix should choose the public config shape, align help/docs with the current 30-minute default, add bounded validation, thread the resolved timeout through exec defaults, config resolution, gateway/node approval registration, and transport waits, then add focused tests for parsing and end-to-end propagation.
