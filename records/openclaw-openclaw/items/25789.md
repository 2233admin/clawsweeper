## Summary

Keep open. Current main still lacks a user-facing configurable exec approval timeout: neither `tools.exec.approvalTimeoutMs` nor `approvals.timeoutMs` exists, while the approval path still uses a fixed 30-minute default plus an internal protocol `timeoutMs`. The PR remains a plausible implementation candidate after its follow-up commit, but it needs maintainer review for config shape, default/docs alignment, bounds, and tests; #51287 is the open canonical feature request and #51292 was closed unmerged.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it under #51287. A complete fix should choose the public config shape, add bounded validation and generated schema/help/labels/docs, thread the resolved timeout through gateway and node exec approval registration plus transport waits, align the documented default with current main, and add focused propagation tests.
