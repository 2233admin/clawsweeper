## Risks / Open Questions

- Exec approval timeout is security-sensitive: increasing how long host-exec approvals remain pending changes the window in which an operator action can authorize host execution.
- The PR schema accepts any positive integer without an upper bound, which could leave host-exec approvals pending unexpectedly long if the key lands as-is.
- The PR help/type text documents a 120000 ms default, while current main uses and documents a 1800000 ms / 30-minute default.
- The public config surface is unresolved: this PR proposes `tools.exec.approvalTimeoutMs`, while related #51287/#51292 discuss an `approvals.timeoutMs` style key.
- The provided PR file list has no focused tests for schema acceptance, bounds, config precedence, agent/gateway/node propagation, expiry behavior, or transport-timeout alignment.
