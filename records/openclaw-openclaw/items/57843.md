## Summary

Keep #57843 open. Current main still returns a bare OutboundDeliveryResult[] from deliverOutboundPayloads, hook cancellations still collapse to empty results with no cancellation metadata, and cron/gateway callers still treat empty arrays or thrown errors as ordinary delivery failures. The PR is a focused implementation of the still-open #57766 and unblocks the related #53961/#57755 work, so it is not obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused delivery contract change that returns results plus hook-cancellation metadata, preserves partial-send information through DeliveryError or equivalent, updates all value-consuming callers, and keeps the cron lazy-loading/test-performance constraints addressed in the latest PR discussion.
