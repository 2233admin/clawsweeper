## Summary

Keep open. Current main still has the Gateway live-chat media gap: assistant events can carry generated-image media, and the non-agent chat completion/history path already knows how to project media, but the agent-run `chat` delta/final bridge still emits only buffered text.

## What This Changes

The issue requests that Gateway WebSocket `chat` events include generated-image information for image-generation replies instead of only assistant text.

## Best Possible Solution

Add a media-aware projection for agent-run Gateway `chat` final events when assistant stream/final data includes generated-image media. Reuse the existing managed outgoing image and webchat media block helpers, keep the behavior generic to assistant media rather than MiniMax-specific, and avoid broadcasting raw unsafe media sources.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat run, emit an assistant event with `data.text` plus `data.mediaUrls` through `createAgentEventHandler`, then emit lifecycle end; current `chat` delta/final payloads contain only text content.

## Solution Assessment

Yes. The narrow maintainable fix is in the Gateway live-event projection path because upstream assistant events already preserve `mediaUrls` and the Gateway already has media display helpers for completion/history paths.
