## Summary

Keep open. Current main still does not carry generated image/media information through the live Gateway `chat` WebSocket event path for agent-run replies; media is projected into `chat.history`, but the reported live `chat.send` event surface remains text-buffer based.

## Best Possible Solution

Keep this open and clarify the Gateway WebSocket media contract. If live `chat` events should carry generated images, add an additive media-bearing projection for agent-run final chat events and cover `image_generate`/`mediaUrls` through `chat.send`, asserting both live final events and `chat.history` expose usable image information without leaking raw base64. If the intended contract is history-only, document that clients must refresh `chat.history` after final events.
