## Summary

Keep this issue open. Current main still returns a hard `status: "timeout"` from `sessions_send` as soon as the nonzero-timeout `agent.wait` call times out, and the reporter’s open implementation PR #68196 still uses closing syntax for the same bug.

## What This Changes

The issue asks `sessions_send` to distinguish wait-window expiry from delivery failure by returning a late reply or accepted delivery when the target agent run was accepted.

## Best Possible Solution

Implement the fix in the `sessions_send` and `run-wait` boundary: after a successful `agent` RPC start, handle wait-window expiry by checking for a new assistant reply since the pre-wait baseline, returning `ok` for a late reply, returning `accepted` only with a real acceptance signal, and preserving hard timeout when delivery cannot be established.
