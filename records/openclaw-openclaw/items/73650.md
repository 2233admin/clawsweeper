## Summary

Keep this PR open for normal maintainer review. Current main still performs the duplicated status-task reconciliation that the PR targets, and the proposed patch is a narrow, backwards-compatible implementation with focused regression coverage and no blocking review findings found in the read-only pass.

## What This Changes

The PR changes `getStatusSummary()` to reconcile inspectable tasks once, passes the shared snapshot to the registry and audit summaries, adds regression tests, and records the performance fix in the changelog.

## Best Possible Solution

Land this shared-snapshot implementation, or an equivalent narrow main-branch change, so `getStatusSummary()` performs one inspectable-task reconciliation per status invocation while keeping existing zero-arg helper callers unchanged. Keep the linked performance issue open until the fix merges.

## Reproduction Assessment

Yes. The duplicated call path is reproducible by source inspection on current main: status calls both helpers, and each helper independently calls `reconcileInspectableTasks()`, which in turn runs the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

Yes. Sharing one reconciled snapshot in the status path is narrower and lower risk than introducing a module-level cache with invalidation, and the optional helper parameter preserves existing callers while testing both the new fast path and fallback behavior.
