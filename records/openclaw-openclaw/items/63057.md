## What This Changes

The PR updates one Anthropic transport source file so legacy non-adaptive thinking payloads use a nullish `thinkingBudgetTokens` fallback and omit `params.thinking` when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that only emits legacy `thinking: { type: "enabled", budget_tokens }` when the resolved budget is valid for Anthropic, preserves the undefined-budget 1024 fallback, omits thinking when no valid budget can be formed, and covers the zero, default, and sub-minimum paths with targeted transport tests.
