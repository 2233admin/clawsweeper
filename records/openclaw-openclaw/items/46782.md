## Summary

Current main still has `openclaw logs` local-time rendering as opt-in, so this PR’s intended behavior has not shipped and it is not closeable as implemented. The PR also is not ready as submitted: it only flips the positive `--local-time` default, does not add a real UTC opt-out or docs/tests, and mixes in an unrelated skill-path warning downgrade that would reduce security observability.

## Best Possible Solution

Keep this PR open for maintainer judgment or replace it with a cleaner focused fix for #46748. The implementation should live in `src/cli/logs-cli.ts`, default human-readable `openclaw logs` and `--follow` output to local timestamps, register and test a real UTC opt-out such as `--no-local-time`, preserve JSON/raw behavior, update `docs/cli/logs.md`, and remove the unrelated `src/agents/skills/workspace.ts` logging downgrade from the logs fix.
