## Summary

Keep this PR open, but it is not merge-ready: the logs timezone request is valid and still tracked by #46748, but this patch does not actually provide the advertised UTC opt-out and it mixes in an unrelated security-observability downgrade for skill path escapes. There is also an overlapping open logs PR, #47365, so maintainers should choose one focused direction rather than land both.

## What This Changes

The PR changes `openclaw logs` so the existing `--local-time` option defaults to `true`, and changes escaped skill-path logging from `warn` to `debug`.

## Best Possible Solution

Land a focused logs-only change that defaults human-readable log output to local time, provides an explicit UTC mode that cannot be overridden by `TZ`, updates tests and `docs/cli/logs.md`, and leaves escaped skill-path warning behavior unchanged unless that security trade-off is reviewed separately.

## Reproduction Assessment

Yes. The source-level reproduction is clear: current `main` has UTC default tests and an opt-in `--local-time`; this PR only flips the default and does not register the promised `--no-local-time` opt-out.

## Solution Assessment

No. Defaulting logs to local time is a reasonable product fix, but this patch is not the best implementation because it lacks the advertised UTC escape hatch, omits docs/tests, and includes an unrelated security-sensitive logging change.
