## Summary

Keep open: current main still treats local log timestamps as opt-in, and this PR is not merge-ready because it promises a UTC escape hatch that the diff does not register and mixes in an unrelated security-observable skill-path logging downgrade. The related user problem remains valid, but the fix should be narrowed and coordinated with the competing open PR for the same issue.

## What This Changes

The PR changes the `openclaw logs` `--local-time` option default from `false` to `true` and lowers escaped skill-path logging from `warn` to `debug`.

## Best Possible Solution

Land or replace this with a focused logs-only change that makes human-readable `openclaw logs` and `openclaw logs --follow` use local timestamps by default, preserves JSON/raw output semantics, provides a tested UTC opt-out, updates `docs/cli/logs.md` and `docs/logging.md`, and leaves escaped skill-path warning behavior unchanged unless a separate owner-reviewed security/observability change narrows the noisy symlink case.
