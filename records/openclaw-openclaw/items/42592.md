## Summary

Current main still does not implement the requested structured installer error handling. The shell installer extracts some npm diagnostics and has broad retry behavior, but it does not classify known npm failures, retry only transient failures, provide code-specific remediation, mirror the policy in PowerShell, or make the Docker allowedOrigins config-read failure explicit.

## Best Possible Solution

Keep this open and implement a focused installer reliability PR: add tested npm error-code extraction/classification, retry only known transient npm/network failures with a bounded budget, print actionable remediation for terminal npm errors, make Docker `gateway.controlUi.allowedOrigins` config-read failures explicit while preserving first-run defaults, and mirror reasonable behavior in `scripts/install.ps1`.

Implement the installer reliability improvement requested by this issue.
