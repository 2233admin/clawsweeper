## Summary

Keep open. Current main still only has partial installer diagnostics and broad retry behavior; it does not implement the requested structured npm error classification, retryable-only policy, code-specific remediation, Windows parity, or explicit Docker config-read failure handling.

## Best Possible Solution

Implement a scoped installer reliability change in the installer-owned surfaces: add tested npm error-code extraction/classification, retry only known transient npm/network failures with a bounded budget, print actionable remediation for terminal npm errors, make Docker gateway.controlUi.allowedOrigins config-get failures explicit without breaking first-run defaults, and mirror reasonable behavior in scripts/install.ps1.
