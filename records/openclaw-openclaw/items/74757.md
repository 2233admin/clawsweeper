## Summary

Keep open. Current main has partial Slack action and user-token plumbing, but the reported gaps remain: the ACP OpenClaw-tools MCP bridge only exposes `cron`, the Slack skill still describes a `slack` tool that is not backed by an MCP tool, and Slack admin channel/app operations are not implemented in the Slack action/config surfaces.

## What This Changes

Expose Slack action tooling to Claude Code ToolSearch/MCP sessions and add gated Slack admin actions for channel, app, invite, member lookup, and archive workflows backed by user/app-configuration tokens.

## Best Possible Solution

Define the canonical Claude Code exposure first, likely by making the general `message`/Slack action surface discoverable through the ACP MCP bridge or by documenting a supported Slack-specific alias, then add a separately gated Slack admin action group with explicit token semantics and docs.
