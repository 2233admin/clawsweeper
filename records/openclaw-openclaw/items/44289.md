## Summary

Keep open. Current main still only partially satisfies the request: the JSON credential matrix is derived from secret target registry metadata, but the Markdown SecretRef credential surface remains a checked-in bullet list that is parity-tested after the fact rather than generated from the same source.

## Best Possible Solution

Keep this issue open. Extend the existing SecretRef credential matrix builder/generator so it updates both the JSON matrix and the supported/unsupported Markdown blocks, then add an explicit package script/check pair while retaining the parity test as a guard.
