## Summary

Current main still has the Control UI bootstrap path composition that can produce the reported duplicated `__openclaw__` config URL. The related fix PRs were closed unmerged, so this remains a concrete behavior bug rather than a cleanup close candidate.

## Best Possible Solution

Keep this issue open. The best fix is a focused core Control UI change that makes the bootstrap config endpoint basePath-relative, updates the gateway contract, frontend bootstrap loader, Vite dev stub, gateway/UI tests, and public Control UI docs, with explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"` and a short-lived compatibility route if maintainers want upgrade tolerance.

## Evidence
