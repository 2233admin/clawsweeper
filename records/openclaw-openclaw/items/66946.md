## Repair Work Prompt

Fix the Control UI bootstrap config duplicate path bug reported here. Current main still has `CONTROL_UI_BOOTSTRAP_CONFIG_PATH = "/__openclaw/control-ui-config.json"` and the UI fetches `${basePath}${CONTROL_UI_BOOTSTRAP_CONFIG_PATH}`, which can produce `/__openclaw__/__openclaw/control-ui-config.json` when the UI is opened at `/__openclaw__/`. Make the endpoint consistently base-path-relative, likely by changing the shared contract to `/control-ui-config.json` or an equivalent helper, and update the gateway handler, frontend bootstrap loader expectations, Vite dev stub, tests, and public docs together. Add explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"` and consider a short compatibility route for the old computed endpoint to avoid cached bundle upgrade failures. Include a user-facing changelog fix entry crediting the reporter `@yndwx01` and, if referencing the prior unmerged attempts, `@nightq`; do not touch unrelated media/canvas routes, auth policy, release metadata, or dependency files.
