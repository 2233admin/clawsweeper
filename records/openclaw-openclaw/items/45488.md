## Summary

Keep open. Current main still implements the reported default behavior: post-compaction code reads user-editable AGENTS.md, extracts configured/default magic headings, injects the excerpt as a system event for the next turn, and also appends extracted AGENTS.md sections into the compaction summary suffix. A config key now exists to customize or disable the post-compaction event injection, but the default remains documented and tested as Session Startup/Red Lines with legacy Every Session/Safety fallback, so the central bloat/magic-string concern is not resolved by current main.

## Best Possible Solution

Keep this issue open and decide the intended product boundary for post-compaction recovery. A conservative fix would make AGENTS.md section reinjection explicitly opt-in, or skip both summary-suffix and system-event reinjection when normal bootstrap Project Context is retained, while preserving a documented config path for deployments that still want post-compaction refresh behavior. Update docs and tests to match the chosen default.
