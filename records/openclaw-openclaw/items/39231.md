## Summary

Keep this PR open. Current main still does not implement the proposed opt-in CaMeL engine, taint-propagating planner/quarantined extractor, config/schema surface, before-tool-call CaMeL policy enforcement, or exec-approval source routing. The PR is a broad security-sensitive architecture change, and the related RFC/follow-up issues do not supersede it.

## What This Changes

This PR adds a new opt-in CaMeL prompt-injection defense engine under `src/agents/camel`, wires it into agent execution, before-tool-call policy, config schemas, gateway exec approvals, and SECURITY.md, and adds focused CaMeL tests.

## Best Possible Solution

Keep this PR open for explicit maintainer/security product judgment. If OpenClaw wants CaMeL in core, the best path is a reviewed design that proves isolated planner and quarantined extractor sessions, durable taint provenance, replay-safe side-effect handling, clear config semantics, gateway approval compatibility, docs, and focused regression tests. If maintainers prefer another structural defense, close only after linking a canonical replacement that covers the same goal.
