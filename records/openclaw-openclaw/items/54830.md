## Summary

Keep this PR open. Current main still lacks context provenance metadata, so the PR is not obsolete, but the patch still has a prompt-integrity bug and needs maintainer product/API review against the open provenance RFC before merge.

## What This Changes

The PR extends embedded context files with optional provenance metadata, tags bootstrap context entries, renders provenance comments and guidance in the agent system prompt, and adds focused tests.

## Best Possible Solution

Rebase the provenance design onto the current project-context helper and cache-boundary layout, validate all rendered provenance fields at runtime, and keep #54373 as the product/design anchor until Phase 1 is accepted or declined.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Coerce volatile before rendering provenance tags:** `src/agents/system-prompt.ts:628`
  - body: The renderer escapes `source` and `injectedAt`, but still interpolates `file.provenance.volatile` directly into an HTML comment. `buildAgentSystemPrompt` receives runtime `contextFiles` and TypeScript types are erased, so a malformed non-boolean value can close the comment and inject unintended prompt text. Normalize it to a strict boolean string before building the tag.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces a prompt-rendered metadata surface with one remaining runtime validation gap.

Concerns:

- **[medium] Unvalidated volatile value can break provenance comment:** `src/agents/system-prompt.ts:628`
  - body: `volatile` is embedded directly in the provenance HTML comment while the other fields are escaped. Runtime callers can still supply malformed data despite the TypeScript type, so the renderer should coerce or validate it before interpolation.
  - confidence: 0.78
