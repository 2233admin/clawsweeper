## Summary

Keep this PR open. Current main still does not implement the central Feishu behavior: explicit `/allowlist channel=feishu` config edits/listing are not wired through plugin-channel normalization plus a Feishu allowlist adapter, Feishu is not marked markdown-capable, and Feishu envelope headers are still not recognized. Some parts of the branch are now superseded on main, and the PR also carries unresolved allowlist-account review concerns plus an unrelated CI/Bun runtime bump, so it should be narrowed rather than closed or merged as-is.

## Best Possible Solution

Keep this PR open for a narrower rework: add a Feishu-owned `allowlist` adapter in `extensions/feishu/src/channel.ts` using the existing plugin SDK allowlist helper and effective merged Feishu account config, set Feishu `markdownCapable` metadata in both runtime and package discovery surfaces, add the Feishu envelope label if current inbound envelopes still use `[Feishu ...]`, add focused regression tests, drop the hook-schema/security-fix chunks already superseded on main, and remove or split the unrelated Bun/CI runtime bump before maintainer review.
