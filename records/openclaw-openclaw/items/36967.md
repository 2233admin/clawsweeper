## Summary

Keep this PR open. Current main still does not implement the central Feishu `/allowlist channel=feishu` config-edit/listing support or mark Feishu as markdown-capable. Some peripheral pieces are now handled generically, but the branch also has unresolved Feishu multi-account review concerns and an unrelated CI runtime version bump, so the best path is a narrower rework rather than closure or merge as-is.

## Best Possible Solution

Keep this PR open for a narrower rework: add a Feishu-owned `allowlist` adapter using the existing plugin SDK surface while preserving effective merged account allowlists, set Feishu `markdownCapable` metadata in both runtime and package discovery surfaces, add focused Feishu `/allowlist channel=feishu` and markdown-capability regression tests, drop already-superseded hook-schema/security-fix chunks, and remove or split the unrelated Bun/CI runtime bump before maintainer review.
