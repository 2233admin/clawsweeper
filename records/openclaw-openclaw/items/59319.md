## Summary

Keep open. Current main still has the duplicate-bootstrap-entry gap: hook-adjusted bootstrap files are sanitized for valid paths, but not deduplicated by path before prompt-context rendering. The linked PR #59344 directly targets this issue, but the provided GitHub context says it is still open and unmerged, so the fix is not implemented on main.

## Best Possible Solution

Keep #59319 open until the path-based dedupe is implemented on main. The narrow fix belongs after `agent:bootstrap` hook overrides and before `buildBootstrapContextFiles`, with a regression test that appends a duplicate resolved path and asserts only one injected entry remains. PR #59344 is the natural candidate if it can be refreshed and validated.
