## Summary

Keep open. Current main still lacks target-derived outbound tool context for exec-approval forwarding, and this PR remains useful but is not merge-ready because the head patch leaves adapter propagation incomplete, drops the new context on queued replay, and adds a compile-breaking test import omission.

## What This Changes

The PR adds outbound `toolContext` plumbing for forwarded exec-approval notices, derives `currentChannelId`/`currentChannelProvider` from resolved targets, and adds regression coverage for forwarded request and resolved deliveries.

## Best Possible Solution

Land an adapted fix on current main that makes target-derived channel identity part of the typed outbound delivery and queue contracts, reaches real channel adapter contexts, and is covered for immediate forwarded approval sends, resolved notices, and recovery replay.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Import filesystem helpers used by the new test:** `src/infra/exec-approval-forwarder.test.ts:382`
  - body: The added regression test calls `fs.mkdtempSync`, `path.join`, `os.tmpdir`, `fs.writeFileSync`, and `fs.rmSync`, but this file does not import `node:fs`, `node:path`, or `node:os` at PR head. TypeScript will fail before the test can run.

  - body: `toolContext` is added to delivery params and the outbound context builder, but the `createChannelHandler(...)` call still omits `toolContext: params.toolContext`. Adapter `sendText`/`sendPayload` contexts therefore continue to receive `ctx.toolContext` as `undefined`.
  - confidence: 0.92
- **[P2] Persist toolContext for queued delivery replay:** `src/infra/exec-approval-forwarder.ts:348-351`
  - body: Forwarded deliveries are queued before send, and recovery rebuilds deliver params from `QueuedDeliveryPayload`. Without adding `toolContext` to the queue payload, enqueue write, and recovery replay path, retries can resend approval notices without the target-derived channel identity.
  - confidence: 0.9
