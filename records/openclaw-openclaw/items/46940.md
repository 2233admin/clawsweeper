## Summary

Keep this PR open. Current main still does not implement the PR’s intended exec-approval forwarding context: forwarded approval delivery resolves channel/to/thread fields, but does not derive or pass target-based toolContext, and the outbound adapter plus queue replay surfaces still have no place to preserve it. The supplied PR file list is source/test only, so the review risk is functional/reliability rather than security or supply-chain.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase/adaptation: add a typed toolContext field to outbound delivery params and ChannelOutboundContext, derive it from the resolved forward target in exec-approval forwarding, thread it through createChannelHandler into real adapter calls, persist/replay it in the delivery queue, and add regression tests for requested/resolved forwarded approvals plus queue recovery.
