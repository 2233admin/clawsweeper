## Summary

Keep this PR open. Current main still lacks the PR’s intended exec-approval forwarding context: forwarded approval delivery passes channel/to/thread fields, but not target-derived tool context, and the outbound adapter/queue paths still have no place to preserve it. The PR is focused and useful, but needs a current-main rebase and complete propagation through adapter contexts plus queue replay.

## Best Possible Solution

Keep this PR open and ask for a rebase/adaptation against current main: add a typed toolContext field to the current outbound delivery params and ChannelOutboundContext, derive it from the resolved forward target in exec-approval forwarding, thread it through createChannelHandler into real adapter calls, persist/replay it in the delivery queue, and add regression tests covering both real adapter context and queue recovery for forwarded exec-approval notices.
