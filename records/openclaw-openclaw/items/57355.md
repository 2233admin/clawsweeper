## Summary

Keep open. Current main does not expose a native Plugin SDK `registerMcpServer` API or registry/runtime path for plugin-owned managed MCP servers; MCP support is still available through bundle/user config surfaces. The provided timeline shows an open related implementation PR, so this should remain open as the tracking issue while that PR or a successor is reviewed.

## What This Changes

Add a native Plugin SDK `api.registerMcpServer` surface so plugins can register managed stdio MCP servers that OpenClaw wires into embedded OpenAI-backed runs.

## Best Possible Solution

Review and either land or revise the linked SDK/runtime implementation so native plugins can register managed MCP servers through a documented, validated, deterministic core seam; keep this tracking issue open until that work merges or a successor replaces it.
