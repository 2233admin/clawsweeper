## Summary

Keep this PR open. Current main still returns `ssh-no-display` on Darwin when `SSH_*` variables are present and no display variables are set, so the linked #67088 dashboard bug is not fixed on main. The PR is narrow and functionally plausible, with no security-sensitive file changes, but it still needs a focused regression test before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch, with a regression test for `resolveBrowserOpenCommand()` that proves Darwin plus lingering SSH variables still resolves to `open`, while Linux without a display keeps returning `ssh-no-display` or `no-display` as intended.
