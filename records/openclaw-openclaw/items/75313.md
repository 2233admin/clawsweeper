## Summary

Keep this issue open. Current main still drops blank user-role session entries during on-disk repair, and a parallel in-memory replay cleanup can drop the same shape before provider submission; the report is valid, narrow, and not an exact duplicate of the related issues listed in the body.

## What This Changes

The issue requests changing session-file repair so blank user-role messages are rewritten to a non-empty placeholder instead of being dropped, preventing strict OpenAI-compatible providers from receiving history with no user query.

## Best Possible Solution

Keep the generic transcript repair invariant in core: blank text blocks should be sanitized, but an only-user turn should be preserved as a non-empty placeholder in both persisted session repair and in-memory replay normalization, with tests covering the strict OpenAI-compatible no-user-history case and docs updated to match.

## Reproduction Assessment

Yes. Without mutating the checkout, the current source and tests provide a high-confidence reproduction path: a user message whose content is `""` or only blank text blocks is dropped by `repairSessionFileIfNeeded`, and the existing test asserts that exact outcome.

## Solution Assessment

Mostly yes. The suggested rewrite-instead-of-drop direction is the narrow maintainable fix, but it should be applied consistently with the in-memory `normalizeAssistantReplayContent` path and should reuse or document a stable placeholder rather than adding a provider-specific Qwen/MLX special case.

## Repair Work Prompt

Fix the blank-user transcript repair bug reported in openclaw/openclaw#75313. Current main drops user messages whose content is blank or only blank text blocks in `src/agents/session-file-repair.ts`, and `src/agents/pi-embedded-runner/replay-history.ts` has a parallel in-memory drop path. Change the generic repair/replay behavior so blank text blocks are removed, but an only-user turn is rewritten to a non-empty placeholder instead of being dropped, preserving at least one user-role message for strict OpenAI-compatible providers.
