## Summary

Keep open. Current main still has the Codex media path normalization bug, while the proposed patch is narrow, uses the existing public file URL helper, adds focused regression coverage, and has no blocking functional or security finding from this review.

## What This Changes

The branch updates the Codex conversation-turn adapter to normalize `file://` image media paths with `safeFileURLToPath` and adds regression tests for decoded paths, remote fallback, and malformed local file URLs.

## Best Possible Solution

Land this as a narrow Codex plugin fix after normal maintainer review, keeping the normalization in the conversation-turn adapter and relying on the shared plugin SDK file-access helper instead of adding another local parser.

## Reproduction Assessment

Yes. A pure unit-level reproduction is available by calling `buildCodexConversationTurnInput` with image `metadata.mediaPath` values such as `file:///tmp/photo%20one.png`, a remote-host `file://` URL plus `metadata.mediaUrl`, and a malformed local file URL; current main’s source shows the old raw pathname behavior directly.

## Solution Assessment

Yes. Reusing `safeFileURLToPath` is the narrowest maintainable fix because it centralizes the repository’s local-file URL validation and decoding rules while preserving remote image fallback when local normalization fails.
