## Summary

Keep this PR open. Current main has already addressed several of the described regressions in a different shape, but this PR still has a blocking correctness issue: its uncaught-exception helper suppresses too broad a class of network failures and can make a failed CLI invocation look successful. The mixed Windows separator skill-path behavior is not clearly covered on current main, so the PR is not obsolete enough to close as implemented.

## What This Changes

The PR modifies skill path compaction and shared CLI uncaught-exception handling/tests, with Slack interaction/pin test changes churned and then restored across the two commits.

## Best Possible Solution

Revise this into a narrower branch that keeps current main's fatal hooks, terminal restore, and narrow benign uncaught-exception policy; if the Windows mixed-separator skill path issue still reproduces, add a focused test and fix around `compactHomePath()` without broadening process-level network failure suppression.

## Reproduction Assessment

Yes for the blocking PR bug: the PR's helper returns `"suppressed"` for a transient network uncaught exception and its installed handler does not exit. Unclear for the mixed Windows separator path issue because current main has no focused test for that exact separator mix.

## Solution Assessment

No. Suppressing every transient network error at the uncaught-exception process handler is broader than the current maintainable approach, which only suppresses a narrow benign set and exits non-zero after terminal cleanup for other uncaught failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exit non-zero for uncaught network failures:** `src/infra/unhandled-rejections.ts:395-397`
  - body: `handleUncaughtProcessError` returns `"suppressed"` for every `isTransientNetworkError`, and the installed CLI handler does not set `process.exitCode` or call `exit`. An uncaught `ECONNRESET` or similar failed network operation can therefore finish as a successful command in scripts even though execution failed; keep the narrower benign gate or exit non-zero for uncaught failures.
  - confidence: 0.93
