## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while the pinned Baileys 7.0.0-rc.9 default derives the omitted predicate from `syncFullHistory`. The recent issue comments and linked #14069 remain consistent with the current source: OpenClaw can process group upserts once Baileys emits them, but the suspected linked-device history-sync gap is still unfixed and undocumented as unsupported.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin either fixes linked-device group inbound delivery or deliberately documents the mode as unsupported. The likely maintainer path is to review the Baileys history-sync contract, update `extensions/whatsapp/src/session.ts` with an explicit `shouldSyncHistoryMessage` predicate or product-backed config option, add a regression assertion in `extensions/whatsapp/src/session.test.ts` for the `makeWASocket` options, and validate with a live linked-device group smoke.
