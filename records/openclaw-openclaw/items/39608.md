## Summary

Keep open. Current main still creates the bundled WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while Baileys 7.0.0-rc.9 defaults that omitted predicate from `syncFullHistory`. The issue and recent comments report the same linked-device symptom on newer releases, docs still present WhatsApp group inbound as supported, and the related open items are adjacent rather than a merged fix or canonical duplicate.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin deliberately fixes or documents linked-device group inbound behavior. The likely code path is a focused change in `extensions/whatsapp/src/session.ts` to pass an explicit Baileys `shouldSyncHistoryMessage` predicate alongside `syncFullHistory: false`, plus a regression assertion in `extensions/whatsapp/src/session.test.ts` and a live linked-device group smoke when possible. If maintainers decide linked-device group inbound cannot be supported reliably, document that limitation in the public WhatsApp/group-message docs instead.
