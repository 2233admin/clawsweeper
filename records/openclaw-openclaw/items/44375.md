## Summary

Current main still has the reported routing foot-gun: once `agents.list` is non-empty, the default agent is `default: true` or the first list entry, unmatched routes use that default, and `openclaw config set agents.list ...` does not warn when implicit `main` is displaced. This should stay open for a narrow UX guard, not be closed as stale, duplicate, or implemented.

## What This Changes

The issue requests a conservative guard so adding an ACP-backed agent to `agents.list` for one binding does not silently move all unmatched routing away from the implicit `main` agent.

## Best Possible Solution

Keep this issue open and implement a conservative config UX guard: warn when `agents.list` becomes non-empty without `default: true` and without `id: "main"`, especially when the previous effective default was implicit `main` or existing `main` state is present. Preserve the documented first-entry fallback unless maintainers decide on a separate breaking routing-policy change.
