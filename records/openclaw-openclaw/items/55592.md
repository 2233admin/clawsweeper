## Summary

Keep this PR open. Current main has not implemented the PR's intended backend-synced message deletion: Control UI message delete still only hides a rendered group through a browser-local DeletedMessages store, and the Gateway protocol/server method surface still has reset/delete/compact but no sessions.truncate or equivalent per-sequence transcript mutation.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for backend-synced Control UI message deletion. The best path is a focused rebase that adds a typed sessions.truncate contract, server-side sequence-based transcript mutation with active-run cleanup and file-safety tests, UI wiring that avoids local-only fallback, generated Swift protocol updates, and targeted regression coverage.
