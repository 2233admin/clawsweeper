## Summary

Keep open. Current main still has the reported Gateway protocol gap, and this PR is a useful narrow compatibility patch, but it adds a Paperclip-named root field to the shared Gateway agent params contract. That needs maintainer/API-owner judgment and duplicate PR consolidation before merge.

## What This Changes

The PR adds an optional opaque root-level `paperclip` field to Gateway `AgentParamsSchema`, regenerates Swift AgentParams models, and adds validator coverage for Paperclip metadata acceptance.

## Best Possible Solution

Land one additive, documented, generic opaque Gateway agent metadata field such as `adapterMeta` or `pluginParams`, regenerate affected client models, keep strict rejection for unrelated root fields, add Paperclip regression coverage, and then close #74635 plus duplicate reports/PRs after the accepted fix merges.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: an `agent` request with `message`, `idempotencyKey`, and root `paperclip` fails `validateAgentParams` because the schema is closed, and #74635 provides the live Paperclip error text and setup path.

## Solution Assessment

No. The proposed patch unblocks the immediate payload shape, but a vendor-named root field is not the narrowest maintainable core protocol shape unless a maintainer explicitly approves that exception; a generic namespaced opaque field is safer.

## Review Findings
