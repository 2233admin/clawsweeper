## Summary

Keep open. Current main parses Telegram reply/quote metadata, but the assistant-facing turn uses `BodyForAgent`, which is still only the standalone inbound text; the formatted reply/quote suffix is only placed on `Body`. That matches the reported symptom and is a narrow, actionable Telegram regression rather than a stale or already-implemented cleanup.

## What This Changes

Telegram reply/quote context should reach the assistant when a user replies to a specific bot message after upgrading to 2026.4.27.

## Best Possible Solution

Keep this issue open and repair the Telegram inbound context boundary so the same visible reply/quote context already computed for `Body` is also available to the assistant turn through `BodyForAgent` or an equivalent consumed field, with regression coverage for normal replies, selected quotes, and external replies.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: construct a Telegram inbound message with `reply_to_message` or `quote`, then inspect the generated payload and dispatch adapter; `Body` contains the reply annotation, while `BodyForAgent` and `textForAgent` do not.

## Solution Assessment

Yes, the requested behavior is the right product direction. The maintainable fix is narrow: adjust Telegram's inbound agent-facing text construction or dispatch mapping, without changing outbound native quote delivery or command parsing semantics.
