## Summary

Keep open. Current main still has the reported numeric ceiling and the config validation mapper still only enriches allowed-value/union cases, not Zod numeric `too_big`/`too_small` bounds. Open PR #73220 explicitly targets this issue and should be reviewed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open. Review PR #73220 against current `src/config/validation.ts`; if it is sound, land it with regression coverage for both `too_big` and `too_small` numeric bounds. If not, implement the same generic bound-hinting in the central mapper and keep coverage in `src/config/validation.allowed-values.test.ts` or a focused sibling test file, while preserving existing allowed-values behavior.
