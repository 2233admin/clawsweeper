## Summary

Keep open. Current main still contains the conflicting Moonshot pinned-tool path: enabled thinking can be rewritten to `thinking: { type: "disabled" }` without clearing any existing `reasoning_effort`, and the current regression test now asserts that behavior. The docs and prior closed-unmerged fix PRs point toward normalizing incompatible Moonshot `tool_choice` values to `auto`, so this is still a valid, narrow bug rather than stale cleanup.

## What This Changes

The issue requests fixing Moonshot/Kimi-compatible requests so enabled reasoning is not sent together with disabled thinking when a pinned tool choice is present.

## Best Possible Solution

Change the Moonshot thinking wrapper so enabled thinking with an incompatible pinned tool choice no longer emits disabled thinking alongside reasoning fields. The docs-aligned end state is to normalize pinned `tool_choice` to `auto`, update the Moonshot regression test accordingly, and avoid touching generated `dist` assets.
