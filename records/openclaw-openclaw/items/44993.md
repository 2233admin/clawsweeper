## Summary

Keep open. Current main still preserves any prompt text that already contains `Current time:`, so heartbeat and main-session cron prompt construction can keep a stale timestamp instead of refreshing it. The latest release tag has the same guard, and the two linked fix PRs are closed unmerged, leaving a narrow valid regression to fix.

## What This Changes

Refresh heartbeat/cron `Current time:` prompt lines on each run instead of preserving an older timestamp line.

## Best Possible Solution

Make the shared current-time helper idempotent for OpenClaw-owned timestamp lines: preserve empty-input behavior, append one fresh line when no helper timestamp exists, refresh or collapse stale helper-emitted lines when present, and cover repeated heartbeat/main-session cron prompt construction with focused regression tests.

## Repair Work Prompt

Fix the stale heartbeat/cron `Current time:` regression tracked by https://github.com/openclaw/openclaw/issues/44993. On current main, `src/agents/current-time.ts` returns any prompt containing `Current time:` unchanged, so heartbeat/main-session cron prompts can preserve an old timestamp even though `runHeartbeatOnce` passes a fresh `startedAt` at `src/infra/heartbeat-runner.ts:1206`. Make `appendCronStyleCurrentTimeLine` idempotent for OpenClaw-owned timestamp lines: keep empty/whitespace behavior, append exactly one fresh line when none exists, and refresh or collapse stale helper-emitted `Current time:` lines on repeated calls. Be careful not to broadly rewrite arbitrary user-authored reminder prose that merely contains `Current time:` unless it matches the helper-owned line format. Use #45071 and #45188 as useful prior proposals, but implement cleanly against current main. Add focused regression coverage in a new `src/agents/current-time.test.ts` plus any narrow heartbeat/session-reset/post-compaction coverage needed to prove repeated construction refreshes without duplication. Add a single-line user-facing changelog fix entry with appropriate reporter/contributor attribution if changelog policy applies. Do not change timezone resolution, the UTC suffix format, provider/model behavior, cron scheduling, or unrelated heartbeat delivery semantics.
