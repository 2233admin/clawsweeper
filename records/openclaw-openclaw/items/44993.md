## Summary

Keep open. Current main still preserves a stale `Current time:` line when the prompt text already contains that prefix, and the related fix PRs were closed unmerged rather than shipped.

## What This Changes

The issue asks for heartbeat and cron prompts to refresh the injected `Current time:` line on every run instead of carrying an earlier timestamp forward.

## Best Possible Solution

Make the shared current-time formatter idempotently refresh OpenClaw-injected `Current time:` lines, keep the existing timestamp format/timezone semantics, and add focused regression coverage so heartbeat and cron prompt construction cannot preserve stale timestamps again.
