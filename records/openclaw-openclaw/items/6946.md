## Summary

Keep this issue open. Current main still lacks the requested Telegram standalone processing-indicator message lifecycle: no accepted `processingIndicator` config, no inbound placeholder message ID carried through dispatch, and no delete/edit cleanup around final replies. Existing typing, ack/status reactions, and native-command progress placeholders are adjacent but do not satisfy the visible in-chat placeholder requested here.

## Best Possible Solution

Keep this issue open as the canonical bundled Telegram feature request. The best path is an opt-in Telegram plugin change that adds typed config/schema/help/docs, sends a standalone placeholder after an eligible inbound message is accepted, carries its message ID through context/dispatch, deletes it after successful final delivery, and edits or replaces it on dispatch failure. Review related PR #40421 separately if maintainers want waiting-tip behavior, but do not close this issue until a focused implementation lands or maintainers explicitly choose a different product direction.
