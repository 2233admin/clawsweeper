## Summary

Current main still does not implement the requested Telegram standalone processing-indicator lifecycle. Telegram has adjacent early typing, ack/status reactions, and draft preview edit/delete behavior, but there is no accepted `processingIndicator`/`waitingTips` config and no inbound placeholder message id carried through dispatch cleanup. The open #40421 waiting-tips PR overlaps and needs maintainer product review before this should become an automatic fix PR.

## Best Possible Solution

Keep this issue open as the canonical bundled Telegram processing-indicator request. The best path is an opt-in Telegram plugin change with typed config/schema/help/docs, dispatch wiring that sends a standalone placeholder after an eligible inbound message is accepted, carries the sent message id through dispatch, deletes it after successful final delivery, and edits or replaces it on failure. Maintainers should first decide how that relates to #40421.
