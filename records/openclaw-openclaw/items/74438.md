## Summary

Keep open. Current main still uses the compatibility agent dir for omitted-`--agent` `models status` and still lacks the doctor live-compatibility exemption, while this PR targets the linked default-agent/auth-store bug with focused code and tests. The PR centralizes `models status` agent-scope resolution, threads the resolved agent id/dir into auth probes, exempts the live compatibility agent dir from doctor orphan warnings, and adds command, doctor, and deadcode-parser regression tests.
