## Summary

Keep open. Current main does not implement the PR’s intended `openclaw gateway status --deep` behavior: the daemon status probe does not capture or return a duration, `DaemonStatus.rpc` has no `durationMs` field, and the text renderer prints only probe status/capability without a duration line. The related health-duration items are for a different command surface and do not supersede this gateway status PR.

## Best Possible Solution

Keep this PR open as a still-relevant small CLI fix. The best path is to review or land a patch that captures probe elapsed time, carries it through `DaemonStatus.rpc`, prints it for successful and failed text status probes, and keeps the JSON output contract stable with targeted tests.
