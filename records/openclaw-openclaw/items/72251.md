## Summary

Keep this draft PR open. Current main at 7e5d6dba80016a5d3502fa6cfc5589aa58a072af still uses the single `tar.c(...)` backup path with only the installed-plugin `node_modules` filter, so the PR's volatile-file filter, EOF retry/backoff, enriched diagnostics, logger plumbing, and `skippedVolatileCount` output are not implemented on main. ## Risks / Open Questions

- The PR is draft and stale against current main; rebasing must compose the volatile filter with current `buildExtensionsNodeModulesFilter` so installed plugin dependency trees remain excluded.
- The proposed logger default can emit retry/skip notices through `runtime.log`; maintainers should verify `openclaw backup create --json` remains pure machine-readable output.
