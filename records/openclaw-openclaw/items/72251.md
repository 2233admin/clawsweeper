## Risks / Open Questions

- The PR is draft and stale against current main; rebasing must compose the volatile filter with current `buildExtensionsNodeModulesFilter` so installed plugin dependency trees remain excluded.
- The proposed logger default can emit retry/skip notices through `runtime.log`; maintainers should verify `openclaw backup create --json` remains pure machine-readable output.
- Retry notices and final diagnostics can include local state file paths; this is useful for debugging but should be intentional for CLI/log surfaces.
- The default volatile skip rules change archive contents for session transcripts/logs, cron run logs, state logs, and `*.sock`, `*.pid`, `*.tmp`, `*.lock`; that needs explicit product/docs acceptance.
- The related deleted-file ENOENT race remains tracked by #67417 and open PR #67472, so this PR should coordinate with that work rather than presenting the broader live-backup race family as complete.
