## Summary

Keep this PR open. Current main still has the channel-summary fallback/default-account behavior and does not contain the PR guard or regression test, but the proposed guard is not a high-confidence fix for the linked Feishu report because Feishu's account helper already synthesizes `default` when no account IDs are configured. The PR also risks hiding status diagnostics for explicit but unresolved channel configs, and the same-author linked issue #64707 remains open.

## Best Possible Solution

Keep this PR open and request changes. The fix should distinguish authored or persisted account configuration from synthetic default-account fallbacks, preserve diagnostics for explicit but unresolved channel configs, and add regression coverage that exercises the real Feishu/shared account-helper path plus an unresolved-config status case.
