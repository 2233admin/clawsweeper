## Summary

Keep this PR open. Current main still has the channel-summary fallback/default-account behavior and does not contain this PR's guard or regression test, but the submitted fix is not a high-confidence implementation for the linked Feishu bug: Feishu's shared account helper already returns a synthetic `default` account ID when no accounts are configured, so a guard based on `accountIds.length` can still treat the phantom default as real. The PR also risks hiding diagnostics for explicitly configured but unresolved channels such as Nostr, and the same-author linked issue #64707 remains open.

## Best Possible Solution

Keep this PR open and request changes. The fix should distinguish authored or persisted account configuration from synthetic default-account fallbacks, suppress only the phantom default status, preserve diagnostics for explicit but unresolved channel configs, and add regression coverage for the real Feishu/shared account-helper path plus an unresolved-config status case.
