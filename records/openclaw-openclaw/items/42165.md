## Summary

Keep open. Current `main` still has the profile-blind implicit grant path reported here: configured `tools.fs` and `tools.exec` sections are merged into the profile allowlist before filtering, while the docs and catalog keep `minimal` to `session_status` and keep filesystem/runtime tools out of `messaging`.

## What This Changes

The issue reports that restrictive `tools.profile` settings such as `minimal` and `messaging` still expose filesystem/runtime tools when `tools.fs` or `tools.exec` config sections are present.

## Best Possible Solution

Make `tools.fs` and `tools.exec` blocks configure behavior only for tools already granted by the effective profile or an explicit allowlist; preserve explicit `alsoAllow` and deny precedence, and add regression coverage for both `minimal` and `messaging`.
