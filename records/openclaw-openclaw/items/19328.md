## Summary

Keep open. Current main still has the stale cached session-entry writeback pattern this PR targets: the agent handler builds `nextEntryPatch` from cached `entry` fields, including `modelOverride` and `providerOverride`, then merges that patch over the fresh store row loaded inside `updateSessionStore`.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent focused fix in `src/gateway/server-methods/agent.ts` that stops cached `entry` fields such as `modelOverride` and `providerOverride` from being merged over the fresh store row. Keep the stale-cache regression tests; leave the narrower residual delivery-field follow-up to #70575.
