## Summary

Keep this issue open. Current main still computes the Discord native slash-command `no_allowlists` audit from legacy `dm.allowFrom` plus the credential-store allowlist, while top-level `channels.discord.allowFrom` is documented, schema-supported, and used by the runtime authorization path. There is already an open exact-scope PR, #72467, with closing syntax for this issue, so the right maintainer action is to review, update, or land that PR rather than close the issue early or queue a duplicate repair.

## What This Changes

The issue requests that the Discord security audit treat top-level `channels.discord.allowFrom` as a valid slash-command owner allowlist.

## Best Possible Solution

Keep this issue open and review, update, or land #72467. The fix should stay narrowly in `extensions/discord/src/security-audit.ts`, normalize top-level `discordCfg.allowFrom` together with legacy `dm.allowFrom` and pairing-store allowlists, and keep the regression in `extensions/discord/src/security-audit.test.ts`. No core API change appears necessary.
