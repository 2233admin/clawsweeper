## Summary

Keep this issue open. Current main still has no `brave.http` diagnostics flag, and the bundled Brave web-search runtime still performs outbound Brave Search/LLM Context requests and cache reads/writes without flag-gated request, response-status, timing, or cache hit/miss diagnostics. The prior ClawSweeper comment remains accurate against the newer main SHA.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by passing the full config or a precomputed `isDiagnosticFlagEnabled("brave.http", config)` boolean into the Brave runtime, then add sanitized diagnostic logs for outbound request URL/query/params, response status and duration, and cache hit/miss for both `web` and `llm-context` modes. Add focused Brave tests and document the flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`.

## Risks / Open Questions

- Implementing this naively could log sensitive search queries or full URLs with user intent. The feature should remain opt-in behind `brave.http`, avoid logging `X-Subscription-Token` or SecretRef-derived credential material, and document that query logging is sensitive.
