## Summary

Current main still does not implement Telegram pinned-message reads. The shared `list-pins` action and CLI command exist, but the bundled Telegram plugin does not advertise, gate, normalize, or execute that action, and its existing `getChat` helper only extracts `result.id`. Closed PRs #38613 and #38718 are useful prior art but neither merged, so this issue remains valid and narrow enough for a focused repair PR.

## What This Changes

Request to let OpenClaw fetch the current pinned Telegram chat message via the Bot API, either on demand or optionally at session start.

## Best Possible Solution

Keep this issue open and land a focused bundled Telegram plugin change that exposes `list-pins`/`pins` through the existing shared message action surface, backed by Bot API `getChat.pinned_message`, with Telegram `actions.pins` config/schema/docs, normalization into the shared pins payload, and tests documenting Telegram's most-recent-pinned-message limitation. Automatic session-start fetching should remain out of scope unless a maintainer explicitly chooses that product behavior.
