## Summary

Current main still does not implement Telegram pinned-message reads. The shared message-action contract already supports `list-pins`, and other channels expose pins through that surface, but the bundled Telegram plugin only advertises and dispatches send/poll/react/delete/edit/sticker/topic actions. The existing Telegram `getChat` helper only reads `result.id`, Telegram config/schema has no `actions.pins` gate, and the docs still list pin reads for Discord/Slack/Matrix rather than Telegram. Closed PRs #38613 and #38718 remain useful prior art, but neither was merged.

## Best Possible Solution

Keep this issue open and land a focused bundled Telegram plugin change that exposes Telegram `list-pins`/`pins` through the existing message action surface, backed by Bot API `getChat.pinned_message`. Add a `channels.telegram.actions.pins` config/schema/docs gate, normalize the single Telegram pinned message into the shared pins payload shape, document the chat-level most-recent-pin limitation, and use #38613 and #38718 as prior art rather than canonical fixes.
