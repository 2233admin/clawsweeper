## Summary

Keep open. Current main still does not expose an OpenAPI/Swagger schema artifact, generator, Swagger UI, or `/openapi.json` discovery route for OpenClaw's Gateway HTTP API. The existing HTTP handlers and docs cover concrete routes, while the only nearby schema generator is the WebSocket protocol JSON Schema pipeline, not an HTTP OpenAPI contract.

## Best Possible Solution

Keep this issue open for maintainer API-boundary design. Define the supported public Gateway HTTP surface first, then add a generated or committed OpenAPI artifact and optionally a served discovery endpoint such as `/openapi.json`, with docs and a drift check/test. Keep the existing WebSocket protocol JSON Schema separate unless maintainers explicitly choose to model that surface in OpenAPI.
