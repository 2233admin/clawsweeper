## Summary

Keep open. Current main still does not expose an OpenAPI/Swagger schema artifact, generator, Swagger UI, or `/openapi.json` discovery route for the Gateway HTTP APIs. The request remains valid product/API-boundary work, not maintainer cleanup that can be closed.

## What This Changes

Expose an OpenAPI/Swagger schema for OpenClaw Gateway HTTP APIs so users can generate custom clients programmatically.

## Best Possible Solution

Keep this issue open for maintainer API-boundary design. Define which Gateway HTTP routes are public and stable, then add a generated or committed OpenAPI artifact, docs, and a drift check; optionally serve the artifact at an authenticated discovery endpoint such as `/openapi.json`. Keep WebSocket protocol JSON Schema and config schema surfaces separate unless maintainers explicitly choose to model them in OpenAPI.
