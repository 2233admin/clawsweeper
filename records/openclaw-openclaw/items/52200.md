## Summary

Keep this PR open. Current main still does not implement the requested Windows normalization: compacted skill paths replace the home prefix with `~/` but leave the remainder unchanged, and that value is emitted directly into `<location>` for the model to read. The PR is a focused implementation candidate for the closed linked bug #52022, and the provided review thread shows the one test-review concern was addressed in the second commit.

## Best Possible Solution

Review and land this PR, or apply an equivalent small fix in `src/agents/skills/workspace.ts` that normalizes prompt-facing Windows skill paths after home compaction, plus the regression test in `src/agents/skills.compact-skill-paths.test.ts`. After that ships on main, this PR can close as implemented or superseded by the landed commit.
