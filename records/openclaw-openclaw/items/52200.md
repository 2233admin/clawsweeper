## Summary

Keep this PR open. Current main still compacts home-prefixed skill paths to `~/` without normalizing remaining Windows backslashes, and both full and compact skill prompt renderers emit that file path directly. The PR remains a focused implementation candidate for the linked Windows path bug, though it should be rebased/adapted to the current `compactHomePath()` helper shape before landing.

## Best Possible Solution

Keep this PR open and land an equivalent small fix on current main: normalize prompt-facing Windows skill paths after home compaction, preferably at `compactHomePath()` or immediately after `compactSkillPaths()`, and include a regression assertion that rendered `<location>` values contain no backslashes for Windows-style paths.
