## Summary

Keep open. Current main still emits prompt-facing skill paths from `compactHomePath()` without Windows backslash normalization, and the PR targets the right bug, but the branch was written against an older compaction shape and needs to be rebased/adapted before merge.

## What This Changes

The PR normalizes Windows backslashes in compacted skill paths and adds a regression assertion for rendered skill `<location>` values.

## Best Possible Solution

Normalize prompt-facing Windows skill paths once at the current compaction boundary, preserving the multi-home and realpath-prefix behavior on main, and add a regression test that actually exercises Windows-style input before running the targeted skills test and changed gate.

## Reproduction Assessment

Yes. The static path is high confidence: on Windows, `path.sep` is `\`, `compactHomePath()` slices after the home prefix and prepends `~/`, and both prompt formatters emit the resulting `filePath` unchanged into `<location>`.

## Solution Assessment

Yes in direction, but not as currently rebased. The narrow maintainable solution is to move the normalization into `compactHomePath()` or immediately after the current `compactSkillPaths()` mapping so both full and compact skill prompt formats receive slash-consistent locations.
