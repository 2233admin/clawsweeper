## Summary

Keep open. Current main has durable background task records and opt-in state-change notifications, but the core request is still not implemented: subagent spawn acknowledgements are not framework-delivered directly to the requester channel, progress lifecycle delivery is not a native default subagent contract, and the announce queue used by completion delivery remains in-process rather than a durable per-announce retry queue.

## What This Changes

The issue requests framework-owned immediate and durable subagent lifecycle notifications that are delivered independently from parent session-lane batching.

## Best Possible Solution

Keep this as the umbrella runtime architecture request. The desired end state is a core-owned subagent lifecycle delivery contract with immediate requester-channel spawn ACKs, per-run progress and terminal events, durable announce persistence with idempotency, and ordering/backpressure rules that coordinate with #22358 and #54488 without routing arbitrary model text around normal channel delivery safeguards.
