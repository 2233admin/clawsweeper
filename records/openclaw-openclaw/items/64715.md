## Summary

Keep this issue open. Current main still exposes Telegram inline keyboards on the outbound agent/message path, but there is no native ReplyKeyboardMarkup or ReplyKeyboardRemove surface for reply keyboard rows, resize_keyboard, one_time_keyboard, or keyboard removal; the related items and open nearby PRs are inline-button/callback work rather than this capability.

## What This Changes

Add native Telegram reply keyboard support to the agent/message send surface, including reply keyboard rows/buttons, keyboard removal, resize_keyboard, and one_time_keyboard.

## Best Possible Solution

Implement an additive Telegram-owned reply keyboard path in the bundled Telegram plugin, with typed schema/docs/tests for ReplyKeyboardMarkup and ReplyKeyboardRemove while preserving the existing inline keyboard behavior and shared presentation contract.
