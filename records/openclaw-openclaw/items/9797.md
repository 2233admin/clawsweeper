## Summary

Keep this issue open. Current main still lacks the requested agent-facing `queue_status` tool or equivalent stable queue snapshot surface for intelligent task dispatch. There is useful related work in #43618, but that PR is still open and its body explicitly says it does not add an in-agent tool.

## Best Possible Solution

Implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy registration, permissions, docs, and focused tests. If #43618 lands first, reuse that backend and keep this issue focused on the in-agent dispatch surface.
