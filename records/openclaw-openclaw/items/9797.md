## Summary

Keep this issue open. Current main still has no agent-facing `queue_status` tool or equivalent stable queue snapshot surface. The existing queue state is internal, `agents_list`/`sessions_list` do not expose workload or lane backlog, and the related open PR #43618 is useful backend/operator work but explicitly does not add the in-agent tool requested here.

## Best Possible Solution

Keep this issue open and implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy registration, docs, and focused tests. If #43618 lands first, reuse its backend and keep this issue focused on the in-agent dispatch surface.
