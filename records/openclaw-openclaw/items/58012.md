## Summary

Keep this issue open. Current main still does not apply Mistral strict9 tool-call-id replay policy when Mistral-family models are routed through proxy providers such as OpenRouter or Together. The issue discussion is consistent with the post-#60126 architecture: replay policy now comes from provider-plugin hooks, while the older provider-capabilities fixes target removed files.

## Best Possible Solution

Keep this issue open until a current-main fix lands. Implement strict9 handling in the provider replay layer, likely by extending OpenRouter/Together or a shared proxy replay helper to return `sanitizeToolCallIds: true` and `toolCallIdMode: "strict9"` only for Mistral-family routes, while preserving existing Gemini passthrough behavior. Add regression coverage for OpenRouter Mistral, OpenRouter Codestral, non-Mistral OpenRouter, and any intended Together/custom-proxy path. Maintainers can separately decide whether to consolidate tracking with #47707.
