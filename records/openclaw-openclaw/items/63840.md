## Summary

Keep this PR open. Current main still lacks the requested Slack Agents & Assistants DM-root thread preservation for non-`all` reply modes, but the proposed patch still has a correctness gap because it keys the override on raw `channel_type` instead of OpenClaw's normalized DM classification.

## What This Changes

The PR changes Slack thread-context resolution and tests to set `MessageThreadId` for DM assistant thread-root messages across reply modes, and also adds `context_window_exceeded` matching to an agents overflow-compaction test harness.

## Best Possible Solution

Revise the Slack fix to use the normalized direct-message classification already computed in the message pipeline, preserve `MessageThreadId` only for assistant-DM self-thread roots, keep non-DM self-thread roots unset, and add focused coverage for raw `im`, D-prefix wrong or missing `channel_type`, non-DM roots, and concurrent assistant-DM roots.

## Review Findings

Overall correctness: patch is incorrect

  - body: The new guard checks only the raw `params.message.channel_type === "im"`, but this path already supports D-prefixed DMs when Slack omits or misreports `channel_type`. Derive this condition from normalized DM state or equivalent channel-id inference.
