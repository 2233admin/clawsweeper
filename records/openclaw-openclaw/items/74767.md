## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit interactive-card updates without an indexed plain-text fallback or documented Feishu search limitation, and PR #74808 is an open, unmerged closing PR for the same issue.

## What This Changes

Feishu streaming-card replies should be searchable in Feishu conversation search, or the channel should provide a documented fallback/limitation for that mode.

## Best Possible Solution

Keep the issue open until the Feishu plugin has an explicit, documented searchable fallback or a maintainer-approved documented limitation; the narrow implementation belongs in the Feishu plugin/config/docs rather than core message dispatch.

## Reproduction Assessment

Yes. The issue gives a high-confidence manual path in Feishu, and current main’s code path matches it: streamed card content is delivered by card create/update/close rather than a final plain text message, though I did not perform a live Feishu search in this read-only review.

## Solution Assessment

Yes for triage: keeping this open while PR #74808 is reviewed is the right path. The maintainable fix should be a Feishu-owned opt-in fallback with tests and docs, or a deliberate documented limitation if Feishu cannot index updated cards.
