## Summary

Keep open. Current main still routes Feishu streaming replies through CardKit interactive cards and suppresses a final plain-text delivery, so it does not solve the reported Feishu conversation-search gap or document it as a known limitation.

## What This Changes

Feishu streaming card replies should remain searchable in Feishu conversation search, matching plain text replies.

## Best Possible Solution

Confirm the Feishu search behavior live, then handle it in the Feishu plugin with an owner-approved delivery policy: either an explicit searchable final-text fallback or opt-in setting with tests and docs, or a clearly documented Feishu limitation if no acceptable indexable streaming path exists.
