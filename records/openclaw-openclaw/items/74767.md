## Summary

Keep open. Current main still sends Feishu streamed replies as CardKit interactive cards without an indexable final plain-text delivery or a documented search limitation, and open PR #74808 explicitly references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Request that Feishu streaming-card replies remain searchable in Feishu conversation search, either through an indexable final fallback delivery path or clear documentation of the Feishu limitation.

## Best Possible Solution

Review #74808 as the active implementation candidate: either land an owner-approved opt-in searchable fallback with tests/docs, or close that PR and document Feishu streaming-card search as a known limitation with `streaming: false` as the workaround.

## Reproduction Assessment

Yes. The issue gives concrete manual Feishu steps, and current main still has the matching source path: streaming card creation/update/close with no final plain-text fallback; only the external Feishu search result itself was not live-tested here.

## Solution Assessment

No for current main; it does not solve the searchability gap. The linked opt-in fallback direction is plausibly the narrowest maintainable fix, but it still needs normal PR review because it changes Feishu delivery semantics and may create visible duplicate messages.
