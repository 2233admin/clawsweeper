## Summary

Keep open. Current main still delivers Feishu streaming replies through CardKit interactive cards without an indexable final plain-text message, and docs only describe disabling streaming as the complete-message path. The linked open PR #74808 explicitly fixes this issue, so this issue should remain open until that PR is reviewed and either merged or closed.

## What This Changes

Make Feishu streaming card replies searchable in Feishu conversation search, likely by adding an indexable final text fallback or documenting the limitation.

## Best Possible Solution

Review and either land or reject PR #74808. The desired end state is an explicit Feishu-owned policy: an opt-in searchable final plain-text fallback with config, tests, and docs, or a documented Feishu limitation if there is no acceptable indexable streaming path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the issue gives concrete Feishu UI steps, and current main can be traced to active streaming-card delivery with no final plain-text send; the external Feishu search index behavior still needs live confirmation.

## Solution Assessment

Is this the best way to solve the issue? The linked opt-in fallback direction appears safer than changing default streaming semantics because it avoids new duplicate messages unless enabled; it still needs maintainer review because it changes Feishu delivery behavior and docs/config surface.
