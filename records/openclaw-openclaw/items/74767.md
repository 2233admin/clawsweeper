## Summary

Keep open. Current main still delivers Feishu streaming replies as interactive CardKit cards without a plain-text searchable fallback, and the related closing PR #74808 is open rather than merged. Because that PR uses closing syntax for this issue, the issue should stay open until the PR lands or is deliberately replaced.

## What This Changes

Request that Feishu streaming card replies be searchable in Feishu conversation search, likely through a final plain-text fallback or documented platform limitation.

## Best Possible Solution

Land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback and docs that describe the visible duplicate-message tradeoff; keep the behavior out of core unless another channel needs the same contract.
