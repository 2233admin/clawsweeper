## Summary

Keep open. Current main still does not expose the five requested active-run steer/abort helpers from `openclaw/plugin-sdk/agent-runtime`, so this PR is not obsolete or implemented. The linked issue #49624 describes the same SDK gap but is already closed; it does not supersede the open implementation candidate, which still needs maintainer judgment on the SDK boundary and `abortEmbeddedPiRun` overload policy.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to expose a supported active-run control surface for external channel plugins covering the five requested helpers, then decide whether that surface belongs in `plugin-sdk/agent-runtime` or a narrower dedicated subpath. Maintainers should also choose whether `abortEmbeddedPiRun` is exported whole as a trusted-plugin capability or wrapped to a single-session abort contract, then require the matching SDK docs, API baseline, and validation proof before merge.

## Risks / Open Questions

- Exporting `abortEmbeddedPiRun` as-is would expose `(undefined, { mode: "all" | "compacting" })` to external plugins. Current tests confirm those modes abort compacting or all active runs, so maintainers should explicitly accept, document, or wrap that capability before merge.
- Adding these helpers to the broad `agent-runtime` barrel may be less aligned with current SDK boundary guidance than a narrow active-run control subpath; the intended public contract needs maintainer choice.
- Because this changes a published plugin SDK surface, it should land with matching docs/API-baseline/export coverage and normal SDK validation. This review stayed read-only and did not run tests or builds.
