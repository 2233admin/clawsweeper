## Summary

Keep this PR open. Current main still does not expose the five requested active-run steer/abort helpers from `openclaw/plugin-sdk/agent-runtime`, and the PR remains a focused one-file implementation candidate for an SDK gap. The remaining maintainer decision is the public SDK shape and whether directly exporting `abortEmbeddedPiRun` should also expose its cross-session abort overload to external plugins.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to expose a supported active-run control surface for external channel plugins covering queue, abort, active, streaming, and wait operations; choose the SDK shape deliberately (`plugin-sdk/agent-runtime`, a narrower active-run subpath, or `api.runtime.agent`), then settle the cross-session abort overload policy and add matching docs/API-baseline/export checks before merge.

## Risks / Open Questions

- Directly re-exporting `abortEmbeddedPiRun` would expose the cross-session `mode: "all" | "compacting"` overload to external plugins; maintainers should explicitly accept, document, or wrap that capability.
- The PR changes a public plugin SDK surface, so it should land with matching SDK docs/API-baseline/export coverage and validation proof.
- The best public shape may be a narrow active-run control subpath or injected `api.runtime.agent` helper instead of appending more names to the broad `agent-runtime` barrel.
