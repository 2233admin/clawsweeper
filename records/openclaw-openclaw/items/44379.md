## Summary

Keep this PR open. Current main has adjacent context-window stop-reason normalization, but it still does not implement the PR's central runtime behavior: a bounded overflow-recovery retry with prompt-hook context injection suppressed after compaction and tool-result truncation fail. The PR remains a plausible implementation candidate, but it should be reworked before merge because the submitted diff includes unrelated extension peer/lockfile churn and review comments identify retry-budget and suppression-scope concerns.

## Best Possible Solution

Keep the PR open, but ask for a focused rework or have a maintainer apply the useful runtime pieces separately: add a one-attempt prompt-hook-context suppression recovery path in src/agents/pi-embedded-runner, preserve compaction_failure semantics, gate the retry on remaining loop budget and meaningful hook context availability, add targeted tests for retry behavior and raw stop-reason UI normalization, and remove unrelated dependency/lockfile changes.
