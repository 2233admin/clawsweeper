## Summary

Keep this PR open. Current main still does not implement configurable TUI status-line command rendering, and the latest PR context shows active author follow-up that addresses earlier schema, lifecycle, and security-documentation review concerns. The petsonality workaround is useful demand evidence, but it remains an external patch rather than shipped OpenClaw behavior or an available TUI plugin seam.

## Best Possible Solution

Keep this PR open for maintainer/product review. If accepted, land it as a core TUI/config feature with the config types, strict Zod validation, generated schema/docs metadata, lifecycle tests, and trusted-local-command documentation shown in the latest PR context. If maintainers prefer externalization, first add a public TUI status/render plugin seam, then move command rendering into a plugin or ClawHub package, using the petsonality workaround as demand and design input.
