## Summary

Keep this PR open. Current main still does not implement configurable TUI status-line command rendering, and the PR remains an active implementation candidate rather than obsolete cleanup. The linked petsonality workaround is useful demand evidence, but it is an external patch, not a supported OpenClaw core/plugin surface.

## Best Possible Solution

Keep this PR open for maintainer/product review. If accepted, land it as a core TUI/config feature with updated config types, strict Zod validation, generated schema/docs baselines, lifecycle tests, and explicit trusted-local-command documentation. If maintainers prefer externalization, first add a public TUI status/render plugin seam, then move command rendering into a plugin or ClawHub package, using the petsonality workaround as demand/design input.
