## Summary

Keep open. Current main still generates the workspace `AGENTS.md` with critical safety and tool-use sections after lower-priority startup and memory text, and the current bootstrap trimmer keeps a fixed head/tail slice without preserving those middle sections under a 1500-character per-file budget. The adjacent tiered-bootstrap PR is open and targets file selection, not priority within the generated file.

## What This Changes

The issue asks OpenClaw to preserve critical generated `AGENTS.md` safety and tool-use guidance when `agents.defaults.bootstrapMaxChars` trims bootstrap context.

## Best Possible Solution

Move the essential safety and tool-use guidance near the top of the generated `AGENTS.md` template, add a regression that proves those sections survive a 1500-character `bootstrapMaxChars` budget, and document the supported path for existing workspaces to adopt the safer ordering without overwriting personal edits.

## Reproduction Assessment

Yes. The current template plus the current trim algorithm provide a high-confidence source-level reproduction: at `bootstrapMaxChars=1500`, `Red Lines`, `External vs Internal`, and `Tools` are absent from the injected `AGENTS.md` content.

## Solution Assessment

Yes for the near-term fix direction. Reordering the generated template is the narrowest maintainable change; content-priority truncation can remain a broader follow-up if maintainers want a generalized mechanism.
