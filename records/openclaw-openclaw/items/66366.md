## Summary

Keep open. Current main still has the reported path: background exec failures enqueue raw command-tail text as untrusted system events, exec-event heartbeats can ask the model to relay those details, and Discord outbound delivery does not have a raw exec/system-event suppression or sanitization policy. The existing `tools.exec.notifyOnExit=false` knob is too broad, and adjacent open PR #68045 is UI/transcript-focused rather than a Discord outbound fix.

## What This Changes

The issue asks OpenClaw to stop Discord from posting raw exec failure/system-event text as a separate visible message after an exec card, either by keeping it on the card/update surface, sanitizing it, or suppressing it.

## Best Possible Solution

Add a scoped Discord-visible exec completion policy that suppresses or rewrites raw `System (untrusted)` / `Exec failed (...) :: ...` follow-ups while preserving exec cards and intentional sanitized async completion summaries where they are useful.
