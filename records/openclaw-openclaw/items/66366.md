## Summary

Current main still has the reported behavior path: backgrounded exec failures can enqueue raw command-tail text as untrusted system events, heartbeat turns can ask the model to relay those details when a delivery target exists, and Discord outbound delivery does not have an exec-failure-specific sanitizer or card-only policy. The issue should remain open because the existing global `tools.exec.notifyOnExit=false` knob suppresses all completion notifications rather than solving the Discord-visible raw follow-up problem.

## What This Changes

The issue requests that Discord stop showing raw exec failure text as a separate visible follow-up message after exec cards.

## Best Possible Solution

Implement a scoped policy that prevents Discord from receiving raw `System (untrusted)` or command-tail exec failure follow-ups after exec-card failures, while still allowing useful sanitized async completion alerts where they are intentional. The regression proof should cover Discord-visible delivery and should distinguish sanitized completion summaries from raw system-event leakage.
