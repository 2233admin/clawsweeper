## Summary

This PR should stay open. Current main still has the unguarded checked-continuation resume in `WebSocketTaskBox.sendPing()`, the keepalive path still calls it, and no equivalent fix or regression test has landed. The PR is narrow and plausibly addresses the reported iOS/macOS crash; overlapping open PRs #41653 and #51311 should be consolidated by maintainers rather than using this review to close one candidate prematurely.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one narrow at-most-once guard around `WebSocketTaskBox.sendPing()` in `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift`, preferably with a focused duplicate-pong regression test in `OpenClawKitTests`, then close overlapping PRs #41653 and #51311 against the selected implementation.
