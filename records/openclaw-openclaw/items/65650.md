## Summary

Current main still does not implement the durable pre-dispatch inbound message store requested here. Slack ingress prepares and dispatches messages directly into the auto-reply path, while the durable SQLite/task and restart-continuation queues on main cover different surfaces. The open related PR #49431 remains the right implementation candidate to review, land, or supersede, but it is open and unmerged, so this issue should stay open.

## Best Possible Solution

Keep this issue open and use it to review, land, or supersede #49431. The best implementation is a durable pending-inbound store at the channel/gateway ingress boundary before active dispatch, with startup replay, shutdown/restart handling, and targeted regression coverage for an accepted Slack turn interrupted before final reply delivery.
