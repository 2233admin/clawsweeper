## Summary

Keep open. Current main still has the reported session-memory slug bug, and the PR is aimed at the right narrow surface, but the head patch remains incomplete because it filters only by response shape and can still accept short `payload.isError` text as a filename slug.

## What This Changes

The PR extracts a testable LLM response slugifier, moves dash stripping after truncation, routes `generateSlugViaLLM` through it, and adds colocated slug-generation regression tests.

## Best Possible Solution

Amend this branch to reject `payload.isError === true` before calling the slugification helper, keep the pure helper and post-truncation dash stripping, add a `generateSlugViaLLM` regression test for an error payload whose text would otherwise look slug-like, then coordinate any broader slug-hardening behavior with #38161.
