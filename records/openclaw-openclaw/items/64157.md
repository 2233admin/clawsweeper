## Summary

Keep open: current main still matches the reported failure mode. Control UI chat turns are sent as internal WebChat traffic and the gateway intentionally blocks WebChat/UI main-session inheritance of stored external routes, so a Feishu-started main-session conversation is not currently converted into Feishu pushback after a Control UI follow-up.

## What This Changes

Allow assistant replies to keep being delivered to Feishu when a Feishu-started conversation is interrupted or continued from the Web Control UI.

## Best Possible Solution

Define the intended cross-surface contract first, then implement an explicit, permissioned route-inheritance or opt-in mirroring path for Feishu-started sessions that preserves the existing WebChat safety guards and covers Feishu main-session delivery in regression tests.
