## Summary

Keep open. Current main still has the reported source-level path: a direct main/TUI session with sticky `entry.channel: "telegram"` and no explicit `entry.displayName` can still be formatted through `buildGroupDisplayName()` before falling back to `origin.label`, so the TUI footer can show `telegram:g-agent-main-main` instead of `openclaw-tui`. The reporter's patch and store-state analysis remain aligned with current code.

## Best Possible Solution

Keep this issue open. The best fix is to guard `buildGroupDisplayName()` in `src/gateway/session-utils.ts` for both `buildGatewaySessionRow()` and `resolveSessionListSearchDisplayName()` so it only runs for parsed group/channel keys or entries whose `chatType`/origin chat type is `group` or `channel`. Add a regression test for a direct `agent:main:main`/TUI session with `channel: "telegram"`, `origin.label: "openclaw-tui"`, and direct chat type that preserves `openclaw-tui` instead of producing `telegram:g-agent-main-main`.
