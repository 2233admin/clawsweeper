## Summary

Keep open. Current main still builds a gateway session row displayName from `entry.channel` alone before falling back to the TUI/WebChat origin label, so a direct session with `chatType: "direct"` and `channel: "telegram"` can still surface a group-style `telegram:g-...` name in the TUI footer.

## Best Possible Solution

Keep this issue open and fix the gateway row display-name derivation: only call `buildGroupDisplayName()` when the session is actually a group/channel session, then add a regression test for a direct Telegram-backed main session that preserves the `openclaw-tui`/origin label instead of showing `telegram:g-agent-main-main`.
