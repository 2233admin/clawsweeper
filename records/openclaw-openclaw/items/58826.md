## Summary

Keep open. Current main has usage/cost visibility, provider quota snapshots, context-size knobs, model fallback after provider errors, opt-in tool-loop detection, and internal retry caps, but it still does not expose built-in operator policy that enforces per-request, per-session, daily token, or request budgets before model dispatch. The related #42475 and #9912 cover narrower slices, and #45879 was closed unmerged, so this issue still tracks broader product work.

## What This Changes

Add configurable built-in token-budget and request-limit controls across request, session, daily, per-provider, and per-agent scopes, including warn/block/fallback behavior and loop safeguards.

## Best Possible Solution

Define an additive core budget policy contract at the gateway/agent-runtime dispatch boundary, reuse existing usage/cost summaries and provider quota snapshots where reliable, enforce configured limits before model calls where possible, surface status/doctor visibility, and keep loop detection as a separate configurable guardrail rather than treating telemetry alone as enforcement.
