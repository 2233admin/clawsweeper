## Summary

Keep this issue open. Current main still renders the Control UI chat session selector, including mobile, as grouped native selects with no clear-completed action, per-row dismiss control, or completed-subagent separator. The session lifecycle fields and existing `sessions.delete` plumbing make a focused fix feasible, but the separate Sessions tab bulk-delete workflow does not satisfy on-demand cleanup from the chat dropdown.

## Best Possible Solution

Implement a focused Control UI cleanup path in the chat session selector, including the mobile selector. Derive clearable rows from subagent session keys plus terminal lifecycle fields such as `status`, `endedAt`, `subagentRunState`, and `hasActiveSubagentRun`; call the existing `sessions.delete` RPC with the intended transcript behavior; refresh `sessions.list`; and add UI/controller tests proving completed subagents can be cleared without affecting active subagents, main sessions, channels, cron rows, or persistent thread-bound sessions.
