## Summary

Keep this issue open. Current main still renders the Control UI chat session selector as native grouped selects with no clear-completed action, per-row dismiss control, or completed-subagent separator. The backend/session data needed for a targeted fix exists, and the Sessions tab has a bulk delete workflow, but that separate management surface does not satisfy on-demand cleanup from the dropdown.

## Best Possible Solution

Keep this issue open and implement a focused Control UI cleanup path in the chat session selector, including the mobile selector. Derive eligible rows from subagent session keys plus terminal lifecycle fields, expose either Clear completed or accessible per-row dismiss controls, call the existing sessions.delete RPC with the intended transcript behavior, refresh sessions.list, and add UI/controller tests proving completed subagents are cleared without affecting active subagents, main sessions, channels, cron rows, or persistent thread-bound sessions.

## Risks / Open Questions

- A dropdown clear action should avoid deleting active subagents or persistent sessions; eligibility should use terminal lifecycle fields such as status, endedAt, subagentRunState, and hasActiveSubagentRun, not key shape alone.
- The existing Sessions tab bulk delete archives transcripts by passing deleteTranscript: true; a dismiss-only dropdown action may need deleteTranscript: false or explicit product copy if transcript archival is intended.
