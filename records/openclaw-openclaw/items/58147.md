## Summary

Keep #58147 open. Current main still formats Control UI chat timestamps and native WebChat session timestamps from browser/Swift locale defaults, and the UI data contracts do not carry `agents.defaults.timeFormat` into either surface. The related ClawSweeper reports concern other `agents.defaults.*` settings and do not provide a canonical duplicate for this time-format UI gap.

## Best Possible Solution

Keep the issue open and implement a shared UI time-format resolver for Control UI and native WebChat. The likely path is to expose the resolved `agents.defaults.timeFormat` through the relevant session/config payloads, pass it into chat/session timestamp renderers, add focused browser and Swift tests for 12h/24h output, and update docs if the setting is intentionally expanded beyond prompt formatting.
