## Summary

Keep open. Current main still lets `channels login` and `channels logout` inherit the parent `channels` startup policy of broad plugin preloading, so the host CLI can import configured plugin modules before `runChannelLogin()` reaches the selected channel plugin's `auth.login` QR flow. The existing scoped channel plugin resolver is available, but the pre-action path has not been narrowed for this command yet.

## Best Possible Solution

Keep this issue open and fix the generic host CLI startup path: add a narrow startup policy for `channels login` and likely `channels logout` so they do not broad-preload all plugins in pre-action, then rely on the existing scoped channel setup resolver to load only the requested or inferred auth-capable channel. Preserve explicit `--channel openclaw-weixin`, single configured channel inference, external plugin compatibility, config auto-enable, and install-on-demand behavior with focused tests.
