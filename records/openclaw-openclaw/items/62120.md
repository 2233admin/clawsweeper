## Summary

Keep open. Current main still lets `channels login` and `channels logout` inherit the broad parent `channels` plugin-preload policy, so the host CLI can preload all plugins during pre-action bootstrap before the selected external channel plugin's QR login path runs.

## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin --verbose` hangs on OpenClaw 2026.4.5 before the Weixin plugin QR login output appears.

## Best Possible Solution

Land a channel-agnostic CLI startup fix that keeps interactive channel auth commands off broad pre-action plugin preload, then explicitly loads only the requested or inferred auth-capable channel through the existing scoped resolver before invoking plugin auth.
