## Summary

Keep open. Current main still cannot distinguish the no-provider media-understanding skip from an empty provider response at the infer CLI boundary, and this PR is a narrow active implementation that preserves and consumes the skipped/no-attempt decision with focused coverage. I found no blocking correctness or security issue in the patch, but it still needs normal maintainer review and CI or changed-gate validation before merge.

## What This Changes

The PR propagates media-understanding decisions through the file runtime result, maps skipped/no-attempt image and audio infer results to explicit missing-provider errors, updates the audio runtime return type, adds regression tests, and records a changelog fix for #73569.

## Best Possible Solution

Land a narrow version of this fix after maintainer validation so infer image/audio commands report missing media-provider configuration distinctly while retaining the existing empty-description/transcript errors for cases where a provider actually ran and returned no text.
