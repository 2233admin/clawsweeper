## Summary

Current main does not implement this PR's intended change. The shared `/think` command still calls `listThinkingLevels(provider, model)` without a catalog, and `resolveCommandArgMenu`/`resolveCommandArgChoices` still only pass provider/model context. The linked issue remains a plausible active bug, because Ollama's provider thinking profile depends on catalog `reasoning` metadata to expose `off, low, medium, high, max`.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73515. The likely best path is to review and land a catalog-aware command menu fix that builds the configured model catalog in the shared resolver, passes it through `CommandArgChoiceContext`, keeps Telegram channel code generic, and includes the proposed regression coverage for configured Ollama models with `reasoning: true`.
