## Summary

Current main does not implement this PR's intended fix. The shared `/think` native argument menu still resolves choices with provider/model only, while Ollama's provider-owned thinking profile needs catalog `reasoning` metadata to expose `off`, `low`, `medium`, `high`, and `max` for configured reasoning-capable models. The PR remains a plausible active implementation candidate for #73515.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73515. The best path is to review and land a catalog-aware shared command menu fix that keeps Telegram generic, builds or passes the configured model catalog in the shared resolver, threads it through `CommandArgChoiceContext`, and keeps the new regression coverage for configured Ollama models with `reasoning: true`.
