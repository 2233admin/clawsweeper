## Summary

Keep open. The PR targets a real defensive-hardening gap in reply payload normalization, but the current patch is not merge-ready: current main already rejects undefined `chat.send.message` at the protocol schema boundary, the sanitizer hunk would regress supported attachment-only sends, and the reply normalizer hunk must be reapplied to the current `hasContent()`/`normalizeOptionalString()` implementation.

## What This Changes

The PR adds defensive guards around message/text trimming in chat send, auto-reply normalization, chat abort command parsing, and TUI stream assembly.

## Best Possible Solution

Adapt the useful runtime narrowing into current main by normalizing reply payload text once before all string helpers, preserving media-only and channelData-only payloads, and leaving chat.send's existing schema validation and attachment-only behavior intact.
