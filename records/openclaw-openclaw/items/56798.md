## Summary

Keep open. Current main still lacks the PR's net-level auto-selection attempt-timeout change, so the branch is not obsolete; however, the current patch still overwrites an operator who explicitly pins Node's network-family timeout to the default numeric value, so it needs another maintainer review pass before merge.

## What This Changes

The PR adds a guarded `net.getDefaultAutoSelectFamilyAttemptTimeout()` / `net.setDefaultAutoSelectFamilyAttemptTimeout(300)` step inside `ensureGlobalUndiciStreamTimeouts()` and tests default-vs-custom timeout behavior.

## Best Possible Solution

Keep the PR open and revise it so OpenClaw either preserves explicit Node network-family timeout policy, including the 250 ms case, or makes the process-wide timeout change opt-in. The PR body and tests should match the latest implementation, and validation should cover the intended built-in-fetch/provider-fetch path without regressing the existing per-dispatcher behavior.

Please detect explicit flags/NODE_OPTIONS or make this mutation opt-in.
