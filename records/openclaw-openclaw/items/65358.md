## Summary

Keep this open. Current main has useful partial mitigations for backups, reset behavior, compaction, and session history safety, but it still does not implement the central upgrade-safety request: automatic pre-update state/workspace/session snapshots, first-class restore/rollback, targeted CLI session repair, stale legacy override diagnosis/repair, or status diagnostics for missing durable memory roots are still missing or only manual.

## What This Changes

The issue requests stronger upgrade hardening for agent memory/session continuity, including automatic pre-update backups, rollback, session override validation, workspace diagnostics, cron validation, and per-session reset tooling.

## Best Possible Solution

Treat this as an upgrade-safety hardening tracker: add an opt-out pre-update backup using the existing backup planner, provide a supported restore/rollback workflow for state/workspace/session data, expose targeted session reset/delete/override repair from the CLI, and add doctor/status checks for stale session overrides and missing MEMORY.md when daily memory or workspace bindings suggest continuity should exist.
