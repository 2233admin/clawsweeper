## Summary

Keep open. Current main still has the Feishu named-account promotion bug and still re-runs Feishu full registration on each active plugin load; the PR addresses real behavior, but its process-wide registration guard is unsafe because active reloads clear plugin state and then need hooks/tools to be registered again.

## What This Changes

This PR adds Feishu full-registration idempotency, Feishu named-account promotion keys plus fallback behavior, and regression tests for duplicate registration and top-level groupPolicy preservation.

## Best Possible Solution

Revise the fix so Feishu named-account configs preserve shared top-level policy fields through the setup-promotion contract or an explicit early-migration fallback, and solve duplicate Feishu registration with reload-aware registry/load-scoped behavior rather than a process-wide one-shot flag. Keep focused regression coverage for both the legacy config normalization case and active reload registration semantics.
