## Summary

Current main still does not implement time-bounded exec approval trust windows. The PR remains substantive, security-sensitive feature work rather than obsolete cleanup: the current CLI, Gateway RPC list, protocol schema, resolver, exec host merge, docs, and history all still show the existing get/set/allowlist plus persistent or session-only no-prompt paths. It also overlaps with open same-author PR #39126 and broader open permission-design issue #48532, so the conservative maintainer path is review/consolidation, not auto-closing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate it with #39126 and the broader permission-model discussion in #48532, then land one canonical implementation with server-side non-agent authorization, a clear policy decision for `security: "deny"`, robust audit semantics, protocol/client/docs updates, and focused security tests.
