## Summary

Current main still does not implement time-bounded exec approval trust windows. The PR remains substantive, security-sensitive feature work: main exposes approvals get/set/allowlist plus persistent or session-scoped no-prompt paths, while the PR proposes new trust/untrust/status CLI and Gateway RPC behavior with audit and authorization semantics that still need maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate it with #39126 and the broader permission-model discussion in #48532, then land one canonical implementation with server-side non-agent authorization, a clear policy decision for `security: "deny"`, robust audit semantics, protocol/client/docs updates, and focused security tests.
