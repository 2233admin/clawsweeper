## What This Changes

This PR extracts shared Feishu document-link parsing into `extensions/feishu/src/document-link.ts`, switches bitable/comment code to it, adds parser/routing tests, and moves duplicated task SQLite number/JSON helpers into `src/tasks/task-store.sqlite-shared.ts`.

## Best Possible Solution

Proceed with maintainer review and merge if the team accepts the stricter shared Feishu URL parser semantics; otherwise keep the extraction but preserve the current bitable parser’s looser token acceptance in the bitable adapter or cover the intentional behavior change with an explicit test.
