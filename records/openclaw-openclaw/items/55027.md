## Summary

Keep open. Current main still routes Feishu block payloads only into the Card Kit streaming-card path when card streaming is enabled, drops non-card block chunks, and hard-disables the generic block-streaming pipeline, so true progressive Feishu block-message delivery remains missing despite the public docs advertising `channels.feishu.blockStreaming`.

## What This Changes

Request Feishu to deliver each runtime-emitted block progressively as normal Feishu messages or cards when block streaming is enabled, rather than relying on Card Kit playback animation or dropping block chunks.

## Best Possible Solution

Implement the Feishu behavior in the Feishu plugin boundary: resolve an explicit Feishu block-streaming setting, enable the core block pipeline only when that setting is on, deliver `info.kind === "block"` immediately through normal Feishu message/card sends, and keep Card Kit preview streaming as a separate mode with existing duplicate-final suppression intact.
