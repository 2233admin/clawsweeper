## Summary

Current main still resolves mixed MCP server configs with both `command` and `url` as stdio before checking `transport: "streamable-http"`, so the reported timeout path remains plausible. An open linked fix PR, #72515, already uses this issue as its closing target, so the issue should stay open while maintainers review that PR.

## Best Possible Solution

Keep this issue open and review #72515. If maintainers accept early rejection as the intended policy, land that PR or an equivalent fix in `src/agents/mcp-transport-config.ts` with regression tests. If managed child-process HTTP servers are desired instead, implement that explicitly with readiness probing, HTTP connection setup, lifecycle cleanup, and docs/tests rather than leaving the silent stdio fallback timeout as the user-visible behavior.
