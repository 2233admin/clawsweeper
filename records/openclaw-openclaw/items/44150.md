## Summary

Keep open. Current main still omits `messages` from the direct `before_compaction` hook helper, while the hook event type already allows the field and another compaction path already sends it. The PR’s intent is valid and was positively acknowledged in discussion, but the branch needs an update for the later helper split plus the existing review concern: pass a stable pre-compaction snapshot, not the live `session.messages` array, and strengthen the tests beyond `expect.any(Array)`.

## What This Changes

The PR adds `messages: session.messages` to the direct compaction `before_compaction` hook payload and adds focused hook payload assertions in `compact.hooks.test.ts`.

## Best Possible Solution

Expose a defensive pre-compaction message snapshot through the direct `runBeforeCompactionHooks` path, keep the plugin hook contract additive, and add focused assertions that the hook receives the exact pre-compaction contents, including the empty-transcript case, without changing compaction policy or after-compaction behavior.
