## Summary

Keep open. Current main still does not implement this PR's intended direct-compaction behavior: the direct before_compaction helper has access to session.messages while computing metrics, but it only sends messageCount/tokenCount to plugin hooks. The feature is coherent because the hook event type already allows messages and the non-direct compaction-start subscriber path already passes messages, but this PR still needs an updated implementation, stronger tests, and explicit transcript exposure/mutation semantics before it should land.

## Best Possible Solution

Keep this PR open and update it: thread a defensive pre-compaction message snapshot through compactEmbeddedPiSessionDirect/runBeforeCompactionHooks, assert exact pre-compaction contents plus the empty-transcript case in compact.hooks tests, and clarify transcript exposure/mutation semantics before running the focused compaction hook tests and changed gate.
