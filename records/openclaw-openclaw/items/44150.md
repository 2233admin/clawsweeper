## Risks / Open Questions

- As submitted, the PR expands plugin hook data exposure to include full pre-compaction transcript messages, but its Security Impact section says data access scope did not change; that should be corrected before landing.
- Passing `session.messages` by live reference would let a hook mutate the in-progress compaction input or observe post-compaction mutations. A landed version should pass a defensive pre-compaction snapshot unless mutation is explicitly intended and documented.
- The current PR patch targets an older direct call site; current main uses `runBeforeCompactionHooks`, so the implementation needs a rebase/refactor rather than closure.
