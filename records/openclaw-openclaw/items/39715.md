## Summary

Keep open. Current main still lacks the PR's central stdout/stderr drain before process completion and finished-session aggregation snapshots, and the PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregation snapshots, retain the settled/cancel/finalize race safeguards from the PR discussion, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
