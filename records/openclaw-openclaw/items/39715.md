## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: child wait and exec/background completion snapshot stdout/stderr without an event-loop drain turn, and the focused flush-at-exit regression coverage proposed by the PR is absent. The PR remains a live implementation candidate for the stdout/stderr data-loss half of the linked sandbox exec issue.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregate snapshots, preserve the settled/cancel/finalize safeguards already discussed on the PR, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
