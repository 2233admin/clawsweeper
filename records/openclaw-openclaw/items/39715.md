## Summary

Keep open. Current main still lacks the stdout/stderr drain barrier this PR is trying to add: the supervisor snapshots output immediately after `adapter.wait()`, the child adapter resolves normal `close` directly, and finished background session logs copy the aggregate as-is. The linked stdin/write fix does not supersede this stdout completion path.

## Best Possible Solution

Keep this PR open. The best path is to land or adapt a minimal stdout/stderr drain before `RunExit` and finished background-session aggregation are snapshotted, keep the later cancel/finalize race safeguards from the PR discussion, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.

There is no `setImmediate` or equivalent drain before the snapshot.
