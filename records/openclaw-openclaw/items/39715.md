## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: there is no stdout/stderr drain turn between child wait completion and the snapshots used for RunExit, exec completion, and finished background-session logs. The PR remains a focused candidate for the stdout half of #30711, with useful prior review discussion around cancel/finalize races that a landed version should preserve.

## Best Possible Solution

Keep this PR open. The best maintainer path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregate snapshots, preserve the settled/cancel/finalize safeguards already discussed on the PR, and include focused regression coverage for backgrounded stdout/stderr flushed at process exit.

## Evidence
