## Summary

Keep this PR open. The current main branch still has the misleading manual `/compact` label behavior, so the branch is useful, but the proposed fix forwards the model context window as `currentTokenCount`, which violates the compactor contract and would produce inflated `tokensBefore` diagnostics.

## What This Changes

The PR modifies `src/auto-reply/reply/commands-compact.ts` to suppress non-decreasing compaction arrows, pass a token count into manual compaction, and estimate stale session totals from the JSONL file size.

## Best Possible Solution

Land a corrected version that keeps the non-decreasing-label guard and stale-total display fallback, but only passes `currentTokenCount` when the manual handler has a true fresh prompt-token usage value; otherwise omit it and let the compactor use its transcript estimates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass usage tokens, not the context window:** `src/auto-reply/reply/commands-compact.ts:166`

Passing the window size makes manual `/compact` report inflated before values and can weaken the post-compaction token sanity check; omit this field unless a real fresh usage count is available.
