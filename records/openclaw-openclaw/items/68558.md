## Summary

Keep open. Current main still renders the manual `/compact` success label with an unconditional `before â†’ after` arrow, still does not forward any caller-observed `currentTokenCount` into `compactEmbeddedPiSession`, and still formats the post-compaction context summary from `tokensAfter` or `resolveFreshSessionTotalTokens()` only. The PR targets a real remaining UX bug, but the proposed implementation needs maintainer review because its own review thread notes that it forwards `contextTokens` as `currentTokenCount`, while the current type contract says that field is live prompt-token usage, not the model context window.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The right fix likely belongs in `src/auto-reply/reply/commands-compact.ts`: avoid rendering a before-to-after arrow unless `tokensBefore > tokensAfter`, preserve the existing context-window denominator behavior, and pass an actual observed usage estimate only if the manual command has one available; otherwise omit `currentTokenCount` rather than using `contextTokens`. Add focused tests in `src/auto-reply/reply/commands-compact.test.ts` for skipped/no-history compaction, `before <= after` label rendering, and stale token-summary fallback if that fallback remains part of the patch.
