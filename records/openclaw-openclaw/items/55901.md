## Summary

Keep this PR open. Current main still does not implement IRCv3 draft/multiline support: IRC outbound text is still flattened, there is no IRCv3 tag parsing or CAP/BATCH handling, and tests still assert the old 350-character IRC outbound limit. The PR is a focused implementation candidate for a bundled IRC plugin capability, but it needs rebase/review rather than cleanup closure.

## Best Possible Solution

Keep the PR open as the implementation candidate. Rebase it onto current main, move the outbound adapter changes into the current `extensions/irc/src/outbound-base.ts` structure, add focused IRC protocol/client tests for draft/multiline negotiation and fallback behavior, then review or merge after the remaining protocol-length concerns are resolved.
