## Summary

Keep open. Current main still overwrites one Mattermost draft preview post across phase transitions, and this PR is a useful direction, but the patch is not merge-ready because boundary splitting is not lifecycle-aware, nested streaming overrides are not merged, and the public docs/changelog remain out of sync.

## What This Changes

The PR adds Mattermost `streaming.mode` and `streaming.toolPreview` config/schema/account resolution, propagates sanitized tool args into tool-start callbacks, wires draft-preview boundary splitting into Mattermost streaming updates, and adds focused tests plus generated config metadata.

## Best Possible Solution

Land a Mattermost-local block preview mode that freezes only materialized preview posts at phase boundaries, preserves nested streaming defaults across per-account overrides, and keeps schema, generated metadata, public docs, and changelog aligned before merge.

## Reproduction Assessment

Yes. Current main can be traced directly from `monitor.ts` and `draft-stream.ts`: phase hooks update the same draft stream and never split the post. The PR regressions have clear focused reproduction paths with a delayed draft create before a boundary and a mixed channel/account `streaming` override.

## Solution Assessment

No, not as currently implemented. The `block` mode is the right narrow direction, but the split operation must be lifecycle-aware, account config merging must preserve nested defaults, and the public documentation/release notes need to match the new schema.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wait for the preview post before splitting:** `extensions/mattermost/src/mattermost/draft-stream.ts:209`
  - body: `signalBoundary()` can split after content has only been queued. If the first create or prior edit is still pending, `forceNewMessage()` clears pending text and resets local ids without waiting, so the next phase can drop the prior preview or later reuse the old in-flight post. Make the boundary wait for the prior update/create to materialize or cover this lifecycle in the draft stream.
  - confidence: 0.89
- **[P2] Deep-merge streaming account overrides:** `extensions/mattermost/src/mattermost/accounts.ts:118`
  - body: Mattermost account merging still deep-merges only `commands`. An account override like `streaming: { toolPreview: "args" }` replaces a channel-wide `streaming: { mode: "block" }`, silently falling back to partial mode. Add `streaming` to the nested merge keys and test mixed root/account overrides.
  - confidence: 0.86
- **[P2] Align Mattermost streaming docs with the schema:** `extensions/mattermost/src/config-schema-core.ts:127`
  - body: This schema now accepts object-form `streaming.mode` values `partial` and `block`, but the checked-in Mattermost and streaming docs still advertise scalar `streaming: "partial"` plus `off` and `progress` modes. Users following those docs will get rejected or stripped config unless the docs are updated or compatibility is added.
  - confidence: 0.83
