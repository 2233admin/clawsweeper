## Summary

Keep this PR open, but do not merge it as-is. Current main still has the Mattermost edit-in-place preview behavior, and the proposed block-mode direction is useful, but the PR has blocking issues around async preview splitting, nested account config merging, and public config documentation.

## What This Changes

The PR adds Mattermost `streaming.mode` and opt-in `streaming.toolPreview`, propagates sanitized tool args into tool-start preview callbacks, wires a Mattermost draft-preview boundary controller, updates generated channel config metadata, and adds focused tests.

## Best Possible Solution

Land a Mattermost-local block preview mode that freezes only materialized preview posts at phase boundaries, preserves predictable nested `streaming` account overrides, and updates schema, generated metadata, docs, changelog, and focused tests together.

## Reproduction Assessment

Yes. Current main can be reproduced with a Mattermost DM turn that emits thinking, partial text, and tool starts; source confirms all phases update the same draft stream. The PR-specific bugs can be reproduced with a delayed `createMattermostPost` unit test and a mixed root/account `streaming` merge test.

## Solution Assessment

No, not as currently implemented. The `block` mode is the narrow maintainable direction, but the split operation must be lifecycle-aware and the public config surface must be made consistent before merge.
