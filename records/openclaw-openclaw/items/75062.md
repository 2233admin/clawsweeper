## Summary

Keep this PR open, but do not merge it as written. The branch addresses the main state-only uninstall data-loss case and the changelog now describes that behavior, but the new pruning path can follow a symlinked state directory and delete children of the symlink target instead of refusing or removing only the state path.

## What This Changes

The PR changes uninstall state cleanup to preserve configured workspaces nested under the state directory during state-only uninstall, prune other state contents, add cleanup regression tests, and add a changelog entry for #75052.

## Best Possible Solution

Land one canonical uninstall fix that preserves configured nested workspaces during state-only uninstall and removes the rest of state/config, but first make the pruning path symlink-safe by lstat/realpathing the state root and adding regression coverage for symlinked state directories.

## Reproduction Assessment

Yes. The original issue is reproducible from current-main source because the default workspace is under .openclaw and state-only uninstall recursively removes that parent; the PR regression is reproducible by setting OPENCLAW_STATE_DIR to a symlink whose apparent path contains a preserved workspace.

## Solution Assessment

No as written. Selective pruning around preserved workspaces is the right direction, but it needs a symlink-aware unsafe-root/boundary guard before enumerating and deleting state children.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refuse symlinked state roots before pruning:** `src/commands/cleanup-utils.ts:156`
  - body: When `OPENCLAW_STATE_DIR` is a symlink and a preserved workspace appears under that symlink path, this branch calls `fs.readdir` on the symlinked directory and then removes child paths. That follows the symlink target, so an unsafe or external directory can be pruned even though removing the state path itself would not enumerate its children. Please lstat/realpath the state root and refuse or otherwise handle symlinked roots before pruning.
  - confidence: 0.88

## Security Review

Status: needs_attention
