## Summary

Keep this PR open, but do not merge it as written. It addresses a real state/workspace data-loss bug, yet the proposed guard skips the entire state directory when a preserved workspace is nested there, which can leave selected state, config, and auth data behind.

## What This Changes

The PR adds a workspace preservation option to state cleanup, passes workspace directories from uninstall when the workspace scope is unselected, and adds a cleanup-utils dry-run regression test.

## Best Possible Solution

Revise the PR so State + config removal still removes state data that can be safely removed while preserving unselected configured workspaces, or blocks before confirmation with clear messaging when selective cleanup is not safe. Add CLI-level coverage for the default nested workspace layout plus cleanup-helper coverage for inside-state config/oauth behavior.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: the default workspace is under the state directory, state-only uninstall calls recursive state removal, and workspace removal is guarded by the separate workspace scope.

## Solution Assessment

No. The PR prevents deleting the workspace, but skipping the whole state directory is not the narrowest maintainable fix because it leaves the selected state/config cleanup incomplete; selective pruning or an explicit refusal is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep removing state data around preserved workspaces:** `src/commands/cleanup-utils.ts:116-122`
  - body: When a workspace is inside the state dir, this branch skips the whole state-dir removal. In the default layout, config and OAuth paths are also inside that dir and the helper skips linked in-state paths afterward, so selecting State + config can leave the selected state/auth data behind. Preserve the workspace by pruning around it or aborting before confirmation, not by treating state removal as complete.
  - confidence: 0.87
