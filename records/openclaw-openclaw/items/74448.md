## Summary

Keep open. Current `main` still has a source-backed path where a non-yielding terminal `agent.wait` result finalizes the subagent run and freezes the child session's current visible assistant text; I did not find a compaction/resume-chain contract that keeps the parent wait pending until the resumed child turn produces the real final handoff. The shipped `sessions_yield` hardening is adjacent and useful, but it does not close this compaction-as-terminal-success report.

## What This Changes

The issue asks OpenClaw to keep subagent completion pending across internal session compaction so the parent receives the final child handoff instead of a mid-task frozen result.

## Best Possible Solution

Add an explicit compaction-aware lifecycle contract: a subagent run that compacts and resumes should remain non-terminal to its parent until the resumed turn actually ends, then freeze and announce exactly one final child handoff.

## Reproduction Assessment

Yes. The issue provides three same-day long-running subagent reproductions with compaction timestamps, premature parent completion timestamps, and later final child messages; current source inspection matches the reported failure mode.

## Solution Assessment

Yes, the issue's Option A is the narrowest maintainable direction: make `agent.wait` or the subagent lifecycle follow the compaction resume chain instead of compensating by guessing at freeze time. A freeze-time deferral can be defense-in-depth, but it should not be the primary contract.
