## Summary

Keep open. The report is concrete and current, and current main still lacks compaction-marker handling in the subagent output capture path. Related PR #73783 is an open implementation candidate for the same auto-compaction announce bug, but it has not merged, so this should not close yet.

## What This Changes

The issue asks for subagent wait/completion handling to remain pending across session auto-compaction so the parent receives the real final handoff instead of a mid-task frozen result.

## Best Possible Solution

Review and land PR #73783 if it fully covers the compaction continuation case, or extend it so subagent completion stays pending across auto-compaction and freezes exactly one final post-compaction handoff.
