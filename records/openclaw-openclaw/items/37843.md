## Summary

Keep this issue open. Current main still treats trimmed cron `payload.model: "default"` as a non-empty explicit model override, and the fallback policy also treats it as an override. The related implementation PRs were closed unmerged, and current docs/tests still do not document or cover `--model default` as shipped behavior.

## Best Possible Solution

Implement a small shared cron payload-model helper that trims `payload.model` and treats exact `default` as unset. Use it in both `resolveCronModelSelection` and `resolveCronFallbacksOverride`, add focused tests for default/empty/explicit/invalid/fallback behavior, and update cron CLI help/docs if `--model default` is intended to be public.
