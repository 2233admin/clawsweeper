## Summary

Keep open. Current main still treats any trimmed non-empty cron `payload.model` as an explicit model reference, so `model: "default"` is not a runtime-default sentinel. The fallback-policy path also treats the same literal as a model override, CLI add/edit can persist it, docs describe `--model` only as a concrete allowed model, and the related PRs were closed unmerged.

## What This Changes

Support `default` as a special cron agentTurn `payload.model` / `--model default` value that resolves to the agent or system default model at runtime.

## Best Possible Solution

Implement a small shared cron payload-model helper that trims `payload.model` and treats exact `default` as unset. Use it in both `resolveCronModelSelection` and `resolveCronFallbacksOverride`, add focused model-selection/fallback/forwarding tests, update cron CLI help and docs if `--model default` is public, and consider a user-facing changelog entry with reporter/contributor credit.
