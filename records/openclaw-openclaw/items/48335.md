## Summary

Keep this PR open. Current main still does not implement the proposed loop-aware compaction guard hook, and the PR is part of an active same-author stack for #48238 rather than obsolete work. The current compaction path supports static/custom instructions and the existing quality guard, but there is no `agents.defaults.compaction.guard` config contract, no `guardEnabled` runtime wiring, no transcript-tail detector/scorer on main, and no conditional guard augmentation before compaction summarization.

## Best Possible Solution

Keep this PR open. If maintainers still want the loop-aware saturated-session compaction guard from #48238, review and land or close the stack in order (#48278, #48293, #48312, then this PR) after rebasing onto the current `src/agents/pi-hooks` compaction pipeline. If product direction changed, close or redirect the whole stack with an explicit maintainer decision rather than sweeping this PR alone.
