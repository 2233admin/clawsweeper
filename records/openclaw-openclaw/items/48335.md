## Summary

Keep this PR open. Current main does not implement the requested loop-aware compaction guard hook: the config schema/types have no `agents.defaults.compaction.guard`, the safeguard runtime has no `guardEnabled`, and compaction instructions are only resolved/composed without risk-based augmentation. The PR is also part of an open same-author stack tied to #48238, #48278, #48293, and #48312, so it should be reviewed or closed as part of that stack rather than swept independently.

## Best Possible Solution

Keep this PR open. If maintainers still want the loop-aware saturated-session guard from #48238, review or land the stack in order (#48278, #48293, #48312, then this PR) and rebase this PR onto the current `src/agents/pi-hooks` compaction pipeline. If the product direction changed, close the whole stack with an explicit maintainer decision rather than auto-closing one middle PR.
