## Summary

Keep this issue open. Current main has adjacent improvements for ACP task tracking, `sessions_send`, and hook delivery suppression via `deliver: false`, but the central channel-less persistent ACP workflow is still blocked: ACP `mode: "session"` still requires `thread: true`, accepted spawn results still do not expose ACP backend/agent resume identity, and `/hooks/agent` still runs the isolated cron agent path rather than dispatching hook-targeted ACP session keys through `acpManager.runTurn`.

## What This Changes

The issue asks OpenClaw to support hook-triggered agents managing ACP child sessions by tracking ACP runs, allowing unbound persistent ACP sessions, exposing ACP resume identity, and avoiding duplicate hook delivery.

## Best Possible Solution

Define an explicit headless/unbound persistent ACP contract, expose stable ACP resume identity through spawn/status surfaces, add focused regression coverage for hook-triggered channel-less ACP workflows, document whether follow-up turns should use `sessions_send` or hook-side ACP dispatch, and keep `deliver: false` as the hook auto-delivery suppression mechanism.
