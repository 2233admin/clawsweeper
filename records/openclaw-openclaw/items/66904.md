## Summary

Keep open. Current main still does not subscribe to Baileys `messaging-history.set`, does not expose opt-in reconnect-history config, and still skips stale append backlog before it can reach agent-visible history/context. The related history-store and offline-reply PRs are closed unmerged, and #56088 is useful implementation caution for on-demand history fetch rather than a merged fix.

## What This Changes

The issue requests opt-in WhatsApp reconnect history sync by handling Baileys `messaging-history.set` events and making missed offline messages available through OpenClaw history/context buffers.

## Best Possible Solution

Add a WhatsApp-plugin-owned, opt-in reconnect-history path that subscribes to typed `messaging-history.set`, configures the needed Baileys history-sync filter, normalizes and dedupes synced messages through existing inbound/history helpers, and populates DM/group context without auto-replying to stale backlog by default. Keep the core extension-agnostic and update config types, strict schema, generated metadata/docs, changelog, and focused regression tests.
