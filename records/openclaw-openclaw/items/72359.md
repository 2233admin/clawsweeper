## Summary

Keep #72359 open. Current main has implemented the hard timeout sibling from #72347, but it still does not implement the requested Active Memory single-shot mode: the config schema has no `mode`, the runtime always invokes the embedded Pi agent loop with memory tools, and docs/tests still describe the feature as a blocking memory sub-agent path.

## Best Possible Solution

Keep the issue open as a focused Active Memory feature request. The likely implementation belongs in `extensions/active-memory`: add a default-preserving mode selector such as `"subagent" | "single-shot"`, update `openclaw.plugin.json`, config parsing, docs, and regression tests, and route the single-shot mode through one memory search-manager query plus one bounded model completion. If the current plugin runtime lacks a suitable one-shot model-completion SDK seam, add a generic runtime seam first rather than importing core provider internals into the plugin.
