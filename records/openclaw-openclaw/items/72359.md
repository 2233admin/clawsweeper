## Summary

Keep open. Current main still lacks the requested opt-in Active Memory single-shot mode: the plugin config, manifest schema, docs, runtime path, and tests remain centered on the blocking embedded memory sub-agent, so this is not implemented or safely closeable as a duplicate.

## What This Changes

The issue requests a default-preserving Active Memory `single-shot` mode that performs one memory search and one LLM completion instead of invoking the embedded agent loop.

## Best Possible Solution

Add a default-preserving Active Memory mode such as `embedded-agent` plus opt-in `single-shot`, keep the current path as default, and implement the single-shot branch inside the Active Memory plugin using supported memory-search and simple-completion surfaces with schema, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. A static current-main check reproduces the implementation gap: the requested config key is absent, the manifest is strict, and eligible turns still call `runEmbeddedPiAgent`. The exact latency numbers from the report are empirical, but proving the missing capability does not require live provider access.

## Solution Assessment

Unclear pending maintainer product choice. If maintainers accept the feature direction, an opt-in default-preserving plugin-local branch is the narrowest maintainable solution; otherwise this should be consolidated into #65309 rather than implemented as a separate mode.
