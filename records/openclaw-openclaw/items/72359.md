## Summary

Keep open. Current main still does not implement the requested opt-in Active Memory single-shot mode: there is no `mode` or `single-shot` config in the runtime type, normalizer, manifest schema, docs, or tests, and the only recall execution path still calls `runEmbeddedPiAgent` with memory tools. The hard-timeout sibling was handled separately in #72347, while this issue remains the narrow low-latency architecture follow-up and overlaps only partially with broader latency tracker #65309.

## Best Possible Solution

Keep this open as the narrow opt-in Active Memory single-shot mode request. The best implementation is a default-preserving `mode` config, with the existing embedded-agent path remaining the default and a new `single-shot` branch that derives the existing query, performs one bounded memory search through the current memory-search helper, runs one bounded simple completion, parses `NONE` or a summary capped by `maxSummaryChars`, and updates manifest/docs/tests. If maintainers want one tracker, explicitly merge this into #65309 rather than closing it as already implemented.

## Clownfish Work Prompt

Implement the open Active Memory single-shot request as an opt-in mode without changing the default embedded-agent behavior. Add a config field such as `plugins.entries.active-memory.config.mode` with a default value that preserves today’s embedded sub-agent path, plus an opt-in `single-shot` value. In single-shot mode, reuse the existing query construction and eligibility gates, run exactly one bounded memory search through the current Active Memory/memory-host search manager with the configured qmd search mode, build a compact prompt from the user/query plus hits, call the existing simple-completion helper once using Active Memory’s current model resolution, and parse `NONE` or a summary capped by `maxSummaryChars`. Respect `timeoutMs` and abort/fail-open behavior, preserve cache/status/debug/log behavior where practical, do not expose tools or call `runEmbeddedPiAgent` in single-shot mode, and do not change the current multi-turn path for default, recall-heavy, or contextual use cases. Update `extensions/active-memory/openclaw.plugin.json`, focused tests, and `docs/concepts/active-memory.md`. Changelog only if maintainers consider the mode user-facing for the active version, and follow the repository attribution rules if adding an entry.
