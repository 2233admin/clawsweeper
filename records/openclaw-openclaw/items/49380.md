## Summary

Keep this issue open. Current main still has the reported visibility gap: `/status` and `session_status` render the persisted session-store `compactionCount`, while context-engine `afterTurn` and turn-maintenance paths still have no typed callback, return value, stats method, or session-store bridge for plugin-owned leaf/proactive compactions. The related #60332 remains overlapping follow-up, but this issue is older, richer, and still valid rather than obsolete.

## Best Possible Solution

Keep this open and implement an additive plugin-facing compaction reporting contract, such as `notifyCompaction(count?)` on `runtimeContext` for `afterTurn` and maintenance. Core should persist that signal into the session-store `compactionCount` used by `/status` and `session_status`, with focused tests for an `ownsCompaction` context engine that compacts during post-turn work.

## Clownfish Work Prompt

Implement a focused fix for the open issue where plugin context engines with `ownsCompaction: true` can perform leaf/proactive compactions in post-turn work but `/status` and `session_status` still show `Compactions: 0`.

Expected boundary:
- Add an additive, backwards-compatible plugin-facing reporting seam, preferably a typed `notifyCompaction(count?: number)` / `reportCompaction(count?: number)` callback on `ContextEngineRuntimeContext`, available to `afterTurn` and turn maintenance.
- Persist plugin-reported compactions into the session-store `compactionCount` used by `src/status/status-message.ts` and `session_status`.
- Cover foreground `afterTurn`; if background/deferred turn maintenance can compact after run persistence, add a store-safe bridge for that path too.
- Do not query plugin-specific databases, do not hard-code `lossless-claw`, `mem-engine`, or other plugin ids, and do not change plugin ownership boundaries.
- Avoid double-counting built-in auto-compaction, manual `/compact`, timeout recovery, and overflow recovery paths that already increment or propagate counts.
- Update plugin/context-engine docs and SDK API baseline if the public type surface changes. Add a changelog entry only if the final change is user-facing; credit the reporter/contributor per repo policy, never Codex/OpenClaw/steipete.

Related refs: openclaw/openclaw#49380, openclaw/openclaw#60332, openclaw/openclaw#69016, Martian-Engineering/lossless-claw#472.
