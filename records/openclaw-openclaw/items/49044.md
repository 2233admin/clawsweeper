## Summary

Keep this PR open. Current main still does not implement the intended per-account SOUL-file behavior: there is no `--soul` CLI flag, no `soulFile` channel/account config surface, no onboarding SOUL-file step, and runtime bootstrap still loads the fixed workspace `SOUL.md`. The older #42861 attempt is closed unmerged, and the review discussion on this PR already identified the missing schema and runtime bootstrap wiring, so the work is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open only as a candidate for revised feature work. The best path is to rebase onto current main, add a documented channel/account config contract for `soulFile`, update the relevant schemas and UI/docs, wire `channels add` and onboarding to write validated relative paths, pass channel/account context into bootstrap resolution and prompt assembly, make bootstrap caching account-aware, preserve workspace-bound file reads, and add regression tests for default fallback plus named-account behavior.
