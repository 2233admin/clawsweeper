## Summary

Keep this PR open. Current main does not implement the central per-account SOUL-file behavior: `channels add` has no `--soul` option, onboarding has no SOUL-file step, config/runtime do not expose or read `soulFile`, and workspace bootstrap still loads the fixed `SOUL.md`. The PR still represents feature work, though it would need a substantial rebase and end-to-end fixes before it is mergeable.

## Best Possible Solution

Keep the PR open only as a candidate for revised feature work. The best path is an end-to-end implementation: add a documented config/schema contract for account-level SOUL overrides, wire CLI and onboarding to write it, pass channel/account context into bootstrap resolution and prompt assembly, keep bootstrap caches account-aware, read only workspace-safe files, and add tests/docs for default fallback plus named-account behavior. Preserve current main’s existing setup-only `channelSetups` support instead of reintroducing the older snapshot workaround.
