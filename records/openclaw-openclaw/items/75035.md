## Summary

Keep open. This is a maintainer-authored implementation PR and current main does not already contain the requested lifecycle gate, but the proposed patch has blocking issues around fail-closed hook ordering and transcript persistence.

## What This Changes

The PR adds a `before_agent_run` lifecycle gate with pass/block/ask decisions, plugin approval routing, blocked prompt transcript sidecars, runner/UI handling, and targeted tests.

## Best Possible Solution

Keep the focused lifecycle-gate direction, but make blocked or denied `before_agent_run` decisions terminate before any downstream raw-content hooks or model submission, then persist the redacted user transcript through a branch-preserving SessionManager-compatible API that can still carry owner-visible sidecar metadata.

## Reproduction Assessment

Yes for the primary review finding: the diff shows block/deny paths only set `skipPromptSubmission`, while current runner code calls `runLlmInput` outside that guard. A focused unit test with both `before_agent_run` and `llm_input` registered should reproduce the leak without a live provider.

## Solution Assessment

No. The proposed feature boundary is reasonable, but the current patch is not the safest implementation until the gate fails closed before `llm_input` and transcript writes preserve the Pi session DAG through the established manager contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop blocked prompts before llm_input hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2646`
  - body: When `before_agent_run` blocks, denies, or times out, this path only sets `skipPromptSubmission`. The existing `llm_input` hook below is outside that guard, so the blocked `effectivePrompt` is still delivered to raw input hook handlers before the run returns blocked. Return/continue before downstream hook execution, or guard `llm_input` on the blocked state.
  - confidence: 0.9
- **[P2] Use SessionManager for blocked transcript entries:** `src/config/sessions/transcript.ts:421`
  - body: This helper appends a Pi `type: "message"` entry directly with `fs.promises.appendFile`. Pi transcripts are a parentId DAG and current readers use `SessionManager.getBranch()`, so bypassing `SessionManager.appendMessage` can choose the wrong leaf during rewrites or concurrent updates. Add a manager-backed append path that preserves the sidecar metadata.
  - confidence: 0.84
