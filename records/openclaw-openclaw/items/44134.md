## Summary

Keep this issue open. Current main still rebuilds attempt-local tools and invokes provider-owned tool-schema normalization on each embedded attempt, with no reuse layer for unchanged normalized schemas. The issue discussion also has concrete follow-up evidence tying the same path to gateway allocation pressure, so this is still actionable.

## What This Changes

Investigate and reduce repeated Google/Gemini provider tool-schema loading/normalization so unchanged tool schemas are not reprocessed every few minutes or every embedded attempt.

## Best Possible Solution

Implement a conservative normalized-tool-schema reuse layer in the runtime/tool-schema path. Cache only normalized schema data for unchanged tools plus provider/model/API/baseUrl context, rebuild per-attempt AgentTool objects so execute callbacks stay fresh, invalidate on actual tool or context changes, and cover both unchanged reuse and changed-schema/context invalidation with focused tests.
