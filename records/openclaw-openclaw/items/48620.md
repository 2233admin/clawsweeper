## Summary

This issue should stay open. Current main at 28ff82dc still lacks a single admin-facing WhatsApp contacts/permissions view: there is no `contacts` CLI command, `pairing list` remains pending-only, approved senders move to the pairing allow-store, the directory command is config-backed lookup rather than a merged permission view, and channel status is account-level rather than per-contact. Related #56621 is open but narrower because it focuses on approved pairing list/revoke, not group policy or per-contact activity.

## Best Possible Solution

Keep this open and design a conservative read-only admin surface first, likely `openclaw contacts list --channel whatsapp [--account <id>] [--json]`. The read model should merge configured `allowFrom`, pairing allow-store entries, `dmPolicy`, `groupPolicy`, `groupAllowFrom`/`groups`, and only include last-active data if an existing reliable source is identified or a deliberate privacy-conscious store is added. Coordinate the approved-sender subset with #56621, then consider revoke/edit verbs after the list semantics are stable.
