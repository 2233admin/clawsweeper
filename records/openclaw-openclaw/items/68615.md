## Summary

Keep open. Current main has partial related fixes and clearer OpenAI/Codex documentation, but the broader stale Codex routing problem is not fully resolved: discovery loads can still execute plugin registration in non-activating mode, stored session overrides can still select an old provider/model when no allowlist rejects them, and the issue has a newer reproduction on openclaw@2026.4.25 showing CLI embedded fallback selecting openai-codex after defaults moved to Copilot.

## What This Changes

The issue asks OpenClaw to stop message-channel and embedded runs from using stale codex or openai-codex routes after Codex has been disabled or replaced by another configured model route.

## Best Possible Solution

Keep this as the umbrella until maintainers split it into focused follow-ups: one for discovery/register semantics, one for stale session/model override cleanup after defaults change, one for auth-profile ordering cleanup, and one regression covering message or CLI embedded fallback after moving from Codex to a proxy/Copilot route.
