## Summary

Current main still does not implement the requested agent-visible pre-compaction warning. The memory-flush config lacks `visibleWarning` / `warningPrompt`, the strict schema would reject those keys, and the runtime still runs memory flush as a silent `NO_REPLY` housekeeping turn; the adjacent `notifyUser` feature only sends user-channel compaction notices.

## What This Changes

Add optional `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config to inject an agent-visible pre-compaction warning before the existing silent memory flush.

## Best Possible Solution

Keep this issue open for maintainer product/API judgment. A core implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject one deduped agent-visible warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add focused config/runtime/docs tests. If maintainers decide this belongs outside core, first add or document a plugin/runtime API that can safely inject an agent-visible pre-threshold warning.
