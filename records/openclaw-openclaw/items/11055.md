## Summary

Keep open. Current main still does not implement the requested agent-visible pre-compaction warning: the memory-flush config has no `visibleWarning` or `warningPrompt` fields, the strict schema would reject them, and runtime memory flush remains a silent embedded run. The adjacent `notifyUser` implementation only sends status notices to the user’s channel, which does not solve the requested model-visible warning moment.

## What This Changes

Add optional `memoryFlush.visibleWarning` and `memoryFlush.warningPrompt` behavior so the agent receives a visible pre-compaction warning before auto-compaction while preserving the existing silent memory flush.

## Best Possible Solution

Decide the product/API boundary, then either add a validated `agents.defaults.compaction.memoryFlush.visibleWarning` / `warningPrompt` surface that injects one deduped agent-visible warning before the soft-threshold flush while preserving silent memory flush, or explicitly document/expose a plugin/runtime hook that can implement this outside core.
