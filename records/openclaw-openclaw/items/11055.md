## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the `memoryFlush` config has no `visibleWarning` or `warningPrompt` keys, the strict config schema would reject them, and the memory-flush runner still launches a silent embedded memory turn. The adjacent merged `notifyUser` work only posts start/completion notices to the user's channel, and the broader compaction-lifecycle request in #38520 is still open, so this is not implemented or superseded.

## What This Changes

The issue requests optional `memoryFlush.visibleWarning` and `warningPrompt` settings that would inject an agent-visible warning before automatic compaction while keeping the existing silent memory flush.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or a documented runtime/plugin extension point that can add the warning while preserving silent memory flush and user-channel `notifyUser` behavior.
