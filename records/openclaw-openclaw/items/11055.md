## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memoryFlush type and strict schema have no visibleWarning or warningPrompt fields, the memory flush path still runs as a silent embedded memory turn, and the existing notifyUser path only sends user-channel status notices. Prior issue discussion also keeps this routed for maintainer product/API judgment rather than cleanup closure.

## What This Changes

The issue requests optional memoryFlush.visibleWarning and warningPrompt config that injects an agent-visible warning before compaction so the model can save in-progress state.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible warning under agents.defaults.compaction.memoryFlush or a documented runtime/plugin extension point that can safely inject that warning before the soft-threshold flush while preserving silent memoryFlush and user-channel notifyUser behavior.
