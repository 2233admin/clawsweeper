## Summary

Keep open. Current main still has silent pre-compaction memory flush and opt-in user-channel compaction notices, but it does not implement the requested `memoryFlush.visibleWarning`, `warningPrompt`, or a deduped agent-visible pre-compaction warning turn. The related channel-notice work is adjacent, not a substitute for this agent-facing API/behavior.

## Best Possible Solution

Keep this issue open for maintainer product/API judgment. A core implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject one deduped agent-visible warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add focused config/runtime/docs tests. If this belongs outside core, add an explicit plugin/runtime API for safe pre-threshold agent-visible injection first.
