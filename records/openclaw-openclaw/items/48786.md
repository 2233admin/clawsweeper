## Summary

Keep open. Current main still parses Feishu fetched quoted/root/thread message content without reading or normalizing fetched-message mention metadata, while the live inbound path already normalizes Feishu mention placeholders. The issue is also paired with open PR #48976, so it should remain open until that implementation candidate is reviewed, narrowed, landed, or closed.

## What This Changes

The issue asks Feishu fetched quoted/replied message content to resolve raw @_user_N mention placeholders into readable Feishu mention tags or names before adding that content to agent context.

## Best Possible Solution

Review and narrow PR #48976 or land an equivalent Feishu-plugin-only patch that models fetched-message mentions, adapts the SDK mention shape into the existing normalization helper, applies it after fetched content parsing for quoted/root/thread messages, and adds focused send.test coverage for text and post/thread content containing @_user_N plus matching mention metadata.
