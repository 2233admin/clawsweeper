## Summary

Keep open. Current main still leaves Feishu fetched quoted, root, and thread message content on the raw parsed `body.content` path without modeling or applying fetched-message mention metadata, while the live inbound path already normalizes Feishu mention placeholders. Open PR #48976 is already paired with this issue using closing syntax, so the issue should stay open until that PR or an equivalent focused fix lands.

## What This Changes

Normalize Feishu `@_user_N` placeholders in fetched quoted/replied message content so quoted context shows resolved `<at user_id="...">name</at>` tags or readable names.

## Best Possible Solution

Review and narrow PR #48976, or land an equivalent Feishu-plugin-only patch that models fetched-message `mentions`, adapts the fetched API shape into the existing mention-normalization helper, applies normalization after fetched content parsing for quoted/root/thread messages, and adds focused `extensions/feishu/src/send.test.ts` coverage for text and post/thread content containing `@_user_N` plus matching mention metadata.
