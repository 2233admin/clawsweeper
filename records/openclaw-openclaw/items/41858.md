## Summary

Keep open. Current main has an alternate root-and-filter based mitigation recorded for the linked FD-exhaustion report, but it has not implemented this PR's actual depth-bound watcher hardening. Both current skills and builtin memory chokidar watchers still run without `depth` or `followSymlinks`, and chokidar's upstream contract says traversal is recursive unless bounded.

## Best Possible Solution

Keep this PR open as a focused resource-hardening candidate, or replace it with a rebased patch against `src/agents/skills/refresh.ts` and `extensions/memory-core/src/memory/manager-sync-ops.ts`. The final fix should deliberately choose depth, `followSymlinks`, and `extraPaths` behavior, preserve or update the documented recursive memory contract, and add regression tests for the watcher options.
