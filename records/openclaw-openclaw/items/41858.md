## Summary

Keep this PR open. Current main shipped an alternative root-and-filter mitigation for the linked FD-exhaustion report, but it has not definitely implemented this PR's intended explicit depth-limit watcher hardening. The active skills, builtin memory, and QMD memory watchers still call chokidar without `depth`, and the old memory file targeted by the PR has moved into the `memory-core` plugin.

## Best Possible Solution

Keep this PR open as a focused hardening candidate, or replace it with a rebased patch against `src/agents/skills/refresh.ts` and `extensions/memory-core/src/memory/manager-sync-ops.ts`. Maintainers should choose the intended depth, symlink, and `extraPaths` contract, include QMD if it shares the risk, update public memory docs if recursive watch behavior changes, and add regression tests asserting the chokidar options.
