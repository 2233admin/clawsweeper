## Summary

Keep open. Current main still lacks the explicit chokidar depth or symlink policy this PR is trying to add, and the active skills, builtin memory, and QMD memory watchers all still call chokidar without `depth` or `followSymlinks`. The PR is not merge-ready as-is because its memory hunk targets the removed `src/memory/manager-sync-ops.ts` path; the active implementation is now under `extensions/memory-core/src/memory/`.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it unchanged. The best path is a rebased watcher hardening patch against `src/agents/skills/refresh.ts`, `extensions/memory-core/src/memory/manager-sync-ops.ts`, and `extensions/memory-core/src/memory/qmd-manager.ts`, with an explicit depth/symlink policy, regression tests asserting chokidar options, and a docs update if recursive memory watcher semantics change.
