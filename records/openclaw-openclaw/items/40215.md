## Summary

Keep this issue open. Current main still has the data in status, diagnostics, and Web UI session usage, but the channel `/usage full` response footer remains a per-response token/cache/cost line with only the session key added in full mode. It does not include active model, cumulative context used/window/percentage, a `context` mode, or inherited `agents.defaults.responseUsage`. Open PR #69618 explicitly targets this issue with closing syntax and is still open/unmerged, so it is an implementation candidate rather than evidence to close.

## Best Possible Solution

Keep this issue open while maintainers review #69618, #48635, or a smaller replacement. The maintainable path is to enrich the normal channel `/usage full` footer with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, reply-runner composition, slash command behavior, config schema/docs, TUI/gateway patch behavior, and reply/session tests.
