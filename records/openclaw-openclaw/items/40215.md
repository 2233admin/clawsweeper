## Summary

Keep this issue open. Current main still does not implement cumulative model/context-window data in the normal channel `/usage full` reply footer, and the requested inherited `agents.defaults.responseUsage` config is also absent. Open PR #69618 explicitly closes this issue and remains the implementation candidate, so this should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while maintainers review #69618 or replace it with a focused patch. The implementation should enrich the normal channel `/usage full` footer with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving explicit session overrides, and cover formatter, reply-runner, slash-command, config schema/docs, gateway/TUI/Web UI, and session tests.
