## Summary

Keep this issue open. Current main still only appends per-response token/cache/cost data to the `/usage` footer, with `full` adding the session key; it does not show active model, cumulative context used/window/percentage, or an inherited `agents.defaults.responseUsage` default. Open PR #69618 still appears to be the closing implementation candidate, so the issue should remain open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while #69618, #48635, or a smaller replacement is reviewed. The maintainer path is to enrich `/usage full` with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, runner composition, slash command behavior, config schema/docs, and reply/session tests.
