## Summary

Keep this issue open. Current main still formats the `/usage` footer from per-response token/cache/cost data, with `full` adding only the session key; it does not include the active model, cumulative context used/window/percentage, or an inherited `agents.defaults.responseUsage` default. Open PR #69618 is a closing implementation candidate and remains unmerged, so the issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while #69618, #48635, or a smaller replacement is reviewed. The maintainer path is to enrich `/usage full` with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, runner composition, `/usage` command behavior, config schema/docs, and reply/session tests.
