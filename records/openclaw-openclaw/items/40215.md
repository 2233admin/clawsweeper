## Summary

Keep this issue open. Current main still keeps the normal channel `/usage full` footer limited to per-response tokens/cache/cost plus the session key; it does not include active model, cumulative context used/window/percentage, or an inherited `agents.defaults.responseUsage` default. Open PR #69618 explicitly closes this issue and remains the implementation candidate, so the issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while maintainers review #69618 or replace it with a focused patch. The fix should enrich the normal channel `/usage full` footer with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving explicit session overrides, and cover formatter, reply-runner, slash-command, config schema/docs, gateway/TUI/Web UI, and session tests.
