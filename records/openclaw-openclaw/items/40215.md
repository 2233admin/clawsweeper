## Summary

Keep this issue open. Current main still only appends per-response token/cache/cost usage, with `full` adding the session key. It does not include model, cumulative context used/window/percentage, or an inherited `agents.defaults.responseUsage` default. Open #69618 is a closing implementation candidate, so this issue should remain open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while #69618, or a smaller replacement, is reviewed. The implementation should enrich `/usage full` with model plus cumulative context/window/percentage data, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, reply runner, `/usage` command, config schema/docs, session patch behavior, and reply-composition tests.
