## Summary

Keep open. The PR targets a real config/schema gap, but it is not mergeable as-is because it makes provider-level `api: "google-vertex"` validate without preserving that inherited API through the inline-provider runtime path that selects the native Vertex transport.

## What This Changes

The PR adds `google-vertex` to the model API enum/generated config schema and adds tests for native Vertex config validation and Google provider normalization.

## Best Possible Solution

Revise the PR so the accepted `google-vertex` config value is handled consistently by the schema, inline provider runtime normalization, and Google provider stream selection. The narrow fix is to add `google-vertex` to `normalizeResolvedTransportApi` and add focused coverage proving a provider-level native Vertex API is inherited by inline models.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to trace a config with provider-level `models.providers["google-vertex"].api = "google-vertex"` and models that omit per-model `api` through `buildInlineProviderModels`; after this PR validates that config, the runtime still materializes `api: undefined`.

## Solution Assessment

No. Adding the enum and regenerating the schema is necessary, but it is not sufficient because the runtime allowlist remains narrower than the config contract; the safer fix keeps validation and model materialization in lockstep.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve provider-level Vertex APIs:** `src/config/types.models.ts:15`
  - body: Adding `google-vertex` to `MODEL_APIS` makes provider-level `api: "google-vertex"` pass config validation, but `buildInlineProviderModels` still filters inherited provider APIs through `normalizeResolvedTransportApi`, whose switch does not include `google-vertex`. For configs where the provider sets the API and models omit it, the resulting inline model has `api: undefined`, so the Google plugin never selects the native Vertex stream. Add the runtime allowlist entry and a provider-level inheritance test.
  - confidence: 0.9
