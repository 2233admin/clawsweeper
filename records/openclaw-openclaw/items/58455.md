## Summary

Keep open. Current main has related compaction safeguards and a pluggable compaction-provider seam, but it does not implement this PR’s central changes: microCompact-style old tool-result clearing, analysis-scratchpad prompt/strip handling, or a post-compact transcript reference. The PR is a concrete implementation attempt for still-open #58398 and has useful tests, but the provided review context also shows remaining correctness concerns, so it needs maintainer/author iteration rather than auto-close.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to either update this PR to fit current main and address the review comments, or split the work under #58398 into smaller changes: core-safe microCompact behavior, robust scratchpad parsing, and transcript-reference handling, with targeted compaction tests for each.
