## What This Changes

The branch adds a metadata-only auth-profile mode-mismatch hint to `resolveApiKeyForProvider`'s final missing-auth error and covers the masked `openai-codex:default` OAuth profile case.

## Best Possible Solution

Land this targeted diagnostic once CI is green and maintainer review is satisfied, while keeping the broader legacy OAuth alias-normalization policy in #47964 unless maintainers choose to scope it into this branch.

## Reproduction Assessment

Yes. On current main, configure `auth.profiles.openai-codex:default.mode` as `api_key` while the auth store contains that same profile as `type: "oauth"`; `resolveAuthProfileOrder` filters it out as `mode_mismatch`, and `resolveApiKeyForProvider` falls through to the generic missing API key error.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix for the newly documented mode-mask failure because it preserves auth precedence and reports only profile/provider/mode/type metadata; broader legacy OAuth alias normalization should remain separate in #47964.
