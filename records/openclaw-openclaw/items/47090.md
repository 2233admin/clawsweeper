## Summary

Keep this PR open. Current main still does not implement the PR's central session-archiving feature: there is no session `archivedAt` state, no Gateway `includeArchived` list parameter, no `archivedAt` patch support, no Control UI archive/unarchive controls, and no archive-summary write path. The existing archive code on main is transcript-file archival for delete/reset/pruning, which is a different behavior from hiding/restoring sessions while preserving history.

## Best Possible Solution

Keep the PR open, but require a rebase and focused maintainer review. The best implementation path is an additive Gateway/session contract for `SessionEntry.archivedAt`, `sessions.list includeArchived`, and `sessions.patch archivedAt`; Control UI Archive/Unarchive plus Archived filtering; preserved lookup paths for archived sessions; auto-unarchive on inbound activity; focused Gateway/UI tests; and a reviewed summary-writing design. Split the Telegram command and unrelated docs tweak unless maintainers explicitly want those surfaces in the same PR.
