## Summary

Current main still has the reported gap. Direct reply delivery resolves and prepends `messages.responsePrefix`, including the account/channel/global cascade, but `message(action=send)` normalizes the raw message, applies only cross-context decoration, and forwards that unprefixed text through gateway plugin dispatch, plugin fallback, and core `sendMessage`. The earlier #9103 PR attempted this area but is closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and implement the prefix once in the shared message-tool `send` boundary before gateway plugin dispatch, plugin fallback dispatch, core fallback delivery, and outbound mirroring. Resolve the same effective `messages.responsePrefix` cascade used by direct replies with agent, channel, and account context, preserve idempotency/template behavior, and add focused regression coverage for plugin, core, media/caption, already-prefixed, and presentation-only cases.
