## Summary

Current main still has the reported gap. Direct reply delivery resolves and applies `messages.responsePrefix`, including per-channel/account cascade, but the message-tool `send` path normalizes the raw message, applies only cross-context decoration, then forwards that same text through gateway plugin dispatch, plugin fallback, and core `sendMessage` without resolving or applying the effective response prefix. The related #9103 PR attempted this area but was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open. Implement prefix application once in the shared message-tool `send` boundary before gateway plugin dispatch, plugin fallback, and core fallback delivery. Resolve the same effective `messages.responsePrefix` cascade used by direct replies with agent, channel, and account context; preserve template interpolation where model context is available; add focused regression coverage for `runMessageAction`/`handleSendAction`, plugin dispatch, core fallback delivery, media-only/presentation-only sends, and already-prefixed text.
