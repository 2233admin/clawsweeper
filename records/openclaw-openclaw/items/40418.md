## Summary

Keep open. Current main has useful session-memory and startup-context plumbing, but it still preserves recent turns as raw role-prefixed text and preloads memory files verbatim; the requested structured synthesis, anchor/receipt generation, and daily summary flow are not implemented, and the related implementation and poisoning reports remain open.

## What This Changes

Request automatic `/new` session preservation that saves session state/transcripts and synthesizes prior sessions into compact linked daily memory for future sessions.

## Best Possible Solution

Adopt or replace the open synthesis design with a bounded, sanitized continuation package: keep full transcripts as receipts, write compact durable summaries with links or anchors, align output with startup-context loading, and avoid promoting raw transcript artifacts back into the next session.
