## Summary

Keep open. Current main has an explicit session-memory hook for `/new` and `/reset`, plus startup-context loading of recent memory files, but it still stores raw recent user/assistant turns in slugged Markdown files and does not implement the requested synthesis, linked full-transcript receipt, or canonical compact continuation package. Open PR #50584 is a live implementation candidate for part of this request, and open #69943 shows why the raw-memory path needs careful sanitization before expanding it.

## What This Changes

The issue requests automatic `/new` session-memory preservation with saved state, linked full transcript export, and startup synthesis into compact daily memory.

## Best Possible Solution

Review #50584 or a narrow replacement that keeps preservation opt-in and bounded, synthesizes durable decisions/TODOs/preferences into a compact continuation package, records links or receipts to full transcripts without injecting full transcripts by default, sanitizes unsafe role/tool tokens, and aligns startup-context loading with dedupe-aware tests and docs.

## Reproduction Assessment

Yes for the remaining behavior gap: the source path for `/new`/`/reset` writes raw recent messages to a slugged memory file, and startup-context only reloads bounded daily/slugged notes. No live run was attempted because this was a read-only review.

## Solution Assessment

Unclear as written. The layered summary-with-receipts direction from the discussion is safer than unconditional full transcript promotion; the best maintainable fix should be opt-in, sanitized, and integrated with the existing session-memory and startup-context surfaces.
