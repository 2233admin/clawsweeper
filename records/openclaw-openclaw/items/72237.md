## Summary

Keep open. Current main still lacks the realtime consult override surfaces, and the PR is a plausible additive implementation; the only concrete defect I found is misleading Talk docs that show optional override values inside defaults examples.

## What This Changes

The PR adds Voice Call and Control UI Talk realtime consult thinking-level and fast-mode config/schema/docs wiring, passes the overrides into consult agent runs and chat.send one-shot options, and adds focused tests.

## Best Possible Solution

Land or adapt the additive override implementation after correcting the Talk defaults examples and getting a green relevant changed gate, while keeping #71849 open for the separate fast memory/context design.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the missing capability: the strict schemas, Talk session result, chat.send schema, Voice Call runtime, and shared consult runtime do not carry consultThinkingLevel or consultFastMode.

## Solution Assessment

Yes for this narrower override feature. Additive config plus one-shot runtime handoff preserves existing unset behavior and is narrower than the separate live-call fast context retrieval work.
