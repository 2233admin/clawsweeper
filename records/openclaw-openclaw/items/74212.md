## Summary

Current main still reproduces the reported behavior: SSH/headless environments are classified as remote, `models auth login` passes that remote flag into the OpenAI Codex provider auth method, and the provider receives the real device user code but replaces it with `Code: [shown on the local device only]` in the remote-mode prompt. This is a valid, narrow OpenAI Codex plugin bug rather than an already-implemented cleanup item.

## Best Possible Solution

Keep this issue open and fix the OpenAI Codex plugin-owned device-code presentation path so interactive SSH/headless `openclaw models auth login --provider openai-codex` users can see the short-lived code needed to complete pairing. The likely change belongs in `extensions/openai/openai-codex-provider.ts`, with regression coverage in `extensions/openai/openai-codex-provider.test.ts`; preserve the existing token exchange helper and avoid logging the code outside the interactive prompt.

Do not change the OpenAI device-code endpoints, token exchange behavior, profile persistence, or provider catalog behavior.
