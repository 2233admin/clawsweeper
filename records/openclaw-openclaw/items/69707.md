## Summary

Keep open. The PR targets a real memory-lancedb compatibility bug, but current main still forwards configured `embedding.dimensions` in the OpenAI-compatible `/embeddings` request and has a regression test expecting that outbound field. The PR branch also patches the older SDK-helper shape instead of current main's direct `client.post("/embeddings")` path, so it needs a rebase/update before it can land safely.

## What This Changes

The PR changes memory-lancedb's OpenAI embeddings wrapper and tests so configured dimensions are omitted from the upstream embeddings request, returned vectors are truncated locally, and short or invalid dimensions fail clearly.

## Best Possible Solution

Keep this PR open and update it against current main. The fix should remove `dimensions` from the initial direct `/embeddings` request body for the raw OpenAI-compatible path, preserve `ensureGlobalUndiciEnvProxyDispatcher`, `client.post`, timeout/maxRetries, and `normalizeEmbeddingVector`, then validate and locally truncate the normalized vector when `embedding.dimensions` is configured. The existing outbound-dimensions test should become omitted-field, local-truncation, short-vector failure, and invalid-dimension coverage.
