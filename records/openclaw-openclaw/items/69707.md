## Summary

Keep open. Current main still forwards configured `embedding.dimensions` to the OpenAI-compatible `/embeddings` request, so the compatibility bug remains real, but the current PR head has a blocking test mismatch introduced by the new config validation: `dimensions: 0` is now rejected during plugin registration, so the added runtime test cannot reach its asserted `memory_recall` error path.

## What This Changes

The PR updates memory-lancedb source, config parsing, manifest metadata, and tests so OpenAI-compatible embeddings omit `dimensions` upstream, locally truncate returned vectors, and reject non-positive configured dimensions.

## Best Possible Solution

Land a narrow memory-lancedb fix that preserves the current direct `/embeddings` transport, proxy setup, timeout/retry behavior, and response normalization, while omitting upstream dimensions, locally truncating normalized vectors, and keeping config/manifest/tests aligned with the earliest point where invalid dimensions are rejected.
