## Summary

Keep this PR open for normal maintainer review. Current main still forwards `dimensions` to the OpenAI-compatible embeddings request, while the PR narrowly removes that outbound field, truncates normalized vectors locally, and addresses the earlier non-positive-dimensions review thread with config and manifest validation plus focused tests.

## What This Changes

The PR updates `memory-lancedb` source, config parsing, plugin manifest metadata, and tests so OpenAI-compatible embedding requests omit `dimensions` and returned vectors are locally truncated to the configured dimension.

## Best Possible Solution

Land a narrow memory-lancedb fix that preserves the current direct `/embeddings` transport, proxy setup, timeout/retry behavior, and response normalization while omitting upstream dimensions, truncating normalized vectors locally, and keeping runtime config, manifest metadata, and regression tests aligned.

Yes. Current main has a source-level reproduction because the OpenAI-compatible request body includes `dimensions`, and the current test explicitly expects that outbound field; the PR body also describes a concrete vLLM-compatible provider reproduction.

## Solution Assessment
