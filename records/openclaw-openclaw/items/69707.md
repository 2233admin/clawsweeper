## Summary

Keep this PR open. Current main still forwards configured `embedding.dimensions` in the `memory-lancedb` OpenAI-compatible `/embeddings` request and still has a regression test expecting that outbound field, so the PR is not implemented on main. The PR idea remains valid, but the branch is based on the older `client.embeddings.create()` implementation; current main now uses direct `client.post("/embeddings")` plus `normalizeEmbeddingVector`, timeout, and retry behavior that a rebase must preserve.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase. The fix should update `OpenAiCompatibleEmbeddings.embed` so the direct `/embeddings` request body contains only `model` and `input`, preserve `client.post` timeout/maxRetries behavior and `normalizeEmbeddingVector`, then validate and locally truncate the normalized vector to configured `embedding.dimensions`. Keep focused regression coverage in `extensions/memory-lancedb/index.test.ts`; docs only need a small note if maintainers want to clarify that `dimensions` is local for OpenAI-compatible providers.
