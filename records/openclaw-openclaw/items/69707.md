## Summary

Keep open. Current main still sends configured `embedding.dimensions` in the OpenAI-compatible `/embeddings` request and still has a regression test expecting that field, so this PR is not obsolete or implemented on main. The updated PR branch targets the right memory-lancedb source/test surface, but exact-head CI is failing and the remaining validation question belongs in normal maintainer review rather than an automated replacement branch.

## What This Changes

The PR changes memory-lancedb's OpenAI-compatible embedding path to omit `dimensions` from `/embeddings` requests, truncate returned vectors locally, and add regression tests for local truncation, short vectors, and non-positive dimensions.

## Best Possible Solution

Land a narrow memory-lancedb fix that omits `dimensions` on OpenAI-compatible requests, preserves the current direct `client.post("/embeddings")` transport with proxy, timeout/retry, and response normalization behavior, locally truncates only after validating configured dimensions, and keeps focused regression coverage for omitted-field, truncation, and failure paths.
