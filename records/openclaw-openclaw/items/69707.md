## Summary

Keep open. Current main still forwards configured memory-lancedb embedding dimensions upstream, while the current PR head addresses the compatibility bug with a narrow source/config/manifest/test change and I did not find a blocking review finding. Merge readiness still depends on normal maintainer review and exact-head validation.

## What This Changes

The PR changes memory-lancedb so OpenAI-compatible embedding requests omit `dimensions`, returned vectors are truncated locally, non-positive dimensions are rejected in config/manifest validation, and focused regression tests cover those paths.

## Best Possible Solution

Land a narrow memory-lancedb fix that preserves the current direct `/embeddings` transport, proxy setup, timeout/retry behavior, and response normalization while omitting upstream dimensions, truncating normalized vectors locally, and keeping runtime config, manifest metadata, and regression tests aligned.
