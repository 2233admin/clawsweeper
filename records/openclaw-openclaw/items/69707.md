## Summary

Keep open. Current main still forwards configured memory-lancedb embedding dimensions upstream, so the compatibility bug is real; the latest PR head now targets that behavior directly and appears to address the earlier zero-dimension test problem. This is an active implementation PR that needs normal maintainer review and exact-head CI completion, not closure or an automated replacement.

## What This Changes

The PR updates memory-lancedb source, config parsing, manifest metadata, and tests so OpenAI-compatible embedding requests omit `dimensions`, returned vectors are truncated locally, and non-positive dimensions are rejected.

## Best Possible Solution

Land a narrow memory-lancedb fix that preserves the current direct `/embeddings` transport, proxy setup, timeout/retry behavior, and response normalization while omitting upstream dimensions, truncating normalized vectors locally, and keeping runtime config, manifest metadata, and regression tests aligned.
