## Summary

Current main does not implement this PR’s intended change. The shipped `sessions_yield` path only records a yielded/end_turn control flow and completion stop/finish metadata; there is no `completion-truth` module, no `onCompletionTruth` callback, and no `meta.completion.truth` or `truthSelection` surface on main. This PR should remain open as an implementation candidate rather than be closed by cleanup.

## Best Possible Solution

Keep this PR open for normal maintainer review. The likely path is to rebase or repair mergeability, review the internal/public API boundary around `createOpenClawCodingToolsInternal`, add or justify deterministic non-live coverage for the full `sessions_yield` path, and verify redaction before exposing completion truth in embedded runner metadata.
