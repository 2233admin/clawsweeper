## Summary

Keep this PR open. Current main still lacks `sessions_broadcast`, so the contribution is not obsolete, but the latest branch is not merge-ready because it adds a new cross-session injection primitive without matching current provenance validation and restricted-tool policy boundaries.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the OpenClaw tool catalog and tool factory, and adds unit tests for broadcasting messages to active sessions with optional channel filtering.

## Best Possible Solution

If maintainers want this capability in core, land it as a policy-aware sibling of `sessions_send`: use the declared `typebox` dependency, reuse or formally extend the gateway provenance contract, annotate received content as non-user inter-session/system data, add matching deny/mutation coverage across subagents and direct Gateway/realtime tool entry points, and document the intended broadcast versus peer-delivery boundary.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the PR sends `inputProvenance.kind: "broadcast"` while current main's gateway schema only accepts `external_user`, `inter_session`, and `internal_system`; the policy gap is visible by comparing the new default registration with existing `sessions_send` deny surfaces.

## Solution Assessment

No. The PR is a useful implementation sketch, but it is not the best mergeable solution because it bypasses current dependency, gateway provenance, and restricted-tool contracts for cross-session injection.
