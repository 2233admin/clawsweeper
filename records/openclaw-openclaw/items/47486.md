## Summary

Keep open. Current main still has no `sessions_broadcast` tool, but this PR is not merge-ready because the proposed tool uses an undeclared TypeBox import, sends an invalid provenance kind that current gateway validation rejects, and exposes a new cross-session injection surface without matching the deny-list coverage already applied to `sessions_send`.

## What This Changes

This PR adds a built-in `sessions_broadcast` session tool, registers it in the core tool catalog and OpenClaw tool factory, and adds tests for broadcasting messages to active sessions.

## Best Possible Solution

If maintainers want this as a core primitive, implement it as a policy-aware sibling to `sessions_send`: use the existing `inter_session` provenance contract or intentionally extend the schema with docs/tests, annotate received content as non-user inter-session data, add deny coverage for subagents, Gateway HTTP, realtime direct tools, and mutation tracking, then update session-tool docs and profile/group expectations.

Full review comments:

- **[P1] Import TypeBox from the declared package:** `src/agents/tools/sessions-broadcast-tool.ts:2`
  - body: Current package metadata declares `typebox`, and existing tools import `Type` from that package. Importing `@sinclair/typebox` here relies on an undeclared package name, so installs using pnpm's strict layout can fail to resolve the new tool.
  - confidence: 0.9
- **[P1] Use a valid input provenance kind:** `src/agents/tools/sessions-broadcast-tool.ts:142`
  - body: Every broadcast `agent` call sends `inputProvenance.kind: "broadcast"`, but current gateway validation only accepts `external_user`, `inter_session`, and `internal_system`. These requests can be rejected as invalid before reaching the target sessions.
  - confidence: 0.94
- **[P1] Apply broadcast to restricted-tool policy:** `src/agents/openclaw-tools.ts:186-191`
  - body: Registering this tool in the default tool set exposes a new cross-session injection path, but the PR does not add `sessions_broadcast` to the subagent, Gateway HTTP, or realtime direct-tool deny surfaces that already block `sessions_send`. Add matching policy coverage and tests before enabling it broadly.
  - confidence: 0.88
