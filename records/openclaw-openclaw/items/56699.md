## Summary

Keep open. Current main still has the Telegram native-command gap: regular Telegram group messages use topic/group policy overrides, but the native slash-command authorization path explicitly disables those overrides. There is also an open, unmerged implementation PR, so the right maintainer path is to review/land or close that PR rather than close this issue early.

## Best Possible Solution

Review #56741 or land an equivalent Telegram plugin patch that changes native command authorization to use topic/group overrides, plus add a regression test for a forum-topic `groupPolicy: "disabled"` native command. Once that lands on main, close this issue as implemented.
