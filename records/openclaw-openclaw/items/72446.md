## Summary

Keep open. Current main still allows every succeeded Microsoft Foundry deployment through onboarding and still routes the built-in provider through OpenAI-compatible Foundry endpoints, so this PR is not obsolete. The patch is focused and has no visible supply-chain-sensitive changes, but it needs maintainer review because it filters prompt choices without filtering the persisted discovered deployment catalog.

## What This Changes

This PR adds a Microsoft Foundry onboarding guardrail that detects Claude/Anthropic deployments, hides them from the selection prompt, explains the custom `anthropic-messages` workaround, errors when only unsupported deployments exist, and adds detection tests.

## Best Possible Solution

Finish the provider-local guardrail so unsupported Claude deployments are removed consistently from both onboarding choices and persisted Microsoft Foundry model catalog entries, with regression coverage for mixed and Anthropic-only resource deployment lists. Full Anthropic Foundry support can remain a separate provider feature.
