## Summary

Keep this PR open. Current main still does not implement the PR’s combined intended behavior: tool-use visible-output suppression is still phase-only, sessions_send still starts the A2A announce/send flow, and the sanitizer does not strip broad self-instruction preludes. The proposed PR also has unresolved review risks around route persistence and sanitizer heuristics, so it should be reviewed or reworked rather than cleaned up as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best fix should split the concerns where possible: land a narrow structured tool-use suppression change, decide whether sessions_send announce/A2A behavior should remain for peer-agent use cases, and if sessions_send needs session-local current-turn handling, suppress outward delivery without rewriting stored external route state. Avoid broad sanitizer heuristics unless they are gated by stronger structural signals and covered by negative tests for valid multi-paragraph replies.
