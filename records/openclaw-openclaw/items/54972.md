## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit `card_id` references, while quoted-message and thread-history reads still parse only inline card payloads and do not request or parse raw CardKit content. A recent merged parser fix improved inline interactive-card extraction, but it explicitly did not cover the broader raw-card readback gap, and the directly relevant raw-card PR remains closed unmerged.

## What This Changes

Make Feishu streaming CardKit replies readable by other agents when quoted or included in thread history instead of returning `[Interactive Card]`.

## Best Possible Solution

Land a Feishu-owned read-path fix that requests validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parses returned summary/body text such as `json_card` before the current inline-card and post fallbacks, and adds focused regression coverage for quoted-message and thread-history reads of streaming CardKit cards. Coordinate that work with #56795 and the prior #55466 approach so only one implementation path moves forward.
