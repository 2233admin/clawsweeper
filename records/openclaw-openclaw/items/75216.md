## Summary

This is a valid optional context-policy idea, but it fits the existing context-engine plugin surface rather than a new built-in core mode. Current main already exposes plugin-owned context assembly and compaction lifecycle hooks, and OpenClaw's project direction sends optional capabilities to plugins/ClawHub when the extension point exists.

## What This Changes

The issue requests a configurable rolling session-context mode that preserves a prioritized FIFO stream as an alternative to the existing compaction behavior.

## Best Possible Solution

Keep core focused on the context-engine extension point and let a rolling/prioritized FIFO policy ship as a ClawHub code plugin; open a new core API issue only if a plugin author identifies a specific missing lifecycle field or runtime hook.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The relevant check is source/docs inspection, which shows the current context-engine plugin path already covers custom context assembly and compaction ownership.

## Solution Assessment

Yes for the close direction: a context-engine plugin is the narrowest maintainable way to experiment with rolling context without adding another built-in core policy. A core change would be appropriate only for a concrete API gap discovered while implementing that plugin.
