## Summary

Keep this issue open. Current main has adjacent restart hardening, but it does not implement the requested startup pass that scans gateway session stores by transcript tail and replays unanswered chat sessions through `sessions.send`. Related #73510 and #71127/#73243 show the stuck-session recovery problem is still active rather than settled.

## Best Possible Solution

Keep this issue open and implement or explicitly supersede the startup recovery design: after gateway request context and channel sidecars are ready, scan chat session stores newest-first with a cap, detect unanswered/empty-visible-tail sessions, skip internal/non-chat channels, replay through `sessions.send`, and add focused tests for selection and safety. Coordinate that with #71127/#73243 for active stuck-run abort behavior and #73510 for the current production evidence.
