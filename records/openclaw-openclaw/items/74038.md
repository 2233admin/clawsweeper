## Summary

Keep open. Current main still enables gateway model pricing whenever `models.pricing.enabled` is not explicitly false, so `models.mode: 'replace'` does not skip OpenRouter/LiteLLM pricing bootstrap yet. The PR head is a focused implementation for the open #74020 behavior, with matching tests/docs/generated config/changelog updates and no unexplained security-sensitive changes, but it remains an open implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #74020. The best path is to land this narrow helper-level default change, or an equivalent maintainer revision, after CI is resolved: explicit `models.pricing.enabled` should win, omitted pricing should disable remote OpenRouter/LiteLLM bootstrap in replace mode, configured local costs should continue to work, and tests/docs/generated config/changelog should stay aligned.
