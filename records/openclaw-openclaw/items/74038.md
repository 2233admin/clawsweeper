## Summary

Keep open. Current main still enables Gateway pricing bootstrap whenever `models.pricing.enabled` is omitted, including replace-mode configs, and this PR is the active narrow implementation candidate for #74020. The patch looks functionally correct, but the branch currently needs maintainer/author follow-up because GitHub reports it as dirty/unmergeable and exact-head checks include failures.

## What This Changes

The PR changes the Gateway model-pricing helper so omitted `models.pricing.enabled` disables remote pricing bootstrap when `models.mode` is `"replace"`, while preserving explicit overrides, and updates related tests, generated config help, docs, and changelog.

## Best Possible Solution

Land this helper-level default change, or an equivalent maintainer revision, after rebase and clean exact-head validation: explicit `models.pricing.enabled` should continue to win, omitted pricing should disable remote OpenRouter/LiteLLM bootstrap only in replace mode, configured local costs should remain usable, and docs/generated config/changelog should stay aligned.

## Reproduction Assessment

Yes. On current main, a config with `models.mode: "replace"` and no `models.pricing.enabled` still makes `isGatewayModelPricingEnabled` return true, and both startup and direct refresh use that helper before the pricing bootstrap path.

## Solution Assessment

Yes, with mergeability and CI follow-up. The PR changes the shared helper rather than duplicating policy at call sites, preserves explicit overrides, and updates the user-visible schema/docs/tests that describe the default.
