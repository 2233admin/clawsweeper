## Summary

Keep open. Current main still enables the Gateway model-pricing bootstrap whenever `models.pricing.enabled` is omitted, including `models.mode: "replace"`, and this PR is the active narrow implementation candidate for the open #74020 report. The related #74045 PR is closed and unmerged, so it does not supersede this PR.

## What This Changes

This PR changes the Gateway model-pricing enablement helper so omitted `models.pricing.enabled` defaults off in `models.mode: "replace"`, adds gateway/config regression tests, and updates generated schema/help, docs, baseline hashes, and the changelog.

## Best Possible Solution

The best path is to land this helper-level default change, or an equivalent maintainer revision, after exact-head CI is clean: explicit `models.pricing.enabled` should win, omitted pricing should skip remote OpenRouter/LiteLLM bootstrap in replace mode, configured local costs should keep working, and docs/generated config/changelog should stay aligned.
