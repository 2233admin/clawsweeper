## Summary

Keep this PR open. The hardening goal is still valid because current main still has unbounded active schemas, but this branch is not mergeable as-is: several new maxima are below current runtime/protocol/provider behavior, and the browser work targets the old core path instead of the active browser plugin schema.

## What This Changes

The PR adds TypeBox descriptions plus maxItems and numeric minimum/maximum metadata across built-in agent tool schemas for browser, canvas, gateway, memory, message, nodes, sessions, subagents, and web_fetch.

## Best Possible Solution

Land a rebased hardening patch that encodes only contract-backed limits, moves browser schema changes to the active browser plugin, preserves current long-running workflows, and adds focused schema contract tests for the affected tools.

## Reproduction Assessment

Yes. Static review reproduces the regressions by comparing the PR's proposed schema maxima with current-main runtime, protocol schema, docs, and provider-backed behavior for node recordings, sessions.list, sessions_spawn, browser snapshots, and Discord media channels.

## Solution Assessment

No. Schema-only hardening is the right direction, but this branch is not the best implementation because several caps are too low for current behavior and one browser change targets a path that current main no longer uses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Raise node timeouts to cover recordings:** `src/agents/tools/nodes-tool.ts:113-114`
  - body: The new timeoutMs maximum is lower than valid media durations: screen_record still allows 300000ms recordings and can run for that full duration, so callers can no longer set a gateway timeout long enough for supported recordings to complete.
  - confidence: 0.93
- **[P2] Keep sessions_list compatible with 200-row lookups:** `src/agents/tools/sessions-list-tool.ts:36-37`
  - body: Capping sessions_list.limit at 50 rejects a pattern OpenClaw still uses internally: announce-target resolution asks sessions.list for 200 rows, and the gateway protocol/runtime accept larger positive limits.
  - confidence: 0.9
- **[P2] Preserve long-running subagent timeouts:** `src/agents/tools/sessions-spawn-tool.ts:44-46`
  - body: The runtime still accepts any non-negative runTimeoutSeconds, with 0 meaning no timeout. Adding maximum: 3600 makes legitimate subagent jobs intended to run past an hour fail schema validation before reaching runtime.
  - confidence: 0.91
- **[P2] Harden the active browser schema:** `src/agents/tools/browser-tool.schema.ts:98-102`
  - body: Current main registers the browser tool from the plugin-owned schema under extensions/browser, so editing the old core browser schema path leaves the active tool surface unchanged. The proposed 1000 snapshot cap is also below the current backend cap of 2000.
  - confidence: 0.84
- **[P2] Permit Discord media channel types:** `src/agents/tools/message-tool.ts:329`
  - body: The type maximum of 15 blocks Discord GuildMedia, which the current Discord runtime treats as forum-like and tests cover for thread creation. Valid media-channel thread behavior would fail schema validation.
  - confidence: 0.88
