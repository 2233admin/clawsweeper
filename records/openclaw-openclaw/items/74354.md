## Summary

Keep open. Current main still lacks the Control UI composer attributes and regression coverage needed to preserve numeric-only input, and open PR #74369 explicitly targets this issue with closing syntax, so this should stay open until that PR lands or is replaced.

## What This Changes

The issue requests that Control UI webchat preserve pure numeric input such as `123456789` instead of allowing it to be rewritten into grouped text like `123-456-789`.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, so the chat composer discourages browser/OS rewriting of numeric text and tests exact numeric payload preservation without adding server-side deformatting that could corrupt intentional hyphenated input.
