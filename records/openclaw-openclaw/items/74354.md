## What This Changes

The issue reports that Control UI webchat can rewrite pure numeric input such as `123456789` into hyphen-grouped text before sending, breaking numeric IDs and URLs.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI fix, so the composer discourages browser/OS text rewriting and tests exact numeric payload preservation while avoiding Gateway-side deformatting that could corrupt intentionally hyphenated user text.

## Reproduction Assessment

Unclear: the report gives concrete Windows 10 Control UI steps and a screenshot, and current source shows the affected draft/send path, but this read-only pass did not live-reproduce the browser/IME auto-formatting behavior.

## Solution Assessment

Yes: the narrow maintainable solution belongs in the Control UI composer plus focused regression tests. A server-side repair of hyphenated digit strings would be broader and could alter valid user input.
