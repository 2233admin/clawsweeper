## Summary

Keep open. Current main still lacks the Control UI composer attributes and regression coverage needed to preserve numeric-only chat input, and PR #74369 remains open and unmerged with closing syntax for this issue.

## What This Changes

The issue reports that Control UI web chat rewrites numeric-only input such as 123456789 into hyphen-grouped text before sending.

## Best Possible Solution

Add a narrow Control UI composer fix that discourages browser or OS rewriting of numeric text and tests exact outgoing chat.send payload preservation, without adding server-side deformatting that could corrupt intentional hyphenated input.

## Reproduction Assessment

Unclear: the report gives a concrete Windows Control UI path, and current main has the vulnerable composer shape, but this read-only Linux review did not live-reproduce the browser/IME rewrite. The source path is still clear enough to keep the issue open.

## Solution Assessment

Yes: composer-level plain-text attributes plus send-boundary regression tests are the narrowest maintainable direction. Repairing grouped numbers in the Gateway would be riskier because it could alter intentionally hyphenated IDs, phone-like strings, or prose.
