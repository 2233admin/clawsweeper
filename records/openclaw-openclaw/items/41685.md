## Summary

Keep open. Current main still leaves the reported Control UI boundary unresolved: the UI lists whatever the Gateway returns from broad combined session discovery, and URL/browser restore paths still preserve explicit stale agent-scoped session keys. The merged deleted-agent guards reduce orphaned send/resolve targeting, but they do not make the Sessions tab or restored selection default to configured agents only.

## What This Changes

The issue asks for Control UI session listing and restored session selection to ignore deleted or unregistered agent namespaces and match the CLI's configured-agent scoping.

## Best Possible Solution

Keep broad Gateway/ACP discovery available for recovery and non-UI consumers, but make Control UI default to configured-agent sessions or request an additive configured-only mode. Validate restored agent-scoped URL/storage selections against configured agents and fall back to the current main session key, with regression coverage for both the Sessions tab and restore path.
