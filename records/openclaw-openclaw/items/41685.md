## Summary

Current main still does not implement the requested Control UI behavior. The Gateway session list is built from a combined store that includes disk-only agent stores, and the Control UI requests that list without filtering to configured agents. Recent deleted-agent purge/targeting guards reduce one failure mode, but they do not hide unregistered agent sessions from the Sessions tab or reject all restored explicit unknown-agent session keys.

## Best Possible Solution

Keep this issue open and make an explicit product/API split: preserve Gateway/ACP disk-only discovery behind an explicit includeDiskOnly/includeAcp-style parameter or separate surface, while making the default Control UI Sessions tab hide agent namespaces that are not currently configured. Also normalize or reject URL/browser-restored agent:<id>:* keys when <id> is not configured, falling back to sessionDefaults.mainSessionKey.
