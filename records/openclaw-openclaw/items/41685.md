## Summary

Keep this open. Current main still lets the Control UI Sessions tab consume broad Gateway session discovery, including disk-discovered agent stores, and the chat restore path still accepts explicit session keys for unknown agent namespaces from URL or browser storage. The related directory/session purge work reduces fresh-delete repros, but it does not close the configured-agent UI boundary or the restore-loop variant described in the discussion.

## Best Possible Solution

Keep this open. Add a Control UI-specific configured-agent boundary: either filter Sessions tab rows against agents.list or add an additive sessions.list option such as configuredOnly for UI callers, while preserving broad Gateway/ACP discovery. Also validate URL and browser-storage restored agent:<id>:* keys against configured agents and fall back to sessionDefaults.mainSessionKey, with regression coverage for both the Sessions tab and restore path.

- A UI-only filter must still leave a deliberate way to inspect ACP/disk-discovered sessions if maintainers consider that a Control UI requirement.
