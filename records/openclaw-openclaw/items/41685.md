## Summary

Keep open. Current main still has the reported Control UI boundary gap: runtime send/resolve paths now reject deleted-agent sessions, but the Sessions tab still consumes broad Gateway session discovery, and URL/browser state can still preserve explicit agent-scoped session keys for agents that are no longer configured.

## What This Changes

The issue requests that Control UI stop listing or restoring sessions for agents that are no longer registered in OpenClaw config.

## Best Possible Solution

Keep broad Gateway/ACP session discovery for non-UI consumers, but make Control UI default to configured-agent sessions and validate restored `agent:<id>:*` URL/storage selections against configured agents, falling back to the current main session key when the agent is unknown.
