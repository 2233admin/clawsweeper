## Summary

Keep this open. Current main still has the Control UI request the broad Gateway `sessions.list` result, and that Gateway path still merges disk-discovered agent session stores without a configured-agent-only default or option. Current main has adjacent safeguards, but it has not implemented the issue's requested Control UI behavior or the follow-up restore guard for explicit unknown-agent session keys.

## Best Possible Solution

Keep this issue open. Implement an explicit split: preserve broad Gateway/ACP disk-only discovery behind an intentional API option or separate surface, make the default Control UI Sessions tab filter to configured agents, and normalize or reject restored `agent:<id>:*` URL/browser session keys when `<id>` is no longer configured, falling back to `sessionDefaults.mainSessionKey`. Add regression coverage for both the Sessions tab list and restore path.
