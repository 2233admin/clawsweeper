## Summary

Keep open. Current main still exposes disk-only/unregistered agent sessions through the Gateway session list that the Control UI consumes, and the UI still preserves explicit restored session keys for unknown agent namespaces. Recent deleted-agent purge/send/resolve guards reduce the original delete path, but they do not implement the requested configured-agent-only Control UI behavior.

## Best Possible Solution

Keep this issue open and implement an explicit split: preserve Gateway/ACP disk-only discovery behind an intentional API option or separate surface, while making the default Control UI Sessions tab filter to currently configured agents. Also normalize or reject restored `agent:<id>:*` URL/browser session keys when `<id>` is no longer configured, falling back to `sessionDefaults.mainSessionKey`, with regression tests for both the UI list and restore path.
