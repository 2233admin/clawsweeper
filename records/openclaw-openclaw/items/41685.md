## Summary

Keep open. Current main still routes the Control UI Sessions tab through broad Gateway session discovery and renders the returned rows without a configured-agent boundary. The related agent-delete/session-store purge work reduces the fresh-delete path, and the closed #41687 PR was not merged, but the Control UI can still surface disk-discovered/unregistered agent sessions and still preserves explicit `agent:<id>:*` URL/storage selections for unknown agents.

## Best Possible Solution

Keep this open and implement a Control UI-specific configured-agent boundary. Filter Sessions tab rows against configured/known agents, or add an additive `sessions.list` option used only by UI callers, while preserving broad Gateway/ACP discovery. Also validate restored URL and browser-storage `agent:<id>:*` keys against configured agents and fall back to `sessionDefaults.mainSessionKey`, with regression coverage for both the Sessions tab and the restore path.

## Clownfish Work Prompt

Fix the Control UI bug where sessions from unregistered/deleted agents can remain visible or be restored from stale browser/URL state. Scope the fix to the Control UI boundary unless an additive Gateway option is clearly cleaner. Do not globally narrow Gateway/ACP `sessions.list` discovery, because docs and tests intentionally allow disk-only ACP/retired stores.

Expected behavior:
- The Control UI Sessions tab should not show `agent:<id>:*` rows when `<id>` is not in the configured/known agents for the current gateway/config.
- URL or browser-storage restored session keys like `agent:local:main` should be validated after the gateway/configured-agent snapshot is known; if the agent id is unconfigured, fall back to `sessionDefaults.mainSessionKey` and update persisted UI state so the bad key does not keep rehydrating.
- Preserve explicit session keys for configured agents, plus existing `global`/`unknown` include toggles.
- Preserve broad Gateway/ACP discovery for non-UI callers and keep existing disk-only ACP tests passing.

Likely implementation paths:
- UI-only path: make Sessions tab load/consume `agents.list` or the existing `agentsList` state, filter rendered/loaded rows by configured agent ids, and sanitize restored `agent:<id>:*` keys after hello/bootstrap/agents state is available.
- Additive protocol path: add a `configuredOnly`/similar `sessions.list` option used by Control UI callers only, update schemas/generated protocol/docs if needed, and keep default Gateway behavior unchanged.

Add regression coverage for both the Sessions tab display path and URL/storage restore path. Add a user-facing changelog Fixes entry credited to @Amateur0x1 and @Jmarzab if the PR changes user-visible behavior.
