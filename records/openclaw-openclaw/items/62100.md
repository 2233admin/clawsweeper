## Summary

Keep open. Current main has related model-selection improvements but does not fully implement this PR's requested raw OpenRouter native slash-ref resolution, and the PR patch introduces provider-scoping regressions already called out in review comments.

## What This Changes

The PR changes model selection to resolve allowlisted OpenRouter native slash-style refs such as `minimax/...` and `qwen/...`, and adds regression tests for those refs.

## Best Possible Solution

Revise this PR or replace it with a narrow resolver change that maps a slash-containing raw selection to OpenRouter only when it uniquely matches a configured/allowlisted OpenRouter model id, while preserving explicit provider refs and keeping allowlist keys provider-qualified.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a config that allowlists `openrouter/minimax/minimax-m2.5:free`, then resolves raw `minimax/minimax-m2.5:free` through `resolveAllowedModelRef`; current main follows the slash-qualified provider path, while the PR's added bare allowlist key creates a separate provider-scope bypass.

## Solution Assessment

No. The PR approach changes the global parser and widens the allowlist; the safer solution belongs in the allowlist/alias resolution layer with unique-match checks and non-regression coverage for real `qwen`/`minimax` providers.
