## Summary

Keep open. Current main still does not expose the requested Gateway v2 routing/config/timing/token metadata on the inspected live response surfaces; the related diagnostics/trajectory work is useful adjacent context but does not implement this benchmark-facing envelope, and the remaining design needs maintainer decisions around schema shape and redaction.

## What This Changes

Request Gateway v2 live responses to emit routing/config lineage, policy/version, timing, decision-trace, and detailed token metadata so benchmark smoke tests can run without stub-only mode.

## Best Possible Solution

Define a typed, additive, redacted Gateway v2 metadata envelope at the protocol boundary, then wire it consistently through the approved benchmark-facing response surfaces with docs and tests for compatibility, omitted-field semantics, timing collection, and provider-specific usage breakdowns.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. The source-inspection path is high confidence because the current schemas and response builders do not include the requested fields.

## Solution Assessment

Unclear because there is no patch yet. Emitting metadata from the routing layer is directionally sound, but the maintainable solution needs maintainer agreement on schema shape, redaction, versioning, target surfaces, and reasoning-token semantics first.
