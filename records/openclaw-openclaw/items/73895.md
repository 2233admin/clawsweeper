## Summary

Keep open. Current main still lacks the requested live Gateway v2 promotion metadata surface; the matching code paths expose standard chat/response usage and separate diagnostics/trajectory export bundles, not a benchmark-facing routing/config/timing envelope.

## What This Changes

Request Gateway live response envelopes to include routing/config lineage, policy metadata, timing fields, and token usage including reasoning tokens so `model_bench.py smoke-gateway-v2` can pass without stub mode.

## Best Possible Solution

Define a redacted, additive Gateway v2 metadata envelope at the typed protocol boundary, then wire it through the maintainer-approved live response surfaces with documented field availability, timing semantics, token semantics, and backward compatibility.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the current schemas and response builders do not contain the requested metadata fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally right, but the maintainable fix needs maintainer-approved schema, redaction policy, target surfaces, and provider-specific token/timing semantics.
