## Summary

Keep open. Current main still does not emit the requested Gateway v2 promotion metadata surface; the typed Gateway chat event schema, agent RPC responses, OpenAI-compatible HTTP responses, OpenResponses resources, SDK run result, and shared usage normalization remain narrower than the requested routing/config lineage, timing, policy, decision-trace, and reasoning-token fields. The request is valid, but it needs maintainer-approved protocol shape and redaction/versioning before implementation.

## What This Changes

Request Gateway v2 responses to emit a promotion-grade metadata envelope with config lineage, routing class/policy, fallback chain, gateway version, decision trace, timing metrics, and token usage including reasoning tokens for benchmark smoke tests.

## Best Possible Solution

Define an additive, redacted Gateway v2 metadata envelope at the typed protocol boundary, then wire it consistently through the benchmark-facing live response surfaces with documented backward compatibility and provider-specific missing-field semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. We do have a high-confidence verification path from source inspection because the current schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally right, but the maintainable fix needs maintainer-approved schema, redaction policy, versioning, target surfaces, and reasoning-token semantics.
