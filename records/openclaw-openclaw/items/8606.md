## Summary

Current main still lacks a context-pruning lifecycle hook. The pruning extension calls the pruner and returns rewritten messages only, the pruner returns `AgentMessage[]` without summary metadata, and the hook catalogs/docs omit any `session:prune` or `context:prune` event. The provided GitHub context shows #12581 remains an open, unmerged closing PR for this issue, so this should stay open until that PR or a maintainer-approved replacement lands.

## Best Possible Solution

Keep this issue open and use #12581, or a maintainer-approved replacement, as the implementation path. The fix should collect safe pruning summary metadata in `src/agents/pi-hooks/context-pruning/*`, emit a `session:prune` or equivalent lifecycle event only after successful pruning, add focused pruning/hook tests, and document the event in `docs/automation/hooks.md` plus plugin docs if it becomes a plugin hook.
