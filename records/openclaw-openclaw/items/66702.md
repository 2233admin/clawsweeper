## Summary

Keep open. Current main includes partial mitigation for stale media markers and separates hidden auto-reply runtime context from the visible transcript prompt, but it still preserves recent prior image blocks and media markers by design, so it does not satisfy the requested current-turn-only media behavior for image-heavy Telegram topic sessions.

## What This Changes

The issue requests that multimodal Telegram topic turns avoid reintroducing stale image attachments from injected context or recent history into the active vision prompt.

## Best Possible Solution

Implement a current-turn media policy in the embedded runner path: when a turn has current attachments, keep only those attachments active by default, mask or ignore prior-turn image blocks and stale media refs in replay, preserve an explicit comparison path, and add regression tests for Telegram topic sessions, injected-context media markers, and recent image-heavy history.
