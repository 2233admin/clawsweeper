## Summary

Keep open. Current main still documents and implements trusted-operator host exec as permissive by default, with safety coming from opt-in exec approvals, sandboxing, tool policy, and advisory prompt guardrails. The issue discussion asks for a native hard/default dangerous-operation gate for destructive commands, so this remains a core safety/product decision rather than cleanup-closeable.

## Best Possible Solution

Keep this issue open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that the current opt-in approvals, sandboxing, and tool-policy model is the final stance. A concrete implementation path should cover default safe mode, Windows PowerShell destructive command classification such as Remove-Item, approval UX, sandbox interaction, and regression tests/docs.
