## Summary

Keep this PR open. Current main still does not implement the central changes: there is no `gateway.restart` RPC handler or restart transaction, config/update/tool restart paths still use the existing SIGUSR1/sentinel flow, CLI restart remains service-manager based, and bundled plugin resolution under Vitest still intentionally avoids using cwd as a candidate. The PR is also size XL and introduces an admin restart control-plane surface, so it needs maintainer/security review or splitting rather than cleanup closure.

This PR adds a turn-aware gateway restart transaction/RPC path, routes config/update/tool/CLI restarts through it, updates gateway protocol/docs/tests/scopes, and changes bundled plugin resolution under Vitest to prefer the current worktree.

## Best Possible Solution

Keep this PR open for maintainer review, preferably split into a narrow bundled-dir Vitest/current-worktree fix and a separate gateway restart transaction/RPC handoff PR. The restart PR should address the existing review findings, update protocol/docs/tests together, and get explicit lifecycle/security review before merge.
