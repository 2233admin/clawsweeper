## Summary

Keep this PR open. Current main still does not implement custom cron job IDs on `cron add`: the CLI has no `--id` add option, the Gateway `cron.add` schema still rejects an `id` field, `CronJobCreate` still omits `id`, and `createJob` still assigns `crypto.randomUUID()` unconditionally. The PR remains the active implementation lane for the open feature request #65636, but it should address the existing case-variant run-log collision concern before merge.

## Best Possible Solution

Keep this PR open as the implementation lane for #65636. Before merge, revise it to canonicalize custom cron job ids or reject case-insensitive duplicates before persistence/run-log use, keep the CLI/gateway/service/normalization tests, and add user-facing docs/changelog if maintainers accept the feature.
