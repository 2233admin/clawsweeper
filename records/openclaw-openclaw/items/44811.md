## Summary

Keep this PR open. Current main still projects runtime-derived config writes onto the stored runtime source snapshot before the inner writer reads the current disk file, so the stale runtime snapshot writeback bug is not implemented on main. The PR is a focused implementation candidate for the linked `session.dmScope` persistence bug, but it should address the existing invalid-readable snapshot and double-read review concerns before merge.

## Best Possible Solution

Keep this PR open and request focused updates/review. The best fix is to make runtime-derived writeback choose the latest existing on-disk source/resolved snapshot as the projection base before computing the runtime patch, fall back only for truly unreadable or missing files as intended, use the existing `baseSnapshot` seam if needed to avoid redundant reads, and land the `session.dmScope` stale-snapshot regression test with the normal changed gate.
