## What This Changes

This PR refactors bundled plugin runtime-dependency planning and repair across plugin loading, Gateway startup/hot reload, CLI plugin commands, post-config setup, docs, changelog, scripts, and tests to use package-level plans, materialization checks, and package-manager convergence.

## Best Possible Solution

Keep the package-level runtime-deps plan/materialization direction, but make the sync staging helper force package-manager convergence whenever OpenClaw has already decided a repair is required. Add a focused regression test for interrupted package-level staging with package sentinels, then rerun targeted runtime-deps/Gateway/CLI tests and a clean changed gate before maintainer landing.

Yes. ## Solution Assessment

No, not as-is. The package-level repair direction is maintainable, but the sync lazy-load staging fallback must pass `force: true` or delegate through the same forced repair helper before the PR fully solves interrupted bundled runtime-deps recovery.
