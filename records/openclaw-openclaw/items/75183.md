## Summary

Keep open. This is an active implementation PR with the protected `maintainer` label, current `main` does not already contain the intended startup-policy/runtime-inspection changes, and the latest PR head still has a blocking channel CLI regression where no-repair mode is applied too broadly.

## What This Changes

The PR changes CLI/channel/plugin inspection and Gateway startup loading so selected paths avoid implicit bundled runtime-dependency repair, adds runtime inspection gating and documentation updates, and adjusts bundled runtime-deps staging tests and helpers.

## Best Possible Solution

Keep the PR open and revise the branch so `installBundledRuntimeDeps: false` is limited to exact static/read-only CLI surfaces, while operational channel commands that load configured channel runtimes either keep repair enabled or use a dedicated pre-stage path before runtime import.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a packaged install with a configured bundled channel plugin whose staged runtime deps are absent, then running a runtime-loading channel command such as `openclaw channels login --channel <id>` or `openclaw channels logout --channel <id>`; the inherited parent policy disables repair before runtime loading.

## Solution Assessment

No. The Gateway pre-stage and explicit repair direction is maintainable, but applying no-repair mode at the parent `channels` catalog entry is broader than the stated read-only boundary; exact per-command policy is the safer fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:
