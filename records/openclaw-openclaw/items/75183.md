## Summary

Keep open. This PR has the protected maintainer label and the current head still has a blocking channel startup-policy regression: the parent `channels` policy disables bundled runtime-dependency repair for commands that still preload runtime plugins, so fresh or pruned packaged installs can fail before operational channel commands run.

## What This Changes

The PR changes CLI/channel/plugin inspection and Gateway startup loading so selected paths avoid implicit bundled runtime-dependency repair, adds explicit repair/runtime-inspect controls, memoizes prepared bundled runtime roots, and updates docs, tests, and changelog.

## Best Possible Solution

Keep the PR open and narrow the no-repair policy to exact static/read-only channel surfaces, or explicitly mark operational channel commands as no-runtime only where that is actually true. Gateway startup can still pre-repair once and then load with synchronous repair disabled.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a packaged install with a configured bundled channel plugin whose staged runtime deps are missing, then running a channel command such as `openclaw channels login --channel <id>`; preaction inherits the parent channel no-repair policy and runtime loading throws before the command can proceed.

## Solution Assessment

No, not yet. The explicit repair and Gateway pre-stage direction is maintainable, but applying `installBundledRuntimeDeps: false` at the parent `channels` policy is broader than the stated read-only boundary; an exact allowlist is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Scope no-repair policy to read-only channel commands:** `src/cli/command-catalog.ts:67`
  - body: The parent `channels` policy still has `loadPlugins: "always"`, but this line now disables bundled runtime-dependency repair for every channel subcommand that inherits it. Real commands such as `channels login`, `channels logout`, and `channels capabilities` still preload channel plugin runtimes, so a fresh or pruned packaged install can now fail with missing bundled deps before the operational command runs. Move this no-repair flag to exact read-only/no-runtime paths, or keep repair enabled for commands that load runtime plugins.
