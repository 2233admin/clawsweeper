## Summary

Keep open. The PR is useful but not mergeable as currently scoped: the pinned acpx runtime still only supports `deny`/`fail`, so accepting and documenting `allow` in OpenClaw would not make non-interactive prompts auto-approve. The branch also introduces a trusted-only auto-approval setting without adding it to the dangerous-config contract.

## What This Changes

The PR adds `allow` to the acpx plugin `nonInteractivePermissions` config schema and manifest enum, documents it, and adds config parser regression tests.

## Best Possible Solution

Implement `allow` in the acpx runtime contract first, covering ACP permission requests plus fs/terminal non-TTY handlers, then bump the pinned package and update OpenClaw schema, manifest, docs, tests, and dangerous-flag metadata together. Until then, keep `permissionMode=approve-all` as the documented full-permission workaround.

## Reproduction Assessment

Yes for the PR defect: inspecting the pinned `acpx@0.6.1` types and compiled runtime shows `allow` is not a supported non-interactive policy and would not auto-approve in non-TTY sessions. Current main also still rejects `allow` at OpenClaw config validation.

## Solution Assessment

No. The branch only widens OpenClaw validation and docs, but the behavior lives in the pinned acpx runtime and the security contract also needs updating.
