## Summary

Keep open. Main now has compaction checkpoint/successor-transcript support, which helps the reset-recovery discussion, but it does not add guard config, transcript-tail scoring, post-compaction validation, or a structured recommend-reset diagnostic.

## Best Possible Solution

Keep this PR open for maintainer review or supersession by a consolidated loop-aware compaction guard implementation. The concrete path is to resolve or replace the prerequisite guard config/detector/scorer/runtime/validator PRs, decide how recommend-reset should relate to existing compaction checkpoints, then land a focused internal diagnostic path with tests and no automatic reset by default.
