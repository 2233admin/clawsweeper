## Summary

Keep open. Current main partially covers the request with the typed `before_install` plugin hook for gateway-managed skill dependency installers, but native ClawHub skill install/update and the Skills UI ClawHub path still route through `installSkillFromClawHub` and copy extracted skill folders without a blocking pre-copy policy hook. There is still no post-install observational hook with audit/outcome metadata.

## Best Possible Solution

Keep this issue open as the concrete tracker for skill install lifecycle policy. The best path is to extend the existing `before_install` contract or add a documented skill lifecycle hook so native ClawHub install/update, gateway `skills.install`, and the Skills UI run a blocking policy point before copying files, then add a post-install observational hook with `installedAt`, `targetDir`, source, outcome, and audit metadata. Clarify separately whether the standalone `clawhub` CLI needs ClawHub-owned support outside this repo.
