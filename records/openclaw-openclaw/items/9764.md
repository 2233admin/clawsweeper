## Summary

Keep this issue open. Current main still has Google Chat authentication wired to service-account credentials and the `chat.bot` scope only; the requested user OAuth config, refresh-token resolution, scoped user-token routing, and graceful fallback for reactions, attachment upload, reaction typing, and email-alias proactive DMs are still missing. The linked external Google Chat work is useful prior art, but it is not shipped in this repository, and #73681 tracks the opposite inbound-reaction gap rather than this outbound/user-auth work.

## Best Possible Solution

Keep this issue open. The bundled Google Chat plugin should add a strict, secret-aware user OAuth credential model, resolve refresh-token credentials through existing config/secret patterns, construct scoped user OAuth clients for reactions, attachment uploads, reaction typing, and proactive DM/email-alias lookup, keep service-account auth for webhook verification and bot-level sends, and document fallback behavior when user OAuth is absent.
