## Summary

Keep open. Current main still resolves a non-subagent forced wake session before `heartbeat.session`, the exec/wake paths still pass the originating session through that forced-session branch, and Telegram private DMs without isolated `dmScope` are still documented and tested as `agent:main:main`. The related open PR #56092 uses GitHub closing syntax for this issue, so this should remain open until that implementation candidate is reviewed and either merged or closed.

## What This Changes

The issue requests stricter isolation so Telegram direct messages and heartbeat/system-event-triggered work cannot pollute each other's sessions or reuse `agent:main:main` unexpectedly.

## Best Possible Solution

Make configured heartbeat sessions take precedence over originating wake sessions, preserve or migrate pending system events without cross-session pollution, and separately settle Telegram DM default behavior through either an explicit docs-backed product decision or a Telegram routing change with matching tests.
