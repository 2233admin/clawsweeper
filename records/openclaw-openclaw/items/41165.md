## Summary

Keep open. Current main still resolves wake-triggered heartbeats through an originating forced session before checking an explicit `heartbeat.session`, and Telegram private DMs without a thread still default to the agent main session. The related open PR #56092 already uses closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed or closed explicitly.

## What This Changes

The issue requests preventing Telegram DMs and heartbeat/system-event traffic from sharing or polluting `agent:main:main`, especially by isolating Telegram DMs and honoring configured heartbeat sessions.

## Best Possible Solution

End state should make configured heartbeat sessions win for system-event-triggered wakes, preserve or migrate pending system events without cross-session pollution, and separately settle Telegram DM default behavior by either documenting intentional main-session sharing or changing the Telegram plugin to isolate DMs with matching tests and docs.
