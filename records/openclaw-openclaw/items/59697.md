## Summary

Keep open. Current main does not yet implement this PR’s intended fix: memory-lancedb still injects leading `<relevant-memories>` context, while the current Gateway and Control UI user-message sanitizers only strip inbound metadata/envelopes and do not remove that XML memory prefix from user-role messages. The PR is also active, recently repaired by a maintainer on the contributor branch, and marked merge-ready, so the best path is review and land rather than close as obsolete.

## Best Possible Solution

Review and land this PR or an equivalent narrow fix that exports/adds an injected-prefix sanitizer and wires it through `src/gateway/chat-sanitize.ts`, `ui/src/ui/chat/message-extract.ts`, and `ui/src/ui/chat/message-normalizer.ts`, with regression tests for string and array content plus preservation of user-authored tags and leading whitespace. After the canonical fix merges, close or supersede duplicate PR #51288 against the landed implementation.
