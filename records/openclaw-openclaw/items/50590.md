## Summary

Current main already fixes the reported sandbox/global skill path mismatch. Read-only sandbox workspaces are prepared by syncing the visible skill set from the agent workspace, including managed skills, into the sandbox workspace before the run; the prompt is then backed by sandbox-local `skills/<name>/SKILL.md` paths. The same behavior was previously reviewed for related #46257 and is present in the `v2026.4.22` release tree.

## Best Possible Solution

Close as implemented. Keep the shipped sandbox-local skill sync behavior; if someone can still reproduce on `v2026.4.22` or newer, the best follow-up is a fresh issue with exact sandbox backend, workspace access, agent/session mode, skill source, prompt path, and failing read/tool call.
