## Summary

Keep this PR open. Current main still forwards Feishu reaction emoji values unchanged into the Feishu add and remove/list reaction API paths, and the intended Unicode-to-Feishu emoji_type normalization is not present on main. The PR is still an active implementation candidate for open bug #66406, with a maintainer-side repair noted in the discussion.

## Best Possible Solution

Keep this PR open for maintainer review and land this repaired contributor branch, or an equivalent Feishu-owned fix, after preserving the lazy runtime boundary and validating the emoji_type vocabulary. The shipped fix should normalize Unicode inputs before addReactionFeishu and listReactionsFeishu, keep Feishu type-string passthrough, and include regression tests for add/remove/list filtering paths.
