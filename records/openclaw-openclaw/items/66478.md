## Summary

Keep this PR open. Current main still lacks Feishu reaction emoji normalization: the Feishu react path trims the user emoji and passes it through to add/list helpers, and those helpers send it directly to Feishu. The linked regression issue remains open, this PR is open and unmerged, and maintainer discussion says this branch was repaired as the canonical contributor path.

## Best Possible Solution

Keep this PR open for normal maintainer review and land this repaired contributor branch, or an equivalent Feishu-owned fix, after validating the Feishu emoji vocabulary. The fix should normalize Unicode emoji and known Feishu type strings before add and remove/list reaction API calls, preserve the lazy runtime boundary, and keep regression coverage for add, remove/list filtering, passthrough, trimming, case normalization, and unknown input behavior.
