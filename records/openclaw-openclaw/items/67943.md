## Summary

Keep open. Current main still has the stale-lineage path described by the report: normal agent runs can reuse stored `spawnedBy`, session merging preserves omitted lineage fields, and `sessions_spawn` computes depth from stored `spawnDepth`/`spawnedBy`. The same-author implementation PR #67946 remains open and unmerged, so the issue should stay open for that review path rather than being closed early.

## What This Changes

The issue requests clearing persisted subagent lineage on ordinary top-level sessions so normal chats do not incorrectly hit `sessions_spawn` depth limits from stale metadata.

## Best Possible Solution

Land #67946 or an equivalent maintainer-approved fix that treats stale persisted lineage as invalid for ordinary top-level sessions, preserves lineage for real spawned/ACP sessions, and adds focused regression coverage for a later top-level run that previously had stored subagent metadata.
