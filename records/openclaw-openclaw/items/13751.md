## Summary

This should stay open because current main still performs contact-backed Feishu sender-name lookup by default and still uses the contact directory for live peer listing. The issue is valid, but it is already assigned and overlaps an open maintainer-labeled Feishu PR, so maintainer coordination is the right next step rather than closing or queuing a competing repair.

## What This Changes

The issue asks for Feishu inbound sender-name handling and live directory listing to stop requiring broad contact-directory read access by default.

## Best Possible Solution

Make ordinary inbound Feishu messages contact-free by default: use verified event or chat-member metadata when available, fall back quietly to the stable sender id, and put contact-backed profile lookup plus live user-directory listing behind explicit operator opt-in with updated docs and regression tests.
