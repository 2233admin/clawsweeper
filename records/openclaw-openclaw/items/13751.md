## Summary

Current main still keeps Feishu sender-name resolution contact-backed and default-on. The stale `contact:contact.base:readonly` notice is suppressed in one error path, but the runtime still calls Feishu contact APIs for sender names and live peer listing, so the central request remains valid. The issue is also currently assigned and overlaps open maintainer PR #73399, so it should stay open for maintainer follow-up.

## Best Possible Solution

Keep this issue open and resolve it with a focused Feishu plugin change: make basic inbound bot operation contact-free by default by using webhook sender fields when available and otherwise falling back to open_id/user_id without calling `contact.user.get`; gate profile lookup and live contact-directory peer listing behind explicit opt-in config. Coordinate with open PR #73399 so display-name fallback work and permission-scope reduction land coherently, then update Feishu config metadata, docs, changelog, and focused regression tests.
