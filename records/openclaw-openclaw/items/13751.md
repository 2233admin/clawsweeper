## Summary

Current main still keeps Feishu sender-name resolution contact-backed and default-on. The exact stale `contact:contact.base:readonly` notice is now suppressed/corrected in one error path, but the underlying runtime still calls Feishu contact APIs for sender names and live peer listing, so the central request is not implemented.

## Best Possible Solution

Keep this issue open and make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise fall back to open_id/user_id without calling `contact.user.get`; keep profile lookup behind an explicit opt-in. For directory peers, prefer configured static peers unless live contact-directory lookup is explicitly enabled. Update Feishu config metadata, docs, changelog, and focused regression tests, coordinated with #73399.
