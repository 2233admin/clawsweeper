## Summary

Keep open. Current main partially implements the narrowed request with `messages.queue.debounceMsByChannel`, but per-channel queue `cap` and `drop` options are still missing from the public config schema and runtime resolver.

## Best Possible Solution

Keep the issue open but narrow it to the remaining core config/runtime work: add backward-compatible per-channel queue cap/drop overrides, preserve existing `messages.queue.byChannel` string mode behavior, update schema/help/docs, and add focused tests for fallback order and overflow behavior per channel.
