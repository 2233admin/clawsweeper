## Summary

Keep open. Current main has generic Gateway/node primitives and an OpenAI-compatible HTTP endpoint that can support an external Raspberry Pi voice bridge, and the discussion includes afalk42's working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is product and security boundary work, not conservative cleanup.

## What This Changes

Request to support Brabble running as a Clawdis/OpenClaw node for distributed Raspberry Pi voice wake, gateway routing, and TTS response playback.

## Best Possible Solution

Maintainers should choose the official boundary first: document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if current Gateway surfaces are insufficient. That path should include explicit security guidance for Gateway credentials and always-on microphone use.
