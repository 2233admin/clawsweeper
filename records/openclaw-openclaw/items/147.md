## Summary

Keep open. Current main has enough gateway/node primitives for an external Raspberry Pi or Brabble-style voice bridge, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining decision is product and architecture scope: external companion/ClawHub, documented minimal node contract, or native node integration.

## What This Changes

Request support for running Brabble or a Raspberry Pi-style distributed voice wake node that connects to the OpenClaw Gateway, sends wake-triggered audio or transcripts to an agent, and returns responses through TTS.

## Best Possible Solution

Keep open for maintainer product review. The likely next step is to document the minimal external voice-node contract and link or promote the afalk42-style implementation through ClawHub or a companion repository, adding core/plugin APIs only if maintainers identify a missing generic node or voice extension seam.
