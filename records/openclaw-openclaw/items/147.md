## Summary

Keep open. Current main has generic Gateway/node primitives that make an external Raspberry Pi voice bridge feasible, and the discussion already includes a working external implementation, but OpenClaw still does not ship or document a first-class Brabble/Raspberry Pi distributed voice-wake node. The remaining gap is a product and security-boundary decision, not an already-completed core cleanup.

## What This Changes

The issue requests a Brabble/Raspberry Pi-style distributed voice wake node that connects to the OpenClaw Gateway and routes wake-word voice commands through an agent with TTS responses.

## Best Possible Solution

Maintainers should choose the product boundary first: either document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or define a small generic node/plugin API only if the current Gateway surfaces are insufficient. That path should include clear security guidance for operator credentials and always-on microphone use.
