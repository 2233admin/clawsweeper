## Summary

Keep open. Current main still ships the bundled 1Password skill with the reported tmux-only `op` guidance, while the same bundled reference describes macOS desktop app integration and signing in with `op vault list`. The latest release tag also still contains the same wording, so this is not implemented, obsolete, or superseded by the related launchd/SecretRef issues.

## Best Possible Solution

Update `skills/1password/SKILL.md` so the workflow is auth-method aware: direct exec for macOS desktop app integration, tmux only where it actually helps an interactive/session-token flow, and explicit service-account/API-token guidance. Keep secret-redaction guardrails, and review adjacent 1Password guidance for consistency without weakening the narrow release-maintainer npm-auth fallback note.
