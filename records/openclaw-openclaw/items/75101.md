## Summary

Keep open. The PR addresses a real gap tracked by the linked security-hardening report, and current main does not already implement this config surface, but the latest PR head still has blocking bypasses in the advertised deny-path behavior for shell-wrapper commands and bare sensitive filenames.

## What This Changes

The PR adds an opt-in `tools.exec.denyPathPatterns` hard-deny config with schema, docs, changelog, helper tests, and enforcement in gateway/node exec policy paths.

## Best Possible Solution

Land a maintainer-reviewed version where one shared deny-path policy evaluates parsed command segments and inner shell payloads before every execution path, treats cwd-resolvable sensitive filenames as candidates, keeps schema/docs aligned, and includes gateway, sandbox/elevated, and node integration coverage.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run gateway/sandbox/elevated `bash -c "cat ~/.openclaw/secrets/foo.env"`; the latest pre-dispatch gate tokenizes the raw wrapper text rather than the inner payload, so the tilde pattern misses. A second static reproduction is `tools.exec.denyPathPatterns: ["**/.env"]` with `cat .env`, which is skipped before cwd resolution.

## Solution Assessment

No. The deny-path gate is the right product direction for the linked incident, but this implementation needs to reuse parsed shell-wrapper payloads on local/elevated paths and broaden candidate selection for bare cwd-relative sensitive filenames before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1621-1622`
  - body: The gateway/sandbox/elevated gate passes the entire raw command as `shellPayload`, so `bash -c "cat ~/.openclaw/secrets/foo.env"` tokenizes the quoted payload as one token starting with `cat `. That token no longer matches the documented `~/.openclaw/secrets/**` deny pattern, leaving the incident path reachable for local/elevated execution.
  - confidence: 0.9
- **[P2] Treat bare deny targets as relative path candidates:** `src/infra/exec-deny-path.ts:86-95`
  - body: `isPathLikeArg` returns false for bare relative filenames such as `.env` or `credentials.json` because they contain no slash. Those arguments never get resolved against `cwd`, so the documented `**/.env` and `**/credentials.json` patterns do not block `cat .env` or `cat credentials.json`.
  - confidence: 0.88
