## Summary

Current main still has the UX bug this PR targets. User-initiated chat focus mode still forces the nav drawer closed, while the visible responsive hamburger only toggles drawer state and never clears `settings.chatFocusMode`. The PR remains useful, but its patch is stale against current main because it modifies the sidebar `.nav-collapse-toggle` path rather than the current `.topbar-nav-toggle` surface.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch. The likely fix belongs in `ui/src/ui/app-render.ts`: update the `.topbar-nav-toggle` click handler so that when `state.settings.chatFocusMode && !state.onboarding` is active, clicking the hamburger clears `chatFocusMode` and opens the nav/drawer; otherwise it should keep the existing drawer toggle behavior. Add a browser regression test that activates `.shell--chat-focus`, clicks `.topbar-nav-toggle`, and verifies focus mode exits and the nav/drawer becomes visible.
