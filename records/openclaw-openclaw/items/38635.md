## Summary

Current main still has the focus-mode hamburger UX bug. This PR has useful intent, but its patch targets the older/sidebar `.nav-collapse-toggle` path while current main’s visible responsive hamburger is `.topbar-nav-toggle`, so it needs updating or replacement before merge.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but update or replace it with a current-main patch that changes the `.topbar-nav-toggle` click handler: when user-initiated `settings.chatFocusMode` is active and onboarding is not controlling focus, clear `chatFocusMode` and open the nav/drawer. Add a focused browser regression test in the current navigation test surface that enters focus mode, clicks `.topbar-nav-toggle`, and verifies `.shell--chat-focus` is removed and the nav becomes visible.
