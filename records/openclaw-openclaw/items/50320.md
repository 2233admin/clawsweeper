## Summary

Keep this PR open. Current main has solved the export-template placeholder problem and replaced the old google-vertex ADC sentinel shape, but it still does not implement the full intended PR: the current-run chat attachment parser can still pass small HEIC/HEIF images through as inline `image/heic`/`image/heif`, and `openclaw config set gateway.nodes.denyCommands ...` still only writes the config with generic update logging rather than warning about ineffective entries. The PR also has unresolved security-sensitive review findings around provider auth behavior, HEIC post-conversion size checks, and denyCommands validation source-of-truth.

## Best Possible Solution

Keep this PR open, but ask for an update before merge: drop or rework the already-solved export/auth pieces against current main, implement HEIC/HEIF normalization inside `parseMessageWithAttachments` before `parsed.images` is returned with post-conversion size checks, and add `gateway.nodes.denyCommands` config-set warnings using a real full node command catalog. The remaining review comments should be addressed before maintainers consider landing it.

## Risks / Open Questions

- Security review: the PR touches provider auth resolution and node command policy UX, both security-sensitive behavior. The provided changed-file list does not include workflows, dependencies, lockfiles, release metadata, package publishing, generated/vendor blobs, or install scripts, so the review concern is behavioral rather than supply-chain.
- The PR’s HEIC conversion path needs post-conversion size validation before returning prompt images, matching the existing `normalizeInputImage` guard.
- The PR’s denyCommands warning approach should validate against a full node command catalog, not the effective allowlist, or it can mislead users trying to block dangerous but valid commands.
- The PR’s google-vertex ADC change should be rebased onto current main’s `gcp-vertex-credentials` marker/runtime-auth model instead of returning oauth-without-apiKey on paths that still require a key.
