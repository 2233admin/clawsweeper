## Summary

Keep open. Current main implements the export-template part differently, but it does not implement the full intended PR: small current-run HEIC/HEIF chat attachments can still flow to the model as image/heic/image/heif, and `openclaw config set gateway.nodes.denyCommands ...` still has no write-time warning. The auth/config parts also need maintainer review because the PR as written has review-identified runtime and operator-warning risks.

## Best Possible Solution

Keep this PR open as a candidate, but it should be updated before merge: either split the already-solved export/auth pieces from the still-open HEIC and denyCommands work, or rebase into focused fixes for #50081 and #50011. The HEIC fix should normalize/revalidate prompt images before `parsed.images` reaches the active session prompt, and the denyCommands fix should validate against a real full command catalog, not only the effective allowlist. The google-vertex ADC fix needs a compatible runtime-auth path rather than returning oauth with no apiKey into code that still rejects that shape.
