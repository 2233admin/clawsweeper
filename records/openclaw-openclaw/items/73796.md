## Summary

Current main still has the reported mismatch: the browser CLI emits screenshot/snapshot MEDIA lines through shortenHomePath, while the shared media parser intentionally rejects tilde-prefixed home-dir paths. An open unmerged implementation candidate, #73798, already carries the narrow producer-side fix and closing syntax for this issue, so the issue should stay open until that PR lands or is closed.

## Best Possible Solution

Keep this issue open while maintainers review, land, or close #73798. The best narrow fix is the producer-side change in the browser CLI: emit absolute paths only on machine-readable `MEDIA:` lines while keeping shortened paths for human-readable logs, and leave parser tilde rejection intact unless maintainers choose a separate security-reviewed parser policy change.
