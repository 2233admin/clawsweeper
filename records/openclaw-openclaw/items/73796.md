## Summary

Keep open. Current main still emits tilde-shortened browser CLI `MEDIA:` paths while the shared media parser still rejects `~`-prefixed paths, so the reported Telegram delivery failure remains plausible. The linked implementation candidate #73798 is closed and unmerged, leaving a narrow repair available.

## What This Changes

The issue asks for browser screenshot media saved under the user's home directory to be delivered to chat instead of being dropped when an agent emits a `MEDIA:~/...` directive.

## Best Possible Solution

Keep the parser's home-dir rejection in place and fix the browser CLI producer so only machine-readable `MEDIA:` lines use absolute paths; retain shortened paths for human display and cover screenshot plus AI snapshot image output with regression tests.

## Reproduction Assessment

Yes. Current source plus existing tests give a high-confidence static reproduction: browser CLI emits `MEDIA:~...` for home paths, and `splitMediaFromOutput` rejects `MEDIA:~/Pictures/My File.png` before outbound delivery.

## Solution Assessment

Yes. The narrow producer-side fix is the best maintainable path because it preserves the media parser's traversal/home-dir guard while making browser-generated `MEDIA:` lines acceptable to the existing validator.
