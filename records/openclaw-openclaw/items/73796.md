## Summary

Keep open. Current main still has the reported producer/validator mismatch: the browser inspect CLI emits machine-readable media lines through `shortenHomePath`, while the shared media parser rejects home-dir-prefixed media before final reply delivery. The related fix PR was closed unmerged, so the requested behavior is not implemented.

## What This Changes

The issue asks OpenClaw to deliver browser screenshot media to chat when the browser plugin emits home-dir-prefixed `MEDIA:` paths that the outbound media parser rejects.

## Best Possible Solution

Use a producer-side browser plugin fix: emit the absolute `path` and `imagePath` values returned by the browser route for machine-readable `MEDIA:` lines, keep shortened paths only for human-readable logs, and add focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: browser inspect output can emit shortened home-dir media paths, and `splitMediaFromOutput` rejects those paths while final reply parsing depends on that parser.

## Solution Assessment

Yes. The narrowest maintainable fix is producer-side in the browser CLI; changing the shared parser to accept home-dir-prefixed paths would touch a security-sensitive boundary unnecessarily.
