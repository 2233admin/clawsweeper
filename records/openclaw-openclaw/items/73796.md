## Summary

Current main still rejects inline `MEDIA:~/...` paths while the browser CLI still emits machine-readable browser screenshot/snapshot `MEDIA:` lines via `shortenHomePath`. The issue also has an open unmerged implementation candidate, #73798, with closing syntax for this issue, so it should stay open until that PR lands or is closed.

## Best Possible Solution

Keep this issue open while maintainers review #73798. The best narrow fix is to make browser machine-readable `MEDIA:` lines emit absolute paths while leaving human-readable display paths shortened, with regression tests for screenshot and AI snapshot output. If maintainers prefer parser normalization instead, that should be handled as a separate security-reviewed media parser policy change.
