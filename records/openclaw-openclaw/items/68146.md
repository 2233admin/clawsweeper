## Summary

Keep open. Current main still does not implement this PR's server-side barrier for startup-gated Gateway methods; `handleGatewayRequest` still fail-fasts requests whose method is in `unavailableGatewayMethods`. Newer mainline startup-handshake retry work overlaps the user-facing race and may supersede part of the need, but it is not the same change and needs maintainer judgment before this PR is closed or rebased.

## What This Changes

This PR adds a Gateway `StartupGateBarrier` so startup-gated `chat.history` and `models.list` requests wait for post-attach sidecars before dispatching, with a retryable `UNAVAILABLE` timeout fallback and focused gateway tests.

## Best Possible Solution

Decide the desired Gateway boundary explicitly: if startup-handshake retry fully resolves the cached-client restart flow, close or supersede this PR with that evidence; otherwise rebase this PR and keep a gateway-owned bounded barrier or equivalent queue with tests for dispatch-after-sidecars, timeout fallback, non-gated bypass, and legacy no-barrier contexts.
