## Summary

Keep this PR open. Current main still does not wire Telegram streamed block replies to `resolveHumanDelayConfig`, while the shared dispatcher, resolver, docs, and peer channels already support the behavior. The PR remains a focused implementation candidate for #68945, and the provided maintainer discussion identifies it as the preferred small fix after unrelated churn was dropped.

## Best Possible Solution

Review and land this PR, or an equivalent scoped patch, after normal maintainer validation. The implementation should re-export `resolveHumanDelayConfig` through `extensions/telegram/src/bot-message-dispatch.agent.runtime.ts`, import it in `extensions/telegram/src/bot-message-dispatch.ts`, pass `resolveHumanDelayConfig(cfg, route.agentId)` as `dispatcherOptions.humanDelay`, keep the focused Telegram regression test and changelog credit, and preserve the scope boundary that this is not a polling-stall fix.
