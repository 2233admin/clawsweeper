## Summary

Keep open. Current main still lacks a CLI-wide root-execution guard or equivalent DigitalOcean 1-Click mitigation, while the home-directory and systemd repair paths remain consistent with the reported corruption path. Open PR #67509 is the direct implementation candidate and uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent maintainer-approved fix lands.

## What This Changes

The issue requests a guard or clear warning to prevent DigitalOcean 1-Click users from running the OpenClaw CLI directly as root and corrupting service state.

## Best Possible Solution

Land PR #67509 or an equivalent early CLI guard before config and state resolution, with an explicit root override, help/version behavior covered by tests, and optional DigitalOcean recovery or wrapper guidance if maintainers want image-specific mitigation.
