## Summary

Keep open. Current main still accepts Telegram `edited_message` update types structurally, but it does not register a runtime `bot.on("edited_message", ...)` handler or contain the focused debounce/test surface this PR proposes. The PR remains a relevant implementation candidate for #72873, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review and land this PR or an equivalent Telegram-owned `edited_message` handler with per-message debounce and focused tests, then let #72873 close after the fix reaches main. Keep the related Discord and Signal edit behavior in their existing channel-specific issues.
