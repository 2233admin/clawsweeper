## Summary

Keep open. Current main still contains the reported unquoted bracket-notation `openclaw config` examples in the English source docs, and a shell-only reproduction still fails before OpenClaw receives the argument. The CLI/config model supports the path once passed through, so this remains an actionable docs shell-quoting regression. Related PR #40642 proposed the right direction but was closed unmerged.

## Best Possible Solution

Keep this issue open and land a small docs fix: update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md` to quote bracket-notation paths or use shell-safe dot-index paths like `agents.list.0.tools.exec.node`, add a short note that paths containing `[]` must be quoted in shells that treat brackets as glob patterns, and let the docs i18n pipeline update zh-CN output. PR #40642 is useful prior direction but needs a fresh merged change on current main.
