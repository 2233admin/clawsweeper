## Summary

Current main still documents the reported unquoted bracket-notation config path, and failglob-style shells reject that argument before OpenClaw can parse it. The CLI path parser supports bracket notation once it reaches argv, so this remains an actionable docs bug rather than a core parser bug or an implemented fix.

## Best Possible Solution

Keep this issue open and land a narrow docs-only patch in the English source docs. Quote bracket-notation path arguments such as `agents.list[0].tools.exec.node`, or switch examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`; add a short Paths note that bracket notation must be shell-quoted in failglob-style shells; update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; let the docs i18n pipeline regenerate localized output.
