## Summary

Keep this issue open. Current main still documents the reported unquoted bracket-notation config path in the English source docs, including the exact `agents.list[0].tools.exec.node` command. The CLI still supports bracket notation after the argument reaches OpenClaw, so the remaining bug is the shell-quoting hazard in docs, not a fixed config parser issue. Related PR #40642 was closed unmerged, so it remains useful prior direction but not an implementation on main.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md` to quote bracket-notation config paths or switch examples to shell-safe dot-index paths like `agents.list.0.tools.exec.node`, and add a short note that bracket paths containing `[]` must be quoted in shells that treat brackets as glob patterns. Use #40642 as useful prior direction, but land a fresh English docs-source change because that PR is closed unmerged.
