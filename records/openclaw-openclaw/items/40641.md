## Summary

Keep open. Current main still contains the reported unquoted bracket-notation config examples in the English source docs, and shell reproduction still shows the path can fail before OpenClaw receives it. The CLI/config model supports the path once passed through, so this remains a docs shell-quoting regression rather than an implemented fix.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md` to quote bracket-notation paths or use shell-safe dot-index paths like `agents.list.0.tools.exec.node`, then add a short note that paths containing `[]` must be quoted in shells that treat brackets as glob patterns. Use #40642 as prior direction, but land a fresh English source-docs change and let the docs i18n pipeline handle zh-CN output.
