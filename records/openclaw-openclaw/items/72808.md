## Summary

Keep open. Current main still contains the deterministic Slack Socket Mode reconnect exhaustion path, and an open PR already references this issue with closing syntax while maintainer discussion is still weighing the local reconnect cap against the upstream Slack SDK pong-stall fix.

## What This Changes

The issue reports Slack Socket Mode silently stopping after days of uptime while OpenClaw remains reachable through the web interface.

## Best Possible Solution

Keep this issue open until #73162 or a replacement lands, then validate OpenClaw with a Slack SDK version that includes slackapi/node-slack-sdk#2565. If reconnect-window message loss remains after those fixes, split that into a narrower follow-up.

## Reproduction Assessment

Yes. The current reconnect loop has a deterministic path where repeated Socket Mode disconnect or start-failure events increment reconnectAttempts until the hardcoded maxAttempts: 12 guard throws, and the discussion includes current-main reproduction plus live logs for the same failure family.

## Solution Assessment

Unclear. Removing the cap is the narrowest local fix for permanent reconnect exhaustion, but the complete resolution should also validate the upstream Slack SDK pong-stall fix and the reported transient delivery failures.
