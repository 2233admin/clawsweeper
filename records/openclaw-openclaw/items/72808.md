## Summary

Keep open. Current main still has a finite Slack Socket Mode reconnect cap, no Slack config override for reconnect attempts, live discussion evidence of the same failure family, and an open unmerged PR with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to recover or surface Slack Socket Mode connections that stop responding after a long-running connection failure while other chat surfaces still work.

## Best Possible Solution

Review #73162 as the current implementation path, land or adjust it with Slack-owned regression coverage for repeated disconnect/start-failure behavior and status reporting, then update `@slack/socket-mode` when the upstream reconnect fix is released and validated.

## Reproduction Assessment

Yes. The discussion provides current-main reproduction and live logs, and the code has a deterministic path where repeated Socket Mode disconnect or start-failure events eventually hit the hardcoded `maxAttempts: 12` guard.

## Solution Assessment

Unclear whether the open uncap alone is the complete fix. It is a narrow fix for permanent reconnect exhaustion, but the complete solution should also account for the upstream Slack SDK pong stall and transient reconnect-window delivery failures described in later comments.
