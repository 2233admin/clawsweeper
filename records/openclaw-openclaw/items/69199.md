## Summary

Keep #69199 open. Current main still sends missing `node:sqlite` failures through the generic memory-search unavailable helper, which only distinguishes quota errors from generic embedding/provider errors. The PR is a narrow, still-relevant fix for the closed UX issue #59457, while #59451 separately tracks the larger fallback-search feature.

## Best Possible Solution

Keep the PR open for maintainer review or merge after normal validation. The best code path is the narrow owner-module fix in `extensions/memory-core`: detect the `node:sqlite` unavailable error in `buildMemorySearchUnavailableResult`, preserve the existing quota and generic branches, and keep the added unit test. Treat #59451 as the separate follow-up for preserving search capability via a file-based fallback.
