## Summary

Keep this PR open. Current main still writes light/REM inline dreaming metadata to `memory/YYYY-MM-DD.md` before request-scoped narrative fallback and has no fallback-aware suppression path. The PR is an active implementation candidate for a still-real memory-core bug, though it should be reviewed against related PR #70332, the security-bot comments, and the unrelated MiniMax FAQ change before merge.

## Best Possible Solution

Keep the PR open as a focused memory-core bug-fix candidate. The best path is to review the current head, remove unrelated docs churn, ensure the light/REM request-scoped fallback tests cover inline suppression, newline/formatting preservation, and fallback-write failure semantics, then either land it or intentionally consolidate the same fix into #70332.
