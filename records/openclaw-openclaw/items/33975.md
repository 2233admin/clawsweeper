## Summary

Current main partially satisfies the request: model attribution can be shown with `messages.responsePrefix` templates such as `{modelFull}`, but there is still no config/schema field or runtime approval hook that pauses before trying a fallback model.

## Best Possible Solution

Keep this open as the fallback-approval tracker. Treat message-level model attribution as satisfied by `messages.responsePrefix` templates such as `[{modelFull}]`, then have a maintainer decide the approval UX and timeout semantics before implementation in the agent model schema/type, fallback runner, delivery/approval surface, docs, and focused tests.
