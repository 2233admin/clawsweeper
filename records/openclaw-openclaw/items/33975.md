## Summary

Keep this issue open. Current main covers the model-attribution half through `messages.responsePrefix` template variables such as `{modelFull}`, but the fallback-approval half is still missing: fallback candidates are tried automatically, and the model config schema/type has no approval policy fields.

## Best Possible Solution

Keep this issue open as the fallback-approval tracker while treating model attribution as satisfied by `messages.responsePrefix` templates. A good implementation would add an opt-in per-agent fallback approval policy to `AgentModelConfig` and `AgentModelSchema`, pause `runWithModelFallback` before non-primary `runFallbackAttempt`, route approval through an existing approval/notification surface, and add docs plus tests for approve, deny, timeout proceed/fail, cancellation, and channel delivery.
