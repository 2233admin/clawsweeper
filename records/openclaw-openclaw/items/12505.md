## Summary

Keep open. Current main has useful opt-in tool sandbox backends, Docker/SSH/OpenShell support, and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered presets, native plugin/process isolation, IPC-proxied plugin execution, per-plugin env/fs/net/resource manifest policy, or default credential scoping. The maintainer discussion also already treats this as the canonical sandbox/isolation architecture tracker rather than resolved work.

## What This Changes

The issue requests a secure-by-default, multi-platform sandbox architecture with tiered presets, process isolation, per-plugin capability manifests, credential scoping, network/filesystem/resource controls, and migration guidance.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into reviewed milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving current Docker/SSH/OpenShell backend groundwork.
