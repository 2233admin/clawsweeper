## Summary

Keep open. Current main has opt-in tool sandbox backends and a plugin SDK backend seam, but the requested secure-by-default tiered, multi-platform plugin/process isolation architecture remains absent; recent maintainer and ClawSweeper comments also treat this as the canonical security architecture tracker.

## What This Changes

The issue requests a secure-by-default unified sandbox architecture with multi-platform process isolation, tiered presets, plugin manifest policies, scoped credentials, IPC/proxy enforcement, and resource controls.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones. The desired end state is a staged design and implementation for secure-by-default presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance while preserving the current Docker/SSH/OpenShell backend groundwork.
