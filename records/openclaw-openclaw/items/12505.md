## Summary

Keep open. Current main has useful opt-in tool sandbox backends and a plugin SDK seam for sandbox backends, but it still does not implement the central request: secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin SDK execution, or per-plugin manifest policies for env/fs/net/resources. The existing maintainer comment also explicitly keeps this open as a sandbox/isolation architecture tracker.

## What This Changes

Requested a secure-by-default unified sandbox architecture with tiered presets, multi-platform backends, plugin/process isolation, IPC SDK proxying, and manifest-declared env/fs/net/resource policies.

## Best Possible Solution

Keep this issue open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower milestones. The best path is a design and staged implementation for secure-by-default tiered presets, plugin or skill execution isolation, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance, while treating current Docker/SSH/OpenShell tool sandbox backends as partial groundwork.
