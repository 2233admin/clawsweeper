## Summary

Current main still does not implement Slack bot-message unfurl controls. The Slack config schema/types and generated metadata lack `unfurlLinks`/`unfurlMedia`, outbound `chat.postMessage` payloads do not pass `unfurl_links`/`unfurl_media`, Slack docs/tests only cover adjacent delivery behavior, and prior matching PRs were closed unmerged rather than landing the feature.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Slack plugin. Add schema-validated and typed `unfurlLinks` and `unfurlMedia` fields at `channels.slack` and `channels.slack.accounts.<id>`, resolve them through the existing account merge path, pass them to Slack as `unfurl_links` and `unfurl_media` on normal `chat.postMessage` text/block/follow-up sends, and update Slack docs, generated config metadata/schema, changelog, and focused tests. Treat #34512/#34526/#34593 as useful prior implementation context, not as superseding trackers.
