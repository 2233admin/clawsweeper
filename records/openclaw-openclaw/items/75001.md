## Summary

Keep open. Current main still intentionally excludes Feishu broadcast-only `@all`/`@_all` from the bot-mention gate and forwarded mention targets, so the report describes a real current behavior dispute rather than an obsolete bug. Because that behavior was introduced to fix the opposite false-positive case, changing it needs Feishu channel owner/product judgment instead of conservative cleanup.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu broadcast `@all`/`@_all` mentions at the channel mention gate and instead expose them to agent prompt or config policy.

## Best Possible Solution

Represent Feishu broadcast mentions as a distinct channel fact that can be included in agent context without automatically satisfying the bot-mention gate, then let maintainers choose prompt/config semantics and update Feishu tests and docs together.

## Reproduction Assessment

Yes. A focused fixture with a group message containing only `@_all` text or mention metadata `{ key: "@_all", id.open_id: "all" }` produces `mentionedBot=false`; with `requireMention=true`, the group handler returns before dispatch.

## Solution Assessment

Unclear. The requested revert is not obviously the best fix because it can restore the broadcast false-positive that PR #72658 addressed; a safer direction is to preserve broadcast context separately from the boolean bot-mention gate.
