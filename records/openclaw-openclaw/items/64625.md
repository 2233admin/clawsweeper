## Summary

Current main still shows the Slack custom-invoke path dropping the already computed agent-scoped media access before the Slack upload runtime sees it. The bundled Slack plugin uses that custom invoke path, the closed related PR was not merged, and current tests cover upload-file mapping but not the createSlackActions custom-invoke media bridge.

## What This Changes

The issue requests preserving agent-scoped mediaLocalRoots/mediaReadFile through Slack upload-file custom invocation so files inside an agent workspace can be uploaded.

## Best Possible Solution

Fix this in the bundled Slack plugin by constructing one augmented SlackActionContext from toolContext plus ctx.mediaLocalRoots/mediaReadFile and passing it through both the custom invoke and fallback runtime paths, with focused regression coverage for upload-file through createSlackActions(options.invoke).
