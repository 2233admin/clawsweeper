## Summary

Keep this PR open. Current main still supports streamable-http MCP servers but the bundled session runtime forwards tool calls to the cached MCP client without lost-session detection, reconnect, metadata refresh, or retry. The PR is a focused implementation candidate, and the provided review context shows the initial reconnect hardening concerns were addressed on the branch rather than making the work obsolete.

## What This Changes

This PR changes the bundled MCP runtime, its tests/harness, and the changelog to reinitialize lost streamable-http MCP sessions in place and retry the original tool call once.

## Best Possible Solution

Land a validated runtime-level reconnect path for stateful streamable-http MCP sessions that preserves the custom schema validator, serializes concurrent reconnects, refreshes tool metadata, retries once, keeps failed reconnect attempts recoverable, and leaves SSE/stdio semantics unchanged.
