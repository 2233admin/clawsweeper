## Summary

Keep open. Current main still omits the state-dir `gateway.env` from the runtime dotenv chain, and this PR is a narrow, source-aligned fix with focused test coverage and no blocking review findings.

## What This Changes

The branch adds `<stateDir>/gateway.env` to `loadGlobalRuntimeDotEnvFiles` after `<stateDir>/.env` and adds a unit test for loading `GATEWAY_AUTH_TOKEN` while preserving `.env` precedence.

## Best Possible Solution

Review and land this PR, optionally adding a docs/changelog note, so the runtime loader reads `$OPENCLAW_STATE_DIR/gateway.env` after `$OPENCLAW_STATE_DIR/.env` and before the legacy XDG fallback.

## Reproduction Assessment

Yes. A source-level reproduction is clear: put `GATEWAY_AUTH_TOKEN` only in `$HOME/.openclaw/gateway.env`, leave it unset in `process.env`, and current main's loader will not read that file; the PR adds a focused unit test for that path.

## Solution Assessment

Yes. The proposed loader insertion is the narrowest maintainable fix because it reuses the existing trusted global dotenv parsing, keeps `.env` precedence, and preserves the old XDG compatibility fallback ordering.
