## What This Changes

The branch changes `parseHeartbeatTasks` to use raw source lines for task-block indentation checks and adds two parser regression tests for unknown indented task fields and top-level task-block termination.

## Best Possible Solution

Heartbeat task parsing should consistently use raw-line indentation for block-boundary decisions, keep top-level non-task termination behavior, include regression coverage for unknown indented fields, and satisfy changelog policy before landing.

## Reproduction Assessment

Yes. A `HEARTBEAT.md` `tasks:` block with an unknown indented field such as `notes:` before a later `- name:` task exercises the two dead `trimmed.startsWith(...)` guards on current main and drops the later task by source inspection.

## Solution Assessment

Yes for the code fix. Replacing only the two block-boundary checks with raw `line` / `nextLine` indentation checks matches the parser’s adjacent `interval:` and `prompt:` contract without a broader YAML/parser refactor.
