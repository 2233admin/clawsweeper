## What This Changes

The PR changes `parseHeartbeatTasks` to test task-block indentation against raw source lines and adds regression tests for unknown indented task fields and top-level task-block termination.

## Best Possible Solution

Land this PR or an equivalent narrow parser fix so heartbeat task-block boundary detection consistently uses raw-line indentation, with regression coverage for unknown indented fields and preserved top-level termination behavior.

## Reproduction Assessment

Yes. A `HEARTBEAT.md` `tasks:` block containing an unknown indented field such as `notes:` before a later task exercises the current-main `trimmed`/`nextTrimmed` guards and drops the later task; the PR adds that fixture as focused regression coverage.

## Solution Assessment

Yes. The proposed change is the narrowest maintainable fix because it changes only the two boundary checks to use the same raw-line indentation contract already used by adjacent recognized-field parsing.
