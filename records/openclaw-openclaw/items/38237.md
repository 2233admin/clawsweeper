## Summary

Keep this PR open. Current main still has the Slack draft-preview identity gap: normal Slack delivery resolves and forwards the active agent identity, but `createSlackDraftStream` has no identity parameter and its initial `sendMessageSlack` call omits identity. The linked #38235 remains open, and this PR is still a useful implementation candidate, but it needs a current `extensions/slack` port and the unrelated changes should be split out or dropped.

## Best Possible Solution

Keep this PR and #38235 open. Port only the Slack identity fix to current `extensions/slack`: add an optional `SlackSendIdentity` parameter to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test. Split or drop the unrelated daemon cleanup and already-separate duplicate-race changes.
