## Summary

Keep this PR open. Current main still has the Slack draft-preview identity gap: `sendMessageSlack` and normal Slack delivery support custom agent identity, and dispatch resolves `slackIdentity`, but `createSlackDraftStream` still has no identity parameter and its initial draft `sendMessageSlack` call does not forward identity. The paired issue #38235 is still open and this PR references it with closing syntax, so this should remain open for a clean current-path port rather than being closed as stale or implemented.

## Best Possible Solution

Keep this PR and paired #38235 open. Port only the Slack identity fix to current `extensions/slack`: add an optional `SlackSendIdentity` parameter to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test. Split or drop the unrelated daemon cleanup and already-separate duplicate-race changes.
