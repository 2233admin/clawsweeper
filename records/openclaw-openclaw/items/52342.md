## Summary

Keep this PR open. Current main does not implement the PR's `chat_type: "p2p"` plus `oc_` OpenChat classification, and linked bug #52238 remains open, but the branch is not merge-ready against current main: Feishu routing has since gained `topic_group` handling, existing tests still preserve `p2p`/`oc_` conversations as direct-message/card-action cases, and the PR branch is dirty against the target branch.

## What This Changes

The PR adds a Feishu chat-type helper that treats normal groups and `p2p` events with `oc_` chat IDs as group chats, wires it into inbound message and mention-forward routing, and updates Feishu unit-test fixtures.

## Best Possible Solution

Rework Feishu chat classification in the Feishu plugin around verified Feishu conversation metadata, preserving real DM semantics while routing native topic/group chats through the group path. The updated change should cover inbound bot events, mention-forward detection, relevant card-action or message lookup fallbacks, and coordinate with #58537 if outbound session routing still needs a separate follow-up.
