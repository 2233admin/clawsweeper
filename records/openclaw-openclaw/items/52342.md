## Summary

Keep this PR open. Current main still classifies Feishu inbound group/direct state from `chat_type` only, so the reported `chat_type: "p2p"` plus `oc_...` topic/open-chat shape is not implemented on main. The PR is still an implementation candidate for #52238, but it needs maintainer review or rework because the prefix-only `oc_` rule can affect Feishu access-control and session-routing paths that currently treat some `oc_`/ambiguous targets as direct unless stronger context is available.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase/adapt it to current main's `isFeishuGroupChatType` helper and either prove prefix-only `oc_` promotion is safe for inbound bot-receive events or use stronger evidence such as `thread_id`, `root_id`, and/or verified Feishu chat metadata. The fix should update both bot routing and mention-forward classification, preserve true p2p DM/card-action behavior, and leave outbound session mismatch tracked by #58537 unless addressed in the same reviewed change.
