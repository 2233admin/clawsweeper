## Summary

Keep open. Current main has not implemented this PR’s intended inbound Feishu OpenChat fix: group classification still depends only on `chat_type`, so a `chat_type: "p2p"` event with an `oc_` chat id remains on the direct-message path. Main does include newer `topic_group` handling, but that does not cover the PR’s reported `p2p` + `oc_` case. Security review of the provided PR diff found no CI, dependency, script, package, permission, secret, or generated/vendor changes; the remaining concern is functional correctness and rebasing against current Feishu code/tests.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. The best path is to rebase it onto current main, fold the group-detection logic into the current Feishu chat-type helper or an equivalent shared helper that accepts both `chat_type` and `chat_id`, update `bot.ts` and `mention.ts`, add regression coverage for the reported `p2p` + `oc_` OpenChat inbound event, and separately track the outbound mismatch in #58537.
