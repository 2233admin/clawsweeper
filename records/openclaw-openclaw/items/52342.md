## Summary

Keep open. The Feishu topic-group routing bug appears real, and current main still treats `p2p` plus `oc_` inbound events as direct traffic, but this PR is not merge-ready because its prefix-only classifier can misroute existing `p2p` `oc_` direct/card-action flows and would regress current `topic_group` group semantics.

## What This Changes

The PR adds a Feishu chat classifier, uses it in inbound bot routing and mention-forward detection, adds classifier tests, and rewrites several Feishu DM fixtures from `oc_` IDs to `p2p_` IDs.

## Best Possible Solution

Revise the Feishu-owned classifier so it preserves current `group` and `topic_group` semantics, distinguishes `p2p` topic-group webhook events using a verified signal such as thread/root fields or Feishu chat metadata, keeps regression coverage for true `p2p` `oc_` direct/card-action flows, applies the final decision consistently in bot routing and mention-forward detection, and coordinates session-key consistency with #58537.
