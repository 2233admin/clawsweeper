## Summary

Current main still does not implement the requested global/per-agent hard iteration budget. `agents.defaults` and `agents.list[]` remain strict schemas without `maxTurns`, `maxToolCalls`, per-turn/per-session tool-call caps, or timeout-after-first-tool-call fields; existing loop detection is disabled-by-default and pattern-based; and OpenResponses `max_tool_calls` is accepted but documented as ignored. This should remain open as the canonical global/per-agent agent iteration-limit request, separate from the open per-channel budget request #13271.

## Best Possible Solution

Keep this issue open as the canonical global/per-agent hard iteration budget request. The best implementation would add typed strict config under `agents.defaults` and per-agent scope, enforce model-tool round-trip and total tool-call counters before executing the next tool, define timeout-after-first-tool-call semantics, document precedence with loop detection and existing timeouts, and add focused runtime/config/docs tests. Keep #13271 separate for per-channel budgets and auto-delegation.
