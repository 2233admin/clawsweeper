## Summary

Current main still does not accept `mediaLocalRoots` under `channels.telegram` or `agents.defaults`, and the current root builder does not read such config. Telegram outbound code can forward a `mediaLocalRoots` value when one is supplied by core, so the report points at a real config-surface gap. The wording is partly stale because current Telegram config types also no longer advertise `mediaLocalRoots`, and open PR #52951 overlaps with a broader `tools.fs.roots` design, so this should stay open for maintainer review rather than close as implemented.

## Best Possible Solution

Keep this open and route it through the shared media/FS policy decision. The best fix is a documented, schema-backed operator allowlist that feeds `getAgentScopedMediaLocalRoots`/outbound media access consistently across channels, with config types, generated schema/help, docs, and regression tests aligned. If #52951 lands and covers Telegram outbound media from configured roots, this issue can then close as implemented by that merged PR.
