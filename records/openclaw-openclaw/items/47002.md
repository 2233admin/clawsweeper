## Summary

Keep open. Current main still lacks a validated persistent outbound-media root allowlist for Telegram or shared filesystem policy: Telegram can consume forwarded `mediaLocalRoots`, but the config schemas expose only Telegram's separate `trustedLocalFileRoots` and the broad `tools.fs.workspaceOnly` flag. Related PR #52951 appears to target the shared filesystem-root direction, but it is still open and changes host filesystem access policy.

## What This Changes

The issue requests a validated configuration surface for outbound local media roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Land a shared, documented, schema-backed operator allowlist, likely `tools.fs.roots` or an equivalent filesystem policy surface, that feeds local media access across channels with config types, generated schema/help, docs, and regression tests aligned.

## Reproduction Assessment

Yes. The config-side failure is reproducible from current source: strict Zod objects for Telegram config and agent defaults omit `mediaLocalRoots`, while Telegram outbound code only consumes roots supplied by core.

## Solution Assessment

No for a Telegram-only validator exception as the final design. The safer maintainable solution is a shared filesystem/media root policy reviewed across host FS tools, media loading, outbound delivery, and channel adapters.
