## Summary

Keep this PR open. Current main still does not serialize the full gateway `config.set`, `config.patch`, and `config.apply` read/base-hash-check/compute/write window, and the PR head is still distinct from `main`. The PR remains a relevant implementation candidate for the lost-update bug, but it needs maintainer review/rebase over current SecretRef and shared-auth behavior plus the existing review gaps addressed before landing.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. Rebase or reimplement it on current main so `config.set`, `config.patch`, and `config.apply` serialize the full read/base-hash-check/validate/write sequence, or otherwise pass the caller base hash into a fresh lower-level mutation that recomputes from current state before writing. Preserve current SecretRef and shared-auth behavior, add stale concurrent-write coverage for all three methods, and resolve the queue rejection review comments before landing.
