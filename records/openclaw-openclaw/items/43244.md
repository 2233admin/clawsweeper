## Summary

Keep this PR open. Current main still does not serialize the full gateway `config.set`, `config.patch`, and `config.apply` read/base-hash-check/compute/write window, and the lower-level write helper does not receive the caller's `baseHash` on this path. The PR remains a relevant implementation candidate, but it needs a current-main rebase plus the existing review gaps addressed before it should land.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. Rebase or reimplement it on current main so `config.set`, `config.patch`, and `config.apply` serialize the full read/base-hash-check/validate/write sequence, or pass the caller `baseHash` into a fresh lower-level mutation that recomputes from current state before writing. Preserve current SecretRef and shared-auth behavior, handle command-lane rejection errors as explicit RPC failures, and add stale concurrent-write tests for all three methods.
