## Summary

Keep this issue open. Current main has managed-restart hardening, restart-sentinel notification for one routed session, and internal main-session recovery, but it still does not provide a Feishu-visible notice for all in-flight Feishu sessions interrupted by a direct supervisor restart such as `systemctl --user restart`.

## Best Possible Solution

Keep this issue open until OpenClaw either implements generic channel-visible interruption handling for direct gateway restarts or documents direct supervisor restarts as outside the supported no-loss boundary. The implementation likely belongs around gateway shutdown/startup recovery: durably identify interrupted in-flight sessions and their delivery contexts, then enqueue idempotent same-channel notices or resume-and-deliver through the original Feishu route.
