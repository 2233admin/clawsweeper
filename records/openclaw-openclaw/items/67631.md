## Summary

Keep open. Current main still lacks automatic macOS app LaunchAgent ownership detection in the CLI doctor/gateway repair path, and open PR #73974 already references this issue with GitHub closing syntax, so the issue should remain open until that PR is merged, replaced, or closed.

## What This Changes

The issue requests preventing the macOS app and npm CLI doctor/gateway repair flows from installing or managing competing LaunchAgent-owned gateway processes on the same machine.

## Best Possible Solution

Make macOS gateway ownership explicit: terminal doctor/gateway repair should detect an installed or loaded app LaunchAgent and skip or defer standalone gateway install/start/restart with clear output, while app and CLI docs describe one canonical owner. The fix should include focused regression coverage for doctor repair, launchd scanning or ownership detection, app attach-existing behavior, and the macOS gateway docs.
