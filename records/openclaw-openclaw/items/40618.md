## Summary

Keep open. The proposed capability is still useful and current main has not implemented it, but the PR branch has a blocking filesystem-boundary issue and needs docs cleanup before it is mergeable.

## What This Changes

The PR adds an optional workspace-relative `hooks.internal.entries.session-memory.outputDir` for the bundled session-memory hook, updates handler tests, and documents the option in English and zh-CN hook docs.

## Best Possible Solution

Add the optional `outputDir` while preserving the default `memory` output, validate empty/absolute/traversal and symlink-alias cases without external side effects, rely on the existing root-scoped safe writer or a root-scoped mkdir helper for parent creation, cover custom/fallback/symlink behavior in tests, and update only English source docs plus bundled hook metadata. Keep #54524 as the broader docs/design follow-up for duplicate indexing guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard output directory before creating it:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: Calling `fs.mkdir(outputRoot, { recursive: true })` before the root-scoped writer can follow an in-workspace symlink such as `link/archive` and create directories outside the workspace even though the later write is rejected. Please remove this raw mkdir or replace it with a boundary-aware helper so the workspace-relative guarantee has no external side effects.
  - confidence: 0.91
- **[P3] Remove localized docs edits:** `docs/zh-CN/automation/hooks.md:39`
  - body: `docs/AGENTS.md` says localized docs under `docs/<locale>/**` are not maintained in this repo. Please keep the English docs as source of truth and drop the zh-CN edits so the i18n pipeline can regenerate them separately.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff has a concrete filesystem-boundary concern in the new output-directory creation path.

Concerns:

- **[medium] Raw mkdir can escape through workspace symlinks:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: The new pre-write `fs.mkdir(outputRoot, { recursive: true })` runs outside the alias-safe write path. If `outputDir` starts with an in-workspace symlink that points elsewhere, this can create external directories before `writeFileWithinRoot` blocks the final file write.
  - confidence: 0.91
