## Summary

Keep #58358 open. Current main still routes Kimi Coding through the Anthropic Messages path, but I did not find a shipped malformed-SSE/message_start-before-message_stop recovery on that path. The directly related PR #58412 is still open and unmerged, so the issue should remain as the bug tracker until a fix lands or a canonical maintainer-owned follow-up supersedes it.

## Best Possible Solution

Keep #58358 open until the malformed Anthropic-format Kimi Coding stream is fixed on main, ideally by landing or revising PR #58412 with a regression test that simulates a second message_start before the previous message_stop/message_end and verifies preserved text plus ordered assistant end/start events.
