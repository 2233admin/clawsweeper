## Summary

Keep this PR open, but do not merge it as-is. The linked #71581 problem is real, yet the latest PR head still rewrites stable exec notification and heartbeat session-resolution contracts in ways that can misroute heartbeats, lose exec completion semantics, and weaken session-isolation checks.

## What This Changes

The PR adds a shared event-routing key normalizer and rewrites exec system-event notification plus heartbeat session selection to route background execution events toward agent sessions based on session/config state.

## Best Possible Solution

Keep the existing exec notification and heartbeat session-resolution contracts intact, then add a narrow dmScope-aware routing seam that reuses the existing single-owner allowlist precondition after canonicalization and ownership checks. Add focused regression coverage for exec-event and cron-event wakes under session.dmScope=main without changing unrelated ProcessSession formatting, delivery context, or subagent safeguards.

## Reproduction Assessment

Yes. The PR diff statically shows the regressed paths, and current tests already exercise notify-on-exit wake scoping, suppression, delivery-context behavior, and heartbeat subagent fallback; a focused #71581 regression should add dmScope=main single-owner exec-event and cron-event routing.

## Solution Assessment

No. The proposed patch is broader than needed and replaces stable routing/notification behavior; the safer solution is to layer dmScope-aware normalization onto the existing canonicalization and ownership checks.

  - body: The new completion text omits the exec id and output tail, so it no longer matches the structured exec completion parser that drives exec-event prompts. Successful and failed background completions can be treated as ordinary heartbeat/cron content instead of async exec results.
  - confidence: 0.92

  - body: Notify-on-exit wakes now use `exec-exit`, but heartbeat reason handling only treats `exec-event` as the event-driven exec path. These wakes may not bypass the file gates or build the exec completion prompt that current behavior relies on.
  - confidence: 0.89

  - confidence: 0.9
