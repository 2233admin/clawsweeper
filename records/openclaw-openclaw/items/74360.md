## Summary

Keep this PR open. Current main still applies the permissive web-tools SSRF policy through the shared trusted web-search helper, and this PR is the active fix for linked #74357, but the latest patch still has a blocking generated Plugin SDK API baseline gap and one environment-dependent SSRF test concern.

## What This Changes

The PR splits web-tools SSRF policies into strict trusted endpoints and explicit self-hosted helpers, switches SearXNG to the self-hosted helper, exports those helpers through the Plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Land a maintainer-approved SSRF fix that makes fixed public web-search providers strict by default, keeps an explicit validated private-network path for SearXNG-style self-hosted endpoints, updates generated SDK baseline artifacts for any public helper exports, and lets #74357 close only after merge.
