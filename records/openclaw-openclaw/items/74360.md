## Summary

Keep open. Current main still has the reported broad private-network SSRF policy, and this PR is the active implementation paired with #74357, but the latest branch still adds public Plugin SDK exports without the generated API baseline update. It needs maintainer/security review and a PR-branch fix before merge, not cleanup closure.

## What This Changes

The PR splits web-tool SSRF handling into strict trusted endpoints and explicit self-hosted helpers, switches SearXNG to the self-hosted helper, exports those helpers through the Plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Update the PR branch so the intentional SDK surface expansion is fully reflected in generated API baseline artifacts, keep the public web-search providers strict by default, preserve a clearly named and validated self-hosted path for SearXNG-style endpoints, and let #74357 close only after the reviewed fix merges.
