## Summary

Keep open. Current main still applies the private-network SSRF policy to the shared trusted web-tools path, so the linked security issue is not implemented on main; this PR is the active fix candidate, but the latest patch still has a blocking Plugin SDK API baseline gap before it can merge.

## What This Changes

The PR splits trusted and self-hosted web-tools SSRF helpers, switches SearXNG to the self-hosted helper, exports the new helpers from the Plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Land a maintainer-approved SSRF fix that makes fixed public web-search providers strict by default, preserves an explicit validated private-network path for SearXNG-style self-hosted endpoints, updates generated SDK API artifacts and any needed docs for the new public helpers, and lets #74357 close only after merge.
