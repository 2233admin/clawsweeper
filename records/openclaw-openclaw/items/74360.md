## Summary

Keep open. Current main still has the broad private-network trusted web policy, and this PR is the active implementation linked to #74357, but the current patch is not merge-ready because it expands public Plugin SDK exports without updating the API baseline. The Dockerfile supply-chain review note appears unrelated to this diff and is separately tracked by #74359.

## What This Changes

The PR makes trusted web-tool endpoints use an empty SSRF policy, adds explicit self-hosted web fetch/search helpers exported through the Plugin SDK, switches SearXNG to the self-hosted helper, and adds SSRF policy tests.

## Best Possible Solution

Land a maintainer-approved SSRF fix that keeps fixed public web-search providers strict by default, preserves a validated private-network path for SearXNG-style self-hosted endpoints, updates generated SDK baseline and any public helper docs, and lets #74357 close only after merge.
