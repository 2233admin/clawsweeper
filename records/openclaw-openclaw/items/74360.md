## Summary

Keep this PR open. Current main still has the broad trusted web-tools SSRF policy, so the proposed SSRF fix is not obsolete; because it changes a security boundary and adds public Plugin SDK helpers, it needs maintainer/security review and normal validation rather than cleanup closure.

## What This Changes

This PR splits web tool SSRF handling into strict trusted endpoints and self-hosted opt-in helpers, switches SearXNG to the self-hosted helper, exports those helpers through the Plugin SDK, and adds SSRF policy tests.

## Best Possible Solution

Land a maintainer-approved SSRF fix that makes fixed public web-search providers strict by default, preserves an explicit validated private-network path for SearXNG-style self-hosted endpoints, and treats any public SDK helper as a documented, baselined contract before allowing #74357 to close on merge.
