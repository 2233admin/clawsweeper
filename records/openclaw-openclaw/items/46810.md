## Summary

Keep this issue open. Current main has the Control UI exec approval modal, countdown, queue indicator, context rows, scrollable command block, and allow-once/allow-always/deny buttons, but the requested command highlighting remains missing. The UI also drops the existing commandPreview field, so it cannot show the requested-versus-effective command distinction when the gateway provides it.

## Best Possible Solution

Keep this issue open and finish the remaining Control UI work in a focused UI PR: parse commandPreview when present, render sanitized command text through a small Lit-safe shell-token highlighter, optionally show the preview/effective distinction without protocol changes, preserve the existing countdown/actions/context rows and scroll behavior, and add focused tests for highlighted output plus escaping/redaction safety.
