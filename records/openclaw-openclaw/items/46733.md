## Summary

Keep open. Current main still removes the Anthropic 1M beta header for OAuth/setup-token credentials, and the matching tests/docs intentionally encode that fallback. The related open context-window PR only covers resolver/default behavior, not the OAuth request-header contract reported here.

## What This Changes

The issue requests Claude Opus 4.6 1M context support for Anthropic OAuth/Claude Max users when `context1m` is configured, including accurate model-list context display instead of a 195k/200k fallback.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, model-list/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate explicit fallback.
