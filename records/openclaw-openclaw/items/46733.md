## Summary

Keep open. Current main and the latest stable tag still encode the older Anthropic long-context contract: OAuth-like Anthropic credentials strip the 1M beta header, Opus/Sonnet 4.6 only resolve to 1M behind `params.context1m`, and docs still describe 1M as beta-gated. Related open work covers only part of the resolver/display behavior, so this is not safely closable as implemented or duplicate.

## What This Changes

The issue asks OpenClaw to support and report Anthropic Opus 4.6 1M context for Claude Max/OAuth users instead of falling back to a 200k-class window or stripping the long-context path.

## Best Possible Solution

Update the Anthropic plugin-owned long-context contract so eligible Opus/Sonnet 4.6 requests use the current GA 1M behavior, retain correct handling for older beta-only models or ineligible credentials, and align runtime headers, context budgeting, models/status display, docs, and tests in one coordinated change.

## Reproduction Assessment

Yes for a source-level reproduction: current main has code and tests that strip the 1M beta for OAuth credentials and keep no-flag Opus 4.6 at 200k. Live provider reproduction was not run because no Anthropic or Claude credential variable names were present in the environment/profile check.

## Solution Assessment

Unclear. The current hardcoded fallback no longer matches Anthropic's published 4.6 GA behavior, but the best fix should be a provider-contract update coordinated with #66862 and #41444 rather than a narrow display-only patch.
