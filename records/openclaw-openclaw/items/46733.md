## Summary

Keep open. Current main still removes the Anthropic 1M beta header for OAuth/setup-token credentials, tests and docs still encode that fallback, and the related open context-display work does not fully cover this request-header/auth contract problem.

## What This Changes

The issue asks OpenClaw to support and display Claude Opus 4.6 1M context for Anthropic OAuth/Claude Max users when `params.context1m` is configured.

## Best Possible Solution

Confirm the current Anthropic OAuth/setup-token long-context contract, then update the Anthropic provider wrapper, context budgeting, models/status display, docs, and tests so eligible OAuth users get a real 1M request path and ineligible credentials get an accurate fallback.
