## Summary

Keep open. Current main still preserves Discord thread paths that can bypass per-message mention gating: bot-owned autoThread threads disable the mention requirement, and bound Discord thread sessions lower the configured mention requirement before the no-mention drop path. The related #52568 is open and narrower, while #41072 was closed unmerged and explicitly scoped away from Discord mention-gating behavior.

## What This Changes

The issue asks for Discord thread follow-up messages to respect mention requirements instead of staying active after an initial bot mention.

## Best Possible Solution

Define an explicit Discord thread-reply mention policy, preferably aligned with the Slack `thread.requireExplicitMention` pattern, then update Discord preflight and allow-list behavior plus config types, strict schema, generated/help metadata, docs, and focused regression tests so autoThread and bound-thread replies can require explicit mentions without breaking intentional thread-bound sessions.
