## Summary

Keep open. Current main and v2026.4.27 cover the Control UI Korean locale work, but the requested AI-agent language persistence remains unresolved: the system prompt has no detected-primary-language instruction, and web search only honors explicit language/search_lang parameters instead of deriving Korean search language automatically. The paired PR is closed unmerged, so there is no implementation path pending merge.

## What This Changes

Request Korean (`ko`) locale support in the Control UI plus persistent Korean agent responses and localized search behavior for Korean-speaking users.

## Best Possible Solution

Keep the shipped Korean Control UI locale as-is, then handle the remaining work as a generic agent language-persistence and search-language design: preserve the user's primary language across final answers and tool-result explanations, define how search language should be selected across providers, and avoid Korean-only hardcoding.
