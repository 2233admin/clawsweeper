## Summary

Keep open. Current main has restored the duplicate heartbeat suppression half, but the plugin-channel fallback half is still not implemented end-to-end: a stored plugin route can be recovered from session metadata and then dropped when heartbeat routing calls the strict outbound resolver against an empty or unbootstrapped plugin registry. The attempted fix in #46203 was closed unmerged and does not appear in current main.

## Best Possible Solution

Keep this issue open and implement the remaining plugin-channel heartbeat fallback in the outbound heartbeat target path. When target last recovers a stored plugin route from session metadata, preserve or safely bootstrap that route if outbound normalization fails only because the plugin registry is empty or unbootstrapped. Leave the restored duplicate suppression intact, and add unmocked regression coverage for resolveHeartbeatDeliveryTarget() with an empty active plugin registry.
