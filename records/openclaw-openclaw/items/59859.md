## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:293-296`
  - body: The edit save path always writes `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may also use `kind: "every"` or `kind: "at"`, so saving one of those jobs after changing unrelated fields can fail validation or change when it runs. Carry the original schedule variant through the dialog or block editing unsupported schedule types.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts by session ID/sessionFile under the sessions directory. As written, the Log action usually misses existing local transcripts and falls back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builders interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6 setups. Normalize IPv6 literals to `[host]` before formatting these URLs.
  - confidence: 0.84
