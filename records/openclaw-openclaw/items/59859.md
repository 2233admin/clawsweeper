This PR adds a native C/GTK4/libadwaita Linux companion under `apps/linux`, a GTK3 AppIndicator tray helper, Linux gateway/systemd/RPC/onboarding/management surfaces and tests, a Linux gateway client ID, and an `agents.update` implicit-default-agent contract fix.

No, merge-as-is is not the best way to solve the issue. The contribution is useful, but the safe path is maintainer product selection, targeted fixes for the remaining contract bugs, and explicit security review before adding this native client surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and uses the textbox as `expr`. Existing jobs can have `kind: "every"` or `kind: "at"`, so saving an unrelated field can fail validation or change when the job runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores and resolves transcripts as direct session-id files under the sessions directory, with optional `sessionFile` metadata. Normal local transcripts will be missed and the Log action falls back to the dashboard even when a local file exists.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal like `::1`, this produces malformed URLs such as `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff on valid custom-bind setups.
  - confidence: 0.84
- **[P2] Resolve cron logs from session transcript paths:** `apps/linux/src/section_cron.c:186-190`
  - body: The cron Log action derives `<dirname(cronStore)>/sessions/<sessionKey>/transcript.jsonl`, but cron stores and session transcripts use different layouts, and transcripts are resolved from session IDs/files under agent session directories. This usually misses existing local transcripts before falling back to the dashboard.
  - confidence: 0.82
