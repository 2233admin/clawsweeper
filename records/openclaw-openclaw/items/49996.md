## Summary

Keep this PR open, but do not land it as-is. Current main only partially implements the intended strict-mode recovery and still uses a trim-based empty-result guard that matches the whitespace-prefix leak flagged in review. The PR also remains much broader than the stated Telegram/text fix, with unrelated generated backup files, hooks, gateway, UI, config, and command changes.

## Best Possible Solution

Keep the linked #49104 problem tracked and replace or reduce this PR to a narrow shared-text fix: use a zero-captured-content guard that counts whitespace as captured, add the whitespace-prefix regression test, and verify the Telegram split/dispatch path. The unrelated dist.bak, hook, gateway, UI, config, and command changes should be split into separate reviewable PRs or dropped from this branch.
