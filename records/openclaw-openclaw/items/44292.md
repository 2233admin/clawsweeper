## Summary

Keep this issue open. Current main still has the original per-key config label parity assertions and a manually maintained `FIELD_LABELS` table, with no autofix, scaffold command, or copy-paste-ready missing-label stub output. The direct implementation candidate #46304 is closed unmerged, so it does not resolve the remaining developer-experience request.

## Best Possible Solution

Keep this issue as the active tracker and implement the narrow developer workflow improvement in `src/config/schema.help.quality.test.ts`: collect all missing root/help label keys, derive conservative title-case label stubs, and fail once with copy-paste-ready additions for `src/config/schema.labels.ts`. A separate generator command can be considered later, but improving the parity test failure is the lowest-risk path because it changes only developer diagnostics and not runtime config behavior.
