## Summary

Keep this PR open. Current main partially mitigates repeated embedding calls with the default embedding cache, but it still does not implement the PR’s intended incremental session sync: changed session files still go through the full `indexFile` path and the writer clears/reinserts indexed rows for that file. The linked bug #40919 remains open, and the PR also needs a rebase plus fixes for the unresolved review comments before it can be a safe implementation candidate.

## Best Possible Solution

Keep this PR open as a possible implementation candidate for #40919. The best path is to rebase the work into `extensions/memory-core/src/memory`, implement transactional/conflict-safe incremental session indexing that preserves unchanged rows and refreshes FTS/vector metadata correctly, and add focused tests for append growth, overlap boundary changes, retry/partial-failure recovery, and keyword citation metadata.
