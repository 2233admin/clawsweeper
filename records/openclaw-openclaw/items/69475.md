## Summary

Keep #69475 open. Current main still has the reported skill-description and gh-issues size problems, and OpenClaw's own prompt path makes skill frontmatter descriptions part of the model's first-pass routing signal.

## Best Possible Solution

Keep the issue open and make a focused repo change. Update the four affected bundled skill `description` fields with clear what-and-when trigger text, including the reporter's follow-up about Contacts coverage for `gog` and less-overlapping Obsidian-specific wording. Split `skills/gh-issues/SKILL.md` into a short orchestrating SKILL.md plus `references/` files so it fits OpenClaw's local progressive-disclosure guidance. A lightweight static test or audit check for these bundled skills would be useful, but no core API change appears necessary.
