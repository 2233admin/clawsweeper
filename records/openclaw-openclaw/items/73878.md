## Summary

Keep this issue open. Current main still has the reported failure shape: gateway model probes are sent as raw model runs with tools disabled, but the embedded runner still evaluates the explicit tool-allowlist precheck and the guard still returns a fatal "No callable tools remain" error when explicit allowlist sources exist and callable tools are empty. The added reporter comment strengthens the embedded `tools.alsoAllow` impact, and current tests still cover flag forwarding/fail-closed behavior rather than the requested raw-model-run bypass.

## Best Possible Solution

Keep the issue open and fix the embedded model-run precheck. The narrow fix should make raw model probes, and possibly only the specific disabled-tools raw-run path, skip `buildEmptyExplicitToolAllowlistError` while preserving fail-closed behavior for normal agent runs with broken explicit allowlists. Add regression coverage for gateway `infer model run`/Gateway `agent` raw runs with configured `tools.allow`, plus a `tools.alsoAllow` case and a normal-agent test proving missing explicit tools still fail.

Keep the fix scoped to raw model probes or the equivalent disabled-tools raw-run path in the embedded runner.
