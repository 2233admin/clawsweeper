## Summary

Keep this issue open. Current main still routes `infer model run --gateway` as a raw model run with tools disabled, but the embedded runner still evaluates explicit tool allowlist sources and the guard still fails when callable tools are empty. The reporter's `tools.alsoAllow` production timeline is also plausible because `alsoAllow` is merged into the effective allow policy before the same empty-allowlist guard path.

## Best Possible Solution

Fix the embedded raw model-run precheck so `modelRun: true`, `promptMode: "none"`, or explicitly disabled-tools provider probes skip the empty explicit allowlist error, while ordinary agent runs with unavailable explicit tools still fail closed. Add regression coverage for `infer model run --gateway`/Gateway raw model runs with `tools.allow` and `tools.alsoAllow`, plus a normal-agent guard test that continues to fail when an explicit missing tool is requested.
