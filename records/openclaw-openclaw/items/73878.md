## Summary

Keep this issue open. Current main still has the reported failure shape: gateway model probes are routed as raw model runs with tools disabled, but the embedded attempt still runs the explicit tool-allowlist precheck and the guard still returns a fatal "No callable tools remain" error when explicit allowlist sources exist and callable tools are empty. The existing tests cover flag forwarding and fail-closed guard behavior, but not the requested bypass for `infer model run --gateway` with `tools.allow` entries.

## Best Possible Solution

Keep the issue open and fix the embedded model-run precheck. The narrowest maintainable path is to make raw model probes or `disableTools: true` runs skip `buildEmptyExplicitToolAllowlistError`, while leaving ordinary agent runs fail-closed when explicit allowlists resolve to no callable tools. Add regression coverage for `infer model run --gateway`/Gateway `agent` raw model runs with a configured `tools.allow`, plus a normal-agent test proving missing explicit tools still fail.
