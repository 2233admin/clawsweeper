## Summary

Keep this PR open. It addresses a real Control UI approval race that still exists on current main, but the proposed patch loses the approval kind after the queue entry is pruned, so plugin approvals would still resolve through the wrong backend method in the same race window.

## What This Changes

The PR updates Control UI approval buttons to pass a render-time approval ID into `handleExecApprovalDecision` and widens the shared view-state handler signature for that optional ID.

## Best Possible Solution

Keep the render-time capture approach, but capture enough stable metadata to resolve the approval through the correct backend path after queue mutation, then add focused UI regression coverage for exec and plugin approval button clicks after the queue entry is removed.
