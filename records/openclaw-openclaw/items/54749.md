## Summary

Keep this issue open. Current main still has report pages in the Memory Wiki/Memory Palace surface and a working wiki content preview bridge, but the report/page card tap path is still wired to expand details rather than open report content; only the nested Open wiki page button calls the content preview path.

## Best Possible Solution

Keep this open and add a focused Control UI fix plus regression test for report/page card selection in the mobile-width Memory Palace report list. The likely product fix is to make report card taps open the wiki.get preview, or to clearly separate expand/details from open-content controls so mobile and desktop cannot fall back to the Overview surface.
