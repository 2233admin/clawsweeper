## Summary

Keep this issue open. Current main still has the Memory Wiki/Memory Palace report surface and a wiki content preview path, but the report/page card tap path remains bound to expand/collapse details, while actual content opening is only wired to a nested button. The prior ClawSweeper comment reached the same conclusion, and current main at a820a307dfeb950fa633c0ae6fb2386a35a91991 does not add a mobile report-card regression or change this behavior.

## Best Possible Solution

Keep the issue open and add a focused Control UI regression around the mobile-width Dreaming/Memory Palace report list. The product fix should make report row/card selection open the wiki.get content preview, or deliberately split expand-vs-open behavior with clear controls that work on mobile and desktop and cannot fall back to the Overview surface.
