## Summary

Keep this issue open. Current main still treats Mattermost `root_id` as thread/session routing metadata, but the inbound Mattermost path does not resolve the referenced post, verify that it was authored by the bot, or pass a `reply_to_bot` implicit mention into the oncall mention gate. The related closed Mattermost threading item covered outbound/thread routing, not this inbound wake behavior.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: add safe referenced-post resolution for `root_id` and any supported parent/inline reply metadata, verify the referenced post was authored by the Mattermost bot user, pass `reply_to_bot` as an implicit mention into the Mattermost/shared mention gate, and add regression tests for unmentioned threaded and inline replies in default oncall mode. Update the Mattermost and group-channel docs when it ships.
