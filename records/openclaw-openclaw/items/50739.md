## Summary

Keep open. Current main still implements `system event --mode now` as queued system-event plus heartbeat wake delivery, and it still lacks a CLI flag, wake protocol field, gateway handling, TUI alert surface, or session-stream bypass for urgent in-session alerts. The KeepMyClaw comments are useful recovery context, but snapshots do not provide the requested immediate operator-visible alert path.

## What This Changes

The issue requests a high-priority or bypass-queue mode for `openclaw system event` so urgent system alerts appear immediately during congested session lanes.

## Best Possible Solution

Define a first-class urgent system alert contract that delivers immediate operator-visible notifications, with transcript or prompt mutation handled through a separately ordered and trust-reviewed path. The implementation should align the CLI, Gateway protocol, wake/cron path, heartbeat/session handling, TUI behavior, docs, and regression tests.
