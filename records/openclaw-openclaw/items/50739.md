## Summary

Keep open. Current main still lacks a priority or queue-bypass path for urgent system events: the CLI and wake protocol only carry `mode` and `text`, the gateway handler drops extra metadata, and `--mode now` still depends on heartbeat delivery that intentionally skips busy session lanes. The KeepMyClaw comments are useful recovery context, but external snapshots do not satisfy the requested immediate in-session alert surface.

## What This Changes

The issue requests a high-priority or queue-bypass mode for `openclaw system event` so urgent in-session alerts are not delayed behind congested session lanes.

## Best Possible Solution

Define an explicit high-priority alert contract that delivers operator-visible notifications immediately, and only mutates the transcript or prompt through a separately ordered, trust-reviewed path with protocol, CLI, Gateway, heartbeat, docs, and regression coverage aligned.

## Risks / Open Questions

- A direct prompt-visible bypass could race an active streaming turn or corrupt transcript ordering; current heartbeat code explicitly avoids busy session lanes.
- Because wake params allow unknown metadata, callers can send a `priority` field today and have it validate while current main silently ignores it.
- System events are formatted as trusted prompt-visible `System:` lines by default, so any bypass path needs spoofing and prompt-injection review.
- The product shape is unresolved: immediate operator/TUI notification may need to be separate from transcript or prompt mutation.
