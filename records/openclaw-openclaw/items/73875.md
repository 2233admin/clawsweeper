## Summary

Keep open. Current main still maps generic ACP runtime options to `model`, `thinking`, `approval_policy`, and `timeout`, applies them before each turn, and the bundled acpx wrapper only skips unsupported timeout/thinking controls for Codex. Claude ACP advertises `mode`, `model`, and `effort`, so this reported Claude failure mode remains valid. Open PR #73693 is the focused unmerged fix candidate, not proof that main is fixed.

## Best Possible Solution

Keep this issue open and use #73693, or a cleaner superseding PR, as the focused fix path. The desired implementation should stop mirroring `timeoutSeconds` as an ACP config option, derive/filter supported option ids from the target ACP session when available, map OpenClaw thinking to adapter-specific ids such as Claude `effort` when advertised, degrade gracefully for clearly unsupported config controls, and add regression coverage with a Claude-like non-Codex adapter that advertises only `mode`, `model`, and `effort`.
