## Summary

Keep open. Current main has the ACP control-plane surface for runtime options, but it still does not solve the reported Claude ACP adapter failure: `sessions_spawn(runtime:"acp")` persists generic model/thinking/timeout runtime options, maps them to ACP config keys such as `thinking`, `approval_policy`, and `timeout`, and either forwards them to non-Codex ACP adapters or fails before the turn when a strict advertised key list omits them. The only post-release mitigation I found is Codex-specific timeout handling in the bundled ACPX wrapper; the Claude path remains uncovered. A related ClawSweeper pass on #40393 already kept ACP thinking behavior open, and this issue is a concrete Claude-specific runtime-control failure in the same area.

## Best Possible Solution

Keep this issue open and fix the ACP runtime-control path generically: derive supported config option IDs per target adapter/session, map OpenClaw concepts to adapter-specific keys such as Claude `effort`/thought-level where advertised, skip unsupported generic keys like `timeout` and `approval_policy` without failing the turn, and add regression coverage with a Claude-like non-Codex ACP runtime that advertises only `mode`, `model`, and `effort`.
