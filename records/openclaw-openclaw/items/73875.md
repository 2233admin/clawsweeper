## Summary

Keep open. Current main still has the reported ACP runtime-control precondition: generic runtime options become ACP config option IDs and are forwarded when the backend does not advertise a strict key list. The issue also has open, unmerged PR #74048 using closing syntax, so this should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue asks OpenClaw-managed ACP sessions to avoid sending unsupported generic config options to strict Claude ACP adapters so trivial Claude ACP runs can complete.

## Best Possible Solution

The ACP runtime-control layer should derive supported config IDs from backend or session capabilities, skip unsupported generic keys for strict or no-list adapters, keep timeout as an OpenClaw execution timeout unless the adapter explicitly advertises it, map adapter-specific effort/thought-level controls where supported, and add regression coverage for strict-key and missing-key-list backends.

## Reproduction Assessment

Yes. The issue gives a concrete live reproduction path, and current-main source shows the static path from `sessions_spawn` generic options to ACP config IDs that are forwarded when no advertised key list exists.

## Solution Assessment

Unclear until PR #74048 is reviewed. The direction of skipping unsupported config options is appropriate, but the maintainable fix should align the capability contract and include regression tests before this issue is considered solved.
