## Summary

Keep open. The report has concrete Docker reproduction data, and current main plus the latest release still expose the affected `node dist/index.js` Docker CLI path while lean startup and benchmark coverage are centered on `openclaw.mjs`.

## What This Changes

The issue asks OpenClaw to remove the 24s-49s startup latency observed when running official Docker image subcommands through `node dist/index.js`.

## Best Possible Solution

Make official Docker CLI workflows and direct packaged `dist/index.js` main execution share the lean startup behavior already used by `openclaw.mjs`, or move the Docker user-facing commands to the wrapper path, with an explicit startup budget for the reported direct-file entry.

## Reproduction Assessment

Yes. The issue provides exact Docker commands and timings, and current main still exposes the same command path in Compose, setup, and docs; the remaining check is to benchmark the latest built image to quantify the current delay.

## Solution Assessment

Unclear because there is no patch yet. The narrow maintainable direction is to route direct-file Docker CLI execution through the lean entry bootstrap or update the official Docker CLI surface to use `openclaw.mjs`/`openclaw`, while preserving package-root exports.
