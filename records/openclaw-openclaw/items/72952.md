## Summary

Keep open. Current main still exposes the reported official-Docker `node dist/index.js` CLI path through compose, setup, docs, and tests, while direct execution of `src/index.ts` still goes through the legacy CLI loader rather than the lean `openclaw.mjs`/`src/entry.ts` startup path. The older matching reports in the provided context were closed as duplicates, not as evidence that the latency was fixed, so this remains an actionable tracker.

## What This Changes

The issue reports that `node dist/index.js` subcommands inside the official Docker image take roughly 24-49 seconds to start, even though a trivial `node -e` command in the same container is fast.

## Best Possible Solution

Make the packaged direct `dist/index.js` CLI path and the official Docker CLI workflows use the same lean startup behavior as `openclaw.mjs`, while keeping package-root library imports stable. Add a `dist/index.js` startup benchmark or budget so this Docker-facing path stays covered.
