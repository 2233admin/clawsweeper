## Summary

Keep open. Current main has adjacent command, tool, install, and agent lifecycle hooks, but it still lacks a skill-specific before/after execution hook or skill invocation event. The enterprise permission-check comment makes this a core/plugin API design request, not a ClawHub-only skill, and related PR #18889 covers agent/tool lifecycle boundaries rather than this skill surface.

## What This Changes

Request an additive skill lifecycle hook surface, such as before/after skill execution events, so plugins or hooks can log, monitor, or block skill invocations.

## Best Possible Solution

Add an explicit, documented plugin hook surface for skill command invocation with a blocking pre-invocation decision hook and a post-invocation observation hook. The contract should carry bounded skill/command/source/session/run metadata, define redaction and failure semantics, and document how it applies to slash-command skills versus model-selected skills.

## Risks / Open Questions

- Skill invocation semantics need product/API design because model-selected skills are prompt instructions and may not correspond to a discrete runtime execution boundary.
- A blocking pre-skill hook for enterprise policy is security-sensitive: it needs fail-closed behavior, bounded metadata, and careful redaction of user input, tool args, secrets, and skill context.
- The API should distinguish slash-command skill invocations, tool-dispatched skill commands, and normal model use of skills before an implementation PR is queued.
