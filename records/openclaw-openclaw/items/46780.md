## Summary

Keep open. Current main has internal command hooks plus plugin agent/tool hooks, and skills can be exposed as slash commands, but there is still no skill-specific before/after lifecycle hook or documented skill execution event. The follow-up comment about enterprise permission checks reinforces that this is a missing core/plugin API design point, not a ClawHub-only implementation request.

## Best Possible Solution

Keep this open and design an additive plugin hook surface for explicit skill invocation first, likely around `src/auto-reply/reply/get-reply-inline-actions.ts` and the skill command/runtime surfaces. A good path would add a pre-invocation decision hook for block/approval plus a post-invocation observation hook with skill name, command name, source, agent/session/run context, success/error, and bounded args metadata, then document the contract in `docs/plugins/hooks.md` and `docs/tools/skills.md` with focused tests. Model-selected skill usage should be called out separately because skills are prompt instructions, not always discrete runtime executions.
