## Summary

Keep open. Current main has the newer provider replay-policy architecture, but it still does not implement this PR's central behavior: no openai-completions thinking-signature boundary sanitizer, no history-aware signature-preservation policy, and no OpenRouter-wrapped Mistral strict9 replay-policy handling. The PR discussion shows concrete review feedback was addressed and a later Codex review found no major issues, so this should be rebased or superseded rather than closed as obsolete.

## Best Possible Solution

Keep this PR open, or replace it with a rebased current-main patch in `src/agents/pi-embedded-runner/replay-history.ts` and provider replay policy helpers. The fix should preserve compatible provider-native thinking signatures until the replay target is known, strip unsupported signatures only for `openai-completions`, account for current completions fields such as `reasoning_details`, and handle OpenRouter-wrapped Mistral strict9 replay policy without a broad global hint table.
