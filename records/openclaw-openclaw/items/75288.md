This PR adds a scoped legacy runtime-deps symlink removal helper, wires it into bundled plugin runtime-deps staging paths, and adds a test for migrating legacy `.local/bundled-plugin-runtime-deps` symlinks.

## Best Possible Solution

Add the required `CHANGELOG.md` entry, then land this PR or an equivalent patch that keeps symlink removal scoped to OpenClaw-owned legacy runtime-deps paths and preserves refusal for arbitrary symlinks.

## Solution Assessment

Mostly yes. The code change is the narrow maintainable fix because it removes only legacy OpenClaw-owned runtime-deps symlinks immediately before the existing replacement/removal operations; the remaining issue is the missing changelog entry required for a user-facing fix.
