## Summary

Keep this PR open. Current main still converts under-cap non-alpha PNGs through JPEG optimization, still renames JPEG outputs to .jpg only for HEIC sources, and the PR targets old `src/web/*` paths that no longer exist. The PR remains a plausible focused fix, but it needs a rebase plus the PNG validation concern from review addressed before merge.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `src/media/web-media.ts` plus current media tests. The best fix is to preserve already-under-cap, validated PNG images without JPEG re-encoding, normalize filenames for every actual JPEG output, and add regression coverage for under-cap PNG preservation plus PNG-to-JPEG filename normalization.
