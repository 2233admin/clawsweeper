## Summary

Keep this PR open. Current main still converts under-cap non-alpha PNGs through the JPEG optimizer, still normalizes JPEG output filenames only for HEIC sources, and the PR's old `src/web` paths need rebasing to the current `src/media/web-media.ts` surface before it can land.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `src/media/web-media.ts` plus current media tests. The best fix is to preserve under-cap PNGs after validating the PNG payload, normalize filenames for every actual JPEG output, and add focused regression coverage for under-cap PNG preservation plus PNG-to-JPEG filename normalization.
