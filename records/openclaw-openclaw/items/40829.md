## Summary

Keep this PR open. Current main still has the behavior this PR is trying to fix: non-alpha under-cap PNGs fall through to JPEG optimization, JPEG output filenames are only normalized for HEIC sources, and the active code has moved from the PR's old `src/web/media.ts` path to the shared plugin-facing loader in `src/media/web-media.ts`.

## Best Possible Solution

Keep the PR open or supersede it with a rebased fix on `src/media/web-media.ts`: preserve valid under-cap PNG bytes without JPEG recompression, normalize filenames whenever the actual optimized output is JPEG, and add focused regression tests in `src/media/web-media.test.ts` plus `extensions/whatsapp/src/media.test.ts`.
