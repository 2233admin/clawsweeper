## Summary

Keep this open. Current main has adjacent hooks for pre-tool blocking, final-answer revision, reply dispatch, and outbound message cancellation, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded retry/fallback policy, and audit records. The discussion usefully identifies complementary plugin implementations, but also preserves the core gap: existing plugins can gate tools or filter/cancel output, not mechanically prove required tools and validators passed before final response delivery.

## What This Changes

The issue requests a pre-response enforcement layer that prevents final answers until mandatory tool calls and configured validators run and pass, with two-phase planning, retry/fallback behavior, and audit logging.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item. The desired end state is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain policy packs implemented as plugins or ClawHub packages.
