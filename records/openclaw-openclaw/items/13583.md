## Summary

Current main still has adjacent hook surfaces for pre-tool blocking, final-answer revision, reply dispatch, and outbound message cancellation/rewrite, but it does not implement the requested fail-closed pre-response `responseGate` with mandatory tool-call accounting, typed validators, bounded retry/fallback behavior, two-phase tool-only planning, or audit records. The issue discussion and prior ClawSweeper review point to useful complementary external/plugin work while identifying the missing response-path core/plugin API seam.

## Best Possible Solution

Keep this issue open as the canonical response-gate policy-as-code design item. The best maintainer path is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records; domain-specific policy packs can then live in plugins or ClawHub.
