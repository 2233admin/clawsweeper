## Summary

Current main still does not implement the requested BlueBubbles 1:1 sender-name enrichment. The existing Contacts resolver is only invoked for group messages, it resolves unnamed phone participants only, tests explicitly preserve skipping email addresses, and there is no open PR found that already carries the proposed fix. The issue also has maintainer signal asking for a PR covering the phone and email pieces, so it should stay open as actionable channel-plugin work.

## What This Changes

Broaden BlueBubbles local Contacts enrichment from group participant lists to 1:1 inbound sender labels, including phone and email handles, while preserving backward-compatible configuration.

## Best Possible Solution

Implement this inside the bundled BlueBubbles plugin by extending the existing local Contacts resolver to handle 1:1 sender labels and email participant handles, while keeping authorization/mention gating before any local Contacts reads and preserving compatibility for existing configs.
