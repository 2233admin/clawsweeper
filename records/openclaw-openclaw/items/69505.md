## Summary

Keep this open. Current main still does not implement an `assistantPrefix` or equivalent provider-prefill contract for cron `agentTurn`, direct agent RPC/CLI runs, or agent defaults; existing `responsePrefix` is only applied after model output exists and does not prevent streamed preamble/narration.

## What This Changes

Add an optional `assistantPrefix` or equivalent field across cron `agentTurn` payloads and agent defaults so provider requests can force assistant replies to start with fixed text.

## Best Possible Solution

Keep this open for maintainer API/design review; if accepted, implement an additive assistant-prefill contract across cron payloads, direct agent params/CLI, agent defaults, cron/direct execution, provider hooks, docs, and focused streaming/fallback tests with explicit unsupported-provider no-op/debug behavior.
