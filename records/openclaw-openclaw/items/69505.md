## Summary

Keep issue #69505 open. Current main does not expose `assistantPrefix` or an equivalent agent-turn/defaults field, the cron protocol/CLI/runtime only carry the existing model/fallback/thinking/timeout/lightContext/tool fields, and the current text-output coalescing mitigation is channel-scoped rather than a protocol-level fixed-prefix guarantee. The reported WhatsApp cron scenario is not solved by the existing Telegram/Discord-style `preferFinalAssistantVisibleText` opt-in because WhatsApp does not opt into that outbound policy on current main.

## Best Possible Solution

Keep this open for maintainer product/API design. The likely implementation is an additive `assistantPrefix`-style field in cron payloads and agent defaults, aligned across TypeScript types, protocol schema, config schema/help, CLI add/edit/run surfaces, and the embedded runner/provider stream path. It should include provider-family implementations for at least Anthropic and one OpenAI-compatible transport, no-op/debug behavior where unsupported, and regression tests proving cron-delivered text starts with the configured prefix rather than relying on prompt obedience or post-hoc channel filtering.
