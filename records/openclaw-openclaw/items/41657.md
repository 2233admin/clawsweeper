## Summary

Keep this issue open. Current main has Slack file data at ingestion and can download files or expose filename/fileId placeholders, but it still does not deliver passive structured attachment metadata with id/name/MIME/size to agents. The related open Slack PR is narrower media hydration work, not a replacement for the requested shared inbound attachment metadata contract.

## Best Possible Solution

Implement a backward-compatible shared inbound attachment metadata contract in MsgContext and the plugin SDK, then populate it from Slack message.files with provider, provider file id, file name, MIME type, size, downloaded/local-path state, and placeholder text. Preserve existing BodyForAgent placeholders and MediaPath/MediaPaths/MediaTypes behavior, add Slack and shared channel contract tests, update Slack/plugin docs, and review #62792 separately as narrower media hydration work.
