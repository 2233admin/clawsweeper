## Summary

Keep this issue open. Current main still hardcodes the DM pairing reply text and CLI hint, and there is no supported `channels.*.pairingMessage` or `showCliHint` config surface. The related same-author PR #42617 remains the active implementation candidate, so maintainers should review, revise, land, or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open and review PR #42617 as the active implementation candidate. The durable fix should add a typed, schema-validated, documented `pairingMessage` config surface, preserve the current default text for backward compatibility, wire it through pairing-capable bundled channels or clearly scope the first supported channels, and add focused tests for text overrides and CLI-hint suppression.
