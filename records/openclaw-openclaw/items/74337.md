## Summary

Keep this PR open. Current main has the related precise UTC quarter-hour message buckets for Peak Error Hours from #49396, but it does not yet have per-session token buckets, and Usage Mosaic plus hour filtering still use the legacy session-span/proportional token allocation path. The PR is still useful implementation work rather than obsolete or already implemented.

## What This Changes

The PR adds per-session UTC quarter-hour token usage buckets to session cost summaries and updates Control UI Usage Mosaic/hour filtering to prefer those buckets while preserving the legacy session-span fallback.

Review and iterate this PR on its merits, then land it if the added optional token-bucket contract, Usage Mosaic aggregation, fallback behavior, tests, changelog entry, and changed-lane validation are acceptable.
