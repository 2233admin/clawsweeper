## Summary

Keep this PR open. Current main has a bundled Groq plugin, docs, env-var discovery, media-understanding, and reasoning-compat support, but it still does not implement this PR’s central text-inference onboarding/catalog path: no `groq-api-key` auth choice, no `--groq-api-key` manifest flag, and no Groq model catalog/static catalog in the bundled plugin. The PR is not mergeable as-is because it targets older core wiring and includes unrelated security-sensitive baseline/allowlist churn, but the user-facing gap remains real and should be reworked in `extensions/groq`.

## Best Possible Solution

Keep the PR open for maintainer/contributor rework or replace it with a fresh PR against current main. The correct implementation should update `extensions/groq`: add plugin-owned API-key auth metadata (`groq-api-key` and `--groq-api-key`), a Groq text model catalog/static catalog, default model config through current provider onboarding helpers, and focused tests for picker, CLI, non-interactive onboarding, and catalog listing. The unrelated secret-scanning baseline/allowlist changes should be removed from this feature PR.
