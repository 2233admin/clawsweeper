## Summary

Keep this PR open. Current main has a bundled Groq plugin, Groq docs, env-var discovery, media-understanding support, and reasoning compatibility, but it still does not implement this PR's central onboarding/catalog behavior: no `groq-api-key` auth choice, no `--groq-api-key` onboarding flag, and no Groq text model catalog/static catalog. The PR should be reworked for the current plugin-owned provider architecture rather than merged as older core wiring.

## Best Possible Solution

Keep the PR open for maintainer or contributor rework, or replace it with a fresh PR against current main. The right implementation belongs in `extensions/groq`: add plugin-owned API-key auth metadata (`groq-api-key` and `--groq-api-key`), Groq text model catalog/static catalog, default-model config through current provider onboarding helpers, and focused picker/CLI/non-interactive/catalog tests. Drop the unrelated secret-scanning baseline and allowlist-comment changes from this feature work.
