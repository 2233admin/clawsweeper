## Summary

Keep this PR open. Current main has a bundled Groq plugin, Groq docs/env-var metadata, media-understanding registration, and reasoning compatibility, but it still does not implement the PR's central user-visible behavior: `groq-api-key` auth choice, `--groq-api-key` onboarding flag, or a Groq text model catalog/default model. The PR is also stale against the current plugin-owned provider architecture and includes unrelated secret-scanning churn that should be dropped during rework.

## Best Possible Solution

Keep the PR open for rework, or replace it with a fresh PR against current main. The right implementation belongs in `extensions/groq`: add plugin-owned API-key auth metadata (`groq-api-key` and `--groq-api-key`), a Groq text model catalog/static catalog, default-model config through the current provider onboarding helpers, and focused tests for picker, CLI, non-interactive onboarding, env-ref storage, and catalog behavior. Drop the unrelated secret-scanning baseline and allowlist-comment changes from the feature PR.
