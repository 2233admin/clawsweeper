## Summary

Current main still has only partial tool-error guidance. Edit mismatch recovery appends bounded current file contents, and exec timeouts have retry guidance, but apply_patch missing-context failures and exec command-not-found failures remain minimal, and the bundled-only tool-result middleware path is not a general external workaround.

## Best Possible Solution

Keep this issue open and implement a focused core tool-runtime improvement: add shared bounded diagnostics for edit/apply_patch missing text and ambiguous matches, include safe line-numbered nearby snippets and concrete retry guidance, and add conservative exec command-not-found hints. Keep the behavior in core tool error paths unless maintainers explicitly design a broader trusted plugin API for tool-result rewriting.
