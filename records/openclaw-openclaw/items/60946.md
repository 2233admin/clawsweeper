## Summary

Keep open. Current main still only partially addresses the request: edit mismatch failures append bounded current file contents, but similar-match guidance, duplicate match locations, apply_patch mismatch diagnostics, and exec command-not-found recovery hints are still missing. This is also not a ClawHub-only close because the public plugin path cannot rewrite high-trust tool results before the model sees them.

## What This Changes

Enhance core tool error responses with contextual troubleshooting hints, nearby or duplicate match information, and actionable retry suggestions for edit, apply_patch, and exec failures.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime, then wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or broadening plugin trust boundaries.
