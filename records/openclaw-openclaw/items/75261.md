## Summary

Keep open. Current main still has the exact remaining gap described by the report: the cron tool preserves a canonical delivery target only when `currentDeliveryContext.to` reaches it, but the MCP loopback path scopes tools with session/channel/account context only and does not pass the current target or thread through. That leaves implicit cron delivery able to fall back to parsing the lowercased session key for Matrix room IDs.

## What This Changes

The issue asks OpenClaw to preserve canonical Matrix room ID casing when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path without explicit delivery.

## Best Possible Solution

Carry canonical delivery context through the MCP loopback path for session-scoped tool calls, or resolve it from persisted session metadata before cron falls back to session-key parsing. The fix should stay generic for case-sensitive channel targets and add regression coverage for mixed-case Matrix room IDs created through Claude CLI bundle-MCP cron calls.

## Reproduction Assessment

Yes. The issue provides a concrete v2026.4.27 reproduction with stored job JSON and Matrix delivery errors, and current main source still shows the missing loopback target propagation needed to avoid the lowercased fallback.

## Solution Assessment

Yes. Extending the existing delivery-context seam into the loopback tool-resolution path is the narrowest maintainable fix; adding Matrix-specific special cases inside cron would be a less desirable fallback.
