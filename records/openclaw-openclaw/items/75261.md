## Summary

Keep open. Current main still preserves canonical Matrix cron delivery only when `currentDeliveryContext` reaches `createCronTool`; the Claude CLI MCP loopback path still passes only session, account, and channel context into tool resolution, so implicit delivery can fall back to the lowercased session key despite #71798.

## What This Changes

Reports that `mcp__openclaw__cron action=add` with `sessionTarget: "current"` and no explicit `delivery` lowercases mixed-case Matrix room IDs when invoked through Claude CLI MCP loopback.

## Best Possible Solution

Make loopback-scoped tool resolution carry a generic, case-preserving `DeliveryContext` into `createOpenClawTools`, or resolve that context from persisted session delivery metadata before cron falls back to session-key parsing. The fix should stay channel-agnostic and cover mixed-case Matrix room IDs through the Claude CLI MCP path.

## Reproduction Assessment

Yes. The issue gives concrete steps, persisted job JSON, and Matrix delivery errors, and current-main source still shows the missing loopback target propagation needed to avoid the lowercased fallback.

## Solution Assessment

Yes. Extending the existing delivery-context seam into the MCP loopback path, or resolving the same generic context from session metadata, is the narrowest maintainable fix; a Matrix-specific cron special case would be less robust.
