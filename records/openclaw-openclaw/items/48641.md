## Summary

Keep this issue open. Current main still resolves Discord account config through the generic account merge helper, so account-scoped `allowFrom` replaces the top-level list; the DM preflight path then authorizes inbound DMs against that resolved per-account list. The related fix PR #49365 is closed and unmerged, and current docs/tests still preserve the override semantics, so this needs a maintainer access-control decision rather than cleanup closure.

## Best Possible Solution

Keep the issue open for a maintainer-owned Discord access-control decision. Define whether top-level `channels.discord.allowFrom` is a shared allowlist, a default inherited only when an account `allowFrom` is unset, or a list that unions with every account. Then implement that chosen behavior in `extensions/discord/src/accounts.ts` or the DM preflight access path, add regression coverage for default and named accounts, improve unauthorized/drop diagnostics if needed, and update `docs/channels/discord.md` plus changelog because the behavior is user-facing and security-relevant.
