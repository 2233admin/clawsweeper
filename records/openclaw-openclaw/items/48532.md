## Summary

Keep open. Current main has partial exec-approval hardening, including Windows-only `argPattern` support, strict inline-eval controls, and exact-command durable fallback for cases that cannot persist a path pattern, but it still does not implement the issue’s central cross-platform binary+argv allowlist model, deny-over-allow full-command rules, dry-run matcher, approval-log clustering, or progressive rule proposal flow.

## Best Possible Solution

Keep this open as a maintainer-owned exec-approval policy design. The durable path should define a cross-platform binary+argv rule model, deny-over-allow full-command matching, a dry-run matcher, audit/approval history, and an opt-in rule-generalization workflow that proposes scoped patterns only after explicit user confirmation. Maintainers should decide how to split or cross-link the work with #41140, #6615, #39979, #46956, and the closed Phase 1 attempts #48586/#48590.
