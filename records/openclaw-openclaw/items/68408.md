## Summary

Keep open. Current main still returns raw daily-memory file content through QmdMemoryManager.readFile(), so this PR targets a real memory isolation gap, but the proposed branch is not mergeable as-is because daily-file detection is based on the raw requested path, the new public SDK export lacks the generated API baseline update, and the helper can rewrite ordinary daily-note formatting.

## What This Changes

This PR adds a Plugin SDK markdown helper for stripping managed Light Sleep and REM dreaming blocks, wires it into memory-core QMD daily-memory full and partial reads, and adds focused helper/read-path tests.

## Best Possible Solution

Land a corrected memory-core read-path filter that classifies daily files from the resolved canonical workspace-relative path, strips only managed Light/REM blocks from both full and paginated memory_get reads, preserves ordinary user-authored formatting when no managed block is removed, keeps dreaming internals reading raw files, and updates the Plugin SDK API baseline if the helper remains public.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Normalize paths before daily-memory detection:** `extensions/memory-core/src/memory/qmd-manager.ts:91`
  - body: resolveReadPath() canonicalizes the request before reading, but the new daily-file check tests the raw relPath string. A request like memory/./2026-04-18.md resolves to the same daily file while skipping the stripping branch, so managed Light/REM blocks can still be exposed. Base this decision on the resolved workspace-relative path instead.
  - confidence: 0.91
- **[P2] Update the Plugin SDK API baseline:** `src/plugin-sdk/memory-host-markdown.ts:65`
  - body: memory-host-markdown is a published Plugin SDK entrypoint, and this PR adds a new public export from it. Without regenerating and committing docs/.generated/plugin-sdk-api-baseline.sha256, pnpm plugin-sdk:api:check will fail in CI.
  - confidence: 0.9
- **[P2] Only collapse blank lines after stripping a block:** `src/plugin-sdk/memory-host-markdown.ts:75`
  - body: stripDreamingManagedBlocks() always rewrites runs of blank lines, even when no dreaming markers were removed. Daily memory_get reads should remain exact excerpts for ordinary notes unless a managed block was actually stripped.
  - confidence: 0.86
