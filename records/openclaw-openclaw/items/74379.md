## What This Changes

Add a config-level exec path denylist that blocks commands whose literal path-like arguments match sensitive glob patterns before execution or approval.

## Best Possible Solution

Review and land PR #75101, or a narrow replacement, that adds a hard-deny `tools.exec.denyPathPatterns` gate before allowlist/approval execution, updates config schema/types/help/docs, and covers literal argv, tilde/home expansion, cwd-relative paths, shell-wrapper payloads, safeBins interaction, and documented bypass limits.
