## Summary

Current main still has the missing-payload completion path: `runSubagentAnnounceFlow` retries child output, then falls through to `(no output)` and delivers that as a task completion result. The regression test suite still asserts that a completion with only child user text announces `(no output)`. Open PR #68464 remains the active unmerged implementation candidate for durable user-safe payloads and fail-closed completion delivery, so this issue should stay open until that or a narrower successor lands.

## Best Possible Solution

Keep this issue open and resolve it through review/landing of PR #68464 or a narrower successor. The code should defer or retry completion delivery when a completion message is expected but no usable child reply, structured delivery payload, or sanitized fallback exists; it should persist a user-safe payload or retryable delivery claim, avoid `(no output)` as a successful completion result, and add a regression test for the missing-payload race.
