## Summary

Keep this PR open. Current main still has the wizard finalize path probe the gateway with `settings.gatewayToken`, but then calls `healthCommand` without passing a config pinned to that setup token, so the linked token-drift bug remains possible. The PR is a focused implementation candidate for open bug #72203 rather than obsolete work.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch that builds a health-check config from `nextConfig` and pins `gateway.auth.token` to the setup session token before calling `healthCommand`, with a regression test asserting the exact token forwarded when config and setup tokens differ. After merge, close linked bug #72203 as implemented if the same failing QuickStart path is covered.
