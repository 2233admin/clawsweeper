## Summary

Keep this PR open. Current main still calls the onboarding final health check without a pinned setup token, while the linked open bug reports that exact QuickStart token-auth mismatch. The PR remains a focused implementation candidate, with a small security-sensitive auth surface to review and unrelated docs whitespace that could be split before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch that lets the setup finalization health check use the just-resolved setup token deterministically. Keep the regression test asserting `healthCommand` receives the setup token when env/config tokens differ, and then close #72203 after the fix merges. Consider splitting the unrelated docs whitespace changes before landing.
