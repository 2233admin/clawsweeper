## Summary

Keep open. Current main does not definitely implement this PR’s intended silent-completion logging behavior: exact `NO_REPLY` is recognized as deliberate silence for incomplete-turn handling, but it can still collapse to empty payloads, and the local delivery and gateway CLI paths still log fallback output when payloads are empty. The PR is narrow runtime/test work with no evident security or supply-chain-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review or update. The best fix is still to carry an explicit, typed signal for exact `NO_REPLY` silent completions from the embedded runner into both local delivery and gateway CLI output, preferably with a shared predicate and focused regression tests for both logging surfaces.
