## Summary

Keep this PR open. Current main still does not implement the intended silent-completion logging behavior: exact NO_REPLY can collapse to an empty payload list, but both local delivery and gateway CLI still log fallback output for empty payloads without a silent-completion marker. The PR remains a focused implementation candidate, though it should be rebased against newer runner metadata.

## Best Possible Solution

Keep this PR open for maintainer review or update. The best path is to preserve an explicit typed signal for exact NO_REPLY silent completions, or reuse current finalAssistantRawText metadata safely, then gate both local delivery and gateway CLI empty-payload logging with focused regression tests for both surfaces.
