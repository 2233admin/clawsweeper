## Summary

Keep this issue open. Current main still converts inbound audio into media-understanding/STT text and only forwards text plus image prompt attachments to the agent; it does not have a typed raw-audio prompt payload or config mode. The directly related native audio/video forwarding PR, #48900, is still an unmerged implementation candidate in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the product tracker for native inbound audio forwarding. The best path is to review and harden #48900 or a successor that adds a typed prompt-media contract for audio/video, config for transcript-only versus raw/audio+transcript behavior, capability detection, payload and token limits, provider adapter support, privacy/egress review, docs, and regression coverage for Telegram/Signal-style voice notes.
