## Summary

Keep this PR open. Current main still lacks the requested Docker-daemon-unavailable handling on the sandbox image-probe paths, the linked bug remains open, and the current PR head is still an implementation candidate rather than something already implemented on main. The PR also needs maintainer fixes before merge because the current diff appears to break `browser.ts` syntax and imports a helper from a barrel that the PR does not update.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reproduce or trace the mode-off cron/heartbeat/sub-agent startup path that reaches Docker, then fix the earliest incorrect probe path or add a narrow shared Docker-daemon-unavailable classifier with regression coverage. The PR should also repair the current `browser.ts` syntax issue, export/import the helper through an intentional internal surface, avoid broad stderr swallowing, and keep the default sandbox image policy unchanged unless maintainers explicitly approve a policy change.
