## Summary

Keep this PR open. Current main still lacks the requested Docker-daemon-unavailable handling on the sandbox image-probe paths, the linked bug #73586 is still open, and this PR is an active implementation candidate rather than work already implemented on main. The current PR head also appears not merge-ready because the diff removes the `ensureSandboxBrowserImage` function declaration and imports `isDockerDaemonUnavailable` through a barrel that current main does not export.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to reproduce or trace the mode-off cron/heartbeat/sub-agent startup path that still reaches Docker, then fix the earliest incorrect probe path or add a narrow shared Docker-daemon-unavailable classifier with regression coverage. Before merge, the PR should repair the `browser.ts` syntax issue, export/import the helper through an intentional internal surface, keep the default sandbox image policy unchanged, and avoid masking unrelated Docker errors.
