## Summary

Current main still has the reported model-usage parsing gap: numeric-string costs are skipped, while booleans and non-finite native numbers are not filtered. The linked implementation PR #37877 is closed and unmerged, and the latest release has the same behavior.

## Best Possible Solution

Implement the narrow parser fix in skills/model-usage/scripts/model_usage.py: add a shared finite cost parser that accepts finite numeric strings, rejects booleans, rejects non-finite values, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py and a user-facing changelog fix entry crediting @shuofengzhang.
