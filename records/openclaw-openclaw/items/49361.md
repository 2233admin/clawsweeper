## Summary

Keep this PR open. Current main still routes the Google Chat typing indicator name through `resolveBotDisplayName`, and that fallback chain only checks the account config name, then `agent.name`, then hardcoded `OpenClaw`; it does not use `agents.list[].identity.name`. The PR is a narrow implementation candidate for the typing-indicator half of the linked Google Chat UX issue, and current main has not implemented the same behavior elsewhere.

## Best Possible Solution

Review and land this narrow Google Chat fix, ideally with a focused regression test that exercises `typingIndicator: "message"` for an agent configured with `identity.name` and no top-level `name`. Track the separate markdown-rendering behavior from the linked issue independently if it is still reproducible.
