## Summary

Keep open. Current main still falls back from Google Chat account name to `agent.name` and then hardcoded `OpenClaw`, so the PR's `agents.list[].identity.name` typing-indicator fix is still not implemented. The linked #49350 also has a separate markdown-rendering concern that this PR does not cover.

## Best Possible Solution

Review and land the narrow Google Chat plugin fallback change, ideally with a small regression test for `typingIndicator: "message"` where the routed agent has `identity.name` but no top-level `name`. Keep the markdown rendering problem from #49350 tracked separately unless another change addresses it directly.
