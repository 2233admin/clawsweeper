## Summary

Keep open. Current main does not implement the intended behavior: chat.history display projection still defaults to an 8,000-character text cap, truncates by character count only, and then applies the 128 KiB JSON-sized message placeholder path. The PR is still a plausible implementation candidate for open companion issue #53242, but it needs rebasing onto the current projection layer and should address the JSON-encoded byte-budget review comment before maintainers consider landing it.

## Best Possible Solution

Keep this PR open as the implementation candidate for #53242. The best path is to update it against current main by moving the cap and truncation change into src/gateway/chat-display-projection.ts, deciding the intended default versus config override behavior, making the byte budget match JSON-serialized message size, and adding focused gateway regressions for long ASCII, emoji/CJK, and escape-heavy assistant text.
