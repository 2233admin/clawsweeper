## Summary

Keep open: current main still does not implement the requested long-text preservation or byte-aware visible-prefix fallback, and this contributor PR remains paired with the open bug report. The branch is useful source work, but it needs maintainer/contributor rework because current main moved the behavior into the shared projection helper and the proposed fallback still budgets raw UTF-8 text instead of serialized history-message bytes.

## What This Changes

The PR raises the old gateway `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation before the oversized placeholder path, and adds gateway tests for long ASCII and emoji-heavy assistant replies.

## Best Possible Solution

Move the fix into the shared chat-display projection path, preserve long assistant text when the fully serialized history entry fits the hard cap, truncate UTF-8-heavy and JSON-escape-heavy text against the serialized message budget before placeholder replacement, and align gateway/UI tests plus config help or docs with the chosen default behavior.
