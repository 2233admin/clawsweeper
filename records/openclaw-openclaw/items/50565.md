## Summary

Keep open. Current main no longer has the old Feishu-specific `tool call` log text, but the remaining attribution bug is still present: embedded tool lifecycle logs are emitted through a static `agent/embedded` logger, while the subscription layer is not passed the originating message channel that the runner already has.

## What This Changes

The issue requests that tool call start/end/failure logs be attributed to the originating channel subsystem, such as `gateway/channels/telegram`, instead of a Feishu or static embedded subsystem.

## Best Possible Solution

Thread a typed channel-origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, derive `gateway/channels/<channel>` only for valid message-channel runs, retain `agent/embedded` fallback behavior, and add focused regression tests around tool lifecycle attribution.
