## Summary

Keep open. Current main still lacks the Linux/systemd self-restart handoff and marker-based SIGTERM restart semantics proposed by this PR; the PR remains a focused implementation candidate with reported live user-systemd smokes and a later branch repair.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and landing after current CI plus a Linux user-systemd smoke confirm self-restarts return `scheduled`, external restarts still return `completed`, and the new handoff/marker path is safe.
