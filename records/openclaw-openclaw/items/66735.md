## Summary

Keep open. Current main still restarts Linux systemd services by running a synchronous `systemctl --user restart <unit>` path and still treats plain `SIGTERM` as stop semantics unless the generic restart intent is present. The PR’s intended behavior, a self-restart-specific `systemd-run --user` handoff plus restart expectation marker handling, is present in the PR diff and active discussion, but not in current `main`.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and landing after CI plus targeted or live Linux user-systemd validation; current main does not yet implement the self-restart handoff or marker-based SIGTERM restart semantics.
