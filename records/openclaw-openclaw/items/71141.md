## Summary

Keep open. Current main still lacks the Telegram reaction-directive normalization and delivery path, and this PR is a useful paired implementation for the open bug, but the submitted branch is stale against current outbound normalization and has blocking TypeScript/API issues before it can merge.

## What This Changes

The PR adds parsing for Telegram reaction directives, stores the reaction in outbound Telegram channel data, allows reaction-only Telegram replies, sends native Telegram reactions, and adds payload/adapter tests.

## Best Possible Solution

Update this PR or a narrow replacement so reaction parsing is added on top of current main, preserves markdown-image extraction, uses the existing Telegram reaction API contract correctly, and covers normalization, adapter, and Telegram delivery behavior for reaction-only and reaction-plus-media replies.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction: `[[react_to_current:...]]` is not parsed into channel data, Telegram delivery rejects empty text/media replies, and the paired issue provides live Telegram confirmation of the same user-visible failure.

## Solution Assessment

No as submitted. The feature direction is the right narrow fix, but this branch needs to be rebased and corrected so it preserves existing outbound normalization behavior and typechecks against `reactMessageTelegram`.
