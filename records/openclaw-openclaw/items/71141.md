## Summary

Keep open: current main still lacks the Telegram reaction directive path, and this draft PR is the paired implementation for the open bug, but the submitted branch is dirty and has blocking source-level correctness/type issues before it can merge.

## What This Changes

The PR parses Telegram reaction directives into outbound channel data, allows reaction-only Telegram replies, calls native Telegram reaction delivery in bot and adapter paths, and adds focused payload and adapter tests.

## Best Possible Solution

Keep the parser and Telegram adapter direction, but make reaction intent part of the shared directive normalization contract used by both final and block replies, wire native Telegram reactions through typed reaction options, preserve current markdown image extraction behavior, and cover reaction-only plus text/media-with-reaction flows in payload, reply-delivery, bot delivery, and adapter tests.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: a Telegram reply containing [[react_to_current:🔥]] is not parsed into reaction metadata, does not produce channelData.telegram.reaction, and reaction-only replies hit the no-text/no-media rejection path.

## Solution Assessment

No, not as submitted. The direction is narrow and appropriate, but the branch needs a rebase plus fixes for reaction metadata preservation through the default assistant reply path and type-safe reaction scope plumbing.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Preserve reaction metadata before reply filtering:** `src/infra/outbound/payloads.ts:181`
  - body: The new channelData merge only happens in createOutboundPayloadPlanEntry. The assistant/block reply pipeline also calls normalizeReplyPayloadDirectives and then filters non-renderable payloads before Telegram delivery; that helper would strip [[react_to_current:...]] but not carry parsed.reactionEmoji into channelData, so reaction-only assistant replies can still be dropped before deliverReplies sees them. Thread the same reaction metadata through that normalizer and cover the final/block reply path.
  - confidence: 0.86

TypeScript will reject the PR until the field is either added through the reaction option contract and caller plumbing or omitted from these calls.
  - confidence: 0.91
