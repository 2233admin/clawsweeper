## Summary

Keep this PR open. Current main still has the reported cron run-log OOM path: both single-job and all-job run-history readers load JSONL files with unbounded fs.readFile before parsing and pagination. The PR is a plausible implementation candidate for #57017, but it is still unmerged, currently unmergeable/stale against main, and its head needs more review before landing.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, preserve the newer run-log query behavior, replace the read-side prune with a robust bounded tail strategy that satisfies keepLines without fixed per-line assumptions and only drops a partial first line when proven, keep the prune serialized and best-effort with runtime pruneOptions, and land it with the focused regression tests. If the branch cannot be refreshed, open a clean replacement PR for #57017 and close this PR only after that successor exists.
