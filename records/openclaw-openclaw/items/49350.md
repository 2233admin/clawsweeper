## Summary

Keep this issue open. Current main still resolves the Google Chat typing display name from account config name, then `agent.name`, then `OpenClaw`, without checking `agents.list[].identity.name`; it also still forwards reply/action/attached-result text to Google Chat without a Google Chat-owned Markdown strip/convert step. The linked typing PR remains open and unmerged, while the Markdown PR was closed unmerged.

## Best Possible Solution

Keep this issue open. Review and either land or close #49361 for the typing fallback, then add a Google Chat-owned Markdown normalization step before every `sendGoogleChatMessage`/`updateGoogleChatMessage` text path used by replies, message actions, and attached-result delivery, with focused regression tests.
