## Summary

This issue should stay open. Current main still implements `/subagents list` and the `subagents` tool as direct-child listing only: depth-2 workers are surfaced as pending descendant counts or `childSessions` metadata on their parent, not as expanded list rows with labels/status/runtime/model for the depth-0 caller. The request is narrow, current source already has descendant traversal helpers, and the related local report #67943 is about stale lineage/depth-limit state rather than recursive list output.

## Best Possible Solution

Keep this issue open and implement a bounded, backward-compatible recursive subagent listing: default `depth: 1`, accept `depth` and `recursive` on the `subagents` tool, parse `/subagents list --depth N` and `/subagents list --recursive`, return flattened rows with `depth` and `parentSessionKey`, and update docs/tests/changelog. The related #67943 should remain separate because it tracks stale lineage/depth-limit behavior, not recursive list visibility.
