## Summary

Keep open. Current main still treats cron as Gateway-owned stored jobs with optional agent binding, and there is no typed or strict schema contract for `agents.list[].cron` or scheduler startup path that compiles agent-declared jobs.

## What This Changes

The issue requests a declarative `cron` array inside each `agents.list[]` entry so agents can own scheduled tasks that the Gateway loads and merges with stored cron jobs.

## Best Possible Solution

If maintainers accept the feature, add an additive core config and scheduler contract: typed per-agent cron entries, strict schema/generated schema/help updates, deterministic compilation into scheduler jobs on startup and reload, documented ID namespacing and stored-job precedence, and tests for validation, merge behavior, duplicates, and inherited agent settings.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-main check: `agents.list[].cron` is not defined by the strict agent config schema and is not loaded by the Gateway scheduler.

## Solution Assessment

Unclear pending maintainer product review. The requested direction is coherent, but the maintainable solution belongs in core config and scheduler semantics rather than a narrow repair because precedence, reload, and autonomous execution safety are central contracts.
