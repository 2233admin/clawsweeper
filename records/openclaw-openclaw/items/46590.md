## Summary

Keep open. Current main still supports Gateway-owned cron jobs in `cron/jobs.json` with optional `job.agentId` binding, but it does not expose or load an `agents.list[].cron` declarative config field. The stale label alone is not enough to close a clear, still-missing feature request.

## Best Possible Solution

Keep this issue open for maintainer product/design review. If accepted, the best implementation is an additive core config contract for `agents.list[].cron` that validates declarative job specs, compiles them into scheduler jobs at Gateway startup/reload, preserves `jobs.json` as the mutable CLI/Control UI store with documented precedence, prevents duplicate runs, and updates config docs plus cron/config tests.
