## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, and recent-session filtering exist, but duration-style older/newer filters, size-threshold filtering, and byte-size row metadata are still absent from the CLI, Gateway protocol, Gateway implementation, and agent tool surface.

## What This Changes

Request to extend `openclaw sessions` with automation-friendly filtering by agent, age, size, and structured JSON output.

## Best Possible Solution

Define the size metric first, then add additive session-list query parameters and row metadata across the CLI, Gateway `sessions.list` protocol, Gateway filtering, and `sessions_list` tool, with docs and focused tests for duration parsing, size parsing/filtering, JSON output, protocol validation, and tool forwarding.
