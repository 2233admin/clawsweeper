## Summary

Keep this issue open. Current main still flattens Signal edit envelopes into the normal inbound path, does not type or use editMessage.targetSentTimestamp, and still dispatches edited content like a fresh message. The closed #9701 PR is useful prior art but was not merged.

## Best Possible Solution

Implement this in the bundled Signal plugin: type editMessage.targetSentTimestamp, route edit envelopes through an explicit edit-aware path, mark edited content or structured metadata, use the target timestamp for dedupe/transcript replacement, suppress non-mentioned group edit replies, and add focused Signal tests. #9701 is useful prior art but needs retargeting and review rather than being treated as shipped.
