## Summary

Keep this issue open. Current main still does not include a built-in Accessible Control UI theme preset: the theme type/validator/resolver, first-paint bootstrap, quick and advanced Appearance pickers, docs, and CSS theme blocks only cover Claw, Knot, Dash, and a browser-local tweakcn custom slot. The related tweakcn custom import and broader #28300 theme-studio request are complementary, not replacements for a shipped high-contrast, color-vision-deficiency-safe preset.

## Best Possible Solution

Implement a focused built-in Accessible theme family for the Control UI: add the theme name and resolved dark token to the theme contract, first-paint bootstrap, quick and advanced Appearance pickers, CSS variable block, storage/theme/app-settings tests, public Control UI docs, and a user-facing changelog entry. Keep #28300 as the broader theme-studio discussion and treat tweakcn import as complementary.
