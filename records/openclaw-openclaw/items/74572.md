## Summary

Keep open. Current main still matches the reported Active Memory / Memory Core composition bug, and open PR #74592 already references this issue with closing syntax, so this issue should stay open until that PR is reviewed and merged or rejected.

## What This Changes

The issue reports that Active Memory lightweight embedded recall runs allowlist Memory Core tools before those plugin tools are materialized, causing an empty callable tool set and a persistent stream-error placeholder in transcripts.

## Best Possible Solution

Review and land or close #74592. The desired end state is a generic, extension-agnostic fix where runtime toolsAllow can materialize explicitly allowlisted plugin tools before final filtering, and unavailable memory tools do not leave a persistent user-visible placeholder in saved transcripts.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is to enable active-memory and memory-core, trigger the Active Memory pre-summarizer, and observe that current main builds tools without using runtime toolsAllow to materialize plugin tools before filtering to the memory allowlist.

## Solution Assessment

Yes for the triage route. Keeping this issue open behind #74592 is the narrow maintainable path; the implementation should stay generic to plugin-tool materialization rather than hardcoding Memory Core behavior in core.
