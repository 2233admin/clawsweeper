## Summary

Keep open. Current main still has the reported Active Memory embedded-run shape: it requests only memory tools while using lightweight bootstrap, Memory Core exposes those names through plugin tool registration, and the embedded runner still fails closed when the resolved callable tool list is empty. The related closed reports cover different Active Memory timeout and MCP over-inclusion failures, not this memory-plugin visibility failure.

## What This Changes

The issue reports that Active Memory's lightweight embedded sub-agent bootstrap cannot see Memory Core's `memory_search` and `memory_get` tools, causing an empty allowlist failure and a persisted placeholder assistant turn.

## Best Possible Solution

Add a focused regression test that enables `active-memory` and `memory-core` together and proves the Active Memory embedded run sees `memory_search`/`memory_get` under lightweight bootstrap. The fix should keep core extension-agnostic, preserve the narrow memory-only allowlist, and make Active Memory degrade to a logged unavailable/empty status instead of writing a user-visible stream-error placeholder when the memory tool set cannot be resolved.
