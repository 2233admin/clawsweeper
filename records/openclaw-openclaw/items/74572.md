## Summary

Keep open. Current main still has the reported Active Memory / Memory Core composition path, and open PR #74592 already references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Active Memory's lightweight embedded recall sub-agent should compose with Memory Core so explicitly allowlisted memory plugin tools remain callable and failures do not persist a placeholder assistant turn.

## Best Possible Solution

A generic embedded-run tool materialization fix should make explicitly allowlisted plugin-provided memory tools available before final allowlist filtering and prevent optional recall setup failures from persisting a user-visible stream-error fallback in transcripts.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is to enable Active Memory and Memory Core, trigger the Active Memory pre-summarizer, and observe that current main builds embedded tools before applying `toolsAllow` while the runtime allowlist is not part of plugin tool materialization.

## Solution Assessment

Yes for the triage route. The issue should stay open behind #74592, and the implementation should remain generic to plugin-tool materialization and embedded gateway subagent binding rather than hardcoding Memory Core behavior.
