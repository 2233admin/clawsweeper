Keep open. Current main still performs the documented default 4:00 AM daily session rollover and does not include the continuity-hint behavior proposed in the related open PR, while the WeChat-specific part of the report lacks enough logs/config/session-key evidence to distinguish expected core rollover from plugin identity churn.
