## Summary

Keep open. Current main still documents and implements a default 4:00 AM daily session rollover that can explain the overnight amnesia, and it does not yet include the scheduled-reset continuity hint proposed in related PR #48235; the WeChat-specific angle also lacks the plugin/session evidence needed to rule out channel session-key churn.

## What This Changes

The issue reports that an overnight WeChat conversation on OpenClaw 2026.04.21 lost prior context without an explicit `/new`, `/reset`, or `/compact` command.

## Best Possible Solution

Keep the documented daily reset policy unless maintainers intentionally change it, but resolve the user-visible amnesia either by landing a reviewed scheduled-reset continuity hint or by proving and fixing a WeChat/session-key stability bug with focused plugin and session-store evidence.

## Reproduction Assessment

Unclear. ## Solution Assessment

Unclear. The narrowest likely core solution is the one-shot scheduled-reset continuity hint proposed in #48235, but a WeChat-specific fix should wait until maintainers can distinguish documented core rollover from channel session identity churn.
