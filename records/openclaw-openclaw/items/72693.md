## Summary

Keep open. Current main has strong partial coverage for the service split-brain and stale pinned-port pieces, but I cannot close the full WSL2 gateway crash-loop report as implemented because the remaining no-listener/EADDRINUSE behavior has not been reproduced or ruled out on current main, and the linked fix PR was closed unmerged while explicitly leaving WSL2 ghost-listener handling out of scope.

## What This Changes

The issue reports that OpenClaw 2026.4.24 on WSL2 can enter an unreachable gateway crash loop involving duplicate systemd supervisors, stale service port pinning, stricter config validation, and a delayed EADDRINUSE failure.

## Best Possible Solution

Keep this issue open until a maintainer verifies the latest build on WSL2 with the reporter's service scenarios, then either close it with current-version live proof or split any remaining ghost-listener failure into a narrow gateway startup/supervised-lock fix with regression coverage.

## Reproduction Assessment

Unclear. The original report gives detailed 2026.4.24 WSL2 logs and recovery steps, and current tests cover mocked service split-brain and stale-port cases, but there is no high-confidence current-main WSL2 reproduction or non-reproduction for the delayed no-listener/EADDRINUSE loop.

## Solution Assessment
