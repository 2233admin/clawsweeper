## Summary

Current main still does not implement externally managed CDP/noVNC endpoints for sandbox browsers; the sandbox browser path remains Docker-lifecycle based, while global remote CDP profiles are a separate browser-control feature.

## Best Possible Solution

Keep this issue open. The best implementation is a core sandbox browser seam: add validated agents.defaults.sandbox.browser external CDP/noVNC endpoint settings or a backend-owned browser capability, skip Docker lifecycle only when that mode is explicitly configured, update config types/schema/help/docs, and add regression tests proving a non-Docker sandbox backend can use browser automation without Docker while preserving current Docker defaults.

## Risks / Open Questions

- External CDP/noVNC support is security-sensitive: endpoint validation, SSRF/private-network policy, authentication, token redaction, and noVNC exposure need explicit review.
- Global remote CDP profile support can be mistaken for this feature, but it does not cover sandbox browser noVNC injection, sandbox lifecycle, or non-Docker backend behavior.
- A minimal external endpoint path needs to avoid stale Docker registry/cleanup assumptions when Docker lifecycle is intentionally skipped.
