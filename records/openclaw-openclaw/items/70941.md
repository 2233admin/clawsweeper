## Summary

Current main still appears to have the reported failure class. `openclaw models set` is a normal model-default mutation with no CLI escape hatch or command-specific write policy, but it persists through the shared config replacement path and the config writer still rejects more-than-half raw size drops with `CONFIG_WRITE_REJECTED` unless `allowDestructiveWrite` is explicitly set. The prior ClawSweeper comment is consistent with current source at edbab0e2db7f450f4d63f9700289ae1ba838a419.

## Best Possible Solution

Keep this issue open. Add a regression test that drives a real `openclaw models set` default-model write against a valid config that legitimately trips the current raw size-drop threshold, then adjust the models/config mutation path so intended `agents.defaults.model.primary` changes can persist while the generic clobber guard still rejects accidental destructive writes.
