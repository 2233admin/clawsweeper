## Summary

Keep open. Current main still routes `openclaw models set` through the generic config replacement path without a command-specific write policy, while the config writer still blocks raw size drops with `CONFIG_WRITE_REJECTED` unless destructive writes are explicitly allowed. The existing models-set tests cover model normalization and object-form rewrites, but they mock `updateConfig` and do not exercise the real writer against a legitimate shrinking config.

## What This Changes

The issue reports that `openclaw models set` can reject a legitimate default-model config update when the serialized config shrinks enough to trip the size-drop clobber guard.

## Best Possible Solution

Add a targeted regression for a real `openclaw models set` config write that legitimately shrinks below the current byte threshold, then implement a narrow semantic policy for intended default-model updates while preserving the size-drop rejection for unrelated clobbers.
