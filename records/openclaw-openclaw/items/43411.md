## Summary

Keep this PR open for maintainer review: current main still does not implement the PR's intended additive `logging.redactPatterns` contract, and the proposed patch fixes only selected paths while leaving other config-derived redaction sinks with replacement semantics.

## What This Changes

The PR adds default-plus-custom redaction pattern merging for no-options `redactSensitiveText()` and `redactToolDetail()`, plus tests for additive behavior and explicit caller-supplied pattern replacement.

## Best Possible Solution

Define a single config-derived redaction contract in the shared resolver/helper used by log output, log tailing, tool-detail display, persisted transcript text, and safety-boundary payloads; keep explicit caller-supplied replacement semantics only where intentional and documented, then add targeted tests for each sink.
