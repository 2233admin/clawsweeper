## Summary

Keep this PR open. Current main does not implement the proposed Telegram streaming-preview recovery: `draft-stream.ts` still sends rendered HTML directly, has no HTML parse-error classifier or plain-text retry, and the outer preview failure path still marks the stream stopped for any thrown error. The PR is focused on the right remaining gap, with discussion showing several race and state concerns were actively addressed, so the best maintainer path is review/refresh and land or request final changes rather than close as obsolete.

## Best Possible Solution

Keep the PR open. The best path is to refresh/review it against current main, preserve the core fix in `extensions/telegram/src/draft-stream.ts` for HTML parse-error plain-text fallback across edit, initial send, draft transport, and the outer catch, verify the generation/race fixes from the review thread, and land it with the focused draft-stream regression tests if maintainer review and gates pass.
