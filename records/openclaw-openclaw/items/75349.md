## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, does not recognize Copilot's reported `model_not_supported` 400 as a model-scoped unsupported-model failure, and only preserves sibling-model auth cooldown bypasses for `rate_limit`, so the reported behavior remains actionable.

## What This Changes

The issue requests removing stale GitHub Copilot `claude-opus-4.6` defaults and making unsupported-model rejections affect only that model instead of cooling down the whole Copilot auth profile.

## Best Possible Solution

Implement a two-part fix: keep the bundled GitHub Copilot catalog conservative or runtime-backed, and classify unsupported-model responses as model-scoped unavailability when the failing model id is known while preserving profile-wide cooldowns for genuine provider/account failures.
