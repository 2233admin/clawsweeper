## Summary

Keep open. Current main still does not implement this PR's intended systemd scope selection for the managed Linux gateway restart/status path, and the PR itself is not merge-ready because it mixes the systemd change with an unrelated Docker mount-validation change and has review feedback on selecting enabled-only units before active units.

## Best Possible Solution

Keep this PR open for maintainer action, but do not merge it as-is. The best fix is a focused systemd lifecycle patch in src/daemon/systemd.ts with tests in src/daemon/systemd.test.ts, optionally updating src/infra/restart.ts only if that helper still needs the same resolver. It should prefer the active/running unit over enabled-only state, report the selected scope, avoid broad Docker validation changes, and use #44772 as prior art rather than treating it as shipped.
