## Summary

Keep open. Current main still intentionally removes the Chrome Extension Relay, rejects `driver: "extension"`, and migrates legacy relay config to Chrome MCP `existing-session`; that does not restore the requested existing-tab relay, auto-reattach, video-control, or chat-automation behavior. The related browser-backend plugin interface is relevant but remains a separate open design item, and restoration needs maintainer product and security direction.

## What This Changes

Request to restore the removed Chrome Extension Relay as an optional browser driver with bundled extension assets, gateway relay support, and backward-compatible `driver: "extension"` config handling.

## Best Possible Solution

Maintainers should decide the browser strategy explicitly: either keep the legacy relay retired and document the missing migration capabilities clearly, or design a vetted browser-backend/plugin path and maintain an optional Chrome Extension Relay with security review, tests, docs, and migration handling.
