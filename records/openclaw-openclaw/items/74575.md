## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin setup prompts, and Crestodian typed setup/repair, but it still lacks the requested agent-guided configuration assistant that interviews the user, probes host capability such as CPU/GPU/RAM, recommends best-practice settings, and refreshes recommendations as new features ship.

## What This Changes

The issue requests a built-in wizard or agent-guided automation that asks users goal-oriented questions, runs basic host checks, and tunes OpenClaw configuration over time.

## Best Possible Solution

Define a scoped next-generation setup UX on top of the existing onboarding and Crestodian boundaries: collect user goals, expose explainable environment/resource discovery, route plugin-specific recommendations through plugin-owned setup metadata, and keep all writes typed, approved, audited, reversible, and documented.
