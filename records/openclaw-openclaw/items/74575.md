## Summary

Keep open. Current main has guided onboarding, fixed configure sections, doctor/health checks, and Crestodian typed setup/repair, but it still does not implement the requested goal-driven assistant that interviews the user, probes host capabilities, recommends best-practice settings, and revisits recommendations after new features ship. The remaining work is broad, product-sensitive, and config-write/security-sensitive, so it should not be closed as conservative cleanup or queued as a narrow automated fix.

## What This Changes

The issue requests a built-in, agent-guided configuration wizard that interviews the user, probes host capabilities, recommends best-practice OpenClaw settings, and refreshes those recommendations as features ship.

## Best Possible Solution

Define a scoped next-generation setup UX that keeps config writes typed, approved, audited, reversible, and explainable; route plugin-specific recommendations through plugin-owned metadata; and document the allowed host/resource probes and recommendation authority before implementation.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is docs/source inspection showing existing onboarding, configure, doctor, and Crestodian surfaces do not provide the requested goal-driven host-capability recommender.

## Solution Assessment

Unclear. A built-in assistant may be the right product direction, but the current request is not yet the narrowest maintainable solution until maintainers define scope, security semantics, plugin recommendation metadata, and rollback behavior.
