## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, doctor repair hints, and Crestodian typed setup/repair, but it still lacks the requested goal-driven configuration assistant that interviews the user, probes host capabilities, recommends best-practice settings, and refreshes recommendations as features ship. The remaining work is broad, product-sensitive, and config-write/security-sensitive, so it needs maintainer scoping rather than conservative cleanup closure.

## What This Changes

Add a built-in, agent-guided configuration automation that asks about user goals, probes host capabilities such as CPU/GPU/RAM, recommends best-practice settings, and updates recommendations as OpenClaw features change.

## Best Possible Solution

Define a scoped next-generation setup UX on top of existing onboarding and Crestodian: collect user goals, expose explainable host/resource discovery, route plugin-specific recommendations through plugin-owned setup metadata, and keep all config writes typed, approved, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from docs/source/test inspection: existing flows are fixed-step or typed repair flows, while the requested goal interview and hardware-based recommender are absent.

## Solution Assessment

Unclear. A guided assistant may be the right UX, but the current proposal needs maintainer design first; the safer direction is a scoped setup/recommendation contract with typed approvals and plugin-owned recommendation metadata rather than an ad hoc agent config writer.
