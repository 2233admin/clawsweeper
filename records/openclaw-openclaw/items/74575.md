## Summary

Keep open. Current main still has explicit onboarding, section-based reconfiguration, plugin field prompts, and bounded Crestodian setup/repair, but not the requested goal-driven assistant that interviews the user, probes host resources, recommends best-practice settings, and revisits recommendations as features ship. The existing ClawSweeper comment and later maintainer mention/subscription both support leaving this for product scoping rather than cleanup-closing it.

## What This Changes

Add built-in guided configuration automation that asks about user goals, probes host capability such as CPU/memory/GPU, recommends OpenClaw settings, and refreshes guidance when new features ship.

## Best Possible Solution

Define a scoped next-generation setup UX that builds on existing onboarding and Crestodian: collect user goals, make host/resource discovery explicit and consented, let plugins contribute typed recommendation metadata where appropriate, and keep every persistent config write approved, audited, reversible, schema-backed, and documented.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap check is high confidence: onboarding, configure, plugin prompts, and Crestodian do not provide a goal-driven host-capability recommender.

## Solution Assessment

No current implementation is sufficient, and a blind automated config writer would not be the right solution. The maintainable path is maintainer-scoped design around typed setup/config surfaces, consented probes, plugin recommendation metadata, and audited writes.
