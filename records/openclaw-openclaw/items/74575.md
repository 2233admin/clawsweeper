## Summary

Keep open. Current main has guided onboarding, section-based reconfiguration, plugin field prompts, and Crestodian typed setup/repair, but it still lacks the requested goal-driven assistant that interviews the user, probes host capability, recommends best-practice settings, and revisits recommendations after new features ship.

## What This Changes

Requests a built-in configuration assistant that asks goal-oriented questions, inspects host capability such as CPU/GPU/RAM, recommends OpenClaw settings, and refreshes those recommendations as features change.

## Best Possible Solution

Define a scoped next-generation setup UX on top of the existing onboarding and Crestodian boundaries: collect user goals, expose explainable host/resource discovery, let plugins contribute recommendation metadata through owned APIs, and keep all persistent config writes typed, approved, audited, reversible, and documented.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection against current main, which shows existing setup helpers but not the requested goal-driven recommender.

## Solution Assessment

Unclear. The requested direction is plausible, but the best maintainable path is a scoped design over the existing typed onboarding/Crestodian/plugin seams rather than an immediate broad auto-config implementation.
