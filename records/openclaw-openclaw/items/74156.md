## Summary

Keep open. This PR is maintainer-authored and has the protected `maintainer` label, so repository policy requires explicit maintainer handling. Current `main` still lacks the PR's scoped external CLI auth overlay path: the overlay loop still reads all built-in external CLI providers without provider/profile eligibility, store load options are not forwarded into runtime overlays, and `models.authStatus` still loads the overlaid store before deriving configured providers. The PR is a live implementation candidate for #73908 and overlaps earlier open scoped-overlay work in #73952 and #74033.

## Best Possible Solution

Keep this PR open for maintainer and Clownfish review. The best path is to land this PR or an equivalent scoped-overlay fix for #73908 that passes provider/profile/config eligibility and `allowKeychainPrompt: false` through external CLI auth overlay/status paths, while preserving intentional external CLI credential reuse for configured providers. Reconcile the overlap with #73952 and #74033 before merge.
