## Summary

Current main still has the focused channel-awareness gap described in the discussion. Ordinary agent runs include channel metadata in the runtime prompt, but slash-command prompt reconstruction for `/context` and `/export-session` still builds runtime params with `host/os/arch` set to `unknown` and omits channel, capabilities, shell, and channel action metadata. The linked fix PR #21271 is closed unmerged, so this should remain open as a narrow fix candidate.

## Best Possible Solution

Keep this issue open and implement the missing runtime metadata only in `resolveCommandsSystemPromptBundle()`. Reuse or extract the embedded-runner pattern so `/context` and `/export-session` prompt reconstruction includes normalized channel, account-aware capabilities, shell, machine/os details, and supported channel actions where the prompt can use them. Treat selective cross-channel reply routing as a separate product/API decision.
