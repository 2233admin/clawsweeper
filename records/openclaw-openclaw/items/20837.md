## Summary

Keep this issue open. Current main already exposes channel metadata in the normal inbound/agent-run path, but slash-command prompt reconstruction still hardcodes unknown runtime host fields and omits channel/capabilities/shell/channel-action prompt metadata; the linked fix PR #21271 is closed unmerged.

## What This Changes

The issue requests that agents receive the incoming communication channel as context and optionally support channel-selective replies.

## Best Possible Solution

Keep this issue open and implement the missing runtime metadata in `resolveCommandsSystemPromptBundle()` only. Reuse or extract the embedded-runner runtime prompt pattern so `/context` and `/export-session` prompt reconstruction includes normalized channel, account-aware capabilities, shell, host/OS/arch, and supported channel action/message guidance where available. Leave selective reply routing for a separate product/API follow-up.

## Repair Work Prompt

Implement the missing command prompt runtime metadata for this issue. Current main already exposes channel metadata in normal inbound/agent-run prompts, but `src/auto-reply/reply/commands-system-prompt.ts` still reconstructs `/context` and `/export-session` prompts with `host/os/arch` as `"unknown"` and without channel, capabilities, shell, reaction guidance, or message-tool/channel-action prompt metadata. Update `resolveCommandsSystemPromptBundle()` only, or extract a small shared helper from the embedded runner pattern, so it supplies real host/OS/arch/shell, a normalized source channel from the command/message context, account-aware capabilities using `params.ctx.AccountId` where relevant, and the same supported channel action/message guidance available to normal runs. Do not implement prompt-driven selective reply routing, do not change deterministic delivery semantics, and do not add core owner-specific channel defaults. Add focused tests for the command prompt bundle and existing `/context`/`/export-session` reconstruction paths. Add a single-line user-facing changelog fix entry if maintainers classify this as user-visible, crediting @SvenMeyer and @evansantos, and do not credit bots.
