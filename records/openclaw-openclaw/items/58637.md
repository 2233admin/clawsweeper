## Summary

Keep open. Current main still does not implement the PR's central UI change, and the backend fallback concern remains real: the sessions table has no Model column even though session rows carry model/modelProvider, and buildGatewaySessionRow only reads transcript usage under token/context/cost fallback conditions while transcript model identity is only considered when runtime model metadata is already present.

## Best Possible Solution

Keep the PR open and ask for a focused update: add the sessions-table Model column using the existing GatewaySessionRow model/modelProvider data, and change buildGatewaySessionRow so transcript model recovery runs when both runtime model metadata and explicit modelOverride are absent, independent of whether token/cost/context usage also needs transcript fallback. Add regression coverage for a session with persisted usage, no entry.model/modelOverride, and transcript model metadata.
