## Summary

Keep open. Current main still selects env proxies from configured environment variables and NO_PROXY rules only; it does not probe proxy reachability or retry direct when a configured proxy refuses connections. Newer managed-proxy work reinforces that proxy routing can be a fail-closed security control, so the requested fallback needs maintainer policy judgment rather than cleanup closure.

## What This Changes

The issue requests proxy reachability detection so OpenClaw can avoid or fall back from a configured proxy when its local port is unavailable.

## Best Possible Solution

Define an explicit proxy-unavailable policy first, then implement it in the shared env-proxy layer so Google auth, guarded fetch, memory host, and channel transports share one behavior. The likely safe endpoint is a cached, timeout-bounded fallback only for approved local/operator convenience proxy cases, while preserving NO_PROXY behavior and fail-closed managed proxy semantics, with tests and docs covering closed loopback proxy, malformed proxy URLs, mandatory proxy mode, and no silent security bypass.
