## Summary

Keep this PR open. Current main still does not implement configurable image-tool JPEG quality or max-side controls, so the PR is not obsolete. It should not merge as-is because the PR head still has unresolved constraint fallback bugs and bundles unrelated docs/gateway/provider-link changes with the image compression feature.

## Best Possible Solution

Keep this PR open as an implementation candidate, or supersede it with a focused rebased PR. The best patch should add a typed and documented `agents.defaults` image-tool compression contract, update generated config metadata/docs, thread the options through `createImageTool`, `loadWebMedia`, and `optimizeImageToJpeg` on current `src/media/web-media.ts`, honor arbitrary out-of-ladder values such as quality 95/85 and maxSide 600, preserve SSRF and local media access guards, add regression tests, and split unrelated docs/gateway/provider-link changes into separate work.
