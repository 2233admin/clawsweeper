## Summary

Keep #58284 open. Current main still implements only cache-TTL-triggered context pruning: the extension skips pruning while the cache TTL window is still warm, the public config/schema only accepts `off` and `cache-ttl`, and docs describe pruning as waiting for TTL expiry. The related #43238 PR appears to be a plausible implementation path, but it has not shipped on main, so this issue still tracks a real missing core behavior.

## Best Possible Solution

Keep the issue open and treat it as a real core context-pruning feature request. The likely fix belongs in the built-in context-pruning path: add a rolling or turn-age option that can run every turn independent of TTL, update config types/Zod/generated schema/docs, and add tests proving active sessions with continuous cache touches still prune old tool results while preserving the protected recent-turn tail.
