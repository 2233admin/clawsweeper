## Summary

Keep open. Current main still lacks the requested default-off force-prepend `<think>` option: it is absent from typed config, strict Zod schema, runner params, subscribe params, and the streaming/non-streaming message parsing path. The related implementation PR #14323 is closed unmerged and remains design context rather than a landed fix.

## What This Changes

Add a user-configurable default-off option to force-prepend an opening `<think>` tag for local reasoning models that omit reasoning delimiters, with streaming/non-streaming support and global or per-model configuration.

## Best Possible Solution

Keep this issue open and implement a focused default-off `forcePrependThinkTag`-style config path with clear global and per-model precedence, resolved before embedded runs, threaded through runner params into `subscribeEmbeddedPiSession`, and applied exactly once before reasoning parsing with a native-tag guard.

## Repair Work Prompt

Implement the default-off force-prepend `<think>` option requested by this issue. Add a typed `forcePrependThinkTag`-style boolean to global agent defaults and per-model `agents.defaults.models[provider/model]` entries, with clear precedence where the per-model value overrides the global value and the default remains false. Resolve the effective value before the embedded run, thread it through auto-reply runner params, `runEmbeddedAttempt`, `SubscribeEmbeddedPiSessionParams`, and into `subscribeEmbeddedPiSession`. In the embedded subscriber/message handler, inject an opening `<think>` exactly once at the start of each assistant message before reasoning parsing for both streaming deltas and non-streaming assistant text, and skip injection when the model already starts with a native opening think tag so `<think><think>` cannot leak. Do not hard-code GLM, MiniMax, step, or any provider/model ids; this should be config-driven. Update config schema/help/generated docs as needed, add docs for the new option, and add a user-facing changelog entry crediting @LisaMacintosh if maintainers keep that attribution. Use #14323 only as design context; do not assume that branch is current or complete.
