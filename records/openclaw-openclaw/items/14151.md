## Summary

Keep open. Current main still lacks the requested default-off force-prepend `<think>` option in the typed config/schema, runner params, subscribe params, and message parsing path; the related implementation PR is closed unmerged, so there is no landed fix to close against.

## What This Changes

The issue requests a default-off global and/or per-model option to prepend an opening `<think>` tag for local reasoning models before reasoning parsing in streaming and non-streaming replies.

## Best Possible Solution

Add a default-off, documented config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, pass it through runner and subscriber params, and apply it once before existing reasoning extraction/stripping with a guard for native opening tags.
