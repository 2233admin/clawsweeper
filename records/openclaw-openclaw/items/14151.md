## Summary

Keep open. Current main still strips/parses model-emitted reasoning tags and handles malformed local-model tags, but it does not provide the requested default-off option to force-prepend an opening <think> tag before parsing for streaming or non-streaming assistant output. The related implementation PR #14323 was closed unmerged and should be treated as context, not as a landed fix.

## Best Possible Solution

Keep this issue open and implement a default-off, typed forcePrependThinkTag-style option with global and per-model resolution, config schema/docs alignment, runner-to-subscribe threading, exactly-once injection before reasoning parsing for streaming and non-streaming paths, and guards for existing tags/native thinking/final enforcement. Use #14323 as useful context, but do not treat it as landed.
