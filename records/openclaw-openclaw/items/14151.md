## Summary

Keep open. Current main still only prompts for, parses, and strips existing reasoning tags; it has no config/schema, runner, or subscriber flag to synthesize an opening `<think>` tag, and the linked implementation PR is closed unmerged.

## What This Changes

The issue asks for a default-off, preferably global and per-model, option to synthesize `<think>` boundaries for local reasoning models before streaming and non-streaming response parsing.

## Best Possible Solution

Add a default-off, documented config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, thread it through runner and subscriber params, and apply it once before existing reasoning parsing with native-tag detection.
