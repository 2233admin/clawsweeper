## Summary

Keep this PR open. Current main still has the ambiguous human `openclaw mcp` CLI wording and MCP docs still lack a prominent note that the `list/show/set/unset` registry excludes mcporter's separate `config/mcporter.json` registry. This PR is a focused implementation candidate for the linked open issue #65209; there is also an older open duplicate attempt (#70495), so maintainers should choose one patch rather than close this as already solved.

## Best Possible Solution

Review and land this PR or the older equivalent #70495, then close the other duplicate and close #65209 after merge. The accepted patch should label human `openclaw mcp list/show/set/unset` output/help as OpenClaw-managed `mcp.servers`, document that mcporter `config/mcporter.json` is handled by `mcporter list`, add regression coverage for the human list wording, and keep JSON output unchanged.
