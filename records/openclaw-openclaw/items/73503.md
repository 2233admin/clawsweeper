## Summary

Keep this PR open. Current main still has the ambiguous `openclaw mcp` human CLI wording and no prominent docs note that `openclaw mcp list/show/set/unset` exclude the separate mcporter `config/mcporter.json` registry. The PR is a focused docs/CLI/test candidate for the linked open issue #65209, and its file set does not introduce security-sensitive workflow, dependency, publishing, or install changes.

## Best Possible Solution

Review and land this PR or an equivalent focused patch. The maintained change should label the human `openclaw mcp list/show/set/unset` surfaces as OpenClaw-managed `mcp.servers`, document that mcporter `config/mcporter.json` is a separate registry handled by `mcporter list`, keep JSON output unchanged, and close #65209 only after the implementation merges.
