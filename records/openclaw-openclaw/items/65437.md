## Summary

Keep open. Current main still drops Slack room/group `/elevated` messages at the Slack control-command gate before `tools.elevated.allowFrom.slack` or the elevated fallback seam can run. The related Discord and command-tier issues are useful context, but they do not supersede this Slack-specific bug.

## Best Possible Solution

Keep this open and fix the Slack room/group `/elevated` path narrowly. Either allow senders in `tools.elevated.allowFrom.slack` to reach `resolveElevatedPermissions`, or add/register an elevated-specific Slack authorizer/fallback without widening access to other control commands. Add a regression test for a Slack channel/group sender listed only in `tools.elevated.allowFrom.slack`, and update the public elevated/Slack docs to describe the resulting behavior.
