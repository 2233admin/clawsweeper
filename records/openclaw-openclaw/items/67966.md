## Summary

Keep open. Current main still does not route local-managed browser profiles through Playwright-backed navigation interception, so the requested strict-SSRF redirect-hop inspection path is absent for that mode. The issue is valid but security-sensitive because the fix changes browser SSRF enforcement and local-managed Playwright/CDP lifecycle behavior, so it should get maintainer/security review before implementation.

## What This Changes

The issue requests enabling Playwright-backed navigation interception for local-managed browser profiles so redirect hops can be inspected under strict browser SSRF policy.

## Best Possible Solution

Implement a targeted browser plugin change that uses the existing `createPageViaPlaywright` / `gotoPageWithNavigationGuard` path for local-managed profiles when strict SSRF policy requires redirect-hop inspection, while preserving Chrome MCP existing-session behavior and raw CDP fallbacks for safe cases such as `about:blank` or non-strict navigation. Add local-managed strict-mode regression tests for allowed IP-literal navigation and blocked private redirect hops, and update browser docs only if the operator-facing capability matrix changes.
