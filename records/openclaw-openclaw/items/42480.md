## Summary

Keep open. The contribution addresses a real Discord debounce mention-metadata problem and the follow-up commit handled the earlier roles/`@everyone` review note, but the branch is stale against current main: it modifies the removed legacy Discord handler while the shipped plugin handler still needs the fix.

## What This Changes

The PR accumulates mentioned users, mentioned roles, and `@everyone` flags across debounced Discord entries when building a synthetic message in the legacy Discord message handler.

## Best Possible Solution

Port the all-entry mention metadata merge into `extensions/discord/src/monitor/message-handler.ts` so debounced synthetic messages expose merged `mentions`, `mention_roles`, and `mention_everyone` through the current internal `Message` getters, then add focused regression coverage for an earlier debounced entry containing mention metadata when the final entry does not.
