## Summary

Keep open. Current main still rejects the reported mixed-selector calls, and the related discussion around #59324, #56203, #63308, and #64699 supports the underlying bug as real. This PR is the right general direction, but the current head should not merge until the denied sessionId response is hardened and the new test stops leaking mock behavior.

## What This Changes

The PR removes the mixed sessionKey/label rejection in `sessions_send`, documents sessionKey/sessionId precedence, adds focused sessions tests, and adds a changelog entry.

## Best Possible Solution

Land a narrow fix that keeps sessionKey/sessionId precedence for redundant selector inputs, but makes denied access fail closed without exposing the resolved canonical key, resets or one-shots the new gateway mock implementation, and then validates the focused sessions test plus the changed gate.

## Reproduction Assessment

Yes. Current main reproduces the reported bug by calling `sessions_send` with a valid `sessionKey`, any non-empty `label`, and a message; source inspection shows the tool returns the mutual-exclusion error before send resolution.

## Solution Assessment

No as submitted. Preferring `sessionKey` is the narrow maintainable fix for the user-facing bug, but this patch is not the best final form until denied sessionId sends stop echoing resolved keys and the test mock isolation is fixed.
