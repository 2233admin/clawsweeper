## Summary

Keep this PR open. The intended `sessionKey` precedence fix is useful and matches the related reports, but the current patch is not merge-ready because it expands the mixed-selector path into session-id resolution without addressing the denied-access echo of resolved canonical keys, and one new test can pollute later tests through a persistent mock implementation.

## What This Changes

The PR removes the `sessions_send` mixed `sessionKey`/`label` rejection, documents `sessionKey` and session-id precedence over redundant hints, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the narrow `sessionKey` precedence direction, but update the PR so denied visibility/A2A responses never reveal a newly resolved canonical key, isolate the gateway mock in the new tests, preserve label-only behavior, and keep the changelog/docs wording aligned with the final contract.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction: a `sessions_send` call with both `sessionKey` and `label` hits the guard in `src/agents/tools/sessions-send-tool.ts:109` and returns the mutual-exclusion error; #64699 provides a concrete Control UI/ClawX payload showing that shape.

## Solution Assessment

No as submitted. Preferring `sessionKey` is the narrow maintainable fix for the reported bug, but the final patch should avoid canonical key disclosure on denied session-id resolution and reset or one-shot the new gateway mock implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid echoing resolved keys after denied sends:** `src/agents/tools/sessions-send-tool.ts:106`
  - body: Removing this guard lets a sessionId-shaped `sessionKey` with redundant `label`/`agentId` reach session-id resolution. If visibility or A2A policy then denies the send, the existing denied branch returns the resolved `displayKey`, so echo the original selector or omit `sessionKey` on denied access.
  - confidence: 0.74
- **[P2] Reset the gateway mock between send-gating tests:** `src/agents/tools/sessions.test.ts:687-695`
  - body: The new session-id test installs a persistent `callGatewayMock.mockImplementation`, but this describe block only calls `mockClear()` before each test. Later tests can inherit the implementation and pass against polluted gateway behavior, so use one-shot implementations or reset the mock after the test.
  - confidence: 0.9
