## Summary

Current main does not already implement this PR's intended behavior. The chat item builder still only suppresses normalized `toolResult` history entries when the user-facing tool-calls toggle is off, while the text extractor has no `SECURITY NOTICE:` or `EXTERNAL_UNTRUSTED_CONTENT` stripping path. This remains a focused but stale UI bug-fix candidate rather than a conservative close.

## Best Possible Solution

Keep this PR open as a stale but plausible bug-fix candidate. The best path is to rebase it onto `ui/src/ui/chat/build-chat-items.ts`, decide the product boundary between hiding raw gateway plumbing and showing collapsible tool cards, filter tool-only history entries before applying the visible history limit, scope any external-content marker stripping to tool-result payloads, and add focused regression tests for the reported role and marker cases.
