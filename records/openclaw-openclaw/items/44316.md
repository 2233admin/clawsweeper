## Summary

Keep open. Current main still does not implement the intended behavior: raw tool-shaped history remains visible when the Control UI tool-calls toggle is enabled, and UI text extraction has no SECURITY NOTICE / EXTERNAL_UNTRUSTED_CONTENT stripping. The contribution is useful, but the branch targets an older chat builder location and has unresolved product/security scope questions, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR branch modifies web chat text extraction and the old chat view builder to hide tool-result/tool/function and tool-call-only assistant plumbing, while stripping SECURITY NOTICE and EXTERNAL_UNTRUSTED_CONTENT markers from extracted UI text.

## Best Possible Solution

Move the useful behavior into the current chat item builder and extraction helpers with focused tests: hide raw tool plumbing from history regardless of the live tool-card toggle, preserve mixed-content assistant turns and intentional live tool cards, and apply external-marker redaction only at a maintainer-approved scope.

## Risks / Open Questions

- The branch modifies the old `ui/src/ui/views/chat.ts` builder path; current main now owns item construction in `ui/src/ui/chat/build-chat-items.ts`, so the patch needs a rebase before it can be evaluated cleanly.

- Filtering tool plumbing after computing the history render limit can show fewer real chat turns in long tool-heavy sessions.
- A fix must preserve intentional live tool-call/tool-output cards controlled by `showToolCalls` while hiding raw transcript plumbing from normal chat bubbles.
