## Summary

Current main still does not implement this PR's intended behavior. The Control UI chat item builder only suppresses normalized toolResult history entries when the user-facing tool-calls toggle is off, tool messages remain a visible UI mode, and message extraction has no SECURITY NOTICE / EXTERNAL_UNTRUSTED_CONTENT stripping path. The PR remains useful, but it is stale against the current build-chat-items split and still needs maintainer judgment on the product boundary and redaction scope.

## Best Possible Solution

Keep this PR open for maintainer review or an updated contributor patch. The best path is to rebase the behavior onto ui/src/ui/chat/build-chat-items.ts and ui/src/ui/chat/message-extract.ts, decide whether tool plumbing should be hidden even when tool cards are enabled, scope marker stripping to confirmed tool-result/external-content payloads, and add focused UI chat tests for the reported role and marker cases.
