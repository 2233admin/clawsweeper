## Summary

This PR adds a new bundled optional auto-recall plugin, but current OpenClaw policy routes optional plugins to external ClawHub/npm distribution unless they need a missing core API or have a clear maintainer-owned product/security reason. Current main already has the core automatic pre-reply memory surface via the bundled Active Memory plugin, and the remaining proposed HTTP/script backend behavior can be implemented as an external native plugin with the existing `before_prompt_build` hook.

## Best Possible Solution

Keep the shipped Active Memory implementation as the core automatic recall path, and move this auto-recall backend-chain implementation to an external ClawHub/npm plugin with explicit compatibility metadata, setup docs, and operator-controlled security guidance. Open a focused core SDK issue only if publishing/running it externally exposes a concrete missing hook or API.
