## Summary

Keep this issue open. Current main still wires the ACP stdio stream directly into `AgentSideConnection`, while `@agentclientprotocol/sdk@0.20.0` validates `initialize.params.protocolVersion` as a uint16 integer before OpenClaw's translator sees the request. The reported VS Code/Cursor date-string protocol version therefore remains a plausible current bug, and open PR #56176 explicitly proposes the missing stream coercion with closing syntax, so the best maintainer path is to review/adapt that PR rather than close the issue early.

## Best Possible Solution

Keep the issue open and use PR #56176, or an equivalent current-main patch, as the implementation path: add a small ACP stream compatibility layer before `AgentSideConnection`, preserve numeric ACP protocolVersion requests unchanged, and add a focused regression test for VS Code/Cursor-style `protocolVersion: "2025-11-25"`. Close the issue only after that fix lands on main.
