## Summary

Keep this PR open. Current main and the latest release still keep Slack thread participation in a process-local dedupe cache, while Slack mention gating uses that cache to allow non-mention replies in bot-participated threads. The PR remains a useful implementation candidate, but it needs maintainer storage/security direction and cleanup because it overlaps with #58731 and the shared persistent-store direction in #70626.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate the cache to the approved shared persistent dedupe/keyed-store primitive with the needed async Slack call-site changes, or explicitly approve a hardened Slack-local store. Before merge, trim unrelated churn and keep focused tests for restart survival, clear semantics, TTL expiry, corrupt or oversized persisted files, entry bounds, and storage failure behavior.
