## Summary

Keep this PR open. Current main still keeps Slack thread participation in a process-local dedupe cache, so the restart-survival behavior this PR targets is not implemented. The PR is active and useful, but it needs maintainer direction on whether to land a hardened Slack-local store or migrate to the shared persistent store work in #70626, and it should shed unrelated formatting churn before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Preserve the user-facing Slack restart-survival behavior, but either migrate the cache to the accepted shared persistent-keyed-store/persistent-dedupe contract once maintainers choose that path, or harden the local Slack implementation with bounded async/locked storage, safe hydration, temp cleanup, observability, and clear regression coverage. The final PR should drop unrelated formatting-only changes and coordinate explicitly with #58731 and #70626.
