## Summary

Keep this PR open. Current main and the latest release still keep Slack thread participation in an in-memory cache, while Slack mention gating uses that cache to continue bot-participated threads without repeated mentions. The PR is still an implementation candidate, but it needs maintainer storage/security direction and consolidation with related persistent-thread work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate it to the approved shared persistent store/dedupe primitive with async Slack call-site updates, or explicitly approve a hardened Slack-local store. Before merge, trim unrelated churn and keep focused tests for restart survival, clear semantics, TTL expiry, corrupt/oversized persisted files, entry bounds, and storage failure behavior.
