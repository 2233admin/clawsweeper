## Summary

Keep this PR open. Current main still keeps Slack thread participation in a process-local dedupe cache, so gateway restarts still lose the non-mention thread-continuation behavior this PR targets. The PR is active and overlaps with unresolved shared persistence work rather than being obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either adapt Slack thread participation to the shared async persistent-dedupe or future persistent-keyed-store API, or explicitly approve a hardened Slack-local store. The final branch should remove unrelated formatting churn outside Slack and keep focused restart-survival regression coverage.
