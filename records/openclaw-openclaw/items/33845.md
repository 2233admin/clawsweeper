## Summary

Keep this PR open. Current main still stores Slack thread participation only in a process-local dedupe cache, so gateway restarts still lose the non-mention thread-continuation behavior this PR targets. The PR remains useful but needs maintainer direction because overlapping shared persistence work is open and the PR still carries storage/security and review-scope questions.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival behavior, then either migrate the cache to the shared persistent dedupe/keyed-store contract once maintainers choose that API, or harden the Slack-local implementation with bounded async/locked storage, safe hydration, temp cleanup, observability, and focused restart regression coverage. The final branch should drop unrelated formatting churn and coordinate explicitly with #58731 and #70626.
