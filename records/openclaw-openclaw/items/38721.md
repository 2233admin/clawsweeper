## Clownfish Work Prompt

Implement a focused fix for gateway shutdowns blocked by active child-process/pipe handles. Work against current main, read the relevant AGENTS.md files and docs list first, and keep the change limited to core process supervision and gateway shutdown/restart wiring.

Observable bug: on stop/restart, current gateway shutdown can hit the force-exit watchdog while active child/PTY processes remain referenced. #61565 only fixed websocket-server close hangs; do not treat it as covering this issue.

Expected fix boundary:
- Add ProcessSupervisor APIs to enumerate active runs with privacy-safe metadata, cancel all active runs or selected active runs, and wait for active supervisor runs to exit with a bounded timeout.
- Wire the gateway stop/restart path to cancel and bounded-drain active supervisor runs before the force-exit watchdog can fire. Preserve existing restart drain behavior for command-queue tasks, embedded runs, and runtime-dependency installs.
- Emit privacy-safe diagnostics/log warnings containing counts plus run id/backend id/pid/scope only. Do not log raw command text, env, cwd, output, or secrets.
- Consider direct runCommandWithTimeout/execFile children only if the source inspection shows they can survive shutdown outside existing timeouts; avoid a broad unrelated process-tracking refactor.
- Keep core extension-agnostic; do not add bundled plugin/channel ids or plugin-specific policy.
- Add focused tests for supervisor active listing/cancel-all/wait behavior and gateway shutdown/restart invoking the new process cleanup. Existing websocket close tests should continue to pass.
- Add a single-line changelog fix entry under Unreleased with `Thanks @carlosrenatoy`; no docs change is required unless a user-visible shutdown diagnostic/config surface is added.
