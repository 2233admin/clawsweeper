## Summary

Keep open. Current main still has bounded gateway shutdown and restart-drain logic, including the adjacent websocket fix from #61565, but it still does not enumerate, cancel, or bounded-drain ProcessSupervisor-managed child/PTY runs during gateway stop or restart. The reported child-process/pipe-handle blocker remains a real core shutdown gap and is narrow enough for a focused fix PR.

## What This Changes

The issue requests that gateway stop/restart handle active child-process and pipe handles from exec or internal CLI runs so shutdown does not time out dirty and leave lane waits behind.

## Best Possible Solution

Keep this open as the canonical gateway shutdown tracker for active ProcessSupervisor child/PTY handles. The best fix is a narrow core contract that exposes privacy-safe active-run enumeration plus bounded cancel/drain operations from `src/process/supervisor/*`, wires that into gateway stop/restart before the force-exit watchdog can fire, preserves the existing command-queue/embedded-run/runtime-deps drain and #61565 websocket hardening, and adds focused supervisor plus gateway shutdown tests.
