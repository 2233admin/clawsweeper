## Summary

Keep this issue open. Current main still reduces the prompt-facing skills catalog to a count/character-bounded alphabetic prefix when compact output exceeds the budget, and the truncation note remains model-facing and count-only. The provided GitHub context shows the omitted-name PR and priority-configuration follow-up are both still open, so the requested visibility and prioritization behavior has not landed.

## What This Changes

The issue asks OpenClaw to prevent eligible user-created skills from disappearing under skills prompt-budget truncation without user-visible warning, documentation, or priority/pinning controls.

## Best Possible Solution

Keep this open until OpenClaw ships #42637 or an equivalent omitted-skill recovery path, documents the skills prompt budget and truncation outcome on the public docs, and decides whether priority/pinning belongs in the skills config contract or remains tracked through #50199.
