## Summary

Keep this issue open. Current main still limits the prompt-facing skills catalog by count and by compact-prefix budget, the truncation note remains prompt-facing and count-only, and there is still no priority, pinning, or context-required skill config surface. Related work is split across the still-open narrow omitted-name PR and the broader priority-configuration issue, so closing this now would lose unresolved product/API work.

## What This Changes

The issue asks OpenClaw to stop silently dropping eligible user-created skills when the skills prompt budget is exceeded, including user-visible warning or recovery behavior plus priority, pinning, required-skill, or context-aware loading controls.

## Best Possible Solution

Keep the issue open until OpenClaw ships an omitted-skill discovery or recovery path, documents truncation and omission behavior on the public skills docs, and explicitly decides whether priority, pinning, or context-required skills belong in the skills config contract or remain tracked in a narrower follow-up.
