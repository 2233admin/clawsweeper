## Summary

Keep open. Current main does not contain the PR’s proposed `approvedExecCommand` change, and the gateway approved-run path still passes `execCommand: enforcedCommand`. The underlying heredoc preservation bug remains plausible, but the PR as written needs revision because it would also route `askFallback === "allowlist"` timeout approvals through raw `params.command`, weakening the enforced allowlist wrapper in a security-sensitive exec approval boundary.

## Best Possible Solution

Keep the PR open and revise it: preserve raw `params.command` only for heredoc runs that received an explicit `allow-once` or `allow-always` decision, keep `enforcedCommand` for timeout or unavailable-UI `askFallback === "allowlist"` approvals, and add gateway-host tests that assert both the explicit heredoc path and the fallback path. This is better than closing because current main has not implemented the intended behavior, but the existing one-line PR should not land as-is.
