## Summary

Keep open. Current main has not absorbed the intended heredoc approval fix, but the proposed one-line change is not safe to merge because timeout fallback approvals share the same follow-up path and would also run raw heredoc command text without an explicit human allow decision.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original `params.command` instead of the rebuilt allowlist `enforcedCommand`, while leaving non-heredoc approvals on the enforced command path.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps `enforcedCommand` for allowlist timeout fallback approvals, and adds focused regression tests for both heredoc branches plus the unchanged non-heredoc path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep enforced command for timeout fallback:** `src/agents/bash-tools.exec-host-gateway.ts:285`
  - body: This switches every heredoc approval follow-up to `params.command`, but the same callback also runs when a request times out and `askFallback === "allowlist"` marks an allowlist match as approved without a human decision. That path should keep the rebuilt `enforcedCommand`; otherwise timeout fallback re-runs raw shell parsing instead of the pinned and quoted allowlist plan.
  - confidence: 0.9
