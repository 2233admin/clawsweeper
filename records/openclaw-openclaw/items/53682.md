## Summary

Keep open. The PR targets a real gateway heredoc approval bug and current main still runs the rebuilt enforced command after approval, but the branch should not merge as-is because it also routes allowlist timeout fallback approvals through raw shell text without an explicit human allow decision.

## What This Changes

The PR changes `src/agents/bash-tools.exec-host-gateway.ts` so heredoc-required gateway approval follow-ups execute the original command text instead of always passing the rebuilt allowlist `enforcedCommand`.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist timeout fallback approvals, and adds targeted tests for both paths plus changelog attribution if maintainers treat this as a user-visible bug fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep enforced command for heredoc timeout fallback:** `src/agents/bash-tools.exec-host-gateway.ts:285`
  - body: This switches every heredoc approval follow-up to `params.command`, but the same callback also runs when a request times out and `askFallback === "allowlist"` marks an allowlist match as approved without a human decision. That path should keep the rebuilt `enforcedCommand`; otherwise timeout fallback re-runs raw shell parsing instead of the pinned/quoted allowlist plan. Use raw command text only for explicit `allow-once` or `allow-always`.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff changes host exec approval security semantics and can bypass enforced allowlist execution on timeout fallback approvals.

Concerns:

  - body: The added `approvedExecCommand` uses raw `params.command` for every heredoc-required approval, including `askFallback: "allowlist"` timeout resolution where no human explicitly approved the run. That drops the enforced command's pinned and quoted argv boundary for a security-sensitive host exec path.
