## Summary

Keep this PR open. Current main still has the Gmail wizard bug the PR targets: `runGmailSetup` writes the Gmail preset but not `hooks.allowedSessionKeyPrefixes`, while the built-in Gmail preset still uses a templated `sessionKey` that `resolveHooksConfig` rejects without prefixes. The PR is not obsolete, but its prefix-only patch is incomplete because current runtime/docs also require `hooks.allowRequestSessionKey=true` for template-derived mapping session keys, and the patch can invalidate existing non-`hook:` default-session configurations.

## Best Possible Solution

Keep this PR open and revise it before merge. The final patch should make `openclaw webhooks gmail setup` emit a config that passes both `resolveHooksConfig` and actual `/hooks/gmail` mapped dispatch: preserve existing prefixes, avoid invalidating existing `defaultSessionKey`, and either explicitly opt into template-derived Gmail session keys with a narrow prefix plus `allowRequestSessionKey=true` or write a static Gmail mapping/session policy when request session keys remain disabled. Add focused regression coverage for plain wizard output, existing-prefix preservation, non-`hook:` default-session configs, static Gmail shadow mappings, and mapped Gmail dispatch behavior.
