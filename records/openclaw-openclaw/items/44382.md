## Summary

Keep this PR open. Current main has shipped the narrower TUI readability fix: a dark/light palette chosen from `COLORFGBG` with `OPENCLAW_THEME=light|dark`. It does not implement this PR's central feature set: five named themes, a theme registry, `--theme`, runtime `/theme`, or `setTheme()`/`getThemeName()`. The PR is still a plausible feature candidate, but it needs maintainer review, a rebase, and a split because the diff also contains unrelated package-resolution/security and generated-protocol changes.

## Best Possible Solution

Keep this PR open as a feature candidate, but require maintainer product review plus a rebase and split: isolate the TUI theme work from package/security/protocol churn, decide whether five named themes and runtime `/theme` are desired core TUI scope, align the design with the shipped light/dark foundation, and add focused tests for startup selection, CLI/env precedence, runtime switching, existing editor/select-list/markdown consumers, and docs. Treat #38982 as a narrower picker candidate only if maintainers choose that path; it should not automatically supersede the five-theme request.
