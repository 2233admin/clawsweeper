## Summary

Keep open. Current main still drops `parsed.offloadedRefs` on the node/iOS `agent.request` path, while this PR is an active, narrow fix for that exact gap; static review found no discrete blocking correctness or security finding, but the PR still needs normal maintainer review and CI/test validation before merge.

## What This Changes

The branch exports the existing `chat.send` media persistence and transcript rewrite helpers, wires them into `server-node-events.ts` for `agent.request`, and adds a node-events regression test for offloaded image refs.

## Best Possible Solution

Land this PR or an equivalent narrow gateway change after normal validation, keeping the shared `chat.send` persistence/rewrite contract rather than adding a separate node-only transcript writer.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: send an `agent.request` with a large image so `parseMessageWithAttachments` returns `offloadedRefs`, then inspect the persisted session transcript on current main and see no `MediaPath`/`MediaPaths` for that user turn.

## Solution Assessment

Yes. Reusing the existing `chat.send` media persistence and transcript rewrite helpers is the narrowest maintainable fix; a separate raw JSONL append or node-only media writer would be more fragile around the SessionManager parent chain.
