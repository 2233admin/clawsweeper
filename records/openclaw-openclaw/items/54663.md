## Summary

Keep open. Current main still hard-codes ordinary group chats to suppress verbose/tool-result text, while preserving media and exec-approval exceptions; the config schema also lacks the requested channel default or Telegram group/topic override. Related #44027 is an open implementation candidate for `/verbose` behavior, but it does not cover the requested typed config policy.

## What This Changes

The issue requests a schema-backed opt-in config option for delivering verbose tool-call summaries/results in group chats while keeping the default off.

## Best Possible Solution

Add a narrow typed group tool-result delivery policy that defaults to false, supports channel defaults plus channel/group/topic overrides, feeds the same resolved policy into normal dispatch and ACP projection, preserves media and exec-approval exceptions, and documents the trusted-group security model.
