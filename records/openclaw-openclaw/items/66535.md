## Summary

Keep open. Current main still has the reported `/compact` cancellation gap: the manual command starts compaction without an `abortSignal` and without registering the compaction in the active-run registry, while the existing cancel paths depend on that registry. The CoClaw report and workaround context are accurate, and open PR #66894 is an unmerged implementation candidate, so this issue should not close before that PR or an equivalent scoped fix lands.

## What This Changes

Allow in-progress manual `/compact` compactions to be canceled through existing cancel paths such as `/stop`, `chat.abort`, and `abortEmbeddedPiRun`.

## Best Possible Solution

Land a scoped fix that registers manual and gateway compactions as compacting active-run handles, passes an `AbortController.signal` into `compactEmbeddedPiSession`, clears the handle in `finally`, and adds regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress compaction.
