## Summary

Keep this PR open. Current main appears to have made the overflow-clipping wrapper portion mostly stale because `.agent-chat__input` no longer has `overflow: hidden`, but it still does not implement the PR’s second user-visible fix: keeping the active slash-menu item scrolled into view during ArrowUp/ArrowDown navigation.

## Best Possible Solution

Keep this PR open and ask the contributor to rebase on current main, narrow the patch to the still-missing slash-menu active-item auto-scroll behavior, and retain wrapper/layout changes only if clipping still reproduces after rebase. The implementation should scope DOM lookup to the current chat instance and add focused UI coverage or explicit manual proof.
