## Summary

Keep this PR open. Current main already avoids the original slash-menu clipping because `.agent-chat__input` no longer has `overflow: hidden`, but it still lacks the keyboard auto-scroll behavior this PR proposes for ArrowUp/ArrowDown navigation in the slash menu. The PR is a focused UI bugfix and is not fully superseded by the related cleanup records.

## Best Possible Solution

Keep the PR open but ask the contributor to rebase on current main and narrow the implementation to the remaining user-visible gap: keep the active slash-menu item in view after ArrowUp/ArrowDown in both command and arg modes. If the wrapper is still needed after rebase, it should be justified with a current reproduction; otherwise a scoped `scrollIntoView({ block: "nearest" })` helper plus focused UI test/manual proof is the best path.
