## Summary

Keep this PR open for maintainer follow-up. Current main still lacks the requested Feishu thread-follow-up mention bypass, so the PR is not implemented, but the latest PR head still has a blocking mention-gate regression and overlaps with #48945's different public API design.

## What This Changes

The PR adds a Feishu `requireMentionInThread` global/per-group config option, passes an `isThreadReply` flag into Feishu reply policy, and adds policy tests for relaxing or tightening @mention requirements in thread replies.

## Best Possible Solution

Land one Feishu-owned thread follow-up policy after maintainer API review: distinguish topic-scoped bot follow-ups from ordinary group quote replies, preserve the existing top-level mention gate, update schema/docs/plugin metadata/generated config together, and cover the bot-level topic, quote-reply, broadcast, and bidirectional override cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421-425`
  - body: The new `isThreadReply` predicate includes `groupSession?.replyInThread`. Current main sets `replyInThread` true for any message with `root_id` or `thread_id`, including ordinary quote replies in group-scoped chats. With `requireMentionInThread: false`, those unmentioned quote replies bypass `requireMention` and dispatch to the agent.
  - confidence: 0.92
- **[P2] Update public Feishu config surfaces:** `extensions/feishu/src/config-schema.ts:146`
  - body: Adding `requireMentionInThread` to the strict Feishu schema makes it a public config key, but the PR does not update Feishu docs, plugin config metadata, or generated bundled channel config metadata. That leaves user-facing config editors and docs out of sync with the accepted schema.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff changes Feishu group mention-gating and currently creates a concrete path for ordinary unmentioned quote replies to reach the agent when the new option is disabled for threads.

Concerns:

- **[medium] Mention-gate bypass for quote replies:** `extensions/feishu/src/bot.ts:421`
  - body: Because the PR treats `groupSession.replyInThread` as a thread override signal, `requireMentionInThread: false` can cause normal group quote replies with `root_id` to bypass the @mention gate and reach the agent. That broadens which group messages can trigger processing under a noise/access-control setting.
  - confidence: 0.9
