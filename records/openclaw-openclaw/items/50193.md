## Summary

Keep open. Current main has not implemented the PR’s central behavior: the Control UI chat model picker still builds and renders a flat option list, not provider-grouped model optgroups. The PR remains a plausible UI enhancement, but it should be updated to preserve the existing disambiguated model labels and avoid hard-coded provider-label drift before maintainers consider landing it.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to add a grouped chat-model select state, render provider `<optgroup>` sections in `ui/src/ui/chat/session-controls.ts`, sort groups by displayed provider label, and preserve the existing disambiguated option-label behavior covered by `ui/src/ui/chat-model-select-state.test.ts`. Add focused tests for provider grouping, duplicate model names across providers, duplicate names within one provider, unknown provider fallback, and session patch behavior; no core/API or security-sensitive changes appear necessary.
