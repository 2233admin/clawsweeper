## Summary

Keep open. Current main does not implement this PR's central behavior: the Control UI chat model picker still resolves and renders a flat option list. The PR remains plausible UI work, but it needs rework against the current `chat-model-select-state.ts` / `chat/session-controls.ts` split and should preserve the existing disambiguated model labels instead of replacing them with raw `entry.name` labels or a drifting provider-label table.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Rebase the work onto the current chat model picker modules, add grouped model select state in `ui/src/ui/chat-model-select-state.ts`, render provider `<optgroup>` sections in `ui/src/ui/chat/session-controls.ts`, sort groups by displayed provider label, and reuse the catalog display helpers so duplicate model names stay distinguishable. Add focused tests for provider grouping, duplicate names across providers, duplicate names within one provider, unknown provider fallback, selected-label clarity, and unchanged `sessions.patch` behavior.
