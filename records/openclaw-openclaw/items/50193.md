## Summary

Current main still exposes the Control UI chat model picker as a flat option list, so this PR’s provider-grouped model selector is not implemented on main. The PR remains plausible, but it is stale against the current split between model-select state and session-control rendering and needs rework before it can land.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. Rebase the work onto `ui/src/ui/chat-model-select-state.ts` and `ui/src/ui/chat/session-controls.ts`, represent provider groups in the model-select state, render model `<optgroup>` sections, sort by displayed provider label, preserve current duplicate-name disambiguation, and add focused tests for provider grouping, unknown-provider fallback, duplicate labels, selected-label clarity, and unchanged `sessions.patch` behavior.
