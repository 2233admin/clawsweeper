## Summary

Keep this PR open, but do not merge it as written. The intended ACPX guard is still useful on current main, but the new version check only handles full x.y.z specs and lets explicit old partial/range pins such as @0.11 or <0.12.0 pass through to session/new, preserving the failure mode the PR is meant to prevent.

## What This Changes

The PR adds a Codex ACP gpt-5.5 compatibility guard for explicit pre-0.12 adapter commands, focused ACPX runtime tests, ACP docs updates, and a changelog entry.

## Best Possible Solution

Keep the wrapper-boundary guard, but evaluate explicit npm version/range specs conservatively so any command that can resolve below @zed-industries/codex-acp@0.12.0 fails before session/new, while unpinned commands, local wrappers, bundled 0.12.0, and explicit 0.12.0+ specs continue to pass.

## Reproduction Assessment

Yes for the review finding. An explicit command such as npx @zed-industries/codex-acp@0.11 produces a version spec without a patch segment, so the added parseSemver returns undefined and supportsCodexAcpGpt55 treats a pre-0.12 range as supported.

## Solution Assessment

No as written. The boundary and error shape are maintainable, but the version check should parse npm semver ranges or conservatively reject explicit specs that can resolve below 0.12.0 instead of assuming unparsable explicit specs are safe.
