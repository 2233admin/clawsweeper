## Summary

Keep open. Current main does not implement the requested alias coverage: the read tool-start warning still only checks `path` and `file_path`, so `file` and `filePath` still take the warning path. The surrounding runtime contract now appears to require canonical `path` only, which makes this a maintainer alignment issue rather than a cleanup-close candidate.

## Best Possible Solution

Keep this open until maintainers choose one coherent contract: either support `path`, `file_path`, `file`, and `filePath` end to end in the read tool schema, validation, execution, diagnostics, and regression tests, or explicitly enforce/document canonical `path` only and close the alias request with that product rationale.
