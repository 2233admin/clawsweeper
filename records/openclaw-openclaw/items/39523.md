## Summary

Keep open. Current main still does not implement this PR's requested Feishu channel/account `mediaLocalRoots` config fallback; it only supports runtime-scoped roots. The PR head is directionally useful but needs rework because `"any"` must honor the current host-read/readFile safety contract and config precedence must account for core-injected roots.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a Feishu-owned patch that adds `mediaLocalRoots` to the Feishu top-level and account schemas, updates generated config metadata and public docs, defines precedence between configured roots and runtime-scoped mediaAccess roots, routes any supported `"any"` mode through the current host-read/readFile safety contract, and adds focused regression tests for array fallback, account fallback, `"any"`, and ordinary runtime-scoped local media sends.

## Risks / Open Questions

- No regression tests currently cover Feishu top-level config fallback, account config fallback, `"any"`, or precedence against runtime-scoped roots.
