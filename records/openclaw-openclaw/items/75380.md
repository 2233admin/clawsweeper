## Summary

Keep open. Current main still has append-only queued JSONL diagnostic sinks for cache trace and Anthropic/provider payload logging, while the documented `logging.maxFileBytes` rotation only applies to the primary Gateway file logger. The prior ClawSweeper comment and the linked related issues correctly separate this from gateway.err.log, journald, transcript-maintenance, and primary tslog rotation work.

## What This Changes

Request bounded max-size and archive rotation controls for cache-trace and provider-payload diagnostic JSONL files under the OpenClaw logs directory.

## Best Possible Solution

Add a reusable bounded-rotation policy for queued diagnostic JSONL writers, expose documented size/archive controls for cache trace and the current payload logger, and keep gateway.err.log, journald, and session transcript growth in their separate tracked lanes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enable cache trace or the Anthropic payload logger, generate repeated model turns, and the active JSONL file appends indefinitely because neither sink passes max-size/archive options and no config field exists for them. I did not run live traffic because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to extend or reuse the existing rotation semantics for these queued diagnostic sinks, with tests and docs for the new config/env knobs; manual truncation or cron cleanup would leave the source-level unbounded writer behavior in place.
