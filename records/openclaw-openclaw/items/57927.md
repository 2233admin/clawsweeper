## Summary

Keep open. Current main has not implemented the core orphaned `toolStartData` cleanup, and one utility cleanup is still present. The PR is still relevant, but its proposed sweep needs revision because the provided review comments identify a real long-running in-flight tool correctness bug that current source confirms would affect `after_tool_call`, messaging-send tracking, and cron-add tracking.

## Best Possible Solution

Keep the PR open, but require a refresh: split or rebase the harmless utility dedupe, drop/update the stale doctor hunk, and replace the idle-timestamp sweep with an in-flight-aware cleanup that only removes truly abandoned entries. Add focused tests proving normal `after_tool_call` duration/params, `cron.add` tracking, abandoned-entry cleanup, and no eviction for a >5 minute active tool call.
